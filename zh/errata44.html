  • 006: 可靠性修正: 2008年11月19日   所有架构
    由于对选项处理的改变，导致某些 DHCP 客户机(例如 Solaris/OpenSolaris 和某些嵌入式路由器)不认可 DHCP OFFER 响应。
    已经发布源代码补丁修正此问题。

  • 005: 可靠性修正: 2008年11月7日   所有架构
    在物理内存耗尽时，一个软件问题会造成分配内存时，指针越界，导致内核崩溃。
    已经发布源代码补丁修正此问题。

  • 004: 可靠性修正: 2008年11月6日   所有架构
    Fix httpd(8)'s mod_proxy 模块在 64 位架构上出错。由于此问题的存在，它导致在 HTTP 会话中使用代理规则时子进程崩溃。
    已经发布源代码补丁修正此问题。

  • 003: 可靠性修正: 2008年11月6日   所有架构
    修正 IPv4 TCP/IP 堆栈中 TIME_WAIT 状态的 socket 重复利用。由于此问题的存在，它导致在 2 个 IP 之间的 TCP 连接中，处于 TIME_WAIT 状态的 socket 不是接受连接，而是重置。
    已经发布源代码补丁修正此问题。

  • 002: 可靠性修正: 2008年11月2日   所有架构
    由于 vr(4) 驱动的问题，它可能导致当 RX 路径耗尽 mbuf 后，使用 vr(4) 驱动的系统在重负荷下崩溃。
    已经发布源代码补丁修正此问题。

  • 001: 安全性修正: 2008年11月2日   所有架构
    邻机发现协议(ndp)没有正确校验邻机的请求，可能导致邻近的攻击者拦截通信。 如果要使用此弱点攻击，攻击者必须与目标都使用 IPv6 连接到同一个路由器上。 CVE-2008-2476.
    已经发布源代码补丁修正此问题。
