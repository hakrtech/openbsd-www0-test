FTP

OpenBSD 的 FTP 镜像由 一个主站点，以及其二级镜像站点组成。二级镜像站点由 OpenBSD 开发社团从具有良好的 网络连接性，及其维护人员所知的站点中选取。如果你希望维护一个 FTP 镜像，请镜像距 你最近的二级镜像站点 - 强烈不赞成镜像主站点。

镜像结构:

每个服务器的文件结构大致如下:

  • /pub/OpenBSD/
    HTTP 和 FTP 镜像的绝对路径，存放 OpenBSD FTP 文件集合，它 必须 存在。如果需要，此位置可以是链接。

  • /pub/OpenBSD/4.3/
    最新发行版本的位置。
    • alpha/
    • amd64/
    • armish/
    • hp300/
    • hppa/
    • i386/
    • landisk/
    • mac68k/
    • macppc/
    • mvme68k/
    • mvme88k/
    • packages/
      可以安装到你的 -stable 系统中的二进制包(预编译的 ports)。
    • sparc/
    • sparc64/
    • tools/
      其它操作系统的工具(Windows，Linux 和类似系统)，用来创建引导软盘，修改分区的 大小等等。
    • vax/
    • zaurus/
    • src.tar.gz
       用户空间源代码，你可能需要将它放到 /usr/src。(它不包含内核源代码)
    • sys.tar.gz
       内核源代码，你可能需要将它放到 /usr/src/sys。
    • ports.tar.gz
       ports 源代码，你可能需要将它放到 /usr/ports。
    • xenocara.tar.gz
       X Window 系统 (X11) 源代码。

  • /pub/OpenBSD/snapshots/
    对于我们的主要架构，我们趋向大约每月构建一次稳定性和质量未知的最小版本。 这个位置用来存放测试版本。

  • /pub/OpenBSD/src/
    为需要读取个别文件的人准备，是每天更新的 CVS 检出副本。通常我们建议使用 CTM 或 AnonCVS 来代替镜像此位置。因此，所有 FTP 镜像都不镜像它。

  • /pub/OpenBSD/doc/
    OpenBSD FAQ 的 PDF 格式和文本格式的文件， 以及其它 OpenBSD 相关的文档。

  • /pub/OpenBSD/tools/
    各种小工具。
  • /pub/OpenBSD/4.2/
    结构与上述的 4.3 发行版本类似.. 怀旧而已。
  • /pub/OpenBSD/4.1/
    结构与上述的 4.3 发行版本类似.. 怀旧而已。
  • /pub/OpenBSD/4.0/
    结构与上述的 4.3 发行版本类似.. 怀旧而已。

配置新的 FTP/ANONCVS 镜像的人可能对 ftp://ftp.openbsd.org/pub/OpenBSD/ 的磁盘空间估算感兴趣。
当前的策略是，如果被列在本页，必须至少提供最新的两个发行版本的二进制格式(当前是 4.2 和 4.3)，以及 OpenSSH/，OpenNTPD/，OpenBGPD/，patches/ 和 tools/ 目录。 说明，“发行版本”意味着在指定目录(例如 4.3/)中的所有文件。它包含所有架 构子目录，和完整的 packages 子目录(它包含每个架构的所有应用程序)。
如果磁盘空间的较大，镜像可以提供更多的内容，例如快照，更旧的版本，当前的源 代码等等。
当前主 FTP 站点的磁盘空间使用情况如下: 


                      当前

pub/OpenBSD/3.7       17.8GB   (OpenBSD 3.7 发行版本)
pub/OpenBSD/3.8       20.0GB   (OpenBSD 3.8 发行版本)
pub/OpenBSD/3.9       22.4GB   (OpenBSD 3.9 发行版本)
pub/OpenBSD/4.0       25.6GB   (OpenBSD 4.0 发行版本)
pub/OpenBSD/4.1       31.7GB   (OpenBSD 4.1 发行版本)
pub/OpenBSD/4.2       35.2GB   (OpenBSD 4.2 发行版本)
pub/OpenBSD/4.3       47.7GB   (OpenBSD 4.3 发行版本)
pub/OpenBSD/snapshots 50.0GB   (当前的快照，尺寸通常 >= 最后的发行版本)
pub/OpenBSD/src         .5GB   (可以通过 FTP 下载的源代码，每天从匿名 CVS 服务
                器更新。你应该避免使用 FTP 镜像获得。更好的方法是你自己从最近的
                匿名 CVS 服务器获得)
因此，包含快照和最新的两个发行版本，镜像服务器至少需要 110GB 的磁盘空间。 如果你有更多的磁盘空间，你可以额外的放置更多的旧版本。 说明，主 FTP站点还提供了下述目录，镜像站点应该不需要提供: 

pub/OpenBSD/distfiles  4.7GB   (ports 系统的 makefile 文件使用的 distfiles。
                你大概不需要镜像这些，除非你自己需要。ports 的 makefiles
                默认只从 ftp.openbsd.org 和 ftp.usa.openbsd.org 下载这些文件。)

如果你增加了新的 FTP，HTTP，AFS 或 RSYNC 镜像站点，请与 www@openbsd.org 联系。
