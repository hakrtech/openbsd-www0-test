OpenBSD/sparc

OpenBSD/sparc draait op de meeste 32bits Sun SPARC werkstations, waaronder de sun4, sun4c en de sun4m architectuur families (behalve de 64bits UltraSPARC computers, deze worden ondersteund door OpenBSD/sparc64).

Er is een mailinglijst beschikbaar voor de OpenBSD/sparc en OpenBSD/sparc64 ports op sparc@openbsd.org. Om in te schrijven op de OpenBSD/sparc mailinglijst, stuur dan een bericht met als inhoud "subscribe sparc" naar majordomo@openbsd.org. Lees ons mailinglijst beleid voordat u uwzelf inschrijft.

De huidige onderhouder van deze port is Art Grabowski (art@openbsd.org), met hulp van voornamelijk Jason Wright, Theo de Raadt en Miod Vallat. Anderen zijn zeker welkom om mee te helpen!

Geschiedenis:

De originele 4.4BSD port werd gedaan door Chris Torek in opdracht van LBL. De code werd vrijgegeven door Chris halverwege 1993 en Theo de Raadt heeft het klaar gemaakt voor de NetBSD tree. Theo en Markus Wild werkten aan het verbeteren van Chris' orginele SunOS compatibiliteits code tot een zeer complete en betrouwbare emulatie. Chuck Cranor heeft vervolgens de sun4c code geport naar de sun4 architectuur met een 8KB pagina-grootte, waarop Theo deze code zo samensmolt dat dezelfde kernel en programma's op de sun4c en sun4m machines konden draaien (dit in tegenstelling met Sun's gescheiden kernel omgevingen). Deze laatste verandering zorgde er tevens voor dat alle apparaat configuratie code herschreven moest worden. Vlak na de vrijgave van NetBSD/sparc 1.0 schreef Peter Galbavy een ESP SCSI stuurprogramma ter vervanging van Chris' sparc-specifieke SCSI code.

Op dat moment ontstond er een conflict tussen Theo en de andere mensen waarmee hij het NetBSD begon, en Theo werd door de core groep gedwongen om z'n werk voor NetBSD neer te leggen. Nadat Theo NetBSD had verlaten hebben een aantal mensen uit de NetBSD groep verschillende dingen gedaan voor de port: Paul Kranenburg heeft een stuurprogramma voor een floppy drive geport en begon aan de ondersteuning van de 4/400. Chuck heeft vele uren gewerkt aan de ie, xy en xd stuurprogramma's.

Theo ging onafhankelijk verder een zorgde voor een aantal toevoegingen: P4 ondersteuning, flexibele boot strategie, een aantal grafische stuurprogramma's met hulp van John Stone, en een groot aantal bug fixes.

Theo begon met het porten van OpenBSD/sparc naar het sun4m platform, maar door een peesontsteking in zijn polsen en de problemen met de NetBSD core moest hij hiermee stoppen. Het eerste werk aan sun4m werd doorgegeven aan Aaron Brown van Harvard, die werd betaald door Margo Seltzer's onderzoeks fonds. Dit eerste werk bestond voornamelijk uit een aantal pogingen tot het bouwen van een pmap structuur dat alle drie de MMU typen efficiënt zou ondersteunen. Echter, David Miller van het UltraLinux project raakte beetje betrokken.

Theo en Jason Downs voegden de NetBSD sparc code samen met OpenBSD, waardoor OpenBSD/sparc weer bruikbaar werd. Jason Wrigt schreef een aantal nieuwe SBUS Ethernet stuurprogramma's met behulp van informatie van het UltraLinux project, en voegde wat NetBSD code toe om de stabiliteit van deze architectuur te verbeteren. Sindsdien heeft Art Grabowski verschillende stabiliteits en betrouwbaarheids problemen opgelost en een niet-uitvoerbare stack en heap aan de sun4m machines toegevoegd. De frame buffer en console code werd volledig herwerkt voor OpenBSD 3.2, waardoor de weergave snelheid werd verhoogd voor enkele frame buffers en waardoor er meer kaarten ondersteund werden.

En het werk gaat verder...

Huidige status:

Alle hardware die wordt genoemd in de ondersteunde hardware sectie kan multi-user booten, en ondersteund genoeg (al dan niet alle) ingebouwde hardware om over het algemeen bruikbaar te zijn. Een toenemend aantal SBus kaarten van derden wordt ook ondersteund.

Het X Window Systeem werkt op alle ondersteunde frame buffer, waaronder opstellingen met meerdere monitoren. Op het moment wordt er een niet-versnelde server aangeboden, hier wordt echter aan gewerkt.

Een belangrijke opmerking over OpenBSD/sparc is dat het is ontworpen zodat een enkele kernel op ALLE ONDERSTEUNDE SPARC machines werkt. SunOS en Solaris hebben altijd een gescheiden `kernel architectuur', bijv. sun4, sun4c en sun4m, een `GENERIC' OpenBSD kernel werkt echter op alle ondersteunde modellen.

Van de meeste problemen met OpenBSD/sparc wordt aangenomen dat dit komt door het grote aantal verschillende SPARC processoren en cache implementaties, samen met hun niet-gedocumenteerde bugs, en dus niet door problemen met de kernel. Feedback over welke modellen werken en welke niet wordt gewaardeerd, helemaal bij nieuwere modellen en upgrades.

Projecten (in willekeurige volgorde):

  • Het schrijven van stuurprogramma's voor meer niet-ondersteunde apparaten (audio, frame buffer...)
  • Een versnelde X server leveren voor de cgsix frame buffer
  • Samensmelten van nuttig NetBSD werk.

Niet ondersteunde machines

  • sun4: 4/400 (ondersteuning voor de I/O cache ontbreekt, en heeft ethernet problemen)
  • sun4d machines:
    • SPARC Server 1000
    • SPARC Center 2000
    Deze machines maken voor hun CPU's gebruik van XD-Bus in plaats van M-BUS, welke nog niet ondersteund wordt.
  • JavaStation-NC (ondersteuning voor de PCI bus ontbreekt)
  • sun4u: UltraSPARC 64-bit machines. De meeste van deze machines worden ondersteund door de OpenBSD/sparc64 port.
  • Solbourne machines: deze machines zijn behoorlijk anders, en sommige hiervan worden geplaagd door processor bugs. Sommige hiervan worden ondersteund door de OpenBSD/solbourne port.
  • Tadpole SPARCbook 1, LC en 2: deze machines bevatten behoorlijk apparte onderdelen en ze beschikken niet over een Sun-compatible PROM.

Verkrijgen en installeren van OpenBSD/sparc:

De laatste ondersteunde OpenBSD/sparc uitgave is OpenBSD 3.8. Hier zijn de OpenBSD/sparc 3.8 installatie instructies .

Van tijd tot tijd worden er snapshots beschikbaar gemaakt op deze locatie en op enkele mirrors. Hier zijn ook de OpenBSD/sparc snapshot installatie instructies .

Er worden verschillende installatie media geleverd, zodat OpenBSD/sparc geïnstalleerd of geüpgrade kan worden via CD, floppy boot images voor sun4c en sun4m, miniroot images voor machines zonder floppy drives (en sun4 machines), als ook netwerk en schijfloze installaties.

  • CD boot

    Booten van een CD geeft u een klein ffs bestandssysteem met een GENERIC kernel met stuurprogramma's voor alle ondersteunde apparaten welke op SPARC machines worden gevonden.
    Voor de nieuwste lijst met stuurprogramma's welke op deze image staan kunt een kijken naar het GENERIC kernel configuratie bestand. Er kan vanaf de CD worden geboot met de volgende OpenPROM opdracht 

      boot cdrom 3.8/sparc/bsd.rd
  • Floppy (floppy38.fs)

    Booten van een CD geeft u een klein ffs bestandssysteem met een kernel met stuurprogramma's voor alle ondersteunde apparaten welke op SPARC machines worden gevonden.
    Voor de nieuwste lijst met stuurprogramma's welke op deze image staan kunt een kijken naar het RAMDISK kernel configuratie bestand. Van het floppy kan met het volgende OpenPROM commando worden geboot 

      boot floppy
  • Miniroot (miniroot38.fs)

    De miniroot levert dezelfde installatie omgeving als de CD waar vanaf geboot kan worden, en het is bedoelt voor een gemakkelijke bootstrap als er al een besturingsstysteem op de machine geïnstalleerd is. Nadat de miniroot met dd op de primaire swap partitie is geplaatst, kan er vanaf deze miniroot worden geboot met een OpenPROM commando als 

      boot disk:b
    (de schijfbenaming kan verschillen, dit hangt af van de gekozen swap partitie).

  • Netboot (boot.net, bsd.rd)

    Wanneer u gebruik maakt van een opstelling zonder harde schijf, dan is het mogelijk om de OpenBSD/sparc boot loader, boot.net, via het netwerkt middels tftp, te booten over het netwerk, om vervolgens de bsd.rd miniroot via NFS te bereiken. Lees verder de diskless(8) manual pagina voor details over het opzetten van een netwerk boot omgeving.
