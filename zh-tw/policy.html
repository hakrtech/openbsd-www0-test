版權採用方針

  • 目標

    著作權(版權)法是複雜的法規, 但 OpenBSD 有個簡單的方針 - OpenBSD 盡全力保有 原本 Berkeley Unix 的版權宣告.

    OpenBSD 能夠有今日的成長, 是因為 Berkeley 的 Computer Systems Research Group 所立下的榜樣, 由於他們與其他人的努力, 才創造出並保有較無法律問題 的源碼散布模式.

    可以 自由地再散布 的權力. "Berkeley" Unix 的進展, 已經達 到可與許多其他作業系統發展團隊交換源碼的程度了, 不管是內部或是跨團隊 的交換皆然. 瞭解有關著作權法的相關考量後, 才能交換並且再散布源碼, 並且仍保有原版 權宣告的精神, 其握有散布程式碼權力的觀念, 也是促使參與其中的人們協力合作的 基石.

  • Berkeley 版權宣告

    Berkeley 版權宣告對於軟體的私人使用或商業使用並無任何限制, 並且僅要求一個簡單 且一致的條件: 對於任何再散布的版本中, 必須保有原本的版權宣告, 及註明原作者的 姓名列表, 純粹只有宣傳的意涵.

    Berkeley 已經在 1999 年七月 22 日取消了第三點版權宣告(廣告宣傳宣告). OpenBSD 源碼目錄的冗長版權宣告中, 也已經移除第三點版權宣告. 此外, 許多其他的 BSD 模式版權宣告, 僅僅包括前兩項版權宣告.

    因為 OpenBSD 的版權宣告僅止於 Berkeley 版權宣告, OpenBSD 希望可以達到與 Berkeley 同樣廣泛的散布與應用. 也因為這點, OpenBSD 不能包含有比 Berkeley 版權宣告更嚴苛的程式碼, 也不能包含更改該著作為另一種版權狀態的版權宣告, 也就 是指 OpenBSD 整體是可以自由免費的再散布的, 但是其中部份元件可能無法單獨自由 免費的再散布.

  • 著作權法

    雖然完整的著作權法已經超過本文的範圍, 一些基本的道理仍是相通的. 依據現行的 法律, 除非是有另行宣告, 否則著作權皆是隨著創作物被產生之時就隨原創者擁有的, 一般來說, 版權僅可及於基本創作本身, 而不能延伸到基於基本創作所衍生的創作物, 也不包含衍生創作物中的基本創作物.

    著作權法包函以下三種的創作物:

    原創創作物
    非基於現有創作物衍生的原創創作物
    衍生創作物
    該創作物是衍生於, 包含, 或修改現有創作物
    編輯創作物
    該創作物是編輯現有或衍生創作物而成

    著作權法的基本精神就是衍生創作物的著作權, 並不影響原基本創作作者的著作權, 只是多增加了衍生創作者的著作權而已. 同樣的, 編輯著作權並不影響其所包含創 作物的著作權, 只有新增編輯著作的權力而已.

    大家要瞭解到著作權法不管是在國內或國際上, 都受到廣泛的保障. 通常放在程式 碼中的"版權宣告"並非指著作權, 而是讓撰寫宣告的人聲明他們擁有這件作品或部 份作品的版權. 通常這些聲明伴隨著一些著作權法所註明的使用許可, 以及擁有此 著作權的所有人/散布人與這項作品的可能使用方式.

  • 使用許可 - 相對於版權的另一面

    由於著作權是來自於作品的創作, 而非經由註冊程續而來, 我們需要一個實際的 方式來擴展作品的使用許可, 這也可能包含著作權法中"正當使用"所允許的使用情況.

    這項許可通常以發行版本或使用許可的方式附於作品中, 其中說明了著作權法規範 以外的允許使用情況. 其中一項極端的例子是 public domain, 意即創作者聲明 他對於該作品的使用方式並不設定限制, 限制較多的聲明可能只限於著作權法中允許 的使用方式, 或是較偏狹的聲明, 讓作品難以使用的各種限制條件.

    同樣的, 發行版本或使用許可的方式只限於該聲明者本身的創作部份, 基於前述創作而 生的衍生創作, 則無法對原創作物的許可權做出或增或減的權力聲明.

    由於著作權是來自於作品的創作, 而非經由註冊程續而來, 移除或修改版權宣告 或是隨作品發行的聲明並不會對著作權有任何影響, 只會讓人對修改者基於該作 品所衍生的新作品合法性產生懷疑而已. 相同的, 增加聲明與使用條件時, 假如與原有聲明與使用條件相衝突, 也無法取代 原有聲明與使用條件, 做出新增聲明與使用條件者的行為, 只會讓人質疑她的修改 作品或衍生作品是否真的可以拿來合法地使用.

    最後一點, 發行版本通常伴隨著該創作物推出. 這意指如果原創作人發行作品時 也附上某些使用許可條件, 則這些使用許可是公平的適用於所有合法取得該作品 的所有人. 也就是說已經發出的使用許可, 原創作人並無法溯及既往的收回他人 的使用權力. 相同的, 假如該原創者決定將作品商品化, 雖然他可以對基於該作 品的衍生創作做出更嚴苛的使用條件, 但他並無法收回或修改已賣出的原創作使 用權.

  • 個別案例

    這個部份是 OpenBSD 對於一些常見版權宣告的使用狀況摘要.

    Berkeley

    Berkeley 版權宣告就是 OpenBSD 所採用的版權宣告模式. 這可以保留原版權擁有者 的所有權利, 也對採用此版權宣告的作品的使用條件, 做出最小的限制. 使用 Berkeley 版權宣告的作品, 或者是近似於 Berkeley 模式的版權宣告, 一般來說都可以放到 OpenBSD 源碼目錄中.

    AT&T

    由於部份的版權讓與 AT&T, 在 4.4BSD lite 與 lite2 版本中的部份檔案, Berkeley 的版權宣告都會包括一份 AT&T 的版權宣告. 這個版權宣告的聲明與標準的 Berkeley 授權是相同的.

    除此之外, OpenBSD 包含一些其他的 AT&T 程式碼, 這些程式碼有較寬鬆的授權 條件, 比如 awk 的公版程式碼.

    Caldera

    Caldera (就是 SCO 集團) 目前擁有 Unix 源碼的版權. 西元二零零二年元月二十三日, Caldera 釋出了原始 Unix 源碼(包括到第七版, 以及 32V)的版權限制. 目前這 份程式碼是採用4-term BSD-style license的版權宣告. 因此, 目前我們已經可以將 Unix 程式碼併入 OpenBSD 之中了(雖然這份程式碼年代久遠, 而且有許多地方需要修改才能使用)

    DEC, Sun, 及其他的軟體公司

    一般來講, OpenBSD 並不會採用軟體公司所撰寫的版權軟體. 只有當該軟體是 著作權人同意無條件使用, 並附上類似 Berkeley 版權宣告時, 或是該軟體屬於 員工所有, 而且其版權宣告已宣告放棄對該軟體的限制下, 才會被納入考量.

    Carnegie-Mellon (CMU, Mach)

    Carnegie-Mellon 版權宣告類似 Berkeley 版權宣告, 差別在於 Carnegie-Mellon 版權宣告要求衍生創作物需要供 Carnegie-Mellon 使用. 由於這項聲明只是一項 請求(request)而非使用條件, 這個版權宣告的軟體仍可以納入 OpenBSD 當中. 需要 注意的是, 現存的 Mach 程式碼, 仍然受限於 AT&T 版權宣告, 這也是 Mach 源碼普遍散布的原因.

    Apache

    Apache 版權宣告類似於 Berkeley 版權宣告, 其差別在於規定所有的衍生創作物, 不可以使用 Apache 字樣在產品中. 這項宣告的目的在於, 避免有人修改了 Apache 程式碼後, 使用 Apache 字樣於新創作產品中, 讓他人誤以為是 Apache 的正式發行 版. 但這點考量對 OpenBSD 並不是問題, 因為 OpenBSD 是一個 編輯物品, 而非衍生創作物.

    ISC

    ISC 版權基本上等同於兩項聲明的 BSD 版權宣告, 因應 Berne 著作權條例, 移除了非必要的語言限制. 這是 OpenBSD 較適用的新增源碼來源. 你可以在 源碼目錄的 /usr/src/share/misc/license.template 檔案中找 到一份該版權宣告的範本.

    GNU General Public License, GPL, LGPL, copyleft, 等相關版權宣告.

    GNU Public License 跟以其為基礎的版權宣告, 均對於源碼的釋出有強制的限制, 以便於源碼, 及 GNU 版權衍生創作品的再散布.

    基於軟體分享的精神, 這是種相當高尚的版權宣告, 但通常也不可能被商業使用軟體 所能接受. 因此, GPL 版權宣告的軟體, 不能被包含進 OpenBSD kernel 或是 runtime 中, 然而, 我們可以用 GPL 版權宣告的軟體當作發展工具, 或是當作系統中 optional 的元件, 只要使用的情形不會讓其他的 OpenBSD 源碼也受到 GPL 版權限制, 都 可以使用 GPL 版權宣告的程式.

    舉例來說, 有一些 ports 軟體包括 GNU 軟體浮點運算程式, 這是可以選擇的, 而且 系統不一定需要用軟體浮點運算程式就可建構完成, 或者你也可以使用其他版權版本 的軟體浮點運算程式來完成. 另一個例子是在 OpenBSD 工具程式中, 使用了 GCC 和其他的 GNU 工具, 要散布一個沒有包括工具程式的 OpenBSD 系統是可行的, 或者散布者可以使用另一組可選擇的, 相容於 GPL 版權宣告的工具程式,

    NetBSD

    大部份的 OpenBSD 程式碼都是基於或發展於 NetBSD 程式碼的, 這是由於一些 OpenBSD 發展者也同時參與了 NetBSD 計畫. 一般的 NetBSD 版權宣告是相容於 Berkeley 版權 的, 也允許相同程度的使用限度. 但這也僅限於一般的 NetBSD 版權才可以放入 OpenBSD 系統中.

    在以前, NetBSD 包含了一些有個人版權宣告的程式, 這些版權比一般的 NetBSD 版權 宣告多了些限制, 但允許由 NetBSD 計畫基金會散布該程式. 這種程式碼無法加入 OpenBSD 當中, 畢竟他們的宣告已經有所限制, 無法符合 OpenBSD 版權宣告, 有些 版權宣告甚至對 OpenBSD 的使用上, 採用偏頗的標準.

    FreeBSD

    大部份的 FreeBSD 程式也是基於 Berkeley 版權宣告, 或者是採用 Berkeley 模式的 其他版權宣告. 這些程式可以放入 OpenBSD 中, 但是使用 GPL 或者個別版權宣告的 部份, 如果不符合 OpenBSD 版權要求, 就無法放入 OpenBSD 系統.

    Linux

    大部份的 Linux 程式碼都是採用 GPL 形式的版權宣告, 因此也無法放入 OpenBSD 之中. 個別的元件如果符合 OpenBSD 版權要求, 才有可能被採用. 要注意的是, Linux 的發行版本的組織, 也可能做出額外的版權宣告, 其中包括非 Linux 核心 的編輯成品及原作品.

    X, XFree86

    X 及 XFree86 並非 OpenBSD 的一部份, 但是 X 或 XFree86 常常伴隨著 OpenBSD port 軟體集散布給使用者, 這種散布的方式也符合其版權宣告.

    Shareware, Charityware, Freeware, 等相關版權宣告.

    大部份的共享軟體版權宣告, 對於再散布, 使用, 可見度都有限制條件, 而這點並 不符合 OpenBSD 計畫目標中的版權條件. 這些軟體的使用, 都需要依照個案, 逐字逐句的檢視其版權宣告, 比如是否是採用"請求"或是"強制要求"等字眼, 或者其軟體版權精神, 是否與 OpenBSD 計畫目標相符合等檢視.

    Public Domain

    當軟體真正被納入公共領域版權宣告, 則可以納入 OpenBSD 當中, 但需要依照個案, 逐個檢視後才可納入. 通常公共領域版權宣告, 是由無法依照著作權法, 去保護其 其著作權利, 或無法履行發行使用許可的個人所做出的聲明. 當一件作品真的被宣 告為公共領域的版權宣告後, 等同於作者放棄對該作品所有相關權利, 在該作品 的散布使用上, 不會有任何限制.
