通信論壇

在 OpenBSD 中，通信論壇是一個讓使用者與開發者溝通的重要方法。除了announce之外， 其他的通信論壇都不需經過仲裁者同意即可散佈的。我們慎重的限制通信論壇的數量。這對減少跨 區發表的數量有幫助並且確保資訊可以充分地傳達到訂閱者的手上。

使用習慣規定

請審慎地考慮到其他的訂閱者

純文字, 每行 72 字元
許多訂閱者用文字模式的收信程式(mail(1), emacs)讀信，通常 HTML 的格式 或是一行超過 72 字元的信是無法閱讀的。 大部分的 OpenBSD 通信論壇在送信之前會將 MIME 內容過濾掉。 如果您不是用純文字的話，您的訊息會試著被重新格式化，抑或如果沒辦法被重新 格式化，則會立刻被拒絕掉。 唯一接受夾檔的通信論壇是ports，這在其他通信論壇是會被移除的。
在您發表之前，您做好功課了嗎
如果您有安裝問題，請確認您已經讀過相關的文件 例如 INSTALL.* 文字檔在 FTP 安裝目錄下， 常見問題及相關的使用手冊(從 afterboot(8)開始). 確認您將相關的細節 (版本, 硬體, dmesg(8)) 附上，那將有助於疑難排解。 我們希望幫助您，但是我們不會剝奪您有價值的學習經驗。
採用有幫助的標題列
採用空標題的訊息會被通信論壇管理者拒絕， 所以他們會過更久時間才會露面. 採用一個恰當的標題才會確保有更多人真正會去看您寫的內容。 另外，避免在標題列使用過度的大寫。
縮減您的簽名檔
控制郵件最後的簽名列維持在合理的長度. 文章很少緊要到要用 PGP 簽章，那些自動附加的通訊錄只是徒增惱人而已。
適切主題
請發表對 OpenBSD 使用者恰當的主題

垃圾信件

垃圾信永遠都是被禁止的，但是有時還是會被錯過。 如果您從 OpenBSD 通信論壇中收到垃圾信，您可到 spamcop送出相關資料. 一般來說，您不用再送一份給通信論壇擁有者，因為很有可能他已經看到了。 請注意，在論壇上抱怨垃圾信的事是會有不良的後果，會產生比垃圾信本身 更多不必要的流量...
OpenBSD 的通信論壇會拒絕來自 MAPS RBL, MAPS Dialup 或 MAPS RSS 的黑名單。 此外, 通信論壇伺服器也會採用 regex-based 的規則，根據某些一般垃圾信的證據拒絕信件往來。

通信論壇

要訂閱某個通信論壇，請寄信至 majordomo@OpenBSD.org 信件內容請寫 "subscribe mailing-list-name".

announce
公告。 這個量少的通信論壇很適合給那些想要偶爾知道新聞消息的人
security-announce
安全公告。 這個量少的通信論壇接收 OpenBSD 的安全報告，並且指出相關的安全修補。
misc
使用者的問題與解答, 一般性的問題. 這是個最活躍的通信論壇. 在您發表之前請 閱讀常見問題及安裝文件。
tech
技術主題，適合 OpenBSD 開發者及進階使用者。 請 導向 '新使用者' 及安裝相關問題 到 misc. 請不要 同時跨區轉貼到 misc 和 tech.
bugs
臭蟲回報 一如透過 sendbug(1)回報及後續討論。
source-changes
自動將 CVS 原始碼更動部分寄出
ports
討論有關使用及發行 'ports' 原始碼
ports-changes
自動將有關 ports 的 CVS 原始碼更動部分寄出
ipv6
討論有關 OpenBSD IP version 6.
advocacy
促進 OpenBSD 的使用. 在 misc非技術相關的討論，通常會轉到這裡。
smp
討論有關 Symmetric Multi-Processing 在 OpenBSD 的設計及實作

OpenBSD 也維護了數個有關特定的平台及特別計畫的通信論壇。 這些論壇都是公開的，所有人都可以訂閱，但是這些應該考量是保留給開發者及測試者們 。為了保持 OpenBSD 開放的原則，並沒有隱密的通信論壇。
如果您想要取得 openbsd.org 完整的通信論壇列表，請送 "lists" 的命令在信件的內容中，並寄至 majordomo@OpenBSD.org.

要取得更進一步的協助, 請送內容為"help"的信件 至 majordomo@OpenBSD.org 然後您會收到一個回覆概述所有的選擇。您的網域 必須 可以被正常的解析否則信件會無法通過!

squish.net的優秀成員 利用通信論壇整理每天及每星期的 OpenBSD source-changes論壇摘要. 這對不喜歡這個量大的論壇的人來說是很便利的。

在 monkey.org 機敏的猴子們維護了 openbsd-mobile 這個通信論壇，給在筆記型電腦上使用 OpenBSD 的人使用. 訂閱方法:
echo subscribe | mail openbsd-mobile-request@monkey.org

在 benzedrine.cx不眠不休的人 維護了 pf 這個通信論壇，給在 OpenBSD 上使用 packet filter 的人. 訂閱方法:
echo subscribe | mail pf-request@benzedrine.cx

給 OpenBSD FTP, Web, AnonCVS and CVSup 映射站維護者的通信論壇 在 rt.fm.

訂閱方法, 請瀏覽下述 URL: http://www.shellhung.org/mailman/listinfo/openbsd/.

捷克語: users@openbsd.cz
訂閱方法, 請瀏覽下述 URL: http://openbsd.cz/mailman/listinfo/users/.

希臘語: openbsd@bsd.gr
訂閱方法請寄信至 majordomo@bsd.gr 內容為 "subscribe openbsd".

義大利語: sikurezza.org, 一個義大利語的非商業安全入口網站 openbsd@sikurezza.org.
如欲訂閱寄封空白信件至openbsd-subscribe@sikurezza.org.

日語: 請瀏覽下述 URL: http://www.openbsd.ics.nara-wu.ac.jp/wakakus 詳看更多細節

波蘭語: 訂閱方法，寄信至 OpenBSD-request@bsdzine.org.

葡萄牙語: openbsd@neei.uevora.pt
訂閱方法，請瀏覽下述 URL: http://neei.uevora.pt/mailman/listinfo/openbsd/.

俄語: openbsd@openbsd.ru
訂閱方法，寄信至 minimalist@openbsd.ru 標題為 "subscribe openbsd".
