郵件論壇

在 OpenBSD 中，郵件論壇是一個讓使用者與開發者溝通的重要方法。除了 announce 及 security-announce 之外，其他的郵件論壇, 都不需經過仲裁者同意即可散佈的。我們慎重的限制郵件論壇的數量。 這對減少跨區發表的數量有幫助, 並且確保資訊可以充分地傳達到訂閱者的手上。

使用習慣規定

請審慎地考慮到其他的訂閱者.

純文字, 每行 72 字元
許多訂閱者用文字模式的收信程式(mail(1), emacs, Mutt)讀信，通常 HTML 的格式 或是一行超過 72 字元的信是無法閱讀的。 大部分的 OpenBSD 郵件論壇, 在送信之前會將 MIME 內容過濾掉。 如果您不是用純文字的話，您的訊息會試著被重新格式化，抑或如果沒辦法被重新 格式化，則會立刻被拒絕掉。 唯一接受夾檔的郵件論壇是ports，這在其他郵件論壇是會被移除的。
在您發表之前，您做好功課了嗎
如果您有安裝問題，請確認您已經讀過相關的文件 例如 INSTALL.* 文字檔在 FTP 安裝目錄下， 常見問答 及相關的使用手冊(從 afterboot(8) 開始), 並請在提問前先查詢 archives 郵件論壇. 我們希望幫助您，但是我們不會剝奪您有價值的學習經驗。而且也沒有人會想在同一個 月內, 看到重複的問題一再出現.
採用有幫助的標題列
採用空標題的訊息會先轉給郵件論壇管理者，所以他們會過更久時間才會露面. 採用一個恰當的標題, 才會確保有更多人真正會去看您寫的內容。 另外，避免在標題列使用過多的大寫字。 "Help!" 或 "I can't get it to work!" 之類的標題, 並不能幫助別人瞭解你的問題. 在同一主題時, 請勿隨意改變標題列. 你也許知道所討論的話題, 但是其他人通常每天 都要閱讀幾百封信件, 並沒辦法時時注意每封信件的來龍去脈.
縮減您的簽名檔
控制郵件最後的簽名列維持在合理的長度. 文章很少緊要到要用 PGP 簽章, 目前會自動過濾掉簽章內容, 那些法律聲明及建議只會讓人感到厭煩而已。
適切主題
請發表對 OpenBSD 使用者相關的主題
講重點
如果你的問題不完整, 只會浪費大家的時間. 除了你之外, 沒有人能提供更多 與問題相關的資訊, 最好每次都能盡你所能, 提供任何與問題相關的資訊, 就算提供過多資訊也總比資訊不足來得好. 每個問題都至少該包含 OpenBSD 的 版本資訊 (比如, "3.2-stable", "2003 年 7 月 20 號的 3.3-current"). 任何硬體相關問題, 都應該 說明所使用的硬體平台 (比如, sparc, alpha 等), 並且提供完整的 dmesg(8) 資訊. 不幸的是, 硬體型號並不能說明特定主機或附件的實際內容, 而且對沒有那台 特定型號的人來說, 也是沒有辦法辨認出來的. dmesg(8) 的資訊則可以告訴我們, 你的主機內究竟含有哪些硬體配備, 而主機外殼的標籤紙, 是沒有意義的.
尊重不同的意見與觀點
聰明的讀者, 可能對於同樣的現象會有不同的解讀. 重複同樣的觀點, 並不能 說服別人, 而且也會惹惱其他讀者.
不要跨區發信及重複發信
跨區發信及重複發信, 並不會增加你得到回應的機會, 比較可能的結果是, 你會惹惱那些原本想幫助你的人. 如果你在正確的信區, 得不到滿意的答案, 通常是因為你未提供足夠的資訊, 或是所提供的資訊不清楚. 不要只是重複發信.

垃圾信件

OpenBSD 郵件論壇使用 SpamAssassin 來減低垃圾郵件的數量, 但仍會有些漏網之魚. 此外, 論壇伺服器也使用字串規則比對, 來阻擋常見的垃圾信. 如果發現有垃圾信件發到 OpenBSD 郵件論壇, 你可以到 spamcop 記錄該圾垃郵件. 一般來說, 你不需要送出郵件副本給論壇管理員, 因為他也應該發現同一封垃圾信件了. 請注意, 在論壇上抱怨垃圾信的事是不切實際的, 這樣只會產生比垃圾信本身 更多不必要的流量...

郵件論壇

一般郵件論壇

這些是大部份 OpenBSD 使用者感興趣的論壇.
misc
使用者的問題與解答, 一般性的問題. 這是個最活躍的通信論壇. 在您發表之前請 閱讀常見問題及安裝文件, 並請在發問前看如何回報問題.
advocacy
Promoting the use of OpenBSD. Non-technical discussions in misc often get shunted here.
announce
重要公告。 這個信件量少的通信論壇, 很適合給那些偶爾想知道計畫新聞消息的人
security-announce
安全公告。 這個信件量少的通信論壇, 會接收到 OpenBSD 的安全報告， 並且指出相關的安全修補。
ports
討論有關使用及新增 'ports' 原始碼目錄的內容.
www
討論 OpenBSD 網站.

發展者的郵件論壇

這些郵件論壇, 討論的是 OpenBSD 的技術問題. 通常這些論壇, 並不適合給初學者 或一般使用者閱讀, 這些論壇不適用於回報問題(除非你附加上很好的修補檔), 也不適合安裝上的問題討論. 如果你不知道該送到哪一個論壇, 通常這表示你的 問題不適合寄送到這些論壇上, 而該送到 misc 郵件論壇. 請千萬 不要跨區發信到多個論壇上.
bugs
系統錯誤回報 是經由 sendbug(1) 及接續的討論回應所組成. 如果你想要讓自己的信息被 GNATS 系統錯誤追蹤系統記錄下來, 請回一封信件到 gnats@openbsd.org, gnats@ 的信件會被轉寄到 bugs@.
ipv6
討論 OpenBSD 上的 IPv6 及 IPsec.
ports-bugs
討論 OpenBSD ports 原始目錄碼的錯誤問題.
smp
在 OpenBSD 支援 multiprocessor 的 開發進度. 需要大家幫忙測試, 但是請不要抱怨其緩慢的發展進度. 歡迎你寄送 自己的實作程式碼.
tech
討論 OpenBSD 發展者及進階使用者的技術問題. 這不是一個"技術支援" 的論壇, 請不要把他當作技術支援論壇.
x11
討論 OpenBSD 上的 X11 發展進度.

CVS 更新紀錄的郵件論壇

每次有發展者將更新 commit 到 OpenBSD CVS 目錄, 就會有一封包含 commit comments 的 信件, 寄送到這些郵件論壇的所有訂閱者.
source-changes
自動寄出的 CVS 原始程式碼目錄的更新紀錄, 包含除了 ports 更新以外 的其他更新紀錄.
ports-changes
自動寄出 ports 相關的 CVS 原始程式碼目錄的更新紀錄.

CTM 郵件論壇

CTM 包含了許多郵件論壇, 是另一種原始碼更新紀錄的 配發系統. 詳情請參閱 CTM 網頁.

Managing Mailing List Membership via Majordomo

如果你想收到一份, 在 openbsd.org 上完整的郵件論壇列表, 請在寄給 majordomo@OpenBSD.org 的郵件內容中, 加入以下指令列 "lists".

想要訂閱郵件論壇時, 請寄一封信到 majordomo@OpenBSD.org 郵件內容, 請加入一列 "subscribe mailing-list-name".

要取得更進一步的協助, 請送內容為"help"的信件 至 majordomo@OpenBSD.org 然後您會收到一個回覆, 概述所有的選擇。您的網域名稱, 必須 可以被正常的解析, 否則信件會無法寄達!

郵件論壇技巧

有許多非常有用的選項可以使用, 可以由 網頁介面 或透過 Majordomo. 你可以在不退訂 郵件論壇的情況下, 改變自己用來收信的電子郵件信箱. 你可以在外出度假時,, 暫停信箱收信, 此外還有更多其他功能. 使用者可以花些時間閱讀這些選項, 請寄一封內容包含 "help" 的信到 Majordomo 或在網頁介面選擇 "Help" 選項.

比如, 你外出度假兩個星期, 也不希望在回家後收到數千封的電子郵件, 你可以暫時 這段時間的訂閱服務, 並且在你回到家之後, 再自動開始收信, 請用指令: 

     set all nomail-14d
這會把你所有的郵件論壇訂閱狀態暫停 14 天 (-14d). 更多其他的功能可以參考 Majordomo 概述網頁.

Digests

如果你比較喜歡看"摘要"(一個定期整合所有信件的列表), 而不喜歡"即時"的收到 各別信件, 你可以使用以下指令: 
     set misc digest-daily
     set source-changes digest-weekly
可以訂閱 misc 論壇的每日摘要, 及 source-changes 論壇的每週摘要. 沒錯, 是可以在同一封 Majordomo 電子郵件中, 發出多個指令.

其他郵件論壇

在squish.net的好心人士, 提供了 OpenBSD source-changes 及 ports-changes 郵件論 壇的每日, 及每週摘要. 這對於那些不想收到這些論壇大量信件的人來說, 相當方便.

The clever monkeys at monkey.org maintain the openbsd-mobile list for people using OpenBSD on mobile and laptop computers. To subscribe:
echo subscribe | mail openbsd-mobile-request@monkey.org

The insomniac at benzedrine.cx maintains the pf list for people using the OpenBSD packet filter. To subscribe:
echo subscribe | mail pf-request@benzedrine.cx

A mailing list for OpenBSD FTP, Web, AnonCVS and CVSup mirror maintainers is available at rt.fm.
