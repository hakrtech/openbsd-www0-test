  • Todas as arquiteturas

    • 002: CORREÇÃO DE CONFIANÇA: 20 de Agosto de 2003
      Uma verificação imprópria na chamada de sistema semget(2) pode permitir a um usuário local causar um pânico no kernel.
      Uma correção via código fonte existe e corrige esse problema.
    • 001: CORREÇÃO DE SEGURANÇA: 4 de Agosto de 2003
      Um erro do tipo off-by-one existe na função da biblioteca C realpath(3). Pelo motivo de que esse bug resultou em um comprometimento do root no servidor ftp wu-ftpd é possível que este bug possa permitir ao atacante ganhar privilégios escalados no OpenBSD.
      Uma correção via código fonte existe e corrige esse problema.
