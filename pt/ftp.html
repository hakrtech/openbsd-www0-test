FTP

O OpenBSD organiza seus espelhos FTP baseando-se em um servidor mestre, usado para alimentar os espelhos de segundo nível, os quais outros sites devem espelhar-se e obter suas informações. Os espelhos de segundo nível são escolhidos a partir de sites com boa conectividade de rede e mantenedores conhecidos pela comunidade de desenvolvimento do OpenBSD. Se você deseja manter um espelho ftp, por favor assegure-se de usar um espelho secundário próximo para obter seus dados - o espelhamento a partir do servidor mestre é fortemente desencorajado.

Layout Estrutural:

A estrutura de arquivos de cada servidor é aproximadamente:

  • /pub/OpenBSD/
    Aonde você encontra nossa mais recente versão.
    • alpha/
    • amd64/
    • cats/
    • hp300/
    • hppa/
    • i386/
    • luna88k/
    • mac68k/
    • macppc/
    • mvme68k/
    • mvme88k/
    • packages/
      Pacotes binários (ports pré-compilado) para instalar em seu sistema -stable.
    • sgi/
    • sparc/
    • sparc64/
    • tools/
      Ferramentas para outros sistemas operacionais (Windows, Linux) para criar discos de boot, trocar tamanho de partições e outras coisas.
    • vax/
    • zaurus/
    • src.tar.gz
       Códigos fonte da parte de aplicação, que colocamos em /usr/src. (Este arquivo não inclui os códigos fonte do kernel)
    • sys.tar.gz
       Códigos fonte do kernel, que você irá colocar em /usr/src/sys.
    • ports.tar.gz
       Códigos fonte do ports, que você irá colocar em /usr/ports.
    • XF4.tar.gz
       Códigos fonte do Sistema X Window (X11).

  • /pub/OpenBSD/snapshots/
    Para nossas arquiteturas principais, tentamos construir mini versões de lançamento de qualidade e estabilidade incertos a cada mês ou mais. Este é o local que colocamos estes arquivos.

  • /pub/OpenBSD/src/
    Este é um checkout diário de nossa árvore de códigos fonte para aqueles que desejam acessar arquivos individualmente. Normalmente nós recomendamos a utilização do CTM ou AnonCVS ao invés de espelhar este local. Além disso, nem todos os espelhos FTP sincronizam este diretório.

  • /pub/OpenBSD/doc/
    O FAQ do OpenBSD em PDF e texto puro, além de outras documentações relacionadas ao OpenBSD.

  • /pub/OpenBSD/tools/
    Vários bits pequeninos.
  • /pub/OpenBSD/3.6/
    O layout é similar a versão 3.7 descrita acima.. para o nostálgico.
  • /pub/OpenBSD/3.5/
    O layout é similar a versão 3.7 descrita acima.. para o nostálgico.
  • /pub/OpenBSD/3.4/
    O layout é similar a versão 3.7 descrita acima.. para o nostálgico.

Pessoas que desejam configurar um novo espelho FTP/ANONCVS devem prestar atenção nas estimativas de espaço em disco para o conteúdo mantido em ftp://ftp.openbsd.org:/pub/OpenBSD/.
A política atual obriga os espelhos listados nesta página a disponibilizar pelo menos duas últimas versões anteriores a atual em formato binário (atualmente 3.6 e 3.7) para todas as arquiteturas, além dos diretórios OpenSSH/, patches/ e tools/.
Dependendo do espaço em disco disponível, os espelhos podem conter mais conteúdo, como snapshots, versões antigas, árvore de códigos fonte atual, etc.
A utilização de espaço em disco no servidor ftp master é a seguinte: 


                   atualmente pode chegar
pub/OpenBSD/3.4       10.0GB       	 (OpenBSD 3.4 release)
pub/OpenBSD/3.5        9.8GB       	 (OpenBSD 3.5 release)
pub/OpenBSD/3.6       17.4GB    	 (OpenBSD 3.6 release)
pub/OpenBSD/3.7       17.3GB    18.5GB   (OpenBSD 3.7 release)
pub/OpenBSD/snapshots 15.9GB    maior!   (snapshots atuais, normalmente >= que a 
					 última distribuição em tamanho)
pub/OpenBSD/src         .5GB      .6GB   (árvore de códigos fonte disponível
					 via ftp, sincronizada toda a noite. Você
					 deve evitar usar um espelho ftp para
					 obter este diretório. É melhor atualizar
					 seus arquivos a partir de um servidor anoncvs.)
Portanto um espelho completo requer aproximadamente 75GB de espaço em disco, porém se limitando somente as duas últimas versões, somente 50GB são necessários. Note que o servidor ftp master também disponibiliza os seguintes diretórios, os quais os espelhos não deve fornecer: 

pub/OpenBSD/distfiles  4.7GB    maior!   (arquivos de distribuição utilizado
                                         pelos makefiles do subsistema de ports.
                                         Você provavelmente não
                                         deve espelhar esses arquivos a não ser
                                         que você possua alguma razão em especial
                                         para isso. Os makefiles do ports irão
                                         procurar por padrão em ftp.openbsd.org
                                         e ftp.usa.openbsd.org)

Se você configurar um novo espelho FTP, HTTP, AFS ou RSYNC, entre em contato com www@openbsd.org.
