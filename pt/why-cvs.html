Porque os desenvolvedores do OpenBSD usam CVS

O "Concurrent Version System" (CVS) foi escolhido para a utilização pelo grupo de desenvolvimento do projeto OpenBSD. Com o CVS, setenta ou mais desenvolvedores são capazes de atualizar o código fonte (com um tamanho aproximado de 1.2GB, possuindo cerca de 140.000 arquivos), que engloba o SO, os portes/pacotes e o sistema X Window. Um grande número de usuários baixam o sistema via CVS anônimo.

O CVS também proporciona a fácil criação e manutenção de ramos, para que assim possamos disponibilizar uma árvore de código fonte com uma versão + correções e uma árvore de código fonte com o último código de desenvolvimento, baseando-se em um ambiente de desenvolvimento paralelo. Com o CVSweb, tudo isso está disponibilizado através de uma interface Web agradável e conveniente.

Pelo fato de que o CVS interage com o Secure Shell (ssh(1)), ele pode disponibilizar autenticação confiável de usuário e máquina. Isso se encaixa perfeitamente nos objetivos de segurança do projeto.
