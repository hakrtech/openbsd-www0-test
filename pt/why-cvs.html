Porque os desenvolvedores do OpenBSD usam CVS

O Sistema de Versões Concorrentes (CVS) foi escolhido pelo grupo de desenvolvimento do projeto OpenBSD para que setenta ou mais desenvolvedores pudessem atualizar o código fonte (que é maior que 1349MB, e possui cerca de 75000 arquivos) que englobam desde o sistema operacional, os ports/pacotes, e o Sistema X Window. Um grande número de usuários baixam o sistema via CVS anônimo, um procedimento muito mais confiável que o Protocolo de Atualização de Software (sup(1)).

O CVS proporciona também a fácil criação e manutenção de ramificações para que assim possamos disponibilizar uma árvore de códigos fonte com uma versão + correções e uma árvore de códigos fonte com os últimos códigos de desenvolvimento, baseando-se em um ambiente de desenvolvimento paralelo. Com o CVSweb, tudo isso está disponibilizado atravéz de uma interface web agradável e conveniente.

Pelo fato de que o CVS interage com o Secure Shell (ssh(1)), ele pode disponibilizar autenticação de usuário e máquina confiáveis. Isso se encaixa perfeitamente nos objetivos de segurança do projeto.
