OpenBSD/vax

Geschichte und Status:

VAX ist seit Anfang 2000 eine unterstützte OpenBSD Architektur, als Code mit dem Port von NetBSD synchronisiert wurde. Da sie die zweite Architektur waren, auf denen überhaupt ein BSD Unix gelaufen ist (im Jahr 1979!), schien es nur logisch, dass diese ehrwührdigen Maschinen ein modernes BS haben sollten. Die Codebasis ist grösstenteils aus Berkeley, mit Zugaben von freiem Code von Digital, Mt Xinu, NetBSD, OpenBSD und anderen. Die Entwicklung geht weiter, und die Änderungen werden bis zum heutigen Tag unter VAX Hackern ausgetauscht. Das Resultat ist ein Betriebssystem, das stabil ist und sich für jeden OpenBSD-Benutzer heimisch anfühlt.

Wichtige Änderungen ziwschen 3.1 und 3.2:

  • Korrektes Habndling von arithmetischen Fehlern führt zu funktionierenden SIGFPE, SIG_IGN
  • Kernel debugger damit er zwei-byte opcodes erkennen und disassemblieren kann
  • Unterstützung für das Booten von integrierten MSCP Platten

Wichtige Änderungen zwischen 3.0 und 3.1

  • UntersT6uulm;tzung für "Cougar" Maschinen ist bestätigt
  • "Legacy" Maschinen werden jetzt auch unterstützt.
  • Binary logical page size auf 4096 bytes geändert.

Wichtige Änderungen zwischen 2.9 und 3.0:

  • Ünterstützung für "Omega" Maschinen hinzugefügt.
  • Event counters zu den meisten devices hinzugefügt.
  • Mehr Addressierungmodi zum Emulationssystem hinzugefügt.

Es gibt signifikante Änderungen zwischen 2.8 und 2.9:

  • Unterstützung für 16 Dateisysteme pro Festplatte
  • Emulation des EMODD Befehls.
  • Unterstützung für KA53 (Cheetah) Maschinen
  • Es gibt keine ksh 'subshell cores' mehr durch Interrupt Signale

OpenBSD läft auf einer großen Breite von VAX Hardware, aber das Inventar unserer Systeme ist begrenzt. OpenBSD/vax könnte sogar auf deiner Hardware arbeiten, auch wenn sie hier nicht extra aufgelistet ist. Wir sind abhängig von eurem Feedback, was bereits unterstützt ist und was noch Arbeit bedarf. Nachdem die Installtion fertig ist und du dein System kurz getestet hast, denk bitte daran ein "dmesg | mail dmesg@openbsd.org" auszuführen, da uns das dabei hilft, festzustellen, was für Hardware die Leute benutzen. Die Maschinen, die auf dieser Seite aufgeführt sind, arbeiten mit dem letzten Snapshot gut.

Dokumentation, die wir noch brauchen:

  • Programmier-Info für LCG oder SPGFX Framebuffer
  • Software Referenzen für SHAC DSSI
  • VARM jeglicher Version

Bekannte Probleme:

  • Toolchain Probleme - übereifriger gcc Optimierer
  • pmap - Adressierungs Limits

Zusätzlich zu den offiziellen Versionen zeigen snapshots den Stand unserer aktuellen Arbeit und sind auf den ftp mirrors erhältlich, ausserdem wurde eine Mailinglist aufgesetzt, die vax@openbsd.org heisst. Um die OpenBSD/vax Mailingliste zu abonnieren, schicke eine Mail mit dem Inhalt "subscribe vax" an majordomo@OpenBSD.org. Lies aber bitte unbedingt unsere Mailinglisten Policy vor dem Einschreiben in die Liste.
