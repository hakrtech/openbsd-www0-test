OpenBSD/vax

OpenBSD/vax läuft auf einer grossen Anzahl von VAX Computern, die von Digital hergestellt wurden (dann Compaq, jetzt Hewlett-Packard).

Eine Mailingliste, die sich nur um den OpenBSD/vax port dreht, ist unter vax@openbsd.org verfügbar. Um daran teilzunehmen, sende eine Mail mit dem Inhalt "subscribe vax" an majordomo@openbsd.org. Lies vorher aber unbedingt unsere Mailinglisten Policy.

Die aktuellen port maintainer sind Brandon Creighton und Hugh Graham. Weitere Mitarbeiter sind definitiv willkommen !

Geschichte:

VAX ist seit Anfang 2000 eine unterstützte OpenBSD Architektur, als unser Code mit dem Port von NetBSD synchronisiert wurde. Da sie die zweite Architektur waren, auf denen überhaupt ein BSD Unix gelaufen ist (im Jahr 1979!), schien es nur logisch, dass diese ehrwührdigen Maschinen ein modernes BS haben sollten.

Die aktuelle Codebasis stammt grösstenteils aus Berkeley, mit Spenden von freiem Code von Digital, Mt Xinu, NetBSD, OpenBSD und anderen.

Nachdem der port mit der NetBSD Codebasis synchronisiert worden ist, wurden Fixes und Verbesserungen vorgenommen, so wie zum Beispiel die Unterstützung von 16 Partitionen pro Laufwerk, Unterstützung für mehr Modelle der VAX, verbesserte Emulation von unimplementierten Anweisungen und so weiter.

Die Entwicklung geht weiter, und Änderungen werden sogar heute noch zwischen den verschiedenen VAX-Hackern hin- und hergegeben. Das Ergebnis ist ein stabiles und jedem OpenBSD-User vertrautes Betriebssystem.

Aktueller Status:

Momentan bootet all die Hardware, die weiter unten in der Liste der unterstützten Hardware zu finden ist, in den Multi-User Modus, und unterstützt genügend der on-Board Geräte, um gut benutzbar zu sein. Einige Modelle können trotzdem zur Zeit nur ohne Festplatte betrieben werden.

Da zur Zeit keiner der VAXstation Frame-Buffer unterstützt wird, gibt es bis jetzt auch keine Unterstützung für X-Window. Trotzdem ist ein komplettes Set von X Clients und Utilities verfügbar, was OpenBSD/vax Maschinen erlaubt, als X11 Font Server zu laufen, oder X Clients auf einem entfernten Display laufen zu lassen.

Trotzdem gibt es noch ein paar Probleme im OpenBSD/vax port:

  • Der aktuelle Compiler hat manchmal Optimizer Probleme. (z.B. produziert gcc -O2 unkorrekten Code, während -O1 oder -O0 funktioniert). Daran wird langsam aber sicher gearbeitet. Der meiste Code sollte aber funktionieren.
  • Addressierungs-Grenzen im pmap Modul. Dies führt manchmal zu einem Kernel Panic. Einige dieser Situationen wurden beseitigt und werden sauber beendet, aber es mag noch bestehende Probleme geben.

Projekte (ohne besondere Reihenfolge):

  • Treiber für die VAXstation 2000's HDC9224 programmieren (Floppy/MFM Festplatten-Controller)
  • Die MSCP Unterstützung ist noch etwas unrund
  • Programmieren von Treibern für Frame-Buffer.
  • Umschwenken auf ELF und shared libraries.
  • Unterstützung für alle neuen Systeme, denen wir begegnen.

Unterstützte Hardware:

Die unten aufgeführte Liste der unterstützten Hardware soll nicht allumfassend sein. Wegen der eingeschränkten Menge an Systemen, die den Entwicklern zur Verfügung steht, wurden nicht alle Kombinationen getestet.

Als Ergebnis funktioniert OpenBSD/vax vielleicht sogar auf deiner Maschine, auch wenn sie nicht unten aufgeführt ist, und wir sind von euren Tests und Rückmeldungen abhängig, um diese Liste auf dem Laufenden zu halten. Um die Genauigkeit dieser Liste zu verbessern, sende uns BITTE dein dmesg nach der Installation an dmesg@openbsd.org.

Unterstützte Modelle

Fast alle on-board Geräte in diesen Modellen werden unterstützt, mit Ausnahme der Frame Buffer und Tastatur-Interfaces. SCSI Controller werden nur dann unterstützt, wenn sie aufgelistet sind.

  • VAXstation 3100/{30,40} (SCSI)
  • VAXstation 3100/{38,48} (SCSI)
  • VAXstation 3100/76 (SCSI)

  • VAXstation 4000/60 (SCSI)

  • MicroVAX 3100/{10,20}{,e} (SCSI)
