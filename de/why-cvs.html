Warum OpenBSD Entwickler CVS benutzen

Das Concurrent Version System (CVS), entwickelt und gepflegt von Cyclic Software wurde für das OpenBSD-Projekt ausgewählt. Mit Hilfe von CVS sind etwa 70 Entwickler in der Lage den Source-Code upzudaten (das sind etwa 1349MB, in ungefähr 1500 Dateien), das schliesst das Betriebssystem, die Ports/Packages und XFree86 ein. Eine grosse Anzahl von OpenBSD-Usern kann die Sourcen per anonymous CVS herunterladen, diese Prozedur ist wesentlich zuverlässiger als das Software Upgrade Protocol (sup).

Da CVS mit der Secure SHell (ssh) zusammenarbeitet, bietet es zuverlässige User und Host-Authentifikation. Das passt gut in die Ziele von OpenBSD, insbesondere das Ziel von Sicherheit.
