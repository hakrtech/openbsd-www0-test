  • Alle Architekturen

    • 009: SECURITY FIX: 3. März 2003
      Ein buffer overflow in der envelope comments Verarbeitung in sendmail(8) könnte einem Angreifer erlauben, root Privilegien zu erlangen.
      Ein Source Code Patch behebt das Problem.

    • 008: SECURITY FIX: 25. Februar 2003
      httpd(8) lässt file inode Nummern via ETag Header fallen, genauso wie auch child PIDs in der multipart MIME boundary Erzeugung. Das kann zum Beispiel zum Ausbeuten von NFS führen, da hier inode Nummern als Teil des Datei-Handle benutzt werden.
      Ein SourceCode Patch behebt beide Probleme.

    • 007: SECURITY FIX: 22. Februar 2003
      In ssl(8) kann durch das Timing beim Berechnen eines MAC ein Informationsleck entstehen, sogar wenn inkorrektes Block Cipher Padding gefunden wurde, ist dies eine Gegenmassnahme. Überprüfe die Zuweisungroutinen auch auf negative Grössen.
      Ein SourceCode Patch behebt beide Probleme.

    • 006: SECURITY FIX: 20. Januar 2003
      Ein "double free" in cvs(1) könnte einem Angreifer erlauben, Code mit den Privilegien desjenigen Users auszuführen, der CVS laufen lässt. Das ist nur ein Problem, wenn der ausgeführte cvs-Befehl unter einem anderen User läuft. Das bedeutet, dass das Problem nur bei Konfigurationen wie pserver Client/Server existiert.
      Es gibt einen Source Code Patch, der das Problem beseitigt.

    • 005: SECURITY FIX: 14. November 2002
      Ein buffer overflow im named(8) kann einem Angreifer erlauben, Code mit den Privilegien des named auszuführen. Auf OpenBSD läuft der named als non-root User in einer chrooted Umgebung, was den Effekt dieses Bugs mildert.
      Es gibt einen Source Code Patch, der das Problem beseitigt.

    • 004: RELIABILITY FIX: 6. November 2002
      Ein Logik-Fehler im pool kernel memory allocator kann eine Speicherverletzung bei Situationen mit wenig Speicher zulassen, was zu einem Systemcrash führen kann.
      Es gibt einen Source Code Patch, der das Problem beseitigt.

    • 003: SECURITY FIX: 6. November 2002
      Ein Angreifer kann die Restriktionen von sendmail's restricted shell smrsh(8) umgehen, und beliebige Befehle mit den Privilegien seines eigenens Accounts ausführen.
      Es gibt einen Source Code Patch, der das Problem beseitigt.

    • 002: RELIABILITY FIX: 6. November 2002
      Netzwerk bridges, die pf mit eingeschaltetem scrubbing benutzen, können eine mbuf corruption verursachen, was zu einem Systemcrash führen kann.
      Es gibt einen Source Code Patch, der das Problem beseitigt.

    • 001: SECURITY FIX: 21. Oktober 2002
      Es kann ein buffer overflow im kadmind(8) Daemon auftreten, was zu einem remote crash oder exploit führen.
      Es gibt einen Source Code Patch, der das Problem beseitigt.
