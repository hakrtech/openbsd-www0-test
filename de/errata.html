Alle Architekturen

  • 020: SECURITY FIX: 20. September 2004
    Eilko Bos hat gemeldet, dass die Radius Authentifizierungs, so wie von login_radius(8) implementiert, nicht das Shared Secret benutzt, das für die Antworten verwendet wird, die vom Radius-Server gesendet werden. Das könnte einem Angreifer erlauben, eine Antwort zu fälschen, um sich Zugriff zu verschaffen. In OpenBSD ist die Radius Authentifizierung standardmässig abgeschaltet.
    Es existiert ein Quellcodepatch, der das Problem behebt.

  • 019: SECURITY FIX: 16. September 2004
    Chris Evans hat mehrere Fehler (Stack und Integer Overflow) im Xpm Library Code gemeldet, der Bild-Daten parst. (CAN-2004-0687, CAN-2004-0688). Einige dieser Fehler wären ausnutzbar, wenn bösartige Bilder in einer Anwendung XPM Bilder behandeln, sofern diese Fehler ProPolice entkommen würden.
    Es existiert ein Quellcodepatch, der das Problem behebt.

  • 018: SECURITY FIX: 10. September 2004
    httpd(8) 's mod_rewrite Modul kann dazu gebracht werden ein Null-Byte an eine beliebige Speicher Position ausserhalb eines Arrays zu schreiben, was zu einem DoS oder möglichen Buffer-Overflow führen kann. Dazu müsste man jedoch dbm für mod_rewrite einschalten und zusätzlich eine bösartige dbm Datei benutzen.
    Es existiert ein Quellcodepatch, der das Problem behebt.

  • 017: RELIABILITY FIX: 29. August 2004
    Wegen einer falschen Fehlerbehandlung in zlib könnte ein Angreifer möglicherweise einen Denial of Service Angriff ausführen. CAN-2004-0797 .
    Es existiert ein Quellcodepatch, der das Problem behebt.

  • 016: RELIABILITY FIX: 26. August 2004
    Wie von Vafa Izadinia berichtet kann eine bridge(4) mit eingeschalteter IPsec Verarbeitung durch ein einziges ICMP echo zum Absturz gebracht werden, das die bridge durchquert.
    Es existiert ein Quellcodepatch, der das Problem behebt.

  • 015: RELIABILITY FIX: 25. August 2004
    Verbesserte Verifikation von ICMP Fehlern um die Auswirkungen von ICMP Angriffen gegen TCP zu minimieren.
    http://www.ietf.org/internet-drafts/draft-gont-icmp-payload-00.txt
    Es existiert ein Quellcodepatch, der das Problem behebt.

  • 014: RELIABILITY FIX: 25. Juli 2004
    Unter einer bestimmten Netzwerklast kann dem Kernel der Stack space ausgehen. Das wurde in einer Umgebung mit CARP und VLAN Interfaces festgestellt. Ursprünglich manifestierte sich dieses Problem als FPU bezogener Crash beim Hochfahren.
    Es existiert ein Quellcodepatch, der das Problem behebt.

  • 013: SECURITY FIX: 12. Juni 2004
    Mehrere Schwachstellen wurden in httpd(8) / mod_ssl gefunden. CAN-2003-0020, CAN-2003-0987, CAN-2004-0488, CAN-2004-0492.
    Es existiert ein Quellcodepatch, der das Problem behebt.

  • 012: SECURITY FIX: 10. Juni 2004
    Wie von Thomas Walpuski veröffentlicht, ist isakmpd(8) immer noch gegen das unautorisierte Löschen von SAs verwundbar. Ein Angreifer kann IPsec Tunnel nach Belieben löschen.
    Es existiert ein Quellcodepatch, der das Problem behebt.

  • 011: SECURITY FIX: 9. Juni 2004
    Es wurden mehrere remote vulnerabilities im cvs(1) Server gefunden, die es einem Angreifer erlauben, den Server zum Absturz zu bringen oder möglicherweise beliebigen Code mit den Rechten des CVS Servers aszuführen.
    Es existiert ein Quellcodepatch, der das Problem behebt.

  • 010: RELIABILITY FIX: 9. Juni 2004
    In OpenBSD 3.5 wurde ein FIFO Bug eingeführt, der auftritt, wenn ein FIFO im nicht-blockierenden Modus zum Schreiben geöffnet ist, wenn es keinen Prozess gibt, der das FIFO liest. Ein Programm, das davon betroffen ist, ist der qmail Mail Server, der in eine unendliche Schleife geraten und alle CPU-Zeit verbrauchen kann.
    Es existiert ein Quellcodepatch, der das Problem behebt.

  • 009: SECURITY FIX: 30. Mai 2004
    Ein Fehler im Kerberos V kdc(8) Server könnte sich so auswirken, dass der Administrator eines Kerberos Realms sich als Principal jedes beliebigen anderen Realms ausgeben kann, das einen Cross-Realm-Trust zum eigenen Realm aufgebaut hat. Dieser Fehler tritt wegen einer unzureichenden Überprüfung des "transited" Felds in einer Kerberos Anfrage auf. Mehr Details findest du in der Ankündigung von Heimdal.
    Es existiert ein Quellcodepatch, der das Problem behebt.

  • 008: SECURITY FIX: 26. Mai 2004
    Durch die Einführung von IPv6 Code in xdm(1) wurde versehentlich ein Test auf der 'requestPort' resource glöscht. Das führt dazu, dass xdm den Chooser Socket erzeugt, auch wenn xdmcp in der xdm-config durch Setzen von requestPort auf 0 abgeschaltet ist. Vergleiche XFree86 bugzilla für weitere Information.
    Es existiert ein Quellcodepatch, der das Problem behebt.

  • 007: SECURITY FIX: 20. Mai 2004
    Es wurde ein heap overflow im cvs(1) Server entdeckt, der von Clients mittels falsch geformter Anfragen ausgenutzt werden kann, was diese Clients in die Lag versetzt beliebigen Code auf dem Server mit den Rechten, unter denen der Server läuft, auszuführen.
    Es existiert ein Quellcodepatch, der das Problem behebt.

  • 006: SECURITY FIX: 13. Mai 2004
    Prüfen auf Integer Overflow in procfs. Von der Benutzung von procfs wird abgeraten.
    Es existiert ein Quellcodepatch, der das Problem behebt.

  • 005: RELIABILITY FIX: 6. Mai 2004
    Antwort auf ein in-window SYN mit begrenzter Rate von ACKs.
    Es existiert ein Quellcodepatch, der das Problem behebt.

  • 004: RELIABILITY FIX: 6. Mai 2004
    Wiederherstellung der Möglichkeit, Tags/Wide/Sync mit einigen SCSI Kontrollern auszuhandeln (z. B. siop(4), trm(4), iha(4) ).
    Es existiert ein Quellcodepatch, der das Problem behebt.

  • 003: RELIABILITY FIX: 5. Mai 2004
    Unter Belastung werden aktuelle Modelle des gdt(4) Kontrollers einfrieren.
    Es existiert ein Quellcodepatch, der das Problem behebt.

  • 002: SECURITY FIX: 5. Mai 2004
    Pfadüberprüfungsprobleme wurden in cvs(1) gefunden, was böswilligen Clients erlaubt, Dateien außerhalb des Repositorys zu überschreiben, und böswilligen Servern, Dateien außerhalb des lokalen CVS Baumes auf dem Client zu überschreiben, und erlaubt Clients, Dateien außerhalb des CVS Repositorys auszuchecken.
    Es existiert ein Quellcodepatch, der das Problem behebt.
