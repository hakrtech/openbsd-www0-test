  • Alle Architekturen

    • 007: SECURITY FIX: 13. November 2001
      Es gibt ein Sicherheitsproblem im vi.recover Skript, das einem Angreifer erlauben kann, eine beliebige Datei der Länge 0 zu löschen, unabhängig davon, wem sie gehört.
      Es gibt einen Source-Code Patch, der das Problem behebt.

    • 006: SECURITY FIX: 13. November 2001
      pf(4) konnte nicht mit einem bestimmten Typ von IPv6 ICMP Paketen umgehen, was zu einem Crash führte.
      Es gibt einen Source-Code Patch, der das Problem behebt.

    • 002: SECURITY FIX: 12. November 2001
      sshd(8) wird von OpenSSH 3.0 auf OpenSSH 3.0.1 upgegradet, um die folgenden Probleme zu beheben:

      • Ein Sicherheitsloch, das einem Angreifer erlauben könnte, sich teilweise zu authentifizieren, wenn -- und nur wenn -- der Administrator KerberosV eingeschaltet hat.
        Standardmässig kann die OpenSSH KerberosV Unterstützung nur aktiviert werden, wenn KerberosV zuvor sauber konfiguriert wurde.

      • Ein exzessiver memory clearing Bug (von dem wir glauben, dass er nicht auszunutzen ist) existiert auch, aber da das auch zu einem Crash des Daemons führen kann, bieten wir hierfür ebenfalls einen Patch.

      • Verschiedene andere unkritische Fixes.

      Effektiv ein Upgrade von OpenSSH 3.0 auf OpenSSH 3.0.1, gibt es einen Sourcecode-Patch, der das Problem behebt.

    • 001: INSTALL ISSUE: 12. November 2001
      Ein kleiner Bug im Installationsskript führt dazu, dass die /etc/hosts Datei falsch angelegt wird.
      Die resultierende Datei enthält eine Zeile wie die folgende:

         #.#.#.# hostname. hostname

      Die Zeile sollte eigentlich wie folgt aussehen:

         #.#.#.# hostname.domainname.com hostname

      Um das Problem zu beheben, editiere einfach die Datei und füge den Domainnamen am passenden Platz ein.
