  • 002: SECURITY FIX: 2. Mai 2006   alle Architekturen
    Eine Sicherheitslücke wurde im X.Org-Server entdeckt - CVE-2006-1526. Clients, die eine Verbindung zum X-Server aufbauen dürfen, können diesen zum Absturz bringen und beliebigen Code im X-Server ausführen.
    Es existiert ein Quelltextpatch, der das Problem behebt.

  • 001: SECURITY FIX: 25. März 2006   alle Architekturen
    In Sendmail wurde eine bestehende Racecondition bei der Verarbeitung von asynchronen Signalen entdeckt. Ein Angreifer kann so von außen beliebigen Code mit den Privilegien des Benutzers ausführen, der sendmail gestartet hat (normalerweise root). Bei diesem Patch handelt es sich um die zweite Fassung.
    Es existiert ein Quelltextpatch, der das Problem behebt.
