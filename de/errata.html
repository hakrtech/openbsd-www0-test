  • Alle Architekturen

    • 021: SECURITY FIX: 8. Mai 2002
      Es gibt eine race condition bei der ein Angreifer file descriptor Tabelle füllen kann, die Sicherung des Kernels der fd slots 0, 1, und 2 für einen setuid oder setgid Prozess ausschalten kann.
      Ein Source Code Patch behebt das Problem.

    • 002: SECURITY FIX: 25. April 2002
      Ein Bug in sudo(8) erlaubt es einem Angreifer den Heap zu korrumpieren, indem er einen eigenen Prompt angibt.
      Ein Source Code Patch behebt das Problem.

    • 001: SECURITY FIX: 22. April 2002
      Ein lokaler User kann super-user Privilegien wegen eines buffer overflow in sshd(8) erlangen, falls AFS auf dem System konfiguriert ist, oder auch KerberosTgtPassing oder AFSTokenPassing im sshd_config File konfiguriert ist. Ticket und token passing sind standardmässig abgeschaltet.
      Ein Source Code Patch behebt das Problem.
