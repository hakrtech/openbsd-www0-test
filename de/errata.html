Alle Architekturen

  • 012: RELIABILITY FIX: 14. Februar 2004
    Es existieren mehrere Buffer Overflows im Code, der die font.aliases Dateien in XFree86 parst. Dank ProPolice können diese nicht dazu ausgenutzt werden, um erweiterte Rechte zu erlangten, aber sie können den X Server dazu bringen, sich zu beenden.
    Es existiert einen Quellcode-Patch, der das Problem behebt.

  • 011: SECURITY FIX: 8. Februar 2004
    Es gibt ein Problem beim Behandeln der IPv6 MTU, das von Angreifern dazu benutzt werden könnte, Denial of Service Angriffe gegen Rechner mit per IPv6 erreichbaren TCP Ports zu fahren.
    Es existiert ein Quellcode-Patch, der das Problem behebt.

  • 010: SECURITY FIX: 5. Februar 2004
    Es gibt einen Bug beim Zählen von Referenzen im shmat(2) Systemaufruf, der von Angreifern unter bestimmten Umständen dazu benutzt werden könnte, Kernel Speicher zu beschreiben.
    Es existiert ein Quellcode-Patch, der das Problem behebt.

  • 009: SECURITY FIX: 13. Januar 2004
    Mehrere Fehler bei der Nachrichtenbehandlung in isakmpd(8) wurden von Thomas Walpuski gefunden. Sie erlauben einem Angreifer, bestimmte SAs zu löschen. Der Patch beinhaltet auch ein Verläßlichkeitsfix für ein Filedeskriptor-Loch, das Probleme verursacht, wenn eine Crypto-Karte installiert ist.
    Es existiert ein Quellcode-Patch, der diese Probleme behebt.

  • 008: RELIABILITY FIX: 20 November 2003
    Durch eine inkorrekte Grenzwertüberprüfung ist es einem lokalen Benutzer möglich durch die Übergabe bestimmter Argumente an die Funktionen semctl(2) und semop(2) das System zum Absturz zu bringen.
    Es gibt einen Source Code Patch, der das Problem behebt.

  • 007: RELIABILITY FIX: 20 November 2003
    Für einen lokelen Benutzer ist es, durch sysctl(3) mit bestimmten Argumenten, möglich einen Absturz zu verursachen.
    Es gibt einen Source Code Patch, der das Problem behebt.

  • 005: RELIABILITY FIX: 4. November 2003
    Es ist für einen lokalen Benutzer möglich eine "system panic" auszulösen, indem er ein spezielles Binärprogramm mit einem ungültigen Header ausführt.
    Es gibt einen Source Code Patch, der das Problem behebt.

  • 004: RELIABILITY FIX: 1. November 2003
    Ein Benutzer mit Schreibrechten auf die httpd.conf oder eine .htaccess Datei kann den httpd(8) zum Absturz bringen oder möglicherweise eigenen Code als Benutzer www laufen lassen (obwohl wir glauben, dass ProPolice die Code-Ausführung verhindern wird).
    Es gibt einen Source Code Patch, der das Problem behebt.

  • 003: RELIABILITY FIX: 1. November 2003
    Einem lokalen Benutzer ist es möglich, eine system panic auszulösen durch das Fluten mit gefälschten ARP-Anfragen.
    Es gibt einen Source Code Patch, der das Problem behebt.

  • 002: SECURITY FIX: 1. November 2003
    Das Benutzen bestimmter ASN.1 Kodierungen oder fehlgeformte Public Keys erlauben es einem Angreifer, einen Denial-of-Service Angriff gegen Anwendungen zu fahren, die gegen ssl(3) gelinkt sind. Dies betrifft die OpenSSH nicht.
    Es gibt einen Source Code Patch, der das Problem behebt.

  • 001: DOCUMENTATION FIX: 1. November 2003
    Die Dokumentation auf dem CD Booklet hat ein falsches Beispiel für eine Standardinstallation.
    Dort, wo geschrieben steht:

               # pkg_add ftp://ftp.openbsd.org/pub/OpenBSD/3.4/packages/i386

    sollte in Wirklichkeit stehen:

               # pkg_add ftp://ftp.openbsd.org/pub/OpenBSD/3.4/packages/i386/

    Der extra / am Ende der Zeile ist wichtig. Wir geben keine Patch files heraus für Sachen, die auf Papier gedruckt sind.

i386

  • 006: SECURITY FIX: 17 November 2003
    Es könnte für einen lokalen Benutzer möglich sein den Stapel von compat_ibcs2(8) zum Überlauf zu bringen.
    Die ProPolicy fängt eine mögliche Eskalation der Privilegien auf und verwandelt diese in einen Denial of Service. Es ist deswegen nicht nötig, die iBCS2 Emulation mit sysctl(8) zu aktivieren.
    Es gibt einen Source Code Patch, der das Problem behebt.
