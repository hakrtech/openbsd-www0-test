Die Patches weiter unten sind mit CVS im OPENBSD_2_7 Patch Branch erhältlich.

  • Alle Architekturen

    • 013: SECURITY FIX: 28. Juni 2000
      libedit würde nach einer .editrc Datei im momentanen (current) Verzeichnis prüfen. Dieses Verhalten ist nicht nett; soweit wir wissen wird das in keiner realistischen Situation zu einem Sicherheitsproblem, aber es gibt trotzdem einen Patch.
      Ein source code Patch, der das Problem löst, existiert.

    • 012: SECURITY FIX: 24. Juni 2000
      Ein schlimmer Bug in dhclient(8) könnte dazu führen, dass Strings von einem nicht-wohlgesonnenen dhcp server in der Shell als 'root' ausgeführt werden.
      Ein source code Patch, der das Problem löst, existiert.

    • 009: SECURITY FIX: 9. Juni 2000
      Ein schlimmer Bug im isakmpd(8) Policy Handling so dass die Policy Verifikation in isakmpd komplett umgangen werden kann.
      Ein source code Patch, der das Problem löst, existiert.

    • 008: RELIABILITY FIX: 8. Juni 2000
      Einige Operationen in msdosfs können in einem "system panic" resultieren.
      Ein source code Patch, der das Problem löst, existiert.

    • 007: RELIABILITY FIX: 8. Juni 2000
      NFS exportieren von CD Dateisystemen führte zu einem "system panic".
      Ein source code Patch, der das Problem löst, existiert.

    • 006: SECURITY FIX: 6. Juni 2000
      Das nicht-standard "UseLogin" Feature in /etc/sshd_config ist kaputt und sollte nicht benutzt werden. Auf anderen Betriebssystemen resultiert das in einem Sicherheitsloch.
      Vermeide die Benutzung dieses Features, oder bringe dein System auf OpenSSH 2.1.1 oder höher, wenn du es benutzen musst.

    • 005: RELIABILITY FIX: 29. Mai 2000
      Parse die IPv4 Optionen sorgfältiger. Es ist aber noch nicht klar, ob das dazu benutzt werden kann, die Maschine lokal oder remote zu crashen.
      Ein source code Patch, der das Problem löst, existiert.

    • 004: RELIABILITY FIX: 29. Mai 2000
      Bestimmte routing table Modifikationen des Superusers können eine "system panic" hervorrufen.
      Ein source code Patch, der das Problem löst, existiert.

    • 003: SECURITY FIX: 26. Mai 2000
      Es ist möglich das learning Flag auf einem Interface zum umgehen, wenn die Frames ,die direkt an die Maschine gehen als bridge agieren.
      Ein source code Patch, der das Problem löst, existiert.

    • 002: DRIVER FIX: 26. Mai 2000
      Der ef(4) Treiber wird sich beschweren, wenn eine Adresse mittels ifconfig hinzugefügt werden soll (ifconfig: SIOCAIFADDR: Invalid argument).
      Ein source code Patch, der das Problem löst, existiert.

    • 001: SECURITY FIX: 25. Mai 2000
      Ein Missbrauch von ipf(8) keep-state Regeln kann in einer durchlässigen Firewall resultieren.
      Ein Source Code Patch existiert, der das Problem löst und ipf auf Version 3.3.16 bringt.

  • i386

    • 011: DRIVER BUG: 17. Juni 2000
      Der an(4) Aironet Communications 4500/4800 IEEE 802.11DS Treiber hat einen Bug, der ancontrol(8) davon abhält korrekt zu funktionieren und stattdessen ein "panic" verursacht.
      Ein source code Patch, der das Problem löst, existiert.
