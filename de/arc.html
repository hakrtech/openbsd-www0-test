OpenBSD/arc

OpenBSD/arc läuft auf Maschinen, die kompatibel zur Advanced RISC Computing Spezifikation sind, bekannt als ARC Maschinen, basierend auf MIPS Prozessoren und urspünglich entworfen, um darauf Windows NT laufen zu lassen. Solche Maschinen beinhalten den bereits lange ausgestorbenen Acer PICA, ebenso wie andere Maschinen, die von Firmen wie MIPS, Deskstation, NEC, und Olivetti hergestellt wurden.

Die ARC Spezifikation ist ausgestorben, und es werden wahrscheinlich keine neuen ARC BIOS Maschinen für MIPS mehr produziert. Microsoft hat angekündigt MIPS nach NT 4.0 nicht mehr zu unterstützen. OpenBSD/arc war eine gute Alternative zu Windows NT.

Die Arbeit am OpenBSD/arc port wurde nach der Version 2.3 nicht mehr fortgesetzt.

Es arbeitet zur Zeit niemand daran, den port wieder zum Leben zu erwecken. Der Source Code ist weiterhin über CVS Attic verfügbar, zusätzlich kann man welchen in NetBSD finden. Wenn du daran interessiert bist, den port wieder zum Leben zu erwecken, bringe ihn zum Laufen und wende dich dann an Per Fogelström und Warner Losh.

Geschichte:

Die frühe Geschichte ist nicht ganz klar. Es scheint, als ob die erste Arbeit von CMU während ihres Mach-Projektes gemacht wurde. Die erste Hardware waren DEC R2000/R3000-basierte DECstations. Dieser Code wurde später von sowohl der Sprite- als auch der BSD-Gruppe benutzt. Der 4.4BSD Code, auch als `pmax'-Port bekannt, wurde etwa Mitte '93 veröffentlicht. Er wurde von einer Vielzahl von Leuten in den NetBSD-Tree eingefügt, wegen Problemen in der toolchain hat es aber Jahre gedauert, bis er wirklich stabil und zuverlässig lief.

Per Fogelström wurde immer vertrauter mit dem Code, nachdem er ihn auf ein selbstgebautes IDT R3081-basiertes Board portiert hatte. Hinterher fügte er R4400-Unterstützung hinzu, als er das System auf ein MIPS R4400 Acer PICA board portierte. Willowglen Singapore kaufte ein weiteres PICA Board für Theo de Raadt, so dass er den Port soweit verbessern konnte, dass er als Entwicklungssystem für ein internes Projekt benutzt werden konnte. Seitdem haben Theo, Per und andere den Port komplettiert.

Als ein Resultat wurde der Code wandelbarer gemacht, und daher sollte er jetzt eine größere Menge von ARC-Maschinen unterstüzen. Unglücklicherweise lenkte der Tod dieser Plattform und der allgemeine Mangel an solcher Hardware viele Leute davon ab, an diesem port zu arbeiten. Schlussendlich wurde die Arbeit daran eingestellt und der Code aus dem source tree entfernt.

Unterstützte Hardware:

Unterstützte Modelle

  • Acer Pica (150MHz R4400PC, ISA, mit on-board Ethernet, SCSI, Video, und seriell)
  • Deskstation Tyne (133MHz R4600, ISA und VLB)
  • Deskstation rPC44 (100MHz R4400PC, EISA Bus)
  • Algorithmics R4000/R5000/R10000 evaluation boards
  • Algorithmics P-4032 und P-5064 boards
  • NEC RiscStation

Unterstützte Peripheriegeräte

  • Grafik
    • der eingebaute S3 VGA Grafik Adapter (PICA) (mit XFree86 Unterstützung).
    • Standard VGA Grafik-Adapter (rPC44, Tyne)
  • Tastatur
    • Standard PC kompatible Tastatur
  • Mäuse
    • PS2 Maus (PICA)
    • Seriell Maus (rPC44, Tyne)
  • Serielle Ports
    • On-board serielle ports (PICA) (kann nicht als serielle Konsole benutzt werden)
    • ISA serielle Port Controller
  • Parallele Ports
    • On-board Parallel Port (PICA)
    • ISA Parallel Port Controller
  • Floppy-Laufwerk
    • ISA Floppy Controller (PICA)
  • Ethernet
    • on-board SONIC Ethernet Controller (PICA)
    • 3Com Etherlink Boards
    • NE2000 kompatible ISA Boards
  • SCSI Controller
    • on-board NCR53C96 SCSI Controller (PICA)
    • VLB Buslogic BT-440C/445C
    • ISA Buslogic BT-545 SCSI Controller (rPC44)
  • IDE Controller
    • ISA Western-Digital kompatible IDE Controller
      (bootbar nur auf Deskstation Systemen)
  • Verschiedene Geräte
    • Joystick am ISA Bus
    • PCI Bus auf Algorithmics P-4032 und P-5064 Boards
