OpenBSD/sgi

OpenBSD/sgi funktioniert zur Zeit auf den 64 bit MIPS-basierenden SGI O2 Workstations. Andere 64 bit SGI Systeme werden in der Zukunft ebenso unterstützt.

OpenBSD/sgi ist ein voll unterstützter 64 bit Port und funktioniert daher nur auf 64 bit Prozessoren, z.B. R4000 und aufwärts.

Geschichte:

Der OpenBSD/sgi port hat seine Wurzeln im OpenBSD/arc port aus den frühen Tagen von OpenBSD. Die Arc Unterstützung wurde jedoch bald wegen mangeldem Interesse und mangelder Hardware wieder eingestellt. Die Unterstützung von OpenBSD Mips wurde dann ausserhalb des OpenBSD Projekts gepflegt. Als SGI Hardware bezahlbar wurde und leicht zu erwerben war, stieg das Interesse für einen SGI port. Das letzte fehlende Stück, eine funktionierende 64 bit toolchain, ist nun da und der 64 bit SGI OpenBSD port ist somit verfügbar und wird im OpenBSD source tree verwaltet.

Aktueller Status:

Aktuell läuft OpenBSD/sgi im 64 bit Modus und bootet auf verfügbaren Systemen in den Multi-user-Modus. Snapshots werden von Zeit zu Zeit zur Verfügung gestellt, während sich der port entwickelt und vergrössert.

Projekte (ohne besondere Reihenfolge):

  • Workaround für das "coherency" Verhalten von R10000 CPUs
  • Wscons Unterstützung für die in die O2 eingebaute Grafik
  • Unterstützung für den eingebauten Sound
  • Unterstützung des X11 Server
  • Grösserer User VM space (aktuell sind 2GB die Höchstgrenze)
  • Unterstützung für mehr SGI Hardware wie Origin 200, Octane, etc
  • Unterstützung für Multiprozessor Systeme (nicht NUMA)

Unterstützte Hardware:

Unterstützte Modelle

  • O2 (auch Moosehead genannt) mit R5000, RM5200, RM7000, R10000 und R12000 CPUs. Level 2 Caches werden auf allen CPUs unterstützt. RM7000 Level 3 Cache wird ebenso unterstützt.
    • Eingebauter Adaptec AIC-7880 SCSI (ahc)
    • Eingebautes Ethernet (mec)
    • Eingebaute serielle Ports (com)
    • 64 bit PCI Erweiterungs-Slot
    • Intel i8255x-basierendes (except the i82556) PCI Ethernet (fxp)

Unterstützte Peripherie

Da die meisten Modelle Erweiterung durch PCI Karten unterstützen funktionieren vermutlich viele PCI Geräte. Bevor hier aber einzelne Geräte als unterstützt aufgelistet werden, muss jedes einzelne davon getestet werden. Wer also Untersützung für ein bestimmtes Gerät haben will, der sollte ein Exemplar davon einem Entwickler zur Verfügung stellen, der dann Tests machen kann und die Funktion verifizieren.

Bitte setze dich mit Per Fogelström in Verbindung, wenn du mehr Informationen haben willst.
