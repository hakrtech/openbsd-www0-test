Unterstützte Hardware:

PowerPC-basierende Macintosh Systeme werden in "Old World" und "New World" Systeme unterteilt, der Unterschied dabei ist the design of the system ROM. Nur "New World" Maschinen werden unterstützt. Bekanntermassen kompatible Systeme sind z.B.:

  • PowerMac G3 (B&W)
  • PowerMac G4 (PCI Grafik? - ungetestet, sollte aber funktionieren)
  • PowerMac G4 (AGP Grafik)
  • PowerMac G4 (Digital Audio)
  • PowerMac G4 (QuickSilver)
  • PowerMac G4 (QuickSilver 2002)
  • PowerMac Dual G4
  • PowerMac G4 Cube
  • iMac Rev A
  • iMac Rev B
  • iMac Rev C
  • iMac DV
  • iMac DV+
  • LCD iMac (erst ab 3.1)
  • PowerBook G3 (FireWire)
  • PowerBook G4
  • iBook
  • iBook2 (Probleme mit dem Combolaufwerk sollten in 3.1 repariert sein)
  • StarMax (laufende Entwicklung)
  • weitere... (laufende Entwicklung)
Um uns dabei zu helfen, diese Liste aktuell und komplett zu halten, schicke uns BITTE dein dmesg nach der Installation an dmesg@OpenBSD.org.

Multiboot/Geteilte HFS Festplatten:

Ein Multiboot-System mit OpenBSD zu haben, bzw. die Festplatte mit OS X/MacOS 9 zu teilen, ist möglich, solange die Platte zuerst mit dem MacOS Partitions-Utility Drive Setup (MacOS9) oder System Disk Utility (Mac OS X) partitioniert wird. Nicht benutzter Platz auf der Platte oder nicht mehr benötigte Partitionen sollten für die Benutzung durch OpenBSD übergelassen werden. Da OpenBSD nur von HFS (nicht MacOS X UFS) Partitionen booten kann, muss die erste Partition mit HFS(+) und nicht UFS formatiert sein, damit man dann davon OpenBSD booten kann.

Genaueres findet sich in INSTALL.macppc.

Stand-alone (keinerlei MacOS mehr enthalten) Installationen werden auch unterstützt.

Root Drive:

Moderne MacPPC Systeme haben zwei IDE Interfaces (Ultra und IDE genannt), wovon jedes zwei Geräte haben kann (ultra0, ultra1, ide0, ide1). Wenn sich mehrere Geräte im System befinden, gibt es Grenzen dessen, welche Festplatte als OpenBSD root drive genutzt werden kann. Ungültige Konfigurationen können nicht erkennen, welches Laufwerk das root Laufwerk ist, also wird jeder Versuch entweder das root Laufwerk sofort erkennen oder endgültig fehlschlagen.

  • ultra0 ist immer ein gültiges root drive.
  • ultra1 ist ebenfalls eine gültige root disk, solange ultra0 eine Festplatte ist und kein anderes ATAPI Gerät.
  • ide1 (Zip drive bay) ist eine gültige root disk falls ultra1 keine Festplatte ist,
  • ide0 (CD-ROM) ist nur dann ein gültiges root drive, wenn keine Festplatte am Ultra Bus hängt.

X Window System:

XFree86 Version 4.2.0 wird auf allen MacPPC Systemen unterstützt, allerdings haben zur Zeit nur die ATI Grafikkarten einen beschleuigten X Server. Die beschleunigten nVidia X server funktionieren zur Zeit nicht richtig, trotzdem können nVidia (und ATI) Karten von X im framebuffer (nicht beschleunigten) Modus betrieben werden, und zwar mit der Boot-Auflösung und -Farbtiefe mit Hilfe des wsfb Treibers (der standardmässig installiert ist).

Anweisungen zum Aktivieren des beschleunigten X Servers sind in der /usr/X11R6/README Datei auf dem installierten System.

Bekannte Probleme

  • Laptop Systeme können sich innerhalb von 2-5 Minuten abschalten, wenn der Deckel geschlossen wird, und das System gerade von der Batterie gespeist wird.
  • Ältere iMacs können ein Problem mit dem USB Tastatur-Treiber haben, wobei die erste Taste, die nach dem Booten gedrückt wird, endlos wiederholt wird. Der workaround ist, zuerst <SHIFT> zu drücken, oder die USB Maus abzunehmen und wieder dranzustecken, wenn das Problem auftaucht.

Andere Features

  • <CTRL><CMD><POWER> auf ADB (nicht USB) Tastaturen resettet den Computer.

  • Backlight auf LCD-Geräten kann mittels der Brightness Control Keys auf ADB (Laptop) Tastaturen kontrolliert werden, oder durch 'wsconsctl -w display.brightness=xx' (0 - 100) und 'wsconsctl -w display.backlight=xx' (0 oder 100)

nicht unterstützte Hardware:

  • Ältere Apple PowerMac Systeme mit einem 601 Prozessor und solche ohne OpenFirmware werden nicht unterstützt.
  • Ältere OpenFirmware Maschinen werden zur Zeit nicht unterstützt. Das liegt hauptsächlich am Mangel an Entwicklern und zu wenig Hardware.
  • Eingebaute Modems in den meisten Maschinen werden nicht unterstützt. In vielen Systemen wird der serielle Port, an den das Modem angeschlossen ist, unterstützt, aber das Modem selber nicht.
  • Neuere PowerBooks haben ein internes Modem, das als USB Gerät angeschlossen wird, das wird auch nicht unterstützt.
  • Das neue 'digital audio' Subsystem in iBooks und neueren PBG4, G4 und iMacs wird nicht unterstützt.
  • Sleep/Suspend wird nicht unterstützt.
