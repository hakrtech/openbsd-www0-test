Alle Architekturen

  • 021: RELIABILITY FIX: 1. Oktober 2003
    Einem lokalen Benutzer ist es möglich, eine system panic auszulösen durch das Fluten mit gefälschten ARP-Anfragen. Es gibt einen Source Code Patch, der das Problem behebt..

  • 020: SECURITY FIX: 1. Oktober 2003
    Das Benutzen bestimmter ASN.1 Kodierungen oder fehlgeformte Public Keys erlauben es einem Angreifer, einen Denial-of-Service Angriff gegen Anwendungen zu fahren, die gegen ssl(3) gelinkt sind. Dies betrifft die OpenSSH nicht.
    Es gibt einen Source Code Patch, der das Problem behebt..

  • 019: SECURITY FIX: 24. September 2003
    Drei Fälle möglichen Zugriffs auf freigegebenen Speicher wurden in pf(4) gefunden. Mindestens einer von ihnen könnte benutzt werden, um im pf mit aktiven scrub-Regeln eine panic auszulösen. Es gibt einen Source Code Patch, der das Problem behebt..

  • 018: SECURITY FIX: 17. September 2003
    Ein buffer overflow im address parsing in sendmail(8) kann einem Angreifer erlauben, root Privilegien zu erlangen.
    Es gibt einen Source-Code Patch, der das Problem behebt.

    HINWEIS: dies ist die zweite Version des Patches, die ein weiteres Problem behebt.

  • 017: SECURITY FIX: 16. September 2003
    Alle Versionen von OpenSSH's sshd vor der Version 3.7 enthalten einen Buffer Management Fehler. Es ist unklar, ob dieser Bug ausnutzbar (exploitbar) ist.
    Es gibt einen Source-Code Patch, der das Problem behebt.
    HINWEIS: dies ist die zweite Version des Patches, die ein weiteres Problem behebt.

  • 016: SECURITY FIX: 25. August 2003
    Fix für einen potentielles Sicherheitsproblem in sendmail(8) in Bezug auf DNS maps. Betroffen sind nur sendmail(8) Konfigurationen, die das "enhdnsbl" Feature nutzen. Die Standard OpenBSD sendmail(8) Konfiguration benutzt das Feature nicht.
    Es gibt einen Source-Code Patch, der das Problem behebt.

  • 015: SECURITY FIX: 4. August 2003
    Es existiert ein off-by-one in der C-Bibliotheks-Funktion realpath(3). Da der selbe Bug zu einer Root-Kompromittierung im wu-ftpd FTP Server geführt hat, ist es möglich, dass dieser Bug einem Angreifer ebenfalls erlaubt, seine Rechte in OpenBSD zu erweitern.
    Es gibt einen Source-Code Patch, der das Problem behebt.

  • 014: SECURITY FIX: 31. März 2003
    Ein buffer overflow beim Verarbeiten von Adressen in sendmail(8) kann es einem Angreifer möglich machen Root Privilegien zu erlangen.
    Es gibt einen Source Code Patch, der das Problem beseitigt.

  • 013: SECURITY FIX: 24. März 2003
    Eine kryptografische Schwäche im Kerberos v4 Protokoll kann unter Kerberos v5 ebenso ausgenutzt werden.
    Es gibt einen Source Code Patch, der das Problem beseitigt.

  • 012: SECURITY FIX: 19. März 2003
    OpenSSL ist angreifbar durch eine Erweiterung des ``Bleichenbacher'' Angriffs , der von den Tschechischen Forschern Klima, Pokorny und Rosa entworfen wurde.
    Ein Source Code Patch behebt das Problem.

  • 011: SECURITY FIX: 18. März 2003
    Verschiedene SSL und TLS Operations in OpenSSL sind mittels "timing attacks" angreifbar.
    Ein ``RSA blinding'' Source Code Patch behebt das Problem.

  • 010: SECURITY FIX: 5. März 2003
    Ein fix für einen lprm(1) Bug, den es seit 1996 gibt und einen Fehler enthält, der zu privilege escalation führen kann. Für OpenBSD 3.2 ist der Effekt nur begranzt, da hier lprm(1) setuid daemon ist, nicht setuid root.
    Es gibt einen Source Code Patch, der das Problem beseitigt.

  • 009: SECURITY FIX: 3. März 2003
    Ein buffer overflow in der envelope comments Verarbeitung in sendmail(8) könnte einem Angreifer erlauben, root Privilegien zu erlangen.
    Ein Source Code Patch behebt das Problem.

  • 008: SECURITY FIX: 25. Februar 2003
    httpd(8) lässt file inode Nummern via ETag Header fallen, genauso wie auch child PIDs in der multipart MIME boundary Erzeugung. Das kann zum Beispiel zum Ausbeuten von NFS führen, da hier inode Nummern als Teil des Datei-Handle benutzt werden.
    Ein SourceCode Patch behebt beide Probleme.

  • 007: SECURITY FIX: 22. Februar 2003
    In ssl(8) kann durch das Timing beim Berechnen eines MAC ein Informationsleck entstehen, sogar wenn inkorrektes Block Cipher Padding gefunden wurde, ist dies eine Gegenmassnahme. Überprüfe die Zuweisungroutinen auch auf negative Grössen.
    Ein SourceCode Patch behebt beide Probleme.

  • 006: SECURITY FIX: 20. Januar 2003
    Eine doppelte Speicherfreigabe cvs(1) könnte einem Angreifer erlauben, Code mit den Privilegien desjenigen Users auszuführen, der CVS laufen lässt. Das ist nur ein Problem, wenn der ausgeführte cvs-Befehl unter einem anderen User läuft. Das bedeutet, dass das Problem nur bei Konfigurationen wie pserver Client/Server existiert.
    Es gibt einen Source Code Patch, der das Problem beseitigt.

  • 005: SECURITY FIX: 14. November 2002
    Ein buffer overflow im named(8) kann einem Angreifer erlauben, Code mit den Privilegien des named auszuführen. Auf OpenBSD läuft der named als non-root User in einer chrooted Umgebung, was den Effekt dieses Bugs mildert.
    Es gibt einen Source Code Patch, der das Problem beseitigt.

  • 004: RELIABILITY FIX: 6. November 2002
    Ein Logik-Fehler im pool kernel memory allocator kann eine Speicherverletzung bei Situationen mit wenig Speicher zulassen, was zu einem Systemcrash führen kann.
    Es gibt einen Source Code Patch, der das Problem beseitigt.

  • 003: SECURITY FIX: 6. November 2002
    Ein Angreifer kann die Restriktionen von sendmail's restricted shell smrsh(8) umgehen, und beliebige Befehle mit den Privilegien seines eigenens Accounts ausführen.
    Es gibt einen Source Code Patch, der das Problem beseitigt.

  • 002: RELIABILITY FIX: 6. November 2002
    Netzwerk bridges, die pf mit eingeschaltetem scrubbing benutzen, können eine mbuf corruption verursachen, was zu einem Systemcrash führen kann.
    Es gibt einen Source Code Patch, der das Problem beseitigt.

  • 001: SECURITY FIX: 21. Oktober 2002
    Es kann ein buffer overflow im kadmind(8) Daemon auftreten, was zu einem remote crash oder exploit führen.
    Es gibt einen Source Code Patch, der das Problem beseitigt.
