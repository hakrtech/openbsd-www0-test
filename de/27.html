Eine CD bestellen

Um die Dateien dieser Version zu bekommen:
  • Bestellen sie eine CDROM mit unserem Bestellsystem.
  • Für FTP die Informationen auf der FTP-Seite beachten, dort gibts auch eine Liste der FTP-Mirrors.
  • Einfach in das Verzeichnis pub/OpenBSD/2.7/ auf einem der FTP-Mirror gehen.
  • Sorgfältig den Rest dieses Dokumentes lesen.
  • Einen Blick auf Die 2.7 Fehler-und-Probleme-Seite gehen, um eine Liste von Fehlern und deren Behebung zu finden.
  • Siehe auch das detaillierte Log der Änderungen zwischen den 2.6 und 2.7 Versionen.



Hinweis:Alle anwendbaren Copyrights und Danksagungen können in den jeweiligen Sourcefiles in den Dateien src.tar.gz, srcsys.tar.gz, X11.tar.gz gefunden werden, oder auch in den Dateien , die via ports.tar.gz geholt werden. Die Distributionsdateien, die benötigt werden, um die Pakete aus der ports.tar.gz Datei zu erzeugen, sind aus Platzgründen nicht auf den CDROMs enthalten.

Was gibt's neues?

Dies ist eine Liste mit einem Teil der neuen Features und Systemen von OpenBSD 2.7. Eine komplette Liste findet sich im changelog

  • OpenSSH unterstützt jetzt sowohl das SSH1 als auch das SSH2 Protokoll. Das SSH2 Protokoll gewinnt langsam an Akzeptanz. Bis vor kurzem war SSH2 nur in verschiedenen kommerziellen Angeboten zu haben. Der Vorteil von SSH 2 ist, dass es den RSA-Algoritmus vermeidet und stattdessen den frei benutzbaren DSA-Algoritmus verwendet. Der Nachteil ist, dass DSA leider nicht annähernd so schnell ist, insbesondere auf alten Maschinen wird das deutlich. Aber da OpenSSH nach wie vor das SSH1 Protokoll unterstützt (solange die RSA Libraries installiert sind), können unsere User das Beste aus beiden Welten bekommen.

  • Der clevere Trick aus 2.6 bleibt, und erlaubt uns nach wie vor die gleiche CDROM mit Kryptosoftware voller Stärke in alle Welt zu vertreiben, ohne das RSA Patent in den USA zu verletzen. Die nächste Version von OpenBSD wird mit standardmässiger RSA-Unterstützung ausgeliefert, da dessen Patent am 21. September 2000 ausläuft.

  • Extensive Änderungen an der Dokumentation, hauptsächlich an den man pages und den Web FAQ. Die manual pages enthalten nun nützliche Beispiele, die die Erklärungen ergänzen.

  • Eine umfangreichere Sammlung von und besser getestete "ports".

  • Über 500 bereits erzeugte und getestete packages.

  • Das System enthält die folgenden Komponenten von Aussenstehenden (also nicht vom OpenBSD-Projekt):

    • XFree86 3.3.6
    • gcc 2.95.2
    • perl 5.6.0 plus einige Fixes.
    • Apache 1.3.12 + Mod_ssl 2.6.2 + OpenSSL 0.9.5a, DSO
    • ipf 3.3.14
    • groff 1.15
    • sendmail 8.10.1
    • lynx 2.8.2 mit zusätzlicher HTTPS Unterstützung
    • sudo 1.6.3
    • Neueste ncurses Version
    • Letztes KAME IPv6 von Mitte Mai 2000.
    • KTH Kerberos 1.0
    • OpenSSH 2.1

  • Deutlich verbesserte Gerätetreiberunterstützung

    • Unterstützung für Crypto Hardware um die IPSEC Performance zu beschleunigen, und zwar mit PowerCrypt.
    • Unterstützung für alle neuen Adaptec, NCR, Qlogic, und Advansys SCSI Controller.
    • Viele neue EthernetGeräte werden untersützt, einschliesslich SiS900/7016, Adaptec Starfire, 21143, Sundance ST201 (DEC DFE-550TX), etc.
    • Neue USB Ethernet-Geräte werden unterstützt, einschliesslich ADMtek USB, CATC USB, und Kawasaki USB.
    • Ein guter Beginn der cardbus Unterstützung.
    • Signifikante Verbesserungen für pcmcia.
    • Unterstützung für viele weitere drahtlose Netzwerkkarten, z.B. WaveLAN, Aironet, Aviator, etc.
    • Viele Verbesserungen in der USB Unterstützung.
    • Hypersparc Unterstützung für den Sparc port.
    • Unterstützung für die meisten Gigabit Ethernet Karten auf dem Markt, z.B.: SysKonnect, Intel, und Alteon.
    • Unterstützung für die ICP Vortex RAID Controller.
    • Unterstützung für grosse PC's, die mehrere PCI Busse basierend auf dem Intel 451 oder RCC ServerWorks Chipsets benutzen.
    • Neue Audio Treiber inklusive Neomagic, C-Media CMI8x38, und USB Audio.
    • Verbesserungen im PCI IDE Subsystem, inklusive der neuen Unterstützung für den AMD 756.

  • Viele Verbesserungen bezüglich Sicherheit und Zuverlässigkeit (siehe die roten Zeilen in den kompletten Changelogs).

  • Viele neue Features, inklusive:

    • Unterstützung für verschlüsselten Swap Space.
    • Verbesserte Linux Emulation
    • Unterstützung für grosse FAT32 Dateisysteme.
    • Signifikante Verbesserungen im Network Bridging Code.
    • Verbesserte pthreads Unterstützung.
    • Unterstützung für vlan Netzwerke.

Die Installation

Es folgen Instruktionen, die sie auf einem Stück Papier hätten, wenn sie eine CDROM gekauft hätten , statt eine alternative Installation zu versuchen. Die Anleitungen für FTP-Installationen sind recht ähnlich, die Anleitungen für die CDROM-Installationen wurden jedoch intakt gelassen, um zu zeigen, wieviel einfacher alles mit einer CDROM gewesen wäre.

Bitte beachten sie auch die folgen Dateien auf den CDROMs, die noch weitere Details enthalten, um OpenBSD 2.7 auf ihrer Maschine zu installieren:

  • CD1:2.7/i386/INSTALL.i386

  • CD2:2.7/sparc/INSTALL.sparc
  • CD2:2.7/mvme68k/INSTALL.mvme68k
  • CD2:2.7/amiga/INSTALL.amiga
  • CD2:2.7/mac68k/INSTALL.mac68k
  • CD2:2.7/hp300/INSTALL.hp300

  • Die pmax & sun3 Versionen sind nur per FTP zu beziehen, leider nicht auf CDROM.

    • Eine schnelle Installationsinformation für Leute, die mit OpenBSD vertraut sind und den Befehl "disklabel -E" kennen. Wenn sie irgendwie unsicher in Bezug auf die Installation sind, lesen sie die entsprechenden INSTALL.* Dateien, die oben angegeben sind !

    i386:

    Spielen sie mit ihren BIOS-Optionen herum, und versuchen sie von CD zu booten. Wenn das nicht klappt, schreiben sie CD1:2.7/i386/floppy27.fs auf eine Floppy, und nehmen sie die als Boot-Medium. Wenn sie OpenBSD neben anderen Betriebssystemen betreiben wollen, sollten sie unbedingt das Dokument INSTALL.i386 lesen.

    Benutzen sie /2.7/tools/rawrite.exe, um eine Floppy unter MS-DOS zu erstellen. Unter Unix einfach "dd if=<file> of=/dev/<device> bs=32k" nehmen, wobei device "floopy", "rfd0c" oder "rfd0a" sein sollte. Nehmen sie nur eine perfekt formatierte Floppy ohne irgendwelche BAD BLOCKS, oder es wird nicht klappen !

    sparc:

    Um von CD2 zu booten, tippen sie "boot cdrom 2.7/sparc/bsd.rd", oder "b sd(0,6,0)2.7/sparc/bsd.rd" , je nach ihrer ROM Version. Atlernativ können sie CD2:2.7/sparc/floppy27.fs auf eine Floppy schreiben und davon booten, indem sie "boot floppy()" oder "boot fd()" eingeben, je nach ROM Version. Schlußendlich gibt es eine dritte Alternative, und zwar das Schreiben von CD2:2.7/sparc/kc.fs und CD2:2.7/sparc/inst.fs auf zwei verschiedene Floppies. Dann einfach die Flopy mit "kc.fs" nehmen und wie oben beschrieben booten. Sobald das Floppy-Drive die erste Floppy ausspuckt, die mit "inst.fs" einlegen. Die anstehenden Fragen beantworten, und danach mit der "kc.fs" Floppy rebooten. Diesmal, sobald die Floppy ausgeworfen wird, einfach die "kc.fs" wieder "reintun", und weitere Fragen beantworten. :-)

    amiga:

    Erzeugen sie BSD-Partitionen mit Hilfe der Anleitung in INSTALL.amiga. Dann die CD2 unter AmigaOS als Device CD0 mounten. Als nächstes folgendes CLI-Kommando eingeben : "CD0:2.7/amiga/utils/loadbsd CD0:2.7/amiga/bsd.rd".

    hp300:

    Booten sie gemäß den Anweisungen in INSTALL.hp300 über das Netzwerk.

    mac68k:

    Booten sie ganz normal das MacOS und partitionieren sie mit Hilfe der passenden A/UX Konfigurationen. Danach packen sie die Macside Utilities von CD1:2.7/mac68k/utils auf ihre Festplatte. Als nächstes lassen sie Mkfs laufen, um die Dateissysteme zu erzeugen, die sie mit Hilfe von A/UX gerade gemacht haben. Danach benutzen sie den BSD/Mac68k Installer um alle Teile aus CD1:2.7/mac68k/ auf ihre Partition zu kopieren. Schlussendlich sind sie bereit, den BSD/Mac68k Booter mit dem genauen Ort des Kernels zu konfigurieren und können das System booten.

    sun3:

    Hol' dir den Release via ftp. Dann kannst du entweder einen "diskless boot" aufsetzen oder ein Installationstape machen, so wie in INSTALL.sun3 beschrieben.

    Hinweise zum Quellcode:

    src.tar.gz enthält eine Archiv mit Quellcode, daß in /usr/src liegen sollte. Diese Datei enthält alles, was man braucht, bis auf die Kernel-Quellen, die sich in einer seperaten Datei befinden. Um den Source zu entpacken einfach : 

    # mkdir -p /usr/src
# cd /usr/src
# tar xvfz /tmp/src.tar.gz
    eingeben.

    srcsys.tar.gz enthält ein Quellcode-Archiv, das in /usr/src/sys beginnt. Diese Datei enthält alle Kernel-Quellen, die man braucht, um einen neuen Kernel zu erzeugen. 

    # mkdir -p /usr/src/sys
# cd /usr/src
# tar xvfz /tmp/srcsys.tar.gz

    Beide Archive enthalten einen regulären CVS checkout. Wenn man diese Dateibäume benutzt, ist es möglich, die anoncvs-Server wie auf http://www.OpenBSD.org/anoncvs.html zu benutzen. Wenn man diese Dateien benutzt, hat man ein viel schnelleres erstes Update, als wenn man einen frischen, eigenen OpenBSD QuellBaum (source tree) mit Hilfe von CVS baut.

    Ports tree

    Es gibt auch ein ports tree Archiv. Um es zu entpacken einfach : 

    # cd /usr
# tar xvfz /tmp/ports.tar.gz
# cd ports
# ls
...
    eingeben.

    Das ports/ Unterverzeichnis ist ein Kontrollpunkt für den OpenBSD ports tree. Lesen sie am besten http://www.OpenBSD.org/ports.html wenn sie noch nichts über ports wissen. Dieser Text ist keine Anleitung, wie man die ports benutzt, sondern nur dazu da, um den Leuten den Einstieg in das OpenBSD ports system zu ermöglichen.

    Wie man sehen kann ist das OpenBSD ports system nicht vollständig. Das liegt daran, dass die volle Integration von ports in OpenBSD noch ein sehr junges Projekt ist. Wir glauben, dass die bereitgestellten ports stabil sind, aber es ist sehr wichtig zu verstehen, dass die ports die Funktionalität in Zukunft noch stärker beeinflussen werden.

    Wie gerade festgestellt, werden die ports in Zukunft stark wachsen. Das ports/ - Verzeichnis hat ein CVS checkout integriert (Lesen sie im Zweifel die manpage von cvs(1) ). So wie unser gesamter Quellcode, gibt es auch den ports tree per CVS. Um also auf einem aktuellenStand zu bleiben, müssen sie den ports tree auf einem Schreib-Lese-Medium (Festplatte, ZIP,CDRW,usw.) mit einem Kommando wie dem folgenden updaten: 

    # cd [portsdir]/; cvs -d anoncvsserver.openbsd.org:/cvs update -Pd

    [Natürlich müssen sie sowohl das lokale Vezeichnis, als auch den ServerNamen mit dem Ort ihres ports tree und ihres bevorzugten anoncvs-Servers ersetzen.]

    Bei Problemen wie immer einfach unsere Webpages kontaktieren, da das ganze eben noch keine vollständig fertige Sache ist.

    Schlussendlich ist Hilfe trotzdem nie weit. Wenn sie gerne einen weiteren Port sehen wollen, selber dabei helfen wollen, oder einfach nur mehr darüber erfahren wollen, abonnieren sie einfach die Mailingliste ports@openbsd.org
