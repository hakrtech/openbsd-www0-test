Geschichte:

Dem Motorola 88k Prozessor wird nachgesagt der beste RISC Prozessor zu sein, der jemals erdacht wurde. Seine kombinierte Einfachheit und Eleganz machen den mvme88k zu einer aufrichtigen, robusten Plattform.

Nivas Madhur hat den ersten mvme88k port für die MVME187 Karte angefangen, aber hat seitdem den Arbeitgegeber gewechselt. Steve Murphree, Jr. hat dann den port im November 1998 komplettiert. Der port hat seitdem einige grosse Änderungen erfahren, wie etwa ein erneuertes autoconf und on-board SCSI Treiber. Neue,stark erweiterte VME Bus Unterstützung. Ein funktionierender install process der einen funktionierenden VID Block auf der Platte erzeugt.

Momentaner Status:

Die Leute, die am OpenBSD/mvme88k port arbeiten sind zur Zeit: Steve Murphree, Jr.
Mehr wären echt nett :)

Email kann an den Verantwortlichen des OpenBSD/mvme88k port geschickt werden: smurph@OpenBSD.org.

Seit Version 2.4 gibt es einen snapshot. Der port unterstützt den MVME187 Single Board Computer (SBC). MVME188 Multi-Prozessor board Unterstützung ist "in Arbeit." Unterstützung für den MVME197 ist geplant, aber es braucht Zeit bis die Dinge laufen. Die Installations-Tools und der Prozess funktionieren seit der Version 2.5. OpenBSD/mvme88k kann via tape ramdisk images oder auch über Netzwerk und diskless Installationen installiert oder upgegraded werden.

  • Snapshots werden von Zeit zu Zeit verfügbar gemacht.

Liste der unterstützten Hardware:

OpenBSD/mvme88k läuft auf folgenden Klassen von Maschinen:

  • MVME187: Motorola 88100 basierte Single Board Computer (SBC)

Unterstützte Geräte

  • Floppy Laufwerke:
    • SCSI floppy disk Laufwerke
  • Serielle Ports:
    • tty00, tty01, tty02 und tty03 on-board Cirrus Lodgic serielle Ports (MVME187)
    • MVME332 8-port serielle Karten
  • Ethernet Adapter:
    • on-board Intel Ethernet (MVME187)
    • MVME376 AMD Lance Ethernet.
  • SCSI Controller:
    • on-board "NCR" SCSI Controller (MVME187)
    • MVME328 SCSI Controller

OpenBSD/mvme88k läuft *nicht* auf diesen Maschinen (noch nicht):

  • MVME188: Die Unterstützung wird gerade in den Source Code eingefügt!
  • MVME197: Ich habe ein Board, aber noch keine Zeit. Bald, vielleicht..

Nicht unterstützte Geräte. Es gibt noch viel mehr...

  • Disk Controller:
    • MVME327 SCSI Controller
  • Ethernet Controller:
    • MVME374 LANCE Controller
  • WAN Controller:
    • MVME333 WAN Controller
