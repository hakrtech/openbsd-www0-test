Wie man die Dateien dieser Version bekommt:
  • Bestelle einfach eine CDROM über unser Bestellsystem.
  • Informationen über mirror Maschinen gibt es auf unsere FTP Seite.
  • Gehe in das pub/OpenBSD/3.0/ Verzeichnis auf einer der mirror sites.
  • Lies den Rest dieses Dokumentes sorgfältig.
  • Wirf einen Blick auf die 3.0 Errata Seite auf der sich gegebenfalls eine Liste von Bugs findet, und wie man sie umgeht oder loswird.
  • Wirf einen Blick auf das detaillierte Logfile der Änderungen zwischen den Versionen 2.9 und 3.0.



Hinweis: Alle anwendbaren Copyrights und Danksagungen können in den passenden Quellcodedateien in src.tar.gz, srcsys.tar.gz, XF4.tar.gz, oder den Dateien gefunden werden, die via ports.tar.gz geholt wurden. Die Distributionsdateien die man braucht um mit Hilfe der ports.tar.gz Datei packages zu bauen sind aus Platzgründen nicht mit auf den CDROMs.

Was gibt es Neues?

Dies ist ein Liste eines Teils der neuen Features und Systeme, die Teil von OpenBSD 3.0 sind. Eine umfassende Liste gibt es im changelog der Änderungen, die zu 3.0 führten.

  • OpenSSH (sowohl das SSH1 als auch das SSH2 Protokoll werden unterstützt) ist jetzt bei Version 3.0. Sichere Dateitransfers sollten jetzt mit dem stark verbesserten SFTP Subsystem gemacht werden, das sowohl einen SFTP Server als auch einen Client enthält.

  • Ausführliche Änderungen an der Dokumentation, insbesondere den man pages und den Web FAQ. Die manual pages enthalten hilfreiche Beispiele in Ergänzung zu den Erklärungen.

  • Umfangreichere und besser getestete Sammlung von "ports".

  • Über 1000 fertig erzeugte und getestete packages.

  • Ein neuer OpenBSD/sparc64 port für UltraSPARC Hardware.

  • Ein neuer Paketfilter, PF, der NAT Fähigkeiten enthält, mit einer fast ipf-kompatiblen Syntax.

  • Das System beinhaltet die folgenden großen Komponenten von Außenstehenden:

    • XFree86 4.1.0 (und i386 enthält auch 3.3.X Server, damit Unterstützung für alle Chipsets gewährleistet ist)
    • gcc 2.95.3 (+ Patches)
    • perl 5.6.1 (+ Patches)
    • Apache 1.3.19, mod_ssl 2.8.2, OpenSSL 0.9.6b (+ Patches), DSO Unterstützung
    • groff 1.15
    • sendmail 8.12.1
    • lynx 2.8.2 mit zusätzlicher HTTPS Unterstützung
    • sudo 1.6.3p7
    • ncurses 5.2
    • Neuestes KAME IPv6
    • KTH Kerberos 1.0.8
    • Heimdal 0.3f (+ Patches)
    • OpenSSH 3.0

  • Viele Verbesserungen für Sicherheit und Stabilität (siehe die roten Einträge im complete changelog).

Installationsanleitung

Es folgen Instruktionen, die du in gedruckter Form haben würdest, wenn du anstatt einer alternativen Installationsmethode eine CDROM gekauft hättest. Die Anleitungen für FTP-Installationen sind recht ähnlich, die Anleitungen für die CDROM-Installationen wurden jedoch intakt gelassen, um zu zeigen, um wieviel einfacher alles mit einer CDROM gewesen wäre.

Bitte beachte auch die folgenden Dateien auf den CDROMs, die noch weitere Details enthalten, um OpenBSD 3.0 auf deiner Maschine zu installieren:

  • CD1:3.0/i386/INSTALL.i386
  • CD1:3.0/alpha/INSTALL.alpha

  • CD2:3.0/macppc/INSTALL.macppc
  • CD2:3.0/mvme68k/INSTALL.mvme68k
  • CD2:3.0/amiga/INSTALL.amiga
  • CD2:3.0/mac68k/INSTALL.mac68k
  • CD2:3.0/hp300/INSTALL.hp300
  • CD2:3.0/vax/INSTALL.vax

  • CD3:3.0/sparc/INSTALL.sparc
  • CD3:3.0/sparc64/INSTALL.sparc64

Hier gibt es kurze Installationsinformationen für Leute, die mit OpenBSD vertraut sind und den Befehl "disklabel -E" kennen. Wenn du irgendwie unsicher in Bezug auf die Installation bist, lies die entsprechenden INSTALL.* Dateien, die oben angegeben sind!

OpenBSD/i386:

    Spiele mit den BIOS-Optionen herum und versuche, von CD zu booten. Die i386 Version von OpenBSD ist auf der CD1. Wenn dein BIOS kein Booten von CD unterstützt, schreibe CD1:3.0/i386/floppy30.fs auf eine Floppy, und nimm diese als Boot-Medium.

    Benutze die CD1:3.0/i386/floppyB30.fs für mehr Unterstützung an SCSI Controllern, oder CD1:3.0/i386/floppyC30.fs für bessere Laptopunterstützung.

    Wenn du OpenBSD neben anderen Betriebssystemen betreiben willst, solltest du unbedingt das Dokument INSTALL.i386 lesen.

    Benutze das "rawrite" Werkzeug von CD:/3.0/tools/rawrite.exe um eine Floppy unter MS-DOS zu erstellen. Unter Unix einfach das dd(1) Utility benutzen. Das folgende Beispiel für dd(1) könnte "floppy", "rfd0c", oder "rfd0a" als Device benutzen. 

      # dd if=<file> of=/dev/<device> bs=32k

    Benutze ausschließlich perfekte und sauber formatierte Floppies OHNE DEFEKTE BLÖCKE oder deine Installation wird wahrscheinlich nicht klappen. Mehr Informationen über die Installation von OpenBSD/i386 gibts in der FAQ4.1.

OpenBSD/alpha:

    Am SRM prompt gibst du boot -fi 3.0/alpha/bsd.rd dka6 ein, wobei dka6 der Kurzname für das CDROM-Laufwerk ist (prüfe das mit show dev).

    Wenn du nicht von CDROM booten kannst, schreibe CD1:3.0/alpha/floppy30.fs oder CD1:3.0/alpha/floppyB30.fs (abhängig von deiner Maschine) auf eine Diskette und gib boot dva0 ein. In INSTALL.alpha gibt es noch weitere Details.

    Benutze ausschließlich perfekte und sauber formatierte Floppies OHNE DEFEKTE BLÖCKE oder deine Installation wird wahrscheinlich nicht klappen.

OpenBSD/macppc:

    Lege die CD2 in dein CDROM Laufwerk und drücke die C Taste während des Einschaltens, bis schließlich die Anzeige OpenBSD/macppc boot erscheint.

    Alternativ kannst du am Open Firmware prompt boot cd:,ofwboot /3.0/macppc/bsd.rd eingeben.

    Aufgrund eines Fehlers bei der CD-Produktion für die Version 3.0 sollte das folgende modifizierte Kommando benutzt werden: boot cd:,OFWBOOT /3.0/macppc/bsd.rd. Dieser Befehl funktioniert vielleicht nicht auf einigen Maschinen aus der neuen Welt, iMac (Rev A - Rev C); auf diesen Maschinen muss man dann die Dateien ofwboot und /3.0/macppc/bsd.rd auf die erste HFS Partition auf der Festplatte kopieren und mittels boot hd:,OFWBOOT /bsd.rd booten.

OpenBSD/hp300:

    Lege die CD2 in dein CDROM Laufwerk und drücke die Leertaste während des poweron self-test. Wähle die boot Möglichkeit, die mit SYS_CDBOOT übereinstimmt.

    Alternativ kannst du über Netzwerk booten, indem du den Anweisungen in INSTALL.hp300 folgst.

OpenBSD/amiga:

    Erzeuge BSD Partitionen gemäß der Vorbereitungssektion in INSTALL.amiga. Mounte die CD2 unter AmigaOS als Device CD0: Danach führe folgendes CLI-Kommando aus: "CD0:3.0/amiga/utils/loadbsd CD0:3.0/amiga/bsd.rd".

OpenBSD/mac68k:

    Boote MacOS ganz normal und partitioniere deine Festplatte mit den passenden A/UX Konfigurationen. Dann packe die Macside Utilities aus dem Verzeichnis CD2:3.0/mac68k/utils aus und speichere sie auf deiner Festplatte. Benutze dann Mkfs, um die Dateisysteme auf den frisch eingerichteten A/UX Partitionen zu erzeugen. Den BSD/Mac68k Installer benutzt du danach, um alle Sets aus CD2:3.0/mac68k/ auf deine Partitionen zu kopieren. Und zum guten Schluss bist du nun soweit, um den BSD/Mac68k Booter mit dem Ort zu konfigurieren, an dem sich dein Kernel befindet und das System zu rebooten.

OpenBSD/mvme68k:

    Du kannst entweder ein bootbares Band erzeugen oder über Netzwerk booten.
    Der Netzwerkboot benötigt eine MVME68K BUG Version, die die NIOT und NBO Debugger kommandos unterstützt. Folge den Anweisungen in INSTALL.mvme68k wenn du mehr Details haben willst.

OpenBSD/vax:

    Boote über das Netzwerk via mopbooting, so wie es in INSTALL.vax beschrieben ist.

OpenBSD/sparc:

    Version 3.0 von OpenBSD/sparc befindet sich auf der CD3. Um von dieser CD zu booten, kannst du eines von den zwei Kommandos benutzen, die weiter unten aufgeführt sind, je nachdem welche ROM Version du hast. 

      > boot cdrom 3.0/sparc/bsd.rd
or
> b sd(0,6,0)3.0/sparc/bsd.rd

    Wenn deine sparc kein CD Laufwerk hat, kannst du alternativ dazu auch von einer Floppy booten. Dazu musst du die Datei "CD3:3.0/sparc/floppy30.fs" auf eine Floppy schreiben. Mehr Informationen gibts es in der FAQ4.1. Um von dieser Floppy zu booten, kannst du eines von den zwei Kommandos benutzen, die weiter unten aufgeführt sind, je nachdem welche ROM Version du hast. 

      > boot floppy
oder
> boot fd()

    Stelle sicher, dass du eine sauber formatierte Floppy OHNE DEFEKTE BLÖCKE benutzt, ansonsten wird deine Installation mit hoher Wahrscheinlichkeit nicht klappen.

    Wenn deine sparc weder Floppy noch CD-Laufwerk hat, kannst du entweder ein bootbares Band erzeugen, oder via Netzwerk installieren, indem du den Anweisungen in der Datei INSTALL.sparc folgst.

OpenBSD/sparc64:

    Lege die CD3 in dein CDROM Laufwerk und gib boot cdrom ein.

    Wenn das nicht klappt oder du kein CDROM-Laufwerk hast, kannst du die Datei CD3:3.0/sparc64/floppy30.fs auf eine Floppy schreiben und sie mit boot floppy booten.
    Stelle sicher, dass du eine sauber formatierte Floppy OHNE DEFEKTE BLÖCKE benutzt, ansonsten wird deine Installation mit hoher Wahrscheinlichkeit nicht klappen.

    Du kannst auch CD3:3.0/sparc64/miniroot30.fs auf die Swap-Partition auf der Platte schreiben und mit boot disk:b booten.

    Wenn nichts davon klappt, kannst du auch vom Netz booten, wie es in INSTALL.sparc64 beschrieben ist.

Hinweise zum source code:

    src.tar.gz enthält ein Source-Code-Archiv, das in /usr/src beginnt. Diese Datei enthält bis auf die Kernel-Sourcen, die sich in einem anderen Archiv befinden, alles, was du brauchst. Um es zu entpacken: 

      # mkdir -p /usr/src
# cd /usr/src
# tar xvfz /tmp/src.tar.gz

    srcsys.tar.gz enthält ein Quellcode-Archiv, das in /usr/src/sys beginnt. Diese Datei enthält alle Kernel-Quellen, die man benötigt, um einen neuen Kernel zu erzeugen. Um sie zu entpacken: 

      # mkdir -p /usr/src/sys
# cd /usr/src
# tar xvfz /tmp/srcsys.tar.gz

    Beide Archive enthalten einen regulären CVS checkout. Wenn man diese Dateibäume benutzt, ist es möglich, die anoncvs-Server wie auf http://www.OpenBSD.org/anoncvs.html beschrieben zu benutzen. Wenn man diese Dateien benutzt, hat man ein viel schnelleres erstes Update, als wenn man einen frischen, eigenen OpenBSD Quellbaum (source tree) mit Hilfe von CVS holt.

Ports Tree

Mitgeliefert wird auch ein ports tree Archiv. Um es zu entpacken: 

    # cd /usr
# tar xvfz /tmp/ports.tar.gz
# cd ports

Das ports/ Unterverzeichnis ist ein checkout für den OpenBSD ports tree. Lies am besten http://www.OpenBSD.org/ports.html wenn du noch nichts über ports weisst. Dieser Text ist keine Anleitung, wie man die ports benutzt, sondern nur dazu da, um den Leuten den Einstieg in das OpenBSD ports System zu ermöglichen.

Natürlich ist das OpenBSD ports System nicht vollständig. Es ist zu bezweifeln, dass es das jemals sein wird. Trotzdem wächst es sehr schnell und wird stabiler. Fast alle ports, die ausgeliefert werden, sollten ohne Probleme auf den meisten Architekturen kompilierbar sein (auf z.B. i386 konnten über 1200 packages ohne Probleme erzeugt werden).

Das ports/ Verzeichnis stellt ein CVS (siehe in der manpage für cvs(1) nach, wenn du mit CVS nicht vertraut bist) checkout unserer ports dar. Wie unser kompletter source tree ist auch auch unser ports tree via anoncvs verfügbar. Um ihn also auf dem Laufenden zu halten, musst du den ports/ tree auf einem schreibbaren Medium haben und den tree mit diesem Kommando updaten: 

    # cd [portsdir]/; cvs -d anoncvsserver.openbsd.org:/cvs update -Pd -rOPENBSD_3_0

[Natürlich musst du die Namen für Server und [portsdir] mit deinen lokalen Verzeichnis, bzw. dem Namen eines anoncvs-Servers in der Nähe austauschen.]

Denk daran, dass die meisten ports via ftp als fertige packages erhältlich sind. Upgedatete packages für 3.0 werden verfügbar gemacht, sobald Probleme bekannt sind.

Falls du gerne einen weiteren port sehen willst, mithelfen willst, oder einfach mehr erfahren willst, solltest du die Mailingliste ports@openbsd.org kennenlernen.
