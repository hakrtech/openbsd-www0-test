OpenBSD/i386

OpenBSD/i386 läuft auf Standard PC's und Klonen davon, mit einer grossen Breite an unterstützten Prozessoren, I/O Bus-Architekturen und Peripheriegeräten.

Inhaltsverzeichnis

  • Geschichte des port
  • Aktueller Status
  • Unterstützte Hardware
  • Installation neben anderen Betriebssystemen
  • Wo man OpenBSD/i386 herbekommt und wie man es installiert

Geschichte:

Der OpenBSD/i386 port war von Anfang an Teil des Projektes. Über die Jahre wurde mehr und mehr Hardware-Unterstützung speziell für OpenBSD geschrieben (so wie die Treiber für die Kryptographie-Beschleuniger), oder von FreeBSD und NetBSD angepasst.

Aktueller Status:

Im Normalfall kann OpenBSD mit minimalen Schwierigkeiten auf aktueller Hardware installiert und benutzt werden. Maschinen die ein spezielles BIOS benutzen (wie etwa ältere propietäre PC's, Laptops und spezielle Servermaschinen) können Probleme verursachen.

Durch eine Binär-Kompatibilitäts-Schicht ist OpenBSD/i386 in der Lage Linux, FreeBSD, BSD/OS, SVR4 (einschließlich Solaris) und IBCS2 Binaries bis zu einem gewissen Punkt auszuführen. Die meisten Anwendungen werden problemlos laufen.

Unterstützung für XFree86 gibt es für die meisten Grafik-Karten, meist mit einem aktuellen XFree86 (bei OpenBSD 3.6 ist das 4.4.0), ansonsten mit XFree86 3.3.6 Servern.

Unterstützte Hardware:

Die meisten Laptops werden unterstützt. Wir haben die Details einiger bekannter Laptops auf einer speziellen Seite dargestellt.

Die Liste der unterstützten Hardware bezieht sich auf OpenBSD-current. Sie weicht leicht von der ab, die für die letzte veröffentlichte Version gilt. Prozessoren

Alle CPU Chips, die kompatibel mit der Intel 80386 (i386) Architektur sind werden unterstützt:

  • 80386 (SX/DX) with 80387 (SX/DX) FPU
  • 80486 (DX/DX2/DX4)
  • Intel Pentium/Pentium-MMX
  • Intel Pentium Pro/II/III/Celeron/Xeon
  • Intel Pentium 4
  • AMD 5x86
  • AMD K5/K6/K6-2/K6-3
  • AMD Athlon/Duron
  • AMD Athlon 64 (Siehe auch OpenBSD/amd64 für 64-bit Unterstützung)
  • Cyrix MediaGX/M1/M2
  • Cyrix 6x86
  • VIA C3
  • Rise mP6
  • IDT WinChip and C3
  • NexGen 586
  • NS Geode GX1 and M1
  • Transmeta TMS3200, TMS5400, TMS5600

Reguläre Fliesskomma-Coprozessoren (80387DX, 80387SX, 80487SX) werden benötigt, wenn sie nicht im Prozessor eingebaut sind.

Alles, was ein Klon des i386 oder höher ist, sollte funktionieren. Die einzig bekannte CPU, die sehr schlecht funktioniert (aufgrund schlechter Motherboard Designs), ist der Cyrix 386DLC.

SMP Unterstützung

Beginnend mit OpenBSD 3.6 unterstützt OpenBSD/i386 SMP (Symmetrical MultiProcessor) Systeme. Um SMP Operationen zu unterstützen enthält das Installations-Datei-Set einen separaten SMP kernel (bsd.mp).

Da dies ein neues Feature ist, sind die Entwickler daran interessiert zu sehen, welche Hardware mit SMP funktioniert und welche nicht, es ist also noch wichtiger nach der Installation ein demsg einzusenden.

  • Alle Standard ISA-, EISA-, VLB-, und PCI-Bus basierte Maschinen, inklusive:
    • Intel 450GX/KX basierter Maschinen.
    • Intel 450NX basierter Maschinen.

  • 32-Bit CardBus PC Karten.
  • Universal Serial Bus (USB).

    Installation neben anderen Betriebssystemen:

    Der OpenBSD/i386 port kann so installiert werden, dass er sich die Festplatten mit anderen Betriebssystemen teilt, wie etwa MS-DOS, Windows oder Linux, wobei man das MSDOS MBR-/Partitionsschema und einen optionalen Boot-Selektor benutzt. Die tägliche Arbeit verläuft ohne Probleme, aber das Setup ist anspuchsvoll und die Windows '95 Installation ist dafür bekannt, sich nicht um bereits existierende MBR/Partitionsschemata zu kümmern.

    Einige Laptop Computer benutzen eine spezielle Partition als suspend-to-disk Speicher, an dem man nichts verändern sollte.

    Im Installation Guide und im Disk Setup Manual gibt's dazu mehr Details.

    Wo man OpenBSD/i386 herbekommt und wie man es installiert:

    Die aktuellste Version von OpenBSD/i386 ist OpenBSD 3.6. Hier ist die OpenBSD/i386 3.6 Installationsanleitung .

    Snapshots werden von Zeit zu Zeit hier genauso wie auf einigen mirrors zur Verfügung gestellt. Hier ist die dazu passende OpenBSD/i386 snapshot Installationsanleitung.

    Es werden mehrere Installationsmethoden angeboten:

    • CD boot

      Beim booten von CD wird ein El Torito 2.88MB Floppy Image geladen, dass fast alle OpenBSD Treiber enthält. Das beinhaltet auch minimale USB-Unterstützung (umass und ukbd Geräte).
      Die neueste Liste von Treibern für dieses Image findet sich im RAMDISK_CD Config File.

    • Floppy A (floppy36.fs) Dieses 1.44MB Floppy Image enhält die gebräuchlichsten Treiber. Es ist dazu bestimmt, auf dem typischen PC zu laufen. Als generelle Regel gilt dabei, dass solche Treiber, die viel Microcode benutzen müssen, nicht dabei sind, genauso wie seltene SCSI Treiber, Gigabit Ethernet Karten, oder RAID Geräte.
      Die neueste Liste von Treibern für dieses Image findet sich im RAMDISK Config File.

    • Floppy B (floppyB36.fs) Dieses Medium soll den Leuten helfen, moderne Serversystem aufsetzen. Es enthält Unterstützugn für die meisten SCSI und RAID Treiber, Gigabit Ethernet Karten, und so weiter. Aufgrund von Platzmangel fehlen einige Treiber. Insbesondere die Unterstützung für die meisten PCMCIA Geräte und wirklich alte Hardware fehlt.
      Die neueste Liste von Treibern für dieses Image findet sich im RAMDISKB Config File.

    • Floppy C (floppyC36.fs) Dieses 1.44MB Floppy Image enthält viele CardBus und PCMCIA Gerätetreiber. Dieses Medium ist zum Installieren von Laptops gedacht. In den meisten Fällen wird auch Floppy A funktionieren, aber einige ungewöhnliche Laptops könnten von dieser Floppy profitieren.
      Die neueste Liste von Treibern für dieses Image findet sich im RAMDISK_CD Config File.

    Verfahre einfach nach der für dich besten Methode.
