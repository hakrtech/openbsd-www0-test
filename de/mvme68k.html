OpenBSD/mvme68k

Geschichte und Status:

Dieser port wurde hautpsächlich von Theo de Raadt im Jahr 1995 erstellt und zwar als Folge eines Vertrages mit Willowglen Singapore. Ein früherer port auf die MVME147 von Chuck Cranor, der auf Paul Mackerras' altem DA30 Code basierte (und Hardware benutzte, die von Jonathan Levine auf Theo's Anfrage gespendet wurde) war eine stabile Entwicklungsplattform. Bizarrerweise schrieb Dale Rahn von Motorola unabhängig davon auch einen port für die MVME147. Für die meisten Kernel-Patches wurde der Code von beiden untersucht, aber der meiste Code wurde von Theo neu geschrieben, oder basierte auf dem hp300 Code. Dale hat sehr bei der Portierung auf die 68040 Modelle geholfen und hat den meisten systemspezifischen Code für das MVME167 Modell geschrieben. Steve Murphee hat die Arbeit fortgeführt und den MVME177 zum Laufen gebracht, und ausserdem Unterstützung für einige VME Geräte hinzugefügt.

Die Leute, die am meisten an OpenBSD/mvme68k arbeiten, bestehen zur Zeit aus Steve Murphee, Theo de Raadt, und Dale Rahn. Natürlich sind weitere Entwickler jederzeit willkommen !

Der selbe Kernel läuft zur Zeit auf den folgenden Hardwareplattformen:

  • MVME147: 68030
    Arbeitet stabil.
    • Alles bis auf den Parallel Port wird unterstützt.
    • Ich habe einen Treiber für den Parallel Port, aber er ist noch nicht eingefügt.

  • MVME162: 68040
    Arbeitet stabil.
    • Alles wichtige funktioniert. Dazu ein paar Hinweise:
    • SCSI funktioniert (4.5MB/sec von einer 4.2G Quantum Grand Prix).
    • Der VME Bus ist noch nicht getestet, läuft aber
    • Parity und ECC funktionieren nicht
    • Flash Treiber ebenso nicht
    • IP Modul Treiber ist noch nicht getestet

  • MVME165: 68040
    Das bisschen, was wir darüber wissen, sagt uns, das es funktionieren kann.

  • MVME166: 68040
    Funktioniert schon. Möchte das irgendjemand testen ?

  • MVME167C: 68040
    Läuft stabil.
    • Arbeitet so stabil wie der MVME162 port oben, zusätzlich aber dieser Hinweis:
    • Der Parallel Port läuft noch nicht.

  • MVME177-001: 68060
    Läuft stabil.
    • Funktioniert genauso gut wie der MVME167 port.

Zusätzliche unterstützte VME Geräte:

  • MVME376: VME Lance Ethernet
  • MVME328: SCSI Kontroller

Auf all diesen Architekturen werden sowohl Booten ohne Festplatte mit Hilfe von Sun-Style Bootparametern/nfs, als auch normales Booten von einer Festplatte unterstützt.

Hinweis: Dieser port hat COMPAT_SUNOS Unterstützung, so dass SunOS sun3 binaries ausgeführt werden können. Daher denke ich, dass die MVME177 wahrscheinlich die schnellste Maschine ist, die in der Lage ist SunOS sun3 Binärdateien auszuführen....

Snapshots werden von Zeit zu Zeit erstellt.
