OpenBSD/mvme68k

OpenBSD/mvme68k läuft auf einer grossen Anzahl von Motorola's 680x0-basierenden VME Motherboards.

Es gibt zur Zeit keinen Verantwortlichen für den mvme68k port.

Geschichte:

Dieser port wurde hautpsächlich von Theo de Raadt im Jahr 1995 erstellt und zwar als Folge eines Vertrages mit Willowglen Singapore. Ein früherer port auf die MVME147 von Chuck Cranor, der auf Paul Mackerras' altem DA30 Code basierte (und Hardware benutzte, die von Jonathan Levine auf Theo's Anfrage gespendet wurde) war eine stabile Entwicklungsplattform.

Bizarrerweise schrieb Dale Rahn, der zu diesem Zeitpunkt für Motorola gearbeitet hat, vollkommen unabhängig davon auch einen port auf den MVME147. Für die meisten Teile des Kernel wurden beide ports analysiert, es wurde aber mehr Code von Theo von Grund auf geschrieben oder basierend auf dem hp300 Code.

Dale hat während des portierens auf die 68040 Modelle sehr geholfen und den meisten für das MVME167 Modell spezifischen Code geschrieben. Später setzte Steve Murphree die Arbeit fort und brachte den MVME177 zum Laufen, und fügte Unterstützung für einige VME-Geräte hinzu.

Aktueller Status:

Momentan booten alle Boards aus der Liste der unterstützten Hardware in den Multi-User-Modus und unterstützen genügend on-Board-Geräte, um brauchbar zu sein.

OpenBSD/mvme68k ist in der Lage sun3 SunOS Binaries mittels der COMPAT_SUNOS Kernel Option laufen zu lassen. Somit ist das MVME177 Board vermutlich die schnellste Maschine, mit der man SunOS m68k Binaries laufen lassen kann!

Da keine der mvme68k Boards Grafikkarten oder -Chips haben, und keine der Motorola VME Frame Buffer unterstützt werden, gibt es keine X Window System Server. Trotzdem ist ein vollständiger Satz von X Clients und Utilities verfügbar, was OpenBSD/mvme68k Maschinen erlaubt als X11 Font Server zu dienen, oder X Clients auf entfernten Displays zu bedienen.

Projekt Liste (ohne besondere Reihenfolge):

  • Unterstützung von MVME172.
  • Unterstützung von MVME135 und MVME136.
  • Unterstützung von MVME165.
  • Arbeiten an noch nicht unterstützten Gerätekarten (MVME327, MVME374...)

Unterstützte Hardware:

Unterstützte Prozessor Boards

  • MVME147 (68030)
    Alle on-board Devices werden unterstützt, mit Ausnahme des Paralell-Ports.
  • MVME162 (68040)
    Fast alle on-board Geräte werden unterstützt, mit den folgenden Ausnahmen:
    • Unterstützung für den VME Bus ist nicht getestet.
    • Parity und ECC Speicher werden nicht unterstützt (aber der Speicher selbst funktioniert prima!)
    • Der Flash Driver funktioniert nicht
    • Der IP Module Treiber ist nicht getestet

  • MVME167 (68040)
    Funktioniert ebenso wie der MVME162 und der Parallel Port wird nicht unterstützt
  • MVME177 (68060)
    Funktioniert genauso wie der MVME167.

Andere Modelle funktionieren möglicherweise auch schon (MVME165, MVME166, z.B.).

Unterstützte Erweiterungs-Boards

  • MVME328: SCSI Controller
  • MVME376: VME Lance Ethernet
