Alle Architekturen

  • 022: SECURITY FIX: 5. Mai 2004
    Es wurden Probleme bei der Überprüfung von Pfadnamen in cvs(1) gefunden, die es bösartigen Clients erlauben, Dateien ausserhalb des Repository zu erzeugen, es bösartigen Servern erlauben, Dateien ausserhalb des lokalen CVS Baums auf dem Client zu überschreiben und es ausserdem Clients erlauben, Dateien von ausserhalb des CVS Repository zu laden.
    Es existiert ein Quellcode-Patch, der das Problem behebt.

  • 021: RELIABILITY FIX: 17. März 2004
    Eine fehlende Überprüfung auf eine NULL-pointer dereference hin wurde in ssl(3) gefunden. Ein Angreifer kann von ausserhalb den Bug dazu ausnutzen, eine OpenSSL Applikation zum Absturz zu bringen, das kann zu einem Denial of Service führen.
    Es existiert ein Quellcode-Patch, der das Problem behebt.

  • 020: RELIABILITY FIX: 17. März 2004
    Defekte in der Üprüfung der Payload und im Verarbeiten von Funktionen von isakmpd(8) wurden entdeckt. Eine Angreifer könnte verformte ISAKMP Nachrichten senden und damit den isakmpd zum Absturz oder in eine Endlos-Schleife zwingen. Dieser Patch beseitigt diese Probleme und einige Speicherlecks.
    Es existiert ein Quellcode-Patch, der das Problem behebt.

  • 019: SECURITY FIX: 13. März 2004
    Wegen eines Fehlers im Parsing der Allow/Deny Regeln von httpd(8)'s Access Modul führt das Verwenden von IP Addresses ohne Netzmaske auf Big Endian 64-bit Plattformen dazu, dass die Regeln nicht zutreffen. Das betrifft nur sparc64.
    Es existiert ein Quellcode-Patch, der das Problem behebt.

  • 018: RELIABILITY FIX: 8. März 2004
    OpenBSD's TCP/IP Stack hat keine Begrenzung dahin gesetzt, wieviele out-of-order TCP segments im System verwaltet werden dürfen. Ein Angreifer kann somit out-of-order TCP Segments senden und damit das System dazu bringen alle verfügbaren Speicher-Puffer zu verwenden.
    Es existiert ein Quellcode-Patch, der das Problem behebt.

  • 017: RELIABILITY FIX: 14. Februar 2004
    Es gibt mehrere Buffer Overflows im Code, der die font.aliases Dateien in XFree86 parst. Dank ProPolice können diese jedoch nicht ausgenutzt werden um erhöhte Rechte zu erlangen, aber sie können den X Server dazu bringen, sich zu beenden.
    Es existiert ein Quellcode-Patch, der das Problem behebt.

  • 016: SECURITY FIX: 8. Februar 2004
    Es gibt ein Problem beim Behandeln der IPv6 MTU, das von Angreifern dazu benutzt werden könnte, Denial of Service Angriffe gegen Rechner mit per IPv6 erreichbaren TCP Ports zu fahren.
    Es existiert ein Quellcode-Patch, der das Problem behebt.

  • 015: SECURITY FIX: 5. Februar 2004
    Es gibt einen Bug beim Zählen von Referenzen im shmat(2) Systemaufruf, der von Angreifern unter bestimmten Umständen dazu benutzt werden könnte, Kernel Speicher zu beschreiben.
    Es existiert ein Quellcode-Patch, der das Problem behebt.

  • 014: SECURITY FIX: 15. Januar 2004
    Mehrere Fehler bei der Nachrichtenbehandlung in isakmpd(8) wurden von Thomas Walpuski gemeldet. Sie erlauben es einem Angreifer, bestimmte SAs zu löschen. +A source code patch exists which remedies these problems.
    +

  • 013: RELIABILITY FIX: 20. November 2003
    Durch eine inkorrekte Grenzwertüberprüfung ist es einem lokalen Benutzer möglich durch die Übergabe bestimmter Argumente an die Funktionen semctl(2) und semop(2) das System zum Absturz zu bringen.
    Es gibt einen Source Code Patch, der das Problem behebt.

  • 012: RELIABILITY FIX: 20. November 2003
    Für einen lokelen Benutzer ist es, durch sysctl(3) mit bestimmten Argumenten, möglich einen Absturz zu verursachen.
    Es gibt einen Source Code Patch, der das Problem behebt.

  • 010: RELIABILITY FIX: 4. November 2003
    Es ist für einen lokalen Benutzer möglich eine "system panic" auszulösen, indem er ein spezielles Binärprogramm mit einem ungültigen Header ausführt.
    Es gibt einen Source Code Patch, der das Problem behebt.

  • 009: RELIABILITY FIX: 29. Oktober 2003
    Ein Benutzer mit Schreibrechten auf die httpd.conf oder eine .htaccess Datei kann den httpd(8) zum Absturz bringen oder möglicherweise eigenen Code als Benutzer www laufen lassen (obwohl wir glauben, dass ProPolice die Code-Ausführung verhindern wird).
    Es gibt einen Source Code Patch, der das Problem behebt.

  • 008: RELIABILITY FIX: 1. Oktober 2003
    Einem lokalen Benutzer ist es möglich, eine system panic auszulösen durch das Fluten mit gefälschten ARP-Anfragen.
    Es gibt einen Source Code Patch, der das Problem behebt.

  • 007: SECURITY FIX: 1. Oktober 2003
    Das Benutzen bestimmter ASN.1 Kodierungen oder fehlgeformte Public Keys erlauben es einem Angreifer, einen Denial-of-Service Angriff gegen Anwendungen zu fahren, die gegen ssl(3) gelinkt sind. Dies betrifft die OpenSSH nicht.
    Es gibt einen Source Code Patch, der das Problem behebt.

  • 006: SECURITY FIX: 24. September 2003
    Drei Fälle möglichen Zugriffs auf freigegebenen Speicher wurden in pf(4) gefunden. Mindestens einer von ihnen könnte benutzt werden, um im pf mit aktiven scrub-Regeln eine panic auszulösen.
    Es gibt einen Source Code Patch, der das Problem behebt.

  • 005: SECURITY FIX: 17. September 2003
    Ein buffer overflow im Address parsing in sendmail(8) kann einem Angreifer erlauben root Privilegien zu erlangen.
    Es gibt einen Source-Code Patch, der das Problem behebt.
    HINWEIS: dies ist die zweite Version des Patches, die ein weiteres Problem behebt.

  • 004: SECURITY FIX: 16. September 2003
    Alle Versionen von OpenSSH's sshd vor Version 3.7 enthalten einen Buffer Management Fehler. Es ist unklar, ob dieser Bug ausnutzbar (exploitbar) ist.
    Es gibt einen Source-Code Patch, der das Problem behebt.
    HINWEIS: dies ist die zweite Version des Patches, die ein weiteres Problem behebt.

  • 003: SECURITY FIX: 10. September 2003
    Root ist in der Lage, den security level zu verringern, in dem er einen Integer Overflow ausnutzt, wenn die Grenzen von Semaphoren sehr gross gemacht werden.
    Es gibt einen Source-Code Patch, der das Problem behebt.

  • 002: RELIABILITY FIX: 20. August 2003
    Eine unsaubere Überprüfung von Grenzen im semget(2) Systemaufruf kann einem lokalen User erlauben, eine Kernel Panic auszulösen.
    Es gibt einen Source-Code Patch, der das Problem behebt.
  • 001: SECURITY FIX: 4. August 2003
    Es existiert ein off-by-one in der C-Bibliotheks-Funktion realpath(3). Da der selbe Bug zu einer Root-Kompromittierung im wu-ftpd FTP Server geführt hat, ist es möglich, dass dieser Bug einem Angreifer ebenfalls erlaubt, seine Rechte in OpenBSD zu erweitern.
    Es gibt einen Source-Code Patch, der das Problem behebt.

    Es könnte für einen lokalen Benutzer möglich sein durch einen Stapelüberlauf in compat_ibcs2(8) und den daraus resultierenden Eskalationen der Privilegien beliebigen Code auszuführen.
    Es gibt einen Source Code Patch, der das Problem behebt.
