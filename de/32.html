Die Dateien dieser Version erhältst du auf einen der folgenden Wege:
  • Bestelle eine CDROM über unser Bestellsystem.
  • Sieh dir die FTP Seite an, um eine Liste der FTP-Mirrors zu betrachten, und den passenden für dich herauszufinden.
  • Gehe in das Verzeichnis pub/OpenBSD/3.2/ auf einer der spiegelnden Maschinen deiner Wahl.
  • Lies den Rest dieser Seite sorgfältig.
  • Sieh dir die 3.2 Errata Seite an, ob es Bugs und/oder Workarounds gibt.
  • Lies dir das detaillierte Log der Änderungen zwischen den Versionen 3.1 und 3.2 durch.


Hinweis: Alle anwendbaren Copyrights und Danksagungen können in den passenden Quellcodedateien in src.tar.gz, srcsys.tar.gz, XF4.tar.gz, oder den Dateien gefunden werden, die via ports.tar.gz geholt wurden. Die Distributionsdateien, die man braucht, um mit Hilfe der ports.tar.gz Datei packages zu bauen, sind aus Platzgründen nicht mit auf den CDROMs.

Was gibt es Neues?

Dies ist eine unvollständige Liste von neuen Features und Teilsystemen, die in OpenBSD 3.2 integriert sind. Eine vollständige Liste findet sich im changelog zu 3.2.

  • OpenSSH (Sowohl das SSH1- als auch das SSH2-Protokoll werden unterstützt) gibt es jetzt in Version 3.5. Privilege Separation ist standardmäßig aktiviert, um eine größere Robustheit zu gewährleisten.

  • Non-executable stack auf i386, sparc, sparc64, alpha, powerpc. No-exec data und bss auf sparc, sparc64, und alpha. Das führt dazu, dass einige Klassen von zukünftigen potentiellen Buffer Overflows nicht mehr ausnutzbar sind.

  • Apache läuft jetzt standardmäßig "chrooted". Um das zu ändern benutze die neue -u Option.

  • Eine deutliche Reduzierung von setuid Binaries. Viele der Binaries, die weiterhin setuid bleiben, wurden so modifiziert, dass die Operationen, die root verlangen, früh ausgeführt werden, und dann die root-Rechte sofort danach entzogen werden.

  • Unterstützung für asymmetrische und symmetrische Hardware-Verschlüsselung ist jetzt standardmäßig aktiviert, wenn du solche Hardware in deiner Maschine hast.

  • Wie immer wurde die Dokumentation verbessert, und zwar besonders die man pages und die Web FAQ. Ein größerer Teil der Webseiten ist jetzt in verschiedenen Sprache vorhanden.

  • Eine vollständigere und besser getestete Sammlung von "ports". Die Anzahl von setuid/setgid ports wurde ebenso wesentlich verringert. Viele derjenigen, die weiterhin setuid sind, wurden so modifiziert, dass diese Prvilegien möglichst früh entzogen werden.

  • Über 1800 vorkompilierte und getestete packages.

  • Bessere Grafik- und X11-Unterstützung für die OpenBSD/sparc, OpenBSD/sparc64 und OpenBSD/alpha ports.

  • Eine Menge Erweiterungen und Verbesserungen der Stabilität an unserem Packet Filter, pf, einschließlich:
    • neues "antispoof" Schlüsselwort: spoofing protection einfach gemacht
    • deutlich vereinfachte Regelsprache
    • erweiterte Filtermöglichkeiten
    • Sehr feine Kontrolle von "state table" Einträgen
    • Dynamische IP-zu-Interface-Zuweisungen werden jetzt unterstützt. Die Regeln müssen nach IP-Änderungen nicht mehr neu geladen werden.

  • Ein neues Tool, systrace, zum detaillierten Überwachen vom Verhalten von Applikationen und Rechten auf Level von Systemaufrufen.

  • Das System enthält die folgenden großen Komponenten von Drittherstellern/Anbietern:

    • XFree86 4.2.1 (und i386 enthält auch noch 3.3.X Server, daher Unterstützung für alle Chipsets)
    • gcc 2.95.3 (+ patches)
    • perl 5.6.1 (+ patches)
    • Apache 1.3.26, mod_ssl 2.8.10, DSO Unterstützung
    • OpenSSL 0.9.7beta3 (+ patches)
    • groff 1.15
    • sendmail 8.12.6
    • lynx 2.8.2rel.1 mit hinzugefügter HTTPS Unterstützung
    • sudo 1.6.6
    • ncurses 5.2
    • Das neueste KAME IPv6
    • KTH Kerberos 1.0.8
    • Heimdal 0.4e (+ patches)
    • OpenSSH 3.5

  • Viele Verbesserungen in Bezug auf Sicherheit und Zuverlässigkeit (siehe auch die roten Zeilen im kompletten changelog).

Wie man es installiert

Es folgen Instruktionen, die du in gedruckter Form haben würdest, wenn du anstatt einer alternativen Installationsmethode eine CDROM gekauft hättest. Die Anleitungen für FTP-Installationen sind recht ähnlich, die Anleitungen für die CDROM-Installationen wurden jedoch intakt gelassen, um zu zeigen, um wieviel einfacher alles mit einer CDROM gewesen wäre.

Bitte beachte auch die folgen Dateien auf den CDROMs, die noch weitere Details enthalten, um OpenBSD 3.2 auf deiner Maschine zu installieren:

  • CD1:3.2/i386/INSTALL.i386
  • CD1:3.2/alpha/INSTALL.alpha

  • CD2:3.2/macppc/INSTALL.macppc
  • CD2:3.2/vax/INSTALL.vax

  • CD3:3.2/sparc/INSTALL.sparc
  • CD3:3.2/sparc64/INSTALL.sparc64

Hier gibt es kurze Installationsinformationen für Leute, die mit OpenBSD vertraut sind und den Befehl "disklabel -E" kennen. Wenn du irgendwie unsicher in Bezug auf die Installation bist, lies die entsprechenden INSTALL.* Dateien, die oben angegeben sind!

OpenBSD/i386:

Spiele mit den BIOS-Optionen herum und versuche, von CD zu booten. Die i386 Version von OpenBSD ist auf der CD1. Wenn dein BIOS kein Booten von CD unterstützt, schreibe CD1:3.2/i386/floppy30.fs auf eine Floppy, und nimm diese als Boot-Medium.

Benutze CD1:3.2/i386/floppyB32.fs für eine bessere Unterstützung von SCSI-Controllern oder CD1:3.2/i386/floppyC32.fs für bessere Laptop-Unterstützung.

Wenn du OpenBSD neben anderen Betriebssystemen betreiben willst, solltest du unbedingt das Dokument INSTALL.i386 lesen.

Benutze das "rawrite" Werkzeug von CD:/3.2/tools/rawrite.exe um eine Floppy unter MS-DOS zu erstellen. Unter Unix einfach das dd(1) Utility benutzen. Das folgende Beispiel für dd(1) könnte "floppy", "rfd0c", oder "rfd0a" als Device benutzen. 

# dd if=<file> of=/dev/<device> bs=32k

Benutze ausschließlich perfekte und sauber formatierte Floppies OHNE DEFEKTE BLÖCKE oder deine Installation wird wahrscheinlich nicht klappen. Mehr Informationen über die Installation von OpenBSD/i386 gibts in der FAQ4.1.

OpenBSD/alpha:

Am SRM prompt gibst du boot -fi 3.2/alpha/bsd.rd dka6 ein, wobei dka6 der Kurzname für das CDROM-Laufwerk ist (prüfe das mit show dev).

Wenn du nicht von CDROM booten kannst, schreibe CD1:3.2/alpha/floppy32.fs oder CD1:3.2/alpha/floppyB32.fs (abhängig von deiner Maschine) auf eine Diskette und gib boot dva0 ein. In INSTALL.alpha gibt es noch weitere Details.

Benutze ausschließlich perfekte und sauber formatierte Floppies OHNE DEFEKTE BLÖCKE oder deine Installation wird wahrscheinlich nicht klappen.

OpenBSD/macppc:

Lege die CD2 in dein CDROM Laufwerk und drücke die C Taste während des Einschaltens, bis schließlich die Anzeige OpenBSD/macppc boot erscheint.

Alternativ kannst du am Open Firmware prompt boot cd:,ofwboot /3.2/macppc/bsd.rd eingeben.

OpenBSD/vax:

Boote über das Netzwerk via mopbooting, so wie es in INSTALL.vax beschrieben ist.

OpenBSD/sparc:

Version 3.2 von OpenBSD/sparc befindet sich auf der CD3. Um von dieser CD zu booten, kannst du eines von den zwei Kommandos benutzen, die weiter unten aufgeführt sind, je nachdem welche ROM Version du hast. 
> boot cdrom 3.2/sparc/bsd.rd
oder
> b sd(0,6,0)3.2/sparc/bsd.rd

Wenn deine sparc kein CD Laufwerk hat, kannst du alternativ dazu auch von einer Floppy booten. Dazu musst du die Datei "CD3:3.2/sparc/floppy31.fs" auf eine Floppy schreiben. Mehr Informationen gibts es in der FAQ4.1. Um von dieser Floppy zu booten, kannst du eines von den zwei Kommandos benutzen, die weiter unten aufgeführt sind, je nachdem welche ROM Version du hast. 

> boot floppy
oder
> boot fd()

Stelle sicher, dass du eine sauber formatierte Floppy OHNE DEFEKTE BLÖCKE benutzt, ansonsten wird deine Installation mit hoher Wahrscheinlichkeit nicht klappen.

Wenn deine sparc weder Floppy noch CD-Laufwerk hat, kannst du entweder ein bootbares Band erzeugen, oder via Netzwerk installieren, indem du den Anweisungen in der Datei INSTALL.sparc folgst.

OpenBSD/sparc64:

Lege die CD3 in dein CDROM Laufwerk und gib boot cdrom ein.

Wenn das nicht klappt oder du kein CDROM-Laufwerk hast, kannst du die Datei CD3:3.2/sparc64/floppy32.fs auf eine Floppy schreiben und sie mit boot floppy booten.
Stelle sicher, dass du eine sauber formatierte Floppy OHNE DEFEKTE BLÖCKE benutzt, ansonsten wird deine Installation mit hoher Wahrscheinlichkeit nicht klappen.

Du kannst auch CD3:3.2/sparc64/miniroot32.fs auf die Swap-Partition auf der Platte schreiben und mit boot disk:b booten.

Wenn nichts davon klappt, kannst du auch vom Netz booten, wie es in INSTALL.sparc64 beschrieben ist.

Hinweise zum source code:

src.tar.gz enthält ein Source-Code-Archiv, das in /usr/src beginnt. Diese Datei enthält bis auf die Kernel-Sourcen, die sich in einem anderen Archiv befinden, alles, was du brauchst. Um es zu entpacken: 

# mkdir -p /usr/src
# cd /usr/src
# tar xvfz /tmp/src.tar.gz

srcsys.tar.gz enthält ein Quellcode-Archiv, das in /usr/src/sys beginnt. Diese Datei enthält alle Kernel-Quellen, die man benötigt, um einen neuen Kernel zu erzeugen. Um sie zu entpacken: 

# mkdir -p /usr/src/sys
# cd /usr/src
# tar xvfz /tmp/srcsys.tar.gz

Beide Archive enthalten einen regulären CVS checkout. Wenn man diese Dateibäume benutzt, ist es möglich, die anoncvs-Server wie auf http://www.OpenBSD.org/de/anoncvs.html beschrieben zu benutzen. Wenn man diese Dateien benutzt, hat man ein viel schnelleres erstes Update, als wenn man einen frischen, eigenen OpenBSD Quellbaum (source tree) mit Hilfe von CVS holt.

Ports Tree

Mitgeliefert wird auch ein ports tree Archiv. Um es zu entpacken: 

# cd /usr
# tar xvfz /tmp/ports.tar.gz
# cd ports

Das ports/ Unterverzeichnis ist ein checkout für den OpenBSD ports tree. Lies am besten http://www.OpenBSD.org/ports.html wenn du noch nichts über ports weisst. Dieser Text ist keine Anleitung, wie man die ports benutzt, sondern nur dazu da, um den Leuten den Einstieg in das OpenBSD ports System zu ermöglichen.

Natürlich ist das OpenBSD ports System nicht vollständig. Es ist zu bezweifeln, dass es das jemals sein wird. Trotzdem wächst es sehr schnell und wird stabiler. Fast alle ports, die ausgeliefert werden, sollten ohne Probleme auf den meisten Architekturen kompilierbar sein (auf z.B. i386 konnten über 2000 packages ohne Probleme erzeugt werden).

Das ports/ Verzeichnis stellt ein CVS (siehe in der manpage für cvs(1) nach, wenn du mit CVS nicht vertraut bist) checkout unserer ports dar. Wie unser kompletter source tree ist auch auch unser ports tree via anoncvs verfügbar. Um ihn also auf dem Laufenden zu halten, musst du den ports/ tree auf einem schreibbaren Medium haben und den tree mit diesem Kommando updaten: 

# cd [portsdir]/; cvs -d anoncvsserver.openbsd.org:/cvs update -Pd -rOPENBSD_3_2

[Natürlich musst du die Namen für Server und [portsdir] mit deinen lokalen Verzeichnis, bzw. dem Namen eines anoncvs-Servers in der Nähe austauschen.]

Denk daran, dass die meisten ports via ftp als fertige packages erhältlich sind. Upgedatete packages für 3.2 werden verfügbar gemacht, sobald Probleme bekannt sind.

Falls du gerne einen weiteren port sehen willst, mithelfen willst, oder einfach mehr erfahren willst, solltest du die Mailingliste ports@openbsd.org kennenlernen.
