Alle Architekturen

  • 027: SECURITY FIX: 31. März 2003
    Ein buffer overflow beim Verarbeiten von Adressen in sendmail(8) kann es einem Angreifer möglich machen Root Privilegien zu erlangen.
    Es gibt einen Source Code Patch, der das Problem beseitigt.

  • 026: SECURITY FIX: 24. März 2003
    Eine kryptographische Schwäche im Kerberos v4 Protokoll kann unter Kerberos v5 ebenso ausgenutzt werden.
    Es gibt einen Source Code Patch, der das Problem beseitigt.

  • 012: SECURITY FIX: 19. März 2003
    OpenSSL ist angreifbar durch eine Erweiterung des ``Bleichenbacher'' Angriffs , der von den Tschechischen Forschern Klima, Pokorny und Rosa entworfen wurde.
    Ein Source Code Patch behebt das Problem.

  • 011: SECURITY FIX: 18. März 2003
    Verschiedene SSL und TLS Operations in OpenSSL sind mittels "timing attacks" angreifbar.
    Ein ``RSA blinding'' Source Code Patch behebt das Problem.

  • 010: SECURITY FIX: 5. März 2003
    Ein fix für einen lprm(1) Bug, den es seit 1996 gibt und einen Fehler enthält, der zu privilege escalation führen kann.
    Es gibt einen Source Code Patch, der das Problem beseitigt.

  • 022: SECURITY FIX: 3. März 2003
    Ein buffer overflow in der envelope comments Verarbeitung von sendmail(8) kann es einem Angreifer erlauben, Root Privilegien zu erlangen.
    Ein Source Code Patch behebt das Problem.

  • 021: SECURITY FIX: 23. Februar 2003
    In ssl(8) kann durch das Timing beim Berechnen eines MAC ein Informationsleck entstehen, sogar wenn inkorrektes Block Cipher Padding gefunden wurde, hier ist eine Gegenmassnahme. Überprüfe die Zuweisungroutinen auch auf negative Grössen.
    Ein SourceCode Patch behebt beide Probleme.

  • 020: SECURITY FIX: 20. Januar 2003
    Ein "double free" in cvs(1) könnte einem Angreifer erlauben, Code mit den Privilegien desjenigen Users auszuführen, der CVS laufen lässt. Das ist nur ein Problem, wenn der ausgeführte cvs-Befehl unter einem anderen User läuft. Das bedeutet, dass das Problem nur bei Konfigurationen wie pserver Client/Server existiert.
    Es gibt einen Source Code Patch, der das Problem beseitigt.

  • 005: SECURITY FIX: 14. November 2002
    Ein buffer overflow im named(8) kann einem Angreifer erlauben, Code mit den Privilegien des named auszuführen. Auf OpenBSD läuft der named als non-root User in einer chrooted Umgebung, was den Effekt dieses Bugs mildert.
    Es gibt einen Source Code Patch, der das Problem beseitigt.

  • 035: SECURITY FIX: 6. November 2002
    Ein nicht korrektes Prüfen der Argumente im getrlimit(2) Systemaufruf kann es einem Angreifer erlauben, den Kernel zum Absturz zu bringen.
    Ein Source Code Patch behebt das Problem.

  • 034: SECURITY FIX: 6. November 2002
    Ein Angreifer kann die Beschränkungen von sendmail's restricted shell smrsh(8) umgehen, und beliebige Befehle mit seinen eigenen Privilegien ausführen.
    Ein Source Code Patch behebt das Problem.

  • 033: SECURITY FIX: 21. Oktober 2002
    Ein Puffer-Überlauf kann im kadmind(8) Daemon passieren, was zu einem möglichen remote Crash oder Exploit führen kann.
    Ein Source Code Patch behebt das Problem.

  • 015: SECURITY FIX: 2. Oktober 2002
    Nicht korrektes Überprüfen von Argumenten im setitimer(2) Systemaufruf kann einem Angreifer erlauben, in den Kernel-Speicher zu schreiben.
    Ein Source Code Patch behebt das Problem.

  • 031: SECURITY FIX: 11. August 2002
    Ein unzureichender boundary check im select(2) Systemaufruf erlaubt einem Angreifer Kernel-Speicher zu überschreiben und beliebigen Code im Kernel-Kontext auszuführen.
    Ein Source Code Patch behebt das Problem.

  • 013: SECURITY FIX: 30. Juli 2002
    Mehrere remote buffer overflows können im SSL2 Server und SSL3 Client der ssl(8) Library auftreten, wie auch im ASN.1 Parser Code in der crypto(3) Library, alle sind potentiell remote exploitbar.
    Ein Source Code Patch behebt das Problem.
    Dies ist die zweite Version des Patches.

  • 012: SECURITY FIX: 29. Juli 2002
    Es gibt einen Buffer Overflow im xdr_array(3) RPC Code, was zu einem möglichen remote crash führen kann.
    Ein Source Code Patch behebt das Problem.

  • 011: SECURITY FIX: 29. Juli 2002
    Es gibt eine Race Condition im pppd(8) daemon, die ihn dazu bringen kann, eigenmächtig die Dateirechte einer anderen Datei zu verändern.
    Ein Source Code Patch behebt das Problem.

  • 010: RELIABILITY FIX: 5. Juli 2002
    Das Empfangewn von IKE payloads ausserhalb der Sequenz kann den isakmpd(8) zum Absturz bringen.
    Ein Source Code Patch behebt das Problem.
    Dies ist die zweite Version des Patches.

  • 009: SECURITY FIX: 27. Juni 2002
    Der Kernel lässt jeden User ktrace(2) Prozesse set[ug]id ausführen.
    Ein Source Code Patch behebt das Problem.

  • 008: SECURITY FIX: 26. Juni 2002
    Ein buffer overflow kann während des Parsens von .htaccess im mod_ssl httpd Modul auftreten, was zu einem remote crash oder exploit führen kann.
    Ein Source Code Patch behebt das Problem.

  • 025: SECURITY FIX: 25. Juni 2002
    Ein potentieller Buffer Overflow wurde im DNS Resolver gefunden.
    Ein Source Code Patch behebt das Problem.

  • 006: SECURITY FIX: 24. Juni 2002
    Alle Versionen von OpenSSH's sshd zwischen 2.3.1 und 3.3 enthalten einen Fehler bei der Eingabe-Validierung, der zu einem Integer-Overflow und "privilege escalation" führen kann. Dieses Problem ist in OpenSSH 3.4 beseitigt, und ein Patch für die angreifbaren Versionen ist als Teil des security advisory verfügbar.

  • 022: SECURITY FIX: 19. Juni 2002
    Ein buffer overflow kann während der Interpretation von "chunked encoding" im http daemon passieren, was zu einem remote crash oder exploit führen kann.
    Ein Source Code Patch behebt das Problem.

  • 004: SECURITY FIX: 22. Mai 2002
    Um bestimmten Umständen ist es auf Systemen, die YP mit Netgroups in der password database benutzen, möglich, dass sshd(8) zwar ACL Checks für den nachgefragten Usernamen durchführt, aber den password database Eintrag eines anderen Users für die Authentifikation benutzt. Das bedeutet, dass unberechtigte Leute Zutritt zum System bekommen könnten, während berechtigte ausgeschlossen bleiben könnten.
    Ein Source Code Patch behebt das Problem.

  • 021: SECURITY FIX: 8. Mai 2002
    Es gibt eine race condition bei der ein Angreifer die file descriptor Tabelle füllen kann und die Sicherung des Kernels der fd slots 0, 1, und 2 für einen setuid oder setgid Prozess ausschalten kann.
    Ein Source Code Patch behebt das Problem.

  • 002: SECURITY FIX: 25. April 2002
    Ein Bug in sudo(8) erlaubt es einem Angreifer den Heap zu korrumpieren, indem er einen eigenen Prompt angibt.
    Ein Source Code Patch behebt das Problem.

  • 001: SECURITY FIX: 22. April 2002
    Ein lokaler User kann super-user Privilegien wegen eines buffer overflow in sshd(8) erlangen, falls AFS auf dem System konfiguriert ist, oder auch KerberosTgtPassing oder AFSTokenPassing im sshd_config File konfiguriert ist. Ticket und token passing sind standardmäßig abgeschaltet.
    Ein Source Code Patch behebt das Problem.
