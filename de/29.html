Bestellung eines CD Sets

Wenn du die CDs dieser Version haben willst:
  • Bestelle eine CD mit unserem Bestellsystem.
  • Sieh dir die Informationen auf der FTP Seite an, wenn du eine Liste der Mirrors sehen willst
  • Gehe in das pub/OpenBSD/2.9/ Verzeichnis auf einer der Mirror-Seiten.
  • Lese den Rest dieses Dokumentes gründlich
  • Werfe einen Blick auf die 2.9 Errata Seite für eine Liste von Bugs und Workarounds.
  • Ein detailliertes Log der Änderungen zwischen den Versionen 2.8 und 2.9.



Hinweis: Alle anwendbaren Copyrights und Danksagungen finden sich in den jeweiligen Quellcodedateien, die in den Archiven src.tar.gz, srcsys.tar.gz und XF4.tar.gz oder auch in den Dateien, die via ports.tar.gz geholt wurden, enthalten sind. Die Distributionsdateien, mit denen die packages aus der ports.tar.gz Datei erzeugt werden, sind aus Platzgründen nicht auf den CDROMs enthalten.

Was gibt es neues ?

Dies ist eine unvollständige Liste der neuen Features und Systeme, die es in OpenBSD 2.9 gibt. Eine umfassendere Liste gibt es im changelog.

  • Die Dateisystem-Performance hat dank neuer Softupdates und neuem dirpref Code einen mächtigen Schub erhalten. Einige Tests haben eine 60-fache Steigerung in der Geschwindigkeit des Dateisystems gezeigt. Wenn es einen Grund für das Update auf 2.9 gibt, dann ist es dieser.

  • OpenSSH (das sowohl SSH1, als auch das SSH2 Protokoll unterstützt) ist jetzt bei Version 2.9. Sichere Dateitransfers sollten mit dem stark verbesserten SFTP Subsystem gemacht werden, welches jetzt als Server und Client geliefert wird.

  • Umfassende Änderungen an der Dokumentation, hauptsächlich den man pages und der Web FAQ. Die manual pages enthalten jetzt brauchbare Beispiel, um die Erklärungen zu ergänzen.

  • Umfangreichere und besser geteste Sammlung von "ports".

  • Über 1000 fertig erzeugte und getestete packages.

  • OpenBSD 2.9 funktioniert auch auf dem neuen Apple Titanium PowerBook G4.

  • Der OpenBSD/alpha port ist nach einer Pause von 2 Versionen zurück, und das mit einer besseren Hardwareunterstützung als je zuvor !

  • Das System beinhaltet die folgenden grossen Komponenten von Aussenstehenden:

    • XFree86 4.0.3 (und i386 enthält zusätzlich 3.3.X Server, um wirklich alle Chipsets zu unterstützen)
    • gcc 2.95.3 (+ patches)
    • perl 5.6.0 (+ patches)
    • Apache 1.3.19, mod_ssl 2.8.2, OpenSSL 0.9.6 (+ patches), DSO Unterstützung
    • ipf 3.4.16 (+ patches)
    • groff 1.15
    • sendmail 8.11.3
    • lynx 2.8.2 mit HTTPS Unterstützung
    • sudo 1.6.3p7
    • ncurses 5.2
    • Latest KAME IPv6
    • KTH Kerberos 1.0.4
    • OpenSSH 2.9

  • Wesentlich verbesserte Treiberunterstützung

    • 3Com 3c990 3XP Typhoon/Sidewinder Treiber (nur Ethernet).
    • Neue Hardware RAID Treiber einschliesslich:
      • Adaptec FSA
      • American Megatrends MegaRAID
      • Compaq Smart ARRAY Controller
    • Neue SCSI Treiber, die folgendes unterstützen:
      • Initio 940/950
      • Symbios Logic 53c1x10
      • AMD Am53c974-basierende PCI Adapter
    • Viel neue Unterstützung für verschiedene Laptop Soundkarten:
      • ESS Maestro 1/2/2E
      • Yamaha DS-1S
      • Cirrus Logic CS4281 Audio Devices
    • USB Hardware wird jetzt standardmässig unterstützt, was auch Tastaturen und Mäuse einschliesst.

  • Viele Verbesserungen für Sicherheit und Stabilität (siehe die roten Einträge im complete changelog).

  • Viele neue Features, einschliesslich:

    • Unterstützung für lange Usernamen.
    • Stark verbesserte Linux emulation, so dass jetzt viel mehr Applikationen funktionieren.
    • Unterstützung für 802.1d spanning tree für bridge(4).
    • vlan(4) bridging Unterstützung erlaubt jetzt viele neue Dinge, die man mit Bridges machen kann (z.B kannst du OpenBSD als 802.1Q kompatiblen Switch einsetzen)
    • kqueue(2) kernel event notification Mechanismus.
    • Einführung des wscons(4) maschinen-unabhängingen Konsolentreibers.
    • Starke Verbesserungen an softupdates, inklusive Geschwindkgkeits- und Stabilitätsfixes.
    • Bug-fixes und Erweiterungen an den atapiscsi(4) und pciide(4) Treibern, einschliesslich UltraDMA/100 Unterstützung für Intel-, Promise-, und Highpoint-Controller.

Installationsanleitung

Es folgen Instruktionen, die du in gedruckter Form haben würdest, wenn du anstatt einer alternativen Installationsmethode eine CDROM gekauft hättest. Die Anleitungen für FTP-Installationen sind recht ähnlich, die Anleitungen für die CDROM-Installationen wurden jedoch intakt gelassen, um zu zeigen, um wieviel einfacher alles mit einer CDROM gewesen wäre.

Bitte beachte auch die folgen Dateien auf den CDROMs, die noch weitere Details enthalten, um OpenBSD 2.9 auf deiner Maschine zu installieren:

  • CD1:2.9/i386/INSTALL.i386
  • CD1:2.9/powerpc/INSTALL.powerpc
  • CD1:2.9/vax/INSTALL.vax

  • CD2:2.9/sparc/INSTALL.sparc
  • CD2:2.9/mvme68k/INSTALL.mvme68k
  • CD2:2.9/amiga/INSTALL.amiga
  • CD2:2.9/mac68k/INSTALL.mac68k
  • CD2:2.9/hp300/INSTALL.hp300
  • CD2:2.9/sun3/INSTALL.sun3

  • Die alpha Version gibt es nur auf den ftp-sites, nicht auf den CDs.

Hier gibt es kurze Installationsinformationen für Leute, die mit OpenBSD vertraut sind und den Befehl "disklabel -E" kennen. Wenn du irgendwie unsicher in Bezug auf die Installation bist, lies die entsprechenden INSTALL.* Dateien, die oben angegeben sind!

OpenBSD/i386:

    Spiele mit den BIOS-Optionen herum und versuche, von CD zu booten. Wenn das nicht klappt, schreibe CD1:2.9/i386/floppy29.fs auf eine Floppy, und nimm diese als Boot-Medium. Wenn du OpenBSD neben anderen Betriebssystemen betreiben willst, solltest du unbedingt das Dokument INSTALL.i386 lesen.

    Benutze das "rawrite" Werkzeug, von CD:/2.9/tools/rawrite.exe um eine Floppy unter MS-DOS zu erstellen. Unter Unix einfach das dd(1) Utility benutzen. Das folgende Beispiel für dd(1) könnte "floppy", "rfd0c", oder "rfd0a" als Device benutzen. 

      # dd if=<file> of=/dev/<device> bs=32k

    Benutze ausschliesslich perfekte und sauber formatierte Floppies OHNE DEFEKTE BLÖCKE oder deine Installation wird wahrscheinlich nicht klappen. Mehr Informationen über die Installation von OpenBSD/i386 gibts in der FAQ4.1.

OpenBSD/sparc:

    Die Version 2.9 von OpenBSD/sparc befindet sich auf CD2. Um von dieser CD zu booten, kannst du, abhängig von deiner BIOS Version, eines der Kommandos unten benutzen: 

      > boot cdrom 2.9/sparc/bsd.rd
oder
> b sd(0,6,0)2.9/sparc/bsd.rd

    Wenn deine sparc kein CD-Laufwerk hat, kannst du alternativ von einer Floppy aus booten. Dazu musst du "CD2:2.9/sparc/floppy29.fs" auf eine Floppy schreiben. Näheres dazu gibts unter der FAQ4.1. Um von dieser Floppy zu booten, kannst du, abhängig von deiner BIOS Version, eines der Kommandos unten benutzen: 

      > boot floppy
oder
> boot fd()

    Benutze ausschliesslich perfekte und sauber formatierte Floppies OHNE DEFEKTE BLÖCKE oder deine Installation wird wahrscheinlich nicht klappen.

    Wenn deine Sparc weder ein Floppy-, noch ein CD-Laufwerk hat, kannst du entweder ein bootbares Tape erzeugen, oder via Netzwerk booten, wie es in der INSTALL.sparc Datei beschrieben ist.

OpenBSD/amiga:

    Erzeuge BSD-Partitionen mit Hilfe der Anleitung in INSTALL.amiga. Dann die CD2 unter AmigaOS als Device CD0 mounten. Als nächstes folgendes CLI-Kommando eingeben : "CD0:2.9/amiga/utils/loadbsd CD0:2.9/amiga/bsd.rd".

OpenBSD/hp300:

    Du kannst mit den Anleitungen aus INSTALL.hp300 über Netzwerk booten.

OpenBSD/mac68k:

    Boote ganz normal das MacOS und partitioniere mit Hilfe der passenden A/UX Konfigurationen. Danach packst du die Macside Utilities von CD1:2.9/mac68k/utils auf deine Festplatte. Als nächstes läßt du Mkfs laufen, um die Dateisysteme zu erzeugen, die du mit Hilfe von A/UX gerade gemacht hast. Danach benutzt du den BSD/Mac68k Installer um alle Teile aus CD1:2.9/mac68k/ auf deine Partition zu kopieren. Schlussendlich bist du nun bereit, den BSD/Mac68k Booter mit dem genauen Ort des Kernels zu konfigurieren und kannst das System booten.

OpenBSD/sun3:

    Entweder machst du einen diskless boot oder erzeugst ein Installtionstape, wie in INSTALL.sun3 beschrieben.

OpenBSD/alpha:

    Hol' dir die Version per ftp. Danach machst du einen diskless boot oder bootest via Floppy, wie in INSTALL.alpha beschrieben.

Anmerkungen zum Quellcode:

    src.tar.gz enthält ein Archiv mit Quellcode, daß in /usr/src liegen sollte. Diese Datei enthält bis auf die Kernel-Quellen, die sich in einer separaten Datei befinden, alles, was man braucht. Um den Quellcode zu entpacken : 

      # mkdir -p /usr/src
# cd /usr/src
# tar xvfz /tmp/src.tar.gz

    srcsys.tar.gz enthält ein Quellcode-Archiv, das in /usr/src/sys beginnt. Diese Datei enthält alle Kernel-Quellen, die man benötigt, um einen neuen Kernel zu erzeugen. Um sie zu entpacken: 

      # mkdir -p /usr/src/sys
# cd /usr/src
# tar xvfz /tmp/srcsys.tar.gz

    Beide Archive enthalten einen regulären CVS checkout. Wenn man diese Dateibäume benutzt, ist es möglich, die anoncvs-Server wie auf http://www.OpenBSD.org/anoncvs.html zu benutzen. Wenn man diese Dateien benutzt, hat man ein viel schnelleres erstes Update, als wenn man einen frischen, eigenen OpenBSD Quellbaum (source tree) mit Hilfe von CVS holt.

Der Ports Tree

Mitgeliefert wird auch ein ports tree Archiv. Um es zu entpacken: 

    # cd /usr
# tar xvfz /tmp/ports.tar.gz
# cd ports

Das ports/ Unterverzeichnis ist ein checkout für den OpenBSD ports tree. Lies am besten http://www.OpenBSD.org/ports.html wenn du noch nichts über ports weisst. Dieser Text ist keine Anleitung, wie man die ports benutzt, sondern nur dazu da, um den Leuten den Einstieg in das OpenBSD ports system zu ermöglichen.

Natürlich ist das OpenBSD ports System nicht vollständig. Es ist zu bezweifeln, dass es das jemals sein wird. Trotzdem wächst es sehr schnell und wird stabiler. Fast alle ports, die ausgeliefert werden, sollten ohne Probleme auf den meisten Architekturen kompilierbar sein (auf z.B. i386 konnten über 1200 packages ohne Probleme erzeugt werden).

Das ports/ Verzeichnis stellt ein CVS (siehe in der manpage für cvs(1) nach, wenn du mit CVS nicht vertraut bist) checkout unserer ports dar. Wie unser kompletter source tree ist auch auch unser ports tree via anoncvs verfügbar. Um ihn also auf dem Laufenden zu halten, musst du den ports/ tree auf einem schreibbaren Medium haben und den tree mit diesem Kommando updaten: 

    # cd [portsdir]/; cvs -d anoncvsserver.openbsd.org:/cvs update -Pd -rOPENBSD_2_9

[Natürlich musst du die Namen für Server und [portsdir] mit deinen lokalen Verzeichnis, bzw. dem Namen eines anoncvs-Servers in der Nähe austauschen.]

Denk daran, dass die meisten ports via ftp als fertige packages erhältlich sind. Upgedatete packages für 2.9 werden verfügbar gemacht, sobald Probleme bekannt sind.

Falls du gerne einen weiteren port sehen willst, mithelfen willst, oder einfach mehr erfahren willst, solltest du die Mailingliste ports@openbsd.org kennenlernen.
