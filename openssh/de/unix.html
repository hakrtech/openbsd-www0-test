Unix Alternativen

Wenn du auf Unix nicht OpenSSH benutzen willst, stehen die folgenden "freien" SSH-Implementationen zur Zusammenarbeit mit OpenSSH zur Verfügung:

  • OSSH ist eine nur-SSH1 Implementation. BSD Lizenz.

    Björn Grönvall war die eigentliche Person, die unsere Gemeinschaft daran erinnert hat, dass sehr alte Versionen von ssh eine freie Lizenz hatten. Dies ist seine eigene Implementation, basierend auf dem alten Code von Tatu. OpenSSH basierte auf einer frühen Version von OSSH, aber OpenSSH fügte ssh2 Protokoll Unterstützung hinzu und erweiterte die Unterstützung des SSH1 Protokolls.

  • LSH/psst ist eine nur-SSH2 Implementation von Niels Möller aus Schweden, GPL Lizenz.

    "ssh benutzt kryptografische Techniken um die Sicherheit von alltäglichen Zielen wie etwa File Transfers, remote logins und remote Command execution zu erhöhen. Es ist als sichere(re) Alternative zu telnet, ftp, rsh, rlogin und rexec gedacht - und braucht viel weniger Aufwand als Kerberos oder IPSEC momentan verlangen."

  • FreSSH ist eine SSH Implementation, die ursprünglich von Eric Haszlakiewicz und Thor Lancelot Simon geschrieben wurde, und zur Zeit von Jason R. Thorpe weiterentwickelt wird. Die Lizenz ist BSD-ähnlich, sollte aber aber gründlich gelesen werden, da sie Software enthält, die von dritter Seite entwickelt wurde und unter einer anderen Lizenz steht.

  • Dropbear ist ein kleiner SSH2 Nur-Server Implementierung, die von Matt Johnston geschrieben wurde. Sie ist unter MIT-style Lizenz erhältlich.
