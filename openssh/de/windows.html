Windows

Die folgenden ,freien' SSH-Implementierungen werden für die Zusammenarbeit von Windows-Maschinen mit OpenSSH empfohlen:

  • PuTTY ist eine SSH1+SSH2-Implementierung. PSCP, ein scp-artiges Programm für Windows, ist auch verfügbar.

    PuTTY unterliegt der MIT-Lizenz (BSD-ähnlich).

    "PuTTY ist eine freie Implementierung von Telnet und SSH für Win32-Plattformen, geschrieben und gepflegt hauptsächlich von Simon Tatham, der in Großbritannien lebt."

  • TTSSH (SSH1) ist eine Nur-SSH1-Implementierung von Robert O'Callahan.

    "TTSSH ist ein freier SSH-Client für Windows. Er ist als Erweiterungs-DLL für Teraterm Pro implementiert. Teraterm Pro ist ein fantastischer freier Terminal-Emulator/Telnet-Client für Windows und sein Quellcode ist verfügbar. TTSSH fügt Teraterm Pro SSH-Fähigkeiten hinzu, ohne irgendwelche bestehenden Funktionen von Teraterm zu opfern. TTSSH kann ebenfalls frei heruntergeladen und benutzt werden und sein Quellcode ist ebenfalls mit freier Lizenz erhältlich. Weiterhin wurde TTSSH vollständig in Australien implementiert [...]."

  • Cygwin (POSIX-Software als Aufsatz zu Windows)

    OpenSSH (Protokolle SSH1 und SSH2) mit Cygwin läuft unter Windows, wenn man die portable Version von OpenSSH benutzt.

  • MSSH

    MSSH vom Metropolitan State College of Denver unterstützt Windows 95 und Windows 98 mit dem SSH1-Protokoll.

  • Secure iXplorer GPL

    Secure iXplorer GPL ist ein grafisches Frontend zu PuTTYs pscp.exe.

  • WinSCP

    WinSCP ist ein scp(1)-Programm für Windows mit integriertem PuTTY.

  • FileZilla

    FileZilla ist ein mächtiger FTP-Client für Windows 9x, ME, NT4, 2000 und XP. Es wurde auf eine leichte Bedienung ausgelegt und soll dabei so viele Features wie möglich haben und trotzdem noch schnell und zuverlässig sein. Es unterstützt SFTP.

  • SSHWindows

    • ,OpenSSH for Windows' ist ein freies Paket, das einen minimalen OpenSSH-Server und -Clientanwendungen im Cygwin-Paket installiert, ohne eine volle Cygwin-Installation zu benötigen. Es bietet volle SSH/SCP/SFTP-Unterstützung, SSH-Terminalunterstützung liefert eine gewohnten Windows-Eingabeaufforderung, während die Unix/Cygwin-style Pfade für SCP und SFTP erhalten bleiben.
