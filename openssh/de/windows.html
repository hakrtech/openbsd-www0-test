Windows & Mac

Die folgenden"freien" Klienten werden für die Zusammenarbeit mit OpenSSH von Windows Maschinen empfohlen:

  • PuTTY ist eine SSH1+SSH2 Implementation. PSCP, ein scp-artiges Programm für Windows, ist auch verfügbar.

    PuTTY unterliegt der MIT Lizenz (BSD-ähnlich). Putty's SSH2 Modus enthält keine unterstützung für User-Authentifikation mit DSA Schlüsseln. User-Authentifikation mit RSA Schlüsseln wird aber in beiden Protokollen unterstützt.

    "PuTTY ist eine freie Implementation von Telnet und SSH für Win32 Plattformen, geschrieben und gepflegt hautpsächlich von Simon Tatham, der in Grossbritannien lebt."

  • TTSSH (SSH1) ist eine nur-SSH1 Implementation von Robert O'Callahan.

    "TTSSH ist ein freier SSH Client für Windows. Er ist als Erweiterungs-DLL für Teraterm Pro implementiert. Teraterm Pro ist ein fantastischer freier Terminal Emulator/Telnet Client für Windows, und sein Quellcode ist verfügbar. TTSSH fügt Teraterm Pro SSH-Fähigkeiten hinzu, ohne irgendwelche bestehenden Funktionen von Teraterm zu opfern. TTSSH kann ebenfalls frei heruntergeladen und benutzt werden und sein Quellcode ist ebenfalls mit freier Lizenz erhältlich. Weiterhin wurde TTSSH vollständig in Australien implementiert [...]."

  • Telneat ist ein Telnet/SSH1/SSH2 Client für Windows von Kosta Schegloff,

    der folgende Features bietet:

    • "volle Telnet Unterstützung
    • SSH Unterstützung (IDEA, DES, TripleDes, ArcFour, BlowFish, Kompression)
    • SSH2 Unterstützung
    • termcap Unterstützung
    • unterstützt Win95, OSR2, Win98, WinNT, Win2000
    • unterstützt externe SSH Verschlüsselungs-Plugins
    • bis zu 10 Konsolen
    • Mausgesteuertes cut-and-paste
    • 'Scroll-Puffer'
    • Autologin
    • konfigurierbare 'recode' Tabellen
    • unabhängige Konfiguration für jeden Host
    • Login Skripte"

  • Cygwin (POSIX Software als Aufsatz zu Windows)

    OpenSSH (Protokolle SSH1 und SSH2) mit Cygwin läuft auf Windows, wenn man die portable version of OpenSSH benutzt.

  • MSSH

    MSSH vom Metropolitan State College of Denver unterstützt Windows 95 und Windows 98 mit dem SSH1 Protokoll.

  • Secure iXplorer

    Secure iXplorer ist ein grafischer Aufsatz für PuTTY's pscp.exe.

  • WinSCP

    WinSCP ist ein scp(1) Programm für Windows mit integriertem PuTTY

Die folgenden Clients werden für die Zusammenarbeit mit OpenSSH von Mac Maschinen aus empfohlen:

  • NiftyTelnet 1.1 SSH ist ein nur-SSH1 Implementation, die mit einem scp-artigen Programm ausgeliefert werden. Geschrieben von Jonas Wallden.

    "NiftyTelnet 1.1 SSH r3 ist eine erweiterte Version von Chris Newman's NiftyTelnet 1.1 Applikation, die Unterstützung für verschlüsselte Terminal Sessions mittles SSH (Secure Shell) Protokoll beinhaltet. Bitte lies die enthaltene Readme Datei bevordu diese Version weitergibst."

  • MacSSH ist eine nur-SSH2 Implementation.

    "MacSSH is a modified version of BetterTelnet with SSH2 support. [...] The only SSH2 client for MacOS that I could find is a commercial product thats costs more than $100, and it crashes my Mac when closing a session... Since it's best to do things by oneself, here's MacSSH."
