OpenSSH für OpenBSD

Da OpenSSH von der OpenBSD Gruppe entwickelt wird, ist OpenSSH Teil der Basis des Betriebssystems, beginnend ab OpenBSD Version 2.6 aufwärts. Es gibt also nichts, was man dem System hinzufügen oder daran ändern müsste. Nichtsdestotrotz möchtest du vielleicht aufgrund neuer Features ein Upgrade machen.

Du kannst die aktuellste OpenSSH Version von diesen ftp mirrors herunterladen.

Du kanst die folgenden Versionen von OpenSSH auf deinen OpenBSD-Maschinen installieren:

  • OpenSSH 3.8: 24. Februar 2004

    ftp://ftp.openbsd.org/pub/OpenBSD/OpenSSH/openssh-3.8.tgz

    Wenn du OpenSSH 3.8 auf OpenBSD 3.4 installierst brauchst du den folgenden Patch:
    ftp://ftp.openbsd.org/pub/OpenBSD/OpenSSH/openbsd33_3.8.patch.

    Wenn du OpenSSH 3.8 auf einer äteren Version installlierst, brauchst du den folgenden Patch:
    ftp://ftp.openbsd.org/pub/OpenBSD/OpenSSH/openbsd33_3.8.patch.

    # cd /usr/src/usr.bin
# tar xvfz .../openssh-3.8.tgz
# cd ssh
# patch -p0 < .../openbsd3x_3.8.patch
# make obj
# make cleandir
# make depend
# make
# make install
# cp ssh_config sshd_config /etc/ssh
# mkdir /var/empty
    Um sshd(8)'s neues "privilege separation" Feature auf einigen älteren Versionen zu unterstützen, wurden dem System ein neuer User und eine neue Gruppe names sshd hinzugefügt. Füge die folgende Einträge mittels vipw(8) hinzu: 
    sshd:*:27:27::0:0:sshd privsep:/var/empty:/sbin/nologin

    Füge das folgende in /etc/group ein: 

    sshd:*:27:
_sshagnt:*:34:
    und starte sshd neu.
