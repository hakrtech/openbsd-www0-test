"Portable OpenSSH"

Le développement normal d'OpenSSH produit une version très petite en termes de taille, sécurisée, et facilement maintenable pour le projet OpenBSD. L'équipe de portage d'OpenSSH prend cette version pûre et y ajoute le code de portabilité nécessaire afin qu'OpenSSH puisse fonctionner sur plusieurs autres systèmes d'exploitation (malheureuseument, et particulièrement parce qu'OpenSSH effectue des opérations d'authentification, on trouve *énormèment* de différences entres les systèmes d'exploitation Unix).

La version d'OpenSSH fournie ici est conçue pour fonctionner sur les systèmes d'exploitation Unix suivants :

  • AIX
  • HP-UX
  • Irix
  • Linux
  • NeXT
  • SCO
  • SNI/Reliant Unix
  • Solaris
  • Digital Unix/Tru64/OSF
  • Mac OS X
  • Cygwin
  • ... et d'autres sont ajoutés régulièrement.

"portable OpenSSH" suit le développement de la version officielle, mais les versions publiées ne sont pas synchronisées. Les versions "Portable" sont indiquées par un 'p' (par exemple 4.1p1). La source officielle pour OpenBSD n'utilisera jamais le suffixe 'p', mais aura une version incrémentée par rapport à la version précédente quand la stabilité sera atteinte lors du développement.

Les fichiers suivants décrivent les efforts de développement de l'équipe de portage d'OpenSSH. Les fichiers sont signés avec la clé PGP publique contenue dans le fichier DJM-GPG-KEY.asc disponible sur le site ftp. Cette clé est aussi disponible sur le réseau des serveurs de clés.

Pour les développeurs

Des snapshots représentant les derniers développements en cours sont mis à disposition tous les soirs dans le répertoire snapshots/ sur les sites miroirs. Les versions portables d'OpenSSH sont aussi disponibles par l'intermédiaire du CVS anonyme. Pour utiliser ce moyen d'accès, vous aurez besoin de saisir les commandes suivantes : 

export CVSROOT=openssh@anoncvs.at.openbsd.org:/cvs
export CVS_RSH=/usr/bin/ssh
cvs get openssh
(Ce dépôt n'est pas protégé par mot de passe. Et bien entendu, il est en lecture seule. N'essayer donc pas d'y effectuer des archivages). Ce dépôt est fourni et maintenu par Wim Vandeputte.

Les développeurs doivent lire les lignes directrices suivantes avant de soumettre du code. Les correctifs doivent être envoyés au format diff unifié.
