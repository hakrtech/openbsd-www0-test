Java

On recommande les clients "libre" suivant pour interagir avec OpenSSH en Java:

  • MindTerm 1 implémente SSH1 uniquement. Licence GPL.

    "MindTerm est un client SSH entièrement LIBRE(*) écrit en Java à 100% (non certifié) supportant le SSH1. Il est distribué avec le code source puisque nous croyons que la sécurité lié à un logiciel ne peut pas être autrement pris sérieusement en compte. MindTerm peut être exécuter comme un programme autonome ou sous la forme d'un applet dans une page web. Il peut être exécuté avec ou sans GUI (autonome). Il a deux particularités notable qui le différencie de quelques autres clients, transfert de fichier par scp intégré et un tunnel ftp spécial qui utilise le ftpd "ordinaire" "derrière" sshd."

    NOTE: Mindterm 2 n'est pas libre.

  • Java-SSH implémente SSH1 uniquement, par Cédric Gourio. Mélange déroutant de licences non commerciale et GPL.

    "Cet applet implémente le protocole 1.5 de SSH. L'émulation terminal est presque compatible VT100. Il fournit des communications sécurisées sur des réseaux non sécurisés. Toutes les communications sont automatiquement et d'une manière transparente chiffrées. Le chiffrage est aussi utilisé pour protéger l'intégrité."

  • SSH Plugin implémente SSH1 uniquement, par Matthias L. Jugel et Marcus Meißner. Licence GPL.

    "Cette implémentation de SSH fournit un chiffrage IDEA et RSA/PKCS#1 et a été à l'origine écrite en 1998 par Cédric Gourio. Il a utilisé une partie de l'ancien applet de Telnet en Java pour écrire son client et donc le code est similaire à notre l'applet."

  • SSH Toolkits for Java. Licence LGPL/GPL.

    "Nous avons actuellement trois projets basés sur LGPL/GPL : J2SSH, une librairie SSH2, SSHTerm, une application terminal client , et un serveur SSH Windows."
