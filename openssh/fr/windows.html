Windows

Les clients "libres" suivants sont recommandés pour interagir avec OpenSSH pour les machines sous Windows:

  • PuTTY implémente SSH1+SSH2. PSCP, un programme type scp pour Windows, est aussi disponible.

    PuTTY est disponible sous licence MIT (type BSD).

    "PuTTY est une implémentation libre de Telnet et SSH pour les plates- formes Win32, écrite et maintenue principalement par Simon Tatham, qui vit en Grande-Bretagne."

  • TTSSH (SSH1) implémente SSH1 uniquement, par Robert O'Callahan.

    "TTSSH est un client SSH libre pour Windows. Il est sous la forme d'une extension DLL pour Teraterm Pro. Teraterm Pro est un excellent client telnet/émulateur libre sous windows, et les sources sont disponibles. TTSSH ajoute la possibilité à Teraterm Pro de pouvoir faire du SSH sans sacrifier aucune des fonctionnalités existantes de TeraTerm. TTSH est aussi libre à télécharger et à utiliser et les sources sont aussi disponibles, avec une licence open source. En outre, TTSSH a été entièrement développé en Australie [...]."

  • Cygwin (logiciel POSIX sous Windows)

    OpenSSH (protocole SSH1 et SSH2) peut fonctionner avec Cygwin sous Windows en utilisant la version portable d'OpenSSH.

  • MSSH

    MSSH du Metropolitan State College de Denver supporte le protocole SSH1 et fonctionne sous Windows 95 et Windows 98.

  • Secure iXplorer

    Secure iXplorer est une interface graphique pour pscp.exe de PuTTY.

  • WinSCP

    WinSCP est un programme type scp(1) pour Windows, avec PuTTY intégré.

  • FileZilla

    FileZilla est un puissant client FTP pour Windows 9x, ME, NT4, 2000 et XP. Il a été conçu dans une optique d'ergonomie et d'un support de fonctionnalités aussi complet que possible, tout en étant rapide et fiable. Il supporte SFTP.
