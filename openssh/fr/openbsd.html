OpenSSH pour OpenBSD

Etant donné que OpenSSH est développé par l'équipe OpenBSD, OpenSSH est inclus dans le système d'exploitation de base, et cela depuis la version 2.6 de OpenBSD. Il n'y a donc rien à installer ou à ajouter à votre système. Cependant, vous souhaiteriez peut-être effectuer des mises à jour pour disposer de nouvelles fonctionnalités ou d'autres correctifs.

Vous pouvez télécharger la dernière version de OpenSSH à partir de ces miroirs ftp.

Les versions suivantes de OpenSSH sont disponibles pour l'installation sur des machines OpenBSD :

  • OpenSSH 4.1: May 26, 2005

    ftp://ftp.openbsd.org/pub/OpenBSD/OpenSSH/openssh-4.1.tar.gz

    Si vous souhaitez installer OpenSSH 4.0 sur OpenBSD 3.5, vous aurez besoin du correctif suivant :
    ftp://ftp.openbsd.org/pub/OpenBSD/OpenSSH/openbsd35_4.1.patch.

    # cd /usr/src/usr.bin
# tar xvfz .../openssh-4.1.tar.gz
# cd ssh
# patch -p0 < /path/to/openbsd35_4.1.patch
# make obj
# make cleandir
# make depend
# make
# make install
# cp ssh_config sshd_config /etc/ssh
    puis redémarrez sshd.

  • OpenSSH 4.0 : 9 Mars 2005

    ftp://ftp.openbsd.org/pub/OpenBSD/OpenSSH/openssh-4.0.tgz

    Si vous souhaitez installer OpenSSH 4.0 sur OpenBSD 3.5 ou une version antérieure, vous aurez besoin du correctif suivant :
    ftp://ftp.openbsd.org/pub/OpenBSD/OpenSSH/openbsd35_4.0.patch.

    # cd /usr/src/usr.bin
# tar xvfz .../openssh-4.0.tgz
# cd ssh
# patch -p0 < /path/to/openbsd3x_4.0.patch
# make obj
# make cleandir
# make depend
# make
# make install
# cp ssh_config sshd_config /etc/ssh
# mkdir /var/empty
    puis redémarrez sshd. Pour disposer de la fonctionnalité de séparation de privilèges de sshd(8), sur certaines anciennes versions d'OpenBSD, un nouvel utilisateur et un nouveau groupe appelés sshd ont été ajoutés au système. Ajoutez l'utilisateur suivant en utilisant vipw(8) : 
    sshd:*:27:27::0:0:sshd privsep:/var/empty:/sbin/nologin

    Et la ligne suivante à /etc/group : 

    sshd:*:27:
_sshagnt:*:34:
