OpenSSH pour OpenBSD

Etant donné que OpenSSH est développé par l'équipe OpenBSD, OpenSSH est inclus dans le système d'exploitation de base, et cela depuis la version 2.6 de OpenBSD. Il n'y a donc rien à installer ou à ajouter à votre système. Cependant, vous souhaiteriez peut-être effectuer des mises à jour pour disposer de nouvelles fonctionnalités ou d'autres correctifs.

Vous pouvez télécharger la dernière version de OpenSSH à partir de ces miroirs ftp.

Les versions suivantes de OpenSSH sont disponibles pour l'installation sur des machines OpenBSD :

  • OpenSSH 3.5 : 14 Octobre 2002

    ftp://ftp.openbsd.org/pub/OpenBSD/OpenSSH/openssh-3.5.tgz

    OpenSSH 3.5 se compilera proprement sur votre système OpenBSD 3.1, sans modifications : 

    # cd /usr/src/usr.bin
# tar xvfz .../openssh-3.5.tgz
# cd ssh
# make obj
# make cleandir
# make depend
# make
# make install
# cp ssh_config sshd_config /etc/ssh
# mkdir /var/empty
    Pour disposer de la fonctionnalité de séparation de privilèges de sshd(8), un nouvel utilisateur et un nouveau groupe appelés sshd ont été ajoutés au système. Ajoutez l'utilisateur suivant en utilisant vipw(8) : 

    sshd:*:27:27::0:0:sshd privsep:/var/empty:/sbin/nologin

    Et la ligne suivante à /etc/group :

    sshd:*:27:
_sshagnt:*:34:
    Si vous ne pouvez pas ajouter le nouvel utilisateur et les nouveaux groupes, vous aurez aussi besoin du correctif suivant :
    ftp://ftp.openbsd.org/pub/OpenBSD/OpenSSH/openbsd31_3.5.patch. 

    # cd /usr/src/usr.bin
# tar xvfz .../openssh-3.5.tgz
# cd ssh
# patch -p0 < .../openbsd31_3.5.patch
# make obj
# make cleandir
# make depend
# make
# make install
# cp ssh_config sshd_config /etc
# mkdir /var/empty

    Remarque : Il n'est pas possible de compiler le code source ci-dessus sur des systèmes OpenBSD antérieurs car le support IPV6 dans la libc était incomplet.
    Si vous effectuez l'installation sur OpenBSD 2.6, vous aurez aussi besoin du correctif suivant :
    ftp://ftp.openbsd.org/pub/OpenBSD/OpenSSH/openbsd26_2.2.0.patch.

    Pour effectuer l'extraction et l'installation sur votre système OpenBSD 2.6 :

    # cd /usr/src/usr.bin

# patch -p0 < openbsd26_2.2.0.patch
# cd ssh
uncomment 'CFLAGS+=-DHAVE_LOGIN_CAP' in sshd/Makefile
# make obj
# make cleandir
# make depend
# make
# make install
# cp ssh_config sshd_config /etc

    Notez que l'utilisation de OpenSSH 2.2.0 sur OpenBSD 2.6 nécessitera l'un des packetages ssl27.tgz ousslUSA27.tgz au lieu des variantes 2.6. Cela met à jour vos librairies ssl avec une nouvelle version de OpenSSL, qui contiennent de nouveaux appels requis par OpenSSH. Et si vous voulez utiliser SSH2 alors que vous n'avez pas déjà une clé DSA, vous devez en créer une en utilisant 

    # /usr/bin/ssh-keygen -d -f /etc/ssh_host_dsa_key -N ''
