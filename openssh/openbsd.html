OpenSSH for OpenBSD

Since OpenSSH is developed by the OpenBSD group, OpenSSH is included in the base operating system, starting at OpenBSD release 2.6. So there's nothing to install or add to your system. However, due to newer features or other patches, you may wish to upgrade.

You can download the current OpenSSH release from these ftp mirrors.

The following versions of OpenSSH are available for putting on your OpenBSD machines:

  • OpenSSH 3.0.1: November 15, 2001

    ftp://ftp.openbsd.org/pub/OpenBSD/OpenSSH/openssh-3.0.1.tgz

    OpenSSH 3.0.1 will compile cleanly on your OpenBSD 3.0 system, without modifications: 

    # cd /usr/src/usr.bin
# tar xvfz .../openssh-3.0.1.tgz
# cd ssh
# make obj
# make cleandir
# make depend
# make
# make install
# cp ssh_config sshd_config /etc
    Make sure you have 'HostKey /etc/ssh_host_dsa_key' in sshd_config.

    If you are installing on OpenBSD 2.8 or 2.9, you also need the following patch:
    ftp://ftp.openbsd.org/pub/OpenBSD/OpenSSH/openbsd2x_3.0.1.patch.

    To extract and install it on your OpenBSD 2.8 or 2.9 system:

    # cd /usr/src/usr.bin
# tar xvfz .../openssh-3.0.1.tgz
# cd ssh
# patch -p0 < .../openbsd2x_3.0.1.patch
# make obj
# make cleandir
# make depend
# make
# make install
# cp ssh_config sshd_config /etc
    If you are installing on OpenBSD 2.7, you also need the following patch:
    ftp://ftp.openbsd.org/pub/OpenBSD/OpenSSH/openbsd27_3.0.1.patch.

    To extract and install it on your OpenBSD 2.7 system:

    # cd /usr/src/usr.bin
# tar xvfz .../openssh-3.0.1.tgz
# cd ssh
# patch -p0 < .../openbsd27_3.0.1.patch
# make obj
# make cleandir
# make depend
# make
# make install
# cp ssh_config sshd_config /etc
    Note: This above source fails to build on earlier OpenBSD, because IPV6 support in libc was incomplete.

  • OpenSSH 2.2.0: Aug 31, 2000

    ftp://ftp.openbsd.org/pub/OpenBSD/OpenSSH/openssh-2.2.0.tgz

    To extract and install it on your OpenBSD 2.7 system:

    # cd /usr/src/usr.bin
# tar xvfz .../openssh-2.2.0.tgz
# cd ssh
comment 'CFLAGS+=-DHAVE_LOGIN_CAP' in sshd/Makefile, using a # character.
# make obj
# make cleandir
# make depend
# make
# make install
# cp ssh_config sshd_config /etc
    Note: This above source fails to build on earlier OpenBSD, because IPV6 support in libc was incomplete.
    If you are installing on OpenBSD 2.6, you also need the following patch:
    ftp://ftp.openbsd.org/pub/OpenBSD/OpenSSH/openbsd26_2.2.0.patch.

    To extract and install it on your OpenBSD 2.6 system:

    # cd /usr/src/usr.bin
# tar xvfz .../openssh-2.2.0.tgz
# patch -p0 < openbsd26_2.2.0.patch
# cd ssh
uncomment 'CFLAGS+=-DHAVE_LOGIN_CAP' in sshd/Makefile
# make obj
# make cleandir
# make depend
# make
# make install
# cp ssh_config sshd_config /etc

    Note that using OpenSSH 2.2.0 on OpenBSD 2.6 will require one of the ssl27.tgz or sslUSA27.tgz packages instead of the 2.6 variants. This updates your ssl libraries to a newer version of OpenSSL, which contains new calls which OpenSSH requires. Also, if you want to use SSH2, and do not already have a DSA key, you need to create one using 

    # /usr/bin/ssh-keygen -d -f /etc/ssh_host_dsa_key -N ''
