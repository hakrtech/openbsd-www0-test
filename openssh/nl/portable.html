Portable OpenSSH

De normale ontwikkeling van OpenSSH produceert een zeer kleine, veilige en gemakkelijk onderhoudbare versie voor het OpenBSD project. Het OpenSSH Portabiliteitsteam voegt portabiliteitscode aan die zuivere versie toe zodat OpenSSH op veel besturingssystemen kan draaien (Helaas moeten zij met *veel* verschillen tussen Unix besturingssystemen rekening houden, niet in de laatste plaats omdat OpenSSH een vorm van authenticatie bevat).

De OpenSSH die hier wordt aangeboden, is ontworpen om op de volgende Unix besturingssystemen te draaien:

  • AIX
  • HP-UX
  • Irix
  • Linux
  • NeXT
  • SCO
  • SNI/Reliant Unix
  • Solaris
  • Digital Unix/Tru64/OSF
  • Mac OS X
  • Cygwin
  • ... en er komen steeds meer bij.

De portable OpenSSH volgt de ontwikkeling van de officiële versie, maar ze worden niet tegelijkertijd uitgebracht. Portable versies worden aangeduid met een 'p' (b.v. 4.0p1). De officiële OpenBSD broncode zal nooit het achtervoegsel 'p' gebruiken, maar zal de versienummer verhogen wanneer ze 'stabiele punten' bereiken in hun ontwikkeling.

De volgende bestanden beschrijven de ontwikkelingen van het OpenSSH portabiliteitsontwikkelteam. De bestanden zijn gesigneerd met de PGP publieke sleutel in het bestand DJM-GPG-KEY.asc op de ftp-site. Deze sleutel is ook beschikbaar via het key server netwerk.

  • Portabiliteits ChangeLog
  • Installatie-instructies
  • FAQ
  • Dagelijkse snapshots zijn beschikbaar in de snapshots/ directory op de mirror sites.

Ontwikkelaars

Elke nacht worden bleeding-edge snapshots hier beschikbaar gemaakt en in de snapshots/ directory op de mirror sites. De portable versie van OpenSSH is ook beschikbaar via anonieme CVS. Om deze te verkrijgen moet u de volgende commando's uitvoeren: 

export CVSROOT=anoncvs@anoncvs.mindrot.org:/cvs
export CVS_RSH=/usr/bin/ssh
cvs get openssh
(Er staat geen paswoord op deze repository. En ja, hij is read-only, dus probeer niet om er naar te committen.) Deze repository wordt aangeboden en onderhouden door Damien Miller. De SSH key fingerprints voor deze repository zijn: 
256 f2:5f:5c:32:1f:7d:06:b7:88:19:2f:56:5e:6b:68:56 ssh_host_ecdsa_key.pub
2048 8e:09:e9:fb:dc:af:e4:a5:e6:a0:e2:28:59:63:84:17 ssh_host_rsa_key.pub
1024 3e:38:50:b8:d7:b7:dc:bc:13:c4:d8:76:c8:31:e3:44 ssh_host_dsa_key.pub

Eens uitgecheckt, moet het configure bestand aangemaakt worden door autoreconf uit te voeren uit het autoconf package (de vereiste versie wordt vermeld in het INSTALL bestand). Het zal ook opnieuw moeten uitgevoerd worden als er veranderingen worden aangebracht in het configure.ac bestand.

Ontwikkelaars wordt gevraagd om deze richtlijnen te lezen en toe te passen aan hun code. Patches moeten worden opgestuurd in unified diff formaat.
