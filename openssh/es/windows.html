Windows

Se recomiendan los siguientes clientes «libres» para la interoperación con OpenSSH desde sistemas Windows:

  • PuTTY es una implementación para el protocolo SSH1+SSH2.

    También existe PSCP, un programa para Windows al estilo scp.

    PuTTY se encuentra disponible bajo la licencencia MIT (del estilo BSD).

    «PuTTY es una implementación libre de Telnet y SSH para plataformas Win32, diseñado y mantenido principalmente por Simon Tatham desde Gran Bretaña.»

  • TTSSH (SSH1) es una implementación sólo para el protocolo SSH1 de Robert O'Callahan.

    «TTSSH es un cliente libre para Windows. Está implementado como una extensión DLL para Teraterm Pro. Teraterm Pro es un fantástico cliente emulador de terminal y cliente de telnet libre para Windows, y su código fuente se encuentra disponible al público. TTSSH añade funcionalidades de SSH a Teraterm Pro sin sacrificar ninguna de las funcionalidades ya existentes de Teraterm. TTSSH está disponible bajo una licencia abierta. Es más, TTSSH ha sido desarrollado completamente en Australia [...].»

  • Cygwin (software POSIX sobre Windows)

    OpenSSH (protocolos SSH1 y SSH2) funciona en Windows con Cygwin usando la versión portable de OpenSSH.

  • MSSH

    MSSH, del Metropolitan State College of Denver, contiene soporte para el protocolo SSH1 en Windows 95 y Windows 98.

  • Secure iXplorer GPL

    Secure iXplorer GPL es una interfaz gráfica para pscp.exe de PuTTY

  • WinSCP

    WinSCP es un programa scp(1) para Windows con PuTTY integrado.

  • FileZilla

    FileZilla es un cliente FTP poderoso para Windows 9x, ME, NT4, 2000 y XP. Ha sido diseñado para ser fácil de usar y con soporte para tantas características como sean posibles sin sacrificar la velocidad ni la confiabilidad. Soporta SFTP.
