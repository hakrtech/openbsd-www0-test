Windows y Mac

Se recomiendan los siguientes clientes «libres» para la interoperación con OpenSSH desde sistemas Windows:

  • PuTTY es una implementación para el protocolo SSH1+SSH2.

    También existe PSCP, un programa para Windows al estilo scp.

    PuTTY se encuentra disponible bajo la licencencia MIT (del estilo BSD). El modo SSH2 de Putty no contiene soporte para la autenticación del usuario con claves DSA. Sin embargo, existe soporte para la autenticación del usuario con claves RSA para ambas versiones del protocolo SSH.

    «PuTTY es una implementación libre de Telnet y SSH para plataformas Win32, diseñado y mantenido principalmente por Simon Tatham desde Gran Bretaña.»

  • TTSSH (SSH1) es una implementación sólo para el protocolo SSH1 de Robert O'Callahan.

    «TTSSH es un cliente libre para Windows. Está implementado como una extensión DLL para Teraterm Pro. Teraterm Pro es un fantástico cliente emulador de terminal y cliente de telnet libre para Windows, y su código fuente se encuentra disponible al público. TTSSH añade funcionalidades de SSH a Teraterm Pro sin sacrificar ninguna de las funcionalidades ya existentes de Teraterm. TTSSH está disponible bajo una licencia abierta. Es más, TTSSH ha sido desarrollado completamente en Australia [...].»

  • Telneat es un cliente de telnet/SSH1/SSH2 para Windows creado por Kosta Schegloff.

    Ofrece las siguientes funcionalidades:

    • soporte completo para telnet
    • soporte para SSH (compresión con IDEA, DES, 3DES, ArcFour y BlowFish)
    • soporte para SSH2
    • soporte para termcap
    • soporte para Win95, OSR2, Win98, WinNT y Win2000
    • soporte externo para plugins de cifrado SSH
    • hasta 10 consolas
    • funcionalidad de «cortar y pegar» con el ratón
    • memoria de pasado de páginas
    • autoingreso
    • tablas de recodificado configurables
    • configuración independiente para cada anfitrión
    • guiones de ingreso ("login scripts")

  • Cygwin (software POSIX sobre Windows)

    OpenSSH (protocolos SSH1 y SSH2) funciona en Windows con Cygwin usando la versión portable de OpenSSH.

  • MSSH

    MSSH, del Metropolitan State College of Denver, contiene soporte para el protocolo SSH1 en Windows 95 y Windows 98.

  • Secure iXplorer

    Secure iXplorer es una interfaz gráfica para pscp.exe de PuTTY

  • WinSCP

    WinSCP es un programa scp(1) para Windows con PuTTY integrado.

Se recomiendan los siguientes clientes para la interoperación con OpenSSH desde sistemas Mac:

  • NiftyTelnet 1.1 SSH es una implementación sólo para el protocolo SSH1 que viene con un programa del estilo de scp. Está escrito por Jonas Wallden.

    «NiftyTelnet 1.1 SSH r3 es una versión mejorada de la aplicación NiftyTelnet 1.1 de Chris Newman, con soporte para sesiones de terminal cifradas usando el protocolo SSH ("Secure Shell"). Por favor, lea el fichero Readme que incluye antes de distribuir esta versión.»

  • MacSSH es una implementación sólo para SSH2.

    «MacSSH es una versión modificada de BetterTelnet con soporte para SSH2. (...) El único cliente de SSH2 para MacOS que he podido encontrar es un producto comercial que cuesta más de $100, y que cuelga mi Mac al cerrar la sesión... Ya que lo mejor es hacer las cosas por uno mismo, aquí está MacSSH.»
