Alternativas para Unix

Si prefiere no usar OpenSSH en Unix, los siguientes clientes «libres» se encuentran disponibles para su interoperación con OpenSSH:

  • OSSH es una implementación sólo para SSH1. Licencia BSD.

    Björn Grönvall fue el primero que recordó a nuestra comunidad de que existían versiones muy antiguas del auténtico ssh que estaban bajo licencias libres. Ésta es su propia implementación, basada en el código viejo de Tatu. OpenSSH se basó en una versión antigua de OSSH, pero para OpenSSH se añadió soporte para el protocolo SSH2 y se extendió el existente para SSH1.

  • LSH/psst es una implementación sólo para SSH2, de Niels Möller en Suecia. Licencia GPL.

    «ssh usa técnicas criptográficas para mejorar la seguridad en las tareas diarias de computación distribuida, como transferencias de ficheros, ingresos remotos, y ejecución de órdenes desde remoto. Su intención es la de ser una alternativa (más) segura a telnet, ftp, rsh, rlogin y rexec, así como requerir mucho menos esfuerzo que la configuración de Kerberos o IPSEC.»
