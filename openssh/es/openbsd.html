OpenSSH para OpenBSD

Ya que el desarrollo de OpenSSH lo lleva a cabo el grupo de OpenBSD, OpenSSH va incluido en el sistema operativo básico, desde la versión 2.6 de OpenBSD. Así que no hay nada que deba ser instalado o añadido a su sistema. Sin embargo, es probable que quiera actualizarlo a causa de la aparición de funcionalidades más recientes u otros parches.

Las siguientes versiones de OpenSSH se encuentran disponibles para instalarlas en sus máquinas con OpenBSD:

  • OpenSSH 2.1.1: 8 de mayo 2.000

    ftp://ftp.openbsd.org/pub/OpenBSD/OpenSSH/openssh-2.1.1.tgz

    Para extraerlo e instalarlo en su sistema OpenBSD 2.7:

    # cd /usr/src/usr.bin
# tar xvfz .../openssh-2.1.1.tgz
# cd ssh
# make obj
# make cleandir
# make depend
# make
# make install
# cp ssh_config sshd_config /etc
    Nota: El código fuente de arriba no compila en versiones anteriores de OpenBSD debido a que el soporte para IPV6 en libc estaba incompleto.
    Si va a instalarlo en OpenBSD 2.6, también necesitará el siguiente parche:
    ftp://ftp.openbsd.org/pub/OpenBSD/OpenSSH/openbsd26_2.1.1.patch.

    Para extraerlo e instalarlo en su sistema OpenBSD 2.6:

    # cd /usr/src/usr.bin
# tar xvfz .../openssh-2.1.1.tgz
# patch -p0 < openbsd26_2.1.1.patch
# cd ssh
# make obj
# make cleandir
# make depend
# make
# make install
# cp ssh_config sshd_config /etc

    El uso de OpenSSH 2.1.1 en OpenBSD 2.6 requerirá el paquete ssl27.tgz o el paquete sslUSA27.tgz en lugar de las variantes de la versión 2.6. De este modo actualizará sus bibliotecas ssl a una versión más nueva de OpenSSL, que contiene nuevas llamadas requeridas por OpenSSH. Además, si quiere usar SSH2 y todavía no tiene una clave DSA, necesita generar una usando la orden 

    # /usr/bin/ssh-keygen -d -f /etc/ssh_host_dsa_key -N ''
