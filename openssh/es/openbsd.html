OpenSSH para OpenBSD

Puesto que el desarrollo de OpenSSH lo lleva a cabo el grupo de OpenBSD, OpenSSH va incluido en el sistema operativo básico desde la versión 2.6 de OpenBSD. Así pues, no hay nada que deba ser instalado o añadido a su sistema. Sin embargo, debido a la aparición de funcionalidades más recientes o de otros parches, es probable que desee actualizar la versión que se instaló con el sistema.

Puede obtener la última versión de OpenSSH desde las réplicas de ftp.

Las siguientes versiones de OpenSSH se encuentran disponibles para su instalación en máquinas con el sistema operativo OpenBSD:

  • OpenSSH 3.6.1: 1 de abril de 2003

    ftp://ftp.openbsd.org/pub/OpenBSD/OpenSSH/openssh-3.6.1.tgz

    OpenSSH 3.6.1 compila perfectamente en sistemas OpenBSD 3.2, sin modificaciones. 

    # cd /usr/src/usr.bin
# tar xvfz .../openssh-3.6.1.tgz
# cd ssh
# make obj
# make cleandir
# make depend
# make
# make install
# cp ssh_config sshd_config /etc/ssh
# mkdir /var/empty
    Para el soporte de la funcionalidad de separación de privilegios de sshd(8) en versiones antiguas de OpenBSD, es necesario añadir un nuevo usuario y grupo llamado sshd al sistema. Añada la siguiente entrada de usuario usando vipw(8): 
    sshd:*:27:27::0:0:sshd privsep:/var/empty:/sbin/nologin

    Añada lo siguiente a /etc/group: 

    sshd:*:27:
_sshagnt:*:34:
    Si no puede añadir el nuevo usuario y grupos, entonces necesita el siguiente parche:
    ftp://ftp.openbsd.org/pub/OpenBSD/OpenSSH/openbsd31_3.6.1.patch.

    # cd /usr/src/usr.bin
# tar xvfz .../openssh-3.6.1.tgz
# cd ssh
# patch -p0 < .../openbsd31_3.6.1.patch
# make obj
# make cleandir
# make depend
# make
# make install
# cp ssh_config sshd_config /etc
# mkdir /var/empty
