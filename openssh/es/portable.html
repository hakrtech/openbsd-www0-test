OpenSSH portable

El equipo de desarrollo de OpenSSH produce una versión para el proyecto OpenBSD muy pequeña, segura, y fácil de mantener. El Equipo de Portabilidad de OpenSSH, a partir de esa versión pura, le añade código de portabilidad para que OpenSSH pueda funcionar en otros sistemas operativos (desafortunadamente, y debido a que OpenSSH usa autenticación, tiene muchas diferencias entre los distintos sistemas operativos Unix).

OpenSSH está diseñado para funcionar en los siguientes sistemas operativos Unix:

  • AIX
  • HP-UX
  • Irix
  • Linux
  • NeXT
  • SCO
  • SNI/Reliant Unix
  • Solaris
  • Digital Unix/Tru64/OSF
  • MacOS X
  • Cygwin
  • ... y más que van siendo añadidos continuamente.

La versión portable de OpenSSH sigue el desarrollo de la versión oficial, pero las versiones finales no están sincronizadas. Las versiones portables están marcadas con una 'p' (v.g. 3.6.1p2). En el código fuente oficial para OpenBSD nunca se usará el sufijo 'p', sino que se incrementará el número de la versión cuando llegue a «puntos estables» en el desarrollo.

Los siguientes ficheros describen los esfuerzos llevados a cabo para su desarrollo por parte del equipo de portabilidad de OpenSSH:

  • Registro de los cambios en las versiones portables (ChangeLog)
  • Instrucciones de instalación
  • Preguntas frecuentes
  • Hay versiones premiliminarias diarias disponibles desde el directorio snapshots/ de las réplicas de FTP

Desarrolladores

Las versiones preliminares extremas están disponibles desde el directorio snapshots/ de las réplicas de FTP. Las versiones portables de OpenSSH también están disponibles mediante CVS anónimo. Para acceder a éstas debe usar las siguientes órdenes: 

export CVSROOT=openssh@anoncvs.be.openbsd.org:/cvs
export CVS_RSH=/usr/bin/ssh
cvs get openssh

(no hay contraseña en este repositorio. Y sí, es de sólo lectura, así que no intente subir nada). Este repositorio lo ha cedido y lo mantiene Wim Vandeputte.

Pedimos a los desarrolladores que lean y sigan esta guía de estilo cuando entreguen código. Los parches se deben enviar en formato diff unificado.
