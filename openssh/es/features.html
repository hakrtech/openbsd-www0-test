OpenSSH es una implementación libre del paquete "secure shell" que provee a los servicios de red de cifrado como ingreso ("login") remoto o transferencia de archivos remota.

La lista siguiente enumera las características de Openssh:

 Características

  • Proyecto de Código Abierto
  • Licencia Libre
  • Cifrado Fuerte (3DES, Blowfish)
  • Envío por X11 (cifra el tráfico de X Windows)
  • Envío por Puertos (cifra los canales para protocolas legados)
  • Autentificación Fuerte (Clave Pública, Contraseña para Uso de Una Sola Vez y Autentificación con Kerberos)
  • Envío por Agente (Un Sólo Ingreso)
  • Interoperabilidad (Conforme con las normativas sobre protocolos remotos de SSH 1.3 y 1.5)
  • Pase de Tickets Kerberos y AFS
  • Compresión de Datos

 Proyecto de Código Abierto

El código fuente de Openssh es de libre disponibilidad para todo aquel que desee obtenerlo en Internet. Se anima a todo el que desee reutilizar el código o hacer una auditoría sobre él a que lo lleve a cabo. La revisión del código asegura que cualquier persona pueda encontrar y corregir errores. El resultado es un código seguro.

 Licencia Libre

OpenSSH no se encuentra bajo ninguna licencia restrictiva. Se puede usar para cualquier propósito, y esto incluye su uso comercial. Sin embargo, el los EE.UU. debe tener en cuenta que la versión 1 del protocolo de SSH requiere el uso del algoritmo de clave pública RSA. La licencia de RSA expirar en septiembre de 2.000.

Todos los componentes de naturaleza restrictiva (o sea, patentes, ver ssl) han sido eliminados del código fuente; los componentes bajo licencia o patentados se sacan de biblioteca externas (v.g. OpenSSL).

 Cifrado Fuerte

OpenSSH usa 3DES y Blowfish como algoritmos de cifrado. Ambos están libres de patentes. 3DES es un algoritmo de cifrado muy conocido y que ha pasado la prueba del tiempo, que provee cifrado fuerte. Blowfish es un bloque de cifrado rápido inventado por Bruce Schneier, que pueden usarlo aquéllos que requieran un cifrado más rápido. El cifrado empieza antes de la autentificación, y ninguna contraseña ni otro tipo de información se transmite sin cifrar. El cifrado también se utiliza como protección contra paquetes falsificados.

 Envío por X11

El envío por X11 permite el cifrado del tráfico en entornos X windows remotos, de tal modo que nadie pueda fisgonear en sus xterm remotas o insertar órdenes dañinas. El programa activa DISPLAY automáticamente en el servidor, y envía cualquier conexión de X11 por un canal seguro. Información falsa sobre Xauthority se genera automáticamente y se envía a la máquina remota; el programa cliente en la máquina local examina las conexiones entrantes de X11 y reemplaza los datos de autorización falsos con datos reales (pero nunca le pasa a la máquina remota la información real).

 Envío por Puertos

El envío por puertos permite enviar conexiones de TCP/IP a una máquina remota por un canal cifrado. Las aplicaciones típicas de Internet como POP se pueden asegurar de este modo.

 Autentificación Fuerte

Una fuerte autentificación protege contra varios problemas de seguridad, como por ejemplo suplantación de IP, rutas falsas, y fisgoneo de DNS. Los métodos de autentificación son: .rhosts junto con huésped de autentificación basado en RSA, autentificación RSA pura, contraseñas para uso de una sola vez, y finalmente autentificación mediante Kerberos.

 Envío por Agente

Un agente de autentificación que se encuentre en la estación de trabajo o el portátil de un usuario, se puede usar para contener las claves de autentificación de RSA. OpenSSH envía la conexión automáticamente al agente de autentificación por medio de cualquier conexión y de este modo no existe la necesidad de guardar las claves de autentificación de RSA en ninguna máquina de la red (exceptuando la máquina del usuario). Los protocolos de autentificación nunca revelan las claves; sólo se pueden usar para verificar que el agente del usuario tenga cierta clave. El agente podría hacer uso de una tarjeta inteligente para llevar a cabo todas las computaciones de autentificación.

 Interoperabilidad

OpenSSH tiene soporte para las versiones 1.3 y 1.5 del protocolo remoto SSH, permitiendo la comunicación con la mayoría de implementaciones de ssh para UNIX, Windows y otras implementaciones comerciales.

OpenSSH todavía no dispone de soporte para la versión 2.0 del protocolo remoto SSH, pero creemos que esto no es tan importante dado que la mayoría de programas ssh se pueden comunicar con el protocolo 1.5. Además, existen unas restricciones de la licencia no comercial bajo la que se encuentra la principal implementación de SSH 2.0.

 Pases de Tickets Kerberos y AFS

OpenSSH también pasa tickets para Kerberos y AFS en la máquina remota. Un usuario puede por tanto acceder a todos sus servicios de Kerberos y AFS sin la necesidad de introducir una contraseña de nuevo.

 Compresión de Datos

La compresión de datos antes del cifrado mejora los resultados en los enlaces con redes lentas.
