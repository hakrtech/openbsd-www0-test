Списки рассылки

Developers private list -- личная рассылка разработчиков

Связанные с безопасностью вопросы могут быть направлены в частную/личную почтовую рассылку openssh@openssh.com, которая предназначенна только для OpenSSH разработчиков. Она не подходит для обсуждения общих вопросов.

Announcement list -- новостная рассылка

Для Linux/Unix-порта есть своя почтовая рассылка openssh-unix-announce@mindrot.org. Она доступна только для чтения и предназначенна в первую очередь для анонсирования новостей.

Информация о подписки и управлении списком рассылки доступна тут.

Developers public list -- открытая рассылка разработчиков

Почтовая рассылка openssh-unix-dev (openssh-unix-dev@mindrot.org) предназначена для общения с разработчиками, желающих усовершенствовать портируемую (например сделать поддержку для другой платформы) или OpenBSD версию OpenSSH.

Трафик рассылки: от нуля до двадцать писем в день. Основня тема - хакинг обоих версий OpenSSH, работа над документацией и обсуждение и/или исправление багов (если вы читали руководства). НЕ РАЗРЕШАЕТСЯ: использованике HTML, форматирование текста или больших прикрепленных файлов (attachments). Разрешается прикреплять только небольшие (например diff) файлы. Ваши просьбы о помощи, которые демонстрируют недостаточное знакомство с OpenSSH может привести только к "RTFM"-ответам, поэтому, пожалуйста, прежде чем что-либо спросить, ознакомтесь с документацией.

Обсуждения коммерческих продуктов являются *оффтопом* для этой рассылки, за исключением тех случав, когда речь идет о разработке OpenSSH (обсуждение взаимодействия).

Список открыт для неподписчиков, хотя они могут быть целью одноразового проверки адреса. Архив доступен на MARC.

Более подробная информация об этой рассылке находится на странице openssh-unix-dev@mindrot.org.

General SSH user discussion list -- основная рассылка для SSH-пользователей

Основной список рассылки SSH secureshell@securityfocus.com. Вы также можете использовать группу новостей Usenet comp.security.ssh.

Чтобы подписаться, отправьте пустое сообщение на адрес secureshell-subscribe@securityfocus.com.

Чтобы отписаться, отправьте пустое сообщение на адрес secureshell-unsubscribe@securityfocus.com.

Архив доступен на MARC.
