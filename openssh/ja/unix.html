Unix における他の選択肢

Unix で OpenSSH を使わないのであれば、他の「フリー」クライアントとして 以下のものが OpenSSH と相互運用が可能です:

  • OSSH は SSH1 プロトコルのみの実装です。BSD ライセンスです。

    私たちのコミュニティでは、Björn Grönvall は 非常に古いバージョンの、フリーのライセンスをもつ真の ssh を作った 人として記憶されています。これはその彼の手による実装です。 OpenSSH は OSSH の初期のバージョンを 元にしていますが、OpenSSH は SSH2 プロトコルと SSH1 プロトコルの 拡張をサポートしました。

  • LSH/psst は SSH2 プロトコルのみの実装で、GPL ライセンスのもとスウェーデンの Niels Möller によって作られました。

    「ssh はファイル転送やリモートログイン、リモートコマンド実行といった 日々の分散コンピューティングの作業の安全性を高めるために 暗号化技術を使っています。これは telnet や ftp, rsh, rlogin そして rexec などに代わる (より) 安全な選択肢となることを目指しており - Kerberos や IPSEC がいまやっているのよりはずっと少ない労力で セットアップできるんです」

  • FreSSH は SSH の実装のひとつで、もともと Eric Haszlakiewicz と Thor Lancelot Simon によって書かれたものです。現在は Jason R. Thorpe によって開発されています。ライセンスは BSD ライクですが、 他の人々によって開発された、異なったライセンスをもつソフトウエアも 含まれているので注意してください。
