OpenBSD 専用の OpenSSH

OpenSSH は OpenBSD グループによって開発されているため、 OpenBSD リリース 2.6 から OpenSSH は基本のオペレーティングシステムに 含まれています。ですから、お使いのシステムに追加したりインストールしたり する必要はありません。けれども新しい機能やその他のパッチのために アップグレードしたいこともあるでしょう。

OpenSSH の最新リリースは ftp ミラーサイト から ダウンロードできます。

以下の OpenSSH のバージョンを、お使いの OpenBSD マシン上で 動かすことができます:

  • OpenSSH 3.0: 2001年 11月 6日

    ftp://ftp.openbsd.org/pub/OpenBSD/OpenSSH/openssh-3.0.tgz

    OpenSSH 3.0 は OpenBSD 2.8 および 2.9 システムで、修正せず そのままコンパイルできます: 

    # cd /usr/src/usr.bin
# tar xvfz .../openssh-3.0.tgz
# cd ssh
# make obj
# make cleandir
# make depend
# make
# make install
# cp ssh_config sshd_config /etc
    必ず、sshd_config に 'HostKey /etc/ssh_host_dsa_key' という行を 追加するようにしてください。

    OpenBSD 2.7 システムで展開しインストールするには:

    # cd /usr/src/usr.bin
# tar xvfz .../openssh-3.0.tgz
# cd ssh
ここで sshd/Makefile にある 'CFLAGS+=-DHAVE_LOGIN_CAP' を、# 文字でコメントアウトします。
# make obj
# make cleandir
# make depend
# make
# make install
# cp ssh_config sshd_config /etc
    注意: 上のソースでは旧バージョンの OpenBSD ではビルドに失敗します。 これは libc の IPV6 サポートが完全ではなかったためです。

  • OpenSSH 2.2.0: 2000年 8月 31日

    ftp://ftp.openbsd.org/pub/OpenBSD/OpenSSH/openssh-2.2.0.tgz

    お使いの OpenBSD 2.7 システムで展開しインストールするには:

    # cd /usr/src/usr.bin
# tar xvfz .../openssh-2.2.0.tgz

ここで sshd/Makefile にある 'CFLAGS+=-DHAVE_LOGIN_CAP' を、# 文字でコメントアウトします。

    注意: 上のソースでは旧バージョンの OpenBSD ではビルドに失敗します。 これは libc の IPV6 サポートが完全ではなかったためです。
    OpenBSD 2.6 にインストールする場合は、以下のパッチも必要になります:
    ftp://ftp.openbsd.org/pub/OpenBSD/OpenSSH/openbsd26_2.2.0.patch

    お使いの OpenBSD 2.6 システムで展開しインストールするには:

    # cd /usr/src/usr.bin
# tar xvfz .../openssh-2.2.0.tgz
# patch -p0 < openbsd26_2.2.0.patch
# cd ssh
ここで sshd/Makefile にある 'CFLAGS+=-DHAVE_LOGIN_CAP' のコメントをはずして生かします。
# make obj
# make cleandir
# make depend
# make
# make install
# cp ssh_config sshd_config /etc

    OpenBSD 2.6 上で OpenSSH 2.2.0 を使うためには、the 2.6 varients のかわりに ssl27.tgz または sslUSA27.tgz パッケージが必要なことに 注意してください。これはお使いの ssl ライブラリを新しい OpenSSL の バージョンにアップデートするもので、OpenSSH に必要な新しい関数を含んで います。また、SSH2 を使う場合で DSA 鍵をもっていないときは、 以下のようにして鍵を生成する必要があります: 

    # /usr/bin/ssh-keygen -d -f /etc/ssh_host_dsa_key -N ''
