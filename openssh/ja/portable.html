移植版 OpenSSH

通常の OpenSSH 開発プロセスはひじょうに小さなもので、安全であり、 簡単に OpenBSD プロジェクトのバージョンを維持することができます。 OpenSSH 移植チーム は この純粋なバージョンをとってきて、これがほかの多くのオペレーティング システム上でも走るように移植可能なコードを追加します (残念ながら OpenSSH は認証を行う必要があるため、異なる Unix オペレーティングシステム の間で *たくさんの* 差が生じてしまいます)。

ここで提供されている OpenSSH は以下の Unix オペレーティング システムで走るよう設計されています:

  • AIX
  • HP-UX
  • Irix
  • Linux
  • NeXT
  • SCO
  • SNI/Reliant Unix
  • Solaris
  • Digital Unix/Tru64/OSF
  • Mac OS X
  • Cygwin
  • ... まだまだ増加中

移植版 OpenSSH は正式バージョンの開発に続きますが、これらの リリースは同時ではありません。移植版のリリースは 'p' で 表されます (例: 4.0p1 など)。これに対して正式な OpenBSD のソースに 'p' サフィックスは決してつきません。かわりに、開発の「安定点」に 達したときバージョン番号が増えるようになっています。

以下のファイルが OpenSSH 移植チームの成果です。 リリースされたファイルは PGP で署名されており、 この公開鍵は各 ftp サイトの DJM-GPG-KEY.asc というファイルにあります。 この鍵は key server network からも取得できます。

  • 移植版の ChangeLog (英語)
  • インストール方法 (英語)
  • FAQ (よくある質問と答え)
  • Daily スナップショット は、ミラーサイト の snapshots/ ディレクトリで入手できます。

開発者

開発の先陣を切っている nightly snapshot は、 ここか、 ミラーサイト の snapshots/ ディレクトリで入手できます。 移植版の OpenSSH も anonymous CVS で入手可能です。 入手するには以下のコマンドを実行する必要があります: 

export CVSROOT=anoncvs@anoncvs.mindrot.org:/cvs
export CVS_RSH=/usr/bin/ssh
cvs get openssh
(このリポジトリにアクセスするはパスワードは必要ありません。それから このリポジトリは読み込み専用なので、commit しようとしないでください。) このリポジトリは Damien Miller がメンテしています。 このリポジトリの SSH 鍵の fingerprint は以下のとおりです: 
256 f2:5f:5c:32:1f:7d:06:b7:88:19:2f:56:5e:6b:68:56 ssh_host_ecdsa_key.pub
2048 8e:09:e9:fb:dc:af:e4:a5:e6:a0:e2:28:59:63:84:17 ssh_host_rsa_key.pub
1024 3e:38:50:b8:d7:b7:dc:bc:13:c4:d8:76:c8:31:e3:44 ssh_host_dsa_key.pub

チェックアウトしたら、autoreconf コマンドを実行して configure ファイルを生成する必要があります。 このプログラムは autoconf パッケージに含まれています (要求されるバージョン番号は INSTALL ファイルの中に書かれています)。 configure.ac ファイルに変更が加えられるたびに、 このプログラムを再実行する必要があります。

開発者がコードを提供するときは これらのガイドライン を読み、これに従うことが求められます。 パッチは unified diff 形式で送るようにしてください。
