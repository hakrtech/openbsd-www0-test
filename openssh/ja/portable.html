移植版 OpenSSH

通常の OpenSSH 開発プロセスはひじょうに小さなもので、安全であり、 簡単に OpenBSD プロジェクトのバージョンを維持することができます。 OpenSSH 移植チーム は この純粋なバージョンをとってきて、これがほかの多くのオペレーティング システム上でも走るように移植可能なコードを追加します (残念ながら OpenSSH は認証を行う必要があるため、異なる Unix オペレーティングシステム の間で *たくさんの* 差が生じてしまいます)。

ここで提供されている OpenSSH は以下の Unix オペレーティング システムで走るよう設計されています:

  • AIX
  • HP-UX
  • Irix
  • Linux
  • NeXT
  • SCO
  • SNI/Reliant Unix
  • Solaris
  • Digital Unix/Tru64/OSF
  • MacOS X
  • Cygwin
  • ... まだまだ増加中

移植版 OpenSSH は正式バージョンの開発に続きますが、これらの リリースは同時ではありません。移植版のリリースは 'p' で 表されます (例: 2.9.9p2 など)。これに対して正式な OpenBSD のソースに 'p' サフィックスは決してつきません。かわりに、開発の「安定点」に 達したときバージョン番号が増えるようになっています。

以下のファイルが OpenSSH 移植チームの成果です:

  • 移植版の ChangeLog (英語)
  • インストール方法 (英語) 日本語訳: 2.9p1 の INSTALL (春山さん)
  • FAQ (よく聞かれる質問と答え)
  • Daily スナップショット (英語)

開発者

開発の先陣を切っている nightly snapshot は ここ から入手できます。 OpenSSH 移植版リリースも anonymous CVS から入手できます。 これらにアクセスするには、以下のコマンドを実行してください: 

cvs -d :pserver:cvs@bass.directhit.com:/cvs login
cvs -d :pserver:cvs@bass.directhit.com:/cvs co openssh_cvs
(パスワードの入力ではただリターンキーを押してください)。 開発者がコードを提供するときは これらのガイドライン を読み、これに従うことが求められます。 パッチは unified diff 形式で送るようにしてください。
