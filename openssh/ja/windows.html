Windows & Mac

以下の「フリー」クライアントが Windows マシンから OpenSSH と相互運用できるものとしておすすめです:

  • PuTTY は SSH1 と SSH2 プロトコルの実装です。PSCP という Windows 用の scp 風プログラムも使えます。

    PuTTY は (BSDライクな) MIT のライセンスで使用できます。

    「PuTTY は Win32 プラットフォーム用の Telnet および SSH の フリーな実装であり、基本的にイギリス在住の Simon Tatham によって 書かれ、メンテナンスされています」

  • TTSSH (SSH1) は SSH1 プロトコルのみの実装で、Robert O'Callahan の手によるものです。

    「TTSSH はフリーの Windows 用 SSH クライアントです。 これは Tera Term Pro の拡張 DLL として実装されています。 Tera Term Pro は非常によくできたフリーの Windows用 端末エミュレータ / telnet クライアントであり、そのソースも利用できます。TTSSH は Tera Term Pro の本来ある機能をまったく損うことなく、SSH の 機能を追加します。TTSSH もまたフリーでダウンロードでき、 そのソースもオープンソースライセンスで使えます。さらに TTSSH は 全部オーストラリアで開発されています (中略)。」

  • Cygwin (Windows 上で動く POSIX ソフトウエア)

    Cygwin を使えば Windows 上で OpenSSH (SSH1 および SSH2 プロトコル) が 走ります。移植版 OpenSSH を使っています。

  • MSSH

    Metropolitan State College of Denver で開発された MSSH は Windows 95 および Windows 98 上で SSH1 プロトコルをサポートしています。

  • OpenSSH for Windows

    Windows 上で走る、もうひとつの OpenSSH です…。

  • Secure iXplorer

    Secure iXplorer は PuTTY の pscp.exe のグラフィカル フロントエンドです。

  • WinSCP

    WinSCP は Windows 上で動く scp(1) プログラムで、PuTTY に統合されています。

以下の「フリー」クライアントが Macintosh マシンから OpenSSH と相互運用できるものとしておすすめです:

  • NiftyTelnet 1.1 SSH は SSH1 プロトコルのみの実装で、 scp 風のプログラムもついてきます。 Jonas Wallden によって書かれました。

    「NiftyTelnet 1.1 SSH r3 は Chris Newman の NiftyTelnet 1.1 アプリケーションに SSH (Secure Shell) プロトコルを使った 暗号化された端末セッションのサポートを追加したものです。 このバージョンを配布する前に、これに含まれている Readme ファイルを 読んでください」

  • MacSSH は SSH2 プロトコルのみの 実装です。

    「MacSSH は SSH2 サポートつきの、BetterTelnet の修正バージョンです。 (中略) 私が見つけた唯一の MacOS 用の SSH2 クライアントは 商用の製品でしたが、 $100 以上もして、おまけに私のマックでは セッションを閉じるときにクラッシュしてしまいました ... 結局自分でやるのが 一番ということになり、MacSSH ができあがったわけです」
