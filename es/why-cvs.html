Por qué usan CVS los desarrolladores de OpenBSD

El Concurrent Version System (CVS, Sistema de Versiones Concurrentes) fue seleccionado por el grupo del proyecto OpenBSD. Con CVS, alrededor de setenta desarrolladores pueden actualizar el código fuente (alrededor de 1.2GB y 140.000 ficheros), que comprende el Sistema Operativo, portes y paquetes y el Sistema X Window. Un gran número de usuarios de OpenBSD pueden bajarse el código fuente del Sistema Operativo por medio de CVS anónimo, un procedimiento que es mucho más fiable que el sup(1) (Software Upgrade Protocol, Protocolo de Actualización de Software).

CVS también permite la fácil creación y mantenimiento de ramas para que podamos proveer un árbol de fuentes de una versión + parches, y un árbol de fuentes de desarrollo a la última con desarrollo paralelo. También está disponible CVSweb, una interfaz web navegable y adecuada para todo esto.

Como CVS dispone de una interfaz para Secure Shell (ssh(1)), puede proporcionar una autentificación fiable del cliente y del servidor. Esto se ajusta muy bien con el objetivo de seguridad del proyecto OpenBSD.
