Por qué usan CVS los desarrolladores de OpenBSD

CVS, Concurrent Version System (Sistema de Versiones Concurrentes) fue elegido por el grupo del proyecto OpenBSD. Con CVS, alrededor de setenta desarrolladores pueden actualizar el código fuente (más de 1349MB, alrededor de 75000 ficheros), que comprende el Sistema Operativo, portes y paquetes, y XFree86. Un gran número de usuarios de OpenBSD pueden bajarse el código fuente del Sistema Operativo por medio de CVS anónimo, un procedimiento que es mucho más fiable que el SUP (Software Upgrade Protocol, Protocolo de Actualización de Software).

CVS también permite la fácil creaci&aocute;n y mantenimiento de ramificaciones, para que podamos proveer un árbol de fuentes de una versión + parches, y un árbol de fuentes de desarrollo a la última con desarrollo paralelo.

Como CVS dispone de una interfaz para Secure Shell (ssh), puede proveer una autenticación del cliente y del servidor fiable. Esto se ajusta muy bien con el objetivo de seguridad del proyecto OpenBSD.
