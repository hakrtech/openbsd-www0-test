  • Todas las Plataformas

    • 032: PARCHE DE SEGURIDAD: 26 de octubre de 2.000
      Hay dos desbordamientos de la memoria intermedia en las bibliotecas de X11 que usan el código xtrans, que podrían constituir un "exploit". Una de esas vulnerabilidades ha sido enviada a la lista de distribución de BUGTRAQ.
      Está disponible un parche para el código que soluciona este problema.

    • 031: PARCHE DE SEGURIDAD: 18 de octubre de 2.000
      Hay varios errores en Apache, en mod_rewrite y mod_vhost_alias, que podrían causar que ficheros aleatorios fueran accesibles para usuarios de www en el servidor y ser expuestos bajo ciertas configuraciones cuando se usan estos módulos (estos módulos no se encuentran activados por definición).
      Está disponible un parche para el código que soluciona este problema.

    • 030: PARCHE DE SEGURIDAD: 10 de octubre de 2.000
      El dæmon de telnet no quita las variables de entorno TERMINFO, TERMINFO_DIRS, TERMPATH y TERMCAP (cuando empieza con un '/').
      Está disponible un parche para el código que soluciona este problema.

    • 029: PARCHE DE FIABILIDAD: 9 de octubre de 2.000
      Hay un desbordamiento de la memoria intermedia en el modo de prueba de sendmail que no constituye un exploit.
      Está disponible un parche para el código que soluciona este problema.

    • 028: PARCHE DE SEGURIDAD: 6 de octubre de 2.000
      Hay errores en cadenas de formato del estilo de printf en varios programas con privilegios.
      Está disponible un parche para el código que soluciona este problema.

    • 027: PARCHE DE SEGURIDAD: 6 de octubre de 2.000
      libcurses hacía honor a las descripciones en el directorio $HOME/.terminfo así como en la variable de entorno TERMCAP en aplicaciones setuid y setgid.
      Está disponible un parche para el código que soluciona este problema.

    • 026: PARCHE DE SEGURIDAD: 6 de octubre de 2.000
      Existe una vulnerabilidad de cadena de formato en talkd(8). Todavía no está claro el alcance de su impacto.
      Está disponible un parche para el código que soluciona este problema.

    • 025: PARCHE DE SEGURIDAD: 3 de octubre de 2.000
      Existe una vulnerabilidad en una cadena de formato en la función pw_error(3). Esta vulnerabilidad se manifiesta como un agujero de seguridad en la utilidad chpass. Se puede solucionar temporalmente desactivando su funcionalidad 
      # chmod u-s /usr/bin/chpass
      Está disponible un parche para el código que soluciona este problema.

    • 024: PARCHE DE SEGURIDAD: 18 de septiembre de 2.000
      Los paquetes ESP/AH malignos pueden llegar a causar una caída bajo ciertas condiciones.
      Está disponible un parche para el código que soluciona este problema.

    • 023: PARCHE DE SEGURIDAD: 16 de agosto de 2.000
      Existe una vulnerabilidad en una cadena de formato en xlock. Puede solucionarlo desactivando su funcionalidad, mediante la orden 
      # chmod u-s /usr/X11R6/bin/xlock
      Está disponible un parche para el código que soluciona este problema.

    • 021: PARCHE DE SEGURIDAD: 14 de julio de 2.000
      Varios problemas en las bibliotecas de X11 con varios efectos colaterales. Disponemos de un megaparche para solucionarlos.

      • Denegación de Servicio (DoS) en el Servidor de X
        Se soluciona aplicando el parche a xc/programs/Xsever/os/secauth.c.
      • Varios agujeros de seguridad en libX11
        Se solucionan aplicando los parches a xc/lib/X11.
      • Denegación de Servicio (DoS) en libICE
        Se soluciona aplicando los parches a xc/lib/ICE.
      • Desbordamiento en el servidor
        Se soluciona aplicando los parches a xc/programs/Xserver/xkb.
        Nótese que la instalación predefinida de OpenBSD no es vulnerable a este problema, ya que el Xwrapper de XFree86 ya ha sido comprobado por argumentos malignos.

      Está disponible un parche para el código que soluciona este problema.
      Nota 1: Para compilar X11 desde el código fuente se requiere tcl/tk.
      Nota 1: Cuando recompile use la orden 'make DESTDIR=/ build' u obtendrá un error en el último paso de la compilación (makewhatis). Si olvida configurar DESTDIR puede ignorar el error de compilación. La base de datos whatis se reconstruirá la próxima vez que se ejecute /etc/weekly.

    • 019: PARCHE DE SEGURIDAD: 5 de julio de 2.000
      De la mismo modo que prácticamente el resto de dæmons de Unix de ftp en el mundo, ftpd contenía un agujero de seguridad en root remoto. Por suerte, ftpd no se encuentra activado por definición. El problema existe si se activa ftp.
      Está disponible un parche para el código que soluciona este problema.

    • 018: SECURITY FIX: 5 de julio de 2.000
      Mopd contenía un desbordamiento de la memoria intermedia.
      Está disponible un parche para el código que soluciona este problema.

    • 017: PARCHE DE INSTALACIÓN: 3 de julio de 2.000

    • 013: PARCHE DE SEGURIDAD: 28 de junio de 2.000

    • 007: PARCHE DE FIABILIDAD: 8 de junio de 2.000

  • mac68k

    • 022: PARCHE DE INSTALACIÓN: 14 de julio de 2.000
      El instalador de MacOS que se incluyó en OpenBSD 2.7 no crea los dispositivos correctamente; para ser más específicos, no crea el dispositivo /dev/arandom necesario para que funcione la criptología en programas de modo usuario como ssh. El problema aparece cuando ssh-keygen no crea las claves RSA o DSA, dando como resultado un mensaje como RSA-generate_keys failed, o como DSA-generate_keys failed.
      Para solucionarlo, una vez que su máquina esté funcionando, ejecute la siguiente orden como root: 
      cd /dev
./MAKEDEV arandom

      Después de esto (y probablemente instalando uno de los paquetes ssl27), reinicie su máquina y generará las claves de ssh correctamente.
