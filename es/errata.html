  • Todas las Plataformas

    • 021: PARCHE DE SEGURIDAD: 14 de julio de 2.000
      Varios problemas en bibliotecas de X11 provocan efectos colaterales. Hemos creado un parche gigante para solucionarlos.

      • Denegación de Servicio (DoS) en el Servidor de X
        Se soluciona aplicando el parche a xc/programs/Xsever/os/secauth.c.
      • Varios agujeros de seguridad en libX11
        Se solucionan aplicando los parches a xc/lib/X11.
      • Denegación de Servicio (DoS) en libICE
        Se soluciona aplicando los parches a xc/lib/ICE.
      • desbordamiento de Servidor
        Se soluciona aplicando los parches a xc/programs/Xserver/xkb.
        Nótese que la instalación predefinida de OpenBSD no es vulnerable a esto, ya que el Xwrapper de XFree86 ha sido comprobado en busca de malos argumentos.

      Está disponible un parche para el código que soluciona este problema.
      Nota 1: se requiere tcl/tk para compilar X11 desde el código fuente.
      Nota 2: Cuando recompile use la orden 'make DESTDIR=/ build' o en caso contrario obtendrá un error en el último paso de la compilación (makewhatis). Si se olvida de configurar DESTDIR, puede ignorar el error de compilación. La base de datos de whatis se reconstruirá la próxima vez que se ejecute /etc/weekly.

  • mac68k

    • 022: PARCHE DE INSTALACIÓN: 14 de julio de 2.000
      El instalador de MacOS que se incluye en OpenBSD 2.7 no crea correctamente todos los dispositivos; más concretamente, no crea el dispositivo /dev/arandom que se necesita para que funcione la criptografía de programas en modo usuario, como ssh. El problema aparece cuando ssh-keygen no puede crear claves RSA o DSA, dando como resultado mensajes como RSA-generate_keys failed o DSA-generate_keys failed. Para solucionarlo, una vez su máquina esté funcionando, invoque la siguiente orden como superusuario (root): 
      cd /dev
./MAKEDEV arandom

      Después de esto, y probablemente después de instalar uno de los paquetes ssl27, reinicie su máquina y generará las claves ssh correctamente.
