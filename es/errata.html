  • Todas las Plataformas

    • 017: PARCHE DE INSTALACIÓN: 3 de julio de 2.000
      El paquete de screen incluido en la versión 2.7 no se instala correctamente. El paquete que se puede encontrar en 2.7/packages/_ARCH_/screen-3.9.5.tgz ahora se llama screen-3.9.5.tgz.old, y se ha substituido el viejo paquete por un nuevo con el nombre de screen-3.9.5p1.tgz.
      Está disponible un parche para el código que soluciona este problema.

    • 013: PARCHE DE SEGURIDAD: 28 de junio de 2.000
      libedit buscaba un fichero .editrc en el directorio actual. Este comportamiento es extraño; no es que sepamos que esto pueda representar un problema de seguridad en ninguna situación real, pero de todos modos hemos hecho un parche.
      Está disponible un parche para el código que soluciona este problema.

    • 012: PARCHE DE SEGURIDAD: 24 de junio de 2.000
      Un error importante en dhclient(8) permitía que cadenas de un servidor de dhcp maligno fueran ejecutadas en la shell como usuario root.
      Está disponible un parche para el código que soluciona este problema.

    • 009: PARCHE DE SEGURIDAD: 9 de junio de 2.000
      Un error grave en el manejo de "policy" de isakmpd(8) por el cual la verificación de "policy" se podía circunvalar en isakmpd.
      Está disponible un parche para el código que soluciona este problema.

    • 008: PARCHE DE FIABILIDAD: 8 de junio de 2.000
      Algunas operaciones en msdosfs pueden dar como resultado un pánico en el sistema.
      Está disponible un parche para el código que soluciona este problema.

    • 007: PARCHE DE FIABILIDAD: 8 de junio de 2.000
      La exportación NFS de sistemas de archivo de CD causaba pánico en el sistema.
      Está disponible un parche para el código que soluciona este problema.

    • 006: PARCHE DE SEGURIDAD: 6 de junio de 2.000
      La funcionalidad no predefinida UseLogin en /etc/ssh_config no funciona y no se debería usar. En otros sistemas operativos esto representa un agujero de seguridad.
      Evite usar esta funcionalidad o actualice a OpenSSH 2.1.1 o a una versión posterior si debe usarla.

    • 005: PARCHE DE FIABILIDAD: 29 de mayo de 2.000
      Analiza las opciones de IPv4 con más cuidado. Todavía no está claro si se puede usar para provocar una caida remota o local de la máquina.
      Está disponible un parche para el código que soluciona este problema.

    • 004: PARCHE DE FIABILIDAD: 29 de mayo de 2.000
      Ciertas modificaciones del superusuario en la tabla de enrutamiento pueden causar pánico en el sistema.
      Está disponible un parche para el código que soluciona este problema.

    • 003: PARCHE DE SEGURIDAD: 26 de mayo de 2.000
      se puede circunvalar el indicador learning en una interfaz si los marcos van directamente a la máquina que actúa como puente (bridge).
      Está disponible un parche para el código que soluciona este problema.

    • 002: PARCHE PARA CONTROLADOR: 26 de mayo de 2.000
      El controlador ef(4) se queja si se añade una dirección con ifconfig (ifconfig: SIOCAIFADDR: Invalid argument).
      Está disponible un parche para el código que soluciona este problema.

    • 023: PARCHE DE SEGURIDAD: 25 de mayo de 2.000
      Una mala utilización de las reglas de ipf(8)keep-state puede hacer que las reglas en cortafuegos sean ignoradas.
      Está disponible un parche para el código, que soluciona este problema y actualiza ipf a la versión 3.3.16.

  • i386

    • 016: ERROR EN CONTROLADOR: 2 de julio de 2.000
      El controlador xl(4), que ofrece soporte para algunas tarjetas 3com, tiene un error que impide que funcione correctamente el filtro multicast en la 3c905B, lo que hace que no funciones muchas cosas con IPv6.
      Hay disponible un parche para el código que soluciona este problema.

    • 015: ERROR EN CONTROLADOR: 30 de junio de 2.000
      El controlador ste(4), que ofrece soporte para las tarjetas Ethernet basadas en el chipset Sundance ST201 (o sea, la D-Link 550TX), tiene un error que produce pánico en la máquina durante el arranque.
      Hay disponible un parche para el código que soluciona este problema.

    • 014: ERROR EN CONTROLADOR: 30 de junio de 2.000
      El controlador para consola de PC (PCVT) tiene dos errores. Al mostrar problemas en pantalla es posible que el modo de vídeo reverso se active o desactive dos veces a un tiempo. Este parche también soluciona un problema con el manejo del área de pasar página, que habrán notado muchos usuarios del cliente de irc BitchX con el programa screen.
      Ahora hay una segunda revisión del parche para el código que soluciona este problema

    • 011: ERROR EN CONTROLADOR: 17 de junio de 2.000
      El controlador an(4) 4500/4800 IEEE 802.11DS de Aironet Communications contiene un error que causa pánico en ancontrol(8) y que no pueda funcionar de forma correcta.
      Hay disponible un parche para el código que soluciona este problema.
