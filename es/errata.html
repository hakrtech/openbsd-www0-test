  • Todas las Plataformas

      +
    • 009: PARCHE DE SEGURIDAD: 9 de junio de 2.000
      Un error grave en el manejo de "policy" de isakmpd(8) por el cual la verificación de "policy" se podía circunvalar en isakmpd.
      Está disponible un parche para el código que soluciona este problema.

    • 008: PARCHE DE FIABILIDAD: 8 de junio de 2.000
      Algunas operaciones en msdosfs pueden dar como resultado un pánico en el sistema.
      Está disponible un parche para el código que soluciona este problema.

    • 007: PARCHE DE FIABILIDAD: 8 de junio de 2.000
      La exportación NFS de sistemas de archivo de CD causaba pánico en el sistema.
      Está disponible un parche para el código que soluciona este problema.

    • 006: PARCHE DE SEGURIDAD: 6 de junio de 2.000
      La funcionalidad no predefinida UseLogin en /etc/ssh_config no funciona y no se debería usar. En otros sistemas operativos esto representa un agujero de seguridad.
      Evite usar esta funcionalidad o actualice a OpenSSH 2.1.1 o a una versión posterior si debe usarla.

    • 005: PARCHE DE FIABILIDAD: 29 de mayo de 2.000
      Analiza las opciones de IPv4 con más cuidado. Todavía no está claro si se puede usar para provocar una caida remota o local de la máquina.
      Está disponible un parche para el código que soluciona este problema.

    • 004: PARCHE DE FIABILIDAD: 29 de mayo de 2.000
      Ciertas modificaciones del superusuario en la tabla de enrutamiento pueden causar pánico en el sistema.
      Está disponible un parche para el código que soluciona este problema.

    • 003: PARCHE DE SEGURIDAD: 26 de mayo de 2.000
      se puede circunvalar el indicador learning en una interfaz si los marcos van directamente a la máquina que actúa como puente (bridge).
      Está disponible un parche para el código que soluciona este problema.

    • 002: PARCHE PARA CONTROLADOR: 26 de mayo de 2.000
      El controlador ef(4) se queja si se añade una dirección con ifconfig (ifconfig: SIOCAIFADDR: Invalid argument).
      Está disponible un parche para el código que soluciona este problema.

    • 023: PARCHE DE SEGURIDAD: 25 de mayo de 2.000
      Una mala utilización de las reglas de ipf(8)keep-state puede hacer que las reglas en cortafuegos sean ignoradas.
      Está disponible un parche para el código, que soluciona este problema y actualiza ipf a la versión 3.3.16.
