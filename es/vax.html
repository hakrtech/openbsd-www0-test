OpenBSD/vax

Cronología y estado actual:

El soporte de OpenBSD para la arquitectura VAX data de principios del año 2000, cuando se sincronizó el código fuente con el porte de NetBSD para VAX. Dado que fue la segunda arquitectura sobre la que funcionón BSD Unix (¡en 1979!), es justo que todavía se pueda disponer de un sistema operativo moderno para esas viejas máquinas VAX. El código base procede en su mayor parte de Berkeley, junto con contribuciones de código libre de Digital, Mt Xinu, NetBSD, OpenBSD y otros. El desarrollo es contínuo, y aun en este momento se comparten los cambios entre los desarrolladores de VAX. El resultado de este trabajo es el de un sistema operativo estable y bien conocido por cualquier usuario de OpenBSD.

Cambios significativos post 3.0:

  • Añadido soporte para máquinas "Legacy".
  • El tamaño de página lógica binaria ha cambiado a 4096 bytes.

Cambios significativos entre las versiones 2.9 y 3.0:

  • Añadido soporte para máquinas "Omega".
  • Añadidos contadores de eventos en la mayoría de dispositivos.
  • Más modos de direccionamiento añadidos al sistema de emulación.

Cambios significativos entre las versiones 2.8 y 2.9:

  • Soporte para 16 sistemas de archivos por controlador.
  • Emulación de la instrucción EMODD.
  • Soporte para máquinas "Cheetah".
  • Las señales de interrupción ya no causan volcados en los subintérpretes (subshell) de ksh.

OpenBSD funciona con un amplia gama de componentes de VAX, pero nuestro inventorio de sistemas personal es limitado. Es posible que OpenBSD/vax funcione en su hardware aunque no aparezca en esta lista, por lo que sus verificaciones e informes son importantes para mantener esta lista actualizada. Una vez acabada la instalación y la verificación de su nuevo sistema, por favor ejecute "dmesg |mail dmesg@openbsd.org" para ayudarnos a determinar los tipos de componentes que utilizan los usuarios.

Proyectos en desarrollo:

  • HDC9224 (controlador de disco duro mfm / disquete) para VAXstation 2000.
  • Soporte MSCP para cintas y controladoras en máquinas QBUS/UQSSP.
  • ¿Bibliotecas compartidas? algún día...
  • Soporte para cualquier nuevo sistema que encontremos.

Documentación que estamos buscando:

  • Información sobre programación de framebuffers para LCG o SPGFX.
  • Referencias de software para SHAC DSSI.
  • Cualquier edición VARM.

Problemas conocidos:

  • problemas con cadenas de herramientas - optimizador gcc demasiado exigente.
  • problemas con pmap - límites de direccionamiento.

Además de las versiones finales (-release), existen unas versiones preliminares (snapshots) que reflejan el progreso actual y que se encuentran disponibles desde las réplicas de ftp; también disponemos de una lista de correo en vax@openbsd.org. Para darse de alta en la lista de correo, envíe un mensaje con la línea "subscribe vax" en el contexto a majordomo@OpenBSD.org. Por favor, lea nuestra política sobre listas de correo antes de suscribirse.

Los mantenedores del porte de OpenBSD/vax son Brandon Creighton y Hugh Graham.
