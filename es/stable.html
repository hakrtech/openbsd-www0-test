¿Qué es una ramificación de parches?

A partir de la versión 2.7, el proyecto OpenBSD ofrece un árbol de fuentes que contiene los parches y soluciones varias que se encuentran en errata, y las pone a disposición a través de CVS aparte del código fuente actual de current. De este modo los usuarios pueden llevar un seguimiento de cualquiera de las dos ramificaciones: current para aquellos que quieran o necesiten todas las funcionalidades más recientes, y la ramificación de parches para aquellos que quieran quedarse en las versiones oficiales y sólo necesiten aplicar los parches y soluciones provistas entre versiones.

Cómo obtener una ramificación de parches

Para obtener una ramificación de parches de una versión en particular de OpenBSD, puede bien empezar con un árbol de fuentes del CD lanzado con la versión de OpenBSD que quiera usar (ver pedidos), o bien bajarse el árbol completo desde un servidor de AnonCVS. Las instrucciones para obtener ramificación de parches y permanecer actualizado se describen en la sección «Para Empezar» de la documentación de AnonCVS.

Por favor, tenga en cuenta que las ramificaciones de parches no sirven para actualizar desde una versión de OpenBSD a otra, v.g. para actualizar desde 2.5 a 2.6. Lo único que proveen es un medio para mantenerse actualizado dentro de una misma versión. Si va a actualizar una versión mediante el código fuente, por favor lea el documento upgrade-minifaq sobre cómo actualizar de las Preguntas Frecuentes.

Compilar desde una ramificación de parches de OpenBSD

Una vez que ya haya obtenido un árbol de fuentes a través de anoncvs, debe recompilar el sistema. Los pasos a seguir para ello son:

  • Recompilar el núcleo del sistema
  • Reiniciar el sistema con el nuevo núcleo
  • Recompilar los binarios

Recompilar el núcleo

Para recompilar el núcleo predefinido de la versión estable: 

    # cd /usr/src/sys/arch/i386/conf
# /usr/sbin/config GENERIC
# cd /usr/src/sys/arch/i386/compile/GENERIC
# make clean && make depend && make

Substituya i386 con su plataforma, v.g. sparc, alpha, etcétera.

Reiniciar con el nuevo núcleo

Para reiniciar con el núcleo que acaba de compilar: 

    # cd /usr/src/sys/arch/i386/compile/GENERIC
# cp /bsd /bsd.old          (Guardar una copia del viejo núcleo)
# cp bsd /bsd               (Copiar el nuevo núcleo a su ubicación)
# reboot

Como en el paso anterior, substituya su plataforma, en este caso i386 por la plataforma correspondiente. Si su sistema tiene problemas con el arranque del nuevo núcleo, puede retroceder y reiniciar desde el viejo núcleo, que ahora se llama bsd.old.

Recompilar los binarios

Para recompilar los binarios del sistema: 

    # cd /usr/src
# rm -r /usr/obj/*
# make obj && make build

Esto tardará bastante tiempo...
