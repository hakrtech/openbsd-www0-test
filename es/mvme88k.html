OpenBSD/mvme88k

OpenBSD/mvme68k es un proyecto para portar OpenBSD a la familia de placas base basadase en la 881x0 VME de Motorola.

En la actualidad no hay nadie que mantenga el porte mvme88k, ya que el soporte no es oficial. Las personas que están trabajando sobre este porte son Steve Murphree y Miodrag Vallat.

Cronología:

Se dice que el procesador Motorola 88k es el mejor procesador RISC que jamás se ha diseñado. Su simplicidad y elegancia se combinan para hacer del mvme88k una plataforma robusta.

Nivas Madhur comenzó el porte al mvme88k inicial para la tarjeta MVME187, pero desde entonces ha cambiado de trabajo. El porte fue importado al árbol de fuentes de OpenBSD por Dale Rahn, pero no disponía de tiempo para trabajar en él. Steve Murphree, Jr. completó finalmente el porte para la MVME187 en noviembre de 1998.

Desafortunadamente, al mismo tiempo una actualización de gcc 2.8.1 a egcs reveló un montón de problemas en el soporte para mvme88k en gcc, y no se pudieron solucionar a tiempo para que OpenBSD/mvme88k fuera una arquitectura con soporte en la versión final de OpenBSD 2.5. En el momento actual, estos problemas todavía no se han solucionado por completo.

La falta de una cadena de herramientas integradas en el árbol de fuentes no impidió que se siguiera trabajando en el porte, y se realizaron muchos cambios en el código base, como rediseñar autoconf y el controlador SCSI integrado, una gran expansión del soporte para el bus VME, trabajar en un proceso de instalación que creara un bloque VID de Motorola en los discos, y el soporte para MVME188 así como la mejora del soporte para MVME197.

Estado actual:

El desarrollo de este porte está parado en este momento debido a que los desarrolladores están ocupados diseccionando el dorsal mvme88k de gcc y reparando los errores de generación de código.

Proyectos (sin orden):

  • Reparar los restantes errores lógicos de generación de código de mvme88k de gcc
  • Mejorar la fiabilidad de MVME197
  • Tarjetas sin soporte (MVME327, MVME374...)
  • Escribir código para un nuevo binutils y migrar a ELF y, luego, a bibliotecas compartidas

Hardware con soporte:

Placas de procesador con soporte

  • MVME187 (un solo procesador 88100)
    Hay soporte para todos los dispositivos integrados, a excepción del puerto paralelo.
  • MVME188 (de uno a cuatro procesadores 88100)
    A diferencia de las otras placas de procesador MVME, esta placa no tiene dispositivos integrados; simplemente actúa como un contenedor para un HyperModule que provee 1, 2 ó cuatro procesadores, y «unidades de gestión de memoria/cache» (Cache/Memory Managemente Units, CMMU). El soporte debería fucionar para todos los modelos de HyperModule que existen, pero hasta ahora sólo lo hemos verificado en los módulos 1P64 (1 cpu, 4 cmmu) y 2P256 (2 cpu, 4 cmmu). Las tarjetas externas específicas para la familia MVME188 proveen memoria y puertos serie.
  • MVME197 (uno o dos procesadores 88110)
    Las placas MVME197 tienen diferentes tipos, diferenciándose por el número de procesadores (1 ó 2) y la presencia opcional de una controladora externa de caché. El soporte debería funcionar para todas las configuraciones que existen, pero sólo lo hemos verificado en una placa MVME197LE (sin controladora externa de caché). El soporte para esta placa todavía es muy experimental y tiene problemas.

Dispositivos integrados con soporte

  • Puertos serie
    • Puertos serie integrados CL2400 Cirrus Logic para MVME187 (tty00-tty03)
    • Puertos serie SYSCON DUART para MVME188 (ttya, ttyb)
  • Adaptadoras ethernet
    • Interfaces integradas Intel i82586 para MVME187 y MVME197
  • Controladoras SCSI
    • Controladoras NCR 53c7xx para MVME187 y MVME197

Placas de extensión con soporte

  • MVME328: controladora SCSI
  • MVME332: placa de 8 puertos serie
  • MVME376: ethernet VME Lance

Cómo obtener e instalar OpenBSD/mvme88k:

Debido a problemas del compilador, no se han generado versiones preliminares (snapshots) desde hace mucho tiempo. Durante un tiempo hubo una versión preliminar de OpenBSD 2.4-current en las réplicas de ftp, que sólo tenía soporte para las placas MVME187 y que se podía usar como base para recomponer un sistema completo... para el hacker aventurero. Para conseguir esta versión preliminar se puede preguntar a miod. El acceso a un sistema OpenBSD/mvme68k ayuda mucho, ya que ambos sistemas son muy parecidos.
