Cronología:

Se dice que el procesador Motorola 88k es el mejor procesador RISC que jamás se ha diseñado. Su simplicidad y elegancia se combinan para hacer del mvme88k una plataforma robusta.

Nivas Madhur comenzó el porte al mvme88k inicial para la tarjeta MVME187, pero desde entonces ha cambiado de trabajo. Steve Murphree, Jr. completó el porte en noviembre de 1.998. El porte ha sufrido desde entonces cambios importantes, incluida la renovación de autoconf y un controlador integrado SCSI, una gran expansión en el soporte del bus VME, un proceso de instalación funcional que genera un bloque VID de Motorola correcto en los discos, y soporte para más placas.

Estado actual:

Las personas que más están trabajando en OpenBSD/mvme88k son: Steve Murphree, Jr. y Miod Vallat (si hubiera más gente sería mejor :)

Se puede contactar con el mantenedor del porte OpenBSD/mvme88k enviando un mensaje a smurph@OpenBSD.org.

Existe una versión preliminar (snapshot) disponible desde el lanzamiento de la versión 2.4 de OpenBSD. El porte tiene soporte para la placa de un solo procesador MVME187 Single Board Computer (SBC), así como para la placa multiprocesador MVME188. (en este momento sólo usa 1 procesador). Existen planes para añadir soporte para la MVME197, pero falta tiempo para llevar esto a cabo. Las herramientas de instalación y proceso funcionan desde la versión 2.5. Es posible instalar o actualizar OpenBSD/mvme88k por medio de imágenes de ramdisk en discos de almacenamiento y también mediante instalaciones de red y sin disco.

Nuevo en 2.8:

  • Soporte para MVME188 completo.
  • Dispositivos de bus VME auto vector, que provee un entorno más "plug and play".
  • Ahora OpenBSD/mvme88k usa el «Sistema de memoria virtual unificada» ("Unified Virtual Memory System", UVM).

Hardware con soporte:

OpenBSD/mvme88k funciona en las siguientes máquinas:

  • MVME187: Single Board Computer (SBC) basado en Motorola 88100
  • MVME188: Sistemas VME basados en Motorola 88100
    Las placas de CPU MVME188 vienen en diversos modelos, que se diferencian por el número de procesadores (1, 2 ó 4) y las CMMU asociadas. Todas las configuraciones existentes deberían tener soporte en OpenBSD, pero sólo se ha comprobado en la placa 1P64 (1 cpu, 4 cmmu) y la 2P256 (2 cpu, 4 cmmu).
  • MVME197: Single Board Computer (SBC) basado en Motorola 88110. Las placas de CPU de MVME197 vienen en diferentes tipos, que se diferencian por el número de procesadores (1 ó 2) y por la presencia opcional de una controdalora de caché externa. Todas las configuraciones existentes deberían tener soporte, pero sólo se ha comprobado en una placa MVME197LE (sin controladora de caché externa). El soporte para esta placa es nuevo y todavía tiene algunas 'funcionalidades' por resolver.

Dispositivos con soporte:

  • Dispositivos de disquete:
    • Dispositivos de disquetera SCSI
  • Puertos serie:
    • Puertos serie Cirrus Logic integrados (MVME187) tty00, tty01, tty02 y tty03
    • Puertos serie integrados DUART ttya y ttyb (MVME188)
    • Tarjeta con 8 puertos serie MVME332
  • Adaptadoras ethernet:
    • Intel Ethernet integrado (MVME187, MVME197).
    • Lance Ethernet MVME376 AMD.
  • Controladoras SCSI:
    • Controladora SCSI "NCR" integrado (MVME187, MVME197):
    • Controladora SCSI MVME328

Dispositivos sin soporte. Hay muchos más...

  • Controladoras de Disco:
    • Controladora SCSI MVME327
  • Contoladoras de Ethernet:
    • Controladora LANCE MVME374
  • Controladoras de WAN:
    • Controladora de WAN MVME333

Proyectos:

  • Empezar a trabajar con controladoras de dispositivos para tarjetas VME sin soporte.
  • Escribir el código para nuevos binutils (solucionará errores de ld) y cambiar a ELF.
  • Bibliotecas compartidas.
