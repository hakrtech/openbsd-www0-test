OpenBSD/powerpc

Estado Actual:

Este porte es bastante nuevo. Tiene soporte para algunos tipos de máquinas con procesadores PowerPC, pero no todos. Aunque existen comités para la creación de normas que traten de resolver este problema, hay diferencias importantes entre las máquinas PowerPC... ¡a pesar de estar hechas por el mismo fabricante!
Este porte funciona en algunas máquinas, pero en otras no. Compruebe la lista que sigue más abajo.

¡Para que funcione, la máquina DEBE tener alguna variante del software OpenBoot de Firmware!

El soporte para el procesador es bastante estable en este momento. El soporte para los controladores es algo limitado, debido a que la mayoría de dispositivos PCI sólo han sido verificados en máquinas "little-endian". Estamos desarrollando soporte para dispositivos adicionales para el hardware de los sistemas Macintosh; cualquier información sobre programación para cualquier tipo reciente de hardware de Mac ayudaría a mejorar esta situación.

Existe soporte para los siguientes procesadores: 603, 603e, 604, 604ev, 750.

El porte todavía no usa controles de dispositivos reales. De momento usa OpenFirmware para todas las necesidades de los controladores de dispositivos de consola, disco, o ethernet. Se puede instalar por arranque en red, disquete y red, o cdrom. Existe soporte para Netroot, pero poco verificado.

Hay dos versiones del núcleo para este porte, una que sólo usa openfirmware para todos los dispositivos, y una versión que ha sido codificada con soporte para algunas máquinas de las series PowerSTack y VI.

Hardware con Soporte (o funcional):

  • Máquinas Motorola con OpenFirmware (PPC1bug firmware no funciona, y las máquinas con procesadores 603* y algunas versiones de Motorola openfirmware tampoco):
    • Series E
    • RiscPC
    • Pro2000
    • Pro3000
    • Pro4000
  • V-I
    • Placas Power4e.

Hardware con Soporte en Desarrollo:

  • Sistemas Apple PowerMac con un mínimo de un procesador 603 y OpenFirmware.
    • Actualmente sólo el Apple imac (333Mhz) tiene soporte para los controladores.
    • Otros sistemas tendrán soporte según disponibilidad de hardware e información sobre dispositivos. Se dará prioridad a los sistemas más recientes sobre los sistemas más viejos y lentos.

Máquinas que NO FUNCIONAN:.

  • Máquinas IBM.
  • Máquinas de la clase PPCP.
  • Sistemas Apple PowerMac con un procesador 601 o sin OpenFirmware no funcionarán nunca.
  • La implementación de Apple de OpenFirmware no tiene soporte para cargar los ejecutables ELF (como los bloques de arranque para este porte). Esto necesita algún tipo de solución alternativa.

Cronología:

La porciones del núcleo del sistema específicas para powerpc vinieron del porte de NetBSD de Wolfgang Solfrank. El modo usuario y las herramientas de compilación vinieron de un porte independiente e incompleto de Dale Rahn. Esperamos poder fusionar piezas de código de otro porte, soporte para controladores de dispositivos para interrupciones del sistema, y algo de soporte real para dispositivos. La conversión al porte de Wolfgang se llevó a cabo debido a que este último funcionaba en modo multiusuario, mientras que el anterior todavía estaba funcionando en modo monousuario sin memoria de ramdisk (los controladores de dispositivos no funcionaban del todo).

Después de llevar a cabo ese trabajo, Per Fogelstrom añadió soporte para la placa vme VI Power4e, eliminando el soporte openfirmware para dispositivos durante el proceso (versión 2.3).

Para la versión 2.4 se añadió soporte para los dispositivos openfirmware, con retrocompatibilidad, de modo que dispositivos de hardware y dispositivos ofw tuvieran un soporte mixto. Después de esto se adaptó el soporte para power4e con el fin de que funcionara en máquinas MCG PowerStack. Se añadió algo de soporte para dec 21040 y vga; vga todavía no se puede usar en consola, pero sí como terminal. Aún no hay soporte para servidores de X, pero el mecanismo para acceder a vga por medio de mmap existe.

Para la versión 2.5 el soporte se mantuvo a los niveles de la versión 2.4.

Para la versión 2.6 no hubo un lanzamiento oficial del porte para powerpc. El trabajo en el iMac había apenas comenzado y no se habían resuelto problemas con el compilador que evitaban que el sistema compilara desde el árbol.

A partir de la versión 2.6, se ha añadido soporte para iMac, existe más desarrollo en progreso para crear mejor soporte para dispositivos imac, y para otros sistemas. Al principio, con el soporte para los dispositivos de imac, se perdiá la compatibilidad con sistemas más antiguos. Esto se debiá principalmente a una falta de verificaciones, pero estos sistemas tendrán soporte una vez que los cambios al sistema sean retroportados.

Contacte con Theo de Raadt o Dale Rahn si está interesado en trabajar en este porte con otras personas, o si necesita más información.
