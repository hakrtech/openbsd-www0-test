i386 Laptop Status

This is a list of laptops known to be working with OpenBSD.

If your laptop isn't listed below doesn't mean that it won't work, and you will most likely find a similar model that is listed below. Most laptops work very well, however some will still have minor issues.

Of those that have problems,

  • Some exhibit problems with APM support. Note that apm -S fails to suspend some laptops.
  • Some lack sound support
  • A few have subtle bugs with their PCMCIA and CardBus support
  • There is currently neither Firewire nor ACPI support in OpenBSD
  • XFree86/X.org does not support some displays properly. Sometimes switching between both of them helps.
  • Some other built-in devices don't work.
  • Some have problems with pcmcia card eject interrupts.
  • Almost all modern laptops with on-board modems have "winmodems", which are not currently and are unlikely to ever be supported. Use a PCCard modem if you need one that works. You can assume any modem on the machines below does NOT work unless indicated.
  • IBM Thinkpads (and perhaps other notebooks, too) with Phoenix-style BIOSes offer a so-called "hibernation mode", where memory is written to disk. You can use this feature with OpenBSD. Generate the partition for hibernation using the /usr/ports/sysutils/tphdisk utility from the ports collection. The hibernation partition requires to be a MS-DOS partition at the beginning of the harddisk. This partition can be of type 16 bit FAT or FAT32 (as such it is possible that a Windows install lives in this partition).
  • IBM Thinkpads with mini-PCI slots will check PCI IDs against a built-in list of "allowed" devices, and thus will not work with third-party cards. This behavior can be changed by using the /usr/ports/misc/tpwireless utility from ports BEFORE inserting the third-party card. This utility sets a "magic bit" in the BIOS to disable this check.
  • In order to get the full native resolution of the LCD screen on some laptops, Intel i8x5/i9x5 chipsets can be adjusted using the x11/915resolution package. This software changes the resolution of an available vbios mode. It patches only the RAM version of the video bios, so the new resolution is lost upon rebooting.
    Note that the 915resolution port is only available in -current, and that it supersedes 855resolution, which is still mentioned in a number of entries below.

  • The on-board fxp(4) and auich(4) are working without problems.
  • The keys for mute, volume up and volume down are working fine. No special software is required. If you'd like to get the big "Access IBM" button to work and have a nice on-screen display, you can install tpb from ports. Sony VAIO PCG-FX602

  • Custom hack for hw.setperf:
    • Apply the ACPI-patches from Alexander Yurchenko.
    • Then edit /usr/src/sys/conf/files and add in line 21: 
      	define  acpibus {}
    • Add to /usr/src/sys/sys/malloc.conf 
      	#define M_ACPI  122     /* ACPI */
    • Comment out the Sony SPIC stuff in /usr/src/sys/arch/i386/conf/files.i386 and /usr/src/sys/arch/i386/conf/GENERIC_ACPI (This stuff does not compile for me and i don't care about it;)
    • rebuild kernel, /usr/src/sbin/sysctl, /usr/src/usr.sbin/acpid[ump]
    • You are now able to set hw.setperf via sysctl.
  • I run additionally a utility I wrote to control this value automatically depending on cpu-load. It is based on Laurie Tratts' perfd. Just unpack and run make && sudo make install.
