H.1 - OpenBSD/hp300 peut-il booter sur le réseau ?

Les stations de travail HP des séries 9000/300 supportent le boot réseau à partir de la révision N de la ROM de Boot. (Ceci inclut tout ROM de Boot avec une révision numérique). La ROM de Boot HP utilise le Protocole de Maintenance Distante HP pour télécharger le code de boot depuis le serveur. Le serveur doit utiliser un démon capable de répondre aux requêtes de boot de HP RMP, rbootd(8).

Pour configurer votre station de travail OpenBSD/hp300, suivez la page de manuel diskless(8).

H.2 - Où en est-on avec X11 sur hp300 ?

Nous avons actuellement la totalité des clients et serveur X11R6 de X.org. Il n'est désormais plus possible d'utiliser le serveur X11R5 de HP-UX via l'émulation.

H.3 - Que signifie 'UNEXPECTED USE OF FFFFFFC4' ?

Ceci est la sortie d'un ensemble de gestionnaires d'exceptions temporaires, à titre indicatif uniquement, qui sont installés par la ROM de Boot avant qu'un OS n'ait été chargé. L'adresse affichée varie selon le type d'exception. La cause la plus fréquente de ceci est la tentative de boot d'un noyau incompatible avec le matériel. Vous obtenez ce genre de résultat lorsque, par exemple, vous essayez de booter un noyau HP-UX 7.0 ou antérieur sur une machine 68040 (qui nécessite au moins 7.05). Si vous vous apprêtez à installer OpenBSD, vous n'avez probablement pas à vous en inquiéter, car il est possible (mais peu fréquent) que cela indique un problème matériel ou un chargeur de boot corrompu.

Information sur la ROM de Boot

H.4 - Quelles commandes comprend la ROM de Boot HP ?

Toutes les anciennes ROMs de Boot hp300 sont très primitives et n'autorisent que quelques opérations simples. Vous ne pouvez qu'interagir avec elle après le premier démarrage -- si vous rebootez la machine, tout ce que vous saisissez sera ignoré et l'OS qui a été démarré lors de la fois précédente le sera à nouveau.

A tout moment après la reconnaissance du clavier, lors de l'auto-test ou de la recherche d'un système démarrable, vous pouvez presser sur reset afin de revenir à une configuration précédente. Sur les claviers HIL, la combinaison est <control>-<shift>-break pour les claviers non-AT (ou break est la touche dans le coin supérieur gauche (à la place de escape sur les claviers corrects) et <control>-<shift>-pause pour les claviers AT. Il n'y a pas d'équivalent sur un terminal série -- vous devrez relancer le processus de mise en route de votre machine.

Après le beep (signifiant que la clavier HIL a été détecté), pressez sur <return> à deux reprises afin d'obtenir la liste des périphériques démarrables. Afin d'effectuer des vérifications matérielles simples, pressez sur <control>-C avant que le démarrage de l'OS soit amorcé. Vous pourrez alors saisir T afin d'effectuer un test de santé étendu ou L afin d'effectuer ce dernier jusqu'à ce qu'une erreur fatale soit trouvée, ou que L soit utilisé à nouveau.

La nouvelle ROM de Boot HP, présente sur les machines des séries 400 et sur les dernières 300s (345, 375, 380, 382, 385) est capable d'en faire plus. Afin de sélectionner le périphérique sur lequel démarrer, pressez <return> une fois, après que la machine ait beepé par deux fois (signifiant la reconnaissance du clavier HIL). Afin d'obtenir une configuration et un menu de test, pressez : 

   C <return>
Ceci vous permettra de configurer les niveaux d'interruption, les codes de sélection, et les propriétés de la console série. Vous pouvez aussi presser <control>-C afin de vous rendre dans un menu de tests étendus avec plusieurs options fantaisistes.

Pour plus d'informations, Michael Wolfson a scanné plusieurs parties de HP Apollo 9000 Series 400 HP-UX Owner's Guide, qui contient des informations intéressantes sur le sujet.

H.5 - Quel ordre est utilisé par la ROM de Boot pour rechercher les périphériques bootables ?

Provenant du Configuration Reference Manual, 98561-90020 :

Révision, Spécifications d'une ROM de Boot

La ROM de boot peut charger un système contenu dans la ROM ou un fichier contenu dans un type de fichier "SYSTEME" LIF ou SRM ayant un nom de la forme SYSa, où "a" est typiquement une lettre ASCII, pouvant être tout caractère autorisé dans un nom de fichier.

Les systèmes ROM se voient assigner une seule lettre d'ID (seulement "B" pour BASIC est actuellement supporté sur les Séries 300).

Tous les systèmes de fichiers trouvés ont un ID assigné de la forme "nna", où "a" est soit la même lettre "a" mentionnée ci-dessus, soit "Z" (si ce n'est pas une lettre ASCII). "nn" est un nombre de la forme " 1" à "99" dénotant l'ordre des occurrences du système, résultant de la même lettre d'ID "a". La plage des IDs systèmes s'étend de "1A" à "99Z".

La ROM de boot charge le premier système trouvé hormis les caractères (autres que ceux de l'ID système) sont saisis sur le clavier de contrôle du boot (voyez ci-dessous). L'ordre de recherche utilisé par la ROM de boot est :

  • For select codes 7 thru 31: disc or tape (HPIB) at bus address 0, unit 0, volume 0
  • SRM at select code 21, node 0, volume 8, "/SYSTEMS" directory
  • 98259A Bubble system at select code 30
  • 98255 EPROM "disc"-type system at unit 0 (lowest address of all 98255s installed)
  • ROM systems (from lowest to highest ROM address)
  • For select codes 0 thru 31, bus addresses 0 to 7, units 0 to 16, volumes 0 to 7: all remaining discs or tapes (HPIB)
  • For select codes 0 thru 31, nodes 1 thru 62, volumes 1 to 50: any other SRM system files in "/SYSTEMS" directories
  • For select codes 0 thru 29, and 31: remaining 98255 Bubble systems
  • Remaining 98255 "disc"-type EPROM units.

La révision B et les suivantes supportent aussi le démarrage réseau, en utilisant la carte 98643 ou la carte ethernet intégrée. Pour les systèmes plus anciens, le meilleur choix est de placer votre lecteur de boot sur "HPIB at address 0". N'oubliez pas que vous devez saisir les lettres en capitales.

Ordre de recherche des nouvelles ROM de Boot

Les nouvelles machines (Modèles 345, 375, 380, 382, 385 et Séries 400) ont un ordre de boot différent. Issu de HP Apollo 9000 Series 400 HP- UX Owner's Guide, A1630-90006 :

Le Scan pour la sélection de recherche de périphériques de stockage de masse des Systèmes à démarrer par un Système d'exploitation. Le premier périphérique de stockage de masse trouvé avec un système d'exploitation compatible HP-UX démarre. Les périphériques de stockage de masse sont recherchés selon les priorités présentes dans ce tableau.

Niveau de priorité Périphérique Code de Sélection Adresse de Bus Numéro d'Unité
1 SCSI 0-31 7-5 0
2 HP-IB 0-31 7-5 0
3 SRM 14 N/A N/A

Ainsi, pour ces systèmes récents, votre meilleure mise est d'utiliser un périphérique de boot SCSI à l'adresse 6 (7 est le contrôleur système de la carte mère).

Il est également possible de configurer la ROM de Boot par défaut pour un périphérique spécifique depuis le menu de configuration.

H.6 - Basculement de votre machine Series 400 du mode Domain au mode "HP-UX Compatible Boot Mode"

Cette étape est nécessaire, depuis qu'OpenBSD peut aussi booter sur une machine des Séries 400 lorsque celle-ci est en mode de boot "HP-UX Compatible". Si, lorsque vous allumez votre machine, le menu n'est pas présenté comme ce qui suit, vous devez alors suivre les instructions suivantes :

BOOTROM  Series 400  Rev. 1.1           
MD12 REV 1.2 1990/08/07.14:27:08        
MC68030 Processor

Configuration EEPROM                    
Utility Chip at 41                      
HP-HIL.Keyboard
[...]
Tout d'abord, vous devez avoir soit un clavier Domain soit un clavier HIL. À présent, placez votre machine en "service mode". Pour une 4XXs, il y a un switch de basculement à l'arrière de la machine (à proximité du haut). Pour une 4XXt ou une 4XXdl, il y a un bouton vert sur le devant, derrière la porte. Pour une 425e, il y a un switch de basculement à l'arrière de la machine (au centre). Une fois en "service mode", l'autre LED devrait être allumée. Appuyez sur "reset". Vous pourriez avoir à presser sur return afin d'obtenir le prompt de boot Domain. Au prompt, vous pouvez saisir H afin d'obtenir la liste des commandes disponibles. Vous devez saisir les choses suivantes afin de convertir le mode HP-UX : 
CF
2
2
P
E
Ceci est la procédure complète capturée depuis une console série.

Assurez-vous d'avoir arrêté le "service mode" une fois ceci terminé. Cela pourrait vous empêcher de sélectionner le périphérique de boot que vous voulez utiliser. De plus, sur les 425e, la sélection de service choisie la console OpenBSD.

H.9 - Configurer une console série sur une 98561, 98562, 98626, 98628, 98642 ou 98644

Éteignez votre système avant d'enlever toute carte. Retirez la carte avec l'interface série.

  • 98561-66530 (Interface Humaine)
    Localisez la banque des 4 switch DIP
    L'un d'entre eux doit être nommé REM
    Positionnez ce switch sur ?
  • 98562 (Interface Système)
    Localisez la banque centrale des swicth DIP (4 switch)
    Le troisième switch est nommé REM
    Positionnez le switch à un (relâchez le dernier nommé un)
    Note : vous aurez besoin d'un câble non-null
  • 98626 (dca)
    Localiser le jumper par les deux banques de switch DIP
    retirez le jumper
  • 98628 (dcl)
    Localisez la banque de switch DIP car le connecteur card-edge
    Le dernier switch (nommé 7) est le switch distant
    Positionnez le switch à zéro (relâchez la fin nommée zéro)
  • 98642 (dcm)
    Localisez les 8 switch DIP
    Le premier switch (nommé 1) est le switch distant
    Positionnez le switch à un (glissez sur le un)
    Note : Selon le manuel, la ROM de Boot ne sait pas comment utiliser ceci sur les anciennes machines, mais OpenBSD (et HP-UX) le savent, vous ne voyez donc rien avant que le chargeur de boot agisse.
  • 98644 (dca)
    Localisez les 10 switch DIP
    Le dernier switch (nommé 1) est le switch distant
    Positionnez le switch à un (relâchez celui nommé un)

À présent, réinsérez la carte et allumez votre machine. Tous les messages de la console seront envoyés via le port série à 9600 bps, 8 bits, sans parité et avec 1 bit de stop. Théoriquement, vous devriez utiliser un câble null-modem, mais j'ai remarqué que j'avais besoin d'un câble non null-modem pour ma 98562.

H.10 - Configurer la console série sur un Model 340

Éteignez votre système
Il y a quatre switch DIP visibles à travers le panneau arrière
Tournez le troisième switch en partant de la gauche sur 1
Allumez votre système

À présent, tous les messages de console seront envoyés via le port série à 9600 bps, 8 bits, sans parité et avec 1 bit de stop.

H.11 - Configurer la console série sur une machine Series 400 ou une 345,375, 380, 382 ou 385

Patientez en attendant que votre système beepe à deux reprises (vous laissant ainsi savoir qu'il a reconnu le clavier). Saisissez C<return> et attendez jusqu'à ce que le menu de configuration apparaisse. Saisissez ensuite l'ensemble suivant de commandes : 
1
5
3
R
X
N

(Si votre machine ne possède pas de contrôleur HP-IB, remplacez 5 par 4 dans les instructions ci-dessus)

Ceci est la procédure complète capturée depuis une console série sur ma 400s. Ceci peut être sensiblement différent pour les machines des séries 300 (345, 375, 380, 382, 385).

Notez que la 425e ne supporte pas la configuration de la console série dans la ROM. Cependant, OpenBSD utilisera le premier port série (sur apci0) comme une console si le swicth service/normal à l'arrière est positionné sur service; aucun autre réglage n'est requis.

À présent, votre machine va se remettre à zéro et envoyer tous les messages de la console via le port série, à 9600 bps, 8 bits, sans parité et avec 1 bit de stop. Utilisez un câble null-modem.

Si vous voulez passer de la console série à la console moniteur/clavier, suivez la même procédure mais saisissez L au lieu de R.

Vous pourriez aussi outrepasser temporairement ce réglage en saisissant L<return> ou R<return> après que votre système ait beepé deux fois et ait reconnu le clavier. Ceci devrait marcher même si vous avez un clavier Domain.

Depuis que la console série est mappée à /dev/console, vous n'avez pas besoin d'ajouter une entrée dans /etc/ttys. Vous n'avez qu'à vérifier que la ligne /etc/ttys pour /dev/console est marquée comme étant on. Vous voudrez probablement changer le type de terminal de vt220 à vt100 ou un quelconque autre type que vous voudrez connecter à celui-ci.

H.12 - Comment construire le câble pour utiliser les ports série cachés sur une machine Series 400 ?

Construisez le sympathique câble répartiteur en trois chemins. Autrement, un câble série DB25 classique devrait bien fonctionner si vous ne désirez qu'un seul port série.
Notez qu'un répartiteur n'est pas nécessaire pour les 425e, sur lesquelles les trois ports série sont à l'extérieur du panneau arrière.
