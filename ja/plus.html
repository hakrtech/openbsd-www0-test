OpenBSD 3.3 と OpenBSD-current との間の変更点

以下は、機種に依存しない主な変更点の部分的なリストで、これらは、 皆さんがかなり頻繁に問い合わされる変更点でもあります。ports に 関する変更も行われており、それらについては、ときどき、それぞれの 機種に関するページに記載されています。

ports コレクションの変更点については、 ここに記述されています。

注: パッチの存在する問題については、朱書きで記されています。

他のリリースの変更点については、こちらをご覧ください。
2.0、 2.1、 2.2、 2.3、 2.4、 2.5、 2.6、 2.7、 2.8、 2.9、 3.0、 3.1、 3.2、 3.3。

私たちは OpenBSD-current 上で作業しています。

以下のリストは、5 月 25 日までに加えられた (ほとんど) すべての変更を要約したものです。

  • xdm(1) の XDMCP の問い合わせを修正 (XFree86 のバグ #277)。
  • 直近のネットマスクチェックの変更後に pf(4) の binat ルールが壊れないように。
  • pfctl(8) のネットマスクの正当性チェックの改善。
  • pfctl(8) がテーブルで動作中に、バッファの解放とポインタの初期化を適正に行うように。
  • ssh(1) の syslog 出力を最初に strnvis(3) を通すように。
  • ssh(1) の fdset のリークを修正。
  • 安全ではない sprintf(9) と vsprintf() 関数をカーネルから削除。
  • sd(4) デバイスに最初に発行されたコマンドのために、メディアの変更を無視。
  • カーネルの vprintf(9) のプロトタイプがユーザランドのそれと一致するように。
  • 元の問い合わせの際に getconf(1) が _POSIX_PIPE_MAX の代わりに _POSIX_PATH_MAX を返すように。
  • カーネルがプロポリス付きでコンパイルされるようになり、モジュールをこれと一緒に再構築が必要。
  • 新しいハードウェアモニタリングセンサ it(4)。
  • Centaur のチップを使用している場合に、マルチキャストの受信が失敗する原因になっていた、dc(4) のエンディアンの問題を修正。
  • altq の HFSC に欠けていた初期化を追加。
  • NetBSD から移植された読み込み専用の NTFS のサポートを追加。NetBSD では有効化されていない。
  • auich(4) のステレオを反転させるためのフラグを追加。
  • nice(3) の戻り値を -NZERO ≤ nice ≤ NZERO(ここで NZERO=20) の範囲に制限。
  • pfctl(8) は、不正なホスト名やネットマスクが導出された場合には断固として失敗するように。
  • NetBSD からの多数のゲームの修正。
  • i386 および hppa で、ブートローダが会話的処理部分を完全にスキップすることを許容。
  • tcp_input.c. で SO_DEBUG に影響をおよぼす、ひどく壊れた switch 文を修正。
  • lpr(1) で、プリントするファイルが実行形式かどうかのチェックを停止。ただし、これは lpd(8) のフィルタとして残存。
  • pfctl(8) で、altq のスループットのより良い統計情報出力を得るため、減衰フィルタを使用。
  • ssh(1) の do_log() で、コードを単一のハンドラから呼び出せるよう、コード中に syslog_r(3) を使用。
  • 特権分離された isakmpd(8) が読み書きできるよう、パスを厳しく限定。
  • インターフェイスの再スキャンを修正するため、isakmpd(8) で sockaddr の代わりに sockaddr_storage を使用。
  • 特権分離された isakmpd(8) の特権部分でのみ、X.509 の秘密鍵を保持。
  • pf(4) の SYN プロキシを使用する場合に、ACK が正しいウィンドウサイズで送信されていることを確実に。
  • 遅い USB デバイスの取り付けが完了するまで、より長く待つように。
  • 構築によって変更されるかも知れないライブラリの詳細を取り出すために Perl を configure するため、'make build' のライブラリのパスで libperl を構築しないように。もうひとつの待ち状態の「回避」。
  • 最新の asn1_compile の依存性を (再び) 除去するため、libkrb5 にターゲットの再生成を追加。
  • 欠落している 80 芯の IDE ケーブル (UDMA モード 2 以上用) についての一貫性についてのさらなる修正。
  • syslogd(8) で、utmp 構造体からの文字列の印刷時に、これらの文字列は NULL ターミネイトしていないので strlcpy(3) を使用しないように。
  • 自身の IP アドレスのエイリアスをローカルと見なして ARP を行わないように。
  • NetBSD からの多数の USB SCSI デバイスの更新のマージ。
  • ssh(1) のサイファの aes-ctr のための試験的サポートを追加。
  • FireWire コードのためのいくつかの USB SCSI の改善を適用。
  • ssh(1) の rhosts 認証コード中の sscanf(3) に対する文字列長を追加。
  • systrace(1) の下でのディレクトリ生成の修正を導入。
  • ビットマスクおよびソースハッシュアドレス選択を使用したアドレスプールを伴なう pf(4) の rdr のルールを修正。
  • pf(4) のタグの逆マッチを許容。
  • Intel の dc(4) デバイスのメディアハンドリングを修正。
  • spamd(8) の接続ハンドラ中の正しいバッファを使用。
  • 引数を保存する際により多くのメモリが必要となった場合に、malloc(3) の代わりに vfprintf(3)mmap(2) を使用。
  • すべてのフェーズ 2 SA の再交渉の原因となる isakmpd.conf(5) の [general] セクションのための新しい Renegotiate-on-HUP オプションを追加。
  • ksh(1) 中のいくつかの符合関連のバグの卵を除去。
  • USB SCSI サポートの改善。
  • mg(1) の上下カーソル移動を修正。
  • ksh(1) は、自身のものではなく libc の dup2(2) を使用するように。
  • さようなら、Kerberos IV。
  • 'synproxy state' (これにより暗黙のうちに変調状態となります) で有効化される pf(4) の TCP SYN プロキシによるもうひとつの高価なファイアウォール機能。
  • -4 や -6 コマンドラインオプションと同じように機能する、ssh(1) のための新しい AddressFamily オプションを追加 (移植版の OpenSSH のバグ [管理番号 534])。
  • wi(4) が sparc64 上で機能するようにアドレス比較を追加。
  • 不正な入力のみからなる設定ファイルによる spamd-setup(8) のクラッシュを回避。
  • IPv6 の近隣発見において、rt->rt_ifp が正しいものと仮定しないように。
  • ssh(1) に新しい ConnectTimeout オプションを追加。
  • Kerberos がコンパイルされていない場合には、ssh(1) 関連プログラムの Kerberos オプションを無効化し、それが使用されていれば警告を出力。
  • 'ssh -V' で long を %s で出力しようとしないようにして、OpenSSL のバージョンが正しく表示されるように。
  • IPv6 の IPsec のフォワーディングを修正 (PR#3231 の修正)。
  • hosts_access(5) ファイルが 2048 文字以上の行を含む場合の libwrap のハングアップを修正 (NetBSD pr#15025)。
  • sftp(1) の ls コマンドにマルチカラム出力を追加。
  • 信用できない mail(1) への入力を、表示する前に vis(3) で洗浄。
  • isakmpd(8) で、多数に回覧するデータ構造の中に秘密鍵を保存しないように。
  • 新しい ssh(1) の dnsfp コード中に欠落していた freerrset(3) を修正。
  • 人間の読めるスケールで読み書きを行うため、新しい fmt_scaled(3) および scan_scaled(3) 関数を libutil に追加。
  • sysctl(8) と同様、mixerctl(1) の -w オプションを沈黙させるための -q オプションを追加。
  • isakmpd(8) に特権分離を準備。これは現時点ではまだ、デフォルトでは有効化されていない。
  • ssh(1) の認証方法ごとの登録解除を修正。
  • faithd(8) で、getaddrinfo(3) が sctp をサポートする際の一日の予想に IPPROTO_TCP を明示的に指定するように。
  • すべての KerberosV ライブラリを /usr/lib に移動するための準備。
  • さらなる TCP の scrub。NAT の検出を妨害し、リモートからの uptime の推測を防止するため、TCP のタイムスタンプを変調。新しい scrub のオプション 'reassemble tcp' を追加。
  • le(4) のさらなる不要な 'lost carrier' メッセージを除去。
  • 短命なカーネルオプション LONGRUN を除去し、SMALL_KERNEL を除く標準を定義。
  • rdr および binat ルール用の pf(4) のタグ付きサポートを有効化。
  • isakmpd(8) の特権分離用に _isakmpd ユーザならびにグループを追加。
  • リモートサーバが BREAK をサポートする場合に、ssh(1) クライアントがそれを送信することを許容 (SSH v2 のみ)。
  • KerberosV の kdc(8) および kadmind(8) デーモンが期待する _kdc および _kadmin ユーザならびにグループを追加。
  • i386 に Transmeta の LongRun 電源管理のサポートを追加 (カーネルオプション LONGRUN、デフォルトで有効化)。
  • pf(4) のタグを最後のマッチではなく、それぞれのマッチしたルール用に追加。
  • /etc/rc および /etc/rc.conf から gated 関連のものを除去。
  • DNS レコード (dnsfp) を使用した ssh(1) のホスト鍵の照合の試験的なサポートを追加。デフォルトでは構築しない。その詳細と構築手順については src/usr.bin/ssh/README.dns を参照。
  • 64 ビットアーキテクチャでテストされた malloc(3) の map_pages() が失敗しないように。
  • 多くの最近の isakmpd(8) の変更を、それらが正しく動作するようになるまで取り消し。
  • XFree 中の KerberosIV のサポートを無効化。
  • 監視中の親プロセスがいなくなった場合に、ssh(1) の特権分離された子プロセスも終了するよう確実に (OpenSSH のバグ [管理番号 560])。
  • 新しい bpf(4) のリンクタイプも使用するよう、pflogd(8) をアップグレード。
  • tcpdump(8) と libpcap に対し、bpf(4) 中の新しい pflog(4) のリンクタイプについて指示。
  • pflog(4) の新しいリンクタイプのための bpf(4) のサポートを「公式」なものに。また、より拡張可能なバージョンを libpcap から導入。
  • プログラムからの KerberosIV のサポートの除去作業の開始。
  • getaddrinfo(3) で数値のノード名を扱う場合に、RFC3493 ごとの数値のアドレスに正規のホスト名を設定するように。
  • vis(3) の VIS_SAFE の挙動が isgraph(3) に関する man ページにマッチするように。
  • pf(4) の block ルールへのタグの指定を許容 (ただし、これは、状態を保持する場合には許容されない)。
  • pf.conf(5) の scrub キーワードが再びプロトコル指定子を伴なうことを許容。
  • KerberosV のコードから KerberosIV のサポートを除去。
  • pf(4) の nat ルールにパケットのタグのサポートを追加。
  • libreadline の文字列長問題と欠落していた NULL による初期化を修正。
  • kerberos-over-ssh2 のサポートを ssh(1) に追加。
  • Ethernet の定義を <net/ethertypes.h> に再移動。ただし、現時点では、互換性のために <netinet/if_ether.h> はこれらを読むように。
  • -w オプションの出力を抑制するための新しい -q オプションを sysctl(8) に追加。
  • libexec/telnetd のファイルから telnet(1)、telnetd(8) および tn3270(1) に直接コンパイルされたコードを入れる代わりに、libtelnet を削除することで循環依存を解消。
  • libkafs のコンテンツを libkrb5 に移動。libkafs は空のダミーライブラリとして残留。
  • カーネルの pf(4) の新しいタグ付きコードの使用後解放 (use-after-free) を修正。
  • i386 の RAMDISK* カーネルで、ますます人気の出ている em(4) ドライバをデフォルトで有効化。
  • rdist(1) を freerdist バージョン 0.92 の圧縮オプションを除いたものに同期。
  • KerberosIV プログラムを構築しないように。現時点ではライブラリはまだ構築。
  • blktochr() および chrtoblk() をカーネルの機種独立コードに移動。
  • カーネル用の便利な関数 [bc]devsw_lookup() を追加。
  • pfctl(8) で、パケットのタグ付けが状態を持つフィルタルールでのみ使用されるよう確実に。
  • スタック保護なしでカーネルを構築するための、カーネル用 config(8) オプションに NO_PROPOLICE を追加。インストールメディア用に便利。
  • libreadline の文字列長のひとつ違い (off-by-one) を修正。
  • pf(4) のパケットのタグ付けサポートのユーザランド部分を追加。
  • afs が KerberosIV なしで動作するようになるまで、これを無効化。
  • sparc の microtime(9) と可能性のある他のものも同様に壊れないよう、グローバルの 'time' 構造体を強制的に quad_t にアラインするように。
  • カーネル内の pf(4) でタグ付けされたパケットのサポートと、そのタグに基づくフィルタリングのサポートを追加。
  • 新しい mbuf(9) のタグ PACKET_TAG_PF_TAG を追加。
  • lndir(1)で、ディレクトリを読み込む際に、配列のインデックスとして -1 を使用しようとしないよう確実に。
  • ELF アーキテクチャ上で、スレッドセーフではないダイナミックローダの処理中のスレッド切り替えのブロックをサポート。
  • ELF の識別インデックスを SystemV ABI の仕様と同期。
  • AM7990 (le(4)) デバイスの 'lost carrier' メッセージの出力を停止。
  • ユーザランドの炉心溶融の原因となったため、<netinet/if_ether.h> の変更を取り消し。
  • スタック保護のプロポリスのサポートをカーネルに追加し、プロポリスをサポートするアーキテクチャ上では、これを使用してカーネルを構築。
  • NULLFS、UMAPFS および UNION を GENERIC カーネルで再有効化。
  • NetBSD 同様、Ethernet の定義を <netinet/if_ether.h> から新しい <net/ethertypes.h> に移動。
  • login_krb5(8) および login_krb5-or-pwd(8) で、KerberosV から IV への変換を無効化。
  • login_krb4(8) および login_krb4-or-pwd(8) の構築を停止。
  • login.conf(5) から krb4 への参照を除去。
  • i386 で取り扱われる保護例外のトラップの方法を変更。詳細と Intel の誤用はチェックインのコメントを参照。
  • Heimdal KerberosV 0.6 をマージ。
  • user(8) がスラッシュから始まるユーザ名を受け付けないように。
  • サポートしていない scsi(4) デバイスをオフラインとして報告しないように。
  • pf(4) が TCP のウィンドウサイズを検査する際に、SYN パケット用のウィンドウスケールの要素を適用しないように。しかしながら、ACK の検査の際にはスケールの要素を適用するように。
  • 再帰的な依存関係のサーチが失敗する原因となる pkg_add(1) のバグを修正。
  • isakmpd(8) に対する、NAT-T、IKEv2 および EAP 用のさらなる記述を追加。
  • unionfs に対するロックや他の修正。
  • isakmpd(8) に BLOCK_SIZE 属性を追加し、AES ESP 変換を AES_128_CBC にリネーム。
  • 将来の NAT-T のサポートのための視点から isakmpd(8) に UDP のカプセル化の (コードではない) 型定義を追加。
  • genfs からの共通コードを使用するための nullfs と umapfs の改造。
  • 階層化ファイルシステムのサポートのための新しい genfs コード。
  • 印刷キュー名を出力する前に vis(3) を通じて洗浄。
  • '//' コメントを理解し、関数型の定義を無視し、__attribute__ を受理するための ctags(1) の改造。NetBSD から移植。
  • at(1) コマンドからの空の出力を正しくチェックするように (PR#3252)。
  • プロセス情報を表示するための新しい ddb(4) のコマンド 'show proc' を追加。
  • popa3d(8) のバージョンを 0.6.2 に同期。
  • インストーラの ftp および http のダウンロードのハンドリングに対する改善とバグ修正。
  • pf(4) での IPv6 のアドレス比較を、より異なり易いので LSB から最初に比較するために並べ替えるように。
  • pf(4) のアタッチの際の初期化で状態探索ツリーが適正であることを確実に。
  • OpenBSD では使用されなくなった KerberosV のファイルを多数削除。
  • ssh(1) で公開鍵認証を行う際に、設定ファイル中を参照するエージェントが保存した鍵を優先するように。これにより、エージェントが多数の鍵を保存している場合に、有効な鍵を取得する前にサーバが接続を切断する可能性を減少。
  • KerberosIV 削除の準備を開始。
  • game/ の下の多数の冗長な定義を削除。NetBSD から。
  • file(1) は、Ogg Vorbis のオーディオファイルを認識可能に。
  • asn1_compile のコンパイルの際に、src/kerberosV ではなく src/usr.bin を使用するように。
  • libreadline に対するさらなる文字列の修正。この時点では ABI は変更せず。
  • csh(1) の符合のオーバーフローを修正。
  • OpenSSL 0.9.7b を (IDEA、MDC2 および RC5 を除いて) マージ。
  • 多数の状態が存在する状況で逆に状態のタイムアウト値を減少するため、適応性のある状態テーブルのタイムアウトを pf(4) に実装。
  • asn1_compile を KerberosV から出して src/usr.bin に移動。
  • pf(4) の状態を持つ scrub の最初のステップ。最も大きな TTL の動的な決定による TTL ベースの NIDS の回避を失敗させ、そのすべての後続のパケットに最小の TTL を強制。
  • ssh(1) で、呼び出し元が remote_name の xstrdup() を行う代わりに、channel_new() の内部でそれを行うように。
  • 生成されたファイルをツリー中に置くことにより、KerberosV のアップグレードの際の迷惑な asn1_compile の飛び越え問題の修正を開始。
  • install(1) がすべてのファイルシステムを書き終えることによって生成されるわずかなファイルの最後の穴を確実に。
  • インストーラは、完全にローカルのセットの絶対パス (これは仮想的なマウントポイントではなく、インストーラのルートディレクトリに関係あり) を受け付けるように。
  • ssh-keygen(1) の -e オプションに、ユーザが SSH1 の鍵を指定した場合に、素直に失敗するように (NetBSD pr#20550)。
  • 'Kang' のスペルの修正により、クリンゴンとの攻防戦を回避。
  • Heimdal KerberosV 0.6rc1 をマージ。
  • mfs はシグナル受信時に強制的なアンマウントを試みないので、プロセスの作業ディレクトリを解決する必要がなかった。- このため、unmount(2) が失敗。
  • isakmpd(8) の IPV6_ADDR ID タイプのハンドリングを修正。
  • 'nat ... -> $if port n' ルールが壊れないようにするため、pfctl(8) の ntohs(3) を修正。
  • pf(4) は、(UDP や ICMP ではなく) TCP 以外のすべてのプロトコルのための ICMP unreachable メッセージを今生成するというキーワードを返すように。
  • カーネル内で安全ではない文字列関数が使用された場合に、コンパイラは警告を生成。
  • 静的な構築が動くようになるまで、libreadline の文字列の修正を取り消し。

  • 4.4BSD.dist ファイル中に記述の存在する /etc/isakmpd/crls ディレクトリが存在していないと言って isakmpd(8) が警告を出力しないよう、このディレクトリを生成。
  • ssh(1) で、オプションの解析前に行末の空白を除去 (OpenSSH のバグ [管理番号 528])。
  • ssh(1) の Kerberos IV サポートを無効化。
  • spamd(8) 中の select(2) のエラーハンドリングを修正。
  • mg(1) は、前回の 'M-x compile' コマンドを記憶しておくように。
  • httpd(8) の dbm の処理に欠落していた free() を追加。
  • osiop(4) に対する、さらなる修正。
  • いくつかの古いスタイル (user.group) の chown(8) を POSIX スタイルの user:group に変更。
  • savecore(8) の NULL 被参照を修正。
  • いくつかの欠落していた NetBSD の著作権情報を ftp(1) に追加。
  • ktutil(8) がネットワーク越えでも正常に稼働するように。
  • adventure(6) の文字列のハンドリング (今回は str[ln]* だけではなく) の改善。
  • ports は XF4 のインストールのためのチェックを行うことができるので、偽の packages 情報を追加。
  • XFree 配布スクリプトで ksh(1) を csh(1) の代わりに使用。
  • pfctl(8) は、不正な ICMP タイプ (>40) およびコード (>255) を排除するように。
  • 取得すべきパケット状態カウンタの不正な状態の原因となっていた、ssh(1) の新しい鍵再生成コードのタイポを修正。
  • sudo(8) を 1.6.7p3 に更新。
  • kvm_mkdb(8) の strlcpy(3) のバッファ長を適正に取り扱うように。
  • osiop(4) に対する多数の修正。
  • (文字列関数の修正の一環として) sort(1) で、古いスタイルの - および + のフォーマット指定子のハンドリングを改善。
  • disklabel の d_packname および d_typename フィールドが NULL ターミネイトされたフィールドではない場合に、そうであるとして仮定しないように。
  • lam(1) の境界チェックのひとつ違い (off-by-one) エラーを修正。
  • pfctl(8) の CIDR マスクの解析部を単純化。
  • 指定さえたインターフェイスにのみ適用するための pf(4) のキュースペックを許容するため、'queue foo on $i_bar' の文法を追加。
  • compat_linux(8) における termios のサポートする速度のリストに 230400 を追加。
  • 多くの NULL テストを省略できるよう、ルールベースの先頭の暗黙の 'pass all' に従うパケットの通過許可時にデフォルトルールを使用。
  • `デフォルト' の pf(4) ルールを追加し、デフォルトのタイムアウトを格納するために使用。
  • XFree86 4.3.0 のマージの際に欠落していた、いくつかの更新されたバージョンのシェアードライブラリを追加。
  • 多数の文字列関数をすべてのツリーにわたって修正。
  • OpenSSL の ASN.1 解析部の不正な境界チェックを修正。
  • tcpdump(8) への以前の realloc() の変更を取り消し。
  • sectok_fmt_fid(3) が文字列長パラメータを取るよう更新され、新しい API に合わせて libsectok のメジャーバージョンを更新。
  • XFree86 4.3.0 のマージに伴ない、ports のライブラリは結局 /usr/local/lib/X11 に入れられることになったので、付加的な記述を追加。
  • sudo(8) を 1.6.7p2 に更新。
  • user(8) の空のグループテストを修正 (PR#3178)。
  • PRIQ キュー ID の割り当てが改善され、異なるインターフェイス上の同一優先度のキューが正しく動くように。
  • tcpdump(8) のメモリリークを修正するため、realloc(3) を使用。
  • ipcomp(4) および ipsec(4) のいくつかのクリーンアップ。
  • ssh(1) に欠落していた初期化を追加 (OpenSSH のバグ [管理番号 #526])。
  • インターフェイスが altq(9) をサポートしない場合、pfctl(8) はエラーメッセージ中にインターフェイス名を表示するように。
  • 現在の secsh の新しいモード案に従って、ssh(1) に自動鍵再生成を追加し、いくつかの鍵再生成バグを修正。
  • 直接モードの bpf(4) のイベントにおける kqueue(2) の通知を修正 (PR#3175)。
  • XFree86 4.3.0 をマージ。
  • ある種の過剰な偏執狂的挙動を修正するため、sudo(8) をバージョン 1.6.7p1 に更新。
  • OpenSSH のバージョンが 3.6 に増加。

  • icmp6 の mbuf のリークを修正。
  • ftp(1) は、環境変数が unset されたかのように、空の環境変数を仮定。
  • ipcomp(4) および ipsec(4) の暗号化エラーのハンドリングの際の、いくつかのメモリ解放後のアクセス (use-after-FREE) を修正。
  • ipcomp(4) に欠落していた splx() を追加。
  • setusercontext(3) に対するクリーンアップと付加的な偏執狂的改修。
  • カーネルの pf(4) ルール構造体を参照する状態がない場合にのみ、それを削除。
  • TCP_DEBUG が定義されている場合に、有用なように netinet/tcp_debug.c のコンパイルを許容。
  • ahc(4) のデュアルチャネル 7899 カードの検出を修正。
  • カーネル内でデバイス名を組み立てる際に、手動ではなく snprintf を使用。
  • ユーザにより、何らかの理由で passwd(1) がアボートさせられた場合に、より一貫性のあるメッセージを出力するように。
  • なおも残る、ツリーからの strcpy、sprintf および strcat 呼び出しの根絶のための作業を開始。
  • spamd(8) のログ機能の不具合を修正。
  • sendmail(8) のアドレス解析部のバッファオーバーフローを修正するため、8.12.9 に更新。この修正は OpenBSD 3.3 の CD に反映されるので、3.3 の errata ではないことに注意。
