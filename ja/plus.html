OpenBSD 3.4 と OpenBSD-current との間の変更点

以下は、機種に依存しない主な変更点の部分的なリストで、これらは、 皆さんがかなり頻繁に問い合わされる変更点でもあります。ports に 関する変更も行われており、それらについては、ときどき、それぞれの 機種に関するページに記載されています。

ports コレクションの変更点については、 ここに記述されています。

注: パッチの存在する問題については、朱書きで記されています。

他のリリースの変更点については、こちらをご覧ください。
2.0、 2.1、 2.2、 2.3、 2.4、 2.5、 2.6、 2.7、 2.8、 2.9、 3.0、 3.1、 3.2、 3.3、 3.4。

私たちは OpenBSD-current 上で作業しています。

以下のリストは、11 月 10 日までに加えられた (ほとんど) すべての変更を要約したものです。

  • OpenSSL 0.9.7c をマージ。
  • dc(1) と bc(1) への移植性のないシンタクス関連の変更。
  • スタック変数の free(9) は良くない考えなので、ubsa(4) でこれを行わないように。
  • ライブラリ名が正しくない場合に ld.so(1) でメモリリークが発生しないように。
  • ld.so(1) で、より良いライブラリのバージョン番号の解析を行うように。これにより、'libpython2.1.so.0.0' と 'libpython2.2.so.0.0' とが同時に存在可能に。
  • bc(1) 用の新しい 'print' 命令。これは移植性のない拡張。
  • ksh(1) で、たとえば、'2>&2' のようなファイルから同じファイルへのリダイレクションのハンドリングを修正。
  • sendmail(8) の cf/openbsd-proto.mc に対する、HELO/EHLO を要求し、EXPN/VRFY を無効化する、さらなるプライバシーフラグの追加。
  • /usr/share/doc/smm/17.password のパスワードセキュリティに対し、古典的な論文を追加。
  • 互換性のため、diff(1) の出力の「ファイル末尾の改行の抑止」を、stdout ではなく stderr に出力するように。
  • pkg_add(1) が、追加済パッケージをさらに追加しようとするエラーを停止。
  • pkg_add(1) で複数のパッケージを追加する際のエラーの追跡を維持し、戻り値として役に立つエラーコードを設定するように。
  • 最初の @cwd からパッキングリストのプレフィクスを自動的に設定するのを除去。
  • carp(4) が追加される際に突発的に壊れる、ifconfig(8) の vlan(4) の情報の表示をリストア。
  • mg(1) のファイルの挿入を本当に修正。
  • mg(1) でより安全な領域のハンドリング。
  • mg(1) がアボートする際、端末を正しくリストア。
  • mg(1) のファイル挿入操作を正しくアンドゥ可能に。
  • pfsync(4) が返すアンカールール番号が壊れないように。
  • プロセスのスワップインの際の競合状態を回避。
  • i386 で、プロセス数が大きい場合に (> 1500) 発生するクラッシュを修正 (PR#3528)。
  • pfsync(4) インターフェイス上に現れる特定のルールでの状態遷移を防止するための、新しい 'no sync' 状態オプション。
  • carp(4) パケットが carp を有効化したインターフェイス上で受信されていることをチェックするように。
  • IPv6 のリンクローカルなマルチキャストの結合用のインターフェイスのインデックスの設定を修正。
  • インターフェイスがダウンしている場合に carp(4) が ARP に反応していたのを停止。
  • sed(1) で正規表現を代用する場合のバッファオーバーフローを修正。FreeBSD から。
  • 移植性のない拡張を dc(1) に追加。'#' (コメント)、'n' (改行なしの表示) および 'a' (バイトから文字への変換)。
  • より良い pkg_add(1) の依存性の解決。
  • 完全な形でインストールされない packages で、インストール後のスクリプトを呼び出さないように。また、そのような packages でも完全に除去することが可能に。
  • pkg_add(1) で、stdin から入力された packages のインストールが可能に。
  • pkg_delete(1) で、インストール済 packages へのパスをコマンドライン上で指定可能に。これにより、たとえば、'pkg_delete /var/db/pkg/zsh-*' が動作するように。
  • パッケージツールは、パッキングリスト中の最初の @cwd のターゲットを自動的にプレフィクスとして使用するように。
  • インターフェイスの互換性確認と pf_test() の最近の順序変更を一時的に取り消し。pf(4) の rdr ルールは、現時点では、ある種の正しくないチェックサムを生成。
  • isakmpd(8) で、暗号化メッセージの鍵の状態を保持しているので、直ちに暗号化メッセージを要求し、ハッシュが添付されるべきペイロードの削除を要求し、動的な交換なしにハッシュのペイロードの妥当性確認を追加。
  • 同じマクロをラベルとして使用するための pf(4) のタグを許容 (pf.conf(5) を参照)。
  • gdb(1) に SIGINFO について指示 (PR#3173)。
  • 標準的なビルドにおける、それぞれの httpd(8) モジュール用に、短かい記述の付加された、コメントアウトされた LoadModule 設定行を追加。
  • newfs(8) で、ファイルシステムの生成が完了するまで、スーパーブロックにマジックを書き込まないように。
  • netstat(1) の IPv6 のリンクローカルなマルチキャストアドレスの表示を修正。
  • 今回はより慎重に wdc(4) のドライブリセットの変更を再実施。
  • pfsync(4) 用に tcpdump(8) の -x フラグが動作するように。
  • リスンするソケットの参照においても可能なところでハッシュテーブルを使用。
  • carp(4) マスタのホストの場合にルートを追加し、ローカルマシンが共通のアドレスを使用できるように。
  • pkg_create(1) は、複製されたパッケージングリストのエントリを修正し、これが行われた場合にそれを表示して終了するように。
  • carp(4) が IPv6 の bpf(4) にすべてを送信しているようふりをするのを停止。
  • dc(1) に GNU 互換の 'r' 演算子 (スタック上の先頭のふたつのアイテムを交換) を追加。
  • インターフェイスアドレスが設定された場合に carp(4) の IPv6 サポート中の、過去の IPv4 サポートを削除。
  • 信頼性のための修正: ローカルユーザは、不正なヘッダを持つ特別に組まれたバイナリを実行することによりシステムをパニックに陥れることが可能です。
    ソースコードパッチはこちら。
    [安定版への適用]
  • pkg_delete(1) は、パッケージ名の幹を使用する場合に依存性を正しく扱うように。
  • ssh(1) のホスト鍵をチェックする際に、静的な文字列の free を試みないように。
  • pf(4) がパケットを落としたり、異なるインターフェイスに送ったりするので、(pf ではない) 通常の IP の出力コードで、ハードウェア IPsec 用のインターフェイスの確認およびチェックサム機能を pf_test() の後まで遅延。
  • pf(4) によってルーティングされたパケットが、IPsec 用のターゲットのインターフェイスとチェックサムの機能をチェックするように。
  • carp(4) がインターフェイスをプロミスカスモードに移行させるのを失敗した場合のメモリリークを修正。
  • IPv6 用の carp(4) に欠落していた、ダウン中のインターフェイスのチェックを追加。
  • pf(4) の有効化時にデバッグフラグを保存。
  • top(1) で、正しい時刻にシグナルをチェックし、stdin の失敗をより良く扱うように。
  • patch(1) は GNU patch と同じ方法でファイル名を決定するように。
  • patch(1) 用の厳密に POSIX に従うための新しい --posix オプション。
  • pkg_add(1) 等の pkgpath を正しい順序で設定。
  • wdc(4) に SATA モードと reset-pause-IDENTIFY の修正を再追加。さらなるテストにはドライブのリセットの修正が必要。
  • hashinit(9) で、正しい要素番号を割り当てるように (PR#3537)。
  • pkg_add(1) 等のコマンドで、tar ファイルから gid を取得する際、(ユーザ名ではなく) グループ名を見るように。また、モードの前にファイルの所有者を設定するように。
  • carp(4) に IPv6 のサポートを追加。
  • 2003 年 10 月 1 日時点の NetBSD の libedit に同期させた上で、いくつかのローカル文字列のクリーニングとヒストリのバグを修正。この更新の結果、いくつかの API を変更。
  • 新プラットフォーム OPENBSD/pegasos のサポート。
  • route6d(8) で、不十分な長さのチェックを修正 (KAME PR#507)。
  • init(8) の strdup(3) の失敗に対する処理の試み。
  • スペル警察署ダブルワード分署の検出作業強化 (ソース・ドキュメント類のスペルチェック強化)。
  • lc(4) のマルチキャストフィルタの初期化の修正。
  • 破壊的なところがあるので、最近の wdc(4) のリセット、識別ならびにモード検出の変更を取り消し。
  • ネットワークブロックが (単なる単一アドレスではなく) ルールの送信元として使用されている場合の、着信して来た接続に対する pf(4) の binat を修正 (PR#3535)。
    [安定版への適用]
  • 安定性のための修正: httpd.conf や .htaccess ファイルへの書き込みパーミッションを持つユーザは、httpd(8) をクラッシュさせることができる可能性があり、あるいは、www ユーザとして任意のコードの潜在的な実行可能性があります (しかし、ProPolice がコードの実行を阻止してくれるはずです)。
    ソースコードパッチはこちら。
    [安定版への適用]
  • growfs(8) で、正しいパーティションを見つけるためにより良い仕事をするように。
  • pkg_delete(1) で、より早い時期に依存性を評価し、そのチェックが失敗した場合には必要な削除リストを与えて終了するように。
  • ssh(1) で、getsockopt(..., TCP_NODELAY, ...) が失敗した場合にも異常終了しないように。
  • リセット後に不具合の出るユニットを使用可能にするため、wdc(4) のドライブリセットと識別コマンドとの間にポーズを追加。
  • いくつかの SATA ドライブでは ATA モードの検出を実際に嫌うため、これを行わないように。
  • carp(4) アドバータイズの再スケジューリング時に正しくスキューを設定するように。
  • mg(1) 起動時のクラッシュを修正。
  • インターフェイスがダウンする最中で mbuf を消費し尽している場合に、carp(4) アドバータイズのスケジューリングを行わないように。
  • carp(4) インターフェイスがダウンしている場合にはアドバータイズの送信を実際に停止するように。
  • sysctl(3) の kern.stackgap_random の最大値を 256MB に設定。
  • growfs(8) で拡大可能なパーティション数の人為的な制限を廃止。
  • PRISM 2.5/3 USB アダプタ用の wi(4) の初期サポート。現状は非常に限定的。
  • wdc(4) のリセットコードがより FreeBSD や NetBSD と同様のものとし、マスタの挙動がおかしい場合のスレーブデバイスの検出を修正。
  • メモリ使用量を 30% 節約するため、pf(4) の状態検索を再構成。
  • carp_output() の失敗時に mbuf をリークしないように。
  • ローカルの IP ポートの参照に使用していたリンクリストをハッシュテーブルに置換し、多数のソケットが存在する場合の参照時間 (in_pcblookup()) を劇的に減少。
  • 可能な限り多数の carp(4) sha1 ハッシュを事前に計算しておくように。
  • syslog.conf ファイル変更後の SIGHUP 受信により、時どき発生していた syslogd(8) のハングアップを防止。
  • 小数の int と NULL との比較を除去。
  • /etc/group が使用中かも知れないので、httpd(8) の chroot 後ではなく、chroot 前に initgroups(3) を実行するように。
  • DHCP がブレークしないよう、フィルタが設定されていない場合にすべてをブロックする新しい bpf(4) 用書き込みフィルタを停止。
  • pkg_delete(1) で、依存パッケージの削除を一度だけ試みるように。
  • carp(4) で、内部ハッシュの sizeof(pointer) バイトだけではなく、完全な内部ハッシュを入れて撹拌するように。
  • 最終的に、フロッピードライブが取り外された i386 ラップトップのユーザのために、長い長いポーズを停止。
  • pkg_info(1) が共通の幹を共用する複数の packages に対して正しく動作するように。たとえば 'pkg_info autoconf' に対する複数の応答など。、
  • pkg_delete(1) が packages 名の幹に対して動作することを許容。
  • 今回は tn3270(1) で、欠落していたもうひとつの strdup(3) のエラーチェックを追加。
  • デフォルトで spamd(8) が記録するログの総量を抑制。新しい -v オプションで詳細なログを取得するように。
  • 新しく分離された pflogd(8) での同様の変更のように、特権分離された syslogd(8) が下位の特権に変更する際に setgroups を呼び出すように。
  • 不整合状態の MSDOS ファイルシステムを横断する際のパニックを修正。NetBSD から。
    [安定版への適用]
  • pflogd(8) で特権分離を実装。_pflogd ユーザならびにグループの生成が必要。
  • root 特権を持たないプログラムが、リンクレイヤや危険な ioctl(2) の発行を好んでも、書き込み権限なしに bpf ディスクリプタを保持できるよう、bpf(4) にロックと書き込みフィルタリングを追加。
  • 新しく比較が拡張された dc(1) の J 演算子を修正。
  • 鍵付き sha1 から hmac-sha1 に carp(4) を切り替え。
  • bc(1) で if ... else の構成を許容するため、dc(1) で拡張比較演算子を実装。
  • dc(1) で、文字列から取得していない文字が、ファイルからの場合と同じように動作するように。
  • libpthread での kqueue(2) のファイルディスクリプタのリークを修正。
  • libpthread で、目前の execve(2) を延命させるため、フラグが設定されていないディスクリプタでわざわざ O_NONBLOCK のリセットを行わないように。
  • tic(1) で、欠落していた strdup(3) のエラーチェックを追加。
  • mg(1) で、バッファ単位ではなくウィンドウ単位で undo が動作するように。
  • <expat.h> の enum XML_Status の最近の定義を修正。expat CVS から。
  • ツリー全体にわたる膨大な数のコメントのスペルの修正。
  • GSSAPI OID が短か過ぎる場合には ssh(1) が異常終了するように。
  • 新しいパッケージツールに切り替え。
  • netstart(8) で、物理インターフェイスが設定された後まで carp(4) インターフェイスを初期化しようとしないように。
  • carp(4) sha1 コードのエンディアン関連のバグを修正。
  • ppp(8) で realloc(3) のクリーンアップ。
  • プロンプトが無効化されている場合、carp(4) のすべてのマスタステータスのホストアドバータイズを停止。
  • carp(4) を実行する場合に、ユーザが advbase と advskew を両方とも 0 に設定しようとしている場合は ifconfig(8) でエラーをひとつだけ出力するように。
  • nc(1) で、欠落していた strdup(3) の戻り値のチェックを修正。
  • pthread を除外した select(2) での numfds==0 の場合を修正。
  • packages 名の '幹' (バージョン番号を抜いた packages 名) を見つけるための関数群を追加し、間もなく利用可能になる pkg_info(1) でそれらを利用。
  • SSHFP 資源レコードの直接のサポートを named(8) に追加。
  • bc(1) の割り当て演算子 (+=, -= etc.) を修正。
  • dc(1) に J(jump) および M(mark) 演算子を追加し、bc(1) の continue ステートメントの実装にそれらを利用。
  • make(1)、libfreetype および xterm(1) の境界外 (out-of-bounds) からの読み込みを修正。
  • 最近の vnd(4) の番号付けの変更が期待どおりに動作するように。
  • IP の高可用性およびロードバランスのための OpenBSD carp(4) (Common Address Redundancy Protocol) を導入。
  • 64 ビットアーキテクチャ上で httpd(8) SHA1 コードがブレークしないように。
  • (arc4random() を使用して得られる) inode の生成番号が正の値であることを確認するように。
  • pciide(4) の DMA の信頼性のための修正。NetBSD から。
  • vi(1) の文字列用ではないバッファ用の strlcpy(3) を memcpy(3) に。いくつかの執着した特別な修正も同時に実施。
  • mount 要求が来るまでの select(2) の前にハンドリングできるよう、mountd(8) の早期のシグナルをチェック。
  • 新しいパッケージ管理ツールを src/usr.sbin/pkg_add に導入。ただし、まだデフォルトではビルドしない。
  • ページサイズかそれより大きな塊の後に保護ページを追加するための、また、ページサイズ未満の塊をランダムな順序で返すための、malloc.conf の新しいオプション 'G'。
  • wdc(4) のより良い SATA のサポート。
  • faithd(8) の poll(2) への引数を修正。
  • libcurses の境界外 (out-of-bounds) からの読み込みを修正。
  • 致命的なエラーの際、tip(1) はターミナルを検知可能な状態に戻すように。
  • 実行時のメモリ不足以外のどのようなエラーでもプロセスを異常終了させるよう、malloc(3) を変更。これは、どのようなエラーでも異常終了させる malloc.conf の 'A' スイッチとは異なるもの。
  • ssh-agent(1) および sshd(8) の生成する一時ディレクトリがよりランダムになるように。
  • これまでデフォルトではビルドされなかった ssh(1) の DNS の指紋 (sshfp) の参照コードをビルドするように。設定ファイル中では今も有効化が必要。
  • たとえば、'MAKEDEV tty08 - tty7f' が動作するように。
  • IPv6 の入力で、分断オフセットのエンディアン変更をただ一度だけ行うように。
  • ハードウェアがときどき間違えるという事実に対する対策を実施し、sk(4) でもハードウェア受信チェックサムを行うように。
  • em(4) をサポートするデバイスで、オフロードの受信チェックサムをハードウェアで行うように。FreeBSD から。
  • timezone ファイルを再更新。今回は tzcode2003d に。
  • バグが修正され、サポートするデバイスが増強された FreeBSD と NetBSD の更新を含む、bge(4) および brgphy(4) のさらなる整備。
  • mg(1) で、^X^W を使用した場合に与えられたファイル名を忘れないように。
  • Linux エミュレーション下の shmat(2) が期待どおりに動作するように。
  • timedc(8) のバッファオーバーフローを修正。FreeBSD で発見され、ここ以外で修正されたもの。
  • dc(1) に除算および剰余の演算子 '~' を追加。
  • ツリーから GNU bc および dc を除去。
  • XFree86 4.3.99.14 から expat 1.95.6 をマージ。
  • 重複鍵の問題を解決するため、ssh(1) のエージェントで鍵を逆順に探索するように (OpenSSH のバグ [管理番号 684])。
