OpenBSD 3.1 と OpenBSD-current との間の変更点

以下は、機種に依存しない主な変更点の部分的なリストで、これらは、 皆さんがかなり頻繁に問い合わされる変更点でもあります。ports に 関する変更も行われており、それらについては、ときどき、それぞれの 機種に関するページに記載されています。

ports コレクションの変更点については、 ここに記述されています。

注: パッチの存在する問題については、朱書きで記されています。

私たちは OpenBSD-current 上で作業しています。

以下のリストは、7 月 16 日までに加えられた (ほとんど) すべての変更を要約したものです。

  • macppc 用のカードバスサポート。
  • chroot された httpd(8) が優雅にリスタートするように修正。
  • milter のビルドのサポートを sendmail(8) に追加。Makefile を参照のこと。
  • Apache 1.3.26 と mod_ssl 2.8.10 とをマージ。
  • /etc/systrace ディレクトリを追加。
  • gnupg をコンパイル可能にするための as(1) への小さな追加。
  • csh(1) のディレクトリ名の補完の際の、巨大なディレクトリでの SIGSEGV を修正。
  • atrun(8) が cron(8) の一部になったので、atrun の cronjob の削除が必要。
  • pf(4) は、より多くの !<interface> シンタクスを受理するように修正。はい、そうです。
  • top(1) は現在は BSD ライセンス。
  • pf(4) パーサは、より多くのばかげた組み合わせを容認 (non-TCP ルールの return-rst や block ルールの keep-state など)。
  • ftpd(8) は、常にパッシブデータコネクション用に高いポート番号を使用 (もう -h オプションは不要)。
  • 残された XFree86 3.3.x サーバは、I/O アクセスを得た後、root 権限を放棄。
  • MMX (SSE ではなく) を as(1) に知らせるように。
  • i386 の AGP GART サポートを追加 (vga(4) および options(4) を参照のこと)、デフォルトで有効化。
  • xterm は root 権限を放棄し、setgid(utmp) コマンドに。
  • 少なくともひとつの tcpdump(8) のバッファオーバーフローを修正。
    [安定版への適用]
  • スケルトンの UserDir ツリーを /var/www/users の下に用意。
  • httpd(8) はデフォルトで chroot() し、root の権限を落とすように変更。
  • pf(4) は、IP アドレスを付与することの可能なほとんどのインターフェイスを受け付け、そのインターフェイス上の IPv4 ならびに IPv6 アドレスをすべて処理可能。
  • cron(8) に対するいくつかの更新。
  • Apache のインストールの際の ab(1) を除去。
  • カーネルから NTP のサポートを除去。
  • ipsec(4) / ipcomp(4) から解放した構造体の再投入の試みは行わない。
  • IP-in-IP カプセル化コードの小さな修正。
  • atactl(8) に対するセキュリティモードオプションの追加。
  • さらに少しの HPT pciide(4) カードのサポート。
  • bsd.port.mk(5) 中の NEED_VERSION を廃止。
  • cvs(1) pserver の IPv6 の NULL ポインタの参照を拡充。
  • いくつかの古いアップグレード用のハックをインストーラスクリプトから除去。
  • 単に '!' を無視する代わりに、pf(4) は不正なシンタクスの '! <interface>' を無効に。
  • pf(4) インターフェイスの統計値を修正し、loginterface 機能の無効化を許容。
  • isakmpd(8) のシグナルハンドラフラグを volatile sig_atomic_t タイプに変更。
  • isakmpd(8) の GCC 3.1 のわずかな不具合を修正。
  • ahc(4) の新コードの復帰。
  • rpcgen(1) のクリーンアップ。
  • 信頼性のための修正: ペイロードの確認中に動的な交換ができると仮定をすべきではなく、さもなくば isakmpd(8) にクラッシュの危険性があります。
    この問題を回避するためのソースコードパッチはこちら。

  • isapnp(4) 上の ep(4) が alpha 上で動作可能に。
  • インストーラのファイルセットの選択ユーザインターフェイスの動作方法を改善。
  • xsystrace の潜在的なバッファオーバーフローを修正。
  • アップグレードの際に、数バージョンをスキップすることの危険性について、distrib/notes に油断に対する注意を追加。
  • 不要なもののために ssh(1) にメモリをアロケートさせず、単に破棄させるだけとした。
  • ftp(1) のアクティブモードのデータチャネルのために IP_PORTRANGE_HIGH をセット。
  • さらにいくつかの usb(4) の製品 ID を追加。
  • rmt(8) のひとつ違い (off-by-one) エラーを修正し、文字列の一般的なハンドリングを改善。
  • nc(1) の EOF ハンドリングを正常化。
  • いくつかの ssh(1) メモリリークを修正。
  • tga(4/ALPHA) ドライバ。
  • いくつかの失われた、あるいは壊れた malloc(3) および realloc(3) のチェックを修正。
  • rcs(1) で、LocalId が長過ぎる問題の後で、実際には exit(3) するように修正。
  • 多数の関数定義やプロトタイプ宣言を ANSI 化。
  • ami(4) RAID コントローラの 'SPL NOT LOWERED' エラーの原因となるバグを修正。
  • ssh-keysign(8) に setuid(root) ビットを取り敢えず戻すが、全体的に HostbasedAuthentication が無効化されている場合にのみ動作。
  • ssh-keysign(8) の Kocher timing attack を回避するため、RSA_blinding_on(3) を使用。
  • ping(8) の signal(3) の競合を修正。
  • 新しい ahc(4) が復活するまで、i386 の RAMDISK カーネルから adv(4) を除去。
  • sysctl(3) からルーティングテーブルを取得する際の NULL ポインタの参照を捕捉。
  • sis(4) が alpha 上でコンパイルおよび動作可能に。
  • ubsec(4) は正しい結果のサイズを返す。
  • bridge(4) は現在、pf(4) なしで ipsec(4) とともにコンパイル。
  • ftpd(8) のクリーンアップ。
  • 何と OpenBSD 2.5 から壊れていた、wdc(4) の PIO モードの書き込みコードを修正。
  • login(1) から不要な longjmp(3) を除去。
  • debug_malloc() で割り当てられたページが利用された試しがないため、VM_PROT_ALL の使用を取り止め。
  • bridge(4) のアドレスキャッシュバグを最終的に修正。
  • ssh(1) Compression オプションの評価の終了ポイントの差異のハンドリングを適正に。
  • X が実行された後の wsdisplay(4) の画面クリア機能を修正。
  • いくつかの X のための '-X ' から始まるパスワードを encrypt(1) のオプションとして間違った解釈を行わないよう、インストーラの動作を正しく修正。
  • いくつかの ppp(8) の互換性の癖を修正。
  • pfctl(8) のパーサにプッシュバックバッファを追加。
  • ssh-keysign(8) から setuid(root) ビットを除去し、無効化。
  • chroot(2) 後は /etc/localtime が不要なので、named(8) は tzset(3) を呼び出す必要あり。
  • 新しい ahc(4) ドライバにさらなる修正。
  • AlphaServer 800 ならびに 1000 のサポートを追加。
  • alpha GENERIC カーネルの lc(4) デバイスを有効化。
  • alpha の isapnp(4) のパニックを修正。
  • xf86config にマウスホイールの設定オプションを付与。
  • X サーバの特権放棄の試行中の i386_iopl(2) の失敗を潔くハンドリング。
  • i386 の fbtab(5) に wscons(4) ファイルを追加。
  • kqueue(2) サポートを syslog(3) に追加。
  • strtou?q() が strtou?ll() に進化。(ラッパーや、その他の) strtou?q() に見せかけることが有効な場合に弱いエイリアスとして使用。
  • inetd(8) から root として rpc.rstatd(8) および rpc.rusersd(8) を実行させることとするが、起動時に nobody 権限に移行。
  • あらゆる場所の、非常に多くの境界チェックを実施。
  • fxp(4) の認識デバイスを少し追加。
  • getcap(3) の man ページの、紛らわしい cgetclose() エントリを修正。
  • 新しい ahc(4) ドライバを再試行。
  • chpass(1) および passwd(1) のクリーンアップ。
  • セキュリティのための修正: カーネルは、どのユーザの set[ug]id プロセスでも ktrace(2) を実行させてしまいます。
