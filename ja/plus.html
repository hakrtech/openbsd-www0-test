OpenBSD 3.4 と OpenBSD-current との間の変更点

以下は、機種に依存しない主な変更点の部分的なリストで、これらは、 皆さんがかなり頻繁に問い合わされる変更点でもあります。ports に 関する変更も行われており、それらについては、ときどき、それぞれの 機種に関するページに記載されています。

ports コレクションの変更点については、 ここに記述されています。

注: パッチの存在する問題については、朱書きで記されています。

他のリリースの変更点については、こちらをご覧ください。
2.0、 2.1、 2.2、 2.3、 2.4、 2.5、 2.6、 2.7、 2.8、 2.9、 3.0、 3.1、 3.2、 3.3、 3.4。

私たちは OpenBSD-current 上で作業しています。

以下のリストは、12 月 26 日までに加えられた (ほとんど) すべての変更を要約したものです。

  • bgpd(8) のエラーハンドリングおよびロギングをオーバホール。
  • pkg_create(1) が packages の中に 'arch' の注釈を含めるのを許容。
  • pkg_add(1) が、それぞれのファイルシステムに対する使用量を報告するように。
  • bgpd(8) で、localhost のネットワーク 127/8 がネットワークに接続されているものと仮定し、その削除を許容しないように。
  • 暗号化または復号化に固有のコンテクストを作成することにより、ipsec(4) の AES 用に割り当てられた空間の大きさを半分に。
  • bgpd(8) で、ネクストホップの確認を有効化。
  • m68k、m88k、sparc、sparc64 および vax で、brk(2) の符号付き比較のバグを修正。
    [安定版への適用]
  • binutils-2.14 から libiberty を更新。
  • (通信先に対する TCP 接続のない) パッシブモードを bgpd(8) に追加。
  • NetBSD への ahc(4) の大規模な同期とそれに続く FreeBSD への同期。多くのバグが修正されたが、修正すべきいくつかが残存。
  • bgpd.conf(5) のサンプルファイルを加えて、bgpd(8) が構築されるように。まだ、多数の作業が必要。
  • AFS のような、オプションなしのマウントポイントをうまく処理できるようにするための pkg_* ツールの新しい仮想ファイルシステム。
  • macppc で sysctl(3) の hw.cpuspeed を有効化。
  • pthread_[gs]etconcurrency(3) に stub を追加。FreeBSD から。
  • fd が 0、1 または 2 を close(2) しようとした場合の libpthread のディスクリプタのリークを修正。
  • libpthread で、accept(2)、connect(2)、recvfrom(2)、recvmsg(2)、sendmsg(2) および sendto(2) のキャンセル点を POSIX.1-2001 で要求される仕様に。FreeBSD から。
  • アタッチ失敗時に xl(4) のリソースの割り当てを解放。
  • fxp(4) のバスマスタを有効化。
  • sshd(8) の新しいオプション KerberosGetAFSToken を追加。
  • ifconfig(8) がネットワーク仮想インターフェイスの生成を自動的に実行。
  • 単調ではない時間の警告を #ifndef DEBUG で停止。
  • url(4) の MAC アドレスの初期化を適切に (PR#3612)。
  • sudo(8) が -k または -K オプション付きで実行された場合、シャットダウン時や YP などの .logout の中から実行された可能性があり、daemon が既に終了しているかも知れないので、ユーザ名の欠落の警告を stderr にのみ出力し、何もログに出力しないように。
  • tcpdump(8) の l2tp 構文解析の、リモートからの不正な特権奪取の可能なクラッシュの原因を修正 (PR#3610)。
    [安定版への適用]
  • ssh(1) で、3DES 暗号のコンテクストを適正にクリーンアップ。
  • sensorsd(8) のシグナルハンドラが変更する変数が、volatile sig_atomic_t 型であることを確認。
  • SIGKILL がプロセスにわたされることはないので、wsmoused(8) (および bgpd) で、このシグナルのハンドラをセットアップしないように。
  • pf(4) の DIOCSTART の実装で、(最近の発信元の追跡に関する追加の際の) ミスを修正。
  • pthread のテープ終端のハンドリングを修正。FreeBSD PR#59291 の修正に対する修正を含む、FreeBSD PR#56274 の修正から。
  • 利用可能なディスク容量や実際の処理の前に書き込み可能かどうかを調べることができるので、pkg_add(1) および pkg_delete(1) で、仮想ファイルシステムを使用。
  • 新しい bgpd に対する多数の作業
  • カーネルの printf(9) から再帰的フォーマット文字列オプションの '%:' を除去。
  • in*_pcbnotify() がマッチした数を返すよう変更。
  • 数々の冗長な処理を減らすため、TCP 入力の処理の際に、より早い時点でマルチキャストのチェックを行うように。
  • 小数の操作を伴なう semop(2) 用に malloc(9) の代わりにスタックを使用するように。FreeBSD からの試み。
  • usbhid(3) ライブラリのいくつかの際限のない sscanf(3) を修正。
  • i386 で、Pentium 4 の温度制御回路用のドライバを追加。
  • 取得長が送信元の MTU より短かい場合に tcpdump(8) がゴミの pfsync(4) ステータスを表示しないように。
  • dc(1) が SIGINT を合理的な方法で扱うようにし、bc(1) も dc に SIGINT を合理的な方法でわたすように。
  • プロセスの統計情報の更新の際に、#ifdef DIAGNOSTIC を外し、カウンタをゼロクリアする代わりに何もしないことによって、microtime(9) からの単調ではない時間増加をチェック。
  • Intel i386 で拡張版 SpeedStep をサポート。
  • i386 で長時間運転を制御するための sysctl(3) の新しい変数 hw.cpuspeed および hw.setperf を追加。
  • (pf(4) scrub の TCP の再構成の際に) 変調すべき正当なタイムスタンプがある場合、TCP のタイムスタンプの変調のみを行うように。
  • いくつかの HA プロセスに必要な Ethernet のマルチキャストアドレスを含む ARP リプライを許容。
  • tcpdump(8) に、どのようにして IKE NAT-D および NAT-OA のペイロードを認識するのかを指示。
  • isakmpd(8) がメッセージを放棄する場合、ログに交換名を残すように。
  • pfsync(4) のマルチキャストグループを 224.0.0.240 に、IP プロトコル (protocols(5) 中の pfsync) を 240 に変更。
  • 新しい仮想ユーザ/グループ _bgpd を追加。
  • bridge(4) インターフェイスが利用可能になった際にスパニングツリーの操作を開始。
  • 新しい BGP デーモン bgpd を追加。ただし、不完全かつデフォルトではビルドせず。
  • rasops(9) ベースのコンソールで、本当の補色カーソルを表示。NetBSD での同様の変更がベース。
  • 新しい kqueue(2) フィルタ NOTE_EOF および NOTE_TR を追加。
  • ccd(4) および、新しく縮小されたバージョンの ccdconfig(8) を i386 CD の RAM ディスクカーネルに追加。
  • カーネルの zlib を 1.2.1 に更新。
  • 特別な gzip を使用してより縮小したブートフロッピー。伸張のみ可能で直接コンパイルされたもの。
  • ローカルな修正を含み、ユーザランドの zlib を 1.2.1 に更新。新しいメジャーバージョン libz.so.3.0。
  • 入力ファイルの stat(2) が失敗した場合に、cvs(1) が null ラベルを、その diff コマンド経由でわたさないように。
  • bridge(4) 上でフィルタリングを行う際に、送信元アドレスではなくパケットの送信先アドレスを、フィルタの送信先と比較するように。
  • 新しい queue(3) マクロ SLIST_FOREACH_PREVPTR (FreeBSD から) および SLIST_REMOVE_NEXT を追加。
  • 複製インターフェイスがその destroy 関数のエラーからの戻りを許容。
  • AFS ライブラリの abort(3) -> exit(3) への変更および多数の文字列のクリーンアップ。
  • 空のコマンドが与えられた場合の newsyslog(8) のセグメンテーションフォールトを回避 (PR#3578)。
  • monop(6) および mopd(8) に欠落していたいくつかの printf(3) の引数を修正。
  • man(1) 用の新しい環境変数 MANPAGER (PR#3563)。
  • サーバの ClientAliveInterval に類似の、上位レイヤのキープアライブオプション 'ServerAliveInterval' を ssh(1) に追加。
  • (たとえば、bpf(4) のリスナがいない場合やネットワーク同期が行われない場合など) 誰も使用していない場合には高価な pfsync(4) の処理を行わないように。
  • カーネル内のいくつかの長いメッセージ文字列を短かく、または '#ifdef SMALL' の外に。
  • ブートフロッピーディスクの空スペース確保のラウンドを開始。
  • CPU 使用時間の計算時に、ある種のデュアルクロックシステムに見られる microtime(9) の時刻逆行バグをチェックするように。
  • rlim_t の符号付きから符号なしへの変更によるいくつかの不具合を修正。
  • 14 〜 18 グループ (modp{2048,2072,4096,6144,8192} - RFC 3526 を参照) のサポートを isakmpd(8) に追加。
  • ネットワーク越えの pf(4) の状態同期の初期サポート。pfsync(4) を参照。
  • sh(1) および ksh(1) の機能が、環境変数の export が行われていないと期待できる状態でマニュアルページに記述されたとおりに動作するように (PR#2450)。
  • pf(4) で、送信元 IP アドレスに基づく状態付き接続の追跡を可能に。特にロードバランシングの設定で便利。
  • awk(1) の USD の文書を /usr/share/doc/usd/16.awk に追加
  • isakmpd(8) のログ出力で、IKE のポートが常に 500 だと仮定しないように。
  • カーネルおよび libc の RMD160 関数のアラインメントの修正。
  • ppp(4) および sl(4) で ifconfig destroy の初期サポートを追加。
  • cvs(1) で、モジュール名の絶対パス名を受け付けないように。CVS 1.11.10 から。
  • join(1) のクリーンアップと POSIX 関連の修正。FreeBSD から。
  • さらなる POSIX の型の定義 (rlim_t は符号なしに、RLIM_SAVED_{CUR,MAX} を定義、id_t を定義)。
  • 目障りな pf(4) のアサートの失敗メッセージを除去し、その根本原因の NAT およびテーブル統計情報の問題を修正 (PR#3587)。
  • sis(4) の短かいケーブルの問題を修正。Linux およびデータシートから FreeBSD で。
  • poll(2) でも、Solaris のように pollfd_t (= struct pollfd) を追加。
  • POSIX に従って、poll(2) に nfds_t 型を追加。
  • 存在しないパッケージの削除しようとすることをエラーとするよう、pkg_delete(1) は警告を出力するように。
  • IPv6 マルチキャストソケットで、ローカルユーザによるカーネルパニックを防ぐため、get/setsockopt(2) の引数の正当性をより厳密に確認。KAME から。
  • sys/net* の下で、レジスタ定義を大規模に除去。
  • IPv4、IPv6 およびレゾルバのクエリ ID 用の非反復的な ID をより良く生成するように。
  • authpf(8) のログ出力に対するいくつかの改善。
  • 複製可能なデバイスの導入後の netinet および netinet6 インターフェイス参照コードの修正。
  • -N オプション付きの ssh(1) が SIGTERM で終了した場合の終了コードをクリア。
  • em(4) を FreeBSD に同期させ、少数のさらなるモデルのサポートを有効化。
  • identd(8) で、いくいかの *printf(char *) を修正。
  • マニュアルページを読まない人たち用に ssh(1) のオプション KeepAlive を TCPKeepAlive にリネーム。
  • 依存関係が存在する場合の pkg_add(1) および pkg_delete(1) で -n のハンドリングを改善。
  • pkg_add(1) の依存関係の解決の出力を適用する基本パッケージを明示。
  • -n オプションに影響する pkg_add(1) のエラーハンドリングロジックを修正。
  • ssh(1) および sshd(8) で、-o オプションの構文解析時に argv を変更しないように (sshd に対する HUP がブレークしないように)。
  • -N オプション (ログインシェルなし) が適用された場合の ssh(1) で、オプション ClientKeepAlive が動作するように。
  • ssh-keygen(1) の -T オプションで、知られた生成器なしにプライムを受け付けないように。
  • lex(1) にいくつかの PostScript のドキュメントを追加。
  • tn3270(1) および kernfs に欠落していた、いくつかの printf(3) の引数を修正。
  • compress(1) で、いくつかのクリーンアップ。
  • ひとり以上のユーザが同時に ftp(1) をアクティブモードで使用するのを許容 (PR#3596)。
    [安定版への適用]
  • xfs シンボリックリンクが壊れないように (PR#3552)。
    [安定版への適用]
  • patch(1) で、デフォルトの (ファイルの) マッピングの代わりに mmap(2) からのプライベートなマッピングを取得するように。
  • troff(1) のクラッシュを修正。
  • 最も古いものを落とすつもりで SYN flood の回避を行う場合、最も新しい TCP 接続を落とさないように。
  • pf(4) で、IPv6 のループバックへのリダイレクトが IPv4 の場合と同様の方法で動作し、付加的な route-to 行を要求しないように。
    [安定版への適用]
  • NFS クライアントのコードの低過ぎる spl(9) を修正。
  • すべての複製能力を持つデバイスをリストアップする、新しい ifconfig(8) の -C オプション (および ioctl(2) SIOCIFGCLONERS をサポート)。
  • ローカルホストへの pf(4) のリダイレクトが、リモート接続からローカルホストへの接続を通知するための portmap(8) のようなプログラムの能力を無意味なものにしてしまわないようにするために使用される、新しい mbuf_tags(9) のタイプ PF_TRANSLATE_LOCALHOST を追加。
  • ppp(4) および sl(4) に ifconfig create のサポートを追加。ifconfig destroy は未サポート。
  • regex(3) の非 ASCII 文字のハンドリングを修正 (PR#3594)。FreeBSD からの修正。
  • ある種のパターンの複数のドットのカウントに関する grep(1) の修正 (PR#3597)。
  • ifconfig の destroy が tun(4) で動作するように。
  • wicontrol(8) のクラッシュの原因となるエンディアン関連のバグを修正。
  • mfs(8) ファイルシステム用に madvise(2) のフラグ MADV_RANDOM を設定。
  • isakmpd(8) の情報交換を行う際の DELETE メッセージに存在する SPI の正当性を確認。
  • 最初のブート時に sshd(8) を有効化すべきかどうかをインストーラが質問するように。デフォルトは有効化。
  • em(4) でマルチキャストの受信を有効化。
  • mg(1) のコマンドライン上で、ひとつ以上のファイルがオープンされている場合に、スクリーンを分割するように。
  • mg(1) の META キーサポートを修復。
  • semop(2) の符合比較のバグを修正。
  • bridge(4)、carp(4)、faith(4)、gif(4)、gre(4)、lo(4)、tun(4) および vlan(4) に複製のサポートを追加。
  • ifconfig(8) のコマンド create および destroy によって、たとえば 'ifconfig vlan100 create' のようにしてアクセスされる、インターフェイスの複製 (クローニング) をサポート。
  • ddb(4) に dmesg コマンドを追加。
  • 多過ぎるネットワークインターフェイス (>65535) の割り当てを許容しないように。
  • Perl 5.8.2 をマージ。
  • gcc3 に hppa 用ターゲットを追加。
  • 新しい sysctl(3) でトグルされる net.inet.esp.udpencap によってトランスポートモード (draft-ietf-ipsec-udp-encaps-XX.txt を参照) が有効化された ESP に、UDP カプセル化のサポートを追加。
  • sshd(8)、ssh-agent(1) およびフォワーディングソケット用に一貫性のある高いリスンバックログを使用。
  • dc(1) のひとつ違い (off-by-one) エラーを修正。
  • ssh(1) の進行表示の見かけに対する改善。
  • 10,000 行以上のプログラムを bc(1) でコンパイル可能に。
  • bc(1) に、もうひとつの移植性のない拡張である、長い変数名のサポートを追加。
  • kqueue(2) のサポートを tun(4) に追加。
  • compat_linux(8) の alarm() 関数用の時間のベースとして、基準時 (エポック) の代わりに現在の時刻を使用。
  • uvm_swap_markbad() の NULL 被参照を回避。
  • カーネルのディスクリプタ管理コードで被参照の前に符合をチェック。
  • 文字列の切り詰め時の csh(1) の変数の置換を修正 (PR#3591)。
  • aliases(5) で、ほとんどの (_syslogd のような) 仮想的なユーザへのメールを root ではなく /dev/null に振り向けるように。
  • gcc3 に amd64 用ターゲットを追加。
  • dc(1) (の -x オプション) で、拡張レジスタのサポートが追加され、bc(1) に間もなく登場する長い変数名のサポートのための準備が完了。
  • mopd(8) のクリーンアップ。
  • gcc3 に OpenBSD 固有のオプションを追加。
  • 容量の関係で Ada 用フロントエンドなしで GCC 3.3.2 を導入 (ただし、有効化はまだ)。
  • pf(4) 用の新しい ':' (包括的な範囲) 演算子を追加。pf.conf(5) で '><' (排他的な範囲) 演算子が動作する場所ならどこにでも使用可能。
  • 有効なグループ名をテストする security(8) の正規表現を修正。
  • pf(4) の状態収集に対するさらなる修正。
  • OpenBSD がサーバで Linux がクライアントである場合の NFS-over-TCP の速度を修正 (PR#3561)。
  • systrace(1) で、'$' で終るユーザ名の受け付けを許容。
  • eeprom(8)、elf2aout および elf2ecoff の printf(3) の欠落していた引数を修正。
  • arc4random(3) の鍵ストリームの最初の 256 バイトを破棄。"Weaknesses in the Key Scheduling Algorithm of RC4" という論文がこれを推奨。
  • 初期化されていない配列の場所を読み込む際の dc(1)のコアダンプを修正。
  • いくつかの gcc3 の互換性に関するクリーンアップ。
  • compat_linux(8) 下の SIOCGIFHWADDR を修正。
  • gcc の準備のため、さらなる libiberty のコンポーネントを構築。
  • GCC 3.3.2 からのバージョンと libiberty とを同期。
  • libedit を 2003 年 11 月 8 日時点の NetBSD のものと同期。
  • libiberty を src/gnu/lib/libiberty に移動し、egcs ディレクトリから削除。新しいライブラリは、いくつかのローカルな変更を含み、「binutils-2.10 と 2.11 の間のどこか」に同期。
  • pcap_setfilter(3) のメモリの二重解放を回避。
  • カーネルの MD5 のコードが bcopy() の自身の実装の代わりに、アーキテクチャごとに最適化された bcopy() を使用 (PR#3549)。
  • ssh(1) の -k オプションに新しい意味を追加し、GSSAPIDelegateCredentials=no と等価に。
  • ssh-keyscan(1) で、任意の数を返す代わりに fd の最大値を取得するために sysconf(3) を使用。
