OpenBSD 3.1 と OpenBSD-current との間の変更点

以下は、機種に依存しない主な変更点の部分的なリストで、これらは、 皆さんがかなり頻繁に問い合わされる変更点でもあります。ports に 関する変更も行われており、それらについては、ときどき、それぞれの 機種に関するページに記載されています。

ports コレクションの変更点については、 ここに記述されています。

注: パッチの存在する問題については、朱書きで記されています。

私たちは OpenBSD-current 上で作業しています。

以下のリストは、5 月 24 日までに加えられた (ほとんど) すべての変更を要約したものです。

  • セキュリティのための修正: httpd(8) デーモンで、chunk エンコード形式の要求を処理中に、リモートからのクラッシュに つながる危険性のあるバッファオーバーフローが発生する危険性があります。
    ソースコードパッチはこちら。
    [安定版への適用]
  • ...
  • pfctl(8) で IPv4/IPv6 混合アドレスリストをサポート。
  • ssh-keysign(8) を追加。
  • 既に使われなくなった dm(8) を除去。
  • splbio() spl(9) が実行したものではない biodone() コールを探し出し、これを修正。
  • cd9660 ファイルシステムの先読性能を改善。
  • さまざまな macppc モデル上でのソフトウェアによる明るさおよびバックライトの制御のサポート。
  • ディスプレイの明るさおよびバックライトの制御をサポートするマシン上では、 wsconsctl(8) にこれらの制御を行わせる。
  • hppa アーキテクチャ用の新しい libc の IEEE 浮動小数点コードおよび libm ルーチン。
  • i386 アーキテクチャ用の splassert (spl(9))。
  • 安全ではない文字列関数の廃止に向けたその他のステップ。
  • sparc64 アーキテクチャ用の splassert (spl(9))。
  • sparc64 の Creator および Creator3D カード用 creator(4) ドライバの追加。
  • IPv6 のサポートを含む、lpr(1) の巨大な変更、新機能およびバグ修正。
  • さらにその他の hppa アーキテクチャのメモリ管理および低レベルコードの修正。
  • macppc の pmap の単純な最適化。
  • 我々はインストールスクリプトのお掃除を行い、機能を追加しながら、サイズを減らすと言いましたっけ ?
  • ddb(4) によるカーネルメッセージバッファのスタックトレースを許可。
  • isp(4) の修正。
  • セキュリティのための修正: sshd(8) が BSD 認証を使用する際の不正な ACL チェックの修正。
    ソースコードパッチはこちら。
    [安定版への適用]
  • mg(1) のメモリリークを修正。
  • 新しい systrace 機能。
  • Cyrix CPU のサポートを改善。
  • ECN をサポート。
  • rdate(8) で SNTP をサポート。
  • VAX アーキテクチャにおける無限の SIGFPE ループの状況を修正。
  • setuid ビットを必要としていないとか、あるいは root 権限を要求する機能が 本来 root のみが呼び出すべきものであるとか、特定のグループへの setgid ビット に置き換えられるとか、そのようなバイナリから不必要な setuid ビットを除去。
  • skey(1) の管理をフラットなファイルからユーザごとのディレクトリとして、関連するツールから setuid ビットを落とした。
  • 数多くの ppp(8) の嬉しい機能。
  • sparc の新しい splassert (spl(9) を参照) デバッグ機能。
  • macppc カーネルにおいて Altivec 命令を有効化。
  • nofn(4) ドライバを利用した、その他の Hifn カード (7814, 7851, 7854) のサポート。
  • OpenSSL 0.9.7。
  • at(1) および関連バイナリを完全に書き直して POSIX 準拠にした。
  • ubsec(4) を利用した、その他のハードウェア暗号カードの機能の利用。
  • hppa アーキテクチャ用メモリ管理のその他の修正。
  • binutils 2.11.2。
  • pf(4) に gid ごとのフィルタリング機能を追加。
  • at(1) も同様に crontab グループへの setgid バイナリに変更。
  • pfctl(8) においていくつかのアドレスのホスト名解決を扱うようにした。
  • 数々のユーザランドプログラムのコンパイル時のウォーニングメッセージを修正。
  • 新しいユーザ crontab を追加し、cron(8) を setuid root から crontab グループへの setgid バイナリに変更。
  • pf(4) に uid ごとのフィルタリング機能を追加。
  • その他の usb(4) サポートの更新。
  • ubsec(4) でより多くの暗号演算ができるようにするためのいくつかのハック、および hifn(4) と lofn(4) とが、これと同時に働くようにするためのハック。
  • インストールスクリプトの平均的な拡張可能なお掃除と、機能追加を行い、かつサイズを減らした。
  • macppc の adb(4) のフランス語キーボード配置の修正。
  • ELF プラットフォームの gprof(1) をネイティブなもの に変更。
  • より良いライセンスの hppa アーキテクチャ用 spmath ルーチンを入手。
  • Realtek RTL8150L ベースの USB カード用 url(4) ドライバを追加。
  • mvme88k アーキテクチャ用 pmap のバグ修正。
  • さまざまな usb(4) ドライバの更新。
  • rlogin(1)、 rlogind(8) および rexecd(8) を除去。
  • newfs(8) のいくつかの誤った計算結果を修正。
  • pchb(4) の影の PCI バス検出が機能するようにした。
  • fms(4) ラジオカード用のチューナドライバを追加。
  • pf(4) のルールが特定の状態を作り出したことをユーザランドプログラムが知ることができるようにした。
  • 3.0 の wsmoused(8) バイナリがカーネルパニックを起こさないようにした。
  • sshd(8) の特権分離をデフォルトで有効化。
  • find(1) の -anewer と -cnewer オプションの挙動を修正。
  • ツリー全体で ptrdiff_t と size_t タイプを int の代わりに使用。
  • wdc(4) で LBA48 アドレッシングをサポート。
  • TURBOchannel alpha ハードウェアのサポートが復活。
  • UKC (boot_config(8)) のデバイスのクローニングのわずかに不正な挙動を修正。
  • セキュリティのための修正: setuid プログラムのために fd の 0 〜 2 に /dev/null を複写する際に資源を割り当てられなければ、exec(3) を失敗させる原因となる。
    ソースコードパッチはこちら。
    [安定版への適用]
  • 拡張属性コードの更新。
  • pckbc(4) の PS/2 マウスポートの検出の改善。
  • より良い hifn(4) の初期化およびメモリ利用。
  • インストールスクリプトの拡張可能なお掃除と、機能追加を行い、かつサイズを減らした。これはデジャブを見ているわけではない。
  • sparc のイーサネットの割り込みレベルの修正、および sparc の割り込みフレームワークを書き直した。
  • より良い、Xwsfb でサポートされる色数の検出。
  • vmstat(8) を 64 ビットレベルで修正。
  • bge(4) の DMA プロセシングを改善。
  • 信頼性のための修正: nfs(8) ファイルシステムの強制 readdirplus 要求カウントの修正。
    [安定版への適用]
  • macppc のコンソールを rcons エンジンから rasops エンジンに変更。
  • もう一度、インストールスクリプトの拡張可能なお掃除と、機能追加を行い、かつサイズを減らした。
  • alpha 用 IEEE754 浮動小数点の補完コードを追加。
  • gx(4) の DMA プロセシングを改善。
  • sparc64 用 XFree86 GLX 拡張機能を構築。
  • キャラクタデバイス用エントリポイントのための古くなったプロトタイプを探し出して、それらの修正。
  • mvme88k アーキテクチャは、新しい MAKEDEV(8) 生成用フレームワークに変更。
  • m4(1) の -s オプションを実装し、これを POSIX 準拠にした。
  • mvme68k カーネルのすべてのコンパイル時のウォーニングメッセージが出ないようにした。
  • 分類された mac68k コードのお掃除。
  • wi(4) の hostap モードの共有鍵をサポート。
  • alpha アーキテクチャで、Xwsfb が tga(4) カードをサポートするようにした。
  • ami(4) のロックのリークを修正。
  • セキュリティのための修正: sudo(8) を sudo 1.6.6 に更新。
    ソースコードパッチはこちら。
    [安定版への適用]
  • 信頼性のための修正: ftp(1) で、悪意のあるサーバからの PASV におけるバッファオーバーランを回避。
    [安定版への適用]
  • Soundforte ラジオドライバ sfr(4) を追加。
  • 動的インターフェース -> pf(4)でのアドレス変換。
  • アドレス変更をトリガーとするイーサネットインターフェース用のカーネルフックを追加。
  • 拡張属性コードの更新。
  • sparc64 アーキテクチャ上の FreeType ライブラリを有効化。
  • カーネル暗号化フレームワークのキューイングを追加。
  • システムの C++ との親和性の向上。
  • pf(4) の再構成されないフラグメントの明示的なフィルタリングを許可。
  • mac68k の sn(4) ネットワークドライバが不安定な問題の修正、および、その他のハードウェアのサポート。
  • wscons(4) のリトアニア語キーボードマップの改善。
  • セキュリティのための修正: sshd(8) の AFS/Kerberos のトークンの取り扱いにおけるバッファオーバーフローの修正、および完全なチケットの送付。
    ソースコードパッチはこちら。.
    [安定版への適用]
  • mg(1) のメモリリークの修正。
  • 分類された hppa アーキテクチャ用メモリ管理の修正。
  • top(1) のわずかな遅延を許容。
  • alpha アーキテクチャ用インストールメディアのアップグレード機能を再び有効化。
  • インストールスクリプトの拡張可能なお掃除と、機能追加を行い、かつサイズを減らした。
  • cvs(1) は、ログイン時に .cvspass ファイルが存在していなければ、失敗せずにそれを作成するようにした。
  • 他のすべてのプラットフォームと同様、mac68k のディスクラベルも 16 個のパーティションに拡張。
  • cdio(1) への cddb のサポートの追加。
  • dc(4) ドライバでより多くのネットワークカードをサポート。
  • メモリが少ない条件での sparc アーキテクチャ用 pmap の挙動を改善。
  • sendmail 8.13。
  • mvme88k アーキテクチャは、新しい MAKEDEV(8) 生成用フレームワークに変更。
  • a.out プラットフォームの速度向上とメモリ使用の低減のための ld(1) のライブラリのロジックを改善。
  • 新しい mvme68k インストールメディア。
  • mac68k の FPU プローブルーチンの変更。
  • sed(1) の不明瞭なバグを修正。
  • wi(4) ドライバでより多くの無線カードをサポートし、内在していたわずかな問題を修正。
  • pfctl(8) の 64 ビット問題を修正。
  • gx(4) ドライバの存在を無意味なものにしていた wx(4) ドライバを除去。

このリストは、そのほとんどが機種に依存しない変更点について記載したものです。特定の 機種に依存する変更点のリストについては、その機種のページをチェックしてみてください。 もし、そちらにリストされた変更点がない場合は、(1) 変更点がドキュメント化されていな い   か、(2) こちらの方に記載されている   かのいずれかでしょう。
