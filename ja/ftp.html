構成・レイアウト:

各サーバのファイルの階層構造はおおまかに次のとおりです。

  • .../OpenBSD/3.4/
    ここに最も最近のリリースが置かれています。
    • alpha/
    • hp300/
    • hppa/
    • i386/
    • mac68k/
    • macppc/
    • mvme68k/
    • mvme88k/
    • sparc/
    • sparc64/
    • vax/
    • src.tar.gz
       /usr/src に置かれるユーザランドソースコード。 (これにはカーネルソースコードは含まれていません)
    • sys.tar.gz
       /usr/src/sys に置かれるカーネルソースコード。
    • ports.tar.gz
       /usr/ports に置かれる ports のソースコード。
    • XF4.tar.gz
       XFree86 (X11) ソースコード。

  • .../OpenBSD/snapshots/
    主なアーキテクチャ用にだいたい毎月ビルドされる、 安定性/品質を保証しないミニリリースの試みです。 これを、私たちはテストリリースと位置付けています。

  • .../OpenBSD/src/
    個々のファイルにアクセスしたい人のための、CVS でチェックアウトされた 毎日のソースツリーです。通常は、CTM か、 ここに位置するミラーの代わりに AnonCVS の使用をお勧めします。これは、すべての FTP ミラーが、この場所を ミラーしているわけではないからです。

  • .../OpenBSD/doc/
    他の OpenBSD 関連のドキュメントと同様の、PDF およびプレーンテキストでの OpenBSD の FAQ です。

  • .../OpenBSD/tools/
    その他もろもろ。
  • .../OpenBSD/3.3/
    上述した 3.4 リリースと同様のレイアウト。懐古趣味用。
  • .../OpenBSD/3.2/
    上述した 3.4 リリースと同様のレイアウト。懐古趣味用。
  • .../OpenBSD/3.1/
    上述した 3.4 リリースと同様のレイアウト。懐古趣味用。
  • .../OpenBSD/3.0/
    上述した 3.4 リリースと同様のレイアウト。懐古趣味用。
  • .../OpenBSD/2.9/
    上述した 3.4 リリースと同様のレイアウト。懐古趣味用。

新しく FTP/ANONCVS ミラーを立てようと する人でしたら、次の ftp://ftp.openbsd.org:/pub/OpenBSD/ で使用される ディスク使用量の見積りに興味があることでしょう。
現状のポリシーでは、このページにリストされたミラーは、すべてのアーキテクチャについて 少なくとも最新の 2 リリース (現時点では 3.3 ならびに 3.4)、および OpenSSH/、patches/ および tools/ ディレクトリの下をバイナリの形で用意することが求められています。
また、利用可能なディスクスペースに依存しますが、スナップショットやより古いリリース、 カレントのソースツリーなど、ミラーはさらに多くのコンテンツを保持することも可能です。
マスタ ftp サイトにおける、現時点でのディスクスペースの使用量は次のようになっています。 


                      現時点    可能性
pub/OpenBSD/2.9        3.0GB     3.0GB   (OpenBSD 2.9 リリース)
pub/OpenBSD/3.0        4.5GB     4.5GB   (OpenBSD 3.0 リリース)
pub/OpenBSD/3.1        6.9GB     6.9GB   (OpenBSD 3.1 リリース)
pub/OpenBSD/3.2       11.8GB    11.8GB   (OpenBSD 3.2 リリース)
pub/OpenBSD/3.3       14.0GB    14.0GB   (OpenBSD 3.3 リリース)
pub/OpenBSD/3.4       11.0GB    14.0GB   (OpenBSD 3.4 リリース)
pub/OpenBSD/snapshots  8.5GB さらに巨大! (現在のスナップショットで、
                                          通常は最新リリース以上のサイズとなります)
pub/OpenBSD/src        0.6GB     0.7GB   (FTP 可能なソースツリーです。ANONCVS から
                                          夜毎チェックアウトされているだけのもの
                                          ですので、たぶん、これを得るための FTP
                                          ミラーの使用は避けるべきで、最寄りの
                                          ANONCVS サーバから単にアップデートする
                                          のが良いでしょう)
このように、完全なミラーは約 50GB ものディスクスペースを必要としますが、最新の 2 リリースに限定すれば、25GB ほど必要なだけで済みます。 また、マスター ftp サイトは、ミラーが保持する必要のない、次のようなディレクトリも 用意していることに注意してください。 

pub/OpenBSD/distfiles  4.4GB さらに巨大! (ports サブシステムの makefile が使用する
                                          配布ファイルです。他の理由で必要でもない
                                          限りは、たぶん、これらのファイルをミラー
                                          すべきではないでしょう。ports の
                                          makefile はこれらのためにデフォルトで
                                          ftp.openbsd.org と ftp.usa.openbsd.org
                                          とをチェックします)

もし、新規に FTP、HTTP、AFS や RSYNC ミラーサイトを追加する場合には、 www@openbsd.org に連絡してください。
