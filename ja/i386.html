OpenBSD/i386

OpenBSD/i386 は標準的な PC やその互換機上で動作し、幅広い種類のプロセッサ、 I/O バスアーキテクチャおよび周辺機器をサポートしています。

歴史:

OpenBSD/i386 ポートは、このプロジェクトの開始当初から存在しているものです。 長年にわたり、(暗号アクセラレータのドライバのように) より多くのハードウェア サポートが特に OpenBSD のために書かれ、あるいは FreeBSD や NetBSD から 移植されてきました。

現在のステータス:

ほとんどの最近のハードウェア上で、最小限の難易度でインストールし、 実行できるでしょう。 ただし、(古めのプロプライエタリな PC やラップトップ、ある種の特化されたサーバマシンのような) 特製 BIOS を搭載したマシンでは問題が出る場合もあります。

OpenBSD/i386 は、そのバイナリ互換レイヤにより、Linux、FreeBSD、BSD/OS、 (Solaris を含む) SVR4 および iBCS2 バイナリをある程度まで実行可能です。 ほとんどのアプリケーションは完全に動作するでしょう。

XFree86 のサポートについては、利用可能な場合には最近の XFree86 (OpenBSD 3.6 の場合は 4.4.0 unencumbered 版) サーバ、そうでない場合でも XFree86 3.3.6 サーバのいずれかが、ほとんどのグラフィックスカードで利用可能です。

サポート済ハードウェア:

ほとんどのラップトップがサポートされています。いくつかの一般的なラップトップの仕様の 詳細についてのページを用意しています。

サポート済ハードウェアのリストは、OpenBSD-current に関するものとなっています。これは、 最新リリースバージョンでサポートされているものとは、わずかに異なっているかも知れません。

プロセッサに浮動小数点ユニットが組み込まれていない場合には、通常の 浮動小数点コプロセッサ (80387DX、80387SX、80487SX) が必要です。

386 やそれ以降のクローン CPU でも、すべて問題なく動作するはずです。 ただし、唯一 Cyrix 386DLC だけは、(マザーボードの設計不良のため) 不十分な動作しかしないことが知られています。

SMP サポート

OpenBSD 3.6 より、OpenBSD/i386 ではほとんどの SMP (Symmetrical MultiProcessor) システムを サポートしています。 SMP の運用をサポートするため、独立した SMP カーネル (bsd.mp) がインストール用のファイル群に含まれています。

SMP マシン上で実行する場合に、この新しい機能が、どのハードウェアでは実行可能で、 どれでは実行できないのか、開発者は確認したいでしょうから、インストール後に dmesg を送ることがさらに重要になります。

    他のオペレーティングシステムとの共存インストール:

    OpenBSD/i386 は、MBR のパーティションテーブルの仕組みや、オプションなブートセレクタ プログラムを利用して、たとえば MS-DOS、Windows や Linux などのような他の オペレーティングシステムとシステムディスクを共有してインストールすることが可能です。 毎日の運用には問題はありませんが、そのセットアップには注意が必要で、たとえば Windows 95 のような、いくつかのオペレーティングシステムは、既存の MBR の情報に 注意を払わないことが知られています。

    また、いくつかのラップトップコンピュータでは、ハイバネーション領域として 特定のパーティションを使用するので、ここを不注意に上書きすべきではありません。

    このプロセスのより詳しいことについては、 Installation Guide および Disk Setup Manual を参照してください。

    OpenBSD/i386 の入手とインストール:

    サポートされている OpenBSD/i386 の最新のリリースは、 OpenBSD 3.6 です。 これには、 OpenBSD/i386 3.6 のインストール手順書 が用意されています。

    折に触れて こちらの場所 やいくつかのミラーサイトに スナップショットが用意されます。 これにも同様に、 OpenBSD/i386 snapshot の導入手順書 が用意されています。

    以下のように、いくつかのインストールメディアが用意されています。

    • CD ブート
      ほとんどの OpenBSD のドライバを含む、El Torito 2.88MB フロッピーイメージが用意されていますので、 これを CD に焼いて、そこからブートしてください。 また、これには、最小限の USB のサポートが含まれています (ストレージデバイスおよびキーボード)。
      このイメージで利用可能なドライバの最新のリストは、カーネル設定ファイル RAMDISK_CD を参照してください。
    • フロッピー A (floppy36.fs)
      この 1.44MB フロッピーイメージは、ほとんどの一般的なドライバを含んでいます。 これは、ほとんどの典型的な PC をカバーするよう設計されています。原則として、欠落しているドライバは、 あまり一般的ではない SCSI カードやギガビット Ethernet カード、あるいは RAID デバイスなどのように、 その実行には大きなマイクロコードが必要になるものだということがわかるでしょう。 このイメージで利用可能なドライバの最新のリストは、カーネル設定ファイル RAMDISK を参照してください。
    • フロッピー B (floppyB36.fs)
      この 1.44MB フロッピーイメージは、フロッピー A には含まれていない いくつかのドライバを含んでいます。 これは、最近のサーバタイプのシステムを構築しようとする人たちに役立つサポートを含みます。 これには、ほぼすべての SCSI および RAID ドライバ、ギガビット Ethernet カード、などなど が含まれています。 容量の問題のため、いくつかの他のドライバが欠落しています。 特に、ほとんどの PCMCIA デバイスや、MS-DOSファイルシステムや、 本当に古いハードウェアデバイスのためのサポートは含まれていません。
      このイメージで利用可能なドライバの最新のリストは、カーネル設定ファイル RAMDISKB を参照してください。
    • フロッピー C (floppyC36.fs)
      この 1.44MB フロッピーイメージは、多くの CardBus ならびに PCMCIA デバイスドライバを含みます。 これは、ラップトップへのインストール用に設計されています。 ほとんどの場合、フロッピー A でも動作可能なことに気付くでしょうが、 変ったラップトップでは、このイメージを使用する方がより便利かも知れません。
      このイメージで利用可能なドライバの最新のリストは、カーネル設定ファイル RAMDISKC を参照してください。
