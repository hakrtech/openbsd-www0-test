What's New

これは OpenBSD 3.0 に含まれた新機能やシステムの部分的な一覧だ。 もっと完全な一覧を見るには 3.0 までずっとつづくchangelog を見よう。

  • OpenSSH (SSH1 と SSH2 プロトコルの 両方をサポート) は現在 3.0 だ。セキュアなファイル転送用もお勧め。 これ用に大幅に拡張された SFTP サブシステムが含まれていて、 これは SFTP サーバとクライアントの両方が入っている。

  • ドキュメンテーションに大幅な変更。特にman ページと Web FAQ。manページは説明を補うために便利な例題も 追加してある。

  • "ports"が拡充されてテストもしっかりされている。

  • 1000 以上のビルド済みテスト済みパッケージ。

  • 新しい UltraSPARC ハードウェアのためのOpenBSD/sparc64 ポート

  • ipf とは文法的にほぼ互換で NAT 機能も持った新しいパケットフィルタ PF,

  • システムは、外部サプライヤからの以下の主要コンポーネントを含む:

    • XFree86 4.1.0 (それとi386版は3.3.x のサーバも入っているので、あらゆるチップセットがサポートされている)
    • gcc 2.95.3 (+ パッチ)
    • perl 5.6.1 (+ パッチ)
    • Apache 1.3.19, mod_ssl 2.8.2, OpenSSL 0.9.6b (+ パッチ), DSO サポート
    • groff 1.15
    • sendmail 8.12.1
    • lynx 2.8.2, HTTPS サポート付き
    • sudo 1.6.3p7
    • ncurses 5.2
    • 最新の KAME IPv6
    • KTH Kerberos 1.0.8
    • Heimdal 0.3f (+ パッチ)
    • OpenSSH 3.0

  • セキュリティと信頼性面で多くの改善点 (完全な変更履歴 の赤字を参照)。

How to install

以下は、 色々あるインストール手段の中で、 CD-ROMセットを買ったら紙に印刷してついてくる指示書だ。 ftp（あるいはその他）でのインストール方法もほとんど同じだ。 ここでは CDROM 版の指示をそのまま記述してある。 ほらね、CD-ROM を買うとこんなに楽なんですよー。

マシンに OpenBSD 3.0 をインストールするための詳しい方法については、 CD 3 枚の上にあるそれぞれ以下のファイルを参照:

  • CD1:3.0/i386/INSTALL.i386
  • CD1:3.0/alpha/INSTALL.alpha

  • CD1:3.0/macppc/INSTALL.macppc
  • CD2:3.0/mvme68k/INSTALL.mvme68k
  • CD2:3.0/amiga/INSTALL.amiga
  • CD2:3.0/mac68k/INSTALL.mac68k
  • CD2:3.0/hp300/INSTALL.hp300
  • CD1:3.0/vax/INSTALL.vax

  • CD3:3.0/sparc/INSTALL.sparc
  • CD3:3.0/sparc64/INSTALL.sparc64

OpenBSD の経験がある人のための、簡単なインストーラ情報と、新しい "disklabel -E" コマンドの説明だ。OpenBSD のインストールがまったく 皆目見当がつかない人は、上に挙げた関連の INSTALL.* ファイルを見てね!

OpenBSD/i386:

    BIOS のオプションをいじくって、CD から起動できるかやってみよう。 CD1 を使うといい。ダメなら起動フロッピーを作ろう。 CD1:3.0/i386/floppy30.fs を フロッピーに書き出して、そこから起動すればいい。

    より多くの scsi 機器のサポートを得たいなら、 CD1:3.0/i386/floppyB30.fs を代わりに、 また、ラップトップのためのより良いサポートを得たいなら、 CD1:3.0/i386/floppyC30.fs を代わりに使う。

    OpenBSD をほかの OS と同居させるつもりなら、 絶対に INSTALL.i386 を読もう。

    MS-DOS で起動フロッピーをつくるには"rawrite"を使う。CD:/3.0/tools/rawrite.exe にある。 Unix OS で起動フロッピーを作るには dd(1) を使う。以下に dd(1) の使用例を書いておく。 device は"floppy" とか "rfd0c" とか "rfd0a".だ。 

      # dd if=<file> of=/dev/<device> bs=32k

    きちんとフォーマットされた不良ブロックなしのフロッピーを使わないと、 ドツボよーん。起動フロッピーの作り方や OpenBSD/i386 のインストールについて もっと詳しく知りたければ、FAQ4.1 を読もう。

OpenBSD/alpha:

    SRM のプロンプトで、boot -fi 3.0/alpha/bsd.rd dka6 と入力する。 この dka6 とは、CDROM ドライブの短縮名のこと (これは show dev で確認することもできる)。

    もし、CDROM から起動できなければ、CD1:3.0/alpha/floppy30.fs か CD1:3.0/alpha/floppyB30.fs (これはマシンに依存する) を フロッピーに書き込んで、boot dva0 と入力する。 より詳しいことは INSTALL.alpha を参照して欲しい。

    きちんとフォーマットされた不良ブロックなしのフロッピーを使わないと、 ドツボよーん。

OpenBSD/macppc:

    CD2 を CDROM ドライブに入れて、C キーを押しながらマシンの電源を入れ、 OpenBSD/macppc boot が表示されるまで C キーを押し続ける。

    代わりに、オープンファームウェアのプロンプトのところで、 boot cd:,ofwboot /3.0/macppc/bsd.rd と入力してもよい。

    3.0 リリースの CD には製造ミスがあったので、 boot cd:,OFWBOOT /3.0/macppc/bsd.rd のように変更したコマンドを使うべきだろう。 このコマンドは, iMac (Rev A 〜 Rev C)など, 古い目のNew World ROMベースのシステムでは 動かないかも知れず、これらのマシンでは ofwboot と /3.0/macppc/bsd.rd とを、ハードディスクの最初の HFS パーティションにコピーする必要があろう。 そして、boot hd:,OFWBOOT /bsd.rd としてブートして欲しい。

OpenBSD/hp300:

    CD2 を CDROM ドライブに入れて、電源投入後の自己診断の間、スペースバーを 押し続ける。そして SYS_CDBOOT 相当の起動方法を選択する。

    代わりに、INSTALL.hp300 の指示に従って、ネットワークから起動することもできる。

OpenBSD/amiga:

    NSATALL.amiga の準備セクションを読んで、BSDパーティションを作ろう。AmigaOS で、 CD2を CD0: にマウントする。それから以下の CLI コマンドを実行する： "CD0:3.0/amiga/utils/loadbsd CD0:3.0/amiga/bsd.rd".

OpenBSD/mac68k:

    ふつうにMacOSを起動して、しかるべき A/UX の設定でディスクにパーティションを 切ってほしい。それから Mac 側のユーティリティを CD2:3.0/mac68k/utils から ハードディスクに展開する。Mkfsを実行して、いま作ったA/UX パーティションに ファイルシステムを作ろう。それから BSD/Mac68k インストーラを使って、 CD2:3.0/mac68k/ の全セットをパーティションに移そう。最後に、 BSD/Mac68k Booter に、カーネルのありかを教えて設定したら、 システムを起動しよう。

OpenBSD/mvme68k:

    起動可能なインストールテープを作るか、ネットワークから起動できる。
    ネットワークからの起動では、NIOT と NBO デバッガコマンドを サポートする MVME68K BUG バージョンが必要となる。より詳しいことは、 INSTALL.mvme68k に従って欲しい。

OpenBSD/vax:

    INSTALL.vax に書いてあるように、mopbooting でネットワークから起動しよう。

OpenBSD/sparc:

    OpenBSD/sparc の 3.0 リリースは CD3 に入っている。この CD から起動するには、 ROMのバージョンに応じて、以下の 2 つのコマンドのうちどちらかが使えるだろう。 

      > boot cdrom 3.0/sparc/bsd.rd
or
> b sd(0,6,0)3.0/sparc/bsd.rd

    CD ドライブの無い sparc だったら、代わりに "CD3:3.0/sparc/floppy30.fs" を フロッピーに書き込んでフロッピーで起動できる。詳しいことは FAQ4.1 を読もう。フロッピーから起動するにはROM のバージョン に応じて、以下の 2 つのコマンドのうちどちらかが使えるだろう。 

      > boot floppy
or
> boot fd()

    きちんとフォーマットされた不良ブロックなしのフロッピーを使わないと、 ドツボよーん。

    もしもフロッピーも CD も無ければ、INSTALL.sparc ファイルに 書いてあるように、起動可能なテープかネットワーク越しにインストール することもできる。

OpenBSD/sparc64:

    CD3 を CDROM ドライブに入れて、後は boot cdrom と打つだけ。

    もし、これで動かなかったら、あるいは CDROM ドライブが無かったら、代わりに CD3:3.0/sparc64/floppy30.fs をフロッピーに書き込んで、 boot floppy として、フロッピーで起動できる。
    きちんとフォーマットされた不良ブロックなしのフロッピーを使わないと、 ドツボよーん。

    また、CD3:3.0/sparc64/miniroot30.fs をディスクのスワップパーティションに 書き込んで、boot disk:b として、そこから起動することもできる。

    もし、どの方法でもダメなら、INSTALL.sparc64 ファイルに 書いてあるように、ネットワークから起動することもできる。

ソースコードについて一言:

    src.tar.gz は /usr/src から始まるソースコードが入っている。このファイルには、 カーネルソース以外の必要なものがすべて入っている。 抽出には: 

      # mkdir -p /usr/src
# cd /usr/src
# tar xvfz /tmp/src.tar.gz

    srcsys.tar.gz は /usr/src/sysからはじまるソースアーカイブだ。このファイルには、 カーネル再構築に必要なカーネルソースがすべて入っている。 抽出には: 

      # mkdir -p /usr/src/sys
# cd /usr/src
# tar xvfz /tmp/srcsys.tar.gz

    これらのツリーはどっちも普通の CVS チェックアウトだ。このツリーを使えば、 http://www.OpenBSD.org/anoncvs.htmlで先陣を 切ることもできるぞ。 このファイルを使えば、完全な OpenBSD ソースツリーを全部まっさらに チェックアウトするよりもずっと高速に、最初の CVS アップデートができる。

Ports ツリー

ports ツリーアーカイブも提供されている。これを抽出するには: 

    # cd /usr
# tar xvfz /tmp/ports.tar.gz
# cd ports

ports/ サブディレクトリは、OpenBSDの ports ツリーのチェックアウトだ。 いま、ports についてまったく知らない人は http://www.OpenBSD.org/ports.html を読んでほしい。このテキストは、 ports の使い方のマニュアルではないのだ。むしろ、OpenBSD ports システムの クイックスタートガイドだと思ってほしい。

確かに、OpenBSD ports システムは完璧じゃない。いつか完璧になるかどう かも疑問だ。しかし、すごい勢いで成長し、どんどん安定性を増しつつある。 このリリースで提供しているほとんどすべての ports はかなりのアーキテクチャ で問題なくビルド出来ると思う。（たとえば、i386 では 1200 以上の package がある。）

ports/ ディレクトリは CVS (なんだか知らなければ cvs(1) の manpageを 見てちょ）の ports チェックアウトになっている。ソースツリーの場合と 同じように、anoncvs でも提供してある。だから ports/ ディレクトリをカレントにしておくには、ports/ ツリーを読み書き可能な メディアにこしらえて、以下のコマンドでツリーを 更新しておこう: 

    # cd [portsdir]/; cvs -d anoncvsserver.openbsd.org:/cvs update -Pd -rOPENBSD_3_0

[もちろん、ローカルのディレクトリとサーバ名は、 自分の ports ツリーの在処と、 最寄りの cvs サーバに置き換えること。]

かなりの ports が packages として ftp で取ってこれる事に注意。3.0 リリース用 に更新された packages は問題が解決すれば配布を開始する予定だ。

もし追加してほしい ports があるとか、手伝いをしたいとか、あるいはもっと 詳しく知りたければ、メーリングリスト ports@openbsd.org に 参加するといいだろう。
