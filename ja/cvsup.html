CVSup とは ?

CVSup は、リモートサーバ上のマスターの cvs(1) リポジトリのソースツリーの配布および更新のためのソフトウェアパッケージです。 OpenBSD のソースはカナダの中央開発用機の上の CVS リポジトリ上でメンテナンス されています。CVSup を使用して、OpenBSD のユーザは、簡単に自身のソースツリー を最新のものにキープすることができます。

CVSup は、更新時プルモデルと呼ばれる方法を使用しています。プルモデルでは、 それぞれのクライアントは、もし、必要なら、サーバに更新があったかどうか を尋ねます。サーバは、クライアントからの更新要求を受動的に待機します。 このように、すべての更新動作はクライアント側から起します。サーバは、決して 求められてもいない更新を行うことはありません。ユーザは、更新結果を取得するために、CVSup クライアントを手動で 起動、または、自動的に定期的にそれを実行する cron(8) ジョブをセットアップしなければなりません。

CVSup という用語は、このとおりに大文字・小文字で記述し、全てのソフトウェア パッケージを指す。その主なコンポーネントは、各ユーザのマシン上で実行される cvsup クライアントであり、そして、各 OpenBSD CVSup ミラーサイトで 実行されているサーバ cvsupd です。

CVSup その前身の sup(1) より、速くそしてより柔軟であることを意図したものです。

OpenBSD プロジェクトは、現状では、次の 5 個の主ソースリポジトリを持っています。

  • src - OpenBSD オペレーティングシステムの全ソースコードを収容。
  • ports - OpenBSD Ports を収容。
  • www - すべての OpenBSD の Web ページ (このページを含む) を収容。
  • X11 - OpenBSD 対応の XFree86-3 ソフトウェアプロジェクトを収容。 これは歴史的な事情によりここにあるだけで、多くのユーザはこのツリーを 使用する必要はありません。もはや使用しません。
  • XF4 - OpenBSD 対応の、アクティブな X のソースツリー。

CVS とは ?

CVS とは、OpenBSD のソースツリーを管理するのに使用されているソースコードコントロールシステムです。 より詳しくは、こちらを参照してください。また、 CVS と CVSup とは、完全に異なるプログラムであることに注意してください。 CVS は、リモートのファイルにアクセスするのに使用できますが、ネット上に 大規模に分散するソースツリーに最適化されているわけではなく、CVSup は、 簡単に、このタスクにはより効率的な、巨大な規模にすることもできるのです。

CVSup を使い始めるために

CVSup は Modula-3 で記述されているので、これをソースからビルドするのは、 面倒なことであるということを理解していただけるかと思います。lang/ezm3 ポートは、CVSup をビルドするためだけに Modula-3 が必要な人たちのための Modula-3 コンパイラおよびランタイムの、より小さなディストリビューションです。それでも、 CVSup を実行することだけに注力したいのなら、net/cvsup port から ビルドする代わりに、ftp ミラーサイトから cvsup の packages をダウンロードしてくることをお薦めします。現時点では Ezm3、および その結果として、CVSup の OpenBSD の ports は i386 プラットフォーム用のみ、 利用可能となっています。

CVSup についてのこれ以上の情報が必要なら、作者の プロジェクトホームページを参照してください。

CVSup で OpenBSD リポジトリをミラーするためには、 次のような設定ファイルを使用する必要があります。 

        # Defaults that apply to all the collections
        *default release=cvs
        *default delete use-rel-suffix
        *default umask=002
        *default host=cvsup.uk.openbsd.org
        *default base=/cvs
        *default prefix=/cvs

        # If your network link is a T1 or faster, comment out the following line.
        *default compress

        OpenBSD-all
        #OpenBSD-src
        #OpenBSD-www
        #OpenBSD-ports
        #OpenBSD-x11
        #OpenBSD-xf4

これは、ローカルリポジトリに対するグループ書き込みを許す umask(2) を使用して、 cvsup.uk.openbsd.org からの、すべての OpenBSD ディストリビューションをリフレッシュすることを、cvsup に指示するものです。

このファイルが、cvs-supfile として保存されていると仮定して、 cvsup の GUI を呼び出すのに、次のコマンドが使用されます。 

        cvsup cvs-supfile

バッチモードの場合には、次のコマンドを使用することになるでしょう。 

        cvsup -g -L 2 cvs-supfile

ソースツリーを更新するために CVS を使う

CVSup のリポジトリから何かをチェックアウトするのは簡単です。 

        # cd /usr
        # cvs -d /cvs checkout src

上記では、最新のソースがチェックアウトされます。しかし、ほとんどの人にとっては、 特別にシステムにパッチを当てたいなど、リリース版のソースが欲しいだけでしょう。 リリース版のソースをチェックアウトするには、コマンドにタグを指定しなければなりません。 たとえば、 

        # cd /usr
        # cvs -d /cvs checkout -rOPENBSD_3_7 src

あるいは、3.6 等の場合なら、OPENBSD_3_6 と指定します。

チェックアウトモードでの CVSup の実行

CVS を使用していて、リポジトリを入手するための代替手段としては、 tag や date キーワードを、*default か 収集用オプションとして supfile に追加して、CVSup をチェックアウトモードで 実行しても良いでしょう。特に、CDROM で出荷されたソースや ports ツリーを効率良く更新するために、 これを使用することができます。

ports ツリーを更新するために、次の supfile が使用できるはずです。 

        # Defaults that apply to all the collections
        *default host=cvsup.uk.openbsd.org
        *default base=/var
        *default prefix=/usr
        *default release=cvs
        *default delete use-rel-suffix compress

        # Ports Collection.
        OpenBSD-ports tag=.

どのような CVS のシンボルタグでも使用できます。ひとつのピリオド "." は HEAD を 意味しますが、これはメインブランチのすべてのファイルの最新のリビジョンになります。 CVSup は間違ったタグを正しいタグとを区別できませんし、間違ったタグで 既存のソースツリーの同期を試みると、すべてのファイルを消去することに なりますので、既存のタグを指定する際には注意してください。

代わりに、リビジョンを日付で指定するための date=[cc.]yy.mm.dd.hh.mm.ss というキーワードを使用すれば良いでしょう。見ていただいたとおりで、 全部で 17 個か 20 個の文字を指定するだけです。2000 年以降なら、世紀 cc を指定します。それ以前なら、下 2 桁 yy を指定する だけでも構いませんし、tag と date キーワードとを同時に 指定しても構いません。

CVSROOT ディレクトリのため、さらに 301MB の付加的な オーバーヘッドがあります。また、全体のリポジトリのサイズは、 現状では、年間 400MB 程度の割合で増加しています。

重要な注意事項: 誰もが知っておくべきこととして、暗号ソフトウェアに関しては 若干の問題があります。

  • OpenBSD のソースはカナダからのものなので、 カナダの個人によって研究されたものとして、また、 カナダの輸出制限リストの記述にあるように、 カナダから世界に向けての暗号ソフトウェアの輸出は合法的なものです。

  • しかしながら、あなたが、米国やカナダ以外の地域に住んでいるのなら、 あなたは米国内にある CVSup サーバから OpenBSD のソースの 暗号セクションを持っていくべきではありません。問題のファイルは次のとおりです。

    • src/kerberosIV/*
    • src/kerberosV/*
    • src/lib/libdes/*
    • src/lib/libc/crypt/crypt.c
    • src/lib/libc/crypt/morecrypt.c
    • src/sys/crypto
    • src/sys/netinet
    • src/usr.sbin/afs/src/rxkad/*
    • XF4/xc-mit/lib/Xdmcp/Wraphelp.c
    • XF4/xc-old/lib/Xdmcp/Wraphelp.c
    • XF4/xc/lib/Xdmcp/Wraphelp.c

    米国の ITAR 軍需品リストによって、暗号ソフトウェアは、 米国からカナダだけに対して輸出されたものかも知れないのです。

OpenBSD プロジェクトでは、さらなる CVSup サーバを募集しています。 ですので、もし、あなたがこれに興味があるなら、設定の詳細に関して、ぜひ www@openbsd.org に 連絡を取ってください。
