CVSup とは ?

CVSup は、リモートサーバ上のマスターの cvs(1) リポジトリのソースツリーの配布および更新のためのソフトウェアパッケージだ。 OpenBSD のソースはカナダの中央開発用機の上の CVS リポジトリ上でメンテナンス されている。CVSup を使用して、OpenBSD のユーザは、簡単に彼ら自身のソースツリー を最新のものにキープすることができる。

CVSup は、更新時プルモデルと呼ばれる方法を使用している。プルモデルでは、 それぞれのクライアントは、もし、必要なら、サーバに更新があったかどうか を尋ねる。サーバは、クライアントからの更新要求を受動的に待機する。 このように、すべての更新動作はクライアント側から起こす。サーバは、決して 求められてもいない更新を行うことはない。ユーザは、更新結果を取得するために、CVSup クライアントを手動で 起動、または、自動的に定期的にそれを実行する cron(8) ジョブをセットアップしなければならない。

CVSup という用語は、このとおりに大文字・小文字で記述し、全てのソフトウェア パッケージを指す。その主なコンポーネントは、各ユーザのマシン上で走る cvsup クライアントであり、そして、各 OpenBSD CVSup ミラーサイトで 走っているサーバ cvsupd である。

CVSup その前身の sup(1) より、速くそしてより柔軟であることを意図したものである。

OpenBSD プロジェクトは、現状では、次の 5 個の主ソースリポジトリを持つ。

  • src - OpenBSD オペレーティングシステムの全ソースコードを収容。
  • ports - OpenBSD Ports を収容。
  • www - すべての OpenBSD の Web ページ (このページを含む) を収容。
  • x11 - OpenBSD 対応の XFree86-3 ソフトウェアプロジェクトを収容。
  • xf4 - OpenBSD 対応の XFree86-4 ソフトウェアプロジェクトを収容。

CVS とは ?

CVS とは、OpenBSD のソースツリーを管理するのに使用されているソースコード コントロールシステムである。より詳しくは、こちらをどうぞ。 CVS と CVSup とは、完全に異なるプログラムであることに注意。CVS は、 リモートのファイルにアクセスするのに使用できるけれど、ネット上に 大規模に分散するソースツリーに最適化されているわけではなく、CVSup は、 簡単に巨大なオーダーにすることもでき、このタスクにはより効率的でもある。

CVSup を使い始めるために

CVSup は Modula-3 で記述されているので、これをソースからビルドするのは、 厄介なことであることを理解してもらえるかと思う。lang/ezm3 ポートは、 CVSup をビルドするためだけに Modula-3 が必要な人たちのための Modula-3 コンパイラおよびランタイムの、より小さなディストリビューションだ。それでも、 CVSup を実行することだけに注力したいのなら、net/cvsup port から ビルドする代わりに、ftp ミラーサイトから cvsup の packages をダウンロードしてくることをお薦めする。現時点では Ezm3、および その結果として、CVSup の OpenBSD の ports は i386 プラットフォーム用のみ、 利用可能となっている。

CVSup についてのこれ以上の情報が必要なら、作者の プロジェクトホームページを見て欲しい。

CVSup で OpenBSD リポジトリをミラーするためには、 次のコンフィグレーションファイルを使用する必要があろう 

        # Defaults that apply to all the collections
        *default release=cvs
        *default delete use-rel-suffix
        *default umask=002
        *default host=cvsup.uk.openbsd.org
        *default base=/cvs
        *default prefix=/cvs

        # If your network link is a T1 or faster, comment out the following line.
        *default compress

        OpenBSD-all
        #OpenBSD-src
        #OpenBSD-www
        #OpenBSD-ports
        #OpenBSD-x11
        #OpenBSD-xf4

これは、ローカルリポジトリに対するグループ書き込みを許す umask(2) を使用して、 cvsup.uk.openbsd.org からの、すべての OpenBSD ディストリビューションをリフレッシュすることを、cvsup に指示するものだ。

このファイルが、cvs-supfile 

        cvsup cvs-supfile

バッチモードの場合には、次のコマンドを使うことになるだろう。 

        cvsup -g -L 2 cvs-supfile

ソースツリーを更新するために CVS を使う

CVSup のリポジトリから何かをチェックアウトすのは、簡単なものだ。 たとえば、Korn/Bourne シェルなら、 

        # cd /usr
        # CVSROOT=/cvs cvs checkout src

csh やそれから派生したものなら、次のようにするだけだ。 

        # cd /usr
        # setenv CVSROOT /cvs
        # cvs checkout src

上記では、最新のソースがチェックアウトされる。だが、たいていの人にとっては、 特別にシステムにパッチを当てたいとかで、リリース版のソースが欲しいだけだろう。 リリース版のソースをチェックアウトするには、コマンドにタグを指定しなければならない。 たとえば、 

        # cd /usr
        # cvs checkout -rOPENBSD_2_9 src

あるいは、2.8 等の場合なら、OPENBSD_2_8 と指定する。

チェックアウトモードでの CVSup の実行

CVS を使用していて、リポジトリを入手するための代替手段としては、 tag や date キーワードを、*default か 収集用オプションとして supfile に追加して、CVSup をチェックアウトモードで 実行しても良い。特に、CDROMsで出荷された ソースや ports ツリーを効率良く更新するために、これを使うことができる。

ports ツリーを更新するために、次の supfile が使えるだろう。 

        # Defaults that apply to all the collections
        *default host=cvsup.uk.openbsd.org
        *default base=/var
        *default prefix=/usr
        *default release=cvs
        *default delete use-rel-suffix compress

        # Ports Collection.
        OpenBSD-ports tag=.

どのような CVS のシンボルタグでも使用できる。ひとつのピリオド "." は HEAD を 意味するが、これはメインブランチのすべてのファイルの最新のリビジョンだ。 CVSup は間違ったタグを正しいタグとを区別できないし、間違ったタグで 既存のソースツリーの同期を試みると、すべてのファイルを消去することに なるので、既存のタグを指定する際には注意しよう。

代わりに、リビジョンを日付で指定するための date=[cc.]yy.mm.dd.hh.mm.ss というキーワードを使用すれば良いだろう。見ていただいたとおりで、 全部で 17 個か 20 個の文字を指定するだけだ。2000 年以降なら、世紀 cc を指定する。それ以前なら、下 2 桁 yy を指定する だけでも良い。tag と date キーワードとを同時に 指定しても構わない。

Currently (as of October 16, 2001), the CVS repository sizes are 現時点 (2001 年 10 月 16 日現在) では、CVS リポジトリのサイズは次のとおり。

OpenBSD-ports - 67MB
OpenBSD-src - 783MB
OpenBSD-www - 46MB
OpenBSD-x11 - 193MB
OpenBSD-xf4 - 340MB

CVSROOT と sup ディレクトリのため、さらに 34MB の付加的な オーバーヘッドがある。全体のリポジトリのサイズは、現状では、年間 170MB 程度の割合で増加している。

重要な注意事項: 誰もが知っておくべきこととして、暗号ソフトウェアに関しては 若干の問題がある。

  • OpenBSD のソースはカナダからのものだが、 カナダの個人によって研究されたものとして、また、 カナダの輸出制限リストの記述にあるように、 カナダから世界に向けての暗号ソフトウェアの輸出は合法的なものだ。

  • しかしながら、あなたが、米国やカナダ以外の地域に住んでいるのなら、 あなたは米国内にある Anonymous CVS サーバから OpenBSD のソースの 暗号セクションを持っていくべきではない。問題のファイルは次のとおり。

    • src/kerberosIV/*
    • src/kerberosV/*
    • src/lib/libdes/*
    • src/lib/libc/crypt/crypt.c
    • src/lib/libc/crypt/morecrypt.c
    • src/sys/crypto
    • src/sys/netinet
    • src/usr.sbin/afs/src/rxkad/*
    • XF4/xc-mit/lib/Xdmcp/Wraphelp.c
    • XF4/xc-old/lib/Xdmcp/Wraphelp.c
    • XF4/xc/lib/Xdmcp/Wraphelp.c

    米国の ITAR 軍需品リストによって、暗号ソフトウェアは、 米国からカナダだけに対して輸出されたものかも知れないのだ。

OpenBSD プロジェクトでは、さらなる CVSup サーバを募集している。なので、 もし、あなたがこれに興味があるなら、コンフィグレーションの詳細に関して、 ぜひ Brian Somers に連絡を取っていただきたい。
