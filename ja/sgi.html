OpenBSD/sgi

OpenBSD/sgi は現在、64 ビット MIPS ベースの SGI O2 ワークステーションで動作します。 他の 64 ビットの SGI システムも、今後対応する予定です。

OpenBSD/sgi は、完全な 64 ビットの移植版であり、 64 ビットのプロセッサ (R4000 以降) を搭載したシステムでしか動作しません。

歴史:

OpenBSD/sgi 移植版は、OpenBSD の初期に作成された OpenBSD/arc 移植版に由来しています。 ハードウェアの入手性と関心が低くなったため、arc の対応はすぐに 打ち切られてしまいました。それ以来、OpenBSD の Mips 対応は OpenBSD プロジェクトの外部で進行していたのですが、 最近は SGI のハードウェアが安価で入手しやすくなったことから、 SGI への移植の関心が高くなってきたというわけです。 最後の懸案だった 64 ビットのツールチェインがようやく利用できるようになり、 64 ビットの SGI 対応 OpenBSD は OpenBSD のソースツリーに統合されました。

現在の状況:

OpenBSD/sgi は現在、対応しているシステムで 64 ビットモードで動作し、 マルチユーザで起動します。移植版の開発の進行に応じて、定期的にスナップショット が作成されています。

プロジェクト (順不同):

  • R10000 CPU のキャッシュ整合動作の対策
  • O2 内蔵ビデオの wscons 対応
  • O2 内蔵オーディオの対応
  • X11 サーバの対応
  • ユーザ VM 空間の拡大 (現在の制限は 2GB)
  • Origin 200 や Octane といった、その他の SGI ハードウェアへの対応
  • (NUMA ではなく)マルチプロセッサシステムへの対応

対応ハードウェア:

対応モデル

  • R5000, RM5200, RM7000, R10000, R12000 CPU を搭載した O2 (別名 Moosehead)。すべての CPU でレベル 2 キャッシュに対応。RM7000 のレベル 3 キャッシュにも対応。
    • 内蔵の Adaptec AIC-7880 SCSI (ahc)
    • 内蔵の Ethernet (mec)
    • 内蔵のシリアルポート (com)
    • 64 ビット PCI 拡張スロット
    • Intel i8255x ベースの (i82556 以外の) PCI Ethernet (fxp)

対応周辺機器

ほとんどのモデルが PCI を使った拡張に対応しており、多くの PCI デバイスが動作するはずです。ただしこのページには、 きちんと試験されたデバイスのみ、対応デバイスとして書いてあります。 つまり、ある特定のデバイスに対応してほしい、という場合には、 機能の検証ができる開発者が、そのハードウェアを入手していなければなりません。

さらに詳しい情報が必要な方は Per Fogelström に連絡してください。

OpenBSD/sgi の入手とインストール:

OpenBSD/sgi がサポートされている最新版は OpenBSD 3.7 です。 OpenBSD/sgi 3.7 のインストール手順はこちらを参照してください。

時々、この場所と いくつかのミラーサイトにスナップショットが置かれます。 OpenBSD/sgi スナップショットのインストール手順も、公開されています。
