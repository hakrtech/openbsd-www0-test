i386 ラップトップ対応状況

OpenBSD の動作が確認されたラップトップ一覧です。

自分のラップトップが載っていないとしても、使えないという意味ではあり ませんし、おそらく似たような型のがあるはずです。ほとんどのラップトップは きちんと動作します。ただ、まだ細かい問題が残っている機種もあります。

問題のある機種というのは、

  • APM サポートで問題を起こすものがあります。apm -S がサスペンドに失敗するラップトップがあるので注意してください。
  • サウンドがサポートされていないものもあります。
  • PCMCIA および CardBus サポートに微妙なバグのある機種が少々あります。
  • Firewire も ACPI も今のところ OpenBSD ではサポートしていません。
  • XFree86/X.org がきちんとサポートしてくれないディスプレイがあります。 もう一方の X に切り替えてみるとうまくいく場合もあります。
  • ほかにもうまく動かない組み込みデバイスがあります。
  • PCMCIA カードの取り出しで割り込みに問題のあるものがあります。
  • 最近のラップトップでオンボードのモデムがあるものは、ほとんどすべて winmodem を使っていますが、これはサポートされておらず、また 将来サポートされる見込みもありません。モデムが必要なら PCCard タイプを使ってください。以下に挙げられているマシンのモデムは、 特に注記のない場合には「使えない」と考えて構いません。
  • Phoenix 系 BIOS を使う IBM Thinkpad (と、おそらく他の) ノートは いわゆる「ハイバネーション・モード」という、メモリをディスクに 書き込む機能を提供します。これは OpenBSD で使うことができます。 ハイバネーション用パーティションを ports の /usr/ports/sysutils/tphdisk ユーティリティで 作成してください。
  • mini-PCI スロット搭載の IBM Thinkpad は PCI ID をチェックして、内蔵 の「許認可」デバイスリストと照合します。そのため、サードパーティ 製のカードでは動作しません。これを変更するには、サードパーティ製 カードを差す「前に」ports の /usr/ports/misc/tpwireless を使います。これは BIOS の「魔法のフラグ」を立てて、 チェックを無効にするユーティリティです。

dmesg ¦ xorg.conf

IBM Thinkpad X31

  • OpenBSD 3.7-current (2005 年 5 月) で使えます。
  • ほぼすべての機器がサポートされていますが、FireWire だけはダメです。
  • APM は良くサポートされています。サスペンドモード (apm -s/zzz) は問題ありません。ハイバネーション (サスペンド・トゥ・ディスク) も動作します。ただしハードディスク先頭に小さな MS-DOS パーティション を置き、ハイバネーションファイルを入れておかなくてはなりません。 これは ports の tphdisk (/usr/ports/sysutils/tphdisk) で作成できます。
  • カーソルキー付近にある二つのキーは xmodmap で設定することが できます。
  • 例: 
    		xmodmap -e "keycode 233 = Page_Down"
		xmodmap -e "keycode 234 = Page_Up"
  • うちの X31 (Model 2672-C8G) に内蔵されている Intel PRO/Wireless 2100 は ipw(4) ドライバで使えます。しかしまずフリーでないファームウェアを ダウンロードしなくてはなりません。詳細については ipw(4) を読んでください。
  • このカードを ral(4) か ath(4) と交換しようと思っています。これならフリーでないファームウェアは 不要です。交換する前には BIOS を開錠する必要があります。 IBM は特製の IBM mini-PCI カードの取り付けしか許可して いないのです。BIOS を変更するのは tpwireless プログラムで可能です (/usr/ports/misc/tpwireless, 自己責任で使ってください!)。
  • 内蔵 CF カードリーダ (すごく便利!) も サポートされています。

  • OpenBSD 3.7 が動きます。

    • まず Alexander Yurchenko の ACPI パッチ を当てます。
    • そして /usr/src/sys/conf/files の 21 行目に追加します: 
      	define  acpibus {}
    • /usr/src/sys/sys/malloc.conf には以下を追加します。 
      	#define M_ACPI  122     /* ACPI */
    • Sony SPIC 関連の部分を (コンパイルできないし、どうでもいいので) /usr/src/sys/arch/i386/conf/files.i386 と /usr/src/sys/arch/i386/conf/GENERIC_ACPI からコメントアウトします。

    • これで hw.setperf を sysctl から設定できるようになりました。
  • このほかに自作 ユーティリティ を走らせて、CPU ロードに合わせて自動的に値を制御させています。 元になったのは Laurie Tratts の perfd です。 展開して make && sudo make install するだけです。
