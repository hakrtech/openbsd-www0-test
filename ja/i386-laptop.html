i386 ラップトップの状況

これは、OpenBSD 2.9 で動作することが知られているラップトップのリストです。

もし、あなたのラップトップが以下にリストアップされていなくても、それが動作しないことを意味するわけではなく、 たいてい、以下にリストアップされている同様のモデルを見つけることができるでしょう。 OpenBSD 2.6 以降、わずかな問題は残っているものの、ほとんどすべてのラップトップは問題なく動作します。 より最新の OpenBSD なら、さらに良好に動作してくれるでしょう。

その、残っている問題には、以下のようなものが含まれます。

  • APM のサポートに関して、いくつかのよく知られた問題があります。いくつかのラップトップでは、 apm -S が失敗するので注意が必要ですが、これはそれほど重要なことではありません。
  • サウンド関連のサポートが不十分です。
  • 一部のラップトップでは、その PCMCIA や CardBus のサポートに捕えにくいバグがあります。
  • XFree86 はいくつかのディスプレイを正しくサポートしていませんが、XFree86 4.x で使用できることがあります。
  • いくつかのその他の組み込みデバイスが動作しません。
  • 一部のラップトップでは、PCMCIA カードの取り出し時の割り込みに問題があります。

もし、あなたが、動作する (あるいは、しない) ラップトップで、以下にリストアップされていないものを お持ちでしたら、それをリストアップできるよう、www@openbsd.org 宛にメールをください。また、その際には、可能な限り多くの情報のご提供をお願いします。

APM 関連の問題報告は、私たちの APM 関連の開発者である Michael Shalayeff と Tobias Weingartner 宛にお願いします。

IBM Thinkpad 770Z

OpenBSD 2.8 上でほとんどすべてが良好に動作します。

このラップトップは実際には BIOS を持っていませんので、 IRQ の割り当てなどのほとんどのデバイスの設定は、Windows 上で行わなければなりません。

Windows が割り当てた IRQ によって com* として認識される、 シリアルポートと赤外線ポートを使用できます。 パラレルポートも同様です (これは lpt2 です)。

APM を動作させるためには、カーネル設定ファイルに 以下の行が必要です。

option         APM_DISABLE_INTERRUPTS=0

(システムメモリより少しだけ多い) 十分なディスクスペースを持つ Windows パーティションがある限り、(ディスクにセーブする) ハイバネーションが 良好に動作します。サスペンド/リジューム/電源断もすべて良好に動作します。 しかし、X Window System (XFree86 3.3.6 でテスト) からのハイバネーションは、 フレームバッファの汚染の原因となり、その状態のリカバーは いったん X サーバを終了して再起動するのが唯一の方法のようで、 仮想ターミナルの切り替えではこの問題は解決しません。異なる解像度間で 切り替えることで、この問題が解決するかも知れないという示唆もあります。 XFree86 4.0 ではこの問題は修正されていますが、最近私がチェックした限りでは、 このドライバ用のアクセラレータのサポートはありません。最も良い解決策は、 ハイバネーションの前にテキストモードに切り替えることでしょう。

また、外部モニタを (Fn+F7 で) 有効化した場合には、XFree86 3.3.6 がある種の不思議な虹色のパターンを表示してしまい、(数秒のうちに、 テキストモードに切り替えるか、X サーバを kill するかしなければ) 突発的にシステムがフリーズしてしまいます。ここで、もし、テキストモードに 切り替えた場合、X サーバを kill しなければなりません (そうしなければ、 グラフィックスモードに戻った時点でシステムがクラッシュします)。ビデオモードで 外部モニタを有効化することには副作用はありません。その後、X を起動することも できますし、既に実行されている場合には、それに切り替えることもできます。

clcs(4) ドライバは どこか変ですので、これは無効化しておく方が良いでしょう。これは実際のものとは異なるのですが、サウンドブラスタ (sb(4)) ドライバなら認識できて良好に動作するでしょう。そのためには、IRQ の割り当てが必要になるかも知れません。

XF86Config ファイルが こちらにあります。

シリアルならびに赤外線ポート (いずれも com* デバイスとして認識) が見えるようにするためには、BIOS で IRQ を割り当てておく必要があります。 これは、GENERIC 設定ファイル中のデフォルト値に設定するので十分でしょう。 これは、パラレルポートについても同様です (OpenBSD では lpt2 として見えます)。 また、この割り当ては、Windows における割り当てでも 同じように使用することができます。

現時点では、ふたつのミニ PCI カードが利用可能で、これらは ひとつが (サポートされていない) WinModem であり、もうひとつが Ethernet/モデムのコンボカードです。Ethernet は (fxp/inphy として 認識され) 良好に動作します。しかし、モデムの方は、実際のものとして 認識されているように見えますが、まだサポートされていません。

サウンド (CS4614) は良好に動作します。CardBus とオーディオとを同時に使用する際の システムのフリーズを回避するため、BIOS の PCI デバイスのための、少なくともふたつの IRQ を設定しなければならないかも知れません。私は 9 と 11 とを使用しています。また、 BIOS の電力設定の PCI パワーマネージメントを使用しないようにする必要もあるでしょう (これはスクロールするメニューでしかありませんので、このページの最終行まで確実に スクロールしてください :-)。

(OS のサポートを必要とする) ハイバネーションを除いて AP は動作します。 "halt -p" が動作するようにするためには、sysctl の変数 machdep.apmhalt を 1 設定する必要があります (1 月中旬かそれ以降の OpenBSD 2.8-current がソース付きで必要です)。また、最終的に APM を動作させるためには、 カーネルの設定ファイルに以下の行が含まれていなければなりません。

option         APM_DISABLE_INTERRUPTS=0

ビデオカードは、XFree86 3.3.6pl8 で動作するはずですが、 私の場合、XFree86 4.0.1 にこちらの パッチを適用したものでなければ動作しませんでした (同じ場所で OpenBSD 2.8 用ドライバのバイナリも見つかるはずです)。また、 (X サーバが /dev/mem をオープンし、BIOS に mmap() 可能とするため) "option INSECURE" を指定したカーネルと、/etc/rc.securelevel で securelevel に -1 を設定する必要があります。この設定で、 16bpp で 1400x1050 の解像度を使用することができます (バージョン 1.1.13 の savage ドライバなら、24bpp でも良好に動作します)。

更新: 上記に示したパッチは、XFree86 4.0.2 に取り込まれ、上記の場所から削除されました。また、 option INSECURE を設定したカーネルも不要で、2000 年 12 月 15 日以降に 更新されたソースから構築された OpenBSD 2.8 (またはそれ以降) の場合、 単に /etc/sysctl.conf で sysctl の変数 machdep.allowaperture を 2 に設定するだけで良くなりました。

いくつかのモデルは 1024x768 の解像度でしか利用できないことに注意してください。 1400x1050 の解像度が利用可能なモデルは、仮想スクリーンで 1600x1200 の解像度 (この場合、スクリーン端点でスクリーン全体がシフトします) でも利用することができるはずですが、 それが動作するのかどうかはテストされていません (ただし、Windows では利用できています)。

ラップトップの蓋を閉じた場合、XFree86 4.0 を利用していれば、 システムはサスペンドします。また、リジュームも良好に動作し、 ディスプレイは期待どおりに元に戻ります (XFree86 3.3.6 の場合、 このように動作せず、ThinkPad 770Z では使用できません)。

XFree86 4.0 用の XF86Config ファイルが こちらにあります。

Toshiba Libretto 50CT APM は動作しますが、BIOS がハイバネーション領域として使用しますので、 ディスクの最後の 32MB かそれくらいを使用しないよう、 注意する必要があります。disklabel の BIOS ジオメトリを見ることで、 この領域のためにどれくらいのセクタ数を必要とするのか知ることができます。 これを守らなければ、サスペンド時にファイルシステムが 壊される原因となってしまいます。

この Libretto はひとつの IRQ を sb(4) および wss の両方に使用しますので、これらのうちのひとつを選択し、他方を カーネルで無効化しておくのが最も良いでしょう。 PCMCIA の外部フロッピーディスクドライブは現時点では OpenBSD でサポートされていませんが、インストール時には使用できます。

インストールは、ひとつだけの PCMCIA スロットを使用した、ややトリッキーな ものになります。最も良いインストール方法はネットワークインストールでしょう。 これを行うには、ブートフロッピーで起動し、ブートプロンプトのところで boot -c と入力します。UKC プロンプトが表示されたら、 フロッピーディスクドライブを取り外して、ネットワークカードを挿入し、exit と入力してください。このようにすることで、カーネルがネットワークカードを プローブして、ネットワークインストールが可能になるはずです。 ブートフロッピーが正常にネットワークカードを 検出できるようにするためには、BIOS でサウンドデバイスを 無効化しておく必要があるかも知れません。

XFree86 4.X 用と XFree86 3.3.X 用のサンプルの XF86Config ファイルがあります。

XFree86 4.1 では、このマシンのチップ用のドライバは壊れていることに 注意してください (これは XFree86 4.2 では修正されています)。ですので、 OpenBSD 3.0 とともに出荷された XFree86 は Libretto 上では動作しません。 (動作することがわかっている) 以前の XFree86 4.01 のこのチップ用のドライバ を XFree86 4.1 用にコンパイルしたものを chips_drv.o として置いていますので、これを以下の場所にコピーしてください。 

	    /usr/X11R6/lib/modules/drivers/chips_drv.o
これによって XFree86 4.1 が 50CT および 70CT 上で動作するようになるでしょう。
