メーリングリスト

メーリングリストは OpenBSD の開発者とユーザの間のコミュニケーションのための重要な 手段です。announce は例外ですが、メーリングリストはモデレータ制ではありません。 私たちは、いくつもの異なるメーリングリストを意図的に限定してきました。これによって、 クロスポストの総量は減少しますので、情報が広く参加者に配布されることが確実となるで しょう。

ネチケット

メーリングリストの他の参加者のことも考えるようにしましょう。

プレーンテキストで、1 行は 72 文字までにしましょう
多くの参加者は、メールをテキストベースのメーラー (mail(1)、emacs) で読んで いますので、HTML でフォーマットされたメールや 1 行が 72 文字以上のメールは、 しばしば読めないという事態が起こります。 ほとんどの OpenBSD のメーリングリストでは、配送する前に MIME コンテントの メッセージを除去します。 もし、あなたが、プレーンテキストでメッセージを再フォーマットしない場合や、 できない場合には、直ちに受け付けは拒否されます。 添付ファイルを許している唯一のメーリングリストが、ports メーリング リストで、他のメーリングリストでは、そのようなメッセージは除去されるでしょう。
投稿する前に、先に宿題を済ませましょう
もし、インストールに関する質問があるならば、FTP サーバのインストレーション ディレクトリにある INSTALL.* テキストファイルのような関連ドキュメント、あるいは、 FAQ や関係する man ページ (これは、 afterboot(8) から始まりまっています) に確実に目を通しておきましょう。 トラブルシューティングの役に立つよう、適切な詳細 (バージョン、ハードウェア、 dmesg(8)) が含まれていることを確認してください。私たちは助けたいとは思いますが、 同時に、あなたの価値ある学習経験の機会を奪いたいとも思わないのです。
役に立つ Subject: にしましょう
空の Subject: のメッセージは、メーリングリストの管理人のところにバウンスするで しょうから、皆さんの目に触れることもないでしょう。適切な Subject: のメッセージは、 より多くの人々が実際にあなたの書いたメールを読んでくれることを保証してくれるでしょう。 同様にして、過度に大文字を使用した Subject: 行も避けるようにしてください。
シグネチャーは短か目に
メールの最後のシグネチャー行は、合理的な範囲の長さに留めておきましょう。投稿する ということは、稀に PGP シグネチャーの使用の根拠となるような、危機的な状況になるかも 知れませんし、また、このような自動的なアドレスカードは、単にうるさいだけでしょう。
トピックに合わせましょう
OpenBSD ユーザに対して、投稿の際に、適正な Subject: を付けるよう心がけてください。

スパム

スパムは常に禁止されていますが、そのような事態が見過ごされることも あります。もし、ひとつのメーリングリストでスパムを受け取れば、 spamcop に報告したくなるかも知れませんが、 一般的には、メーリングリスト管理人に、そのコピーを送る必要はありません。 なぜなら、彼も既にそれを見る機会があったわけですから。ただ、スパムに対して そのメーリングリスト上で苦情を言ったりするのは、スパムそのものよりも多くの トラフィックを生ずるため、むしろ逆効果であることに注意してください。
OpenBSD のメーリングリストは、 MAPS RBL、 MAPS Dialup や MAPS RSS などのブラックリストに載っている すべてのホストからのメールを拒絶しています。さらに、メーリングリストのサーバは、 ある種の共通のスパムの証拠に基づく、正規表現ベースの拒絶ルールも持っています。

メーリングリスト

用意されているメーリングリストに subscribe するには、 "subscribe mailing-list-name" という本文のメールを majordomo@OpenBSD.org に送ってください。

announce
重要なアナウンスを行うためのメーリングリストです。この流量の少ないメーリングリストは、 プロジェクトに関する、そのときどきのニュースを望む人々にとって非常に便利なものでしょう。
security-announce
セキュリティに関するアナウンスを行うためのメーリングリストです。この流量の少ない メーリングリストは、OpenBSD のセキュリティアドバイザリを受け取り、セキュリティパッチが 利用可能になれば、それを告知するためのものです。
misc
利用者の質問とそれに対する回答や一般的な疑問のためのメーリングリストです。これは、 最も活発なメーリングリストです。ここに投稿する前に、ぜひ、 FAQ とインストレーションドキュメントを読んでください。、
tech
OpenBSD の開発者や上級者のための技術的なトピックを扱うメーリングリストです。 「新規ユーザ」やインストールに関連の質問は、misc の方に お願いします。また、misc と tech の両方に クロスポストしないよう、お願いします。
bugs
sendbug(1) ユーティリティで送付されたバグレポートと、それに対するフォローアップ の議論を行うためのメーリングリストです。
source-changes
CVS ソースツリーの変更時点で自動的に流されるメールのためのメーリングリストです。
ports
'ports' の使用法や、そのソースツリーへの寄付に関する議論を行うためのメーリングリストです。
ports-changes
ports に限定した CVS ソースツリーの変更時点で自動的に流されるメールのためのメーリングリストです。
ipv6
OpenBSD の IP バージョン 6 に関する議論を行うためのメーリングリストです。
advocacy
OpenBSD の使用のプロモーションを行うためのメーリングリストです。misc の技術的ではない 議論が、しばしばこちらに移ってくることがあります。
smp
OpenBSD の対称マルチプロセシング (SMP) の設計と実装に関する議論を行うための メーリングリストです。

OpenBSD では、特別なプロジェクトや、特定のアーキテクチャに特化したメーリングリストをいくつか 運用しています。これらのメーリングリストは、誰でも subscribe できる、公開されたメーリングリスト ですが、これらは開発者やテスターのために用意されたメーリングリストだと見なされているものです。 OpenBSD の公開の原則にしたがって、これらはプライベートなメーリングリストにはなっていないのです。

より詳しいことが知りたい場合には、"help" とだけ本文に書いたメールを、 majordomo@OpenBSD.org に送付してください。 そして、すべてのオプションの概要が書かれた返信を受け取ることができるでしょう。 あなたのドメインが正しく引けなければ、メールは受け付けられないでしょう。

squish.net のすばらしい皆さんが、 OpenBSD の source-changes メーリングリストの、毎日および毎週の ダイジェスト版メーリングリストを運用してくれています。これは、典型的にオリジナルの メーリングリストだと流量が多くて困る人たちに便利なメーリングリストです。

monkey.org の賢いお猿さんたちが、モバイルや ラップトップパソコン上で OpenBSD を使用している人たちのための、 openbsd-mobile メーリングリストを運用しています。このメーリングリストに subscribe するには、次のようにしてください。
echo subscribe | mail openbsd-mobile-request@monkey.org

OpenBSD の FTP、Web、AnonCVS および CVSup のミラーのメンテナのためのメーリングリストとして、 rt.fm があります。
