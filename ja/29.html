What's New

これは OpenBSD 2.8 に含まれた新機能やシステムの部分的な一覧だ。 もっと完全な一覧を見るには 2.9 までずっとつづくchangelog を見よう。

  • ファイルシステムの性能が、新しい softupdatesとdirpref コードのおかげですさまじく向上。一部のテストだと、ファイルシステムの速度が60倍も向上している。2.9にアップグレードする理由を一つあげろと言われたら、まずこれだね。

  • OpenSSH (SSH1 と SSH2 プロトコルの 両方をサポート) は現在 2.9 だ。セキュアなファイル転送用もお勧め。 これ用に大幅に拡張された SFTP サブシステムが含まれていて、 これは SFTP サーバとクライアントの両方が入っている。

  • ドキュメンテーションに大幅な変更。特にman ページと Web FAQ。manページは説明を補うために便利な例題も 追加してある。

  • "ports"が拡充されてテストもしっかりされている。

  • 1000 以上のビルド済みテスト済みパッケージ。

  • OpenBSD 2.9 は新しい Apple Titanium Powerbook G4で動く。

  • リリース二回お休みをもらったけれど、OpenBSD/Alphaポートが復活。これまで以上のハードをサポート！

  • システムは、外部サプライヤからの以下の主要コンポーネントを含む:

    • XFree86 4.0.3 (それとi386版は3.3.x のサーバも入っているので、あらゆるチップセットがサポー
    • gcc 2.95.3 (+ パッチ)
    • perl 5.6.0 (+ パッチ)
    • Apache 1.3.19, mod_ssl 2.8.2, OpenSSL 0.9.6 (+ パッチ), DSO サポート
    • ipf 3.4.16 (+ パッチ)
    • groff 1.15
    • sendmail 8.11.3
    • lynx 2.8.2, HTTPS サポート付き
    • sudo 1.6.3p7
    • ncurses 5.2
    • 最新の KAME IPv6
    • KTH Kerberos 1.0.4
    • OpenSSH 2.9

  • デバイスドライバサポート大幅改善

    • 3Com 3c990 3XP Typhoon/Sidewinder driver (Ethernet のみ).
    • 新しいハードウェア RAID ドライバ、たとえば:
      • Adaptec FSA
      • American Megatrends MegaRAID
      • Compaq Smart ARRAY コントローラ
    • 新しい SCSI ドライバのサポート。たとえば:
      • Initio 940/950
      • Symbios Logic 53c1x10
      • AMD Am53c974-based PCI adapters
    • 各種ラップトップオーディオサポートを大量追加。たとえば:
      • ESS Maestro 1/2/2E
      • Yamaha DS-1S
      • Cirrus Logic CS4281 audio devices
    • USB がデフォルトで有効にされていて、キーボードとマウスもサポート。

  • セキュリティと信頼性面で多々の改善点（完全な変更履歴 の赤字を参照)。

  • 数々の新機能、たとえば:

    • 長いユーザ名のサポート。
    • Linux エミュレーションを大幅に改善して、ずっと多くのアプリケーションが走るようにした。
    • bridge(4)用の 802.1d spanning tree サポート。
    • vlan(4)によるブリッジ のサポートで、より多くのブリッジが可能に。（これで OpenBSD を802.1Q互換スイッチとして も使うことがでるようになりました。）
    • kqueue(2) カーネルイベント通知メカニズム。
    • wscons(4) 機種非依存コンソールドライバの追加。
    • softupdate が、速度面と安定性などで大幅に改善。
    • atapiscsi(4) と pciide(4) ドライバーにバグフィックスと機能拡張、 Interl, Promise そして Highpoint 製 UltraDMA/100 コントローラのサポートなどです。

How to install

以下は、 色々あるインストール手段の中で、 CD-ROMセットを買ったら紙に印刷してついてくる指示書だ。 ftp（あるいはその他）でのインストール方法もほとんど同じだ。 ここでは CDROM 版の指示をそのまま記述してある。 ほらね、CD-ROM を買うとこんなに楽なんですよー。

マシンに OpenBSD 2.9 をインストールするための詳しい方法については、 CD 2 枚の上にあるそれぞれ以下のファイルを参照:

  • CD1:2.9/i386/INSTALL.i386
  • CD1:2.9/powerpc/INSTALL.powerpc
  • CD1:2.9/vax/INSTALL.vax

  • CD2:2.9/sparc/INSTALL.sparc
  • CD2:2.9/mvme68k/INSTALL.mvme68k
  • CD2:2.9/amiga/INSTALL.amiga
  • CD2:2.9/mac68k/INSTALL.mac68k
  • CD2:2.9/hp300/INSTALL.hp300
  • CD2:2.9/sun3/INSTALL.sun3

  • alpha リリースは ftp サイトにはあるけれど、CDには入っていない。

OpenBSD の経験がある人のための、簡単なインストーラ情報と、新しい "disklabel -E" コマンドの説明だ。OpenBSD のインストールがまったく 皆目見当がつかない人は、上に挙げた関連の INSTALL.* ファイルを見てね!

OpenBSD/i386:

    BIOS のオプションをいじくって、CD から起動できるかやってみよう。 CD1 を使うといい。ダメなら起動フロッピーを作ろう。 CD1:2.9/i386/floppy29.fs を フロッピーに書き出して、そこから起動すればいい。

    OpenBSD をほかの OS と同居させるつもりなら、 絶対に INSTALL.i386 を読もう。

    MS-DOS で起動フロッピーをつくるには"rawrite"を使う。CD:/2.9/tools/rawrite.exe にある。 Unix OS で起動フロッピーを作るには dd(1) を使う。以下に dd(1) の使用例を書いておく。 device は"floppy" とか "rfd0c" とか "rfd0a".だ。 

      # dd if=<file> of=/dev/<device> bs=32k

    きちんとフォーマットされた不良ブロックなしのフロッピーを使わないと、 ドツボよーん。起動フロッピーの作り方や OpenBSD/i386 のインストールについて もっと詳しく知りたければ、FAQ4.1 を読もう。

OpenBSD/sparc:

    OpenBSD/sparc の 2.9 リリースは CD2 に入っている。CD2 から起動するには、 ROMのバージョンに応じて、以下の 2 つのコマンドのうちどちらかが使えるだろう。 

      > boot cdrom 2.9/sparc/bsd.rd
or
> b sd(0,6,0)2.9/sparc/bsd.rd

    CD ドライブの無い sparc だったら、代わりに "CD2:2.9/sparc/floppy29.fs" を フロッピーに書き込んでフロッピーで起動できる。詳しいことは FAQ4.1 を読もう。フロッピーから起動するにはROM のバージョン に応じて、以下の 2 つのコマンドのうちどちらかが使えるだろう。 

      > boot floppy
or
> boot fd()

    きちんとフォーマットされた不良ブロックなしのフロッピーを使わないと、 ドツボよーん。

    もしもフロッピーも CD も無ければ、INSTALL_sparc ファイルに 書いてあるように、起動可能な tape かネットワーク越しにインストール することもできる。

OpenBSD/amiga:

    NSATALL.amiga の準備セクションを読んで、BSDパーティションを作ろう。AmigaOS で、 CD2を CD0: にマウントする。それから以下の CLI コマンドを実行する： "CD0:2.9/amiga/utils/loadbsd CD0:2.9/amiga/bsd.rd".

OpenBSD/hp300:

    INSTALL.hp300の指示に従って、ネットワークから 起動できる。

OpenBSD/mac68k:

    ふつうにMacOSを起動して、しかるべき A/UX の設定でディスクにパーティションを 切ってほしい。それから Mac 側のユーティリティを CD2:2.9/mac68k/utils から ハードディスクに解凍すること。Mkfsを実行して、いま作ったA/UX パーティションに ファイルシステムを作ろう。それから BSD/Mac68k インストーラを使って、 CD2:2.9/mac68k/ の全セットをパーティションに移そう。最後に、 BSD/Mac68k Booter に、カーネルのありかを教えて設定したら、 システムを起動しよう。

OpenBSD/sun3:

    ディスクレスブートを設定するか、インストールテープをつくろう。 これは INSTALL.sun3に説明してある。

OpenBSD/alpha:

    リリースを ftp から持ってくる。そうしたらディスクレスブートを設定するか、フロッピー で起動しよう。これは INSTALL.alpha に説明してある。

ソースコードについて一言:

    src.tar.gz は /usr/src から始まるソースコードが入っている。このファイルには、 カーネルソース以外の必要なものがすべて入っている。 抽出には: 

      # mkdir -p /usr/src
# cd /usr/src
# tar xvfz /tmp/src.tar.gz

    srcsys.tar.gz は /usr/src/sysからはじまるソースアーカイブだ。このファイルには、 カーネル再構築に必要なカーネルソースがすべて入っている。 抽出には: 

      # mkdir -p /usr/src/sys
# cd /usr/src
# tar xvfz /tmp/srcsys.tar.gz

    これらのツリーはどっちも普通の CVS チェックアウトだ。このツリーを使えば、 http://www.OpenBSD.org/anoncvs.htmlで先陣を 切ることもできるぞ。 このファイルを使えば、完全な OpenBSD ソースツリーを全部まっさらに チェックアウトするよりもずっと高速に、最初の CVS アップデートができる。

Ports ツリー

ports ツリーアーカイブも提供されている。これを抽出するには: 

    # cd /usr
# tar xvfz /tmp/ports.tar.gz
# cd ports

ports/ サブディレクトリは、OpenBSDの ports ツリーのチェックアウトだ。 いま、ports についてまったく知らない人は http://www.OpenBSD.org/ports.html を読んでほしい。このテキストは、 ports の使い方のマニュアルではないのだ。むしろ、OpenBSD ports システムの クイックスタートガイドだと思ってほしい。

確かに、OpenBSD ports システムは完璧じゃない。いつか完璧になるかどう かも疑問だ。しかし、すごい勢いで成長し、どんどん安定性を増しつつある。 このリリースで提供しているほとんどすべての ports はかなりのアーキテクチャ で問題なくビルド出来ると思う。（たとえば、i386 では 1200 以上の package がある。）

ports/ ディレクトリは CVS (なんだか知らなければ cvs(1) の manpageを 見てちょ）の ports チェックアウトになっている。ソースツリーの場合と 同じように、anoncvs でも提供してある。だから ports/ ディレクトリをカレントにしておくには、ports/ ツリーを読み書き可能な メディアにこしらえて、以下のコマンドでツリーを 更新しておこう: 

    # cd [portsdir]/; cvs -d anoncvsserver.openbsd.org:/cvs update -Pd -rOPENBSD_2_9

[もちろん、ローカルのディレクトリとサーバ名は、 自分の ports ツリーの在処と、 最寄りの cvs サーバに置き換えること。]

かなりの ports が packages として ftp で取ってこれる事に注意。2.9 リリース用 に更新された packages は問題が解決すれば配布を開始する予定だ。

もし追加してほしい ports があるとか、手伝いをしたいとか、あるいはもっと 詳しく知りたければ、メーリングリスト ports@openbsd.org に 参加するといいだろう。
