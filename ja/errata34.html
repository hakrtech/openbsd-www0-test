OpenBSD パッチブランチの情報はこちらをご覧ください。
主要な packages のアップデート情報はこちらをご覧ください。

今までのリリースの正誤表は、こちらをご覧ください。
2.1、 2.2、 2.3、 2.4、 2.5、 2.6、 2.7、 2.8、 2.9、 3.0、 3.1、 3.2、 3.3、 3.5、 3.6。
次のすべてのパッチを含む tar.gz ファイルを取得可能です。 このファイルは、毎日 1 度更新されます。

OPENBSD_3_4 のパッチブランチ の CVS では、以下のパッチが利用可能となっています。

OpenBSD へのパッチの適用のしかたについてのより詳しい情報は、 OpenBSD FAQ をご参照ください。

全アーキテクチャ共通

  • 035: セキュリティのための修正: December 13, 2004
    On systems running isakmpd(8) it is possible for a local user to cause kernel memory corruption and system panic by setting ipsec(4) credentials on a socket.
    この問題に対処するためのソースコードパッチはこちらです。

  • 034: 信頼性のための修正: November 10, 2004
    Due to a bug in lynx(1) it is possible for pages such as this to cause lynx(1) to exhaust memory and then crash when parsing such pages.
    この問題に対処するためのソースコードパッチはこちらです。

  • 033: 信頼性のための修正: November 10, 2004
    pppd(8) contains a bug that allows an attacker to crash his own connection, but it cannot be used to deny service to other users.
    この問題に対処するためのソースコードパッチはこちらです。

  • 032: 信頼性のための修正: November 10, 2004
    BIND contains a bug which results in BIND trying to contact nameservers via IPv6, even in cases where IPv6 connectivity is non-existent. This results in unneccessary timeouts and thus slow DNS queries.
    この問題に対処するためのソースコードパッチはこちらです。

  • 031: セキュリティのための修正: September 20, 2004
    Eilko Bos reported that radius authentication, as implemented by login_radius(8), was not checking the shared secret used for replies sent by the radius server. This could allow an attacker to spoof a reply granting access to the attacker. Note that OpenBSD does not ship with radius authentication enabled.
    この問題に対処するためのソースコードパッチはこちらです。

  • 030: セキュリティのための修正: September 16, 2004
    Chris Evans reported several flaws (stack and integer overflows) in the Xpm library code that parses image files (CAN-2004-0687, CAN-2004-0688). Some of these would be exploitable when parsing malicious image files in an application that handles XPM images, if they could escape ProPolice.
    この問題に対処するためのソースコードパッチはこちらです。

  • 029: セキュリティのための修正: September 10, 2004
    httpd(8) 's mod_rewrite module can be made to write one zero byte in an arbitrary memory position outside of a char array, causing a DoS or possibly buffer overflows. This would require enabling dbm for mod_rewrite and making use of a malicious dbm file.
    この問題に対処するためのソースコードパッチはこちらです。

  • 028: 信頼性のための修正: August 26, 2004
    As reported by Vafa Izadinia bridge(4) with IPsec processing enabled can be crashed remotely by a single ICMP echo traversing the bridge.
    この問題に対処するためのソースコードパッチはこちらです。

  • 027: 信頼性のための修正: August 25, 2004
    Improved verification of ICMP errors in order to minimize the impact of ICMP attacks against TCP.
    http://www.gont.com.ar/drafts/icmp-attacks-against-tcp.html
    この問題に対処するためのソースコードパッチはこちらです。

  • 026: 信頼性のための修正: Jul 25, 2004
    Under a certain network load the kernel can run out of stack space. This was encountered in an environment using CARP on a VLAN interface. This issue initially manifested itself as a FPU related crash on boot up.
    この問題に対処するためのソースコードパッチはこちらです。

  • 025: セキュリティのための修正: June 12, 2004
    Multiple vulnerabilities have been found in httpd(8) / mod_ssl. CAN-2003-0020, CAN-2003-0987, CAN-2004-0488, CAN-2004-0492.
    この問題に対処するためのソースコードパッチはこちらです。

  • 024: セキュリティのための修正: June 10, 2004
    As disclosed by Thomas Walpuski isakmpd(8) is still vulnerable to unauthorized SA deletion. An attacker can delete IPsec tunnels at will.
    この問題に対処するためのソースコードパッチはこちらです。

  • 023: セキュリティのための修正: June 9, 2004
    Multiple remote vulnerabilities have been found in the cvs(1) server that allow an attacker to crash the server or possibly execute arbitrary code with the same privileges as the CVS server program.
    この問題に対処するためのソースコードパッチはこちらです。

  • 022: セキュリティのための修正: May 30, 2004
    A flaw in the Kerberos V kdc(8) server could result in the administrator of a Kerberos realm having the ability to impersonate any principal in any other realm which has established a cross-realm trust with their realm. The flaw is due to inadequate checking of the "transited" field in a Kerberos request. For more details see Heimdal's announcement.
    この問題に対処するためのソースコードパッチはこちらです。

  • 021: セキュリティのための修正: May 20, 2004
    A heap overflow in the cvs(1) server has been discovered that can be exploited by clients sending malformed requests, enabling these clients to run arbitrary code with the same privileges as the CVS server program.
    この問題に対処するためのソースコードパッチはこちらです。

  • 020: セキュリティのための修正: May 13, 2004
    Check for integer overflow in procfs. Use of procfs is not recommended.
    この問題に対処するためのソースコードパッチはこちらです。

  • 019: 信頼性のための修正: May 6, 2004
    Reply to in-window SYN with a rate-limited ACK.
    この問題に対処するためのソースコードパッチはこちらです。

  • 018: 信頼性のための修正: May 5, 2004
    Under load "recent model" gdt(4) controllers will lock up.
    この問題に対処するためのソースコードパッチはこちらです。

  • 017: セキュリティのための修正: May 5, 2004
    Pathname validation problems have been found in cvs(1), allowing malicious clients to create files outside the repository, allowing malicious servers to overwrite files outside the local CVS tree on the client and allowing clients to check out files outside the CVS repository.
    この問題に対処するためのソースコードパッチはこちらです。

  • 016: 信頼性のための修正: 2004 年 3 月 17 日
    欠落していた NULL ポインタ被参照のチェックが ssl(3) に発見されました。 リモートの攻撃者は、このバグを利用して、OpenSSL を使用するアプリケーションをクラッシュさせることができます。 これによって、サービス拒否攻撃を行うことができてしまいます。
    この問題に対処するためのソースコードパッチはこちらです。

  • 015: 信頼性のための修正: 2004 年 3 月 17 日
    isakmpd(8) のペイロードの検証および処理関数の欠陥が発見されました。 攻撃者は、異常な ISAKMP メッセージを送信し、isakmpd のクラッシュや無限ループを 引き起こすことができてしまいます。このパッチは、これらの問題を修正し、さらに いくつかのメモリリークを除去します。
    この問題に対処するためのソースコードパッチはこちらです。

  • 014: セキュリティのための修正: 2004 年 3 月 13 日
    httpd(8) のアクセスモジュール用の Allow/Deny ルールの構文解析のバグが原因で、ビッグエンディアンの 64 ビットプラットフォーム上で、ネットマスクを指定せずに IP アドレスを使用していると ルールのマッチが失敗します。これは sparc64 にのみ影響します。
    この問題に対処するためのソースコードパッチはこちらです。

  • 013: 信頼性のための修正: 2004 年 3 月 8 日
    OpenBSD の TCP/IP プロトコルスタックには、順序どおりではない TCP セグメントを システムがキューイングする数に制限がありません。攻撃者は順序どおりではない TCP セグメントを送信し、すべての利用可能なメモリバッファを使用し尽すよう システムを騙すことができてしまいます。
    この問題に対処するためのソースコードパッチはこちらです。

  • 012: 信頼性のための修正: 2004 年 2 月 14 日
    XFree86 において、font.aliases ファイルの構文解析を行うためのコードに、いくつかの バッファオーバーフローが存在しています。しかし、ProPolice のお蔭で、これらを利用して 特権を奪取することはできませんが、X サーバを異常終了させることができてしまいます。
    この問題に対処するためのソースコードパッチはこちらです。

  • 011: セキュリティのための修正: 2004 年 2 月 8 日
    IPv6 には、IPv6 の TCP ポートがリーチャブルなホストに対して 攻撃者がサービス拒否攻撃を実行するために使用できる、MTU のハンドリングの問題が存在しています。
    この問題に対処するためのソースコードパッチはこちらです。

  • 010: セキュリティのための修正: 2004 年 2 月 5 日
    攻撃者が特定の環境下でカーネルメモリへの書き込みに使用することができる 参照カウントのバグが shmat(2) システムコールに存在しています。
    この問題に対処するためのソースコードパッチはこちらです。

  • 009: セキュリティのための修正: 2004 年 1 月 13 日
    いくつかのメッセージハンドリングに関する欠陥が isakmpd(8) に存在することが Thomas Walpuski によって報告されました。この欠陥によって、攻撃者は任意の SA を消去することができてしまいます。パッチには、暗号カードがインストールされている場合に問題となる、 ファイルディスクリプタのリークに関する信頼性のための修正も含まれています。
    この問題に対処するためのソースコードパッチはこちらです。

  • 008: 信頼性のための修正: 2003 年 11 月 20 日
    不正確な境界チェックのため、ローカルユーザは semctl(2) と semop(2) 関数に ある種の引数をわたすことにより、 クラッシュの原因となることができてしまいます。
    この問題に対処するためのソースコードパッチはこちらです。

  • 007: 信頼性のための修正: 2003 年 11 月 20 日
    ローカルユーザは sysctl(3) に、ある種の引数を指定することで、システムをクラッシュさせることができてしまいます。
    この問題に対処するためのソースコードパッチはこちらです。

  • 005: 信頼性のための修正: 2003 年 11 月 4 日
    ローカルユーザは、不正なヘッダを持つ特別に組まれたバイナリを実行することによりシステムをパニックに陥れることが可能です。
    この問題に対処するためのソースコードパッチはこちらです。

  • 004: 信頼性のための修正: 2003 年 11 月 1 日
    httpd.conf や .htaccess ファイルへの 書き込みパーミッションを持つユーザは、 httpd(8) をクラッシュさせることができる可能性があり、あるいは、www ユーザとして任意のコードの 潜在的な実行可能性があります (しかし、ProPolice がコードの実行を阻止してくれるはずです)。
    この問題に対処するためのソースコードパッチはこちらです。

  • 003: 信頼性のための修正: 2003 年 11 月 1 日
    ローカルユーザは、偽の ARP リクエストで ARP テーブルを溢れさせることによって、 システムをパニックに陥れることができてしまいます。
    この問題に対処するためのソースコードパッチはこちらです。

  • 002: セキュリティのための修正: 2003 年 11 月 1 日
    ある種の ASN.1 エンコーディングや奇形の公開鍵の使用によって、攻撃者は ssl(3) がリンクされたアプリケーションに対してサービス拒否攻撃を行うことができる可能性があります。 この問題は、OpenSSH には影響を及ぼしません。
    この問題に対処するためのソースコードパッチはこちらです。

  • 001: ドキュメントの修正: 2003 年 11 月 1 日
    CD に添付されたドキュメントには、パケットのインストールについて正しくない例が含まれています。
    以下のように書かれた部分がそれです。

               # pkg_add ftp://ftp.openbsd.org/pub/OpenBSD/3.4/packages/i386

    ここは、以下のように読んで欲しい部分です。

               # pkg_add ftp://ftp.openbsd.org/pub/OpenBSD/3.4/packages/i386/

    最後に追加された / が重要なのです。私たちは、 紙に印刷されたものに対して利用可能なパッチファイルは提供しません。

i386

  • 006: セキュリティのための修正: 2003 年 11 月 17 日
    compat_ibcs2(8) のスタックを、ローカルユーザはオーバーランさせることができてしまいます。
    ProPolice がこれを捕えると、潜在的な権限昇格をサービス拒否へと変換します。 sysctl(8) を使用して iBCS2 エミュレーションを有効化することなく、 この現象を引き起こすことができてしまいます。
    この問題に対処するためのソースコードパッチはこちらです。
