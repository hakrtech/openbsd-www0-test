OpenBSD 2.9 と OpenBSD 3.0 との間の変更点

以下は、機種に依存しない主な変更点の部分的なリストで、これらは、 皆さんがかなり頻繁に問い合わされる変更点でもあります。ports に 関する変更も行われており、それらについては、ときどき、それぞれの 機種に関するページに記載されています。

ports コレクションの変更点については、 こちらに記述されています。

注: パッチの存在する問題については、朱書きで記されています。

他のリリースの変更点については、こちらをご覧ください。

OpenBSD 3.0 リリース (2001 年 12 月 1 日)

  • pf(4) は、IP オプションがセットされたパケットをデフォルトでブロック。
  • fd(4) の共有を無効化。
  • pflogd(8) の umask(2) を修復。
  • ssh(8) および ssh(1) のチャネルの取り扱いを調整。
  • pfctl(8) にインターフェイス名からアドレスへの変換を追加。
  • mktemp(1) を利用して cvs(1) の /tmp の競合を回避。
  • whois(1) サーバの、終端改行文字が不足するという不正な出力を取り扱い。
  • FM radio(9) ドライバのサポート。これは、radioctl(1) を通じて設定。
  • ポート番号だけを含む場合に Host: フィールドを要求する、バギーな HTTP サーバでもうまく動作するように改善。
  • pcap(3)、isakmpd(8)、bpf(4) および tcpdump(8) で、timeval に代えて bpf_timeval を使用し、移植性を改善。
  • DF フラグがセットされた icmp(4) パケットの状態の検索を壊してしまう pf(4) のネットワーク/ホスト順序のバグを修正。
  • 新しい ssh(1) 設定オプション NoHostAuthenticationForLocalhost。
  • 新しい rmd160(1) の実装をインポート。
  • 潜在的なローカルセキュリティホールが修正された sendmail(8) 8.12.1 にアップグレード。
  • TMPDIR 環境変数を評価させるため、mktemp(1) に新しい -t および -d フラグを追加。
  • mtree(8) に、パーミッションチェックを緩和するための -l フラグを追加。
  • pfctl(8) で -N および -R フラグが指定されている場合に、特定のファイルがオープンできない場合のエラーを報告。
  • 改行文字を含むファイル名をスキップさせるよう、rcp(1) および scp(1) を修正。
  • インストールドキュメンテーションの大がかりなオーバーホール。
  • pf(4) で、選択可能なプリセット FSM の最適化をサポート。
  • 多数の RAIDFrame 関連の作業。.
  • フォワードされた ip6(4) パケットを pf(4) でフィルタリング。
  • cvs(1) を更新。
  • デーモンの PID ファイルに書き込むための、新しい pidfile(3) 呼び出しを追加。
  • ssh(1) が、ユーザの設定を最初に読むように変更。
  • pf.conf(5) で、アンダースコアを含むマクロ名を許容。
  • gzip(1) での /tmp の取り扱いをオーバーホール。
  • rwho(1) の文字列の取り扱いを改善。
  • pf(4) の icmp(4) パケットの取り扱いを修正。
  • ssh(1) の圧縮でのメモリリークを修正。
  • アタッチ後の usb(4) キーボードのディスプレイへの接続をサポート。
  • ip6(4) サブシステムで、パケットを廃棄した際の mbuf の解放を適切に。
  • cu(1) のデフォルトを 9600 ボーに。
  • malloc(3) のひとつ違い (off-by-one) エラーおよびバッファオーバーフローを含む、多数の getcap(3) の修正。
  • 新しい grep(1) のインポート。
  • popa3d(8) を、単独、あるいは inetd や tcpd(8) の使用を選択可能とするサポートが追加された 0.4.9.4 に更新。
  • ディスクレスの場合に NFS をブロックしないよう、pflogd(8) の変更作業を開始。
  • uvm(9) のメモリリークを回避。
  • rpc.rstatd(8) からシグナルの競合を削除。
  • local/lib より X11R6/lib を優先するよう、rc(8) で shlib_dirs の検索順序を変更。
  • 共通部分にインラインを使用することで m4(1) を高速化。
  • user(8) ツールをクリーンアップ。
  • syncer 関連の問題を解決し、非常に多くのバッファの性能を改善するための、バッファフラッシュデーモンを実装。
  • m4(1) から、バギーな STATIC メモリの最適化を削除。
  • vi(1) が一時ファイルを生成できない場合に、アボートするよう修正。
  • クロスビルドのサポートを改善。
  • mail(1) で、いくつかのバッファの取り扱いを修正。
  • ユーザランドからカーネルにデータをわたす際に、より多くのデータ長のチェックを追加。
  • pf(4) で traceroute(8) 等が止まってしまわないよう、ip(4) の icmp(4) パケットの内部プロトコルを無視しないように修正。
  • resolv.conf(5) で、insecure1 および insecure2 オプションをサポート。
  • セキュリティのための修正: uuxqt で長いコマンドラインオプションの取り扱いを注意深く行い、また、普段の uucp をノン root で実行するように修正しました。

  • バッファの依存性を B_DEFERRED としてマークし、sync(2) を実行する際にそれらを一度スキップ。

  • maestro(4) オーディオドライバに、ステレオジッター抑制機能を追加。
  • select(2) の FD_SETSIZE の間違った使用を修正。
  • wall(1) の -g フラグを修復。

  • より良いロシア語 calendar(1) のサポート。
  • pf(4) で、インターフェイスのパラメータリストや段階のスキップを実装。
  • rmuser(8) のユーザ名の長さ制限を更新。
  • クロスビルド用にセットする必要のある、すべての環境変数を表示するための新しいソースコンパイル用ターゲット cross-env を追加。
  • USB HID デバイスを扱うためのユーザランドプログラム usbhidctl(1) をインポート。
  • USB HID デバイスを扱うための usb(3) ライブラリ、libusb をインポート。
  • powerpc ベースの機種間でコード共有を可能とするため、powerpc のポート名を macppc に改名。
  • vlan(4) のボーレートを親から継承。
  • 各種の lpd(8) の改善と修正。
  • セキュリティのための修正: lpd のキューファイルの読み込み時のバッファオーバーフローを修正しました。

  • scp(1) および rcp(1) のメモリリークを修正。
  • サーバが自身の名前を指定した場合の dhclient(8) のセグメンテーションフォールトを回避。
  • pfctl(8) のホストパラメータリストの ! オペレータをサポート。
  • 暗号化辞書が同期できていない場合に、ppp(8) で受信した各パケットのリセット要求を送信。
  • pf(4) が (位相変調として知られている) ISN ランダム化をサポートするように変更。
  • argc を int ではなく long としてスタック上に蓄積。
  • rijndael 用のコードを最適化された AES のリファレンスリリースのものに変更。

  • sudo(8) で、ログインクラスが見つからなかった場合に、デフォルトのログインクラスを適用。
  • pf.conf(5) のマクロの拡張をサポート。
  • フロッピー用インストールスクリプトを小さくするためのちょっとしたマジックの作業。
  • pf(4) の初期の tcp(4) のステートウィンドウを正しく設定。
  • tcpdump(8) のバイナリフォーマットで pf(4) のログを記録するデーモンである、pflogd(8) をインポート。
  • セキュリティのための修正: sendmail のデバッグのハンドリングで、境界を取り扱いを修正しました。

  • ppp(8) の iface-alias オプションを修正。
  • bridge(4) デバイスのルールをフラッシュするコードを修正。
  • ppp(8) での ip6(4) のサポート、IPV6CP のおおまかなサポート、および、他の多くの小さな変更。
  • vipw(8) および crontab(1) のファイル変更の検出を拡張。
  • pf(4) に、ルールごとの統計やバイトカウンタを追加。
  • サスペンド後の最初の電源 ON なしに xl(1) の Rx/Tx をリセットしない。
  • pfctl(8) でパラメータリストをサポート。
  • bpf(4) でも使用可能なネットサブシステムで、ipip_input() に確実にインターフェイスがわたされるように。
  • 透過型 ftp プロキシ、ftp-proxy(8) をインポート。
  • "前もって存在する接続のより良い取り扱いを用意し、SYN の乱射するまぬけなプロトコルスタック"を許容する pf(4) の tcp(4) のステートコードを緩和。
  • sparc64 のインポート第一弾。リストするには多過ぎるほどの継続的な開発項目。
  • exec(3) のスタックへのランダムなオフセットを加えるための可能性を追加。
  • siop(4) ドライバが変った癖のあるテーブルに注意を払い、INQUIRY の盗聴および ifdef のトグルを除去するように。
  • 安全な .db ファイルだけを更新するよう pwd_mkdb(8) の新しい -s スイッチ。
  • 拡張シンボル情報を表示するための nm(1) の -e スイッチ。
  • 新しいパスワードをユーザから受け取るまでパスワードファイルのロックを遅延。また、ファイルロックルーチンを使用。
  • hifn(4) ドライバで、PCI のアボートからの回復を試行。
  • openssl(1) と DNSSEC 鍵フォーマットとの間で変換を行う小さなユーティリティ、keyconv をインポート。
  • isakmpd(8) で、信用のある公開 RSA 鍵をファイルとしてサポート。
  • PFKEY v2 の長さと IPv4-in-IPv6 および IPv6-in-IPv4 のフロー中の KAME のメッセージとを修正。
  • 「太陽」の設計者の POP3 デーモン、popa3d(8) をインポート。
  • 価値のある vm_extern.h、vm_inherit.h、vm_map.h および vm_pager.h ファイルを廃止。
  • icmp(4) のクエリ/リプライを参照する icmp(4) のエラーのための pf(4) のサポート。
  • mtree(8) でファイルフラグを許容。
  • ipsec(4) の処理の際に、パケットの中のソースアドレスと SA の中のそれとを比較しないように。
  • rijndael のブロックの整合性を修復。
  • exec(3) の sigaction(2) のシグナルハンドラを共用しない。
  • altq(9) のオプションを base + red + cbq 用の "altq" にマージし、それをデフォルトで有効化。
  • ディレクトリへのシンボリックリンクの追跡を抑止する -h フラグを ln(1) でサポート。
  • 広範囲のデバイスで、多くの新しいタイムアウトを導入。
  • IPSEC オプションの一部となった IPCOMP オプションを除去。
  • 競合状態でも安全となるよう、altqd(8) の signal(3) ハンドラを書き直し。
  • hifn(4) ドライバに RC4 の演算のサポートを追加。
  • mailwrapper(8) の割り当てられていないメモリを free(3) しない。
  • tsleep(9) を ltsleep ラッパーに変更。ltsleep は tsleep(9) よりひとつ多くの引数を取り、それは安全な場合に解除される単純なロック。
  • シェアードライブラリなしで、その機種上での kerberosV 全体のコンパイルが可能に。
  • mktemp(1) で使用される rcs2log の /tmp の競合を回避。
  • 短かい実行のための妥当なバンド幅の指標とするための dd(1) の時間保持のコードを微調整。
  • デフォルトで ipsec(4) の ESP および AH プロトコルを有効化。
  • Kernel crash(8) dump が mvme68k の下で稼働するように。
  • もはや ACQUIRE ポリシー使用時のパケットを廃棄せず、鍵管理の通知時にエラーとなるように。
  • 任意の DNS レコードを受け取るための新しい getrrsetbyname(3) 関数。
  • ssh-keyscan(1) がプロトコルバージョン 2 をサポート。
  • 保守の一貫性の理由から、/var/run ディレクトリに置かれる xdm(1) の PID ファイルのファイル名を xdm-pid から xdm.pid に変更。
  • 新しい hash(3) テーブルを作成するかどうかの決定を遅延するか、open(2) の呼び出し後まで作成しないことで、ファイルロックの問題をクリーンアップ。
  • vlan(4) の変更。IFCAP_VLAN_MTU および IFCAP_VLAN_HWTAGGING の能力を利用。LINK0 および MTU の曖昧さを排除。MTU は親の MTU のスコープの範囲内でのみ変更可能に。
  • ロックされたターミナルの悲劇を避けるため、syslogd(8) でノンブロッキングモードの問題を修復。
  • マニュアルページにおいて .Sh AUTHOR と .Sh EXAMPLE のすべてのケースを複数形に。
  • 現在いくつのページが kmem_map 中にあるのかを報告する、新しい sysctl(3) の nkmempages。
  • pf(4) で、状態なしの tcp(4) の正規化をサポート。
  • ソフトウェアによる x99 のトークン計算機である x99token(1) をインポート。
  • EDNS0 拡張フラグ DNSSEC OK のサポートの resolver(3) ルーチンへの追加。
  • WindowsME や Windows98 で暗号化を有効にした場合に不思議な失敗がないよう、ppp(8) の CHAP SUCCESS パケットの最後で NUL を送信しないように修正。
  • 大容量のメモリを搭載したマシンの kmem_map 空間を保存するのに、kernel_map からの uvm(9) のページバケットを割り当て。
  • ppp(8) で、CHAP81 チャレンジ・レスポンスを送信する際の Windows98 のバグを補正。
  • ssh(1) で、どのスマートカードデバイスを使用するのかを指定するための SmartcardDevice オプションをサポート。
  • Ultra DMA モードをダウングレードする場合に、モードをひとつだけ落とすように。
  • DMA のタイムアウトが発生した場合に、バスマスタ PCIIDE を停止し、チャネルをリセット。
  • 多過ぎる malloc(3) によって、ELF の実行が抑制されないように修正。
  • pf(4) で、ルールのスキップをサポート。
  • lockf(3) で負のロック長を許容し、その仕様に互換となるよう変更。
  • tty の消失を制御する場合の出力の錯乱を防止するため、top(1) で、stdout(4) が EOF を発行した場合にアボート。
  • dohooks(9) とそのファミリを経由するスタートアップおよびシャットダウンフックを実装。
  • セキュリティのための修正: ユーザランドからコピーされたファイルハンドルのサイズをチェックするようにしました。
