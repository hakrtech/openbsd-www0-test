What's New

This is a partial list of new features and systems included in OpenBSD 2.8. For a comprehensive list, see the changelog leading to 2.8.

  • OpenSSH (supporting both the SSH1 and SSH2 protocols) is now at version 2.3.0. An SFTP server is now included for secure file transfers.
  • Since the RSA patent has expired, there is no longer a need to install the SSL libraries. They are included in the system.
  • Extensive changes to the documentation, notably the man pages and the Web FAQ. The manual pages now include useful examples to supplement the explanations.

  • More complete collection and better tested set of "ports".

  • Over 500 pre-built and tested packages.

  • OpenBSD 2.8 will run on Apple iMac, G3, G4, and G4 Cube machines.

  • The system includes the following major components from outside suppliers:

    • XFree86 3.3.6-current
    • gcc 2.95.3
    • perl 5.6.0 plus a few fixes.
    • Apache 1.3.12 (+ patches), Mod_ssl 2.6.2, OpenSSL 0.9.5a, DSO support
    • ipf 3.3.18
    • groff 1.15
    • sendmail 8.10.1
    • lynx 2.8.2 with HTTPS support added
    • sudo 1.6.3p5
    • ncurses 5.2
    • Latest KAME IPv6
    • KTH Kerberos 1.0.2
    • OpenSSH 2.3.0

  • Much improved device driver support

    • Improved hardware crypto support, now including Hifn PowerCrypt and Broadcom Bluesteelnet (uBsec) hardware accelerator boards.
    • Many new Ethernet devices supported, including National Semiconductor DP83815-based adapters, 3Com MiniPCI adapters, 3Com 574-based PCMCIA, and many new CardBus devices (Xircom, Intel 21143, Intel CardBus II).
    • Support for most of the Gigabit Ethernet cards on the market, i.e., SysKonnect, Intel, and Alteon-based.
    • Support for most types of USB devices, including Ethernet, printers, audio, etc. Sync your Handspring Visor, or transfer MP3's to your Rio.
    • Support for 3ware Escalade 3W-5x000 and 3W-6x000 series RAID controllers.
    • New audio support, including the Cirrus Logic CS4280, ForteMedia FM801, and integrated audio chips found on newer VIA and Intel motherboards.
    • Improvements and new support in the PCI IDE subsystem.

  • Many improvements for security and reliability (look for the red print in the complete changelog).

  • Many other new features, including:

    • Console mouse support on the i386 platform.
    • Support for the AES encryption standard.
    • Import of tcfs, a file content crypto file system.
    • New sftp-server in OpenSSH.

How to install

Berikut adalah petunjuk yang anda dapatkan dalam selembar kertas jika anda membeli serangkai CDROM dari http://www.OpenBSD.org/id/orders.html, daripada mencoba melakukan bentuk instalasi alternatif lain. Petunjuk untuk melakukan instalasi lewat ftp (atau bentuk instalasi lain) adalah saling mirip, petunjuk yang ada pada CDROM akan dapat anda rasakan betapa lebih mudah jika anda membeli sebuah CDROM.

Mohon mengacu pada berkas-berkas berikut pada dua CDROM untuk rincian lebih jauh tentang bagaimana memasang OpenBSD 2.8 di mesin anda:

  • CD1:2.8/i386/INSTALL.i386
  • CD1:2.8/powerpc/INSTALL.powerpc
  • CD1:2.8/vax/INSTALL.vax

  • CD2:2.8/sparc/INSTALL.sparc
  • CD2:2.8/mvme68k/INSTALL.mvme68k
  • CD2:2.8/amiga/INSTALL.amiga
  • CD2:2.8/mac68k/INSTALL.mac68k
  • CD2:2.8/hp300/INSTALL.hp300
  • CD2:2.8/sun3/INSTALL.sun3

  • Sebuah rilis pmax tersedia juga di situs ftp, tidak pada CD.

    • Petunjuk penginstalan cepat untuk orang yang terbiasa dengan OpenBSD, dan penggunaan perintah baru "disklabel -E". Jika anda terlalu bingung ketika menginstal OpenBSD, baca berkas INSTALL.* yang sesuai pada daftar di atas!

    i386:

    Play with your BIOS options, and see if you can enable booting off a CD; try using CD1. If not, write CD1:2.8/i386/floppy28.fs to a floppy, then boot that. If you are mixing OpenBSD with another operating system, you will surely need to read the INSTALL.i386 document.

    To make a floppy under MS-DOS, use /2.8/tools/rawrite.exe. Under Unix, use "dd if=<file> of=/dev/<device> bs=32k" (where device could be "floppy" or "rfd0c" or "rfd0a"). Use properly formatted perfect floppies with NO BAD BLOCKS or you will lose.

    If you experience any boot hangs on PCI devices, or any panic early in boot that seems related to pcibios(4), you might have to disable the pcibios device as a workaround. At the boot> prompt, type "boot -c" to enter User Kernel Configuration. At the UKC> prompt, enter "disable pcibios" then "quit". See config(8) after a successful boot for instructions on how to re-write your kernel to disable pcibios(4) permanently.

    sparc:

    To boot off CD2, type "boot cdrom 2.8/sparc/bsd.rd", or "b sd(0,6,0)2.8/sparc/bsd.rd" depending on your ROM version. Alternatively, write CD2:2.8/sparc/floppy28.fs to a floppy and boot it using "boot floppy" or "boot fd()" depending on your ROM version.

    amiga:

    Buat partisi BSD berdasar pada bagian persiapan di INSTALL.amiga. Tautkan CD2 dalam AmigaOS sebagai divais CD0: Lalu, eksekusi perintah CLI berikut: "CD0:2.8/amiga/utils/loadbsd CD0:2.8/amiga/bsd.rd".

    hp300:

    Anda bisa boot lewat jaringan dengan mengikuti petunjuk pada INSTALL.hp300

    mac68k:

    Boot MacOS secara normal dan partisi harddisk anda dengan konfigurasi A/UX yang sesuai. Lalu, mekarkan peralatan Macside dari CD1:2.8/mac68k/utils ke harddisk anda. Jalankan Mkfs untuk membuat sistem berkas anda pada partisi A/UX yang baru saja anda buat. Lalu, gunakan Penginstal BSD/Mac68k untuk menyalin semua isi CD1:2.8/mac68k/ ke partisi anda. Akhirnya, anda siap mengkonfigurasi Booter BSD/Mac68k dengan tempat kernel dan anda bisa boot sistem tersebut.

    sun3:

    Get the release via ftp. Then, you can either setup a diskless boot or create an installation tape, as described in INSTALL.sun3.

    Notes about the source code:

    src.tar.gz contains a source archive starting at /usr/src. This file contains everything you need except for the kernel sources, which are in a separate archive. To extract: 

    # mkdir -p /usr/src
# cd /usr/src
# tar xvfz /tmp/src.tar.gz

    srcsys.tar.gz contains a source archive starting at /usr/src/sys. This file contains all the kernel sources you need to rebuild kernels. To extract: 

    # mkdir -p /usr/src/sys
# cd /usr/src
# tar xvfz /tmp/srcsys.tar.gz

    Both of these trees are a regular CVS checkout. Using these trees it is possible to get a head-start on using the anoncvs servers as described at http://www.OpenBSD.org/anoncvs.html. Using these files results in a much faster initial CVS update than you could expect from a fresh checkout of the full OpenBSD source tree.

    Struktur Ports

    Sebuah struktur port termampat juga tersedia. Untuk memekarkan: 

    # cd /usr
# tar xvfz /tmp/ports.tar.gz
# cd ports
# ls
...

    Subdirektori ports/ merupakan pemeriksaan pada struktur port OpenBSD. Baca http://www.OpenBSD.org/id/ports.html jika anda tidak tahu sama sekali tentang port pada saat ini. Teks ini bukan manual penggunaan port. Tetapi, lebih sebagai rangkaian catatan yang membawa pengguna menuju pada sistem port OpenBSD.

    Tentu saja, sistem port OpenBSD tidak lengkap. Hal ini karena integrasi penuh port ke lingkungan OpenBSD adalah proyek sangat dini pada rilis ini. Kami percaya port yang tersedia di sini adalah stabil, tetapi lebih penting menyadari bahwa port akan terus berkembang pada fungsional yang besar di masa depan.

    Seperti kami katakan, port akan berkembang cepat di masa depan. Direktori ports/ mewakili sebuah pemeriksaan CVS (lihat halaman manual cvs(1) jika anda terbiasa dengan CVS) dari port kami. Sebagaimana dengan struktur lengkap kode sumber kami, kami akan segera membuat port tersedia melalui anoncvs. Sehingga, untuk menjaga tetap terkini dengannya, anda harus membuat struktur port berada dalam modus baca-tulis dan perbarui struktur port dengan perintah seperti berikut: 

    # cd [portsdir]/; cvs -d anoncvsserver.openbsd.org:/cvs update -Pd

    [Tentu saja, anda harus mengganti direktori setempat dan nama server dengan tempat port anda dan server anoncvs terdekat.]

    Sekali lagi, penting untuk melihat petunjuk khusus karena hal ini merupakan layanan sangat baru dan belum dikerjakan sepenuhnya.

    Akhirnya, meskipun port masih baru, bantuan tidak akan pernah jauh. Jika anda tertarik melihat port ditambahkan, ingin membantu kami, atau hanya ingin mengetahui lebih,daftar mailing ports@openbsd.org adalah sebuah tempat yang bagus untuk mengetahui semuanya.
