Apakah CVSync itu?

CVSync adalah sebuah peranti lunak untuk mendistribusi dan update struktur kode sumber dari sebuah gudang cvs(1) utama di server lain. Kode sumber OpenBSD dikelola dalam sebuah gudang CVS pada mesin pengembangan terpusat di Canada. Dengan CVSync, pengguna OpenBSD dapat dengan mudah menjaga struktur kode sumber lokalnya tetap terbarukan.

CVSync menggunakan jenis update tarikan terpanggil (so-called pull). Dalam model tarikan, setiap klien meminta ke server agar diberi update, jika dan ketika diinginkan. Server menunggu secara pasif akan permintaan update dari kliennya. Sehingga seluruh update dikendalikan oleh klien. Server tidak pernah mengirimkan update yang tak diminta. Pengguna harus juga menjalankan aplikasi CVSync klien secara manual agar bisa melakukan update, atau harus di atur sebuah tugas dengan cron(8) agar berjalan secara otomatis seperti kebutuhan dasar umumnya.

Kata CVSync, penggunaan beberapa huruf besar tidak bermaksud apa-apa, merujuk pada paket peranti lunak secara utuh. Bagian utamanya adalah cvsync klien yang berjalan pada setiap mesin pengguna, dan cvsyncd yang berjalan di setiap situs miror CVSync OpenBSD.

CVSync dimaksudkan lebih cepat maupun lebih fleksibel daripada sup(1) dan and CVSup.

Proyek OpenBSD saat ini mempunyai lima gudang utama kode sumber:

  • src - Menampung seluruh kode sumber untuk Sistem Operasi OpenBSD.
  • ports - Menampung Port OpenBSD.
  • www - Menampung seluruh halaman web OpenBSD (termasuk halaman yang anda baca ini).
  • X11 - Menampung adaptasi OpenBSD pada proyek peranti lunak XFree86-3 .
    • Disebut disini hanya untuk kepentingan historis, kebanyakan pengguna tidak punya alasan untuk mengunakannya, karena sudah tidak dipakai lagi.
  • XF4 - Rumah untuk penyesuaian proyek peranti lunak X OpenBSD yang aktif.

Apakah CVS itu?

CVS adalah sistem kendali kode sumber yang digunakan untuk mengelola struktur kode sumber OpenBSD. Dijelaskan lebih lanjut di sini. Menjadi perhatian bahwa CVS dan CVSync adalah dua program yang berbeda. Meskipun CVS dapat digunakan untuk akses file secara jarak jauh, ia tidak dioptimalkan untuk distribusi struktur kode sumber secara penuh melalui jaringan komputer, dan CVSync dapat dengan mudah sebuah perintah pada skala ini lebih efisien untuk tugas tersebut.

Persiapan Menggunakan CVSync

Jika anda hanya ingin menjalankan CVSync saja, disarankan untuk download paket cvync dari situs miror ftp daripada membangunnya dari port net/cvsync .

Untuk keterangan lanjutan tentang CVSync, lihat homepage proyek ini.

Untuk melakukan miror gudang OpenBSD dengan CVSync, berkas konfigurasi berikut bisa digunakan: 

	config {
	    hostname anoncvs.de.openbsd.org
	    # If your network link is a T1 or faster, comment out the following line.
	    compress
	    collection {
		name openbsd release rcs
		prefix /cvs
		umask 002
	    }
	}

Hal tersebut akan mengarahkan cvsync membarui seluruh distribusi OpenBSD dari anoncvs.de.openbsd.org dengan sebuah umask(2) yang membolehkan perijinan hak tulis oleh grup tertentu pada gudang di mesin lokal anda.

Anggap berkas ini disimpan sebagaicvs-syncfile, perintah berikut bisa digunakan untuk menjalankan cvsync 

        cvsync -c cvs-syncfile

Menggunakan CVS untuk Update Struktur Kode Sumber Anda

Sekarang menjadi mudah untuk memeriksa apapun dari gudang yang menggunakan CVSync. Untuk shell Korn/Bourn: 

        # cd /usr
        # CVSROOT=/cvs cvs checkout src

Untuk csh dan turunannya: 

        # cd /usr
        # setenv CVSROOT /cvs
        # cvs checkout src

Hal di atas tersebut akan memeriksa kode sumber yang paling terkini. Banyak dari anda akan hanya memeriksa kode sumber rilis, khususnya jika anda melakukan patch pada sistem anda. Untuk memeriksa kode sumber rilis harus anda tentukan sebuah penanda bersama perintah anda. Sebagai contoh: 

        # cd /usr
        # cvs checkout -rOPENBSD_3_5 src

Or OPENBSD_3_4 untuk 3.4, dan lainnya yang sesuai.

Ukuran gudang keseluruhan saat ini terus berkembang pada laju sekitar 400MB per tahun.

CATATAN PENTING: Terdapat beberapa masalah terkait dengan peranti lunak kriptografi yang setiap orang harus perhatikan benar-benar:

  • Kode sumber OpenBSD berasal dari Kanada. Seturut penelitian oleh seorang Kanada dan sebagaimana penjelasan dalam daftar Kendali Ekspor Kanada, merupakan hal yang legal untuk mengekspor peranti lunak kripto dari Kanada ke seluruh dunia.

  • Namun, jika anda berada di luar Amerika Serikat atau Kanada, anda jangan mengambil bagian kriptografi kode sumber OpenBSD dari sebuah server CVSync yang terletak di Amerika Serikat. Berkas yang dimaksud di sini adalah...

    • src/kerberosIV/*
    • src/kerberosV/*
    • src/lib/libdes/*
    • src/lib/libc/crypt/crypt.c
    • src/lib/libc/crypt/morecrypt.c
    • src/sys/crypto
    • src/sys/netinet
    • src/usr.sbin/afs/src/rxkad/*
    • XF4/xc-mit/lib/Xdmcp/Wraphelp.c
    • XF4/xc-old/lib/Xdmcp/Wraphelp.c
    • XF4/xc/lib/Xdmcp/Wraphelp.c

    Karena menurut daftar amunisi(baca: senjata, dalam arti luas) dari lembaga ITAR di Amerika Serikat, peranti lunak kripto hanya bisa diekspor ke Kanada saja dari Amerika Serikat.
