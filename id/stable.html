Apakah ranting perbaikan itu?

Bermula dari 2.7, OpenBSD menyediakan struktur kode sumber yang berisi perbaikan dan pembetulan penting (contohnya.yaitu pada kisah masalah ditambah lainnya yang tampak jelas dan sederhana, tetapi tidak dinaggap sebagai bagian kisah masalah) dan menyediakannya lewat CVS sebagai tambahan pada kode sumber terkini. Maka, pengguna bisa memilih 3 opsi :

  • Bertahan dengan rilis terakhir kami dan melakukan perbaikan sendiri.
  • Menggunakan ranting perbaikan yang mempunyai banyak penerapan perbaikan.
  • menggunakan kode sumber terkini untuk semua fiotur terakhir.

Perubahan log per hari menjelaskan perbaikan rilis akhir yang telah dimasukkan dalam ranting perbaikan. Sesuai prinsip umum, semual masukan Kisah masalah akan disatukan pada ranting perbaikan dalam 48 jam ketika sebuah masalah di umumkan. Rilis akhir perbaikan lainnya mungkin juga disatukan, berdasar pada sejumlah kondisi:

  • Perbaikan harus sederhana, singkatt, dan terbukti 100% benar.
  • Masukan masalah yang dianggap sebagai bug mungkin memepengaruhi banyak orang. Perbaikan lain mungkin disatukan pada ranting perbaikan jika mempengaruhi beberapa orang secara drastis.
  • Mohon tidak meminta subsistem besar atau perbaikan tertentu untuk disatukan. Mengelola struktur perbaikan memebutuhkan banyak usaha yang bisa lebih menghemat waktu dalam pembuatan rilis kami berikutnya yang lebih baik.
  • Kecuali dari 2 hal di atas, versi rilis OpenSSH akan disatukan dalam ranting perbaikan.

Mendapatkan ranting stabil

Untuk mendapatkan ranting perbaikan pada rilis OpenBSD tertentu, anda dapat mengkinikan puncak struktur kode sumber yang telah ada (dari FTP atau CD) atau anda bisa mendapatkan struktur kode sumber segar dari sebuah server AnonCVS. Petunjuk untuk mendapatkan ranting perbaikan dan tetap terkini dijelaskan pada bagian Mulai pada dokumentasi AnonCVS. Menjadi catatan bahwa ranting perbaikan tidak membantu untuk membarui satu rilis OpenBSD ke rilis lainnya, misal dari 2.7 ke 2.8. Hanya bermaksud membantu tetap terkini dengan perbaikan dalam satu rilis tertentu saja. JIka anda mencoba membarui satu rilis ke rilis lainnya lewat kode sumber, mohon lihat faqmini-pembaruan. Juga, anda tidak bisa kembali ke belakang, dari -current kembali ke -stable, karena masalah versi pustaka.

Membangun dari ranting perbaikan OpenBSD

Setelah anda mendapatkan struktur kode sumber lewat anoncvs, anda harus membangun ulang sistem. Tahapan untuk melakukannya sebagai berikut:
  • Buat ulang kernel
  • Reboot dengan kernel baru
  • Buat ulang binari

Membuat ulang kernel

Untuk membuat ulang kernel stabil dasar: 

    # cd /usr/src/sys/arch/i386/conf
# /usr/sbin/config GENERIC
# cd /usr/src/sys/arch/i386/compile/GENERIC
# make clean && make depend && make

Ganti i386 dengan arsitektur anda, misal sparc, alpha, dll.

Reboot dengan kernel baru

Untuk reboot dengan kernel yang baru saja dikompail: 

    # cd /usr/src/sys/arch/i386/compile/GENERIC
# cp /bsd /bsd.old          (Menyinmpan sebuah salinan lama kernel anda)
# cp bsd /bsd               (Salin kernel baru pada tempatnya)
# reboot

Seperti hal di atas, ganti arsitektur mesin anda daripada i386. Jika sistem anda mengalami masalah boot dengan kernel baru, mudahnya anda bisa kembali dan reboot dengan kernel lama, yang telah kita beri nama bsd.old.

Membuat ulang binari

Untuk membuat ulang binari sistem: 

    # cd /usr/src
# rm -r /usr/obj/*
# make obj && make build

Hal ini akan membutuhkan beberapa waktu sampai selesai...
