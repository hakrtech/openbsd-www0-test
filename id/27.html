Berkas distribusi yang digunakan untuk membangun paket aplikasi tambahan dari berkas ports.tar.gz tidak termasuk dalam CDROM karena keterbatasan kapasitas ruang.

Apa yang Baru

This is a partial list of new features and systems included in OpenBSD 2.7. For a comprehensive list, see the changelog leading to 2.7.

  • OpenSSH now supports both the SSH1 and SSH2 protocols. The SSH2 protocol is slowly gaining acceptance. Up until recently, SSH2 was just available in various commercial offerings. The benefit with SSH2 is that it avoids the RSA cipher and uses DSA instead, which is freely reusable. The downside is that DSA is not nearly as fast on old slow machines. But since OpenSSH still supports SSH1 protocol (if the RSA libraries are installed) our users get the best of both worlds.

  • The clever trick used in 2.7 remains, allowing us to distribute the same CD-ROM worldwide containing full strength crypto without violating the RSA patent in the USA. The next OpenBSD release will ship with RSA support in the operating system by default, since the patent expires on September 21, 2000.

  • Extensive changes to the documentation, notably the man pages and the Web FAQ. The manual pages now include useful examples to supplement the explanations.

  • More complete collection and better tested set of "ports".

  • Over 500 pre-built and tested packages.

  • The system includes the following major components from outside suppliers:

    • XFree86 3.3.6
    • gcc 2.95.2
    • perl 5.6.0 plus a few fixes.
    • Apache 1.3.12 + Mod_ssl 2.7.2 + OpenSSL 0.9.5a, DSO
    • ipf 3.3.14
    • groff 1.15
    • sendmail 8.10.1
    • lynx 2.8.2 with HTTPS support added
    • sudo 1.6.3
    • Recent ncurses release
    • Latest KAME IPv6 as of mid-May 2000.
    • KTH Kerberos 1.0
    • OpenSSH 2.1

  • Much improved device driver support

    • Support for crypto hardware to accelerate IPSEC performance, ie. PowerCrypt.
    • Support for all new Adaptec, NCR, Qlogic, and Advansys scsi controllers.
    • Many new ethernet devices supported, including SiS900/7016, Adaptec Starfire, 21143, Sundance ST201 (DEC DFE-550TX), etc.
    • New USB ethernet devices supported, including ADMtek USB, CATC USB, and Kawasaki USB.
    • A good start at cardbus support.
    • Significant improvements to pcmcia.
    • Support for many more wireless networking cards, ie. WaveLAN, Aironet, Aviator, etc.
    • Many improvements in USB support.
    • Hypersparc support for the sparc port.
    • Support for most of the gigabit ethernet cards on the market, ie. SysKonnect, Intel, and Alteon.
    • Support for the ICP Vortex RAID controller.
    • Support for large PC's using multiple PCI busses based on the Intel 451 or RCC ServerWorks chipsets.
    • New audio drivers, including Neomagic, C-Media CMI8x38, and USB audio.
    • Improvements in PCI IDE subsystem, including new support for AMD 756.

  • Many improvements for security and reliability (look for the red print in the complete changelog).

  • Many other new features, including:

    • Support for encrypting swap space.
    • Improved Linux emulation
    • Support for large FAT32 filesystems.
    • Significant improvements to the network bridging code.
    • Improved pthreads support.
    • Support for vlan networking.

Bagaimana menginstal

Berikut adalah petunjuk yang anda dapatkan dalam selembar kertas jika anda membeli serangkai CDROM dari http://www.OpenBSD.org/id/orders.html, daripada mencoba melakukan bentuk instalasi alternatif lain. Petunjuk untuk melakukan instalasi lewat ftp (atau bentuk instalasi lain) adalah saling mirip, petunjuk yang ada pada CDROM akan dapat anda rasakan betapa lebih mudah jika anda membeli sebuah CDROM.

Mohon mengacu pada berkas-berkas berikut pada dua CDROM untuk rincian lebih jauh tentang bagaimana memasang OpenBSD 2.7 di mesin anda:

  • CD1:2.7/i386/INSTALL.i386

  • CD2:2.7/sparc/INSTALL.sparc
  • CD2:2.7/mvme68k/INSTALL.mvme68k
  • CD2:2.7/amiga/INSTALL.amiga
  • CD2:2.7/mac68k/INSTALL.mac68k
  • CD2:2.7/hp300/INSTALL.hp300

  • Sebuah rilis pmax&sun3 tersedia juga di situs ftp, tidak pada CD.

    • Petunjuk penginstalan cepat untuk orang yang terbiasa dengan OpenBSD, dan penggunaan perintah baru "disklabel -E". Jika anda terlalu bingung ketika menginstal OpenBSD, baca berkas INSTALL.* yang sesuai pada daftar di atas!

    i386:

    Atur opsi BIOS anda, dan lihat apakah anda bisa boot dengan sebuah CD; coba gunakan CD1. Jika tidak, tuliskan CD1:2.7/i386/floppy21.fs ke sebuah disket, lalu boot dengan disket itu. Jika anda memasang OpenBSD bersama sistem operasi lain, anda dipastikan harus perlu membaca dokumen INSTALL.i386.

    Untuk membuat sebuah disket boot dengan MS-DOS, gunakan /2.7/tools/rawrite.exe. Dengan Unix, gunakan "dd if=<file> of=/dev/<device> bs=32k" (dengan divais bisa "floppy" atau "rfd0c" atau "rfd0a"). Gunakan disket bagus yang diformat benar dan TANPA BLOK RUSAK atau anda akan gagal.

    sparc:

    Untuk boot dengan CD1, tulis "boot cdrom 2.7/sparc/bsd.rd", atau "b sd(0,6,0)2.7/sparc/bsd.rd" bergantung versi ROM anda. Cara lain, tuliskan CD1:2.7/sparc/floppy21.fs ke sebuah disket dan boot dengannya memakai "boot fd()" atau "boot floppy" bergantung versi ROM anda. Akhirnya, cara ketiga adalah menuliskan CD1:2.7/sparc/kc.fs dan CD1:2.7/sparc/inst.fs ke dua disket terpisah. Lalu masukkan "kc.fs", dan boot seperti penjelasan di atas. Segera setelah drive disket mengeluarkan disketnya, masukkan "inst.fs". Jawab sejumlah pertanyaan. Boot ulang dengan disket "kc.fs". Pada saat tersebut, ketika disket dikeluarkan langsung masukkan "kc.fs" lagi dan jawab serangkaian pertanyaan yang berbeda.

    amiga:

    Buat partisi BSD berdasar pada bagian persiapan di INSTALL.amiga. Tautkan CD2 dalam AmigaOS sebagai divais CD0: Lalu, eksekusi perintah CLI berikut: "CD0:2.7/amiga/utils/loadbsd CD0:2.7/amiga/bsd.rd".

    hp300:

    Anda bisa boot lewat jaringan dengan mengikuti petunjuk pada INSTALL.hp300

    mac68k:

    Boot MacOS secara normal dan partisi harddisk anda dengan konfigurasi A/UX yang sesuai. Lalu, mekarkan peralatan Macside dari CD1:2.7/mac68k/utils ke harddisk anda. Jalankan Mkfs untuk membuat sistem berkas anda pada partisi A/UX yang baru saja anda buat. Lalu, gunakan Penginstal BSD/Mac68k untuk menyalin semua isi CD1:2.7/mac68k/ ke partisi anda. Akhirnya, anda siap mengkonfigurasi Booter BSD/Mac68k dengan tempat kernel dan anda bisa boot sistem tersebut.

    sun3:

    Dapatkan rilisnya lewat ftp. lalu, anda bisa membuat boot tanpa disk ataupun membuat sebuah pita instalasi, seperti dijelaskan pada INSTALL.sun3.

    Catatan tentang kode sumber:

    src.tar.gz berisi sebuah kode sumber termampat yang bermula di /usr/src. Berkas ini berisi setiap hal yang anda perlukan kecuali kode sumber kernel, yang terdapat dalam berkas termampat terpisah. Untuk memekarkan: 

    # mkdir -p /usr/src
# cd /usr/src
# tar xvfz /tmp/src.tar.gz

    srcsys.tar.gz berisi kode sumber termampat yang bermula di /usr/src/sys. Berkas ini berisi semua kode sumber kernel yang anda perlukan untuk membuat ulang kernel. Untuk memekarkan: 

    # mkdir -p /usr/src/sys
# cd /usr/src
# tar xvfz /tmp/srcsys.tar.gz

    Kedua struktur ini adalah pemeriksaan pada susunan CVS. Dengan struktur ini dimungkinkan mendapatkan permulaan dalam penggunaan server anoncvs seperti dijelaskan pada http://www.OpenBSD.org/id/anoncvs.html. Dengan berkas-berkas ini membuat inisialisasi pembaruan CVS akan lebih cepat daripada waktu yang anda harapkan saat melakukan pemeriksaan dari struktur lokal kosong terhadap struktur penuh kode sumber OpenBSD.

    Struktur Ports

    Sebuah struktur port termampat juga tersedia. Untuk memekarkan: 

    # cd /usr
# tar xvfz /tmp/ports.tar.gz
# cd ports
# ls
...

    Subdirektori ports/ merupakan pemeriksaan pada struktur port OpenBSD. Baca http://www.OpenBSD.org/id/ports.html jika anda tidak tahu sama sekali tentang port pada saat ini. Teks ini bukan manual penggunaan port. Tetapi, lebih sebagai rangkaian catatan yang membawa pengguna menuju pada sistem port OpenBSD.

    Tentu saja, sistem port OpenBSD tidak lengkap. Hal ini karena integrasi penuh port ke lingkungan OpenBSD adalah proyek sangat dini pada rilis ini. Kami percaya port yang tersedia di sini adalah stabil, tetapi lebih penting menyadari bahwa port akan terus berkembang pada fungsional yang besar di masa depan.

    Seperti kami katakan, port akan berkembang cepat di masa depan. Direktori ports/ mewakili sebuah pemeriksaan CVS (lihat halaman manual cvs(1) jika anda terbiasa dengan CVS) dari port kami. Sebagaimana dengan struktur lengkap kode sumber kami, kami akan segera membuat port tersedia melalui anoncvs. Sehingga, untuk menjaga tetap terkini dengannya, anda harus membuat struktur port berada dalam modus baca-tulis dan perbarui struktur port dengan perintah seperti berikut: 

    # cd [portsdir]/; cvs -d anoncvsserver.openbsd.org:/cvs update -Pd

    [Tentu saja, anda harus mengganti direktori setempat dan nama server dengan tempat port anda dan server anoncvs terdekat.]

    Sekali lagi, penting untuk melihat petunjuk khusus karena hal ini merupakan layanan sangat baru dan belum dikerjakan sepenuhnya.

    Akhirnya, meskipun port masih baru, bantuan tidak akan pernah jauh. Jika anda tertarik melihat port ditambahkan, ingin membantu kami, atau hanya ingin mengetahui lebih,daftar mailing ports@openbsd.org adalah sebuah tempat yang bagus untuk mengetahui semuanya.
