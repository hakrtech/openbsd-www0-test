Untuk mendapatkan berkas-berkas pada rilis ini:
  • Pesanlah sebuah CDROM dari sistem pemesanan kami.
  • Lihat keterangan daftar mesin miror pada halaman FTP
  • Masuk ke direktori pub/OpenBSD/3.1/ pada salah satu situs miror
  • Segera baca bagian lain dari dokumen ini.
  • Lihat daftar masalah dan penangannya pada halaman kisah masalah 3.1
  • Lihat perubahan log terinci antara rilis 3.0 dan 3.1



Catatan: Semua hakcipta yang dipakai dan anggota tim yang ikutserta dapat ditemukan dalam berkas kode sumber yang bisa digunakan dan didapat dari berkas src.tar.gz, srcsys.tar.gz, XF4.tar.gz, atau pada berkas yang didapat melalui ports.tar.gz. Berkas distribusi yang digunakan untuk membangun paket aplikasi tambahan dari berkas ports.tar.gz tidak termasuk dalam CDROM karena keterbatasan kapasitas ruang.

What's New

This is a partial list of new features and systems included in OpenBSD 3.1. For a comprehensive list, see the changelog leading to 3.1.

  • OpenSSH (supporting both the SSH1 and SSH2 protocols) is now at version 3.1. Secure file transfers are encouraged using the greatly enhanced SFTP subsystem which comes both with an SFTP server and client.

  • As usual, improvements to the documentation, notably the man pages and the Web FAQ. A larger part of the website is now available in several languages.

  • More complete collection and better tested set of "ports".

  • Over 1000 pre-built and tested packages.

  • Greatly improved hardware support in the OpenBSD/sparc64 port, and the addition of X11 support.

  • Numerous performance improvements to the OpenBSD/macppc port, as well as accelerated X11 servers for some models.

  • A lot of enhancements to our new packet filter, pf, including performance improvements, as well as the ability to filter other protocols than the usual tcp, udp and icmp, such as esp.

  • An utility to achieve per-user pf rules changes, typically intended for gateways, authpf.

  • The in-kernel software raid (RAIDframe) can now automatically detect and configure its raid devices at boot. Taking advantage of this, the system can have its root partition directly mounted from an auto-detected raid partition.

  • Wavelan bridging is now possible on Prism-II based cards.

  • The system includes the following major components from outside suppliers:

    • XFree86 4.2.0 (and i386 contains 3.3.X servers also, thus providing support for all chipsets)
    • gcc 2.95.3 (+ patches)
    • perl 5.6.1 (+ patches)
    • Apache 1.3.14, mod_ssl 2.8.8, OpenSSL 0.9.6b (+ patches), DSO support
    • groff 1.15
    • sendmail 8.12.2
    • lynx 2.8.2rel.1 with HTTPS support added
    • sudo 1.6.5p2
    • ncurses 5.2
    • Latest KAME IPv6
    • KTH Kerberos 1.0.8
    • Heimdal 0.4e (+ patches)
    • OpenSSH 3.1

  • Many improvements for security and reliability (look for the red print in the complete changelog).

How to install

Berikut adalah petunjuk yang anda dapatkan dalam selembar kertas jika anda membeli serangkai CDROM dari http://www.OpenBSD.org/id/orders.html, daripada mencoba melakukan bentuk instalasi alternatif lain. Petunjuk untuk melakukan instalasi lewat ftp (atau bentuk instalasi lain) adalah saling mirip, petunjuk yang ada pada CDROM akan dapat anda rasakan betapa lebih mudah jika anda membeli sebuah CDROM.

Mohon mengacu pada berkas-berkas berikut pada tiga CDROM untuk rincian lebih jauh tentang bagaimana memasang OpenBSD 3.1 di mesin anda:

  • CD1:3.1/i386/INSTALL.i386
  • CD1:3.1/alpha/INSTALL.alpha

  • CD2:3.1/macppc/INSTALL.macppc
  • CD2:3.1/vax/INSTALL.vax

  • CD3:3.1/sparc/INSTALL.sparc
  • CD3:3.1/sparc64/INSTALL.sparc64

    • Only available via FTP:

  • CD4:3.1/mvme68k/INSTALL.mvme68k
  • CD4:3.1/amiga/INSTALL.amiga
  • CD4:3.1/mac68k/INSTALL.mac68k
  • CD4:3.1/hp300/INSTALL.hp300

Petunjuk penginstalan cepat untuk orang yang terbiasa dengan OpenBSD, dan penggunaan perintah baru "disklabel -E". Jika anda terlalu bingung ketika menginstal OpenBSD, baca berkas INSTALL.* yang sesuai pada daftar di atas!

OpenBSD/i386:

    Ubah opsi BIOS anda untuk mengaktifkan booting melalui sebuah CD. Rilis OpenBSD/i386 berada pada CD1. Jika BIOS anda tidak mendukung booting dari sebuah CD, anda perlu membuat disket booting untuk menginstall. Untuk membuat sebuah disket booting tuliskan CD1:3.1/i386/floppy32.fs ke sebuah disket dan booting melalui drive disket.

    Pakai CD1:3.1/i386/floppyB32.fs untuk dukungan pengendali scsi yang lebih luas, atau CD1:3.1/i386/floppyC32.fs untuk dukungan yang lebih baik pada laptop.

    Jika anda merencanakan booting ganda OpenBSD dengan SO lain, anda perlu untuk membaca dokumen INSTALL.i386.

    Untuk membuat sebuah disket booting dengan MS-DOS, gunakan "rawrite" yang terletak di CD:/3.1/tools/rawrite.exe. Untuk membuat disket booting dengan SO Unix, gunakan dd(1). Berikut ini adalah sebuah contoh penggunaan dd(1), dengan divaisnya harus "floppy", "rfd0c", atau"rfd0a". 

      # dd if=<file> of=/dev/<device> bs=32k

    Pastikan anda menggunakan disket terformat sempurna yang sesuai dengan TANPA BLOK RUSAK atau instalasi anda akan menemui kegagalan seperti kebanyakan. Untuk keterangan lanjut tentang membuat sebuah disket booting dan menginstall OpenBSD silahkan mengacu pada FAQ4.1.

OpenBSD/alpha:

    Pada prompt SRM prompt, ketikboot -fi 3.1/alpha/bsd.rd dka6 dengan dka6 adalah kependekan untuk nama drive CDROM (bisa anda periksa dengan show dev).

    Jika anda tidak bisa boot dari CDROM, tuliskan CD1:3.1/alpha/floppy32.fs atau CD1:3.1/alpha/floppyB32.fs (bergantung mesin anda) ke sebuah disket dan ketik boot dva0 dengan disket itu. Lihat INSTALL.alpha untuk rincian lebih.

    Pastikan anda menggunakan disket terformat sempurna TANPA BLOK RUSAK atau instalasi anda kelihatannya akan gagal.

OpenBSD/macppc:

    Pasang CD2 dalam drive CDROM anda dan nyalakan mesin anda disertai menekan kunci C key sampai monitor nyala dan layar menampilkan OpenBSD/macppc boot.

    Alternatifnya, pada prompt Open Firmware prompt, enter boot cd:,ofwboot /3.1/macppc/bsd.rd

OpenBSD/hp300:

    Pasang CD4 pada drive CDROM anda dan tekan spacebar selama poweron self-test. Masukkan pilihan boot yang berkorespondensi dengan SYS_CDBOOT.

    Cara lain, anda bisa boot lewat jaringan dengan mengikuti petunjuk pada INSTALL.hp300.

OpenBSD/amiga:

    Buat partisi BSD berdasar pada bagian persiapan di INSTALL.amiga. Tautkan CD2 dalam AmigaOS sebagai divais CD0: Lalu, eksekusi perintah CLI berikut: "CD0:3.1/amiga/utils/loadbsd CD0:3.1/amiga/bsd.rd".

OpenBSD/mac68k:

    Boot MacOS secara normal dan partisi harddisk anda dengan konfigurasi A/UX yang sesuai. Lalu, mekarkan peralatan Macside dari CD4:3.1/mac68k/utils ke harddisk anda. Jalankan Mkfs untuk membuat sistem berkas anda pada partisi A/UX yang baru saja anda buat. Lalu, gunakan Penginstal BSD/Mac68k untuk menyalin semua isi CD4:3.1/mac68k/ ke partisi anda. Akhirnya, anda siap mengkonfigurasi Booter BSD/Mac68k dengan tempat kernel dan anda bisa boot sistem tersebut.

OpenBSD/mvme68k:

    You can create a bootable installation tape or boot over the network.
    The network boot requires a MVME68K BUG version that supports the NIOT and NBO debugger commands. Follow the instructions in INSTALL.mvme68k for more details.

OpenBSD/vax:

    Boot over the network via mopbooting as described in INSTALL.vax.

OpenBSD/sparc:

    The 3.1 release of OpenBSD/sparc is located on CD3. To boot off of this CD you can use one of the two commands listed below, depending on the version of your ROM. 

      > boot cdrom 3.1/sparc/bsd.rd
or
> b sd(0,6,0)3.1/sparc/bsd.rd

    If your sparc does not have a CD drive, you can alternatively boot from floppy. To do so you need to write "CD3:3.1/sparc/floppy31.fs" to a floppy. For more information see FAQ4.1. To boot from the floppy use one of the two commands listed below, depending on the version of your ROM. 

      > boot floppy
or
> boot fd()

    Make sure you use a properly formatted floppy with NO BAD BLOCKS or your install will most likely fail.

    If your sparc doesn't have a floppy drive nor a CD drive, you can either setup a bootable tape, or install via network, as told in the INSTALL.sparc file.

OpenBSD/sparc64:

    Put the CD3 in your CDROM drive and type boot cdrom.

    If this doesn't work, or if you don't have a CDROM drive, you can write CD3:3.1/sparc64/floppy31.fs to a floppy and boot it with boot floppy.
    Make sure you use a properly formatted floppy with NO BAD BLOCKS or your install will most likely fail.

    You can also write CD3:3.1/sparc64/miniroot31.fs to the swap partition on the disk and boot with boot disk:b.

    If nothing works, you can boot over the network as described in INSTALL.sparc64

Notes about the source code:

    src.tar.gz contains a source archive starting at /usr/src. This file contains everything you need except for the kernel sources, which are in a separate archive. To extract: 

      # mkdir -p /usr/src
# cd /usr/src
# tar xvfz /tmp/src.tar.gz

    srcsys.tar.gz contains a source archive starting at /usr/src/sys. This file contains all the kernel sources you need to rebuild kernels. To extract: 

      # mkdir -p /usr/src/sys
# cd /usr/src
# tar xvfz /tmp/srcsys.tar.gz

    Both of these trees are a regular CVS checkout. Using these trees it is possible to get a head-start on using the anoncvs servers as described at http://www.OpenBSD.org/anoncvs.html. Using these files results in a much faster initial CVS update than you could expect from a fresh checkout of the full OpenBSD source tree.

Struktur Port

Sebuah versi termampat struktur port juga disediakan. Untuk memekarkan: 

    # cd /usr
# tar xvfz /tmp/ports.tar.gz
# cd ports

Subdirektori ports/ adalah sebuah pemeriksaan dari struktur port OpenBSD. Silahkan baca halaman ports jika anda tidak tahu apapun tentang port pada saat ini. teks ini bukan manual tentang bagaimana menggunakan port. Terima kasih, ini adalah rangkaian catatan yang bermaksud untuk mengarahkan pengguna pada sistem port OpenBSD.

Tentu saja, sistem port OpenBSD tidak lengkap. Diragukan kondisi port hanya akan seperti yang sudah terjadi. Namun, port berkembang sangat cepat dan menjadi semakin stabil. Hampir semua port yang disediakan bersama rilis ini seharusnya bisa di buat paketnya tanpa masalah pada sebagian besar arsitektur (lebih dari 2000 paket dibuat pada i386 sebagai contoh).

Direktori ports/ mewakili sebuah pemeriksaan CVS (lihat halaman manual untuk cvs(1) jika anda tidak terbiasa dengan CVS) pada port kami. Sama dengan struktur sumber lengkap kami, struktur port juga tersedia melalui anoncvs. Sehingga, dalam tujuan untuk menjaga tetap terkini dengan struktur tersebut, anda harus membuat struktur ports/ berada dalam sebuah media baca-tulis dan perbarui struktur dengan sebuah perintah seperti: 

    # cd [portsdir]/; cvs -d anoncvsserver.openbsd.org:/cvs update -Pd -rOPENBSD_3_1

[Tentu saja, anda harus mengganti direktori lokal dan nama server disini dengan lokasi koleksi port anda dan sebuah server anoncvs terdekat.]

Perhatikan bahwa kebanyakan port tersedia sebagai paket lewat ftp. Paket terbarukan untuk rilis 3.1 akan disediakan jika muncul suatu masalah.

Jika anda tertarik melihat suatu port ditambahkan, ingin menolong kami, atau hanya ingin mengetahui lebih jauh, daftar mailing ports@openbsd.org merupakan tempat yang tepat untuk semuanya.
