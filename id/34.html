Untuk mendapatkan berkas-berkas dari rilis ini:
  • Pesan sebuah CDROM dari sistem pemesanan kami.
  • Lihat keterangan pada Halaman FTP yang memuat daftar mesin miror.
  • Menuju ke direktori pub/OpenBSD/3.4/ pada salah satu situs miror.
  • Segera baca sisa dari dokumen ini.
  • Lihat pada Halaman kisah masalah 3.4 untuk melihat daftar bug dan pemecahannya.
  • Lihat perubahan log terinci antara rilis 3.3 dan 3.4.


Catatan: Seluruh hakcipta dan pendukung yang disebutkan dapat ditemukan pada kode sumber terpakai dalam berkas src.tar.gz, sys.tar.gz, XF4.tar.gz, atau dalam berkas yang didapat melalui ports.tar.gz. Berkas terdistribusi yang digunakan untuk membangun paket dari ports.tar.gz tidak disertakan pada CDROM karena keterbatasan ruang.

Apakah yang Baru

Ini adalah daftar parsial fitur baru dan sistem yang disertakan dalam OpenBSD 3.4. Untuk daftar yang lebih lengkap, lihat log perubahan untuk 3.4.

  • Arsitektur i386 telah dialihkan ke format executable ELF

  • Peningkatan lanjutan untuk W^X, termasuk dukungan untuk arsitektur i386. Binari standar i386 mempunyai pengaturan ulang segmen tereksekusi untuk mendukung isolasi kode terhadap data, dan batasan CS cpu digunakan untuk mendapatkan batasan usaha terbaik dalam eksekusi kode.

  • ld.so(1) pada paltform ELF saat ini memuat pustaka dalam aturan acak untuk resistensi lebih kuat terhadap serangan. Arsitektur i386 juga memetakan pustaka ke suatu alamat acak. Bersama dengan W^X dan ProPolice, perubahan-perubahan ini meningkatkan tingkat kesulitan kesuksesan eksploitasi sebuah kesalahan aplikasi, seperti buffer overflow.

  • Sebuah pemeriksa batasan statis telah ditambahkan pada compiler untuk membentuk pemeriksaan dasar pada fungsi yang menerima buffer dan ukurannya. Pemeriksa ini bertujuan menemukan kesalahan umum dalam penggunaan fungsi pustaka seperti strlcpy(3) atau sscanf(3) tanpa membuat kesalahan positif apapun. Menjalankan pemeriksa ini untuk kode sumber dan struktur port memunculkan ratusan bug nyata, yang telah diperbaiki dan dilaporkan kembali ke pembuat asli aplikasi sejauh mungkin.

  • Pemisahan priviles telah diimplementasikan untuk daemon syslogd(8) , membuatnya makin lebih robust terhadap kesalahan yang mungkin terjadi. Proses anakan yang memantau lalulintas jaringan saat ini berjalan sebagai user biasa sekaligus masuk ke mode chroot sementara proses induk melacak status proses anakan dan melakukan operasi dengan priviles pada kedua sisi.

  • Banyak fungsi string tidak aman telah dihapus dari kernel dan utilitas userland. Audit ini adalah salah satu usaha paling komprehensif yang pernah OpenBSD lakukan, dengan ribuan kemunculan strcpy(3), strcat(3), sprintf(3), dan vsprintf(3) telah diganti dengan yang lebih aman, dengan alternatif seperti strlcpy(3), strlcat(3), snprintf(3), vsnprintf(3), dan asprintf(3).

  • Banyak peningkatan perbaikan pada bug di perlindungan stack ProPolice. Beberapa kode yang menghasilkan bug untuk arsitektur RISC telah diperbaiki.

  • Perlindungan stack ProPolice telah diaktifkan juga pada kernel.

  • Pemisahan priviles telah dimplementasikan pada server X. Priviles dari proses anakan bertanggungjawab pada operasi yang tidak bisa diselesaikan setelah proses utama berpindah ke user tanpa priviles. Hal ini sangat mengurangi potensi kerusakan yang bisa disebabkan oleh client X 'jahat', salah satu kasus bug pada server X.

  • Dukungan emulasi untuk kompatibilitas binari saat ini dikendalikan melalui sysctl(8). Emulasi saat ini deaktif untuk konfigurasi dasar bertujuan membatasi akses pada binari 'jahat', dan dapat diaktifkan pada berkas sysctl.conf(5).

  • Halaman manual telah banyak dirapikan dan diperbaiki.

  • Struktur port saat ini mendukung pembuatan program di dalam systrace(1), mencegah kemungkinan aplikasi merusak sistem saat waktu compile melalui sebuah skrip konfigurasi tersisip trojan atau hal semacam itu

  • Pengingatan simbol pada ld.so(1) mengurangi waktu startup pada aplikasi besar.

  • Banyak perbaikan lisensi, termasuk penghapusan klausa iklan untuk sebagian besar struktur kode sumber.

  • Penggantian perintah-perintah GNU untuk diff(1), diff3(1), grep(1), egrep(1), fgrep(1), zgrep(1), zegrep(1), zfgrep(1), gzip(1), zcat(1), gunzip(1), gzcat(1), zcmp(1), zmore(1), zdiff(1), zforce(1), gzexe(1), dan znew(1) dengan persamaannya dalam lisensi BSD.

  • Tambahan dukungan read-only untuk filesistem NTFS

  • Peningkatan kehandalan pada filesistem berlapis, mengaktifkan NULLFS bekerja kembali.

  • Mengimpor utilitas growfs(8) , membolehkan ekspansi filesistem yang telah ada.

  • Peningkatan pada emulasi linux membuat lebih banyak aplikasi bisa berjalan.

  • Peningkatan signifikan pada pustaka pthreads(3).

  • Menghilangkan banyak penggunaan fd_set statis, selain itu digunakan poll(2) atau alokasi dinamis.

  • ANSIfikasi dan pengaturan ketat prototipe untuk sebuah bagian besar pada struktur kode sumber

  • Dukungan Legacy KerberosIV telah dihapuskan, dan kode dasar KerberosV yang masih ada telah disusun ulang untuk pengelolaan yang lebih mudah.

  • Lebih dari 2400 port, 2200 paket pre-built.

  • Sejumlah besar perbaikan bug, perubahan, dan optimasi pada paket filter pf(4) termasuk:
    • packet tagging (misal, filter pada tag yang ditambahkan oleh bridge berdasar MAC address)
    • stateful TCP normalization (mencegah penghitungan uptime dan pendeteksian NAT)
    • passive OS detection (filter atau redirect koneksi berdasar OS)
    • SYN proxy (melindungi server terhadap serangan SYN flood)
    • adaptive state timeouts (mencegah status tabel berlebih ketika diserang)

  • Peningkatan dukungan peranti keras, termasuk:
    • Kauai ATA controllers (Apple ATA100 wdc) kauaiata(4) mengaktifkan dukungan untuk Powerbook model 12" dan 17".
    • Dukungan untuk kendali register LongRun pada CPU-CPU Transmeta
    • Banyak perbaikan pada kendali SCSI aac(4), ahc(4), osiop(4), dan siop(4).
    • Kendali peranti keras monitoring baru it(4), lm(4), dan viaenv(4).
    • Kendali baru untuk safe(4) akselerator kripto SafeNet.
    • Kendali baru mtd(4) untuk kartu jaringan Myson Technologies.
    • Dukungan kartu ethernet lebih untuk sk(4), wi(4), fxp(4), dan dc(4).
    • Overhaul masif dan sinkroniasasi dengan NetBSD pada keseluruhan sistem pendukung usb(4).
    • Dukungan baru dan lebih baik untuk berbagai kendali pada pciide(4), termasuk dukungan eksperimental untuk Serial ATA.
    • Kendali baru untuk mendukung mgx(4) dan framebuffer SPARC pninek(4). Kendali vigra(4) juga mendukuing lebih banyak model.
    • Dukungan pcmcia(4) untuk Tadpole SPARCBooks dan SPARCs dengan pcmcia-sbus bridges.
    • Dukungan watchdog untuk elansc(4) dan geodesc(4) sebagaimana digunakan dalam papan Soekris.

  • Pada sistem ini disertakan beberapa komponen utama berikut dari penyedia aplikasi yang lain:
    • XFree86 4.3.0 (+ beberapa perbaikan, dan untuk i386 berisi server 3.3.X juga, untuk menyediakan dukungan bagi semua chipset)
    • Gcc 2.95.3 (+ beberapa perbaikan)
    • Perl 5.8.0 (+ beberapa perbaikan)
    • Apache 1.3.28, mod_ssl 2.8.15, dukungan untuk DSO (+ beberapa perbaikan)
    • OpenSSL 0.9.7b (+ beberapa perbaikan)
    • Groff 1.15
    • Sendmail 8.12.9 (+ perbaikan keamanan parse8.359.2.8)
    • Bind 9.2.2 (+ beberapa perbaikan)
    • Lynx 2.8.4rel.1 dengan dukungan HTTPS dan IPv6 (+ beberapa perbaikan)
    • Sudo 1.6.7p5
    • Ncurses 5.2
    • Latest KAME IPv6
    • Heimdal 0.6rc1 (+ beberapa perbaikan)
    • Arla-current
    • OpenSSH 3.7.1 (sekarang mendukung GSSAPI)

  • Banyak peningkatan dalam hal keamanan dan kehandalan (lihat tanda merah pada log perubahan lengkap).

  • dan masih banyak lainnya.

Bagaimana Cara Install

Berikut adalah petunjuk yang bisa anda dapatkan dalam selembar kertas jika anda membeli serangkai CDROM daripada melakukan sebuah bentuk alternatif instalasi. Petunjuk untuk melakukan instalasi lewat ftp ( atau cara lain ) adalah sangat mirip; petunjuk pada CDROM sangat lengkap sehingga bisa anda lihat betapa lebih mudahnya hal itu jika anda membeli serangkai CDROM.

Mohon merujuk pada berkas-berkas berikut pada tiga CDROM atau miror ftp untuk rincian lanjut tentang bagaimana cara install OpenBSD 3.4 pada mesin anda:

  • CD1:3.4/i386/INSTALL.i386

  • CD2:3.4/macppc/INSTALL.macppc
  • CD2:3.4/vax/INSTALL.vax

  • CD3:3.4/sparc/INSTALL.sparc
  • CD3:3.4/sparc64/INSTALL.sparc64

  • FTP:.../OpenBSD/3.4/alpha/INSTALL.alpha
  • FTP:.../OpenBSD/3.4/hp300/INSTALL.hp300
  • FTP:.../OpenBSD/3.4/hppa/INSTALL.hppa
  • FTP:.../OpenBSD/3.4/mac68k/INSTALL.mac68k
  • FTP:.../OpenBSD/3.4/mvme68k/INSTALL.mvme68k

Keterangan instalasi cepat adalah untuk orang yang telah terbiasa dengan OpenBSD, dan terbiasa dengan penggunaan perintah "disklabel -E". Jika anda sepenuhnya bingung ketika melakukan install OpenBSD, baca berkas INSTALL.* yang sesuai sebagaimana disebutkan diatas !

OpenBSD/i386:

    Atur opsi BIOS anda agar bisa booting dari CD. Rilis OpenBSD/i386 ada pada CD1. Jika BIOS anda tidak mendukung booting dari CD, anda perlu membuat disket boot untuk menginstall. Untuk membuatnya masukkan berkas CD1:3.4/i386/floppy34.fs ke sebuah disket dan lalu booting lewat disket drive.

    Gunakan CD1:3.4/i386/floppyB34.fs untuk dukungan kendali scsi yang lebih bagus , atau CD1:3.4/i386/floppyC34.fs untuk dukungan pada laptop yang lebih bagus.

    Jika anda berencana membuat OpenBSD bisa dual booting dengan OS lain, anda perlu mambaca dokumen INSTALL.i386 yang disertakan.

    Untuk membuat disket boot lewat MS_DOS, gunakan "rawrite" yang berada di CD:/3.4/tools/rawrite.exe. Untuk membuat disket boot dari sebuah OS Unix, gunakan dd(1). Berikut ini adalah contoh penggunaan dd(1) , dengan divais bisa berupa "floppy", "rfd0c", atau "rfd0a". 

      # dd if=<file> of=/dev/<device> bs=32k

    Pastikan anda menggunakan disket bagus yang telah diformat dengan tepat TANPA BAD BLOCK atau proses instalasi anda umumnya akan menemui kegagalan. Untuk keterangan lanjut pembuatan sebuah disket boot dan instalasi OpenBSD/i386 mohon merujuk ke FAQ4.1.

OpenBSD/macppc:

    Put the CD2 in your CDROM drive and poweron your machine while holding down the C key until the display turns on and shows OpenBSD/macppc boot.

    Alternatively, at the Open Firmware prompt, enter boot cd:,ofwboot /3.4/macppc/bsd.rd

OpenBSD/vax:

    Boot over the network via mopbooting as described in INSTALL.vax.

OpenBSD/sparc:

    The 3.4 release of OpenBSD/sparc is located on CD3. To boot off of this CD you can use one of the two commands listed below, depending on the version of your ROM. 
      > boot cdrom 3.4/sparc/bsd.rd
or
> boot sd(0,6,0)3.4/sparc/bsd.rd

    If your sparc does not have a CD drive, you can alternatively boot from floppy. To do so you need to write "CD3:3.4/sparc/floppy34.fs" to a floppy. For more information see FAQ4.1. To boot from the floppy use one of the two commands listed below, depending on the version of your ROM. 

      > boot floppy
or
> boot fd()

    Make sure you use a properly formatted floppy with NO BAD BLOCKS or your install will most likely fail.

    If your sparc doesn't have a floppy drive nor a CD drive, you can either setup a bootable tape, or install via network, as told in the INSTALL.sparc file.

OpenBSD/sparc64:

    Put the CD3 in your CDROM drive and type boot cdrom.

    If this doesn't work, or if you don't have a CDROM drive, you can write CD3:3.4/sparc64/floppy34.fs to a floppy and boot it with boot floppy.
    Make sure you use a properly formatted floppy with NO BAD BLOCKS or your install will most likely fail.

    You can also write CD3:3.4/sparc64/miniroot34.fs to the swap partition on the disk and boot with boot disk:b.

    If nothing works, you can boot over the network as described in INSTALL.sparc64

OpenBSD/alpha:

    Write FTP:3.4/alpha/floppy34.fs or FTP:3.4/alpha/floppyB34.fs (depending on your machine) to a diskette and enter boot dva0. Refer to INSTALL.alpha for more details.

    Make sure you use a properly formatted floppy with NO BAD BLOCKS or your install will most likely fail.

OpenBSD/hp300:

    Boot over the network by following the instructions in INSTALL.hp300.

OpenBSD/hppa:

    Boot over the network by following the instructions in INSTALL.hppa or the hppa platform page.

OpenBSD/mac68k:

    Boot MacOS as normal and partition your disk with the appropriate A/UX configurations. Then, extract the Macside utilities from FTP:3.4/mac68k/utils onto your hard disk. Run Mkfs to create your filesystems on the A/UX partitions you just made. Then, use the "BSD/Mac68k Installer" to copy all the sets in FTP:3.4/mac68k/ onto your partitions. Finally, you will be ready to configure the "BSD/Mac68k Booter" with the location of your kernel and boot the system.

OpenBSD/mvme68k:

    You can create a bootable installation tape or boot over the network.
    The network boot requires a MVME68K BUG version that supports the NIOT and NBO debugger commands. Follow the instructions in INSTALL.mvme68k for more details.

Catatan tentang kode sumber:

    src.tar.gz berisi arsip kode sumber yang bermula di /usr/src. Berkas ini berisi apapun yang anda perlukan kecuali kode sumber kernel, yang terpisah pada berkas termampat lainnya. Untuk memekarkan: 

      # mkdir -p /usr/src
# cd /usr/src
# tar xvfz /tmp/src.tar.gz

    sys.tar.gz berisi arsip kode sumber yang bermula di /usr/src/sys. Berkas ini berisi seluruh kode sumber kernel yang anda perlukan untuk membangun ulang kernel. Untuk memekarkan: 

      # mkdir -p /usr/src/sys
# cd /usr/src
# tar xvfz /tmp/sys.tar.gz

    Kedua struktur ini adalah checkout regular CVS. Menggunakan struktur ini memungkinkan untuk mendapat sebuah awalan-baik dalam menggunakan server anoncvs seperti dijelaskan di sini. Menggunakan berkas-berkas ini akan mempercepat inisialisasi pembaruan CVS daripada yang anda harapkan dengan sebuah checkout dari 'kondisi nol' pada struktur kode sumber OpenBSD yang lengkap.

Struktur Port

Sebuah struktur port termampatkan juga disediakan. Untuk memekarkan: 

    # cd /usr
# tar xvfz /tmp/ports.tar.gz
# cd ports

Subdirektori ports/ adalah sebuah checkout struktur port OpenBSD. Mohon baca halaman ports jika anda tidak tahu apapun tentang port pada saat ini. Tulisan ini bukanlah sebuah manual tentang bagaimana menggunakan port. Daripada itu, ini adalah serangkaian catatan yang dimaksudkan untuk mengenalkan pengguna pada sistem port OpenBSD.

Direktori ports/ mewakili sebuah ckeckout CVS (lihat manual cvs(1) jika anda tidak paham apakah CVS itu) dari port kita. Seperti struktur kode sumber lengkap kita, struktur port kita juga tersedia melalui akses anoncvs. Sehingga, dalam tujuan agar tetap terbaru dengan struktur CVS, anda harus membuat struktur ports/ berada pada media read-write dan perbarui struktur dengan perintah seperti: 

    # cd [portsdir]/; cvs -d anoncvsserver.openbsd.org:/cvs update -Pd -rOPENBSD_3_4

[Tentu saja, anda harus mengganti direktori lokal dan nama server disini dengan lokasi dari koleksi port anda dan server anoncvs terdekat.]

Menjadi catatan bahwa hampir semua port disediakan paket aplikasinya lewat ftp. Paket yang telah diperbarui untuk rilis 3.4 akan dibuat jika ada suatu masalah.

Jika anda tertarik untuk melihat ada tambahan port aplikasi, ingin membantu, atau hanya ingin tahu lebih banyak, daftar mailing ports@openbsd.org adalah sebuah tempat yang bagus untuk diketahui.
