Mengapa Pengembang OpenBSD Memakai CVS

Concurrent Version System (CVS) telah dipilih untuk digunakan dalam grup proyek OpenBSD. Dengan CVS, tujuh puluh atau lebih pengembang dapat mengkinikan kode sumber (lebih dari 1349 MB, sekitar 75000 berkas) yang mencakup seluruh SO, ports/paket, Sistem Window X . Sejumlah besar pengguna OpenBSD kemudian mendapatkan sumber SO lewat CVS anonymous, sebuah prosedur yang jauh lebih handal daripada Software Upgrade Protocol (sup(1)).

CVS juga membuat penciptaan dan pengelolaan ranting dengan mudah sehingga kita dapat menyediakan sebuah rilis + struktur kode sumber perbaikan dan sebuah pengembangan kode sumber terkini setiap menit dengan pengembangan paralel. Dengan CVSweb, sebuah interface web yang bisa dinavigasi dan tepat untuk melihat seluruh isi gudang CVS.

Karena antarmuka CVS dengan Secure Shell +(ssh(1)) maka bisa membuat otentikasi pengguna dan host yang handal. Hal ini tercakup dalam tujuan keamanan proyek OpenBSD.
