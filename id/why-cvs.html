Mengapa Pengembang OpenBSD Menggunakan CVS

Concurrent Version System (CVS) telah dipilih untuk digunakan dalam grup proyek OpenBSD. Dengan CVS, tujuh puluh atau lebih pengembang dapat mengkinikan kode sumber (lebih dari 1349 MB, sekitar 75000 berkas) yang mencakup seluruh SO, port/paket, dan XFree86. Sejumlah besar pengguna OpenBSD kemudian mendapatkan sumber SO lewat CVS anonymous, sebuah prosedur yang jauh lebih handal daripada Software Upgrade Protocol (sup).

CVS juga membuat penciptaan dan pengelolaan ranting dengan mudah sehingga kita dapat menyediakan sebuah rilis + struktur kode sumber perbaikan dan sebuah pengembangan kode sumber terkini setiap menit dengan pengembangan paralel.

Karena antarmuka CVS dengan Secure Shell (ssh) maka bisa membuat otentikasi pengguna dan host yang handal. Hal ini tercakup dalam tujuan keamanan proyek OpenBSD.
