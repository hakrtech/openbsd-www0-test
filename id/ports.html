Latar belakang

OpenBSD sendiri sesungguhnya sebuah sistem lengkap, tetapi tetap banyak peranti lunak yang mungkin ingin ditambahkan. Namun, terdapat masalah ketika harus menentukan apa saja yang harus ditambahkan, berkait masalah lisensi umum dan batasan ekspor. Karena OpenBSD bermaksud menjadi sistem operasi kecil tunggal sepadan UNIX , beberapa hal tidak bisa disertakan bersama sistem.

Koleksi port dan paket TIDAK melalui keseluruhan audit keamanan yang dilakukan OpenBSD. Meskipun kami berusaha keras untuk menjaga tinggi kualitas paket, kami hanya tidak mempunyai cukup sumberdaya manusia dalam memastikan kemanan dan kehandalan pada tingkat yang sama.

Koleksi port, pada dasarnya dipinjam dari FreeBSD, mengisi jeda diantaranya. Konsepnya adalah untuk memiliki, pada setiap peranti lunak pihak ketiga, sebuah Makefile yang mengatur

  • tempat mendapatkan,
  • cara mendapatkan,
  • pada apa bergantung (jika ada),
  • bagaimana mengubah kode sumber (jika perlu),
  • dan bagaimana mengkonfigurasi, build dan memasangnya.
Keterangan ini disimpan dalam sebuah hirarki direktori dibawah direktori /usr/ports.

Paket adalah ekivalensi binari dari port. Sebuah port terkompail akan menjadi sebuah paket yang bisa di daftarkan pada sistem menggunakan pkg_add(1)

Paket terlihat sebagai berkas .tgz sederhana, tetapi harus selalu ditambahkan dengan menggunakan pkg_add(1), mungkin terdapat beberapa keterangan tambahan yang hanya bisa dikelola oleh pkg_add(1) untuk penanganannya. Petunjuk: anda dapat membedakan antara paket dan berkas .tgz memakai pkg_info(1).

Mendapatkan Paket

Sejumlah besar koleksi paket pracipta yang tersedia untuk hampir semua arsitektur umum.
  • Pada CD-Rom (bisa anda pesan di sini),
  • Pada situs miror ftp.
  • Jelajahi dan ambil dari daftar paket web:
    • Paket untuk OpenBSD 3.2
    • Paket untuk OpenBSD 3.1
    • Paket untuk OpenBSD 3.0
    • Paket untuk OpenBSD 2.9
    • Paket untuk OpenBSD 2.8
    • Paket untuk OpenBSD 2.7
Menambahkan paket semudah pkg_add pkgname.tgz. Jika anda mendapatkan paket dari sumber tunggal (gudang paket), atur PKG_PATH ke URL gudang paket, untuk mendapatkan keterkaitannya.

Sebagai contoh, untuk memasang paket Gimp pada rilis 3.2 di sebuah mesin i386 machine lewat situs ftp (termasuk keterkaitannya), lakukan: 

    # setenv PKG_PATH ftp://ftp.openbsd.org/pub/OpenBSD/3.2/packages/i386/
    # pkg_add ${PKG_PATH}gimp-1.2.3.tgz

Pembaruan Ports dan Paket untuk sebuah Rilis OpenBSD

Pada rilis 2.7 terlihat perkenalan ranting stabil untuk struktur port.

Sebagai contoh, untuk mendapatkan ranting stabil rilis 3.2: 

    $ cd /usr/ports
    $ cvs -q -d anoncvs@some.anon.server:/cvs up -r OPENBSD_3_2 -Pd

Bermula dari rilis 2.8, paket binari terpilih juga tersedia. Mohon lihat halaman paket stabil untuk menemukan paket terbarui dan perbaikan penting pada ranting stabil.

Nama paket selalu berubah berkait pembaruan, untuk menghindari resiko kebingungan antara sebuah paket rilis dan paket terbaiki.

Mengelola Paket Terpasang

Halaman manual packages(7) berisi keterangan bermanfaat tentang cara mengelola paket terinstall, memperbaiki konflik (berkas yang telah ada) dan menangani keterkaitan.

Seperti pada OpenBSD 2.8, untuk membarui paket anda harus:

  • Buang paket lama dengan pkg_delete(1)
  • Tambahkan paket baru dengan pkg_add(1)
Hal ini sangat menjengkelkan, jika sebuah paket meminta keterkaitannya, dan anda harus membuang sejumlah besar paket hanya untuk pembaruan sebuah paket.

Menggunakan Port

Jika sebuah paket tidak ada untuk arsitektur anda, anda masih mungkin mengkompail port tersebut. Disamping itu, beberapa user akan ingin mengkompail apapun dari sumber port untuk beberapa alasan.

Anda bisa meng-ftp versi rilis dari direktori pub/OpenBSD/[versi] (dengan [version] adalah nomor rilis) pada setiap situs miror ftp. Versi rilis adalah yang kami sertakan dalam CDROM kami, dan telah berubah melalui banyak percobaan dari snapshot manapun. Keterangan lanjut tersedia di halaman manual ports(7).

Daur Hidup Struktur Port

Struktur port, seperti lainnya dalam OpenBSD, secara konstan mengalami perubahan.

Struktur port terkini mungkin tidak bisa digunakan dengan rilis sebelumnya. Hal ini karena ada perubahan-perubahan, khususnya dengan proses pembuatan port, hal tersebut membutuhkan kode berbasis pada struktur kode sumber OpenBSD-current.

Struktur port berjalan sebagai entitas tunggal. Membarui sebuah direktori tunggal tidak kami jamin akan berhasil, seturut ketergantungan paket mungkin memaksa anda untuk membarui dan kompail ulang bagian struktur port yang sangat banyak. Sangat disarankan setiap orang jangan mengikuti ports-current kecuali mereka menyiapkan diri berurusan dengan beberapa masalah. Milis seperti ports-changes@openbsd.org atau tech@openbsd.org akan mungkin sangat bernilai.

Anda disarankan untuk mengikuti ports-stable untuk mesin produksi: kami akan menjaga struktur port stable tetap handal dalam menghadapi msalah, dan menyediakan pembaruan binari tepat pada waktunya sebaik mungkin.

Menjadi catatan bahwa distfile yang hilang bukanlah suatu masalah, sebagaimana ftp.openbsd.org merupakan juga gudang distfile yang lengkap. Meskipun mengubah checksum bukanlah sebuah masalah: anda dapat memberikan perintah 

	make checksum REFETCH=true
untuk memastikan anda mendapatkan distfile yang benar untuk struktur port anda. Untuk definisi current dan stable, lihat Upgrade Mini-FAQ.

Tersedia sebuah daftar perubahan harian untuk ports dan ports-current.

Struktur ports-current dapat diperoleh melalui:

  • CVS Anonymous CVS (lihat sambungan). Perintah yang penting adalah cvs get ports.
  • Ftp anonymous ftp dari ftp://ftp.openbsd.org/pub/OpenBSD/snapshots/ports.tar.gz. Berkas termampat ini diperbarui setiap malam.
  • CVSup. Lihat contoh penggunaan CVSup dalam mode checkout.
  • Web browser, tengoklah antarmuka web untuk CVS.

Ports dan XFree86

Ada beberapa keajaiban khusus pada berkas konfigurasi XFree86 OpenBSD yang membolehkan struktur port menginstall aplikasi berbasis imake pada /usr/local.

Mulai 2.9, OpenBSD disertai XFree86 4.0.3. Jika anda menggunakan distribusi sebelum itu, bersama dengan binari XFree86 dari proyek XFree86, anda mungkin perlu melakukan perbaikan pada /usr/ports/infrastructure/patches/patch-xfree-4.0 menjadi sebuah distribusi binari XFree 4.0.x yang bisa berjalan sesuai struktur port OpenBSD.

Contoh Penggunaan Struktur Port

Anggap anda ingin mendapatkan struktur port dan anda ingin untuk melakukan kompail dan menginstall utilitas pemampatan unzip. Anda harus bisa melakukan hal sebagai berikut: 

    % cd /usr/ports/archivers/unzip
    % su
    # make
    # make install
    # exit
Mudah bukan ? Khususnya memahami semua apa yang terjadi di belakang:
  • Mengambil kode sumber unzip dari situs ftp,
  • Memeriksa integritas kode sumber termampat,
  • Memekarkan kode sumber unzip,
  • Menerapkan perbaikan spesifik,
  • Konfigurasi dan membangun program,
  • Membuat binari paket pada /usr/ports/packages,
  • Menginstall paket tersebut.

Bersama OpenBSD 3.0, hampir semua port secara otomatis membangun paket sewaktu dinstall.

Sebagaimana ports dibangun, direktori /usr/ports/distfiles mulai dipenuhi dengan kode sumber program, dan /usr/ports/packages mulai dipenuhi dengan paket binari. User dengan koneksi lambat mungkin bisa merujuk ke mirroring-ports(7) untuk cara efisien dalam mendapatkan semua distfile dalam sekali kesempatan. Pada OpenBSD-current, anda dapat menggunakan skrip /usr/ports/infrastructure/fetch/clean-old untuk melacak distfile lama. Menjadi catatan bahwa CD OpenBSD hanya berisi struktur port dan paket terpilih. Jika anda ingin mempunyai distfile, anda harus mendapatkannya melalui cara tersendiri.

Penggunaan Lebih lanjut Struktur Port

Struktur port mempunyai banyak fitur bagi pengguna lanjut yang membuat port menjadi alat yang lebih bernilai lebih dari instalasi dasar. Pengguna lanjut mungkin berharap untuk 'bermain' dengan makefiles (anda seharusnya membaca halaman manual make(1) terlebih dahulu) atau mengatur berbagai variabel dari baris-perintah make atau pada /etc/mk.conf. Variabel-variabel ini dijelaskan secara rinci pada halaman manual bsd.port.mk(5) , dan dokumen porting berikut.

Membuat Port Baru

Jika anda tertarik dalam membantu memperluas struktur port OpenBSD anda harus membaca dahulu porting.html. Halaman tersebut merujuk pada FreeBSD Porter's Handbook sebagaimana petunjuk dan kebijakan khusus OpenBSD.

Ketika sebuah Versi Port Tertinggal dari Versi Utamanya

Koleksi port adalah proyek sukarela. Kadang suatu projek tidak mendapatkan waktu dari pengembangnya untuk dijaga tetap terbarukan. Pengembang secara mudah akan mengambil apa yang menarik baginya dan dapat di ujicoba pada lingkungan mereka. Sumbangsih anda sangat membantu untuk platform mana suatu port bisa diujicoba.

Beberapa port mungkin tertinggal versinya dari versi utama karena hal diatas. Koleksi port mungkin mempunyai versi program yang dirilis pada Januari lalu sementara versi baru program tersebut telah dirilis oleh pengembangnya pada Mei dan itu 3 bulan yang lalu. Sering hal ini menjadi keputusan berdasar kesadaran, versi baru mungkin mempunyai masalah dengan OpenBSD yang oleh pengelola port coba untuk diselesaikan, atau versi baru jelas merupakan aplikasi yang lebih buruk dari versi sebelumnya: OpenBSD mungkin mempunyai tujuan berbeda dari pengembang utama aplikasi proyek lain, yang seringkali pada fitur dan desain atau pemilihan implementasi yang tidak sesuai oleh sudut pengembang OpenBSD. Pembaruan yang ada mungkin juga diabaikan karena versi yang baru tidak dianggap sebagai pembaruan yang penting.

Jika anda benar-benar menginginkan versi terbaru dari sebuah port, anda harus meminta MAINTAINER port tersebut untuk memperbarui port (lihat ini tentang bagaimana cara mencari pengelola sebuah port); jika anda bisa mengirim perbaikan versi baru tersebut, semuanya akan menjadi lebih baik. Untuk menciptakan perbaikan yang tepat, anda harus merujuk pada dokumentasi pembuatan port.

Masalah dan Pihak yang bisa dihubungi

Jika anda mempunyai masalah dengan port yang telah ada, mohon kirim email ke maitainer port. Untuk melihat siapa yang mengelola port tersebut, ketik, sebagai contoh: 

       % cd /usr/ports/archivers/unzip
       % make show VARNAME=MAINTAINER
Cara lain, jika tidak ada maintainer, atau anda tidak bisa menghubunginya , kirim email ke milis OpenBSD, ports@openbsd.org. Harap tidak menggunakan milis misc@openbsd.org untuk menanyakan tentang port. Koreksi selalu kami terima, tetapi pada setiap masalah mohon sediakan:
  • Keluaran uname -a,
  • Versi OpenBSD anda, termasuk perbaikan/patch yang anda terapkan,
  • Keterangan lengkap masalah yang ditemui.
Untuk port yang tidak bisa dibangun dengan sempurna, sebuah transkrip build lengkap hampir selalu diperlukan. Anda bisa menggunakan skrip portslogger, yang bisa ditemukan di /usr/ports/infrastructure/build, untuk keperluan ini. Contoh menjalankan portslogger seperti berikut: 
       % cd /usr/ports/archivers/unzip
       % su
       # mkdir -p ~/portslogs
       # make clean install 2>&1 | /usr/ports/infrastructure/build/portslogger \
               ~/portslogs
Setelahnya, anda harus mempunyai sebuah berkas log dari proses build di direktori ~/portslogs anda yang bisa anda kirimkan ke maintainer port. Juga, pastikan anda tidak menggunakan opsi tambahan apapun dalam proses build anda, misal tambahan opsi di /etc/mk.conf.

Cara lain, anda bisa

  • Mengunakan script(1) untuk membuat transkrip build lengkap. Jangan menghilangkan keterangan configure.
  • Sertakan keluaran dari pkg_info(1) jika terlihat bermanfaat.
  • Kesalahan dalaman kompiler gcc(1) meminta anda untuk melaporkan bug tersebut ke daftar mailing gcc. Hal ini menghemat akan waktu jika anda mengikuti petunjuk mereka, dan sediakan paling tidak beberapa berkas yang dihasilkan dengan gcc -save-temps.
