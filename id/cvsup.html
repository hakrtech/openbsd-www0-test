Apakah CVSup itu?

CVSup adalah sebuah paket piranti lunak untuk mendistribusikan dan membarui struktur kode sumber dari sebuah gudang cvs(1) utama pada sebuah server di tempat lain. Kode sumber OpenBSD telah dikelola dalam sebuah gudang CVS pada sebuah mesin pusat pengembangan di Kanada. Dengan CVSup, pengguna OpenBSD bisa dengan mudah membuat struktur kode sumber mereka selalu terbarukan.

CVSup menggunakan model pembaruan yang disebut pull. Dengan model pull , setiap klien meminta ke server untuk pembaruan, jika dan ketika pembaruan diperlukan. Server menunggu dengan pasif permintaan pembaruan dari para klien. Sehingga seluruh pembaruan dilakukan berdasar keinginan klien. Server tidak pernah mengirim pembaruan yang tidak diinginkan klien secara langsung. Pengguna harus menjalankan klien CVSup secara manual untuk mendapatkan pembaruan, atau harus di konfigurasi tugas cron(8) untuk menjalankannya secara otomatis.

Istilah CVSup, tertulis huruf besar sama persis, mengacu pada keseluruhan paket piranti. Komponen utamanya adalah cvsup klien yang berjalan di setiap mesin pengguna, dan server cvsupd yang berjalan di setiap situs miror CVSup OpenBSD.

CVSup dimaksudkan lebih cepat dan lebih fleksibel daripada sup(1), pendahulu CVSup.

Proyek OpenBSD saat ini mempunyai 5 gudang kode sumber utama:

  • src - Berisi semua kode sumber untuk Sistem Operasi OpenBSD.
  • ports - Berisi Port OpenBSD.
  • www - Berisi seluruh halaman-halaman situs web OpenBSD. (termasuk yang saat in anda baca).
  • x11 - Berisi adaptasi OpenBSD pada proyek piranti lunak XFree86-3 .
    • Disebut disini hanya untuk kepentingan historis, kebanyakan pengguna tidak punya alasan untuk mengunakannya, karena sudah tidak dipakai lagi.
  • XF4 - Rumah untuk penyesuaian proyek peranti lunak X OpenBSD yang aktif.

Apakah CVS itu?

CVS adalah sistem kendali kode sumber yang digunakan untuk mengelola struktur kode sumber OpenBSD. Dijelaskan lebih lanjut di sini. Perhatikan bahwa CVS dan CVSup seluruhnya adalah program yang berbeda. Meskipun CVS bisa digunakan untuk akses berkas jarak jauh, tidak dioptimasi untuk distribusi seluruh struktur kode sumber lewat jaringan, dan CVSup dapat dengan mudah meminta tingkat efisiensi yang lebih untuk tugas tersebut.

Memulai Penggunaan CVSup

Membangun CVSup dari kode sumber dapat membuktikan tidak praktisnya karena ditulis dalam bahasa Modula-3. Port lang/ezm3 memberikan distribusi sistem kompiler dan runtime Modula-3 yang lebih kecil bagi siapa yang hanya perlu Modula-3 untuk membangun CVSup. Namun, jika anda hanya ingin menjalankan CVSup tanpa peduli hal lain, di sarankan anda mengambil paket cvsup dari situs miror ftp daripada membuat paket anda sendiri dari port net/cvsup . Saat ini port OpenBSD ezm3 dan sebagai konsekuensi dari CVSup tersedia di platform i386.

Untuk keterngan lanjut tentang CVSup, lihat homepage proyek pembuatnya.

Untuk melakukan miror terhadap gudang OpenBSD dengan CVSup, berkas konfigurasi berikut bisa digunakan: 

        # Konfigurasi dasar yang sesuai untuk seluruh koleksi
        *default release=cvs
        *default delete use-rel-suffix
        *default umask=002
        *default host=cvsup.id.openbsd.org
        *default base=/cvs
        *default prefix=/cvs

        # Jika sambungan jaringan anda sebuah T1 atau lebih cepat, beri tanda # diawal 1 baris berikut.
        *default compress

        OpenBSD-all
        #OpenBSD-src
        #OpenBSD-www
        #OpenBSD-ports
        #OpenBSD-x11
        #OpenBSD-xf4

Secara langsung cvsup akan membarui seluruh distribusi OpenBSD dari cvsup.id.openbsd.org dengan sebuah umask(2) yang mengijinkan hak tulis grup ke gudang kode sumber lokal di mesin anda.

Anggap berkas tersebut disimpan sebagai cvs-supfile, perintah berikut bisa digunakan untuk menjalankan GUI cvsup: 

        cvsup cvs-supfile

untuk mode batch, berikut ini bisa dipakai: 

        cvsup -g -L 2 cvs-supfile

Menggunakan CVS untuk Membarui Struktur Kode Sumber Anda

Sekarang akan mudah untuk memeriksa gudang terCVSup manapun. Untuk shell Korn/Bourne: 

        # cd /usr
        # CVSROOT=/cvs cvs checkout src

Untuk csh dan turunannya: 

        # cd /usr
        # setenv CVSROOT /cvs
        # cvs checkout src

Hal diatas itu akan memeriksa kode sumber terbaru. Banyak dari anda akan hanya perlu kode sumber release, khususnya jika anda memperbaiki(patching) sistem anda. Untuk memeriksa kode sumber release anda harus menentukan sebuah penanda(tag) bersama perintah anda. Sebagai contoh: 

        # cd /usr
        # cvs checkout -rOPENBSD_2_9 src

Atau OPENBSD_2_8 untuk 2.8, dst.

Menjalankan CVSup dalam Mode Pemeriksaan(Checkout)

Sebagai sebuah alternatif untuk mendapatkan gudang kode sumber dan memakai CVS, anda bisa menjalankan CVSup dalam modus pemeriksaan dengan menambahkan sebuah kata kunci tag atau date dalam supfile anda, baik sebagai sebuah *default atau sebagai sebuah opsi pada sebuah koleksi. Khususnya, anda bisa menggunakan hal ini untuk mengefisienkan pembaruan struktur kode sumber dan port yang disertakan bersama CDROM.

supfile berikut dapat digunakan untuk membarui struktur port anda: 

        # Konfigurasi dasar yang sesuai untuk seluruh koleksi
        *default host=cvsup.id.openbsd.org
        *default base=/var
        *default prefix=/usr
        *default release=cvs
        *default delete use-rel-suffix compress

        # Koleksi Port.
        OpenBSD-ports tag=.

Setiap penanda(tag) simbolis CVS dapat digunakan. Sebuah titik tunggal "." berarti HEAD, misal revisi terbaru dari seluruh berkas pada ranting(branch) utama. Hati-hati untuk menentukan sebuah penanda yang ada, karena CVSup tidak bisa membedakan penanda valid dengan penanda invalid, dan tetap berusaha menyelaraskan sebuah struktur kode sumber ke sebuah penanda invalid hasilnya akan menghapus seluruh berkas yang sudah anda punyai.

Sebagai pilihan lain, gunakan kata kunci date=[cc.]yy.mm.dd.hh.mm.ss untuk memilih sebuah revisi berdasat tanggal. Seluruh 17 atau 20 karakter harus di berikan seperti format contoh. Untuk tahun 2000 dan sesudahnya, tentukan abad cc. Untuk tahun sebelum itu, gunakan hanya 2 tempat angka terakhir tahun itu yy. Anda juga bisa menggabungkan kata kunci tag dan date .

Catatan:Jika server anda terdaftar disini dengan keterangan yang salah, mohon hubungi www@openbsd.org

Anda bisa juga menggunakan traceroute(8) untuk menemukan server yang terdekat dengan anda. Masalah terkait server cvsup mohon dilaporkan ke pengelola dari server bersangkutan.

Saat ini (per tanggal 8 Desember 2004), ukuran gudang CVS kode sumber adalah

OpenBSD-ports - 168MB
OpenBSD-src - 1310MB
OpenBSD-www - 161MB
OpenBSD-x11 - 200MB
OpenBSD-xf4 - 524MB

Terdapat sebuah tambahan lebih dari 271MB untuk direktori CVSROOT . Keseluruhan ukuran gudang kode sumber saat ini meningkat pada laju sekitar 400MB per tahun.

CATATAN PENTING: Terdapat beberapa hal terkait dengan piranti lunak kriptografi yang harus disadari setiap orang:

  • Kode sumber OpenBSD berasal dari Kanada. Seturut riset penelitian seorang Kanada dan dijelaskan pada daftar Kendali Ekspor Kanada, merupakan hal legal untuk mengekspor piranti lunak crypto dari Kanada ke seluruh dunia.

  • namun, jika anda tinggal diluar Amerika Serikat, seharusnya anda tidak mengambil bagian kriptografi pada kode sumber OpenBSD dari sebuah server CVSup yang berlokasi di Amerika Serikat. Berkas-berkas dimaksud adalah...

    • src/kerberosIV/*
    • src/kerberosV/*
    • src/lib/libdes/*
    • src/lib/libc/crypt/crypt.c
    • src/lib/libc/crypt/morecrypt.c
    • src/sys/crypto
    • src/sys/netinet
    • src/usr.sbin/afs/src/rxkad/*
    • XF4/xc-mit/lib/Xdmcp/Wraphelp.c
    • XF4/xc-old/lib/Xdmcp/Wraphelp.c
    • XF4/xc/lib/Xdmcp/Wraphelp.c

    Berdasar daftar senjata ITAR Amerika Serikat, piranti lunak crypto hanya boleh di ekspor ke Kanada dari Amerika Serikat.

Proyek OpenBSD mencari lebih banyak lagi server CVSup -- jika anda tertarik, mohon hubungi www@openbsd.org untuk rincian konfigurasi.
