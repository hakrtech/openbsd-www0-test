Apakah CVS Anonymous itu?

CVS Anonymous adalah sebuah cara untuk menjaga salinan setempat anda dari struktur kode sumber OpenBSD tetap terkini sesuai dengan perubahan yang terjadi pada kode sumber OpenBSD terkini. Sebagai tambahan untuk mengikuti sisi keras pengembangan, mungkin juga untuk melacak perbaikan pada masalah sebuah rilis.

Keuntungan utama dari CVS Anonymous CVS dibdaning teknik pembaruan kode sumber lainnya adalah kerjanya yang langsung ke gudang kode sumber utama ataupun miror. Ini berarti anda memiliki rangkaian lengkap perintah CVS untuk mengendalikan penyatuan dan pembaruan perubahan anda dengan perubahan kode sumber lain dan melakukan diff, merubah kisah dan permintaan lain yang langsung ke gudang utama.

Proyek OpenBSD saat ini mempunyai lima gudang kode sumber utama:

  • src - Rumah bagi semua kode sumber untuk Sistem OpenBSD.
  • ports - Rumah untuk Port OpenBSD.
  • www - Rumah untuk semua halaman web OpenBSD. (termasuk yang satu ini).
  • X11 - Rumah untuk penyesuaian proyek peranti lunak XFree86-3 OpenBSD. Disebut disini hanya untuk kepentingan historis, kebanyakan pengguna tidak punya alasan untuk mengunakannya, karena sudah tidak dipakai lagi.
  • XF4 - Rumah untuk penyesuaian proyek peranti lunak X OpenBSD yang aktif.

Untuk merangkum, kekuatan sesungguhnya dari penggunaan CVS Anonymous adalah hal ini sebuah "toleransi" sistem kendali kode sumber - dalam hal ini mnegikuti mengikuti perubahan yang telah anda lakukan pada kode sumber setempat dan membuat "usaha terbaik" untuk mengkinikankeseluruhan struktur kode sumber anda, daripada memberi anda sebuah daftar masalah yang harus ditangani sebelum berlanjut.

Apakah CVS itu?

CVS adalah sistem kendali kodesumber yang dipakai untuk mengelola struktur kodesumber OpenBSD. CVS mengimplementasikan sebuah gudang terpusat untuk semua kode sumber rilis resmi dan perubahannya, dengan membolehkan pengembang untuk mengelola salinan setempat kode sumber mereka dengan perubahan yang mereka lakukan. Terdapat 2 tingkat akses struktur kodesumber:
  • Akses baca-tulis untuk pengembang: Pengembang yang perlu menambahkan perubahan pada struktur kodesumber harus mempunyai sebuah akoun pada mesin OpenBSD. Mendapatkan akses ini secara langsung akan bekerja pada kodesumber bersama pengembang OpenBSD lainnya. Jika seseorang melakukan beberapa pekerjaan bagus dan menunjukkan mereka bisa berkeja dalam tim, mereka akan mendapatkan akoun.
  • Akses baca-saja untuk siapapun: Setiap orang bisa mengakses gudang CVS baca-saja. Salinan dari gudang baca-tulis CVS jarang ada mirornya.

Kekuatan utama CVS adalah kemampuannya melakukan penyatuan yang pandai dari perubahan ke gudang pusat dengan perubahan yang anda buat pada salainan setempat anda. Ini berarti bahwa jika anda membuat sebuah perubahan ke sebuah modul dan melakukan pembaruan, perubahan anda tidak akan "tertiup angin", jelasnya CVS membuat usaha terbaik untuk menyatukan perubahan ke kode sumber pusat dengan perubahan yang telah anda lakukan di salinan setempat anda

Pada kejadian perubahan tidak bisa disatukan sempurna, CVS menyediakan sebuah "pengembalian halus", dalam tataran menyediakan anda catatan kisah perubahan ke dalam salainan setempat anda, menyimpan sebuah salinan asli dari versi anda dan melanjutkan untuk membarui modul kode sumber lain yang anda minta.

Mulai Menggunakan CVS Anonymous

Sementara anda bisa mendapatkan keseluruhan struktur kodesumber dari sebuah server AnonCVS, anda bisa menghemat waktu dan bandwidth melalui "pramuat" struktur kodesumber anda dengan berkas sumber baik dari CD OpenBSD maupun dari sebuah server FTP. Hal ini relevan jika anda menjalankan -stable, karena relatif hanya sedikit berkas berubah antara -release dan -stable. Menyalin struktur CVS dari CD ke /usr/src (anggap CD telah ditautkan pada /mnt): 
    # cd /usr/src; tar xzf /mnt/src.tar.gz
    # cd /usr; tar xzf /mnt/XF4.tar.gz
    # tar xzf /mnt/ports.tar.gz
Berkas-berkas sumber di server FTP dipisahkan menjadi dua berkas untuk mengurangi waktu mendapatkannya bagi siapa yang ingin bekerja hanya dengan satu bagian dari struktur. Dua files tersebut adalah sys.tar.gz, yang berisi berkas-berkas yang dibutuhkan untuk membuat kernel, dan src.tar.gz, yang berisi seluruh perangkat "userland". Secara umum, bagaimanapun, anda biasanya akan menginginkan keduanya terpasang di harddisk anda. Dengan asumsi berkas, src.tar.gz dan sys.tar.gz berada di /usr: 
    # cd /usr/src
    # tar xzf ../sys.tar.gz
    # tar xzf ../src.tar.gz
    # cd /usr
    # tar xzf XF4.tar.gz
    # tar xzf ports.tar.gz

Tidak semua orang berencana akan memekerkan seluruh rangkaian berkas, tetapi karena sistem harus dijaga tetap sinkron, secara umum anda perlu mengatur seluruh struktur.

Setelah ini, /usr/src merupakan tempat checkout yang manis bagi seluruh perintah cvs(1) bekerja dengan semestinya.

Menggunakan CVS untuk mendapatkan dan membarui struktur kode sumber anda

CVS telah didesain agar mudah dalam mendapatkan dan membarui kode sumber anda. Pertama anda harus menentukan apakah anda ingin untuk mengikuti current atau sebuah ranting stable(perbaikan). Struktur current mempunyai semuanya sampai pada menit terakhir perubahan, sementara ranting perbaikan berisi rilis formal ditambah perbaikan dari masalah dan sedikit perbaikan yang telah diterapkan. Untuk keterangan lanjut tentang "citarasa" OpenBSD ini, lihat di sini.

Setelah anda menentukan struktur yang akan diikuti, anda harus memilih server CVS Anonymous yang akan digunakan. Sebuah daftar server ada di bawah ini.

Sekali setelah anda memilih Server CVS Anonymous yang akan anda gunakan, anda bisa mulai menggunakan cvs. Bagi anda yang mempunyai rangkaian CD dapat memulai pemeriksaan CVS yang ada pada CD dengan menggunakan cara di atas untuk mendapatkan kode sumber ke sistem anda. Jika anda tidak mempunyai rangkaian CD, gunakan cara di bawah untuk melakukan pemeriksaan kode sumber. Cara ini akan meletakkan struktur kode sumber OpenBSD di /usr/src. 

	# cd /usr; cvs checkout -P src

Cara diatas akan melakukan pemeriksaan struktur kode sumber current. Banyak dari anda hanya menginginkan kode sumber ranting perbaikan. Untuk melakukan pemeriksaan pada ranting perbaikan, anda harus menentukan sebuah tanda dalam perintah anda. Contoh: 

	# cd /usr; cvs checkout -P -rOPENBSD_3_7 src

Atau OPENBSD_3_6 untuk 3.6, dsb.

Tanda OPENBSD_3_7 berisi kode sumber rilis dan perbaikan masalah yang telah diterapkan.
