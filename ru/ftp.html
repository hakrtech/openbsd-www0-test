Файловая структура

Файловая структура каждого сервера приблизительно следующая:

  • .../OpenBSD/3.4/
    Здесь находится самый последний релиз.
    • alpha/
    • hp300/
    • hppa/
    • i386/
    • mac68k/
    • macppc/
    • mvme68k/
    • mvme88k/
    • sparc/
    • sparc64/
    • vax/
    • src.tar.gz
       Исходной код всех программ (размещается в /usr/src). Сюда не входит исходной код ядра.
    • sys.tar.gz
       Исходной код ядра (размещается в /usr/src/sys).
    • ports.tar.gz
       Исходной код портов (размещается в /usr/ports).
    • XF4.tar.gz
       Исходной код XFree86 (X11).

  • .../OpenBSD/snapshots/
    Для основных архитектур, примерно раз в месяц, создаются мини-релизы неизвестной стабильности и качества. Эти релизы размещаются здесь.

  • .../OpenBSD/src/
    Здесь находится ежедневная выборка исходных текстов из cvs, для тех, кто хочет иметь доступ к отдельным файлам. Вместо того, чтобы зеркалировать данную директорию, мы рекомендуем воспользоваться CTM или AnonCVS.

  • .../OpenBSD/doc/
    OpenBSD FAQ в PDF и текстовом форматах, а так же другая относящаяся к OpenBSD документация.

  • .../OpenBSD/tools/
    Различные утилиты.

  • .../OpenBSD/3.3/
    Структура как и у 3.4, описана выше.. для ностальгирующих.
  • .../OpenBSD/3.2/
    Структура как и у 3.4, описана выше.. для ностальгирующих.
  • .../OpenBSD/3.1/
    Структура как и у 3.4, описана выше.. для ностальгирующих.
  • .../OpenBSD/3.0/
    Структура как и у 3.4, описана выше.. для ностальгирующих.
  • .../OpenBSD/2.9/
    Структура как и у 3.4, описана выше.. для ностальгирующих.

Тем, кто создает новые FTP или AnonCVS зеркала, может быть интересен примерный размер необходимого дискового пространства. Ниже приводятся размеры для ftp://ftp.openbsd.org:/pub/OpenBSD/.

Зеркала должны предоставить доступ по крайней мере на два последних релиза в бинарном виде (на текущий момент, это 3.3 и 3.4) для всех архитектур, а так же директорий OpenSSH/, patches/ и tools/. В зависимости от свободного места, зеркала также могут предоставлять snapshots, старые релизы, дерево исходных текстов и т.д.

Используемое место на диске, на главном ftp сервере: 

                       сейчас  может быть
pub/OpenBSD/2.9        3.0GB     3.0GB   (релиз OpenBSD 2.9)
pub/OpenBSD/3.0        4.5GB     4.5GB   (релиз OpenBSD 3.0)
pub/OpenBSD/3.1        6.9GB     6.9GB   (релиз OpenBSD 3.1)
pub/OpenBSD/3.2        11.8GB    11.8GB  (релиз OpenBSD 3.2)
pub/OpenBSD/3.3        14.0GB    14.0GB  (релиз OpenBSD 3.3)
pub/OpenBSD/3.4        11.0GB    14.0GB  (релиз OpenBSD 3.4)
pub/OpenBSD/snapshots  8.5GB     больше! (текущий снапшот, обычно >=
                                          размеру последнего дистрибутива)
pub/OpenBSD/src        .6GB      .7GB    (дерево src, доступное по FTP,
                                          создается при помощи AnonCVS каждую
                                          ночь. Вы не должны использовать
                                          зеркала для его получения, лучше
                                          скопировать его с ближайшего
                                          AnonCVS сервера)
Поэтому, для полного зеркала необходимо около 50GB, а для частичного (последние два релиза) только 25GB.

Кроме того, главный ftp сервер предоставляет следующие директории: 

pub/OpenBSD/distfiles  4.4GB     больше! (distfiles, используются подсистемой
                                          портов. Вы не должны зеркалировать
                                          данную директорию, если только для
                                          своих нужд. Make-файлы из портов по
                                          умолчанию будут искать эти файлы на
                                          ftp.openbsd.org и ftp.usa.openbsd.org)

Если вы создали новое ftp, http, AFS или rsync зеркало, пожалуйста, напишите в www@openbsd.org.

www@openbsd.org
