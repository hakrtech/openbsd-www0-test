Commander les CD

Pour obtenir les fichiers de cette version :
  • Commandez un CDROM depuis notre système de commandes.
  • Regardez les informations sur la page FTP pour une liste des machines miroirs
  • Allez dans le répertoire pub/OpenBSD/2.9/ sur l'un des sites miroirs
  • Lisez brièvement le reste de ce document.
  • Jetez un oeil sur la page des errata 2.9 pour une liste des bugs et des solutions.
  • Regardez la liste détaillée des changements entre les version 2.8 et 2.9.



Remarque : Tous les copyrights applicables et les différents crédit se trouvent dans les fichiers source se trouvant dans src.tar.gz, srcsys.tar.gz, XF4.tar.gz ou dans les fichiers récupérés par l'intermédiaire de ports.tar.gz. Les fichiers utilisés pour construire les paquetages composant le fichier ports.tar.gz ne sont pas inclus sur le CDROM par manque de place.

Quoi de neuf ?

Ceci est une liste partielle des nouvelles fonctionnalités incluses dans OpenBSD 2.9. Pour une liste complètes, voir le journal des changements jusqu'à la 2.9.

  • Les performances du système de fichier ont été considérablement améliorées à l'aide des nouveaux codes de softupdate et dirpref. Certains tests montrent une amélioration de 60x en vitesse du système de fichiers. S'il y a une raison de mettre à jour en OpenBSD 2.9, c'est celle là.

  • OpenSSH (qui supporte les protocoles SSH1 et SSH2) en est à la version 2.9. Les transferts de fichier sécurisés sont encouragés à l'aide du sous-système SFTP qui est fourni avec un serveur et un client.

  • Beaucoup de changements dans la documentation, notamment les pages du manuel et la FAQ en ligne. Les pages du manuel incluent maintenant des exemples utiles pour complèter les explications.

  • Une collection de "ports" plus complète et mieux testée.

  • Environ 1000 paquetages pré-compilés et testés.

  • OpenBSD 2.9 fonctionne sur le nouveau Apple Titanium PowerBook G4.

  • Après deux versions à problèmes, le port OpenBSD/alpha est de retour avec un meilleur support matériel !

  • Le système comprend les éléments majeurs suivants venant de sources extérieures :

    • XFree86 4.0.3 (et i386 contient aussi des serveurs 3.3.X, fournissant un support pour tous les chipsets)
    • gcc 2.95.3 (+ correctifs)
    • perl 5.6.0 (+ correctifs)
    • Apache 1.3.19, mod_ssl 2.8.2, OpenSSL 0.9.6 (+ correctifs), support DSO
    • ipf 3.4.16 (+ correctifs)
    • groff 1.15
    • sendmail 8.11.3
    • lynx 2.8.2 avec support HTTPS ajouté
    • sudo 1.6.3p7
    • ncurses 5.2
    • Dernière version de la pile IPv6 KAME
    • KTH Kerberos 1.0.4
    • OpenSSH 2.9

  • Support périphérique grandement amélioré.

    • Pliote 3Com 3c990 3XP Typhoon/Sidewinder (seulement Ethernet).
    • Nouveaux pilotes RAID hardware incluant :
      • Adaptec FSA
      • American Megatrends MegaRAID
      • Contrôleurs Compaq Smart ARRAY
    • Nouveaux pilotes SCSI incluant :
      • Initio 940/950
      • Symbios Logic 53c1x10
      • Adaptateurs PCI basés sur l'AMD Am53c974
    • Support pour les cartes audio des portables, incluant :
      • ESS Maestro 1/2/2E
      • Yamaha DS-1S
      • Périphériques audio Cirrus Logic CS4281
    • Le matériel USB est maintenant activé par défaut et comprend le support pour les claviers et les souris.

  • Nombreuses améliorations pour la sécurité la fiabilité (voir les lignes en rouge dans le journal des changements).

  • De nouvelles capacités, incluant :

    • Support des noms de connexion longs.
    • Émulation Linux grandement améliorée permettant à bien plus d'applications de tourner.
    • Support 802.1d spanning tree pour bridge(4).
    • Support vlan(4) en pont ce qui permettant plus de configurations en pont (on peut même utiliser OpenBSD comme un commutateur compatible 802.1Q)
    • Mécanisme de notification dans le noyau de type kqueue.
    • Ajout du pilote de console wscons(4) indépendant du type de machine.
    • Améliorations majeures dans les softupdates, incluant des correctifs pour la vitesse et la stabilité.
    • Corrections de bugs et améliorations dans les pilotes atapiscsi(4) et pciide(4), comme les contrôleurs UltraDMA/100 d'Intel, Promise et Highpoint.

>Comment installer

Suivent ici les instructions reproduites sur le livret qui se trouvent sur la jaquette du CDROM. Les instructions pour effectuer une installation par FTP (ou autre) sont très semblables, les instructions pour l'installation par le CDROM sont laissées telles quelles pour vous montrer qu'il est beaucoup plus simple d'installer depuis le CDROM.

Veuillez consulter les fichiers suivants sur les CDROMs pour plus de détails quant à l'installation d'OpenBSD 2.9 sur votre machine :

  • CD1:2.9/i386/INSTALL.i386
  • CD1:2.9/powerpc/INSTALL.powerpc
  • CD1:2.9/vax/INSTALL.vax

  • CD2:2.9/sparc/INSTALL.sparc
  • CD2:2.9/mvme68k/INSTALL.mvme68k
  • CD2:2.9/amiga/INSTALL.amiga
  • CD2:2.9/mac68k/INSTALL.mac68k
  • CD2:2.9/hp300/INSTALL.hp300
  • CD2:2.9/sun3/INSTALL.sun3

  • La version pour alpha est disponible depuis les sites FTP mais pas sur les CD.

Informations pour une installation rapide, faites pour les personnes familières avec OpenBSD, et avec l'utilisation de la commande "disklabel -E". Si vous n'êtes pas à l'aise avec l'installation d'OpenBSD, lisez les fichiers INSTALL.* correspondants cités ci-dessus !

OpenBSD/i386:

    Manipuler les options du BIOS, et voyez si vous pouvez démarrez depuis le CD, en utilisant le CD1. Si ce n'est pas possible, copiez CD1:2.9/i386/floppy29.fs sur une disquette, et démarrez avec ça.

    Si vous désirez mélanger OpenBSD avec un autre système d'exploitation, vous aurez sans doute besoin de lire INSTALL.i386.

    Pour créer une disquette depuis MS-DOS, utilisez /2.9/tools/rawrite.exe. Sous Unix, utilisez l'utilitaire dd(1). L'exemple suivant est un exemple d'utilisation de dd(1), où le périphérique peut être "floppy", "rfd0c", ou "rfd0a". 

      # dd if=<file> of=/dev/<device> bs=32k

    Utilisez une disquette correctement formatée SANS BLOCS DÉFECTUEUX ou vous n'y arriverez pas. Pour de plus amples informations sur la création d'une disquette de démarrage et sur l'installation d'OpenBSD/i386, veuillez lire la FAQ4.1.

OpenBSD/sparc:

    La version 2.9 de OpenBSD/sparc se trouve sur le CD2. Pour démarrer depuis le CD, vous pouvez utilisez l'une deux commandes listées ci-dessous, en fonction de votre ROM. 

      > boot cdrom 2.9/sparc/bsd.rd
or
> b sd(0,6,0)2.9/sparc/bsd.rd

    Si votre sparc n'a pas de lecteur de CD, vous pouvez démarrer depuis la disquette. Pour ce faire, vous devez écrire "CD2:2.9/sparc/floppy29.fs" sur une disquette. Pour plus d'information, voir FAQ4.1. Pour démarrer depuis la disquette, utilisez l'une des deux commandes listées ci-dessous, selon la version de votre ROM. 

      > boot floppy
or
> boot fd()

    Soyez certain d'utiliser une disquette correctement formatée SANS BLOC DÉFECTUEUX ou vous n'y arriverez pas.

    Si votre sparc n'a pas de lecteur de CD ou de lecteur de disquette, vous pouvez soit installer à l'aide d'une bande de lancement, soit via le réseau, comme indiqué dans le fichier INSTALL.sparc.

OpenBSD/amiga:

    Créer les partitions BSD suivant les instructions de la section préparation de INSTALL.amiga. Montez le CD2 sous AmigaOS en tant que périphérique CD0, ensuite exécutez la commande CLI suivante : "CD0:2.9/amiga/utils/loadbsd CD0:2.9/amiga/bsd.rd".

OpenBSD/hp300:

    Vous pouvez démarrer depuis le réseau en suivant les instructions de INSTALL.hp300.

OpenBSD/mac68k:

    Démarrer MacOS normalement et partitioner votre disque avec les configurations A/UX appropriées. Extrayez les utilitaires Macside de CD1:2.9/mac68k/utils sur votre disque dur. Lancer Mkfs pour créer votre système de fichiers sur les partitions A/UX que vous venez de faire. Ensuite utilisez l'installeur BSD/Mac68k pour copier tous les fichiers dans CD1:2.9/mac68k/ sur vos partitions. Pour finir, vous pourrez configurer le booter BSD/Mac68k avec l'emplacement de votre noyau et démarrer votre système.

OpenBSD/sun3:

    Vous pouvez soit faire un démarrage sans disque ou créer une bande d'installation, comme décrit dans INSTALL.sun3.

OpenBSD/alpha:

    Récupérer la version par ftp. Ensuite vous pouvez soit faire un démarrage sans disque ou par disquette comme décrit dans INSTALL.alpha.

Remarques à propos du code source :

    src.tar.gz contient une archive des sources se trouvant dans /usr/src. Ce fichier contient tout excepté les sources du noyau, qui sont dans une archive séparée. Pour l'extraire : 

      # mkdir -p /usr/src
# cd /usr/src
# tar xvfz /tmp/src.tar.gz

    srcsys.tar.gz contient une archive des sources se trouvant dans /usr/src/sys. Ce fichier contient toutes les sources du noyau nécessaires pour reconstruire un noyau. Pour l'extraire : 

      # mkdir -p /usr/src/sys
# cd /usr/src
# tar xvfz /tmp/srcsys.tar.gz

    Ces deux arborescences sont régulièrement mises à jour dans le CVS. En utilisant ces arborescences, il est possible d'utiliser les serveurs anoncvs immédiatement comme décris dans http://www.OpenBSD.org/fr/anoncvs.html. Utiliser ces fichiers est beaucoup plus rapide que d'obtenir la totalité des sources depuis un serveur anoncvs.

Arborescence des ports

Une archive de l'arborescence des ports est aussi fournie. Pour l'extraire : 

    # cd /usr
# tar xvfz /tmp/ports.tar.gz
# cd ports

Le sous-repértoire ports/ est un "checkout" de l'arborescence des ports OpenBSD. Veuillez lire la page concernant les ports si vous ne savez pas de quoi il s'agit. Ce texte n'est pas un manuel pour utiliser les ports. C'est plutôt un ensemble de notes permettant à l'utilisateur de commencer rapidement à utiliser le systèmes de ports OpenBSD.

Bien sur le système des ports d'OpenBSD n'est pas complet et ne le sera sans doute jamais. Cependant, il grossit rapidement et devient de plus en plus stable. Quasimment tous les ports fournis avec cette version devrait compiler sans problèmes sur la plupart des architectures (plus de 1200 paquetages compilés sur i386 par exemple).

Le répertoire ports/ représente une version CVS des ports (voir la page du manuel cvs(1) si vous n'êtes pas familier avec CVS). Comme pour l'arborescence des sources, notre arborescence des ports est disponible par anoncvs. Donc pour mettre à jour votre arborescence des ports/ depuis un media en lecture-écriture, il suffit de lancer la commande suivante : 

    # cd [portsdir]/; cvs -d anoncvs@server.openbsd.org:/cvs update -Pd -rOPENBSD_2_9

[Bien sur, vous devez remplacer le répertoire local et le nom du serveur ici par l'emplacement de votre collection des port et le serveur anoncvs le plus proche.]

Remarquez que la plupart des ports sont disponibles en tant que paquetages via ftp. Des mises à jour de paquetage pour la version 2.9 seront disponibles si des problèmes surviennent.

Si vous êtes intéressé par l'ajout d'un port, voulez aider, ou voulez juste en savoir plus, la liste de discussion ports@openbsd.org est la liste à connaître.
