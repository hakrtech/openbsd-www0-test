OpenBSD/cats

OpenBSD/cats fonctionne sur la Simtec Electronics StrongARM 110 Evaluation Board.

Une liste de diffusion dédiée aux ports à base d'ARM est disponible à l'adresse arm@openbsd.org. Pour vous inscrire à la liste de diffusion OpenBSD/arm, envoyez un message dont le corps contient "subscribe arm" à majordomo@openbsd.org. Veuillez lire notre politique des listes de diffusion avant de vous inscrire.

Le mainteneur actuel du port est Dale Rahn (drahn@openbsd.org). Si d'autres personnes souhaitent contribuer, elles sont bien entendu les bienvenues ! D'autres plates-formes ARM possibles sont en cours d'évaluation.

Historique :

OpenBSD/cats a été démarré en janvier 2004 à partir du code NetBSD -current.

Etat actuel :

La Simtec StrongARM 110 Evaluation Board est la seule carte supportée à l'heure actuelle.

X11 :

Le port est encore trop jeune pour parler du graphisme. Néanmoins, le mode texte de la console vga est supporté.

Projets (sans ordre particulier) :

  • Meilleur support du démarrage à partir de firmware ABLE (en cours).
  • Symbôles noyau (indisponible à l'heure actuelle en raison de limitations du processus de démarrage).

Périphériques supportés :

A l'heure actuelle, seul le matériel suivant a été testé :

  • Modules PCI intégrés (pci)
    • Acer Labs M5229 UDMA IDE
  • PCI-Bridges (pchb)
    • Acer Labs M1533 ISA
  • Cartes vidéo
    • S3 PCI VGA
  • Ethernet
    • Adaptateurs PCI à base de ASIX 88140A/88141 (dc), y compris :
      • CNet Pro110B
      • Alfa Inc. GFC2204
    • Novell NE1000, NE2000 (ISA, et les variantes PCI) (ne) (B)
    • Adaptateurs Ethernet VIA Rhine/RhineII/RhineIII (vr), y compris :
      • Addtron AEF-360TX
      • Hawking PN102TX
      • D-Link DFE530TX
    • DEC DECchip 21143 (clones Tulip) (dc)
    • Adaptateurs PCI à base d'Intel i8255x (excepté le i82556) (fxp), y compris :
      • Intel EtherExpress PRO/10+
      • Intel EtherExpress PRO/100, PRO/100B, and PRO/100+
      • Intel EtherExpress PRO/100+ "Management Adapter"
      • Intel EtherExpress PRO/100 Dual Port
      • Intel PRO/100 VE, PRO/100 VM, and PRO/100 S
    • Adaptateurs PCI à base d'Intel 21145 (dc)
  • Adaptateurs SCSI et IDE
    • Adaptateurs SCSI à base d'Adaptec AIC-7770, AIC-7850, AIC- 7860, AIC-7870, AIC-7880, AIC-7890, AIC-7891, AIC-7892, AIC-7895, AIC-7896, AIC-7897 et AIC-7899 (ahc) y compris les cartes Adaptec
      • AHA-274X[W,T]
      • AHA-284X
      • AHA-2910, AHA-2915
      • AHA-2920
      • AHA-2930[C,U2]
      • AHA-2940[J,N,U,AU,UW,UW Dual,UW Pro,U2W,U2B]
      • AHA-2950[U2W,U2B]
      • AHA-3940[U,AU,UW,AUW,U2W]
      • AHA-3950U2
      • AHA-3960
      • AHA-3985
      • AHA-4944UW
      • AHA-19160B
      • AHA-29160[B,N]
      • AHA-39160
    • Adaptateurs PCI à base de Symbios Logic (NCR) 53C8xx, 53C1010 et 53C1510D y compris les cartes génériques, d'anciennes cartes ASUS, la série DTC-3130 et les séries Diamond Fireport (siop)
    • Tout les adaptateurs PCI IDE (pciide) supportés par la plate-forme i386
  • Disques/Lecteurs
    • Tous les périphériques IDE supportés sur la plate- forme i386.
    • Tous les périphériques ATAPI supportés sur la plate-forme i386.
    • Tous les périphériques SCSI supportés sur la plate-forme i386.
  • Contrôleurs/Hubs USB

    Le support USB ne fonctionne pas correctement sur toutes les cartes à l'heure actuelle.

    • OHCI: Acer Labs M5237 USB (ohci)
  • Périphériques USB
    • Alps Electric Apple USB Keyboard (ukbd)
    • Solid Year Keyboard and Mouse (ukbd) (ums)
    • Logitech M4848 (ums)
    • Kensington USB/PS2 Wheel Mouse (ums)
    • Lecteurs Zip USB (umass)
  • Autres périphériques
    • Ports série : le port série intégré devrait fonctionner comme une console série fcom ou comme un port série standard.
