Ceci est la liste des errata & des correctifs d'OpenBSD 5.3:

Pour des informations concernant la branche stable d'OpenBSD, référez-vous ici.

Pour les errata concernant une version précise, cliquez ci-dessous:
2.1, 2.2, 2.3, 2.4, 2.5, 2.6, 2.7, 2.8, 2.9, 3.0, 3.1, 3.2, 3.3, 3.4, 3.5, 3.6,
3.7, 3.8, 3.9, 4.0, 4.1, 4.2, 4.3, 4.4, 4.5, 4.6, 4.7, 4.8, 4.9, 5.0, 5.1, 5.2, 5.4.
Vous pouvez également télécharger un fichier tar.gz contenant tous les correctifs suivants. Ce fichier est mis à jour une fois par jour.

Les correctifs ci-dessous sont disponibles sur CVS via la branche OPENBSD_5_3 stable.

Pour des informations plus détaillées à propos de l'installation de correctifs sur OpenBSD, consultez la FAQ OpenBSD.

  • 001: CORRECTIF DE FIABILITE: 15 mars 2013   Toutes les architectures
    Une condition rare durant le démarrage des sessions peut faire que bgpd remplace une session active, menant à des conséquences inconnues. Ce bogue a été trouvé par inspection (nous ne savons pas comment le reproduire, considérez cela comme un challenge).
    Un correctif corrigeant le problème au niveau du code source est disponible.

  • 002: CORRECTIF DE FIABILITE: 5 mai 2013   Toutes les architectures
    Une vulnérabilité présente dans le pilote vr(4) pourrait provoquer un arrêt de celui-ci suite à des erreurs.
    Un correctif corrigeant le problème au niveau du code source est disponible.

  • 003: CORRECTIF DE FIABILITE: 17 mai 2013   Toutes les architectures
    Un problème existe dans le processus nginx(8) si proxy_pass est utilisé avec des serveurs backend HTTP non fiables. Ce problème pourrait mener à un déni de service ou à l'exposition de la mémoire du processus lors de la réception d'une requête spécialement conçue à cette fin. Ce problème est assigné CVE-2013-2070.
    Un correctif corrigeant le problème au niveau du code source est disponible.

  • 004: CORRECTIF DE FIABILITE: 17 mai 2013   Toutes les architectures
    Une découverte de Peter Philipp montre qu'il est possible qu'un processus utilisateur supprime la route RNF_ROOT supposée insupprimable, résultant en un kernel panic.
    Un correctif corrigeant le problème au niveau du code source est disponible.

  • 005: CORRECTIF DE FIABILITE: 31 mai 2013   Toutes les architectures
    Un déni de service local est susceptible de survenir si un utilisateur (privilégié ou non) effectue un ioctl SIOCSIFADDR sur une socket AF_INET6 avec un paramètre malformé.
    Un correctif corrigeant le problème au niveau du code source est disponible.

  • 006: CORRECTIF DE FIABILITE: 12 juin 2013   Toutes les architectures
    Un déni de service a été découvert dans TFTP. La mauvaise gestion d'options TFTP à la réception d'un paquet pourrait causer un double-free, faisant crasher le service.
    Un correctif corrigeant le problème au niveau du code source est disponible.

  • 007: CORRECTIF DE FIABILITE: 12 juin 2013   Toutes les architectures
    Deux vulnérabilités dans le pilote vio(4) pourraient causer un kernel panic et créer des problèmes de réception multicast avec IPv6.
    Un correctif corrigeant le problème au niveau du code source est disponible.

  • 008: CORRECTIF DE FIABILITE: 7 novembre 2013   Toutes les architectures
    Un crash peut survenir lors de la destruction d'une interface pflow(4).
    Un correctif corrigeant le problème au niveau du code source est disponible.

  • 009: CORRECTIF DE SECURITE: 7 novembre 2013   Toutes les architectures
    Une vulnérabilité de corruption de mémoire a été découverte dans le processus post-authentification de sshd lorsqu'un cipher AES-GCM (aes128-gcm@openssh.com ou aes256-gcm@openssh.com) est sélectionné durant l'échange kex. Reportez-vous au bulletin sur gcmrekey pour de plus amples informations.
    Un correctif corrigeant le problème au niveau du code source est disponible.

  • 010: CORRECTIF DE FIABILITE: 11 novembre 2013   Toutes les architectures
    Un utilisateur non-privilégié pourrait bloquer le système.
    Un correctif corrigeant le problème au niveau du code source est disponible.

  • 011: CORRECTIF DE SECURITE: 21 novembre 2013   Toutes les architectures
    Un problème existe dans nginx(8) qui pourrait permettre à un attaquant d'outrepasser les restrictions de sécurité dans certaines configurations en effectuant des requêtes spécialement forgées. Ce problème a été assigné CVE-2013-4547.
    Un correctif corrigeant le problème au niveau du code source est disponible.

  • 012: CORRECTIF DE FIABILITE: 19 décembre 2013   Architectures à alignement strict
    Dans OpenSSL, l'utilisation de l'algorithme SSL/TLS SHA384 pourrait résulter en un crash de l'application. Les plates-formes i386, amd64, vax et m68k ne sont pas affectées.
    Un correctif corrigeant le problème au niveau du code source est disponible.

  • 013: CORRECTIF DE SECURITE: 10 janvier 2014   Buffer overflow lors du parsage de fichiers BDF
    Un fichier de police BDF contenant une chaîne de caractère plus longue que prévu pourrait overflower un buffer sur la stack dans le serveur X. Cette vulnérabilité a été assignée CVE-2013-6462.
    Un correctif corrigeant le problème au niveau du code source est disponible.
