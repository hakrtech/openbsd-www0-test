FTP

OpenBSD organise les miroirs FTP selon un principe de maître/esclave. Un site maître est utilisé pour alimenter des miroirs de second niveau. Ces derniers servent à alimenter les autres sites miroirs. Les miroirs de second niveau sont choisis à partir d'une liste de sites dotés d'une bonne connectivité réseau et maintenu par des personnes connues par la communauté de développement OpenBSD. Si vous souhaitez maintenir un miroir FTP, veuillez synchroniser votre serveur avec un miroir de second niveau proche de vous. La synchronisation à partir du site maître est vivement découragée.

Disposition d'un miroir

La structure des fichiers de chaque serveur est en gros :

  • .../OpenBSD/3.6/
    Où vous trouverez notre dernière version.
    • alpha/
    • amd64/
    • hp300/
    • hppa/
    • i386/
    • luna88k/
    • mac68k/
    • macppc/
    • mvme68k/
    • mvme88k/
    • sparc/
    • sparc64/
    • vax/
    • src.tar.gz
       Sources des programmes utilisateurs, que vous placerez dans /usr/src (ceci n'inclut pas les sources du noyau)
    • sys.tar.gz
       Les sources du noyau, que vous placerez dans /usr/src/sys
    • ports.tar.gz
       Les sources des ports, que vous placerez dans /usr/ports
    • XF4.tar.gz
       Les sources de XFree86 (X11)

  • .../OpenBSD/snapshots/
    Pour nos architectures principales, nous construisons des mini versions dont la stabilité et la qualité sont inconnues environ tous les mois. C'est l'endroit où il faut rechercher ces versions.

  • .../OpenBSD/src/
    Il s'agit d'une version des sources par cvs faite tous les jours pour ceux qui veulent accéder aux fichiers individuellement. Normalement, nous recommandons l'usage de CTM ou AnonCVS plutôt que de faire un miroir de cette partie. Ainsi, tous les miroirs FTP ne font pas un miroir de ce site.

  • .../OpenBSD/doc/
    La FAQ OpenBSD au format PDF et texte simple, ainsi que d'autres documents relatifs à OpenBSD.

  • .../OpenBSD/tools/
    Autres petites choses.
  • .../OpenBSD/3.5/
    Disposition similaire à la version 3.6 décrite ci-dessus...
  • .../OpenBSD/3.4/
    Disposition similaire à la version 3.6 décrite ci-dessus...
  • .../OpenBSD/3.3/
    Disposition similaire à la version 3.6 décrite ci-dessus...
  • .../OpenBSD/3.2/
    Disposition similaire à la version 3.6 décrite ci-dessus...
  • .../OpenBSD/3.1/
    Disposition similaire à la version 3.6 décrite ci-dessus... pour les nostalgiques.
  • .../OpenBSD/3.0/
    Disposition similaire à la version 3.6 décrite ci-dessus... pour les nostalgiques.
  • .../OpenBSD/2.9/
    Disposition similaire à la version 3.5 décrite ci-dessus... pour les nostalgiques.

Les personnes désirant mettre en place de nouveaux miroirs FTP/ ANONCVS seront intéressées par l'estimation de la place disque de ce qui est maintenu à ftp://ftp.openbsd.org:/pub/OpenBSD/.
La politique en cours requiert que chaque miroir listé sur cette page fournisse au minimum les deux dernières versions sous forme binaire (actuellement 3.5 et 3.6) pour toutes les architectures ainsi que les répertoires OpenSSH/, OpenNTPD/, OpenBGPD/, patches/ et tools/.
Selon l'espace disque disponible, les miroirs peuvent fournir d'autres ressources telles que les snapshots, les anciennes versions, la dernière arborescence des sources, etc.
L'utilisation actuelle de l'espace disque sur le site ftp maître est la suivante : 


                      actuel    pourrait être
pub/OpenBSD/3.2       11.8GB    11.8GB       (OpenBSD version 3.2)
pub/OpenBSD/3.3       14.0GB    14.0GB       (OpenBSD version 3.3)
pub/OpenBSD/3.4       11.0GB    14.0GB       (OpenBSD version 3.4)
pub/OpenBSD/3.5       14.5GB    15.0GB       (OpenBSD version 3.5)
pub/OpenBSD/3.6       18.0GB    18.0GB       (OpenBSD version 3.6)
pub/OpenBSD/snapshots 14.4GB    plus grand!  (derniers snapshots, d'une 
                                             taille normalement >= à la 
                                             dernière version)
pub/OpenBSD/src         .6GB      .7GB       (arborescence des sources 
                                             téléchargeable via ftp 
                                             et mise à jour toutes les
                                             nuits à partir de anoncvs.
                                             Il serait préférable d'éviter
                                             l'utilisation d'un miroir ftp
                                             pour la télécharger et de 
                                             la mettre à jour directement
                                             à partir d'un serveur anoncvs
                                             proche.)
Un miroir complet requiert donc approximativement 76GB d'espace disque mais si on se limite aux dernières versions, l'espace requis n'est que de 52GB. Il est à noter que le site FTP maître fournit aussi les répertoires suivants que les miroirs ne devraient pas avoir besoin de fournir : 

pub/OpenBSD/distfiles  4.7GB   plus grand!  (distfiles utilisés par le
                                            système des ports. Vous ne
                                            devriez normalement *pas* faire un 
                                            miroir de ceci à moins que vous ne vouliez
                                            ces fichiers pour des raisons diverses.
                                            Les makefiles des ports n'iront
                                            par défaut que dans 
                                            ftp.openbsd.org et ftp.usa.openbsd.org)

Si vous voulez ajouter un nouveau site miroir FTP, HTTP, AFS ou RSYNC, veuillez contacter www@openbsd.org.
