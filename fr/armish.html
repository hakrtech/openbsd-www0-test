OpenBSD/armish

OpenBSD/armish fonctionne sur plusieurs systèmes embarqués d'architecture ARM utilisant le chargeur de démarrage Redboot (qui démarre Linux par défaut).

Une liste de diffusion pour les ports basés sur l'architecture ARM est disponible sur arm@openbsd.org. Pour vous abonner à la liste de diffusion OpenBSD/arm, envoyez un message contenant dans son corps "subscribe arm" à majordomo@openbsd.org. Nous vous prions de vous assurer d'avoir consulté la politique des listes de diffusion avant de vous abonner.

Le mainteneur actuel de ce portage est Dale Rahn (drahn@openbsd.org).

Historique :

OpenBSD/armish est le 3ème portage d'OpenBSD vers des machines ARM, après cats et zaurus. Il prévoit se supporter une variété de machines ARM similaires :

  • Certance CP3100
  • Thecus N2100 (et versions renommées : Allnet ALL6500, Evesham SilverSTOR M-Box)
  • Thecus N4100 (et versions renommées : Allnet ALL6400, Evesham SilverSTOR XS)
  • Séries IO-Data HDL-Gxxx, HDL-GWxxx, and HDL-GZxxx

Etat actuel :

Le développement est actuellement très actif. Le Thecus démarre du réseau (en utilisant TFTP ou HTTP, grâce à redboot) et tourne en mode multi utilisateurs.

Les applications et bibliothèques clientes X sont disponibles.

Projets (sans ordre particulier) :

  • Finir le support pour le Certance CP3100
  • Supporter plus de matériel
  • Support des périphériques i2c sur le Thecus
  • Support de la configuration du flash Redboot
  • Démarrage natif à partir du disque...
