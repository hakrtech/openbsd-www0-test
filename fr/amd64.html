OpenBSD/amd64

OpenBSD/amd64 fonctionne sur les processeurs AMD de la famille Athlon-64 en mode 64-bit.

Ce port fonctionne aussi sur les processeurs Intel ia32e qui supportent les extensions AMD64. Malheureusement, Intel a abandonné le support du bit NXE (No-EXecute) de la table de pagination. W^X n'est donc pas supporté sur les processeurs Intel. Ce port fonctionne aussi sur des processeurs provenant d'autres fabricants ayant cloné les extensions AMD64. (Certains processeurs Intel ne supportent pas le bit PAE NX. Ce qui signifie que les machines utilisant ce type de processeurs fonctionneront sans support W^X -- Il est donc plus sûr de les faire fonctionner en mode i386). Cette famille de processeurs fonctionne aussi avec OpenBSD/i386 mais uniquement en mode 32-bit.

Le support du système X Window est disponible pour la plupart des cartes graphiques en utilisant le serveur X.Org. Comme d'autres systèmes d'exploitation libres il est hautement recommandé de ne plus utiliser de cartes Nvidia puisque ce vendeur continue de montrer une résistance tenace pour fournir des informations qui permettrait à X.Org de supporter ce matériel correctement.

Etat actuel :

Le seul problème majeur subsistant à l'heure actuelle est la pauvreté relative du débogueur noyau ddb.

Matériel supporté :

Processeurs

Toutes les versions des processeurs AMD Athlon 64 et leurs clones sont supportés.

Support SMP

A partir de la version OpenBSD 3.6, OpenBSD/amd64 supporte la plupart des systèmes SMP ("Symmetrical MultiProcessor"). Pour le fonctionnement en mode SMP, un noyau SMP séparé (bsd.mp) est fourni avec les ensembles de fichiers d'installation.

Cette fonctionnalité étant nouvelle, les développeurs souhaitent avoir un retour sur le matériel SMP qui fonctionne ou ne fonctionne pas. Il est donc très important d' envoyer un dmesg après l' installation.

Bus

  • Tous les ponts PCI standards
  • PC Cards PCMCIA 16-Bit
  • PC Cards CardBus 32-Bit
  • Universal Serial Bus (USB)

Interfaces pour bus

  • Ponts standards PCI-PCI, incluant les backplanes d'extension PCI (ppb)

Sources d'entropie

  • Récupération du temps de latence entre les interruptions venant de nombreux périphériques.

Obtenir et installer OpenBSD/amd64

La dernière version disponible et supportée d'OpenBSD/amd64 est OpenBSD 4.9. Voici les instructions d'installation d'OpenBSD/amd64 4.9.

De temps à autre, des snapshots sont mis à disposition, à cet endroit ainsi que sur quelques miroirs. Et voici les instructions d'installation des snapshots OpenBSD/amd64 .

Plusieurs média d'installation sont fournis :

  • Démarrage à partir du CD
    Le CD fournit une image amorçable El Torito sans émulation incluant pratiquement tous les pilotes OpenBSD. Cette image inclut aussi un support USB minimal (périphériques de stockage et clavier).
    Pour la liste la plus récente des pilotes disponibles sur cette image, consultez le fichier de configuration noyau RAMDISK_CD.
  • Floppy A (floppy49.fs)
    Cette image de disquette de 1.44MB contient les pilotes les plus communs. Elle a été conçue pour le PC le plus usuel. En règle générale, les pilotes manquants sont ceux nécessitant des microcodes conséquents pour fonctionner tels que les pilotes de certaines cartes SCSI rares, des cartes gigabit ethernet, ou des périphériques RAID.
    Pour la liste la plus récente des pilotes disponibles sur cette image, consultez le fichier de configuration noyau RAMDISK.
