OpenBSD/sun3

Le port OpenBSD/sun3 fonctionne actuellement sur tous les ordinateurs Sun3 à base de 68020, à l'exception des modèles 3/E.

Histoire et status :

Le port Sun3 d'OpenBSD est dérivé du port NetBSD/sun3, à l'époque où les port sun3 et sun3x étaient gérés séparément. Les petits bouts d'OpenBSD qui dont totalement spécifiques au port Sun3 sont globalement les mêmes que leurs équivalents chez NetBSD.

Matériel supporté :

OpenBSD/sun3 fonctionne sur les modèles de Sun3 suivants :
  • Sun-3/50
  • Sun-3/60, 3/60LE
  • Sun-3/75, 3/140, 3/150, 3/160 et 3/180
  • Sun-3/110
  • Sun-3/260 et 3/280
Les autres modèles de Sun3 ne sont PAS supportés, spécialement ceux basés sur 68030 (sun3x). OpenBSD ne fonctionne pas sur les machines 3/160 avec uniquement 2Mb de mémoire, et très difficilement sur les machines à 4Mb.

Périphériques supportés :

  • Ports série intégrés
  • Clavier et souris Sun type 3
  • Vidéo intégré ou VME (bwtwo, cgtwo, cgfour)
  • Ethernet intégré ou VME (intel, lance)
  • Contrôleur scsi sun-3 intégré ou VME On-board , actuellement avec quelques limitations
  • Les cartes VME xylogics sont reconnues mais ne fonctionnent pas pour l'instant
Le contrôleur scsi sun-2 n'est pas actuellement supporté.

La version complète d'OpenBSD/sun3 est disponible par ftp ; les snapshots sont disponibles de façon irrégulière.
Vous devriez lire les notes d'installation pour cette version avant d'essayer d'installer OpenBSD/sun3.

Le premier démarrage du système prendra un temps certain, du fait de la génération d'un clé DSA pour ssh. Ceci peut prendre jusqu'à 15 minutes sur une 3/160, mais cette durée peut beaucoup varier suivant votre machine. Ceci n'a lieu qu'une seule fois et il n'y a pas lieu de s'en soucier.

Vous pouvez utiliser cvs pour obtenir le système X Window. Kenneth Stailey a écrit des patches pour OpenBSD/sun3, qui aurait du être intégré dans les sources principales de Xfree86. Les clients X et les bibliothèques sont disponibles, mais il n'y a pas encore de serveur X.

Projets en cours (sans ordre particulier) :

  • périphériques "call up" (/dev/cua*).
  • pilote scsi sun-2 (sc).
  • migrer le port du système de mémoire VM en UVM (en progrès)
  • récrire le chargeur de boot et les procédure d'installation pour partager le plus de code possible avec le port sparc.

Le port Sun3 d'OpenBSD est maintenu par Miodrag Vallat, miod@openbsd.org.
