i386

Installation avec d'autres systèmes d'exploitation :

Le port d'OpenBSD/i386 peut être installé de façon à partager les disques du système avec d'autres systèmes d'exploitation tels que MS-DOS, Windows ou Linux en utilisant le système de partitionnement (MBR) de MS-DOS ainsi qu'un système permettant de sélectionner le système à démarrer. L'utilisation journalière ne pose aucun problème, par contre, l'installation nécessite plus d'attention. En effet, l'installation de Windows 95 ne se soucie pas du partitionnement/MBR déjà existant. Veuillez consulter les notices d'installation pour plus de détails.

Média d'installation Plusieurs options d'installation existent :
  • Démarrage à partir du CD :

    Démarrer à partir du CD permet d'obtenir l'équivalent d'une disquette El Torido 2,88Mo qui contient presque tous les pilotes d'OpenBSD.

  • Disquette A :

    Cette disquette 1,44Mo contient les pilotes les plus courants. En règle générale, les pilotes qui manquent sont ceux qui nécéssitent de gros microcodes pour tourner. Cela inclut quelques unes des cartes SCSI les plus rares, les cartes Gigabit Ethernet ou les périphériques RAID.

  • Disquette B :

    Cette disquette contient les pilotes qui sont manquants sur la disquette A. Pour des questions de place, vous verrez que certains périphériques sont aussi manquants. En particulier, le support pour les périphériques PCMCIA est manquant ainsi que pour les périphériques matériels les plus vieux. D'un certain côté, cette disquette est faite pour aider les personnes qui veulent mettre en place des systèmes modernes, étant donné qu'elle contient à peu près tous les pilotes SCSI et RAID.

  • Disquette C :

    Cette disquette contient la plupart des pilotes Cardbus et PCMCIA. C'est un excellent choix pour la plupart des portables. Dans la plupart des cas, la disquette A fonctionne aussi, mais certains portables bénéficieront grandement de cette disquette.

Utilisez la méthode qui vous convient le mieux.

Compatibilité binaire avec d'autres systèmes d'exploitations :

Le port OpenBSD/i386 peut faire tourner les binaires Linux, FreeBSD, BSD/OS, SVR4 (incluant Solaris) et IBCS2.

Matériels supportés :

Le port OpenBSD/i386 fonctionne sur un grand nombre de PC et différents clones, ainsi que sur une très grande variété de processeurs et de bus d'E/S différents. Il est fait pour être installé et fonctionner avec un minimum de difficulté sur la plupart des produits existants. Les cas où des problèmes ont été rencontrés sont typiquement le cas de vieux matériels propriétaires PC, les portables ou les serveurs spécialisés qui reposent sur un BIOS fait sur mesure.

OpenBSD ne supporte pour l'instant le multi-processeurs (SMP) mais peut tourner avec un processeur sur un carte mère multi-processeurs.

La plupart des portables sont supportés. Nous maintenons une page qui contient les détails concernant les portables les plus courants.

Tous les processeurs compatibles avec l'architecture Intel 80386 (i386) :

  • 386/486 (SX/DX/DX2/DX4)
  • Intel Pentium/Pentium-MMX
  • Intel Pentium Pro/II/III/Celeron/Xeon
  • AMD 6x86
  • AMD K5/K6/K6-2/K6-3
  • AMD Athlon
  • Cyrix MediaGX/M1/M2
  • Rise mP6

  • NexGen 586
  • VIA Joshua
Tout ce qui est un clone du 386 (ou mieux) devrait fonctionner correctement. Le seul CPU qui est connu comme ne fonctionnant pas correctement (à cause d'un mauvais design des cartes mères) est le Cyrix 386DLC.

Bus

  • Toutes les machines basées sur des bus ISA, EISA, VLB, et PCI incluant...
    • Les machines basées sur l'Intel 450GX/KX
    • Les machines basées sur l'Intel 450NX
    • Les machines basées sur le chipset ServerWorks
    (Nous ne mentionnons que ces trois dernières car elles sont difficiles à supporter).
  • À la fois les cartes PCMCIA 16-bit et les nouvelles cartes CardBus 32-Bit.
  • Universal Serial Bus (USB).
Le bus MCA qui se trouvent dans certains PC IBM officiels n'est pas supporté.

  • Ponts PCI-CardBus compatibles YENTA (cbb)

(2) Le support pour les périphériques marqués d'un (2) EST inclus dans les noyaux "generic", mais ne se trouvent pas sur le noyau des disquettes d'installation.

Matériels non supportés

Le matériel non supporté peut l'être ultérieurement, vos dons de temps, matériels ou documentation peuvent accélerer le support !

  • Le bus MCA "Micro Channel" utilisé dans beaucoup de modèles IBM PS/2.
  • Les adaptateurs SCSI basés sur le NCR 5380 et le 53400.
  • Lecteurs de bandes QIC-40 et QIC-80 (il s'agit des lecteurs de bandes qui se connectent sur le contrôleur du lecteur de disquettes).
  • Cartes-mères multi-processeurs (bien qu'elles fonctionnent très bien avec un seul processeur). Il existe un projet qui vise à les supporter.
  • Les interfaces CD-ROM propriétaires de Sony et Panasonic
  • Les lecteurs ZIP sur port parallèle (Les lecteurs ZIP sur SCSI et ATAPI fonctionnent bien)
  • Les adaptateurs Ethernet Intel i82556 (EtherExpress PRO/100A) et i82596 (EtherExpress PRO/10 PCI)
  • Les adaptateurs Ethernet Hewlett-Packard PC-LAN+ (HP27xxx)
  • Les périphériques son SoundBlaster Live! (EMU10k1)
  • Les périphériques son basés sur l'ESS Maestro
  • Les périphériques basés sur le Yamaha DS1.
  • Les périphériques son basés sur le Cirrus Logic CS4281 (bien que le CS4280 fonctionne)
  • Cartes son Aureal Vortex
  • Winmodems
  • Les périphériques infrarouge, tels que ceux qui se trouvent sur les portables.
  • Firewire
  • Accélérateurs cryptographiques 3C990 (3Con ne rendra pas public ses spécifications)
  • Adaptateurs SCSI Initio
  • Adaptec RAID, AMI RAID, Mylex RAID, Compaq RAID
  • PCMCIA:
    • Cartes mémoires
    • La plupart des cartes multifonctions
