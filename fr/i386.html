i386

Installation avec d'autres systèmes d'exploitation :

Le port d'OpenBSD/i386 peut être installé de façon à partager les disques du système avec d'autres systèmes d'exploitation tels que MS-DOS, Windows ou Linux en utilisant le système de partitionnement (MBR) de MS-DOS ainsi qu'un système permettant de sélectionner le système à démarrer. L'utilisation au jour le jour ne pose aucun problème, par contre, l'installation nécessite plus d'attention. En effet, l'installation de Windows 95 ne se soucie pas du partitionnement/MBR déjà existant. Veuillez consulter le Guide d'Installation pour plus de détails.

Média d'installation

Plusieurs options d'installation existent :
  • Démarrage à partir du CD :

    Démarrer à partir du CD permet d'obtenir l'équivalent d'une disquette El Torito 2,88Mo qui contient presque tous les pilotes d'OpenBSD. Ceci inclut un support USB minimal (périphériques umass et ukbd).
    Pour obtenir la liste la plus récente des pilotes disponibles dans cette image, regardez le fichier de configuration RAMDISK_CD.

  • Disquette A :

    Cette disquette 1,44Mo contient les pilotes les plus courants. Elle a été faite pour supporter les configurations PC les plus courantes. En règle générale, les pilotes qui manquent sont ceux qui nécessitent de gros microcodes pour tourner, telles que ceux pour les cartes SCSI les plus rares, les cartes Gigabit Ethernet ou les périphériques RAID.
    Pour obtenir la liste la plus récente des pilotes disponibles dans cette image, regardez le fichier de configuration RAMDISK.

  • Disquette B :

    Cette disquette contient les pilotes qui sont manquants sur la disquette A. Cette image est faite pour aider les gens installant des systèmes de type serveurs avec du matériel récent. Elle contient le support pour pratiquement tout les pilotes SCSI et RAID, les cartes ethernet gigabit et autres. Pour des questions de place, vous verrez que certains périphériques sont aussi manquants. En particulier, le support pour les périphériques PCMCIA est manquant ainsi que pour les périphériques matériels les plus vieux.
    Pour obtenir la liste la plus récente des pilotes disponibles dans cette image, regardez le fichier de configuration RAMDISKB.

  • Disquette C :

    Cette disquette contient la plupart des pilotes Cardbus et PCMCIA. Cette image est faite pour des installations sur des portables. Dans la plupart des cas, la disquette A fonctionne aussi, mais certains portables bénéficieront grandement de cette disquette.
    Pour obtenir la liste la plus récente des pilotes disponibles dans cette image, regardez le fichier de configuration RAMDISKC.

Utilisez la méthode qui vous convient le mieux.

Compatibilité binaire avec d'autres systèmes d'exploitation :

Le port OpenBSD/i386 peut faire tourner les binaires Linux, FreeBSD, BSD/OS, SVR4 (y compris Solaris) et IBCS2.

Matériel supporté :

Le port OpenBSD/i386 fonctionne sur un grand nombre de PC et différents clones, ainsi que sur une très grande variété de processeurs et de bus d'E/S différents. Il est fait pour être installé et fonctionner avec un minimum de difficulté sur la plupart des produits existants. Les cas où des problèmes ont été rencontrés sont typiquement le cas de vieux matériels propriétaires PC, les portables ou les serveurs spécialisés qui reposent sur un BIOS fait sur mesure.

OpenBSD ne supporte pour l'instant le multi-processeurs (SMP) mais peut tourner avec un processeur sur une carte mère multi-processeurs.

La plupart des portables sont supportés. Nous maintenons une page qui contient les détails concernant les portables les plus courants.

La liste du matériel supporté correspond à OpenBSD-current. Elle diffère du support fournit par la dernière version.

Tous les processeurs compatibles avec l'architecture Intel 80386 (i386) :

  • 386/486 (SX/DX/DX2/DX4)
  • Intel Pentium/Pentium-MMX
  • Intel Pentium Pro/II/III/Celeron/Xeon
  • Intel Pentium IV
  • AMD 6x86
  • AMD K5/K6/K6-2/K6-3
  • AMD Athlon/Duron
  • Cyrix MediaGX/M1/M2
  • VIA Cyrix III
  • Rise mP6
  • IDT WinChip
  • NexGen 586
  • VIA Joshua
  • Transmeta TMS3200, TMS5400, TMS5600
Tout ce qui est un clone du 386 (ou mieux) devrait fonctionner correctement. Le seul CPU qui est connu comme ne fonctionnant pas correctement (à cause d'une mauvaise conception des cartes mères) est le Cyrix 386DLC.

Bus

  • Toutes les machines basées sur des bus ISA, EISA, VLB, et PCI, y compris :
    • Les machines basées sur l'Intel 450GX/KX.
    • Les machines basées sur l'Intel 450NX.
    • Les machines basées sur le chipset ServerWorks.
    (Nous ne mentionnons que ces trois dernières car elles sont les plus difficiles à supporter).
  • À la fois les cartes PCMCIA 16-bit et les nouvelles cartes CardBus 32-Bit.
  • Universal Serial Bus (USB).
Le bus MCA qui se trouvent dans certains PC IBM officiels n'est pas supporté.

Interfaces pour les Bus

  • Les ponts PCI-PCI standards, y compris les backplanes d'extension PCI (ppb)
  • Ponts PCI-CardBus compatibles YENTA (cbb)

  • Le générateur de nombres aléatoires Intel 82802 existant sur les systèmes i810, i815, i820, i840, i850 et i860.

Matériel non supporté

Le matériel non supporté peut l'être ultérieurement, vos dons de temps, matériels ou documentation peuvent accélerer le support !

  • Le bus MCA "Micro Channel" utilisé dans beaucoup de modèles IBM PS/2.
  • Les adaptateurs SCSI basés sur le NCR 5380 et le 53400.
  • Lecteurs de bandes QIC-40 et QIC-80 (il s'agit des lecteurs de bandes qui se connectent sur le contrôleur du lecteur de disquettes).
  • Cartes-mères multi-processeurs (bien qu'elles fonctionnent très bien avec un seul processeur). Il existe un projet qui vise à les supporter.
  • Les interfaces CD-ROM propriétaires de Sony et Panasonic
  • Les lecteurs ZIP sur port parallèle (Les lecteurs ZIP sur SCSI et ATAPI fonctionnent bien)
  • Les adaptateurs Ethernet Intel i82556 (EtherExpress PRO/100A, Intel PRO/100 EISA et PRO/100 Smart) et i82596 (EtherExpress PRO/10 PCI) (mickey y travaille)
  • Les adaptateurs Ethernet Hewlett-Packard PC-LAN+ (HP27xxx)
  • Les périphériques son SoundBlaster Live! (EMU10k1) (mickey y travaille)
  • Cartes son Aureal Vortex (mickey y travaille)
  • Winmodems
  • Les périphériques infrarouge, tels que ceux qui se trouvent sur les portables.
  • Firewire
  • Contrôleurs RAID Intel et Mylex, ceux ne supportant pas l'interface I2O
  • PCMCIA:
    • Cartes mémoires
    • La plupart des cartes multifonctions

Si vous cherchez à tester des fonctionnalités avant un nouvelle version d'OpenBSD, vous pouvez essayer un des snapshots. Pour l'architecture i386, des snapshots sont disponibles de temps en temps.
