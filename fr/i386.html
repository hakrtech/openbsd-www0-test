OpenBSD/i386

OpenBSD/i386 fonctionne sur le PC standard et ses clones, supportant une large variété de processeurs, d'architectures E/S et périphériques.

Historique :

Le port OpenBSD/i386 a été présent dès le commencement du projet. A travers les années, de plus en plus de support matériel a été écrit spécifiquement pour OpenBSD (tels que les pilotes d'accélérations cryptographiques), ou a été adapté à partir de FreeBSD et NetBSD.

Etat actuel :

On peut s'attendre à ce qu'il puisse s'installer et fonctionner avec un minimum de difficultés sur la plupart des matériels actuels. Les machines utilisant des BIOS adaptés (tels que les anciens PC propriétaires, les portables, et quelques machines serveurs spécialisées) peuvent causer des problèmes.

A travers sa couche de compatibilité binaire, OpenBSD/i386 peut faire fonctionner les binaires Linux, FreeBSD, BSD/OS, SVR4 (y compris Solaris) et IBCS2, jusqu'à un certain degré. La plupart des applications fonctionneront sans problèmes.

Le support XFree86 est disponible pour la plupart des cartes graphiques, en utilisant soit une version récente des serveurs XFree86 (4.4.0 dans la version OpenBSD 3.5) quand c'est possible ou des serveurs XFree86 3.3.6 autrement.

Matériel supporté :

La plupart des portables sont supportés. Nous avons une page qui donne les spécificités de certains portables communs.

La liste du matériel supporté est établie par rapport à OpenBSD-current. Elle sera légérement différente de la liste fournie par la dernière version.

Processeurs

Tous les processeurs compatibles avec l'architecture Intel 80386 (i386) sont supportés :

  • 80386 (SX/DX) avec une FPU 80387 (SX/DX)
  • 80486 (DX/DX2/DX4)
  • Intel Pentium/Pentium-MMX
  • Intel Pentium Pro/II/III/Celeron/Xeon
  • Intel Pentium IV
  • AMD 5x86
  • AMD K5/K6/K6-2/K6-3
  • AMD Athlon/Duron
  • AMD Athlon 64 (Consultez OpenBSD/amd64 concernant le support 64-bit)
  • Cyrix MediaGX/M1/M2
  • Cyrix 6x86
  • VIA C3
  • Rise mP6
  • IDT WinChip et C3
  • NexGen 586
  • NS Geode GX1 et M1
  • Transmeta TMS3200, TMS5400, TMS5600

Les coprocesseurs de calcul à virgule flottante habituels (80387DX, 80387SX, 80487SX) sont nécessaires lorsqu'ils ne sont pas intégrés au processeur.

Tout ce qui est un clone du 386 (ou mieux) devrait fonctionner correctement. Le seul CPU qui est connu comme ne fonctionnant pas correctement (à cause d'une mauvaise conception des cartes mères) est le Cyrix 386DLC.

Support SMP

OpenBSD/i386 -current (ET NON 3.5-release/-stable) supporte la plupart des systèmes SMP ("Symmetrical MultiProcessor"). Pour le fonctionnement en mode SMP, Un noyau SMP séparé (bsd.mp) est fourni avec les ensembles de fichiers d'installation.

Vu que cette fonctionnalité est nouvelle, les développeurs souhaitent avoir un retour sur le matériel SMP qui fonctionne ou ne fonctionne pas. Il est donc très important d'envoyer un dmesg après installation.

Bus

  • Tous les bus ISA, EISA, VLB, et PCI standards y compris :
    • Intel 450GX/KX
    • Intel 450NX
    • ServerWorks chipset
    (Nous ne mentionnons que ces trois dernières car elles sont les plus difficiles à supporter).
  • Cartes PCMCIA PC Cards 16-Bit
  • Cartes CardBus PC Cards 32-Bit
  • Universal Serial Bus (USB)
  • Inter IC (I2C) bus *

Le bus MCA qui se trouve dans certains PC IBM officiels n'est pas supporté.

Interfaces pour les bus

  • Les ponts PCI-PCI standards, y compris les backplanes d'extension PCI (ppb)
  • Ponts PCI-CardBus compatibles YENTA (cbb)

Sources d'entropie

  • Récupération du temps de latence entre les interruptions venant de nombreux périphériques.
  • Le générateur de nombres aléatoires Intel 82802 existant sur les systèmes i810, i815, i820, i840, i850 et i860.

Installation avec d'autres systèmes d'exploitation :

OpenBSD/i386 peut être installé de telle façon à partager les disques système avec d'autres systèmes d'exploitation, tels que MS-DOS, Windows ou Linux, en utilisant le système de partitionnement MBR/partition et un gestionnaire de sélection du système à démarrer en option. L'utilisation au jour le jour ne pose aucun problème, par contre, l'installation nécessite plus d'attention. En effet, l'installation de Windows 95 ne se soucie pas du partitionnement/MBR déjà existant.

De même, certains portables utilisent une partition spécifique pour le stockage "suspend-to-disk", qui ne doit pas être touchée.

Pour de plus amples informations, veuillez consulter le Guide d'Installation et le Manuel de configuration des disques.

Obtenir et installer OpenBSD/i386 :

La dernière version supportée d'OpenBSD/i386 est la version OpenBSD 3.5. Voici les instructions d'installation d'OpenBSD/i386 3.5.

Des snapshots sont mis à disposition de temps à autre, à cet endroit ainsi que sur quelques miroirs. Voici de même les instructions d'installation d'OpenBSD/i386 snapshot.

Plusieurs options d'installation existent :

  • Démarrage à partir du CD :
    Démarrer à partir du CD permet d'obtenir l'équivalent d'une disquette El Torito 2,88Mo qui contient presque tous les pilotes d'OpenBSD. Ceci inclut un support USB minimal (périphériques umass et ukbd).
    Pour obtenir la liste la plus récente des pilotes disponibles dans cette image, regardez le fichier de configuration RAMDISK_CD.
  • Disquette A (floppy35.fs)
    Cette disquette 1,44Mo contient les pilotes les plus courants. Elle a été faite pour supporter les configurations PC les plus courantes. En règle générale, les pilotes qui manquent sont ceux qui nécessitent de gros microcodes pour tourner, telles que ceux pour les cartes SCSI les plus rares, les cartes Gigabit Ethernet ou les périphériques RAID.
    Pour obtenir la liste la plus récente des pilotes disponibles dans cette image, consultez le fichier de configuration noyau RAMDISK.
  • Disquette B (floppyB35.fs)
    Cette disquette contient les pilotes qui sont manquants sur la disquette A. Cette image est faite pour aider les gens installant des systèmes de type serveurs avec du matériel récent. Elle contient le support pour pratiquement tout les pilotes SCSI et RAID, les cartes ethernet gigabit et autres. Pour des questions de place, vous verrez que certains périphériques sont aussi manquants. En particulier, le support pour les périphériques PCMCIA et le système de fichiers MS-DOS est manquant ainsi que pour les périphériques matériels les plus vieux.
    Pour obtenir la liste la plus récente des pilotes disponibles dans cette image, consultez le fichier de configuration noyau RAMDISKB.
  • Disquette C (floppyC35.fs)
    Cette disquette contient la plupart des pilotes CardBus et PCMCIA. Cette image est faite pour des installations sur des portables. Dans la plupart des cas, la disquette A fonctionne aussi, mais certains portables bénéficieront grandement de cette disquette.
    Pour obtenir la liste la plus récente des pilotes disponibles dans cette image, consultez le fichier de configuration noyau RAMDISKC.
