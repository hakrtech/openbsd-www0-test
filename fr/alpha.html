OpenBSD/alpha

État et histoire :

Le port OpenBSD/alpha a commencé durant l'été 1996 en utilisant du code venant du port NetBSD/alpha qui avait été démarré par Chris Demetriou <cgd@netbsd.org> en 1994. Après le port originel OpenBSD, différentes questions de copyright ont empêché OpenBSD de suivre le code de NetBSD. Cependant, le travail sur le port alpha a continué et de nombreuses choses ont été corrigées. Pendant que nous faisions des corrections, NetBSD continuait de faire des améliorations dans le port alpha qu'il nous était malheureusement impossible d'utiliser. Ensuite, entre autre par manque de programmeurs et l'impossibilité d'utiliser le code de NetBSD, le support matériel et logiciel s'est retrouvé dépassé. Ce qui a entraîné que les versions 2.5 et 2.6 de l'alpha ont eu peu d'améliorations dans le noyau. Le résultat est que le port alpha ne fut pas inclus dans la version 2.7

Récemment, un regain d'intérêt pour la mise à jour du port alpha a été constaté. Ceci couplé au fait que NetBSD a changé sa licence sur son code alpha de façon à éliminer la question du copyright, pousse à reprendre le port alpha. (Pour plus de détails sur le fait que nous ne pouvions pas suivre le développement du code NetBSD, voir notre page politique.)

Pour l'instant, il n'y a pas de mainteneur pour le port alpha. Les discussions techniques à propos du port alpha doivent être adressées à <alpha@openbsd.org>. Si vous êtes intéressé par le développement du port alpha, envoyer vos requêtes à cette adresse. Si vous êtes intéressé par le poste de mainteneur du port alpha, aidez le projet pendant quelques temps et lancez vous dans ce poste.

Projets (sans ordre particulier) :

  • Mettre à jour le noyau alpha avec ce que possède NetBSD.
  • Nouveaux Binutils pour supporter le format de fichier objet ELF. (ce qui donne les bibliothèques partagées)
  • Nouveaux bootblocks pour supporter ELF.
  • Mise à jour de compat_osf1.

Matériel supporté :

À cause de changements qui sont actuellement en cour dans l'alpha, le matériel supporté peut varier. Au dernier renseignement, les matériels suivants sont supportés, mais cela peut changer n'importe quand.

  • Série DEC 3000/[3456789]00 avec les périphériques suivants :
    • Matériels supportés :
      • Ports séries intégrés.
      • Ethternet LANCE intégré.
      • Processeurs SCSI intégré (53c[f]94), bien que les deux peuvent ne pas fonctionner au même moment.
    • Ce qui peut fonctionner mais n'a pas été testé :
      • TurboChannel option slot LANCE (PMAD-A).
    • Ce qui n'est *pas* supporté actuellement :
      • Tous les types de Frame buffers.
      • Les autres cartes TurboChannel.
      • Le processeurs RNIS/Audio.
  • AlphaStation {200,250,255,400,500}, AXPpci,
    Multia (bien que les Multias ont eu des problèmes de mortalité infantile et de développement)
    Systèmes utilisant les périphériques suivants :
    • Matériels supportés :
      • NCR 53c810 SCSI (intégré ou carte PCI).
      • Ports série ns16550 intégrés.
      • Ethernet basé sur le DC21040 (intégré ou carte PCI).
      • Cartes Ethernet PCI basées sur le DC21140.
      • Ponts PCI-PCI basés sur le DC21050 (bien que d'autres types de ponts PCI-PCI devraient fonctionner).
      • Cartes vidéo PCI VGA.
      • Cartes ISA VGA.
      • Cartes vidéo DEC ZLXp-E1 (basées sur le DC21030-based, "TGA").
      • Cartes Ethernet PCI et ISA compatibles NE2000.
      • Cartes Ethernet ISA SMC/WD 8003, 8013, et la SMC "Elite16".
      • Cartes Ethernet 3Com 3c503.
      • Cartes Ethernet 3Com 3c5xx/3c9xx.
      • Cartes Ethernet basées sur le Lite-On PNIC (82c168, 82c169).
      • Clavier.
      • Souris de style PS/2.
      • Contrôleurs IDE (incluant le contrôleur IDE intégré sur certaines machines).
      • Cartes Intel Etherexpress Pro/100B.
      • Cartes séries multiport ISA Cyclades.
    • Ce qui pourrait marcher mais qui n'a pas été testé :
      • Ports parallèles intégré.
      • Cartes séries de la famille ISA ns16x50.
      • Cartes séries multiports ISA ns16x50.
      • Autres cartes SCSI NCR 53c8xx.
      • Les cartes ISA contrôleur SCSI Adaptec 152*.
    • Ce qui n'est *pas* supporté actuellement :
      • Matériel de son (incluant le Windows Sound System se trouvant sur certaines machines).
      • Cartes vidéo PCI DEC ZLXp-E2 et ZLXp-E3.
      • Le contrôleur SCSI intégré sur l'AlphaStation 600.
      • Cartes PCI et ISA qui ne sont pas listées ci-dessus.
      • Le lecteur de disquettes.
  • Actuellement, les systèmes suivants ne sont pas supportés :
    • Systèmes DECpc AXP 150 (bus EISA comme dans les PC)
    • Les systèmes Alpha "serveur" (autres que l'AlphaServer 400, qui n'est autres qu'une AlphaStation 400 déguisée).
    • Les systèmes Alpha multi-processeurs
    • EB164, PC164, PC164[LSR]X
    • Les systèmes Digital Personal Workstation (Miata)
    • Les systèmes basés sur le 21264
    • Les systèmes API (Alpha Processor, Inc.)

Snapshots:

Pendant que nous ramenons le port alpha à la vie, l'apparition de snapshots sera non prédictible et sporadique au mieux. Si vous êtes intéressé pour aider, nous vous encourageons à récupérer un snapshot, à vous synchroniser avec -current et de creuser !
