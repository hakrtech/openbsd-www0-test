OpenBSD/vax

History and Status:

OpenBSD/vax is currently working on expansion of the supported hardware, graphics console support, and shared libraries. The architecture specific code is largely based on the NetBSD/vax port with the end result being a very comfortable and familiar OpenBSD environment.

Significant changes contained in the post 2.8 snapshot:

  • Support for 16 filesystems per drive. (work by hugh)
  • Emulation of the EMODD instruction. (work by bjc)
  • Support for KA53 (Cheetah) machines. (work by bjc & hugh)

OpenBSD is currently known to run on these vax models:

  • vaxstation/microvax 2000
  • vaxstation 3100/20
  • vaxstation 3100/38
  • vaxstation 3100/76
  • vaxstation 4000/60
  • microvax 3300
  • vax 4000/100
  • vax 4000/300

SCSI should be working on the following:

  • microvax 3300
  • vaxstation 4000/60
  • vaxstation 4000/vlc?
  • most 3100 series vaxstations and microvaxes

Projects being worked on:

  • vaxstation 2000's hdc9224 (floppy/mfm hard disk controller)
  • qbus tk50/tk70 tape drives
  • shac dssi support
  • VAX 4000-10X support (docs and hardware sought)

Known problems:

  • optimizer bugs in gcc
  • intermittent lance ethernet timeouts
  • some signal delivery problems

The current release OpenBSD/vax is 2.8; however, snapshots will be made from time to time.

The OpenBSD/vax port maintainer is Brandon Creighton.
