7.1 - Comment est-ce que je redéfinis le clavier ? (spécifique à wscons)

En utilisant le pilote console wscons(4): alpha, i386, et macppc.

A partir de OpenBSD 3.0:

Depuis OpenBSD 2.9, OpenBSD a changé et utilise maintenant le pilote console wscons(4). Sous OpenBSD 2.8 et avant, le pilote console pcvt(4) était utilisé.

Avec wscons(4), plusieurs options peuvent être configurées en utilisant l'utilitaire wsconsctl(8). Par exemple, pour changer la configuration des touches du clavier avec wsconsctl(8) vous devez exécuter ceci: 

    # wsconsctl -w keyboard.encoding=fr

Dans l'exemple suivant, on redéfinit la touche "Caps Lock" en "Control L": 

    # wsconsctl -w keyboard.map+="keysym Caps_Lock = Control_L"
Pour OpenBSD 2.9:

Les arguments à utiliser avec wsconsctl sont légèrement différents sous OpenBSD 2.9. En particulier, vous ne devez pas utiliser le mot clé ``keyboard'', mais l'option ``-k" pour spécifier que la commande doit s'appliquer au clavier. 

    # wsconsctl -k -w encoding=fr

et

    # wsconsctl -k -w map+="keysym Caps_Lock = Control_L"
Pour OpenBSD 2.8 (et versions antérieures):

En utilisant kcon(1), "kcon - keyboard control and remapping for the pcvt driver".
Avant d'utiliser ceci, vous aurez besoin d'activer scon(1) pour la redéfinition des caractères.

Exemple: 

    # scon -o
# kcon -m f8

Cela chargera le fichier keycap pour la France. Remarquez que si vous lancez scon -o encore une fois, cela désactivera la redéfinition des touches, ce qui n'est sans doute pas ce que vous désirez si vous utilisez kcon pour les redéfinir. Gardez ceci à l'esprit si vous utilisez scon de manière automatique comme dans les scripts.

Ces commandes ne peuvent être exécutées qu'à partir du terminal virtuel pctv.

7.2 - Est-ce que OpenBSD contient gpm ou quelque chose du même genre ?

Pour les plates-formes utilisant wscons(4) il y a un port du démon moused(8) de FreeBSD. Le premier port de moused(8) à été réalisé pour OpenBSD 2.7 et utilisait l'interface de la console pcvt(4), ceci a changé plus tard pour utiliser wscons. Le démon a été renommé en wsmoused(8), et il est inclut par défaut dans OpenBSD à partir de la version 2.9.

wsmoused(8) fonctionne avec les architectures: alpha, i386, et macppc.

7.3 - Effacer la console à chaque fois qu'un utilisateur se déconnecte.

Pour ce faire, il vous faut ajouter une ligne dans /etc/gettytab(5). Changer la section: 

    P|Pc|Pc console:\
        :np:sp#9600:

Ajouter une ligne ":cl=\E[H\E[2J:" à la fin, ce qui donne: 

    P|Pc|Pc console:\
        :np:sp#9600:\
	:cl=\E[H\E[2J:

7.4 - Accéder au tampon de la console (spécifique alpha/macppc/i386)

Dans OpenBSD 2.6 fut ajoutée la capacité de revenir en arrière dans la console. Ceci permet de voir les informations inscrites précédemment sur votre écran. Pour monter et descendre dans le tampon, utilisez simplement les combinaisons de touches:

Pour monter: 
    "[shift]+[pgup]"
Pour descendre: 
    "[shift]+[pgdn]"

La configuration par défaut permet de pouvoir visualisez 8 pages en arrière.

7.5 - Pourquoi est-ce que F9 jusqu'a F12 ne permettent plus de changer de console ? (spécifique i386)

Dans OpenBSD 2.6 (et versions antérieures), le pilote de la console pcvt sur le port i386 permettait d'appuyer sur F9, F10, F11, F12 et d'obtenir les terminaux virtuels correspondant 1, 2, 3 et 4. A partir de 2.7, cette fonctionnalité a été supprimée vu que plusieurs applications utilisent ces touches.
