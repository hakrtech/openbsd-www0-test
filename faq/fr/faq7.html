7.1 - Comment est ce que je redéfinis le clavier ?(spécifique pcvt/i386)

En utilisant kcon(1), "kcon - keyboard control and remapping for the pcvt driver" Avant d'utiliser ceci, vous aurez besoin d'activer scon pour la redéfinition des caractères

Exemple : 

    bsd# scon -o
bsd# kcon -m f8

Cela chargera le fichier keycap pour la France. Remarquez que si vous lancez scon -o encore une fois, cela désactivera la redéfinition des touches, ce qui n'est sans doute pas ce que vous désirez si vous utilisez kcon pour les redéfinir. Gardez ceci à l'esprit si vous utilisez scon de manière automatique comme dans les scripts.

Ces commandes ne peuvent être exécutées que depuis un terminal virtuel pcvt.

7.2 - Est-ce qu'il y a gpm ou quelque chose du même genre dans ?

Après la sortie d'OpenBSD 2.7, un port du démon moused(8) de FreeBSD a été réalisé. Il a été modifié pour utiliser l'interface de la console pcvt(4) d'OpenBSD. Il sera inclut par défaut pour i386 à partir d'OpenBSD 2.8. Les autres plate-formes ne peuvent pas faire usage de cet utilitaire pour l'instant.

7.3 - Comment est-ce que j'efface la console à chaque fois qu'un utilisateur se déconnecte ?

Pour ce faire, il vous faut ajouter une ligne dans /etc/gettytab. Changez la section : 

    P|Pc|Pc console:\
        :np:sp#9600:

Ajoutez la ligne ":cl=\E[H\E[2J:" à la fin ce qui donne : 

    P|Pc|Pc console:\
        :np:sp#9600:\
	:cl=\E[H\E[2J:

7.4 - Accéder au buffer de retour de la console (spécifique pcvt/i386).

Dans OpenBSD 2.6 fut ajouté la capacité de revenir en arrière dans la console pour les machines i386. Ceci permet de voir les informations qui ont disparu de l'écran. Pour pouvoir vous déplacer de haut en bas dans le buffer, il suffit juste d'utiliser la combinaison de touches "[shift]+[pgup]" pour monter et "[shift]+[pgdn]" pour descendre. Le buffer par défaut (le nombre de pages que vous pouvez voir) est de 8, ce qui peut être changé grâce à l'utilitaire scon(1). Par exemple, pour changer le nombre de pages à 10, il faut faire ceci : 

    bsd$ scon -b 10

Cette commande ne peut être exécutée que depuis un terminal virtuel pcvt.

7.5 - Pourquoi est-ce que F9 jusqu'à F12 ne permettent plus de changer de console ? (spécifique pcvt/i386)

Dans OpenBSD 2.6 et en dessous, le pilote de la console pcvt sur le port i386 permettait d'appuyer F9, F10, F11, F12 et d'obtenir les terminaux virtuels correspondants 1, 2, 3 et 4. À partir de 2.7, cette fonctionnalité a été supprimée, parce que plusieurs applications utilisent ces touches.
