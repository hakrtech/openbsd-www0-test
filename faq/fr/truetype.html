Que faut-il faire pour les obtenir ?

OpenBSD inclut XFree86 4 avec les "Xft Render Extensions" activées. Pour pouvoir l'utiliser, votre serveur X (pilote de votre carte graphique) doit les supporter. Certains pilotes n'ont pas encore été convertis au code 'fb' et ne peuvent donc pas utiliser l'extension RENDER nécessaire. Vérifiez comme ceci : 
$ /usr/X11R6/bin/xdpyinfo | grep RENDER
RENDER
Si RENDER n'est pas spécifié dans sa liste des extensions, vous devrez attendre que quelqu'un implémente l'extension pour cette carte.

L'étape suivante consiste à trouver et d'installer quelques polices "TrueType". Une collection de polices populaires a été compilée et mise à disposition via l'arborescence des ports OpenBSD, sous /usr/ports/x11/msttcorefonts. Elles peuvent être installées comme suit : 

# cd /usr/ports/x11/msttcorefonts
# make
# make install
Si vous êtes le seul utilisateur sur le système voulant ces polices, créez un fichier $HOME/.xftconfig contenant la ligne suivante : 
dir "/usr/local/lib/X11/fonts"
Pour activer les polices pour tout le système, vous devrez éditer /usr/X11R6/lib/X11/XftConfig et ajouter 
dir "/usr/local/lib/X11/fonts"
juste aprèsdir "/usr/X11R6/lib/X11/fonts/Type1".

Quelles applications peuvent les utiliser ?

xterm(1), xditview(1), et toute applicaion utilisant QT2 (ç.à.d. toutes les applications KDE) reconnaissent Xft(3).

xterm

Lancez-le comme ceci : 
$ xterm -fa 'Andale Mono' -fs 14
option:   ressource:   signification
 -fa     *faceName    Nom de la police
 -fs     *faceSize    Taille de la police
Les autres polices "TrueType" (Arial, Georgia, Verdana, ...) ne fonctionnent pas très bien avec xterm. xterm perd un pixel par caractère, car xterm n'est pas disposé à traiter des caractès qui se prolongent au delà de leur cellule.

Les applications QT2/KDE2

Pour lancer une application QT2/KDE2, comme le navigateur konqueror, il n'y a qu'1 (!) seule variable d'environnement à définir :
Lancez l'application en définissant la variable QT_XFT comme "true" : 
$ env QT_XFT=true konqueror
ou pour kdbg: 
$ env QT_XFT=true kdbg
et vous avez fini.
