Par utilisateurs de haut niveau, nous désignons les personnes qui ont une expérience terrain de la manière d'utiliser et d'administrer un système d'exploitation type Unix. Si vous suiviez les instructions de cette section sans comprendre ce que vous êtes entrain de faire, vous causeriez des problèmes au niveau du fonctionnement de votre système.

12.1 - Forcer l'accès DMA sur les disques IDE

Votre chipset n'est peut-être pas connu du code PCI IDE. Si c'est le cas, vous verrez le message suivant :: 

    pciide0: DMA, (unused)

Si vous voyez ce message, vous pouvez essayer de forcer le mode DME en utilisant 'flags 0x0001' dans l'entrée pciide dans votre fichier de configuration du noyau comme ceci :

    pciide* at pci ? dev ? function ? flags 0x0001

Après avoir effectué cette opération, le code pciide essayera d'utiliser le mode DMA qu'il sache ou pas comment faire ça avec votre chipset. Si ça marche et que votre système passe les étapes de fsck et de démarrage, alors il est fort probable que ça marche pour de bon. Si ça ne marche pas et que votre système se fige ou panique après la séquence de démarrage cela voudrait dire que vous ne pouvez pas utiliser le mode DMA (jusqu'à ce que le support de votre chipset soit ajouté). Il est à noter que si vous trouvez la documentation du chipset de votre contrôleur PCI-IDE, c'est un bon début pour un support complet de votre chipset par le code PCI-IDE. Vous pouvez effectuer des recherches sur le site Web du fabricant ou appelez les. Si le contrôleur PCI-IDE fait partie de votre carte mère, essayez de trouver le fabricant du chipset et sollicitez leur ressources !

Vous saurez que le support DMA a été activé si vous voyez le message suivant :

    cd0(pciide0:1:0): using PIO mode 3, DMA mode 1 (using DMA data transfers)

Cela veut dire que pciide0, canal 1, pilote 0 (qui semble être un CD-ROM ATAPI) utilise les transferts de données DMA.

Quelques remarques :

DMA n'est pas supporté sur wdc* à moins qu'un canal DMA (drq) est spécifié. Nous ne sommes pas sûrs quels drqs sont "standards" pour les contrôleurs de disques durs. Pour activer :

wdc0	at isa? port 0x1f0 irq 14 flags 0x00

Comparé à PIO, le DMA non-ultra ne fournit pas forcément une meilleure bande passante. Cependant, il diminue la charge CPU de manière significative.
