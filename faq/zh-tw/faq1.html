1.1 - OpenBSD是什麼?

OpenBSD是一種免費的, 多平臺的, 4.4BSD 基礎上象UNIX的操作系統. 我們的目標 側重在正確性, 安全性, 標準化和移植性. 大量在SVR4(Solaris), FreeBSD, Linux, BSD/OS, SunOS 和 HP-UX 上運行的程式, OpenBSD都有 對應的模擬程式.

1.2 - OpenBSD運行在什麼系統上?

OpenBSD 3.3 運行在下列平臺上:

  • i386 - CD 可啟動
  • sparc - CD 可啟動
  • sparc64 - CD 可啟動
  • hp300 - 只有FTP
  • mac68k - 只有FTP
  • macppc - CD 可啟動
  • mvme68k - 只有FTP
  • alpha - 只有FTP
  • vax
  • hppa - 只有FTP * 3.3 新平臺*

可啟動 是說OpenBSD將從CD直接啟動. CD可以啟動在幾種硬件平臺上, 詳見 FAQ的第三章 關於得到OpenBSD CD的細節.

以前的 OpenBSD版本還有下列的移植:

  • amiga - 取消于3.2版本
  • sun3 - 取消于2.9版本
  • arc - 取消于2.3版本
  • mvme88k
  • pmax

OpenBSD現在還不支持多處理器. 詳見FAQ 8, SMP.

1.3 - OpenBSD是真的免費麼?

OpenBSD是完全免費的. 執行版本是免費的. 原碼也是免費的. 所有OpenBSD的部份有合理的版權 允許再分佈. 這也包括可以再使用大量OpenBSD的原碼, 無論是為個人還是商業原因. 除了原來 BSD 執照的規定, OpenBSD 沒有任何更多的限制. 擁有嚴格版權的軟件不可以包含在OpenBSD通常的 分佈裡. 這是為了保護OpenBSD的免費使用. 比如, OpenBSD可以個人自由使用, 學校, 政府, 非福利和福利組織都可使用.

關於更多執照的詳情, 請參閱: http://www.openbsd.org/policy.html.

OpenBSD 的維護者主要是花費自己的資金去開發. 這包括開發的時間, 各種設備做各種移植, 網絡資源把OpenBSD帶給用戶, 還有時間包括回答用戶提出的問題和報告的BUG. OpenBSD並不是 自立性的富有, 哪怕微量的時間, 設備和資源貢獻都會帶來很大的轉變.

1.4 - 為什麼我要使用OpenBSD?

新用戶會經常問到OpenBSD是不是比其它的UNIX-like操作系統好. 這個問題基本上 沒有答案, 而且是一種以信仰為主題的爭論. 在任何條件下請不要在OpenBSD的群體 郵件上問此問題.

我們認為下列原因使OpenBSD成為非常有用的操作系統. OpenBSD是不是適合你, 其實 是一個只有你可以回答的問題.

  • OpenBSD運行在不同的硬件平臺上.
  • OpenBSD被很多安全專家認為是最安全的 UNIX-like 操作系統. 這是十位成員花了一年半時間鑒定原碼的結果.
  • OpenBSD是擁有全方位特性的UNIX-like操作系統, 而且原碼免費.
  • OpenBSD整合了很多先進的技術關於建立防火牆和 私人網絡服務在 分佈狀的環境中.
  • OpenBSD受益于從許多正在開發的領域, 通過國際的用戶和編程人士,它有機會體現 最新發展的技術.
  • OpenBSD給予普通用戶機會參與開發和測試產品.

1.5 - 我怎麼幫助支持OpenBSD?

我們很感激對OpenBSD 工程貢獻的個人和組織. 他們被承認在 捐助頁.

OpenBSD對廣大用戶有幾點需要.如果你發現OpenBSD有用, 我們強烈支持你以各種方式 做貢獻. 如果下列建議中沒有適合你的, 不要猶豫請把你的想法通過電子郵件告訴我們 donations@openbsd.org.

  • 購買OpenBSD CD. 它包括當前的 完整版本, 在很多平臺上可啟動. 它也為OpenBSD工程帶來一些資金, 解決網絡 資源限制在互聯網上發行版本. 這三張便宜的CD包括了所有的原碼. 記住, 你的 朋友們需要他們自己的拷貝!
  • 捐錢. 這項工程總是需要資金 去購買設備, 網絡連接, 和CD發行成本. 製作CD需要開發者們預先付下款, 而且 沒有成品保證. 發郵件到 donations@openbsd.org得到 信息如何做. 即便捐不多的錢也會有改觀.
  • 捐獻設備. 這項工程總是需要通常的和 特殊的設備. 象IDE和SCSI盤, 不同種的RAM都是受歡迎的. 其它的硬件比如電腦系統和 母板, 你應該詢問當前的需要. 寫信到 donations@openbsd.org去安排 發貨.
  • 捐助你的時間和技能. 喜歡寫操作系統的程序員自然總是受歡迎, 但也有非常多 其它的方式. 跟隨 郵件群體,回答新人的問題.
  • 幫助維護文獻, 通過呈交新的FAQ材料(寄到 faq@openbsd.org). 建立本地的 用戶群, 掛鉤你的朋友和OpenBSD. 在公司使用OpenBSD. 如果你是學生, 像你的教授推薦OPENBSD, 作為學習電腦科學和工程學的工具. 值得一提的非常 重要的是, 你不應該通過花費時間跟人爭論操作系統來幫助OpenBSD. 這不但沒有幫助發掘新人, 而且會影響到開發者與其它開發者的關係.

1.6 - 誰維護OpenBSD?

OpenBSD是由開發小組維護, 小組人員遍布全球. 由在加拿大的Theo de Raadt協調.

1.7 - 何時OpenBSD會發行下個版本?

OpenBSD小組每六個月發行一次新的版本, 發行日為五一和十一月一日. 更多的關於開發詳情請參閱 這裡.
