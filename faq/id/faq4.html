4.1 - Ikhtisar prosedur instalasi OpenBSD

OpenBSD mempunyai sebuah prosedur instalasi berbasis teks yang handal dan bisa beradaptasi, dan dapat diinstall dari sebuah disket tunggal. Hampir semua platform mengikuti sebuah prosedur instalasi yang mirip; namun ada beberapa perbedaan dalam rinciannya. Pada semua kasus, anda didorong untuk membaca dokumen INSTALL spesifik platform di direktori platform pada CD-ROM atau situs FTP (sebagai contoh, i386/INSTALL.i386, mac68k/INSTALL.mac68k atau sparc/INSTALL.sparc).

Pada kebanyakan platform, instalasi OpenBSD menggunakan sebuah kernel khusus dengan sejumlah utilitas dan skrip install yang di tanamkan dalam sebuah RAM disk pra-muat. Setelah kernel ini diboot, sistem operasi dimekarkan dari sejumlah berkas termampat tar(1) (.tgz). Ada beberapa cara untuk boot kernel instalasi ini:

  • Floppy disk: Citra floppy disk disediakan untuk digunakan membuat disket instalasi pada sistem sepadan Unix lainnya, atau pada sebuah sistem DOS/Windows. Nama khususnya adalah floppy34.fs, walaupun beberapa platform mempunyai multi citra floppy disk yang tersedia.
  • CD-ROM: Pada beberapa platform ISO CD-ROM (cd34.iso) disediakan untuk membuat CD-ROM bootable. Ini hanya berisi kernel instalasi - berkas instalasi masih harus di ambil dari FTP atau sumber lain. Anda bisa, tentu saja, membuat CD-ROM anda sendiri dengan berkas apapun dan utilitas yang anda inginkan didalamnya.
  • bsd.rd: RAM disk kernel, diharapkan untuk boot dari partisi yang telah terinstall OpenBSD atau boot lewat jaringan.
  • Jaringan: Beberapa platfoem mendukung booting lewat jaringan.
  • Menuliskan citra sistem ke disket: sebuah citra filesistem yang bisa dituliskan ke partisi yang ada, dan lalu dapat di boot.
  • Bootable Tape: Beberapa platform mendukung booting dari divais tape/pita Penggunaan pita ini dengan mengikuti petunjuk INSTALL.platform berikut.
Tidak setiap platform mendukung semua pilihan booting:
  • alpha: Floppy, CD-ROM, menuliskan sebuah citra floppy ke hard disk.
  • hp300: CD-ROM, jaringan.
  • hppa: Jaringan.
  • i386: Floppy, CD.
  • mac68k: Diinstall (dan di booting) menggunakan utilitas Mac OS. Lihat INSTALL.mac68k untuk jelasnya.
  • macppc: CD-ROM, jaringan.
  • mvme68k: Jaringan, pita bootable.
  • sparc: Floppy, CD-ROM, jaringan, menuliskan citra filesistem ke partisi yang ada, pita bootable.
  • sparc64: Floppy (hanya U1/U2), CD-ROM, jaringan, menuliskan citra filesistem ke partisi yang ada.
  • vax: Floppy, jaringan.
Seluruh platform lain selain mac68k dapat juga menggunakan sebuah bsd.rd untuk install ulang atau upgrade.

Sekali kernel instalasi di booting, anda mempunyai beberapa pilihan cara untuk mendapatkan rangkaian berkas instalasi. Sekali lagi, tidak setiap platform mendukung setiap pilihan.

  • CD-ROM: Tentu saja, kami memilih agar anda menggunakan rangkaian CD-ROM Resmi, tetapi untuk kebutuhan khusus, anda bisa juga membuat CD-ROM anda sendiri.
  • FTP: Salah satu dari situs miror FTP OpenBSD atau server FTP lokal terdekat anda yang mempunyai rangkaian berkas instalasi OpenBSD.
  • HTTP: Salah satu dari situs miror HTTP OpenBSD atau server web lokal terdekat anda yang mempunyai rangkaian berkas instalasi OpenBSD.
  • Partisi disk lokal: Dalam banyak kasus, anda bisa menginstall rangakain berkas instalasi dari partisi lain pada partisi disk lokal. Sebagai contoh, pada i386, anda dapat menginstall dari sebuah partisi FAT atau sebuah CD-ROM dalam format ISO9660, Rock Ridge ataupun Joliet. Pada beberapa kasus, anda perlu melakukan mount file sistem secara manual sebelum digunakan.
  • NFS: Beberapa platform mendukung penggunaan tautan NFS untuk rangkaian berkas instalasi.
  • Pita: Rangkaian berkas instalasi dapat juga dibaca dari pita terdukung.

4.2 - Daftar periksa sebelum instalasi

Sebelum anda memulai instalasi, anda seharusnya mempunyai beberapa gagasan tentang apa yang akan anda selesaikan. Anda akan perlu mengetahui hal-hal berikut, paling tidak:

  • Nama mesin
  • Peranti keras terpasang dan tersedia
    • Verifikasi kompatibilitas peranti keras pada platform anda dengan halaman kompatibilitas
    • Jika ISA, anda perlu juga mengetahui setting peranti keras, dan konfirmasikan sesuai yang dibutuhkan OpenBSD.
  • Metode instalasi yang digunakan (CD-ROM, FTP, dll.)
  • Bagaimanakah sistem akan akan diperbarui dan diperbaiki(patch)?
    • Jika diselesaikan setempat, anda perlu memiliki ruang cukup yang tersedia untuk struktur kode sumber dan membangunnya.
    • Jika tidak, anda perlu mengakses mesin lain untuk membangun rilis terbaiki(ter-patch) aktif.
  • Layout hard disk yang diinginkan
    • Apakah data yang telah ada perlu disimpan di tempat lain?
    • Apakah OpenBSD bisa berada dalam sistem ini OS lain? Jika ya, bagaimana keduanya akan bisa di booting? Akankah anda memerlukan memasang sebuah "boot manager"?
    • Akankah keseluruhan hard disk digunakan OpenBSD, atau anda ingin untuk menjaga partisi/OS yang telah ada (atau ruang disk untuk OS yang lain di kemudian hari)?
    • Bagaimanakah anda mengharapkan untuk mensubpartisikan bagian OpenBSD dari hard disk anda?
  • Setting jaringan, jika tidak menggunakan DHCP:
    • Nama domain
    • Alamat beberapa Domain Name Server (DNS)
    • Alamat-alamat IP dan subet mask untuk setiap kartu jaringan
    • Alamat Gateway
  • Akankah anda menjalankan Sistem Window X?

4.3 - Membuat media instalasi OpenBSD yang bisa di booting

Sebagai contoh, kita akan tengok pada citra instalasi yang tersedia untuk platform i386 dan sparc.

Platform i386 mempunyai lima citra disk instalasi berbeda yang bisa dipilih:

  • floppy34.fs (PC Desktop) mendukung banyak kartu jaringan PCI dan ISA, IDE dan adapter sederhana SCSI dan beberapa dukungan PCMCIA. Kebanyakan pengguna akan menggunakan citra ini.
  • floppyB34.fs (Server) mendukung banayk pengendali RAID, dan beberapa adapter SCSI kurang terkenal. Namun, dukungan untuk banyak adapter SCSI standar dan banyak kartu jaringan EISA dan ISA telah di pindahkan dari citra ini.
  • floppyC34.fs (Laptop) mendukung divais CardBus dan PCMCIA yang banyak ditemukan pada kebanyakan laptop.
  • cdrom34.fs adalah, hasil dari kombinasi seluruh tiga citra disket booting diatas. Dapat digunakan untuk membuat disket booting 2.88M , atau lebih umumnya, digunakan sebagai citra untuk pembuatan CD bootable.
  • cd34.iso adalah sebuah ISO berformat ISO9660 yang bisa digunakan untuk membuat CD bootable dengan hampir semua aplikasi populer pembuat CD-ROM pada kebanyakan platform. Ini adalah cdrom34.fs dalam sebuah format "siap-untuk-rekam".
Ya, terdapat situasi yang mungkin bahwa satu disket instalasi diperlukan untuk mendukung adapter SCSI anda dan disket lain diperlukan untuk mendukung adapter jaringan anda. Untungnya, ini kejadian langka, dan biasanya bisa dilakukan.

Platform sparc mempunyai tiga citra disk instalasi berbeda yang bisa dipilih:

  • floppy34.fs: Mendukung sistem dengan sebuah disket.
  • cd34.iso Sebuah ISO bisa digunakan untuk membuat CD anda sendiri guna booting sistem SPARC dengan sebuah CD-ROM.
  • miniroot34.fs Dapat dituliskan ke partisi swap dan di booting.

4.3.1 - Membuat disket booting di Unix

Untuk membuat disket terformat, gunakan perintah fdformat(1) untuk sekaligus format dan memeriksa bad sector. 

  # fdformat /dev/rfd0c
  Format 1440K floppy `/dev/rfd0c'? (y/n): y
  Processing VVVVVVVVVVVVVVVVVVVVVVVVVVVVVVVVVVVVVVVV done.

Jika keluaran perintah fdformat anda seperti contoh di atas, maka disket anda dalam keadaan BAGUS. Namun, jika anda tidak melihat SELURUH "Vr" maka disket anda umumnya rusak, dan anda harus mencoba pada disket lainnya.

Menjadi catatan beberapa sistem sepadan Unix mempunyai perintah berbeda untuk format disket. Rujuklah dokumentasi sistem anda untuk prosedur yang pasti.

Sekali anda mempunyai disket yang terformat dan bersih bad sector inilah waktunya menuliskan citra instalasi ke disket. Untuk ini, anda bisa gunakan utilitas dd(1). Sebuah contoh penggunaan dd(1) sebagai berikut: 

  # dd if=floppy34.fs of=/dev/rfd0c bs=32k

Sekali citra dituliskan, periksa untuk memastikan bahwa salinan citra pada disket sama dengan aslinya gunakan perintah cmp(1). Jika isi diket identik dengan citra, anda akan mendapatkan prompt lagi. 

  # cmp /dev/rfd0c floppy34.fs

4.3.2 - Membuat disket booting di Windows atau DOS

Bagian ini menjelaskan bagaimana menuliskan citra instalasi ke disket dibawah sistem DOS atau Windows. Anda bisa mendapatkan alat yang disebutkan berikut dari direktori tools pada miror ftp manapun, atau dari direktori 3.4/tools pada CD1 dari rangkaian CD OpenBSD.

Untuk menyiapkan sebuah disket pada MS-DOS atau Windows, pertama-tama gunakan alat format bawaan MS-DOS/Windows untuk format disket.

Untuk menuliskan citra instalasi ke disket yang telah anda siapkan anda bisa gunakan rawrite, fdimage, atau ntrw. rawrite tidak akan berjalan pada Windows NT, 2000 atau XP.

Menjadi catatan bahwa FDIMAGE.EXE dan RAWRITE.EXE keduanya aplikasi MS-DOS, dan karenanya terbatas untuk konvensi penamaan berkas "8.3" dari MS-DOS. Seperti floppyB34.fs dan floppyC34.fs mempunyai nama file lebih panjang, harus anda temukan cara sistem anda menyimpan berkas tersebut dalam format "8.3" sebelum menggunakan FDIMAGE.EXE atau RAWRITE.EXE untuk membuat disket booting anda.

Contoh penggunaan rawrite: 
  C:\> rawrite
  RaWrite 1.2 - Write disk file to raw floppy diskette

  Enter source file name: floppy34.fs
  Enter destination drive: a
  Please insert a formatted diskette into drive A: and press -ENTER- : Enter

Contoh penggunaan fdimage: 
  C:\> fdimage -q floppy34.fs a:

COntoh penggunaan ntrw: 
  C:\> ntrw floppy34.fs a:
  3.5", 1.44MB, 512 bytes/sector
  bufsize is 9216
  1474560 bytes written

4.3.3 - Membuat sebuah CD-ROM

Anda bisa membuat sebuah CD-ROM menggunakan baik berkas cd34.iso maupun, pada kasus platform i386, anda bisa juga gunakan cdrom34.fs sebagai citra disket bootable yang bisa digunakan untuk boot sistem i386 dari CD-ROM. Rincian pasti disini diserahkan pada pembaca untuk menentukan memakai alat mana yang mereka punyai.

Beberapa utilitas pada OpenBSD adalah:

  • mkhybrid(8)
  • cdrecord, bagian dari koleksi cdrtools pada Sistem Port OpenBSD.

4.4 - Booting dari media instalasi OpenBSD

Booting i386

Booting dari citra instalasi pada platform PC i386 bukanlah hal baru bagi sebagian besar orang. Jika anda menggunakan sebuah disket, semudah memasukkan disket ke drive disket dan boot sistemnya. Citra instalasi akan kemudian dimuatkan, boot dari disket diaktifkan dari BIOS anda. Jika anda ingin booting dari CD, anda harus menuju sistem BIOS anda dan mengatur pilihan booting agar bisa boot dari CD. Beberapa BIOS yang lama tidak mempunyai pilihan ini, dan anda harus menggunakan disket untuk booting citra instalasi anda. Jangan terlalu cemas; meskipun jika anda boot dari disket anda masih bisa menginstall dari CD.

Booting sparc/sparc64

CATATAN: Pada platform sparc64, hanya mesin-mesin SBus (Ultra 1, Ultra 2) yang bisa di boot dari disket.

Untuk boot dari disket, tempatkan disket dengan citra instalasi OpenBSD tertulis didalamnya pada drive disket. Lalu gunakan perintah berikut ini untuk boot dari disket: 

  ok boot floppy

Untuk boot dari CD-ROM, tempatkan disk CD-ROM OpenBSD pada drive. Jika Sun anda hanya mempunyai 1 drive CD-ROM, lalu tinggal menuju ke prompt boot, tempat anda bisa 'boot cdrom': 

  ok boot cdrom

Tentu saja, ini hanya akan berjalan pada modus perintah baru. Jika anda pada prompt modus perintah lama (sebuah panah kanan), ketik 'n' untuk modus perintah baru. (Jika anda menggunakan sebuah sparc lama yaitu sebelum sun4c, anda mungkin tidak mempunyai modus perintah baru. Pada kasus ini, anda perlu bereksperimen.) Jika anda mempunyai multi divais CD-ROM, anda perlu boot dari salah satu yang bisa untuk boot. Coba probe-scsi dari modus perintah baru. 

  ok probe-scsi

  Target 0
    Unit 0   Disk     QUANTUM LIGHTNING 365S
  Target 1
    Unit 0   Removable Disk     QUANTUM EMPIRE_1080S
  Target 3
    Unit 0   Removable Disk     Joe's CD-ROM

Temukan disk mana merupakan CD-ROM yang ingin anda gunakan untuk boot. Perhatikan angka Target. 

  ok boot /sbus/esp/sd@X,0

4.5 - Melakukan sebuah instalasi

4.5.1 - Memulai instalasi

Apapun cara anda booting, sekarang waktunya mnggunakannya. Selama proses boot, kernel dan seluruh program yang digunakan untuk instalasi OpenBSD dimuatkan ke memori. Masalah paling umum ketika booting adalah disket yang jelek atau masalah pensejajaran barisan drive. Disket booting sangat rapat dipaketkan -- titik rusak manapun akan menyebabkan masalah.

Pada hampir setiap tahapan selama proses instalasi OpenBSD, anda dapat menggagalkan usaha instalasi dengan menekan CTRL-C dan dapat restart tanpa reboot dengan menjalankan install pada prompt shell.

Ketika boot anda berhasil, anda akan melihat sejumlah pesan teks tergulung di layar. Teks ini, pada banyak arsitektur dengan tulisan putih diatas latar belakang biru, adalah dmesg, kernel memberitahu anda divais yang ditemukan, dan lokasinya. Jangan khawatir tentang mengingat teks ini, salinannya disimpan sebagai/var/run/dmesg.boot. Pada beberapa arsitektur, SHIFT+PGUP akan membolehkan anda memeriksa teks yang telah digulung hilang dari layar.

Lalu, anda akan melihat seperti ini: 

  rootdev=0x1100 rrootdev=0x2f00 rawdev=0x2f02
  erase ^?, werase ^W, kill ^U, intr ^C, status ^T
  (I)nstall, (U)pgrade or (S)hell? i

Dan dengan itu, kita capai pertanyaan pertama kita. Hampir selalu, anda punya tiga pilihan tertampil:

  • Install: memuat OpenBSD ke sistem, menimpa apapun yang telah ada sebelumnya. Menjadi catatan bahwa dimungkinkan untuk membiarkan beberapa partisi tidak di ubah pada proses ini, seperti /home, tetapi sebaliknya, anggap semuanya ditimpa.
  • Upgrade: Install serangkai berkas instalasi baru pada mesin, tetapi jangan menimpa keterangan konfigurasi apapun, data user, atau program tambahan. Tanpa format hard disk dilakukan, maupun penimpaan direktori /etc atau /var . Sedikit catatan penting:
    • Anda tidak akan diberikan pilihan untuk menginstall berkas etc34.tgz. Setelah instalasi, anda akan secara manual menggabungkan perubahan dari etc34.tgz ke sistem anda sebelum anda dapat mengharapkan instalasi sepenuhnya berfungsi. Ini tahapan penting yang harus dilakukan, sebagaimana jika tidak beberapa kunci layanan (seperti pf(4)) tidak akan berjalan.
    • Proses upgrade tidak didesain untuk melompati rilis sebelumnya! Sementara hal ini sering berhasil, hal ini tidak didukung. Untuk OpenBSD 3.4, upgrade 3.3 ke 3.4 hanyalah upgrade yang didukung. Jika anda harus upgrade dari versi yang lebih lama, sebuah install ulang lengkap disarankan.
    • Platform i386 telah dialihkan ke format binari ELF , dan membutuhkan uninstall seleuruh paket dan port yang telah diinstall sebelum upgrade. Install ulang SANGAT disarankan untuk upgrade.
  • Shell: Seringkali, anda perlu melakukan perbaikan atau pengelolaan sistem yang tidak (atau harus tidak) boot ke kernel normal. Pilihan ini membolehkan anda melakukan pengelolaan sistem.
Pada suatu waktu, anda tidak akan melihat pilihan "Upgrade" tertampil. Setelah sebuah peristiwa hari penandaan/flag day, tidak mungkin melakukan upgrade secara langsung; salah satu harus membangun ulang sistem dari awal.

Pada contoh ini, kita akan melakukan instalasi, tetapi proses upgrade merupakan hal yang serupa. 

  Welcome to the OpenBSD/i386 3.4 install program.

  This program will help you install OpenBSD in a simple and rational way. At
  any prompt except password prompts you can run a shell command by typing
  '!foo', or escape to a shell by typing '!'. Default answers are shown in []'s
  and are selected by pressing RETURN. At any time you can exit this program by
  pressing Control-C and then RETURN, but quitting during an install can leave
  your system in an inconsistent state.

  Specify terminal type: [vt220]
  Do you wish to select a keyboard encoding table? [n] ENTER

Pada kebanyakan kasus, tipe terminal bawaan adalah sesuai; meskipun jika anda menggunakan sebuah serial console untuk instalasi, tinggal jangan ambil terminal bawaan instalasi, sesuaikan.

Jika anda tidak memilih tabel enkoding keyboard, sebuah layout keyboard US akan dianggap dipakai. 

  IS YOUR DATA BACKED UP? As with anything that modifies disk contents, this
  program can cause SIGNIFICANT data loss.

  It is often helpful to have the installation notes handy. For complex disk
  configurations, relevant disk hardware manuals and a calculator are useful.

  Proceed with install? [n] y

Jika anda menulis nilai bawaan [n] disini, proses install akan di hentikan dan membawa anda pada prompt shell.

4.5.2 - Mengatur hard disk

Mengatur harddisk pada OpenBSD agak berbeda diantara platform-platform. Untuk i386 dan macppc, pengaturan harddisk diselesaikan dlaam dua tahap. Pertama, OpenBSD mambagi irisan hard disk yang ditentuakn dengan menggunakan fdisk(8), lalu kemudian irisan tersebut di buat subbagian menjadi partisi OpenBSD menggunakan disklabel(8).

Beberapa user mungkin sedikit bingung dengan terminologi yang digunakan di sini. Ini akan muncul kita gunakan kata "partisi" dalam dua cara berbeda. Observasi ini adalah benar. Ada dua lapisan pemartisian pada beberapa platform OpenBSD, pertama, salah satu mengacu pada pemartisian Sistem Operasi, tentang bagaimana multi OS pada satu komputer menandai ruang mereka sendiri pada harddisk, dan kedua adalah bagaimana partisi OpenBSD di subpartisi ke filesistem individual. Lapisan pertama terlihat sebagai sebuah partisi disk dari DOS, Windows, dan OS lain manapun yang berada dengan Sistem Operasi lain yang merupakan turunan mesin IBM AT. Lapisan kedua adalah pemartisian yang hanya terlihat dari OpenBSD dan OS yang secara langsung dapat membaca filesistem OpenBSD. 

  Cool!  Let's get to it...

  You will now initialize the disk(s) that OpenBSD will use. To enable all
  available security features you should configure the disk(s) to allow the
  creation of separate filesystems for /, /tmp, /var, /usr, and /home.

  Available disks are: wd0.
  Which one is the root disk? (or done) [wd0] Enter

Disk root adalah disk tempat sistem untuk boot, dan normalnya tempat swap mengambil ruang. Biasanya, ini akan merupakan konfigurasi bawaan -- jika ini tidak, anda akan perlu mengetahui bagaimana memaksa komputer anda boot dari sebuah disk tidak standar. Disk IDE akan tertampil sebagai wd0, wd1, dll., disk SCSI dan divais RAID akan tertampil sebagai sd0, sd1, dan selanjutnya. Seluruh disk OpenBSD dapat ditemukan terdaftar di sini -- jika anda mempunyai drive yang tidak tertampil, anda mempunyai peranti keras tidak terdukung atau terkonfigurasi tidak tepat. 

  Do you want to use *all* of wd0 for OpenBSD? [no] Enter

Jika anda menjawab "yes" pada pertanyaan ini, keseluruhan disk akan dialokasikan untuk OpenBSD. Ini akan menghasilkan pada sebuah Master Boot Record standar dan tabel partisi ditulis ulang ke disk -- satu partisi, berukuran dari keseluruhan harddisk, diatur menjadi tipe partisi OpenBSD, dan ditandai sebagai partisi bootable. Ini akan merupakan pilihan umum untuk kebanyakan sistem produksi menggunakan OpenBSD; namun, terdapat beberapa sistem hal ini seharusnya tidak dilakukan. Banyak sistem Compaq, banyak laptop, beberapa sistem Dell dan sistem lainnya menggunakan partisi "maintenance" atau "Suspend to Disk", yang seharusnya tetap dijaga. Jika sistem anda mempunyai partisi lain dari tipe manapun yang tidak anda harapkan dihapus, jangan memilih "yes" pada pertanyaan di atas.

Untuk demi kepentingan contoh ini, akami akan anggap disk harus dipilah antara OpenBSD dan partisi Widows 2000 yang telah ada sebelumnya, sehingga kita ambil jawaban "no", yang akan membawa kita pada program fdisk(8) . Anda dapat juga mendapatkan keterangan lanjut tentang fdisk(8) di sini.

Catatan Penting: User dengan harddisk besar (lebih dari 8GB pada i386 terbaru, meskipun pada mesin yang lebih tua dan platform berbeda, seringkali lebih kecil) akan ingin melihat bagian ini sebelum melanjutkan hal lain. 

  You will now create a single MBR partition to contain your OpenBSD data. This
  partition must have an id of 'A6'; must *NOT* overlap other partitions; and
  must be marked as the only active partition.

  The 'manual' command describes all the fdisk commands in detail.

Sekali lagi, beberapa perintah ini dapat digunakan sejumlah perluasan:

  • p - menampilkan (mencetak) disklabel saat ini ke layar, dan anda bisa menggunakan pemodifikasi k, m atau g untuk kilobyte, megabyte atau gigabyte.
  • D - Menghapus disklabel yang ada sebelumnya, membuat disklabel baru yang akan mencakup hanya partisi OpenBSD untuks aat ini. Ini sangat berguna jika disk sebelumnya mempunyai disklabel, dan partisi OpenBSD dibuat ulang pada ukuran yang berbeda -- label disk lama mungkin tidak terhapus, dan bisa menyebabkan kebingungan.

Gunakan perintah ini dan mengharapkannya mengubah ukuran partisi yang telah ada adalah cara yang tepat untuk kehilangan sejumlah besar data.

Mengiris disk anda secara tepat adalah penting. Jawaban untuk pertanyaan, "Bagaimana saya harus mempartisi sistem saya?" adalah "Tepatnya bagaimana anda memerlukannya". Ini akan berbeda dari aplikasi satu ke aplikasi lainnya. Tidak ada jawaban umum. Jika anda tidak yakin tentang bagaiamana anda ingin mempartisi sistem anda, lihat diskusi ini.

Pada sistem ini, kita punyai lebih dari 17G tersedia untuk OpenBSD. Itu ruang yang sangat besar, dan tidak umum kita membutuhkan semua ruang itu. Sehingga, kita akan dengan hati-hati tidak menggunakan ukuran minimum absolut, kita lebih memilih mempunyai beberapa ratus megabyte ruang kosong daripada satu kilobyte terlalu kecil.

Pada disk root, dua partisi 'a' dan 'b' harus dibuat . Proses instalasi tidak akan berjalan sampai dua partisi ini tersedia. 'a' akan digunakan untuk filesistem root (/) dan 'b' akan digunakan sebagai ruang swap.

Setelah sedikit berpikir, kita putuskan untuk membuat cukup partisi untuk membolehkan pembuatan filesistem terpisah yang dianjurkan (/, /tmp, /var, /usr, /home) bersama dengan sebuah partisi swap:

  • wd0a: / (root) - 150M. Seharusnya sudah lebih dari cukup.
  • wd0b: (swap) - 300M.
  • wd0d: /tmp - 120M. /tmp digunakan utuk membangun beberapa software, 120M akan mungkin cukup untuk kebanyakan hal.
  • wd0e: /var - 80M. Jika ini merupakan server web atau mail , kita buat partisi menjadi cukup lebih besar lagi, tetapi, itu bukanlah yang sedang kita lakukan.
  • wd0g: /usr - 2G. Kita ingin partisi ini cukup besar untuk memuat beberapa aplikasi user, ditambah kemampuan untuk update dan membangun ulang sistem jika dibutuhkan atau diinginkan. struktur Ports akan berada di sini, yang akan mengambil ruang hampir 100M sebelum port di buat paketnya.
  • wd0h: /home - 4G. Ini akan membuat ruang berkas bagi user cukup besar.

Sekarang anda harus memberi nama host sistem. Berikut ini adalah ukuran filesistem minimal yang dianjurkan untuk instalasi sistem lengkap. Termasuk ruang tambahan yang bisa kita buat untuk menjalankan sebuah sistem home khusus yang terhubung ke Internet.

  • Ini adalah nilai minimal.
  • Jika anda berencana menginstall sejumlah software tambahan signifikan, buatlah partisi /usr anda lebih besar! paling tidak tiga kali dari angka yang dberikan disini!
  • Untuk sistem yang mengelola sejumlah besar email atau halaman web (disimpan, berurutan, pada /var/mail dan /var/www) anda akan ingin membuat partisi /var anda lebih besar secara signifikan, atau meletakkan pada partisi yang berbeda.
  • Untuk sistem multiuser yang membuat banyak log, anda akan masih ingin membuat partisi /var anda lebih besar pada (/var/log) secara siginifikan.
  • Jika anda berencana membangun kernel atau sistem dari kode sumber, anda akan membuat partisi /usr lebih besar secara signifikan, paling tidak lebih besar 800M-1G dari yang disebutkan di sini.

Ketika anda pada editor disklabel, anda mugnkin memilih untuk membuat sistem keseluruhan hanya mempunyai sebuah 'a' (filesistem utama) dan 'b' (swap) . Filesistem 'a' yang anda tentukan dengan disklabel akan menjadi partisi root anda, yang seharusnya gabungan dari seluruh 3 partisi utama di atas (/, /usr, dan /var) ditambah beberapa ruang untuk /tmp. Jika anda mempunyai banyak ruang disk untuk dipakai buatlah 256MB, atau bahkan 512MB.

Ruang swap yang digunakan untuk menyimpan dump core sistem pada kejadian ini dari sebuah crash(8). Jika hal ini menjadi petimbangan anda, ruang swap anda seharusnya sedikit lebih besar daripada jumlah memori utama anda yang dipunyai pada sistem anda. Perhatikan sewaktu reboot, savecore(8) akan mencoba menyimpan isi dari partisi swap ke sebuah berkas di /var/crash sehingga sekali lagi, jika ini prioritas bagi anda, partisi /var anda harus mempunyai cukup ruang kosong untuk menyimpan berkas dump ini.

Ada lima latar belakang utama yang melandasi penggunaan filesistem terpisah, daripada menyatukan semuanya dalam satu atau 2 filesistem:

  • Keamanan: Anda dapat menandai beberapa filesistem menjadi 'nosuid', 'nodev', 'noexec', 'readonly', dll. Ini bisa diselesaikan sewaktu proses instalasi, nyata, jika anda menggunakan partisi seperti penjelasan di atas.
  • Stabilitas: Seorang user, atau program yang tidak diinginkan , bisa memenuhi filesistem dengan sampah jika mereka mempunyai izin menulis. Program kritis anda, yang tentu saja berjalan pada filesistem berbeda, tidak akan terinterupsi.
  • Kecepatan: Sebuah filesistem yang sering ditulis mungkin akan terfragmentasi. (Untungnya, filesistem ffs, yang OpenBSD gunakan, tidak terlalu banyak terjadi fragmentasi.)
  • Integritas: Jika satu filesistem terkorupsi karena sebab tertentu maka filesistem anda yang lain masih tetap BAGUS.
  • Ukuran: Banyak platform mempunyai batasan daerah disk tempat ROM boot dapat memuat kernel. Pada beberapa kasus, batasan ini mungkin sangat kecil (504M untuk mesin 486 lama), pada kasus lain, batasan bisa lebih besar (8G pada mesin i386 baru). Karena kernel bisa berada dimana saja pada partisi utama, keseluruhan partisi utama seharusnya berada dalam daerah ini. Untuk lebih jelas, lihat bagian ini. Panduan yang bagus mungkin adalah untuk menjaga partisi / anda berada dalam 2G ruang awal disk anda, kecuali anda paham platform anda ( dan mesin tertentu!) dapat mengelola lebih (atau kurang!) dari batasan tersebut.

Beberapa gagasan tambahan tentang pemartisian:

  • Untuk usaha pertama anda pada sebuah eksperimen dengan sistem, satu partisi / yang besar dan swap mungkin yang termudah sampai anda tahu seberapa besar ruang yang anda perlukan. Dengan melakukan hal ini anda akan mengorbankan beberapa fitur keamanan bawaan OpenBSD yang membutuhkan filesistem terpisah untuk /, /tmp, /var, /usr dan /home.
  • Sebuah sistem dihubungkan dengan Internet atau gugus kerja host lain seharusnya mempunyai /var terpisah (dan mungkin juga /var/log terpisah ) untuk logging.
  • Sebuah partisi /home hal yang cantik. Ada versi baru dari OS? Hapus dan masukkan lagi semuanya, tetapi partisi /home anda jangan di sentuh dalam proses upgrade. Ingatlah untuk menyimpan salinan dari berkas konfigurasi anda, pikirkan hal ini sebelumnya!
  • Sebuah partisi terpisah untuk semua hal mungkin akan terakumulasi dalam sejumlah besar kuantitas berkas yang mungkin perlu untuk dihapus akan lebih cepat untuk format ulang dan buat partisi ulang daripada proses penghapusan (delete). Lihat upgrade-minifaq untuk contoh pada (/usr/obj).
  • Jika anda ingin membangun ulang sistem anda dari kode sumber untuk berbagai alasan, kode sumber berada di /usr/src. Jika anda tidak membuat sebuah partisi terpisah untuk /usr/src, pastikan /usr mempunyai sisa ruang yang cukup.
  • Fakta umum yang sering terlupakan: anda TIDAK perlu mengalokasikan seluruh ruang disk ketika menginstall! Karena anda saat ini akan berpikir untuk tidak membeli hard disk baru berkapasitas kurang dari 20G, hal ini akan bermanfaat dengan membiarkan sebuah chunk pada disk belum teralokasikan. Jika anda memperbesar sebuah partisi, dapat anda alokasikan partisi baru tersebut dari ruang belum terpakai, partisi kembar menjadi partisi yang baru, ubah /etc/fstab untuk rujukan partisi baru, tautkan ulang, sekarang anda mempunyai ruang kosong lebih siap pakai.
  • Jika anda membuat partisi anda terlalu mendekati kebutuhan sistem minimal, mungkin anda akan menyesalinya kemudian hari, ketika anda melakukan upgrade sistem.
  • Jika anda membolehkan user untuk menulis pada /var/www (misal, halaman web pribadi), mungkin anda akan meletakkannya pada sebuah partisi terpisah, sehingga anda dapat menggunakan quota untuk membatasi ruang yang mereka gunakan, dan jika mereka memenuhi quota ruang, tidak satupun bagian sistem anda akan terpengaruh.

4.8 - Multibooting OpenBSD/i386

Multibooting adalah mempunyai beberapa sistem operasi dalam satu komputer, dan bisa berarti pemilihan OS mana yang akan di boot. Ini bukan hal yang biasa! Jika anda tidak paham apa yang anda lakukan, bisa jadi anda akan menghapus sejumlah besar data dari komputer anda. Pengguna baru OpenBSD SANGAT disarankan untuk memulai dengan sebuah hard disk kosong pada sebuah mesin khusus, dan kemudian praktekkan konfigurasi yang anda inginkan pada sistem non-produksi sebelum mencoba konfigurasi multibooting pada mesin produksi. FAQ 14 mempunyai lebih banyak keterangan tentang proses boot OpenBSD.

Sewaktu multibooting, kebutuhan seluruh sistem operasi harus sesuai dengan konfigurasi anda. Orang sering bertanya adakah cara mengatasi batasan boot 8G pada OpenBSD. Sementara sejumlah program menyatakan bisa melebihi batasan boot tersebut pada sejumlah sistem operasi lain, tidak satupun program itu yang tahu bagaimana harus melakukan hal tersebut pada versi OpenBSD terkini.

Berikut ini adalah beberapa pilihan untuk melakukan multibooting:

Mengatur partisi aktif

Hal ini mungkin paling banyak dilupakan, dan masih, terkadang solusi terbaik untuk multibooting. Mudahnya aturlah partisi aktif dari OS yang anda jalankan saat ini sehingga bisa langsung boot ke OS tersebut setelah reboot. Secara tidak nyata setiap OS menawarkan sebuah program untuk hal ini; pada OpenBSD adalah fdisk(8), program bernama mirip ada pada Windows 9x atau DOS, dan banyak sistem operasi lainnya. Bisa sangat diinginkan OS atau sistem mengambil waktu cukup lama untuk shut down dan reboot -- anda bisa mengaturnya dan memulai proses reboot, lalu berjalan ke dapur, mengambil secangkir kopi, dan kembali ke komputer telah booting ke OS yang anda inginkan -- tidak ada waktu untuk menunggu Kejadian Ajaib guna meilih OS pada waktu berikutnya.

Disket booting

Jika nada mempunyai sebuah sistem yang jarang boot ke OpenBSD (atau tidak ingin user lain pada komputer tersebut tahu setiap perubahan sistem), pilihlah menggunakan disket booting. Mudahnya gunakan salah satu disket instalasi stnadar OpenBSD, dan buat sebuat berkas /etc/boot.conf (ya, anda juga akan harus membuat sebuah direktori /etc pada disket tersebut) berisi: 
     boot hd0a:/bsd
agar sistem boot dari hard drive 0, partisi 'a' OpenBSD, berkas kernel /bsd. Perhatikan anda bisa juga boot dari drive lain dengan sebuah baris seperti: "boot hd2a:/bsd" untuk boot dari hard drive ketiga dari sistem anda. Untuk boot OpenBSD, masukkan disket anda tadi, lalu reboot. Untuk boot OS lain, keluarkan disket tadi, lalu reboot.

Pada kasus ini, program boot(8) dimuatkan dari disket, mencari dan membaca /etc/boot.conf. Baris "boot hd0a:/bsd" memerintahkan proses boot(8) pada tempat kernel dimuatkan -- pada kasus ini, HD pertama yang dilihat oleh BIOS. Mohon diingat, hanya sebuah berkas kecil (/boot) dimuatkan dari disket -- sistem memuat keseluruhan kernel dari hard disk, sehingga ini hanya menambahkan sekitar 5 detik waktu untuk booting.

NTLDR pada Windows NT/2000/XP

Untuk multiboot OpenBSD dengan Windows NT/2000/XP, dapat anda gunakan NTLDR, pemuat boot yang digunakan NT. Untuk multiboot dengan NT, anda perlu sebuah salinan Partition Boot Record (PBR) OpenBSD. Setelah installboot, anda dapat menyalinnya ke sebuah berkas dengan dd(1): 
	# dd if=/dev/rsd0a of=openbsd.pbr bs=512 count=1

Sekarang boot NT dan letakkan berkas openbsd.pbr di C:. Tambahkan baris seperti ini pada akhir berkas C:\BOOT.INI: 

	c:\openbsd.pbr="OpenBSD"

Ketika anda reboot, anda seharusnya bisa memilih OpenBSD dari menu pemuat NT. Terdapat keterangan lanjut tentang NTLDR di Panduan Hacking NTLDR.

Pada Windows XP dapat juga anda sunting keterangan boot menggunakan GUI; lihat XP Boot.ini HOWTO.

Catatan: Pemuat boot Windows NT hanya bisa untuk boot OS dari hard drive primer. Anda dapat menggunakannya untuk boot OpenBSD yang berada pada drive kedua di sistem anda.

Pemuat boot lainnya

Beberapa pemuat boot lainnya yang berhasil dipakai pengguna OpenBSD termasuk diantaranya GAG, OSBS, The Ranish Partition Manager dan GRUB.

OpenBSD dan Linux (i386)

Mohon merujuk ke INSTALL.linux, yang memberikan intruksi mendalam untuk membuat OpenBSD berjalan bersama Linux.

4.9 - Kirimkan dmesg anda ke dmesg@openbsd.org setelah instalasi

Hanya mengingatkan orang-orang, ini hal penting untuk pengembang OpenBSD menelusuri perantikeras mana yang berjalan pada OpenBSD, dan peranti keras keras mana yang tidak berjalan sempurna.

Nukilan dari /usr/src/etc/root/root.mail 

If you wish to ensure that OpenBSD runs better on your machines, please do us
a favor (after you have your mail system configured!) and type something like:
 # dmesg | mail -s "Sony VAIO 505R laptop, apm works OK" dmesg@openbsd.org
so that we can see what kinds of configurations people are running.  As shown,
including a bit of information about your machine in the subject or the body
can help us even further.  We will use this information to improve device driver
support in future releases.  (Please do this using the supplied GENERIC kernel,
not for a custom compiled kernel, unless you're unable to boot the GENERIC
kernel).  The device driver information we get from this helps us fix existing
drivers. Thank you!

Jika anda ingin memastikan OpenBSD berjalan lebih baik lagi pada mesin anda, mohon
dengan senang hati (setelah sistem mail anda terkonfigurasi!) melakukan hal semisal berikut:
# dmesg | mail -s "Sony VAIO 505R laptop, apm works OK" dmesg@openbsd.org
sehingga bisa kami lihat jenis konfigurasi yang orang jalankan dengan OpenBSD. Seperti tertampil,
termasuk sejumlah kecil keterangan tentang mesin anda pada subyek atau badan pesan
dapat menolong kami lebih jauh. Kami akan gunakan keterangan ini untuk meningkatkan dukungan
pengendali divais di rilis berikutnya. (Mohon lakukan ini dari kernel GENERIC,
bukan dari kernel yang anda atur sendiri, kecuali anda tidak bisa boot dengan kernel
GENERIC.) Keterangan pengendali divais yang kami dapatkan disini membantu kami untuk memperbaiki
pengendali yang telah ada. Terima kasih!

Pastikan anda mengirim email dari sebuah akoun yang dapat juga menerima email sehingga pengembang OpenBSD dapat menghubungi anda jika mereka mempunyai sesuatu yang mereka ingin agar anda uji atau perubahan yang bisa membuat konfigurasi anda berjalan. Bukanlah hal penting sama sekali untuk mengirim email dari mesin yang sama dengan menjalankan OpenBSD, sehingga jika mesin tersebut tidak bisa menerima email, hanya tinggal lakukan 

$ dmesg | mail akoun-anda@mailanda.dom
dan kemudian alihkan pesan tersebut ke 
 dmesg@openbsd.org
dengan akoun-anda@mailanda.dom adalah akoun email aktif anda. (atau transfer keluaran dmesg menggunakan ftp/scp/floppydisk/carrier-pigeon/...)

CATATAN - Mohon hanya mengirim dmesg dari kernel GENERIC. Kernel kustomisasi dengan beberapa dukungan pengendali driver dihilangkan tidaklah membantu.

4.10 - Menambahkan rangkaian berkas install setelah instalasi

"Ah tidak!" Saya lupa untuk menambahkan sebuah rangkaian berkas install setelah selesai instalasi!"

Seringkali, anda menyadari anda benar-benar TIDAK memerlukan comp34.tgz (atau bagian sistem lainnya) sama sekali, tetapi anda tidak menyadari hal ini sewaktu anda menginstall sistem anda. Kabar baik: Terdapat dua cara mudah untuk menambahkan rangkaian berkas install setelah inisialisasi instalasi:

Menggunakan proses upgrade

Mudahnya anda boot media instalasi anda (CD-ROM atau disket), dan pilih Upgrade (daripada Install). Ketika nada mendapat daftar rangkaian berkas install , pilih yang anda lupa pilih saat instalasi awal, pilih sumber terdapat berkas tersebut, dan biarkan proses upgrade menginstallnya untuk anda.

Memakai tar(1)

Rangkaian berkas instalasi adalah berkas tar termampat, dan anda bisa memekarkannya secara manual dari filesistem utama: 

  # cd /
  # tar xzvpf comp34.tgz

JANGAN lupa 'p' pada perintah di atas guna mengembalikan perizinan berkas dengan tepat!

Satu kesalahan umum adalah anda pikir anda dapat gunakan pkg_add(1) untuk menambahkan sebuah rangkain berkas install yang terlupa. Tidak akan bisa. pkg_add(1) adalah untuk berkas paket, bukan berkas tar biasa seperti rangkaian berkas install.

4.11 - What is 'bsd.rd'?

bsd.rd adalah sebuah kernel "disk RAM". Berkas ini bisa sangat berguna; banyak pengembang yang secara cermat menjaga berkas ini tetap berada pada filesistem utama mereka sepanjang waktu.

Disebut sebuah "kernel disk RAM" untuk menjelaskan pada filesistem root kernel -- daripada di sebuah drive fisik, utilitas tersediakan setelah boot bsd.rd disimpan di kernel, dan dijalankan dari sebuah filesistem RAM. bsd.rd juga disertai dengan rangkaian utilitas yang bagus agar anda bisa mengelola dan menginstall sistem.

Pada beberapa platform, bsd.rd nyatanya merupakan teknik instalasi yang harus dipakai -- anda tempatkan kernel ini pada filesistem yang ada, boot dengannya, dan jalankan instalasi darinya. Pada kebanyakan platform, jika nada mempunyai sebuah versi sistem OpenBSD sebelumnya yang berjalan, anda bisa FTP bsd.rd versi baru, reboot dari berkas yang anda FTP tadi, dan install OpenBSD versi baru tanpa menggunakan media instalasi CD-ROM atau disket sama sekali.

Berikut ini adalah contoh booting bsd.rd pada sistem i386: 
  Using Drive: 0 Partition: 3
  reading boot.....
  probing: pc0 com0 com1 apm mem[639k 255M a20=on]
  disk: fd0 hd0+
  >> OpenBSD/i386 BOOT 2.02
  boot> boot hd0a:/bsd.rd
. . . normal boot to install . . .
Seperti terlihat, anda akan dibawa ke program instalasi, tetapi anda dapat juga menuju ke shell untuk melakukan pengelolaan sistem anda.

Aturan umum pada booting bsd.rd adalah mengubah kernel boot anda dari /bsd ke bsd.rd dengan cara penggunaan apapun pada platform anda.


4.12 - Masalah yang umum dijumpai saat instalasi

4.12.1 - Compaq saya hanya mengenali RAM sampai 16M

Beberapa sistem Compaq mempunyai masalah dengan sistem RAM lengkap tidak dikenali oleh pemuat boot OpenBSD tingkat kedua secara tepat, dan hanya 16M yang dikenali dan digunakan oleh OpenBSD. Hal ini bisa diperbaiki baik dengan membuat/menyunting berkas /etc/boot.conf , atau dengan memasukkan perintah pada prompt "boot>" sebelum OpenBSD dimuatkan. Jika anda mempunyai sebuah mesin dengan RAM 64M, tetapi OpenBSD sebelumnya hanya mengenali 16M memori awal, perintah yang bisa anda gunakan adalah: 
     machine mem +0x3000000@0x1000000
untuk menambahkan 48M (0x3000000) setelah 16M pertama (0x1000000). Khususnya, jika anda mempunyai mesin dengan masalah ini, anda masukkan perintah di atas pertama kali saat prompt boot> dari disket/CD-ROM instalasi, memuatkan sistem, reboot, dan membuat sebuah berkas /etc/boot.conf dengan baris di atas didalamnya sehingga semua boot selanjutnya akan mengenali seluruh kapasitas RAM yang sebenarnya.

Juga telah dilaporkan bahwa sebuah update ROM akan memperbaiki hal ini pada beberapa sistem.

4.12.2 - i386 saya tidak mau booting setelah instalasi

Instalasi anda kelihatannya berjalan lancar, tetapi pada booting pertama, tidak anda lihat pertanda OpenBSD mencoba untuk booting. Terdapat beberapa alasan umum untuk masalah ini:
  • Tidak ada partisi yang ditandai aktif saat fdisk(8). Untuk memperbaiki ini, reboot mesin dengan disket booting atau media instalasi lain, dan "flag" sebuah partisi menjadi "active" (bootable). Lihat here dan here
  • Tidak ada pemuat boot sahih yang diletakkan pada disk. Jika jawaban anda "Y" saat ditanya "Use entire disk for OpenBSD?" selama instalasi, atau mnggunakan pilihan "reinit" pada fdisk(8), boot recoed OpenBSD terpasang pada Master Boot Record disk; jika jawaban anda tidak, kode boot utama tidak diubah sama sekali. Hal ini akan menjadi sebuah masalah jika tidak ada boot record lain yang ada. Solusinya adalah boot media instalasi lagi, masuk ke shell dan lakukan fdisk(8) dan gunakan pilihan "update" .
  • Pada kejadian kasus yang sangat jarang, seringkali terjadi kesalahan pada pemasangan pemuat boot tingkat kedua. Pemasangan ulang pemuat boot tingkat kedua dibicarakan di sini.
  • Anda menggabungkan berkas instalasi dan disket boot a.out dan ELF . OpenBSD/i386 telah bertransisi dari fromat a.out sebelumnya menjadi format ELF pada binari yang dipakai sejak beberapa waktu setelah 3.3-release. Pemuat boot OpenBSD tidak bisa booting kernel OpenBSD 3.3-release, atau sebaliknya pemuat boot 3.3-release tidak bisa booting sebuah kernel OpenBSD 3.4. Anda harus menggunakan sebuah disket booting yang sesuai dengan OpenBSD yang akan anda install. Kegagalan pada hal ini akanmenampilkan pesan seperti "failure(79)" dan/atau "Inappropriate file type or format".
  • Anda abaikan seluruh peringatan dan penjelasan tentang batasan boot 8G. Hal in akan membawa anda pada sebuah kondisi menggantung pada proses terawal booting, seringkali dengan pesan, "Bad magic".

4.12.3 - Mesin saya (yang lama, lambat) bisa boot, tetapi menggantung saat tahap ssh-keygen

Hal ini terlihat mesin anda berjalan wajar, hanya mengambil waktu sementara untuk melakukan proses pembangkitan kunci ssh. Sebuah SparcStation2 atau sebuah Macintosh Quadra bisa membuatuhkan waktu sampai 45 menit atau lebih untuk menyelesaikan tiga tahap ssh-keygen(1) , beberapa mesin akan perlu waktu lebih lama. Biarkan saja proses itu selesai; hal ini hanya terjadi sekali setiap instalasi.

4.12.4 - Saya mendapat pesan "Failed to change directory" saat instalasi

Ketika instalasi dengan FTP dari sebuah snapshot selama tahap -beta dari daur pengembangan OpenBSD, anda mungkin akan melihat ini: 

  Do you want to see a list of potential FTP servers?  [y] ENTER
  Getting the list from 192.128.5.191 (ftp.openbsd.org)... FAILED
  Failed to change directory.
  Server IP address or hostname?
Hal ini wajar dan kebiasaan yang diharapkan selama pra-rilis ini bagian dari daur. Program instalasi mencari daftar FTP di server FTP utama pada sebuah direktori yang tidak akan tersedia sampai tanggal rilis, sehingga anda mendapat pesan di atas.

Mudahnya gunakan daftar miror FTP untuk menemukan miror FTP kesukaan anda, dan secara manual masukkan nama miror ketika ditanyakan pada prompt.

Catatan: Anda seharusnya tidak melihat hal ini jika anda menginstall -release atau dari CD-ROM.

4.12.5 - Saat saya login, saya dapati "login_krb4-or-pwd: Exec format error"

Kerberos IV telah tidak dipakai pada OpenBSD 3.4, tetapi jika anda melakukan upgrade, binari Kerberos IV yang lama akan masih berada dalam sistem anda. Ini sebuah masalah pada platform i386, sebagaimana berkas Kerberos IV yang lama dalam format a.out, dan karenanya tidak bisa berjalan pada kernel ELF standar (yang mematikan emulasi a.out, seperti disebut di sini). Jika anda menemui masalah ini, anda perlu menimpa metode otentikasi krb4 saat anda log in: 
    OpenBSD/i386 (puffy.openbsd.org) (ttyC0)

    login: joeuser:passwd
    password:
Bisa anda gunakan sintaks "username:passwd" yang sama dengan sebuah koneksi ssh dan dengan su(1) untuk mengakses sistem anda. Kemudian suntinglah /etc/login.conf, dan hilangkan semua referensi pada krb4.

4.13 - Atur sendiri proses instalasi sesuai kebutuhan

berkas siteXX.tgz

Skrip install/upgrade OpenBSD membolehkan pemilihan sebuah rangkaian berkas instalasi buatan user sendiri yang disebut "siteXX.tgz", dengan XX adalah versi rilis (misal, 3.4). Rangkaian berkas instalasi siteXX.tgz adalah, seperti rangkaian berkas instalasi lainnya, sebuah arsip gzip(1) termampat tar(1) bermula pada '/' dan dimekarkan seperti rangkaian berkas instalasi dengan pilihan xzpf. Rangkaian berkas ini akan diinstall terakhir, setelah semua rangkaian berkas instalasi terpasang.

Dengan ini membuat user bisa menambahkan dan/atau menimpa berkas instalasi 'normal' dan akan bisa mengatur proses instalasi atau upgrade sesuai keinginan.

Beberapa contoh menggunakan berkas siteXX.tgz :

  • Buat sebuah berkas siteXX.tgz yang berisi seluruh berkas dengan perubahan yang anda buat sejak menginstall OpenBSD pertama kali dulu. Lalu, jika anda perlu membuat ulang sistem mudahnya pilih siteXX.tgz selama instalasi ulang dan seluruh perubahan yang anda buat dimasukkan ke sistem baru.
  • Buat serangkaian direktori khusus untuk mesin-mesin tertentu yang masing-masing berisi sebuah berkas siteXX.tgz yang berisi file tertetnu dari masing-maisng tipe mesin. Instalasi pada mesin (misal, boks dengan kartu grafis berbeda) dari sejumlah kategori khusus dapat diselesaikan dengan memilih berkas siteXX.tgz yang sesuai.
  • Letakkan berkas yang sering anda atur sesuai keinnginan anda pada sebuah berkas siteXX.tgz yang sama atau mirip -- /etc/skel, /etc/pf.conf, /var/www/conf/httpd.conf, /etc/rc.conf, dll.

Skrip install.site/upgrade.site

Sebagai tahap terakhir pada proses install/upgrade, skrip akan mencari pada direktori utama sistem yang baru terinstall/terupgrade install.site atau upgrade.site, seperti sesuai dengan proses saat ini, dan menjalankan skrip ini dalam sebuah lingkungan terchroot pada root sistem terinstall dan terupgrade. Ingat, upgrade diselesaikan dari sebuah filesistem yang telah di boot, sehingga filesistem tujuan anda aktualnya akan ditautkan di /mnt. Namun, skrip anda bisa ditulis seperti jika berjalan pada root filesistem normal. Karena skrip ini berjalan setelah semua berkas utama terinstall, anda mendapat hampir fungsionalitas penuh pada sistem anda (meskipun, dalam mode single user) saat skrip anda berjalan.

Perhatian bahwa skrip install.site bisa jadi ada pada sebuah berkas siteXX.tgz file, sementara skrip upgrade.site bisa diletakkan dalam direktori utama sebelum upgrade, atau bisa diletakkan pada sebuah berkas siteXX.tgz .

Skrip ini bisa digunakan untuk melakukan apa saja yang dimungkinkan dalam sebuah skrip.

  • Hapus berkas yang terinstall/terupgrade yang tidak anda inginkan berada pada sistem.
  • Hapus/upgrade/install paket yang anda inginkan berada pada sistem.
  • Laukan sebuah backup/arsip segera dari sistem baru sebelum anda menghubungkannya dengan jaringan komputer atau Internet.
Gabungan siteXX.tgz dan berkas install.site/upgrade.site dimaksudkan memberi user kemampuan pengaturan sesuai kehendak sendiri tanpa harus membangun ulang rangkaian instalasi terkustom mereka sendiri.

4.14 - Bagaimana saya bisa menginstall sejumlah sistem yang mirip?

Ada beberapa alat yang bisa anda gunakan untuk membangun sejumlah sistem mirip OpenBSD.

Berkas siteXX.tgz dan install/upgrade.site

Lihat artikel di atas.

Pemulihan dari dump(8)

Pada kebanyakan platform, media booting menyertakan program restore(8) , yang bisa digunakan untuk mengembalikan sebuah backup yang dibuat dengan dump(8). Karenanya, anda bisa boot dari sebuah disket, CD, atau berkas bsd.rd , kemudian fdisk, disklabel, dan kembalikan konfigurasi yang diinginkan dari pita atau media lain dan memasang blok boot. Lebih jelas di sini.

Pencitraan disk

Sayangnya, tidak ada paket pembuatan citra disk yang cermat menangani FFS dan bisa membuat sebuah citra berisi hanya ruang berkas yang aktif. Kebanyakan solusi utama pembuatan citra disk akan memperlakukan partisi OpenBSD sebagai sebuah partisi "generik", dan dapat membuat citra dari keseluruhan disk. Hal ini sering sudah memenuhi keinginan anda, tetapi umumnya dengan sejumlah besar ruang tersia-siakan -- sebuah partisi kosong, 10G /home akan membutuhkan 10G ruang pada citra, meski tidak ada satupun berkas dalam partisi tersebut, anda tidak akan bisa langsung menggunakan ruang tambahan, dan anda tidak akan bisa menginstall citra tersebut pada hard disk yang lebih kecil.

Jika berikut situasi yang bisa diterima, bisa anda temukan perintah dd akan melakukan apa yang anda perlukan, membuat anda bisa menyalin satu disk ke disk lainnya, sektor-per-sektor. Ini akan menyediakan fungsi yang sama seperti program komersial tanpa biaya.

4.15 - Bagaimana saya mendapatkan pesan dmesg(8) untuk melaporkan masalah instalasi?

Ketika melaporkan sebuah masalah, sesuatu yang kritis untuk menyertakan dmesg(8) sistem secara lengkap. Namun, seringkali anda perlu melakukannya, ini karena sistem tidak berjalan bagus atau tidak mau terinstall sehingga anda tidak punya disk, jaringan, atau sumber lain yang anda perlukan untuk mengirimkan dmesg yang sesuai ke milis. Namun, ada beberapa cara lain:
  • Floppy disk: Disket dan CD-ROM booting mempunyai cukup utilitas untuk membuat anda mencatat dmesg ke disket DOS guna dibaca pada mesin lain. Tempatkan sebuah disket MSDOS pada disk drive anda dan eksekusi perintah berikut: 
         mount -t msdos /dev/fd0a /mnt
     dmesg >/mnt/dmesg.txt
     umount /mnt
    Jika anda memiliki sistem OpenBSD lainnya, anda dapat juga menuliskannya pada sebuah disket yang bisa dibaca OpenBSD -- sering, disket booting mempunyai cukup ruang untuk menyimpan hasil dmesg. Pada kasus tersebut hilangkan "-t msdos" di atas.

  • Serial Console: Lihat artikel ini tentang cara konfigurasi serial console, lalu menangkap keluarannya ke file.

  • FTP: Dibawah beberapa keadaan, anda mungkin bisa menggunakan klien ftp(1) pada disket booting atau CD-ROM untuk mengirim dmesg ke server FTP lokal , yang bisa anda peroleh kembali.

4.16 - Upgrade/install-ulang OpenBSD/i386 menggunakan bsd.rd-a.out

Merupakan hal yang mungkin untuk melakukan upgrade dan install menggunakan kernel bsd.rd . Namun, dengan OpenBSD 3.4, platform i386 dialihkan format terkeksekusi dari a.out menjadi ELF, sehingga boot loaders yang lebih tua (OpenBSD 3.3 dan sebelumnya) tidak bisa menjalankan kernel bsd.rd format-baru.

Untuk mengatasi masalah ini, dan membuat upgrade bisa dilakukan menggunakan bsd.rd, sebuah bsd.rd versi a.out telah disediakan di distribusi FTP. Berkas ini, bsd.rd-a.out, bisa dibooting oleh OpenBSD 3.3 dan sebelumnya, tetapi ini adalah kernel OpenBSD 3.4 asli, termasuk pemuat boot ELF yang baru, sehingga bisa digunakan untuk bootstraping OpenBSD/i386 3.4 dari sebuah sistem yang lebih tua.

Segera download bsd.rd-a.out dan letakkan pada direktori root mesin anda. Boot kernel tersebut dan bukannya kernel bsd atau bsd.rd normal seperti ditunjukkan di sini (memilih bsd.rd-a.out sebagai kernel booting anda, sudah pasti itu).

Jika anda ingin menginstall -current, sangat disarankan anda pertama kali menginstall sebuah sistem 3.4-release minimal (base34.tgz, etc34.tgz, bsd), lalu install-ulang menggunakan berkas bsd.rd dari -snapshot .
