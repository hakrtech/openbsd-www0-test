Antes de Empezar

Antes de intentar instalar OpenBSD es necesario llevar a cabo unas cuantas comprobaciones.

  • La primera y principal es la de asegurarse de que su hardware tenga soporte en OpenBSD. Puede ver una lista actualizada de hardware para el que existe soporte en i386.html.
  • Si tiene pensado instalar OpenBSD por medio de su dispositivo de red, es primordial que se asegure de que exista soporte para su dispositivo de red.
  • Asegúrese de haber hecho copias de seguridad de toda la información que necesite, especialmente si quiere configurar un sistema con multiarranque.

Crear Discos de Arranque de OpenBSD

Si la BIOS de su máquina tiene implementado el arranque directo desde el CDROM, puede arrancar desde el CD de OpenBSD y de este modo no necesitará crear ningún disco de arranque. Si esta opción existe la podrá econtrar en la BIOS de su máquina, en la sección que trate sobre la configuración de arranque.

OpenBSD se distribuye con dos imágenes de los discos de arranque separadas, las cuales contienen soporte para diferentes dispositivos. Puede ver las diferencias a continuación. De cualquier modo necesitará leer i386.html para ver si existe soporte para otros dispositivos en uno u otro disco.

  • floppy26.fs tiene soporte para los siguientes dispositivos (que no están soportados en floppy26B.fs):

    Ethernet:

    • NE2000 PCMCIA
    • 3com 3c5xx PCMCIA
    • Xircom PCMCIA
    Disco:
    • ISAPNP/PCMCIA IDE (las dos imágenes tienen soporte para ISA y PCI)
    • BusLogic [57]4X SCSI
    • Adaptec 154[02] SCSI
    • Adaptec 174[024] SCSI
    • Adaptec 274X/284X/2940 SCSI
    • NCR 538XX SCSI
    • AdvanSys 1200A/B/Ultra/Ultrawide SCSI
    • Seagate ST0[12] SCSI

  • floppyB26.fs contiene soporte para los siguientes dispositivos (que no están soportados en floppy26.fs):

    Disco:

    • Qlogic ISP [12]0x0 SCSI/FibreChannel
    • UltraStor [123]4f SCSI
    • WD7000/TMC-7000 SCSI

Una vez haya obtenido la imagen del disco de arranque correcta, debe crear su disco de arranque. Para ello asegúrese de que utiliza un disquete que ya haya sido formateado (el proceso de formateado le dará la composición básica del disco, con sectores y pistas, de modo que el dispositivo de la disquetera lo pueda reconocer).

En DOS, use la orden FORMAT. En OpenBSD debe usar fdformat(1) del modo siguiente (nota: fd0a se refiere a todo el disquete, a menos que tenga un "disklabel" en su disquete): 

    # fdformat /dev/fd0a
Format 1440K floppy `/dev/fd0a'? (y/n): y
Processing VVVVVVVVVVVVVVVVVVVVVVVVVVVVVVVVVVVVVVVV done.

Según la arquitectura que use, existen varias maneras diferentes de crear un disco de arranque. Hay algunas herramientas disponibles en el directorio ftp tools, existente en las réplicas del servidor de ftp, o en el directorio 2.6/tools del primer CD de OpenBSD. Si lo hace desde un Sistema Operativo de tipo UNIX, puede usar la orden dd(1) para copiar la imagen a su disquete. A continuación puede ver un ejemplo de cómo usar dd(1) para copiar una imagen del disco de arranque de OpenBSD a un disquete: 

    # dd if=floppy26.fs of=/dev/rfd0c bs=126b

Windows o NT no disponen de esta opción. Para éstos existen otras opciones. Los usuarios de Windows necesitarán el programa rawrite que pueden encontrar en el CD de OpenBSD o por ftp.

    Ejemplo del uso de rawrite: 
    C:\> rawrite
RaWrite 1.2 - Write disk file to raw floppy diskette

Enter source file name: floppy26.fs
Enter destination drive: a
Please insert a formatted diskette into drive A: and press -ENTER- : >Enter<

Rawrite no funciona en NT. Sin embargo existen otras herramientas como fdimage o ntrw para copiar imágenes. Estas dos se encuentran disponibles en el CD de OpenBSD o por ftp.
Ejemplos del uso de cada uno de estos programas:

    Ejemplo del uso de fdimage: 
    C:\> fdimage -q floppy26.fs a:
    Ejemplo del uso de ntrw: 
    C:\> ntrw floppy26.fs a:
3.5", 1.44MB, 512 bytes/sector
bufsize is 9216
1474560 bytes written

Del Arranque al ``fdisk''

Ahora que tiene su disco de arranque creado, ya puede arrancar el sistema de instalación. Cuando arranque con este disquete, el núcleo del sistema y todos los programas que usa OpenBSD para instalarse se cargarán en la memoria. Si surge algún problema y su disco de arranque no inicia, es probable que se deba un disquete en malas condiciones. ¡Este problema es muy común! Si su disco de arranque falla al iniciar, inténtelo una segunda vez, y si ésta fallara, tire el disquete y obtenga uno nuevo. Recuerde que, aunque ese disquete le funcionara correctamente con anterioridad, OpenBSD usa prácticamente todos y cada uno de los bits disponibles en el disquete, y la causa puede estar en que alguno de ellos se encuentre en una mala condición. Cualquier error en el disquete impedirá que OpenBSD arranque normalmente.

Si inició sin problemas verá un montón de texto blanco sobre fondo azul; ese texto son mensajes del núcleo del sistema que le indica los dispositivos que ha encontrado, y dónde los ha encontrado. A continuación verá lo siguiente: 

    sh: /etc/rc: No such file or directory
Enter pathname of shell or RETURN for sh: <Enter>
erase ^?, werase ^W, kill ^U, intr ^C
Lógicamente su respuesta será I, para Instalar 
    (I)nstall, (U)pgrade or (S)hell? I
A continuación verá una descripción de la instalación, y le preguntará por su tipo de terminal... pulse intro para aceptar la opción predefinida, pcvt es el tipo de terminal correcto.
    You can run a shell command at any prompt via '!foo' or escape to a shell by simply typing '!'. Specify terminal type [pcvt25]: <Intro>
El siguiente paso es seleccionar el disco duro en el que va a instalar OpenBSD. Si tiene datos en este disco y quiere hacer una copia de seguridad de éstos, asegúrese de hacerlo antes de comenzar este proceso. 
    Available disks are:

wd0

Which disk is the root disk? [wd0]
Si va a usar un disco primario IDE, la respuesta correcta será wd0. Si va a usar un disco primario SCSI, la respuesta correcta será sd0. Si va a instalar OpenBSD en otro disco, como pueda ser el secundario, seleccione la respuesta apropiada (wd1, sd1, etc...) para éste. Si la respuesta entre corchetes (en el ejemplo anterior, [wd0]) es la apropiada, puede pulsar intro. A continuación le preguntará: 
    Do you want to use the *entire* disk for OpenBSD? [no]
Si no quiere usar el disco entero para OpenBSD, pulse intro. Pasará al juego de las particiones con fdisk. En la sección 14.2 de las Preguntas Frecuentes encontrará las instrucciones sobre cómo jugar a este juego. Recuerde que estas particiones no son sus particiones Unix. Sólo necesita una partición en esta tabla para OpenBSD (cualquier otra partición será para otros sistemas operativos, como Windows).

Si contestó "yes" a esta sección, OpenBSD se encargará de hacer el proceso de fdisk por Vd, y le colocará en el disco correcto.

Configurar el Disco de OpenBSD con ``disklabel''

Para una explicación más detallada sobre cómo usar disklabel, lea la sección 14.1 de las Preguntas Frecuentes. Para obtener una idea aproximativa de los tamaños necesarios para sus sistemas de archivo en el disco, lea la sección 4.3 de las Preguntas Frecuentes.

En una máquina multiusuario es conveniente crear, como mínimo, cinco sistemas de archivos diferentes: uno para /; otro para swap; otro para /var; otro para /tmp; y otro para /usr. Si sus usuarios van a almacenar ficheros de gran tamaño, entonces debería crear también un sistema de archivos diferente para /home.

Existen cuatro razones principales para usar sistemas de archivos separados en lugar de tenerlo todo en uno o dos sistemas de archivos:

  • Seguridad: Puede marcar algunos sistemas como 'nosuid', 'nodev', 'noexec', 'readonly', etc...
  • Estabilidad: Un usuario o un programa con un funcionamiento incorrecto, puede llenar un sistema de archivos con basura si tiene permiso de escritura sobre éste. Sus programas más críticos, que por supuesto funcionarán sobre un sistema de archivos diferente, no se verán interrumpidos.
  • Velocidad: Un sistema de archivos sobre el que se escribe con frecuencia puede sufrir algo de fragmentación (por suerte, el sistema de archivos ffs, que es el que usa OpenBSD, no es propenso a la fragmentación).
  • Integridad: Si un sistema de archivos se corrompe por cualquier motivo, sus otros sistemas de archivos se mantendrán en buen estado.

Configurar los Puntos de Montaje y Formatear los Sistemas de Archivos

Configurar los puntos de montaje es mucho más fácil de lo que pueda parecer. Después de configurar los sistemas de archivos con disklabel, verá algo como lo siguiente: 

    The following filesystems will be used for the root filesystem and swap:
        wd0a    /
        wd0b    swap

Mount point for wd0d (size=65520k) [/tmp, RET, none, or done]? <Intro>
Mount point for wd0e (size=65520k) [/var, RET, none, or done]? <Intro>
Mount point for wd0g (size=307440k) [/usr, RET, none, or done]? <Intro>
Mount point for wd0h (size=1926256k) [/home, RET, none, or done]? <Intro>

Aquí se puede ver que wd0a se ha configurado como root, y wd0b como swap. Así será en todos los casos a menos que wd0 cambie por cualquier otro dispositivo que esté usando. En el ejemplo anterior nuestros puntos de montaje ya han sido definidos por disklabel(8), pero aún así deberían ser revisados para poder estar seguros de que toda la información es correcta, ¡es su última oportunidad para cambiarla! Para aquellos usuarios con más de un disco duro, ésta también será su oportunidad para configurar sus otros discos. Hágalo con calma y no intente meter todos sus puntos de montaje en un sólo disco. A continuación aparecerá una lista como la siguiente: 

    Now you can select another disk to initialize.  (Do not re-select a
disk you have already entered information for).  Available disks are:

wd0
wd1

Which one? [done] <Intro>

You have configured the following devices and mount points:

wd0a /
wd0d /tmp
wd0e /var
wd0g /usr
wd0h /home

Filesystems will now be created on these devices.
If you made any mistakes, you may edit this now.
Edit using ed? [n] <Enter>

Como puede ver, en el ejemplo se decidió no usar wd1 en este momento. Si se hubiera escogido la opción de configurar otro disco, se habrían seguido exactamente los mismos pasos que se siguieron para la configuración del primero. También tiene una última oportunidad para hacer correcciones con ed(1), pero es mucho más facil verificar la información con anterioridad. Una vez que los sistemas de archivos han sido configurados, es el momento de formatear los discos. Aparecerá un aviso diciendo que toda la información será borrada. Por esta razón, si está en un sistema de arranque múltiple, es muy importante haber hecho copias de seguridad. He aquí un ejemplo del aviso: 

    ============================================================
The next step will overwrite any existing data on:
        wd0a wd0d wd0e wd0g wd0h

Are you really sure that you're ready to proceed? [n] y

Nótese que la respuesta a esta pregunta es no por definición; así pues, tendrá que indicarle explícitamente que continúe con el formateo. En caso contrario pasaría a la "shell" desde donde podría empezar a instalar de nuevo escribiendo la orden install, o simplemente reiniciando otra vez con su disco de arranque. Si ha respondido yes, comenzará a dar formato a todos los sistemas de archivos; esto puede tardar un tiempo, dependiendo del tamaño de su disco.

Después de haber configurado su red, el guión de instalación le ofrecerá la oportunidad de realizar cambios a su configuración de red (principalmente a posibles errores). Instalar desde un CDROM

Para instalar desde un CD-ROM debe usar la opción 'C' cuando escoja su medio de instalación. De este modo aparecerá una lista de los dispositivos que sean identificados como CD-ROM en su máquina. La mayor parte de los usuarios sólo dispondrán de un dispositivo de CDROM, pero si no es su caso entonces tendrá que asegurarse de que elije bien el dispositivo desde el que va a instalar OpenBSD. Después de escoger el dispositivo correcto le preguntará por el sistema de archivos en el que se instalarán los ficheros. Si está usando el CDROM de OpenBSD tendrá que escoger 'c', ya que 'c' aquí representa al CDROM. Siguiendo con los ejemplos: 

    Install from (f)tp, (h)ttp, (t)ape, (C)D-ROM, (N)FS or local (d)isk? c

the CD-ROM device containing the partition with the installation sets:

cd0

Lo siguiente será indicar '/2.6/i386/' como el directorio en donde se encuentran los ficheros (también por definición). Después de esto le mostrará una lista de paquetes para instalar. Extract more sets? [n] <Intro>

La elección de paquetes no está completa todavía, falta una biblioteca que no se encuentra en el CD y que deberá bajarse Vd. mismo. Como se explica durante la instalación, si todavía no dispone de una conexión de red, esta biblioteca se puede añadir más tarde. En este ejemplo del proceso de instalación se asume que existe la disponibilidad de una red, y que ésta se encuentra ya preparada durante el proceso de instalación. Lo primero que verá es una explicación, y luego se le ofrecerá la elección del paquete que desee seleccionar. Después de seleccionar su paquete, tendrá que configurar su red. 

    Two OpenBSD libraries (libssl and libcrypto, based on OpenSSL) implement many
cryptographic functions which are used by OpenBSD programs like ssh, httpd, and
isakmpd.  Due to patent licensing reasons, those libraries may not be included
on the CD -- instead the base distribution contains libraries which have had
the troublesome code removed -- the programs listed above will not be fully
functional as a result.  Libraries which _include_ the troublesome routines
are available and can be FTP installed, as long as you meet the follow (legal)
criteria:
  (1) Outside the USA, no restrictions apply. Use ssl26.tar.gz.
  (2) Inside the USA, non-commercial entities may install sslUSA26.tar.gz.
  (3) Commercial entities in the USA are left in the cold, due to how the

Queremos recordar a los usuarios que es importante para los desarrolladores de OpenBSD el poder llevar un seguimiento del hardware que funcione, así como del hardware que no funcione bien...

Si desean asegurarse de que OpenBSD funcione mejor en sus máquinas, háganos
el favor (¡Después de haber configurado su sistema de correo electrónico!) de

	dmesg | mail dmesg@openbsd.org
para que podamos ver los tipos de configuraciones que tengan los usuarios.
Usaremos esta información para mejorar el soporte de los dispositivos de
control en versiones venideras (esta información nos servirá si nos la
proveen con el núcleo GENERIC que viene en la instalación, no con un núcleo
compilado a medida).  La información sobre dispositivos de control que
obtengamos de aquí nos ayudará a arreglar los dispositivos ya existentes.
Revise también la sección 14.7

Asegúrese de que nos envía el mensaje desde una cuenta en la que también pueda recibir correo, para que los desarrolladores puedan contactar con Vd si hubiera algo que quisieran comprobar o cambiar con el fin de afinar su configuración. No es importante que envíe el mensaje desde la misma máquina en la que esté instalado OpenBSD; si desde esa máquina no puede recibir correo, puede hacer lo siguiente 

 dmesg | mail su-cuenta@sucorreo.dom
y reenviar el mensaje a 
 dmesg@openbsd.org
en donde su-cuenta@sucorreo.dom es us cuenta de correo normal (o puede transferir la salida de dmesg usando ftp, scp, disquete, paloma mensajera, etc... )

NOTA - Por favor, envíe sólo el dmesg correspondiente a un núcleo GENERIC. Los núcleos a medida en los que se han eliminado dispositivos de control no son de ninguna ayuda.

Pregunta: - ¿Es posible obtener una imagen ISO de OpenBSD?

Respuesta: - No. Los derechos de autor de la composición del CDROM oficial de OpenBSD pertenecen a Theo de Raadt, a modo de incentivo para que compren los CDs. Nótese que estos derechos sólo son aplicables a la composición del CDROM, y que OpenBSD en sí mismo es libre. No hay nada que pueda impedir a alguien de coger OpenBSD y hacer su propio CD.

Pregunta: - Tengo problemas de instalación con 8MB de RAM. ¿Qué puedo hacer?

Respuesta: - Muchos usuarios tienen problemas al final de la instalación porque el sistema se les cuelga en MAKEDEV all .... Existe una solución a este problema. Debe activar el sistema de archivos de swap durante la instalación. Para ello, siga los pasos que vienen a continuación:

  • Comience la instalación normalmente.
  • Configure sus discos con fdisk(8) y disklabel(8) como se ha descrito anteriormente.
  • Una vez que sus discos estén configurados, y DESPUÉS de que hayan sido formateados, pulse <CNTRL>-C para salir a la shell.
  • Reinicie el sistema con reboot.
  • Esta vez, evite las secciones fdisk(8) y disklabel(8). Esto lo puede hacer desde fdisk(8) escribiendo exit, y desde disklabel(8) escribiendo q.
  • Proceda con la instalación hasta que haya completado la Sección de Red de la instalación. Cuando haya finalizado, pulse <CNTRL>-z para salir a la shell.
  • Es desde aquí desde donde tendrá que activar swap. Para ello escriba /sbin/swapon wd0b, o sd0b, o cualquiera que sea su dispositivo de swap.
  • Para proceder con el resto de la instalación escriba fg.

Si ha seguido estas instrucciones y aún tiene problemas, envíe un mensaje a Eric Jackson <ericj@openbsd.org>.
