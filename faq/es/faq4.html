4.1 - Breve introducción a la instalación de OpenBSD

Esta sección de las preguntas frecuentes ahora contiene instrucciones para la instalación de OpenBSD 3.0. Pueden existir diferencias entre la instalación de la versión 3.0 y la de versiones anteriores de OpenBSD.

OpenBSD dispone de un procedimiento de instalación muy robusto, basado en texto. Este procedimiento de instalación fue diseñado para que se pudiera adaptar a casi todas las situaciones posibles con las que se puede encontrar un usuario. Además de la robustez, el procedimiento de instalación se puede llevar a cabo usando el disquete número 1. El procedimiento de instalación es muy parecido para todas las arquictecturas, aunque con frecuencia se requieren distintos grupos de herramientas para cada una de las arquitecturas. A continuación enumeraremos las diferentes opciones de las que dispondrán los usuarios de OpenBSD cuando instalen el sistema.

En la mayoría de arquitecturas hay varias opciones de instalación, incluido FTP, CDROM y archivos de disco local. Una opción para la que no hay soporte es la de descargar una imagen ISO para poder crear un CDROM propio. Los CDROM están disponibles a la venta.

La información que sigue a continuación se ofrece independientemente de la arquitectura que se utilice. Por favor, lea las instrucciones de instalación correspondientes a cada arquitectura, o busque información concreta sobre las arquitecturas en la cubierta del CD de OpenBSD.

4.1.1 - Arquitecturas con soporte en OpenBSD

OpenBSD 3.0 dispone de soporte para una número de arquitecturas que se mencionan por orden alfabético a continuación. Por favor, lea la página correspondiente a cada arquitectura para saber con detalle para qué componentes o modelos existe soporte en el sistema.

  • alpha - Máquinas DEC basadas en procesadores Alpha.
  • amiga - Modelos de Amiga basados en procesadores m68k (requieren MMU).
  • hp300 - Máquinas HP300/HP400 de Hewlett-Packard.
  • i386 - Computadoras (ordenadores) personales basados en procesadores Intel.
  • mac68k - La mayoría de modelos de Apple Macintosh basados en el procesador MC680x0.
  • mvme68k - Tarjetas VME MVME147/16x/17x 68K de Motorola.
  • macppc - Modelos de Apple con procesadores PowerPC.
  • sparc - La plataforma SPARC de Sun Microsystems.
  • sun3 - Modelos Sun3 basados en el procesador 68020 de Sun.
  • vax - Máquinas VAX de DEC.

4.1.2 - Medios de instalación con soporte

Existe la posibilidad de instalar OpenBSD desde varios tipos distintos de medios. A continuación se mencionan las opciones que son más comunes y que son independientes de la arquitectura. Estas opciones se pueden utilizar después de arrancar desde un CD-ROM o desde un disquete. Podrá encontrar más información sobre cómo crear un disco de instalación de OpenBSD más adelante.

CD-ROM Para instalar desde un CD-ROM es necesario haber adquirido un CDROM oficial de OpenBSD, o haber creado uno. Sin lugar a dudas ésta es la forma más fácil para instalar un sistema OpenBSD.
NOTA: Se puede arrancar desde los CD oficiales de OpenBSD siempre que su BIOS permita esta opción.
FTP Esta opción de instalación le permite instalar OpenBSD bajándose antes los paquetes de instalación en tiempo real a través de la red. Con esta opción puede escoger una dirección IP estática u obtener una dirección IP por medio de DHCP.
Local Filesystem Esta opción le permite instalar desde archivos disponibles previamente desde un sistema de archivos. En el disco de instalación hay soporte para sistemas de archivos de DOS, EXT2FS y FFS.

4.1.3 - Cómo crear disquetes de instalación con arranque de OpenBSD

Para crear un disquete con la imagen de instalación, primero debe obtener la imagen de arranque correcta desde uno de los sitios de distribución de OpenBSD. Puede encontrar una lista de servidores de FTP en la páginas de distribución por FTP de OpenBSD. Cada arquitectura dispone de una imagen que se puede bajar, excepto la plataforma i386, para la que se puede escoger entre 3 imágenes. Las diferencias entre los diferentes disquetes de instalación para la plataforma i386 se detallan más adelante. Para el resto de las arquitecturas sólo tiene que bajarse la imagen floppy30.fs correspondiente.
NOTA: La imagen cdrom30.fs se puede utilizar para crear un CD-ROM de instalación con arranque de OpenBSD.

La plataforma i386 tiene 3 discos de instalación distintos entre los que deberá escoger uno. Las diferencias se muestran a continuación:

  • floppy30.fs contiene soporte para la mayoría de sistemas de escritorio, con soporte para NIC PCI e ISA, IDE y adaptadoras SCSI simples. También algo de soporte PCMCIA.
  • floppyB30.fs contiene soporte para muchas controladoras de RAID, sin embargo se ha eliminado el soporte para muchas adaptadoras SCSI y NIC EISA e ISA. Éste será el disco de arranque a elegir para muchos servidores grandes. También hay aquí soporte para algunas adaptadoras SCSI poco frecuentes.
  • floppyC30.fs contiene soporte para los dispositivos Cardbus y PCMCIA que se encuentran en muchos portátiles.
  • cdrom30.fs contiene soporte para la mayoría de controladores del disco y de red del sistema instalado; en efecto, es una combinación de los tres discos de arranque. Se puede utilizar para crear un disquete de arranque de 2.88MB, o una imagen de arranque para un CD grabable.

La mayoría de usuarios de i386 usarán el disquete de instalación floppy30.fs.

Y sí, pueden existir situaciones en las que un disco de instalación se requiera para el soporte de su adaptadora de red. Por fortuna, esto es muy extraño, y generalmente se puede encontrar una solución.

Una vez que ya disponga de la imagen de disquete correcta, necesitará un disquete en blanco. Si hubiera ALGÚN sector erróneo en el disquete, la instalación fallaría con casi total seguridad. Si no está seguro, use la orden fdformat(1) para dar formato y verificar los sectores erróneos.

    # fdformat /dev/fd0a
    Format 1440K floppy `/dev/fd0a'? (y/n): y
    Processing VVVVVVVVVVVVVVVVVVVVVVVVVVVVVVVVVVVVVVVV done.

Si en su pantalla ve algo como lo anterior, entonces su disquete está en buen estado. Sin embargo, si no ve TODAS las «V», entonces es muy probable que su disco esté en mal estado, y debería probar con uno virgen. Si va a formatearlo desde MS-DOS, use la orden format de DOS.

Una vez que tenga un disquete vacío y con formato, puede empezar a volcar la imagen de instalación a éste. Si lo va a hacer desde un sistema OpenBSD u otro sistema operativo tipo UNIX, puede usar la utilidad dd(1). A continuación tiene un ejemplo de cómo hacerlo:

    # dd if=floppy30.fs of=/dev/rfd0c bs=126b

En cuanto haya volcado la imagen al disquete, asegúrese que la imagen volcada sea la misma que la original con la siguiente orden. Si está bien no verá ninguna salida, sólo otro punto de pedido.

    # cmp /dev/rfd0c floppy30.fs

Si va a crear esta imagen desde una plataforma de DOS/Windows, puede obtener las herramientas necesarias desde el directorio tools en cualquiera de las réplicas de ftp, o del directorio 3.0/tools en el CD1 de OpenBSD. Todo lo que necesitarán los usuarios de DOS/Windows 9x será el programa rawrite para volcar la imagen a su disquete de arranque.

    Ejemplo de utilización de rawrite:
    C:\> rawrite
    RaWrite 1.2 - Write disk file to raw floppy diskette

    Enter source file name: floppy30.fs
    Enter destination drive: a
    Please insert a formatted diskette into drive A: and press -ENTER- : <Intro>

Si usa NT, rawrite no le servirá. Puede utilizar otros programas como fdimage o ntrw para volcar la imagen al disquete de arranque. Estos dos programas están disponibles en los sitios de ftp o en los CD de OpenBSD. He aquí un ejemplo sobre cómo usar cada uno de ellos:

    Ejemplo de utilización de fdimage:
    C:\> fdimage -q floppy30.fs a:
    Ejemplo de utilización de ntrw:
    C:\> ntrw floppy30.fs a:
    3.5", 1.44MB, 512 bytes/sector
    bufsize is 9216
    1474560 bytes written

4.1.4 - Cómo arrancar con las imágenes de instalación de OpenBSD

Esta sección está dividida en subsecciones, de acuerdo con las diferentes arquitecturas más usadas y para las que existe soporte en OpenBSD. La hemos subdividido de este modo para que podamos dar a cada usuario las instrucciones correctas sobre lo que deben hacer en sus respectivas plataformas.

Arranque de i386

    Para la mayoría de personas, arrancar e instalar desde una imagen en la arquitectura i386 no es nada nuevo. Si va a usar el disquete con la imagen, introdúzcalo en su disquetera y encienda su máquina. Si va a usar el CD para el arranque, antes debe entrar en la BIOS de su máquina y configurar las opciones de arranque para permitir el arranque desde el dispositivo de CD. Algunas BIOS ya viejas no presentan esta opción, y por tanto tendrá que usar un disquete para el arranque. Pero no se preocupe, ya que aun arrancando desde un disquete podrá instalar desde el CD.

Arranque de sparc

    Para arrancar desde un disquete, introduzca su disquete con la imagen de instalación de OpenBSD en su disquetera. A continuación use la siguiente orden para arrancar desde el disquete:

      ok boot floppy

    Para arrancar desde el CDROM, introduzca su CD de OpenBSD en el dispositivo de CDROM. Si su estación Sun sólo tiene un dispositivo de CDROM, vaya al punto de arranque y desde éste ejecute 'boot cdrom':

      ok boot cdrom

    Esto sólo funcionará en el modo de órdenes nuevo. Si su punto de órdenes es el antiguo (una flecha a la derecha), escriba 'n' para entrar en el modo de órdenes nuevo (si está usando una vieja sparc anterior al modelo sun4c, es probable que no tenga un modo de órdenes nuevo, en cuyo caso tendrá que experimentar Vd mismo). Si tiene varios dispositivos de CDROM, tiene que arrancar desde el correcto. Pruebe probe-scsi desde el modo de órdenes nuevo.

      ok probe-scsi
      Target 0
      Unit 0 Disk QUANTUM LIGHTNING 365S
      Target 1
      Unit 0 Removable Disk QUANTUM EMPIRE_1080S
      Target 3
      Unit 0 Removable Disk Joe's CD ROMs

    Averigüe cuál es el CDROM desde el que quiere arrancar. Anote el número que le corresponda.

      ok boot /sbus/esp/sd@X,0

    Ahora que su disco de arranque ya ha sido creado, es el momento de arrancar desde él. Cuando arranque con este disquete, el núcleo del sistema y todos los programas que OpenBSD usa para instalarse se cargarán en la memoria. Si surgiera algún problema y su disco de arranque no funcionara, es probable que se deba a que el disquete estaba en mal estado. ¡Éste es un problema muy común! Si el disquete le falla durante el arranque, inténtelo de nuevo, y si le vuelve a fallar tírelo y utilice un disquete virgen y nuevo. Cuando un disquete falla en el proceso de arranque, aun cuando ese mismo disquete haya funcionado bien anteriormente, tenga en cuenta que OpenBSD usa prácticamente todos y cada uno de los bits disponibles en el disquete. Cualquier error en el disco impedirá que se cargue OpenBSD.

    Cuando haya arrancado con éxito verá una cantidad de texto en blanco sobre fondo azul; este texto son mensajes del núcleo del sistema que le indica qué dispositivos está encontrando y dónde los encuentra.

    Ahora le pedirá que introduzca el camino hasta el intérprete (shell) que vaya a utilizar o que pulse intro para seleccionar sh como intérprete predeterminado. Acepte este último, y a continuación le preguntará si desea instalar, actualizar o invocar al intérprete de órdenes ((I)nstall, (U)pgrade or (S)hell?). Seleccione la opción i para instalar, después de lo cual le mostrará un mensaje:

      rootdev=0x1100 rrootdev=0x2f00 rawdev=0x2f02
      Enter pathname of shell or RETURN for sh:
      erase ^?, werase ^W, kill ^U, intr ^C
      (I)nstall, (U)pgrade or (S)hell? i
      ==================================================
      Welcome to the OpenBSD/i386 3.0 installation program.

      This program is designed to help you put OpenBSD on your disk in a simple and
      rational way.

      As with anything which modifies your disk's contents, this program can cause
      SIGNIFICANT data loss, and you are advised to make sure your data is backed
      up before beginning the installation process.

      Default answers are displayed in brackets after the questions. You can hit
      Control-C at any time to quit, but if you do so at a prompt, you may have
      to hit return. Also, quitting in the middle of installation may leave your

      install, the install program will remember many of your old answers.

      You can run a shell command at any prompt via '!foo'

      Specify terminal type [vt220]: <Intro>

    En este mensaje le advierte que el uso de el programa puede provocar una pérdida de datos en el disco, y le aconseja que haga antes una copia de seguridad (respaldo) de sus datos. También le indica que puede abandonar el programa de instalación en cualquier momento, presionando las teclas Control-C (si lo hace desde un punto de pedido tendrá que pulsar intro a continuación), y que puede ejecutar órdenes mediante el intérprete (shell), precediéndolas con un signo de exclamación final (!orden), o invocar al intérprete mediante la orden ´!´.

    Después le preguntará por el tipo de terminal de su equipo. Dependiendo de la arquitectura que esté usando, el tipo de terminal que necesita puede variar. Por ejemplo, OpenBSD/sparc usa el tipo de terminal «sun». En OpenBSD/i386 puede escoger el predeterminado, «pcvt25», pulsando intro.

4.1.5 - Cómo configurar los discos durante la instalación

La configuración de los discos duros (rígidos) en OpenBSD es muy parecida para todas las plataformas. La configuración del disco duro para i386 se lleva a cabo en dos partes. Una con fdisk(8), y la otra con disklabel(8).

    The installation program needs to know which disk to consider the root disk.
    Note the unit number may be different than the unit number you used in the
    boot program (especially on a PC with multiple disk controllers).
    Available disks are:

    wd0

    Which disk is the root disk? [wd0] <Intro>
    Do you want to use the *entire* disk for OpenBSD? [no] yes

    [...]

Ésta es la parte de la instalación en la que escogerá qué disco quiere que sea su disco «raíz» (root). En este disco raíz será en donde se ubicarán los sistemas de archivos root ('/') y swap. wd0 indica un disco IDE, pero si sus discos son SCSI verá sd0 en su lugar. Debería existir una entrada para cada disco que encuentre OpenBSD. Si decide utilizar el disco completo para OpenBSD, el programa de instalación configurará el espacio del disco de forma automática; si, por el contrario, decidiera utilizar sólo parte del disco, el programa de instalación abriría el editor de discos, fdisk(8). Puede ver más información sobre el uso de fdisk(8) en la sección 14.2 de las preguntas frecuentes.

NOTA: OpenBSD no dispone de soporte para ningún sistema de archivos tipo journaling filesystems, como ReiserFS, JFS de IBM o XFS de SGI. En lugar de éstos usamos Soft Updates. El sistema de archivos de OpenBSD es FFS (Fast File System).

    label offsets are absolute, ie. relative to the start of the disk... NOT

En OpenBSD, la partición a corresponde al sistema de archivos raíz (root), y la partición b al espacio de memoria de intercambio (swap). c representa siempre al disco entero, y puede usar el resto de las letras para crear particiones para sistemas de archivos adicionales hasta un número total de 16 por disco. Antes de crear una nueva partición (v.g. >a a), debe comprobar que ésta no exista, y si existiera deberá eliminarla antes de crearla (v.g. >d a).

Para una máquina multiusuario es mejor crear un mínimo de cinco sistemas de archivos diferentes, uno para '/' (root), otro para swap, otro para /var, otro para /tmp, y otro para /usr. Si los usuarios del sistema van a guardar ficheros de gran tamaño, será conveniente tener también un sistema de archivos /home separado.

4.1.6 - Cómo configurar los puntos de montaje y dar formato a los sistemas de archivos

Existen cuatro razones principales para la utilización de sistemas de archivos separados, en lugar de meter todo junto en uno o dos sistemas de archivos:

  • Seguridad: Puede marcar algunos sistemas de archivos como 'nosuid', 'nodev', 'noexec', 'readonly', etc...
  • Estabilidad: Un usuario, o un programa que funcione mal, podría llenar un sistema de archivos con basura si tuviera permiso de escritura para éste. Con sistemas de archivos separados, sus programas críticos, que por supuesto se ejecutarían en ese sistema de archivos separado, no se interrumpirían.
  • Velocidad: Un sistema de archivos sobre el que se escribe con frecuencia puede llegar a sufrir algo de fragmentación (por suerte, los sistemas de archivos ffs que usa OpenBSD no sufren de fragmentaciones fuertes).
  • Integridad: Si un sistema de archivos se corrompiera por cualquier motivo, el resto estaría a salvo.

La configuración de los puntos de montaje es mucho más simple de lo que parece. Después de que haya configurado sus sistemas de archivos con disklabel(8), verá unas líneas de texto como las que siguen a continuación. Si configuró sus puntos de montaje con disklabel(8), esto se hará de forma automática, de lo contrario lo puede hacer desde aquí.

4.3 - ¿Cuánto espacio necesito para instalar OpenBSD?

Se sugiere usar los siguientes tamaños de espacios para una instalación completa del sistema. Las cifras incluyen el suficiente espacio adicional como para una instalación típica de un sistema en su casa conectado a internet.

  • Para un sistema multiusuario es recomendable triplicar estos valores.
  • Si va a instalar una cantidad significativa de software de terceras partes, haga una partición /usr grande. ¡Como mínimo triplique estos valores!
  • Para un sistema que vaya a gestionar mucho correo electrónico o páginas web (almacenados respectivamente en /var/mail y /var/www), la partición /var deberá ser bastante más grande.
  • Para un sistema multiusuario que genere muchos mensajes de registro (log), la partición /var aún tendrá que ser mucho más grande (/var/log).
  • Si va a recompilar el núcleo o el sistema desde el código fuente, le interesa crear una partición /usr bastante más grande, probablemente entre 800MB y 1GB más grande de la que se indica abajo.
Mientras lea esto recuerde que tanto /usr como /usr/X11R6 son, por regla general, parte del mismo sistema de archivos; o sea, parte de /usr, ya que no hay ninguna ventaja importante en mantenerlos como sistemas de archivos separados. 
    SYSTEM          /       /usr    /var    /usr/X11R6
alpha           56M     540M    27M     161M

hp300           31M     234M    24M      47M
i386            35M     229M    24M      72M

Cuando se encuentre en el editor del disco ("disklabel"), puede escoger que su sistema tenga sólo 'a' (sistema de archivos principal) y 'b' (memoria de intercambio, swap). El sistema de archivos 'a' que configure en disklabel será su partición raíz (root), que se debe componer de la suma de los tres valores principales anteriores (/, /usr, y /var), más algo de espacio para /tmp. La partición 'b' será la partición swap del sistema. Para esta partición recomendamos un mínimo de 32MB, pero si dispone de suficiente espacio en el disco use un mínimo de 64MB. Si tiene mucho espacio de sobre en el disco, lo mejor es que la haga de 256MB, o incluso de 512MB.

Existen cuatro razones principales para usar sistemas de archivos separados en lugar de meterlo todo en dos sistemas de archivos:

  • Seguridad: puede marcar algunos sistemas de archivos como 'nosuid', 'nodev', 'noexec', 'readonly', etc...
  • Estabilidad: un usuario, o un programa con un comportamiento incorrecto, puede llenar un sistema de archivos de basura si tiene permiso de escritura sobre éste. De este modo, los programas más críticos, que por supuesto funcionarán en un sistema de archivos diferente, no serán interrumpidos.
  • Velocidad: un sistema de archivos sobre el que se escriba con frecuencia puede acabar algo fragmentado. Por suerte, el sistema de archivos de tipo ffs, que es el que usa OpenBSD, no es propenso a la fragmentación.
  • Integridad: si un sistema de archivos se corrompiera por cualquier motivo, el resto de sistemas quedarían intactos.

4.4 - Multiarranque de OpenBSD (i386, alpha)

OpenBSD y NT (o XP)

Para tener OpenBSD y NT (o XP) en una misma máquina puede usar el gestor de arranque del último, NTloader. Para el multiarranque con NT necesita una copia de su «registro de la partición de arranque» (pbr, partition boot record de OpenBSD. Después de ejecutar installboot puede copiarlo del siguiente modo: 

    	# dd if=/dev/rsd0a of=openbsd.pbr bs=512 count=1

A continuación arranque NT y copie openbsd.pbr en c:. Añada una línea como la siguiente al final de c:\boot.ini: 

    	c:\openbsd.pbr="OpenBSD"

Cuando reinicie podrá seleccionar OpenBSD desde el menú del gestor de arranque de NT. Puede encontrar más información sobre NTloader en NTLDR Hacking Guide.

En Windows XP también puede editar la información de arranque usando la interfaz gráfica de usuario; puede ver más información en XP Boot.ini HOWTO.

OpenBSD y Windows o DOS (i386)

Para arrancar OpenBSD junto con Windows 3.1, Windows95, o DOS, debe usar un gestor de arranque en el sistema que pueda manejar OpenBSD, Windows y DOS. Algunos de estos gestores de arranque son osbs20b8.zip o The Ranish Partition Manager. Estos dos pueden arrancar las particiones de OpenBSD.

OpenBSD y Linux (i386)

Por favor, lea el documento INSTALL.linux, que incluye instrucciones a fondo sobre el funcionamiento de OpenBSD con Linux.

Usando el proceso de actualización

Arranque con su medio de instalación (CD-ROM o disquete) y elija la opción de actualización, Upgrade (en lugar de Install, para instalar). Cuando vea la lista de archivos de instalación, escoja los archivos que olvidó instalar la primera vez, seleccione su fuente de instalación, y deje que el programa de instalación lo instale.

Usando tar(1)

Los archivos de instalación son simples paquetes comprimidos, y se pueden extraer de forma manual desde el sistema de archivos raíz: 

    # cd /
# tar xzvpf comp31.tgz

NO se olvide de usar la opción p con la orden anterior, para que restaure los permisos de los ficheros correctamente.

Un concepto erróneo es el de pensar que puede usar pkg_add(1) para añadir archivos de instalación del sistema. Esto no funciona. pkg_add(1) es para archivos de programas, no para archivos genéricos como los de instalación.

4.7 - ¿Qué es bsd.rd?

bsd.rd es un núcleo del sistema en un «disco RAM». Este archivo puede ser de gran utilidad, y muchos de los desarrolladores se aseguran de que se encuentra disponible en sus sistemas de archivos raíz en todo momento.

El término «núcleo en un disco RAM» describe el sistema de archivos del núcleo (en lugar de un controlador físico), y las utilidades disponibles después del arranque de bsd.rd que se almacenan en el núcleo y que se ejecutan desde un sistema de archivos basado en RAM. bsd.rd también incluye algunas utilidades que permiten mantener e instalar el sistema.

En algunas plataformas, bsd.rd es la técnica de instalación preferida (basta con copiarlo en un sistema de archivos existente, iniciarlo, y ejecutar la instalación desde él. En la mayoría de plataformas, si tiene una versión más antigua de OpenBSD, puede obtener una versión más reciente de bsd.rd, iniciar con ésta, e instalar una nueva versión de OpenBSD sin usar ningún tipo de medio de instalación extraíble.

He aquí un ejemplo de arranque de bsd.rd en un sistema i386: 

    Using Drive: 0 Partition: 3
reading boot.....
probing pc0 com0 com1 apm mem[639k 255M a20=on]
disk fd0 hd0
>> OpenBSD/i386 BOOT 1.28
boot> boot hd0a:/bsd.rd
. . . arranque normal de instalación . . .

Tal y como se indica, pasará a la instalación del programa, pero también puede activar el intérprete de órdenes para llevar a cabo tareas de mantenimiento del sistema.

La regla general para arrancar con bsd.rd es cambiar el núcleo de arranque /bsd por el de bsd.rd por cualquier medio que use su plataforma.
