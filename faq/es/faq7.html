7.1 - ¿Cómo puedo reasignar el mapa del teclado? (específico para wscons)

Plataformas que usan el control de consola wscons(4): alpha, i386, y powerpc.

Para OpenBSD 2.9 y posteriores:

A partir de la versión 2.9, OpenBSD cambió al controlador de consola wscons(4).

Con las consolas wscons(4), la mayoría de opciones se pueden controlar usando la utilidad wsconsctl(8). Para cambiar la asignación del teclado con wsconsctl(8) siga el ejemplo, con el que se cambiará la asignación del teclado a ES. 

    # wsconsctl -w encoding=es

En el siguiente ejemplo, reasignaremos la tecla "Bloq Mayús" para que sea "Control L": 

    # wsconsctl -w map+="keysym Caps_Lock = Control_L"
Para OpenBSD 2.8 y anteriores:

Usando kcon(1), «kcon - control del teclado y reasignación para el controlador de pcvt». Antes de usarlo debe activar la reasignación del mapa de caracteres con scon.

Ejemplo: 

    # scon -o
# kcon -m es

De este modo cargará el fichero keycap para el teclado español. Tenga en cuenta que si vuelve a ejecutar scon -o, volverá a desactivar la reasignación del mapa de caracteres, lo que probablemente no querrá hacer si está usando kcon para reasignar. Recuerde esto cuando use scon de un modo automatizado, como por ejemplo en guiones del intérprete de órdenes ("shell scripts").

Estas órdenes sólo se pueden ejecutar desde un terminal virtual pcvt.

7.2 - ¿Existe algo parecido a gpm en OpenBSD?

Para las plataformas que usen el controlador de consola de wscons(4) existe un porte del dæmon del ratón para modo de consola, moused(8). El porte original de moused(8) fue hecho para OpenBSD 2.7 usando el controlador de consola pcvt(4), que más tarde se modificó para su uso con wscons. El nombre del dæmon se cambió por el de wsmoused(8), y se incluye con la instalación predeterminada de OpenBSD 2.9.

wsmoused(8) debería funcionar en las siguientes plataformas: alpha, i386, y powerpc.

7.3 - ¿Cómo refresco la consola cada vez que salga un usuario?

Para refrescar la consola debe añadir una línea al fichero /etc/gettytab. Cambie la sección actual: 

    P|Pc|Pc console:\
        :np:sp#9600:

Añada la línea ":cl=\E[H\E[2J:" al final, de modo que quede como lo siguiente: 

    P|Pc|Pc console:\
        :np:sp#9600:\
	:cl=\E[H\E[2J:

7.4 - Cómo acceder a la memoria intermedia para retroceder páginas en la consola (específico para alpha/powerpc/i386).

A partir de la versión 2.6, OpenBSD incluye la posibilidad de acceder a la memoria del retroceso de páginas. Esto le permite ver información que ya haya pasado por su pantalla. Para poder retroceder y avanzar en la memoria, use las siguientes combinaciones de teclas:

para avanzar, 
    "[shift]+[pgup]"
para retroceder, 
    "[shift]+[pgdn]"

La memoria de retroceso predeterminada, o la cantidad de páginas de memoria que puede retroceder para ver, es 8.

7.5 - ¿Por qué las teclas F9 a F12 ya no cambian de consola? (específico para i386)

Hasta la versión 2.6 de OpenBSD incluida, el porte de i386 del controlador pcvt de la consola permitía cambiar a las terminales virtuales 1, 2, 3, y 4 pulsando las teclas F9, F10, F11, y F12 respectivamente. A partir de la versión 2.7 y posteriores, esta funcionalidad se ha eliminado porque varias aplicaciones utilizan estan teclas para obtener datos de entrada.
