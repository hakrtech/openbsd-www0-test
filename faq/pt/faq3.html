3.1 - Como comprar um conjunto de CDs do OpenBSD

Comprar um conjunto de CDs é geralmente o melhor jeito de começar. Visite a página de pedidos para comprar sua cópia: página de pedidos do OpenBSD.

Há muitas boas razões para ter seu próprio conjunto de CDs do OpenBSD:

  • Vendas de CDs ajudam no progresso do desenvolvimento do OpenBSD.
  • O desenvolvimento de um sistema operacional multiplataforma requer investimentos constantes em equipamentos.
  • Sua ajuda, na forma de uma compra do conjunto de CDs, tem uma influência real no desenvolvimento futuro.
  • Os CDs contêm os binários (e o código fonte) para a maioria das plataformas populares suportadas.
  • Os CDs são inicializáveis em várias plataformas, e podem ser usados para iniciar uma máquina sem um sistema operacional pré-existente instalado.
  • Os CDs são úteis para iniciar mesmo se você escolheu instalar uma "snapshot".
  • Instalação pelo CD é rápida! Instalação pelo CD preserva recursos de banda da rede.
  • Os CDs do OpenBSD vêm com adesivos muito legais. Seu sistema não está completo sem eles. Você somente pode receber esses adesivos comprando um conjunto de CDs ou doando hardware.
  • O conjunto de CDs é completo o bastante para criar um ambiente de trabalho e desenvolvimento sem qualquer conexão de rede para isso.

Para instalar uma versão (release) do OpenBSD, você deve usar um conjunto de CDs oficial.

3.3 - O OpenBSD fornece uma imagem ISO disponível para download?

Para determinadas plataformas, sim!

A maioria das plataformas que são capazes de inicializar a partir de CD-ROM, tem as imagens ISO disponíveis para download, que podem ser usado para criar CD-ROMs inicializaveis instalar o OpenBSD. Tanto "instalação completa" (todos os arquivos necessários para uma instalação completa do OpenBSD, chamado install54.iso) e "apenas boot" (para a instalação de uma fonte de arquivos de rede, chamado cd54.iso) ISOs são fornecidos para essas plataformas.

Lembre-se: esses arquivos ISO não é o mesmo do CD oficial. Essas imagens são para plataformas únicas, e não incluem nenhum dos pacotes pré-compilados, adesivos, ou o trabalho artístico que o CD oficial inclui.

No entanto, a instalação a partir de um arquivo ISO não é um bom método de instalação para muitas pessoas. Ainda é mais rápido e simples fazer o download da mídia de inicialização e então instalar apenas as partes necessárias. No entanto, para aqueles que desejam fazer várias instalações, ou não podem colocar dez arquivos em um CD-ROM ou configurar um servidor HTTP/FTP local, as ISOs estão disponíveis.

O projeto OpenBSD não disponibiliza para download as imagens ISO usadas para criar os discos-mestre dos CDs oficiais. A razão é simples, nós queremos que você compre o conjunto de CDs para ajudar no progresso do desenvolvimento do OpenBSD. Os direitos autorais do layout oficial do CD-ROM do OpenBSD são de Theo de Raadt. Como um incentivo para as pessoas comprarem o conjunto de CDs, alguns extras são incluídos no pacote (trabalho de arte, adesivos, etc.).

Note que somente o layout do CD possui direitos autorais, o OpenBSD em si é livre. Nada impede alguém de fazer o download do OpenBSD e criar seu próprio CD.

Para aqueles que precisam de um CD inicializável para seus sistemas, imagens ISO de disco de inicialização (chamadas de cd54.iso) são disponibilizadas para várias plataformas e permitem que o resto do sistema seja instalado via HTTP/FTP. Essas imagens ISO possuem apenas alguns poucos megabytes de tamanho e contêm apenas as ferramentas de instalação, sem os pacotes de arquivos reais.

3.4 - Download via HTTP, FTP, RSYNC ou AFS

Existem numerosos sítios de espelhos internacionais que oferecem acesso FTP e HTTP à releases e snapshots do OpenBSD. Acessos RSYNC e AFS também estão disponíveis. Você deve sempre usar o sítio mais próximo de você. Antes de você iniciar o download de uma release ou uma snapshot, você pode usar o ping(8) e o traceroute(8) para determinar qual espelho está mais próximo de você e se ele tem um desempenho adequado. Naturalmente, seu CD com a release do OpenBSD está sempre mais próximo que qualquer espelho. Informações de acesso estão aqui:

Página de Download do OpenBSD.

3.5 - Verificando seu Download

No mesmo diretório em que estão os arquivos de instalação, cada espelho inclui um arquivo chamado SHA256 que contém checksums dos diferentes arquivos de instalação. Você pode confirmar que nenhum dos arquivos baixados foi corrompido em trânsito utilizando o comando sha256(1): 
  $ sha256 -c SHA256
O projeto OpenBSD não assina digitalmente suas releases. O comando acima só detecta danos acidentais, não adulteração. Se os Homens de Preto estão atrás de você, eles vão te pegar.

3.6 - Escolhendo o Hardware

Selecionar o hardware apropriado para executar o seu sistema OpenBSD é importante, já que isso pode significar a diferença entre o sucesso e o fracasso de um projeto.

Se você está planejando a compra de um PC novo, seja peça por peça ou completamente pré-montado, você primeiro precisa ter certeza de que vai comprar partes confiáveis. No mundo do PC, isso não é fácil. Partes ruins ou, de outra forma, não-confiáveis podem fazer o OpenBSD funcionar precariamente e travar frequentemente. O melhor conselho que podemos lhe dar é ser cuidadoso e comprar marcas e partes que foram revisadas por uma autoridade que você confia. Algumas vezes, uma máquina de alto custo é uma máquina de melhor qualidade. Outras vezes, esta é somente mais cara.

Existem algumas coisas que ajudam você a obter o desempenho máximo do seu sistema:

  • Deixe a aplicação escolher o hardware: Normalmente é melhor fazer um ajuste no hardware que você estava planejando usar do que comprometer o projeto da aplicação porque você possui algo que "quer realmente usar".
  • Identifique os seus gargalos: Não gaste dinheiro com processadores poderosos se a sua aplicação é restrita à operações de E/S de disco. Não gaste dinheiro com discos rápidos se o seu sistema é limitado pela velocidade da rede. Não gaste dinheiro com mais nada se o seu gargalo é uma linha DSL de 128kbps.
  • Mantenha isso simples: Hardware simples normalmente possui problemas simples. Hardware complexo, pensado para nunca quebrar, pode render-lhe um longo tempo de conserto quando parar de funcionar.
  • Utilize hardware que você entende, ou aprenda o novo hardware antes de você implementá-lo como um sistema de produção: Independente dos méritos técnicos do hardware, usar um tipo particular de hardware antes mesmo de você se familiarizar com ele, e entender como ele funciona e como ele falha, é tolice.
  • Utilize múltiplos discos: Em vez de comprar um disco grande, compre vários discos pequenos. Embora isso custe mais, distribuir a carga em múltiplos eixos ("spindles") diminui a quantidade de tempo necessário para acessar os dados nos discos. E, com mais eixos, você consegue um acesso mais rápido e confiável aos dados com RAID.
  • Separe os grandes blocos de armazenamento: Muitas pessoas cometem o erro de projetar um sistema que guarda grandes quantidades de dados em um Grande Bloco de Armazenamento. Normalmente esse é um projeto ruim. Você vai perceber que é melhor separar seu sistema de armazenamento em blocos gerenciáveis. Isso possui várias vantagens, duas das grandes vantagens é que você pode adicionar facilmente mais espaço de armazenamento quando você precisar (você não vai ter que estimar corretamente os requerimentos; ninguém faz isso), e que você pode comprar uma pequena quantidade de espaço para armazenamento agora, e adicionar mais quando o preço cair e a capacidade de armazenamento aumentar.
  • Evite adaptadores de rede baratos: O OpenBSD suporta vários adaptadores de rede baratos. Esses adaptadores funcionam bem em sistemas domésticos e ambientes de negócios ou pesquisas com transferência de dados baixa ou moderada. Mas, se você precisa de alta transferência de dados e pouco impacto no seu servidor, é melhor que você compre um adaptador de rede de qualidade. Infelizmente, alguns adaptadores de marcas renomadas não são muito melhores do que os adaptadores baratos, e alguns bons adaptadores não possuem documentação completa disponível para o desenvolvimento de bons drivers. Adaptadores Gigabit frequentemente funcionam melhor que adaptadores de 10Mbps/100Mbps, mesmo quando usados em redes de baixa velocidade, devido ao sistema de buffer superior.

3.7 - O que é apropriado como um "primeiro sistema" para aprender o OpenBSD?

Embora o OpenBSD funcione em computadores pequenos, velhos e de baixa potência, diferentemente de qualquer outro SO moderno, se você está dando seus primeiros passos com o OpenBSD, usar uma máquina muito fraca pode ser frustrante. O seguinte guia é SOMENTE um guia, o OpenBSD funciona muito bem em um equipamento mais modesto que o listado aqui, mas ele pode ser mais frustrante que o necessário para um usuário de primeira viagem.
  • Plataforma: Utilize uma plataforma que você está familiarizado. Quando você está aprendendo um novo sistema operacional, não é um bom momento para aprender uma nova plataforma. Assumimos que você esteja usando a plataforma i386 ou a plataforma amd64, já que elas são, provavelmente, as plataformas mais familiares para a maioria das.

    Uma vez estando familiarizado com o OpenBSD, acredita-se que essas plataformas te proporcionam um excelente ponto de partida para aprender sobre outras plataformas.

  • Processador: Um processador Pentium de 100MHz ou superior. Sim, o OpenBSD funciona em um 80486 de 25MHz, mas você não vai querer ter todo o trabalho necessário para aprender o sistema em uma máquina lenta. A primeira irritação que você vai ter com um processador mais lento que esse é o tempo que ele leva para iniciar um sessão SSH na máquina. Se você deseja executar o X, você provavelmente vai ter que usar um processador de 200MHz, no mínimo. Na verdade, o X funciona muito bem em máquinas lentas após ter sido carregado, mas ele leva um certo tempo para carregar e iniciar.
  • 32M de RAM ou mais: Se você deseja executar o X, 128M pode ser um ponto de partida melhor.
  • Disco rígido: Um disco rígido de 1G pode ser usado para um sistema simples, como um firewall, um servidor DNS, ou algo similar. Se você deseja recompilar o sistema a partir do código fonte, você provavelmente vai precisar de um disco de 4G, e se você deseja compilar o X também, você vai precisar de 6G ou mais. A interface IDE é recomendada para um começo. Se você tem um disco muito grande, não sinta-se obrigado a alocar todo o disco -- não há nada de errado em deixar 72G de um disco rígido de 80G desalocados se tudo que você precisa são 8G.
  • Adaptador de rede: Utilize um adaptador PCI. Se você está planejando colocar múltiplos adaptadores de rede na máquina, escreva o endereço MAC no corpo da placa antes de colocar a tampa. Mesmo que a vontade de usar um adaptador ISA antigo seja forte, resista. Você provavelmente não se lembra sobre como configurá-lo apropriadamente.
  • Inicialização múltipla: Na sua primeira instalação do OpenBSD, não tente fazer inicialização múltipla com outro SO. Inicialização múltipla é um processo difícil de ser feito corretamente, e você deve entender bem todos os SOs envolvidos antes de tentar isso, o que certamente não é o caso na sua primeira instalação. É muito possível que você apague acidentalmente todos os dados no sistema. Utilize um computador dedicado ou, no mínimo, um disco dedicado somente para isso.
  • Laptops: Enquanto muitos laptops funcionam bem com o OpenBSD, algumas vezes eles não são sistemas fáceis de fazer funcionar bem; então, um laptop pode não ser a melhor escolha para a sua primeira instalação do OpenBSD. Entretanto, uma vez que você estiver confortável com o OpenBSD, um laptop pode ser uma ferramenta muito útil.
  • Hardware novo: Hardware de marca nova, ou recém-lançado, algumas vezes não é suportado pelo OpenBSD; então, para um primeiro sistema OpenBSD, uma máquina levemente velha é recomendada.
Evidentemente, "quanto mais, melhor". Alguns aplicativos populares podem aparentemente usar quase todo o processador e memória que você possui no seu sistema.
