1.1 - OpenBSDって何?

OpenBSD プロジェクトは、フリーで入手可能なマルチプラットホームの 4.4BSD-ベースの UNIX系オペレーティングシステムだ。 われわれが強調するのは 移植性、 標準化、正しさ、セキュリティだ。 OpenBSDは SVR4 (Solaris)、 FreeBSD、Linux、BSDI、SunOS、HPUX の各種バイナリ・エミュレーションをサポートしている。


1.2 - OpenBSD はどんなシステム上で動くの?

OpenBSD 3.1 は以下のプラットホームで動く:

  • i386 - CD ブート可能
  • sparc - CD ブート可能
  • sparc64 - CD ブート可能
  • hp300 - CD ブート可能
  • amiga
  • mac68k
  • macppc - CD ブート可能
  • mvme68k
  • alpha - CD ブート可能
  • vax

ブート可能 というのは、OpenBSD が CDから直接起動できるということ。 CD セットは、いくつかのハードウェアプラットホーム上でブートできる。 OpenBSD の CD 入手については、この FAQ の 3 章を参照のこと。

過去の OpenBSD リリースでは、以下のプラットホーム用のポートもあった:

  • sun3 - このポートは、2.9 リリースから はずされた
  • arc - このポートは、2.3 リリースから はずされた
  • mvme88k
  • pmax

もし、マルチプロセッサのサポートに興味があるなら、より詳しいことは、 FAQ 8, マルチプロセッサを参照して欲しい。

1.3 - OpenBSD って本当にフリーなの?

OpenBSD はすべてフリーだ。バイナリはフリー。ソースもフリー。 OpenBSD のすべての部分は、フリーの配布を認める、 まっとうなコピーライト条件を持っている。 これは、個人的にだろうと商業的な目的だろうと、OpenBSD のほとんどの部分を再利用できるということだ、OpenBSD は、 もともとの BSD ライセンスで含意されているもの以上の制限は一切つけていない。 もっときびしいライセンス条件のもとで書かれたソフトは、 OpenBSD の正規ディストリビューションには含められない。 これは、OpenBSD のフリーな利用を守るためのものだ。 たとえば、OpenBSD は個人利用でも、アカデミックな利用でも、 政府機関でも非営利組織でも、商業組織でも自由に使うことができる。

他のよくあるライセンスについてもっと知りたければ、以下を参照してほしい: http://www.openbsd.org/policy.html

OpenBSD のメインテナーたちは、プロジェクトをもっぱら自分の持ち出しでサポートしている。 これはプロジェクトのためのプログラミングにかけた時間、各種の移植をサポートするための機器、 OpenBSD をあなたに届けるために使われたネットワーク資源、そしてユーザからの質問に答えたり、 バグ報告を調べたりするのに使う時間なんかも含む。 OpenBSD 開発者たち個人はお金持ちではないから、 ほんのちょっとでも時間や機器やリソースで貢献してもらえると、大きく効いてくる。


1.4 - なぜあたしが OpenBSD なんか使わなきゃいけない?

新人ユーザは、OpenBSD がほかのフリーなUnix系OSに比べて優れているかどうかをしょっちゅう知りたがる。 この質問は、おおむね回答不能だし、無数の (そして無益な) 宗教論争のネタになってきたものだ。 いかなる状況においても、こんな質問を OpenBSD のメーリングリストに投げたりしないこと。

以下に、なぜわれわれが OpenBSD を有用な OS だと思うか、いくつか理由をあげておいた。OpenBSD があなたにとって適切かどうかは、あなた自身が判断するしかない。

  • OpenBSD はいろんなハードウェアプラットホーム上で動く。
  • OpenBSD は、10人による1.5年にわたる徹底的なソースコードセキュリティ監査のおかげで、 多くのセキュリティ専門家によっていちばんセキュリティの高い UNIX-系 OS と判断されている。
  • OpenBSD は、ソースコードの形で提供されている、 完全な UNIX-系 OS で、無料だ。
  • OpenBSD は最先端のセキュリティ技術を統合してあるので、 ファイアーウォールや プライベートネットワークサービス を分散環境で構築するのに適している。
  • OpenBSD は、いろんな分野での強力な継続的開発の成果をもらっていて、 できたてのテクノロジーを国際的なプログラマやエンドユーザたちとともに 活用する機会を提供する。
  • OpenBSD は、ふつうの人が製品の開発やテストに 参加する機会を提供する。


1.5 - OpenBSD をサポートするにはどうしたらいい?

われわれは、OpenBSD プロジェクトに貢献してきた人々や組織に大いに借りがある。 かれらの名前を以下にあげておこう:

http://www.openbsd.org/donations.html

OpenBSD は、ユーザコミュニティからいつも何通りかのサポートを必要としている。 もし OpenBSD が使えると思ったら、ぜひともプロジェクトに貢献していただきたい。 もし以下の提案がどれもあなたにとって適切でないなら、 メールで別の方法を提案していただければ幸甚。送り先は： donations@openbsd.org

  • OpenBSD CD セットを買おう。いまの OpenBSD のフルリリースが入っているし、 多くのプラットホームでは起動 CD として使える。それに、OpenBSD を支える 収入を生み出してくれるし、ディストリビューションをインターネット経由で 配布するための、ネットワーク資源を節約してくれる。この、廉価な 3 枚組の CD セットには、すべてのソースコードも含まれている。そうそう、お友だちの ためにも CD を買っておいてあげよう！
  • お金の寄付。プロジェクトは、機器やネットワーク接続、CD 発行のために費用など、いつも現金が必要だ。 CD のプレスは、OpenBSD 開発者にとって前払いの持ち出し投資を必要とするもので、 しかもそれが帰ってくるという保証はない。 寄付の方法について、詳しくはdonations@openbsd.org までメールを送ってほしい。ほんのちょっとのお金でも、 ずいぶんと差が出るもんだ。
  • 機器やパーツを寄付。プロジェクトはたえず、一般的なハードや、 特定のハードがほしいと思っているのだ。 IDE や SCSI ディスク、各種 RAM はいつでも大歓迎。その他、 各種コンピュータシステムやマザーボードの場合、 いま何が必要とされているかについてまず問い合わせてほしい。 発送のための手続きなんかについては、 donations@openbsd.orgまでご連絡を。
  • 時間や技能で貢献。OS を書くのが 好きなプログラマはもちろん大歓迎だけれど、 役にたつ方法はそれ以外にも何十とある。 メーリングリストを読んで、新人の質問に答えてあげよう。
  • FAQ のネタを送って、ドキュメンテーションの充実に貢献しよう (送り先は faq@openbsd.org)。 地域のユーザグループを組織して、友だちを OpenBSD にハメてしまおう。 雇い主に、仕事で OpenBSD を使うよう説得しよう。学生なら、教授にコンピュータ科学や エンジニアリングの講義で OpenBSD を使いませんか、と話してみよう。 それと一つ言っておくけれど、OpenBSD プロジェクトを助けるためにやってはいけない一番だいじなこと： OS フレーム合戦に加わって時間を無駄にするようなことはしないでほしい。 プロジェクトの新しいユーザ探しにも役にたたないし、 開発者同士の関係にも非常に有害な影響を与える場合だってある。


1.6 - OpenBSD はだれがメンテナンスしているの?

OpenBSD は、世界各国に散らばった開発チームが維持している。 プロジェクトを仕切っているのは、カナダにいる Theo de Raadt だ。
