1.1 - OpenBSD って何でしょう ?

OpenBSD プロジェクトは、フリーで入手可能なマルチプラットホームの 4.4BSD ベースの UNIX 系オペレーティングシステムです。 私たちが強調するのは 移植性、標準化、正しさ、 セキュリティです。 OpenBSDは SVR4 (Solaris)、FreeBSD、Linux、BSDi、SunOS、HPUX の各種バイナリ・エミュレーションをサポートしています。

この FAQ は、特に最も最近の OpenBSD のリリースであるバージョン 3.6 だけを網羅しています。

1.2 - OpenBSD はどのようなシステム上で動くのでしょう ?

OpenBSD 3.6 は以下のプラットホーム上で動きます。

  • alpha - FTP のみ
  • amd64 - CD ブート可能
  • cats - FTP のみ
  • hp300 - FTP のみ
  • hppa - FTP のみ
  • i386 - CD ブート可能
  • mac68k - FTP のみ
  • macppc - CD ブート可能
  • mvme68k - FTP のみ
  • mvme88k - FTP のみ
  • sparc - CD ブート可能
  • sparc64 - CD ブート可能
  • vax

ブート可能というのは、OpenBSD が CD から直接起動できるということです。 CD セットは、いくつかのハードウェアプラットホーム上でブートできます。 OpenBSD の CD 入手については、この FAQ の 3 章を 御参照ください。

過去の OpenBSD リリースでは、以下のプラットホーム用のポートもありました。

  • amiga - 3.2 リリース後に削除
  • sun3 - 2.9 リリース後に削除
  • arc - 2.3 リリース後に削除
  • pmax - 2.7 リリース後に削除

1.3 - OpenBSD って本当にフリーなのでしょうか ?

OpenBSD はすべてフリーです。バイナリはフリーですし、ソースも フリーです。OpenBSD のすべての部分は、フリーの配布を認める、 まっとうなコピーライト条件を持っています。これは、個人的に であっても商業的な目的であっても、OpenBSD のほとんどの部分を 再利用できるということです。OpenBSD は、もともとの BSD ライセンスで含意されているもの以上の制限を一切加えていません。 もっと厳しいライセンス条件のもとで書かれたソフトは、 OpenBSD の正規ディストリビューションには含められません。 これは、OpenBSD のフリーな利用を守るためのものです。 たとえば、OpenBSD は個人利用でも、アカデミックな利用でも、 政府機関でも非営利組織でも、商業組織でも自由に使うことができます。 OpenBSD やその構成部品はまた、自由に 商用製品に利用できます。

他の一般的なライセンスについてのより詳しいことは、 OpenBSD の著作権のポリシーを参照してください。

OpenBSD のメンテナたちは、プロジェクトをもっぱら自分の持ち出しで サポートしています。これはプロジェクトのためのプログラミングにかけた時間、 各種の移植をサポートするための機器、OpenBSD をあなたに届けるために使われた ネットワーク資源、そしてユーザからの質問に答えたり、バグ報告を調べたり するのに使う時間なんかも含んでいます。しかし、OpenBSD 開発者たち個人は お金持ちではありませんから、ほんの少しでも時間や機器やリソースで貢献して いただければ、大きな違いとなって表われてきます。

1.4 - なぜ私は OpenBSD を使わなければならないのでしょう ?

新しいユーザは、OpenBSD が他のフリーな UNIX 系 OS に比べて優れている のかどうか、頻繁に知りたがるものです。しかし、この質問は、おおむね 回答不能ですし、無数の (そして無益な) 宗教論争のネタになってきたものです。 いかなる状況においても、このような質問は OpenBSD のメーリングリストに 投げたりしないようにしてください。

以下に、なぜ私たちが OpenBSD を有用な OS だと思うのか、 いくつか理由をあげてみました。しかし、OpenBSD があなたに とって適切かどうかは、あなた自身が判断するしかありません。

  • OpenBSD はいろいろな ハードウェアプラットホーム上で動きます。
  • OpenBSD は、決して終ることのない徹底的な ソースコードのセキュリティ監査のおかげで、多くのセキュリティ専門家 によって、最も高いセキュリティ を誇る UNIX 系 OS と判断されています。
  • OpenBSD は、ソースコードの形で提供されている、 完全な UNIX 系 OS で、無料です。
  • OpenBSD には、最先端のセキュリティ技術が統合されていますので、 ファイアウォールやプライベート ネットワークサービスを分散環境で構築するのに適しています。
  • OpenBSD は、いろいろな分野での強力な継続的開発の成果の提供を受けて、 できたての技術を国際的なプログラマやエンドユーザのコミュニティとともに 活用する機会を提供しています。

1.5 - OpenBSD をサポートするにはどうすれば良いでしょう ?

私たちは、OpenBSD プロジェクトに貢献してきた人々や組織に大いなる 恩があります。彼らのお名前を 寄付のページに掲載しています。

OpenBSD は、ユーザコミュニティからの、常に何通りかのサポートを必要としています。 もし OpenBSD が使えると思ったなら、ぜひともプロジェクトに貢献してください。 もし以下の提案がどれもあなたにとって適切でないのでしたら、 メールで別の方法を提案していただければ幸いです。送り先は donations@openbsd.org です。

  • OpenBSD CD セットの購入。今の OpenBSD のフルリリースが入っていますし、多くのプラットホームでは起動 CD として使えます。 それに、OpenBSD を支える収入を生み出してくれますし、ディストリビューションを インターネット経由で配布するための、ネットワーク資源を節約してくれます。 この、廉価な 3 枚組の CD セットには、すべてのソースコードも含まれています。 そうそう、お友だちのためにも CD を買っておいてあげましょう !
  • お金の寄付。プロジェクトは、 機器やネットワーク接続、CD を発行するための費用など、いつも現金を 必要としているのです。CD のプレスは、OpenBSD 開発者にとって前払いの 持ち出し投資を必要とするもので、しかもそれが帰ってくる保証はありません。 寄付の方法についての詳しいことは、 donations@openbsd.org までメールをお送りください。 ほんの少しのお金でも、大きな差が出るものなのです。
  • 機器やパーツの寄付。プロジェクトは絶えず、 一般的なハードや、特定のハードを欲しいと思っています。IDE や SCSI ディスク、 各種 RAM はいつでも大歓迎しています。その他、各種コンピュータシステムや マザーボードの場合、いま何が必要とされているかについては、 まずお問い合わせください。発送のための手続き等については、 donations@openbsd.org まで御連絡ください。
  • 時間や技能で貢献。オペレーティングシステムを書くのが 大好きなプログラマはもちろん大歓迎ですが、 役にたつ方法はそれ以外にも何十とあります。 メーリングリストを読んで、 新しいユーザの質問に答えてあげてください。
  • FAQ のネタを送って、ドキュメンテーションの充実に貢献してください (送り先は faq@openbsd.org です)。 地域のユーザグループを組織して、友だちを OpenBSD の虜にしてしまいましょう。雇用者に、仕事で OpenBSD を使うよう 説得しましょう。学生なら、コンピュータ科学やエンジニアリングの講義で OpenBSD を使いませんか、と教授に話してみましょう。 また、OpenBSD プロジェクトを助けようとして、オペレーティングシステムの フレーム合戦に加わって時間を無駄にしたりしないことが、最も重要なことの ひとつであると意識することも、価値のあることです。 プロジェクトの新しいユーザ探しにも役に立ちませんし、 開発者同士の関係にも非常に有害な影響を与える場合さえあるのです。

1.6 - OpenBSD は誰がメンテナンスしているのでしょう ?

OpenBSD は、世界各国に分散した開発チーム によって維持されています。また、プロジェクトをとりまとめているのは、 カナダ在住の Theo de Raadt です。

1.7 - OpenBSD の次のリリースはいつでしょうか ?

OpenBSD チームでは、5 月と 11 月のリリースを目標に、新しい リリースを 6 ヶ月ごとに行っています。開発サイクルに関するより詳しい情報は、 here を参照してください。

1.8 - OpenBSD には何が含まれているのでしょう ?

OpenBSD は、以下のものを含む、多数のサードパーティソフトウェア製品とともに 出荷されています。
  • X Window 環境 XFree86 4.4.0。 これは新しく採用されたライセンスの制限がない、ローカルパッチが適用された X Window 環境です。 i386 版には、より多くのグラフィックチップセットに対応した v3.3 の X サーバも 含まれています。x*.tgz という インストール用ファイルセット を指定するとインストールされます。
  • GNU C コンパイラ GCC 2.95.3 および 3.3.2。 OpenBSD チームは、スタック保護技術 Propolice をこれに追加し、デフォルトで有効化しています。そして、OpenBSD のユーザランドすべてと OpenBSD 上でコンパイルされたアプリケーションは、デフォルトでこれが使用されています。 インストール用ファイルセット comp36.tgz の一部としてインストールされます。
  • Perl 5.8.5 (+ パッチ) および OpenBSD チームによる改良。
  • Apache 1.3.29 web サーバ。 OpenBSD チームは、デフォルトでの chroot、特権分離、およびその他のセキュリティ関連の改良を 追加しています。また、mod_ssl 2.8.16 と DSO のサポートも組み込まれています。
  • OpenSSL 0.9.7d (+ パッチ) および OpenBSD チームによる改良。
  • Groff 1.15 テキストプロセッサ。
  • Sendmail 8.13.0 メールサーバ。セキュリティパッチ parse8.359.2.8 適用済。
  • BIND 9.2.3 DNS サーバ。 OpenBSD では、chroot による運用や他のセキュリティ関連の問題等に対する 多数の改良を実装しています。
  • テキストベース web ブラウザ Lynx 2.8.5rel.2。 また、OpenBSD チームによるパッチと HTTPS サポートが追加されています。
  • Sudo v1.6.7p5。これは、ユーザが 個々のコマンドを root として実行することを許可できるようにするためのものです。
  • Ncurses 5.2。
  • KAME IPv6。
  • パッチが適用された Heimdal 0.6rc1
  • Arla 0.35.7
  • OpenSSH 3.9
  • gdb 6.1
ご覧のように OpenBSD チームでは、(ほとんどの場合) セキュリティやコードの品質の改良のため、 サードパーティの製品にパッチを当てることがよくあります。ユーザがそれによる 動作の違いを感じない場合もあるでしょうし、動作の違いが問題となる場合もあるでしょう。 そのソフトウェアの違うバージョンのを何も考えずに追加する前に、そういう改良が 含まれているということを思い出してください。新しいバージョンのものを追加すること はできますが、セキュリティの低いシステムになってしまうかも知れません。

もちろん OpenBSD では、アプリケーションを packages や ports システムを使用して追加することができます。

1.9 - OpenBSD 3.6 では何が新しくなったのでしょうか ?

OpenBSD 3.5 から OpenBSD 3.6 へのバージョンアップで加えられた変更の全リストは こちらにありますが、以前のバージョンを熟知し、 OpenBSD 3.6 にアップグレード、またはこれをインストールしようとする人たちに対し、 OpenBSD チームが予想するいくつかの変更は、以下のようにいくつかの特別な注意が 必要となっています。
  • i386 および amd64 が、対称型マルチプロセッサ (SMP) に対応しました! SMP 動作用のカーネル (bsd.mp) が、別に用意されています。
  • ネットワークタイムプロトコルデーモン (ntpd) のベースシステムへの統合。 いつものように、このデーモンは確実に動作し、 セキュアでフリーなものとなるよう設計されたものです。 また、このデーモンは簡単に使うことができ、NTP サービスを必要とするユーザの大部分が満足できるはずです。 現在、ports にある ntpd を使っているユーザも、 ぜひこの新しい ntpd を試してみてください。 OpenBSD をアップグレードすると、同じシステムに ntpd というプログラムが 2 個存在する状態になることがありますので、どちらのプログラムを実行しているのか 間違えないように注意してください。
  • 新しいユーザとグループ。 OpenBSD には、権限分離を実現するための新しいユーザとグループがいくつか追加されています。 アップグレードする場合には、upgrade-minifaq に ある手順にしたがって、/etc ディレクトリも注意深くアップグレードする必要があります。
  • 新しい CD ブートのシステム。 従来の OpenBSD は、CD から「フロッピーエミュレーション」を使ってブートしていました。 この機能では、インストール用カーネルの大きさが 2.88M に制限されており、 2.88M エミュレーション CDROM はすべてのコンピュータで使えるわけでもありません。 そのため、古い CDROM でブートできない、新しいコンピュータの多くで ブート可能な、「非エミュレーション」ブートが導入されました。

1.10 - OpenBSD をデスクトップシステムとして使うことはできますか?

これは、「デスクトップ」という言葉の意味をきちんと説明せずに尋ねられることが 多い質問です。この質問は「何が必要なのか」に依るため、答えることができるのは 質問した人だけです。

OpenBSD は「サーバ用途の」オペレーティングシステムとして高い評価を得ているのですが、 デスクトップ用途に使うことも可能です。多くのデスクトップアプリケーションを、 ports と packages を使ってインストールすることができます。 そして、オペレーティングシステムを選ぶ時に考えたこと、つまり「それは自分のやりたい内容を、 自分の望むようにやってくれるだろうか」という質問を考えてください。 この質問には、あなたが自分自身で回答しなければなりません。

1.11 - どうしてあのソフトウェアが入っている/いないのでしょう?

そのソフトウェアが OpenBSD に入っている、あるいは入っていないのは なぜか、という質問は多く寄せられます。これは、開発者の希望および、 プロジェクトの目標と合致するかという 2 つの理由からそうなっている、というのが回答です。 単に性能が良いというだけで、そのソフトウェアが OpenBSD に含まれることはありません。ソフトウェアはフリーで利用可能で、 わたしたちの考え方にもとづいた改変・再配布が可能でなければなりません。 また、そのソフトウェアは安定していてセキュアでなければなりません。 バージョン番号が大きいからといって、以前より改良されているとは限らないのです。

よく最も大きな問題となるのは、ライセンスです。わたしたちは OpenBSD を 世界中の誰もが、どんな目的にも使えるようにしたいと考えています。

以下は、サードパーティ製のソフトウェアに関して良く尋ねられる質問です。

  • どうして Sendmail が含まれているのでしょう? あれはセキュアでないことで有名ではないですか? Sendmail は、 今までセキュリティに関してあまり良くない経歴を持っていましたが、 Sendmail の作者や保守担当者は、コードをよりセキュアにするための作業に 非常に高い理解を示してくれています (これは滅多にないことです)。 最近の Sendmail のセキュリティに関する経歴は、他の「よりセキュアである」と されているソフトウェアと、ほとんど変わりません。
  • どうして Postfix が含まれていないのでしょう? ライセンスがフリーではありませんので、検討されることはありません。
  • どうして qmail や djbdns が含まれていないのですか? ライセンスが原因です。改変物の再配布ができないため、検討されることはありません。
  • どうして Apache が含まれているのですか? ほとんど人には必要ないじゃないですか! 開発者が含めることを望んでいるからです。
  • どうして新しいバージョンの Apache が含まれていないのですか? 新しいバージョンのライセンスは、受け入れることができないものだからです。
  • どうして XFree86 v4.4 が含まれていないのですか? v4.4 で採用された新しいライセンスは、受け入れることができないものだからです。 したがって、他の多くの主要なオープンソースソフトウェアプロジェクトと同様、 OpenBSD はほぼ v4.4 と同じ内容の、フリーだった最後のバージョンを含んでいます。
これらの話題の多くは、メーリングリストで 嫌になるほど議論されたものです。情報が必要ならば、 これらのアーカイブをご覧ください。

もちろん、このようなパッケージを使いたいと望んでおり、ライセンスに違反しない 使い方をしている限り、誰もあなたがそれを使うのを妨げるものはありません (わたしたちがそうしてしまうと、真にフリーではなくなってしまうのです)。 ただし、あなたが必要としているものは変わるかも知れません。 つまりあなたは、フリーでないソフトウェアを組み込むことによって、 販売や再配布、利益を得ることを禁じるような「キラーアプリケーション」を 開発してしまうかも知れないわけです。そういうのは止めた方がいいでしょう。
