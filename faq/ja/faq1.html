1.1 - OpenBSD って何でしょう ?

OpenBSD プロジェクトは、フリーで入手可能なマルチプラットホームの 4.4BSD ベースの UNIX 系オペレーティングシステムです。 私たちが強調するのは 移植性、標準化、正しさ、 セキュリティです。 OpenBSDは SVR4 (Solaris)、FreeBSD、Linux、BSDi、SunOS、HPUX の各種バイナリ・エミュレーションをサポートしています。

1.2 - OpenBSD はどのようなシステム上で動くのでしょう ?

OpenBSD 3.3 は以下のプラットホーム上で動きます。

  • i386 - CD ブート可能
  • sparc - CD ブート可能
  • sparc64 - CD ブート可能
  • hp300 - FTP のみ
  • mac68k - FTP のみ
  • macppc - CD ブート可能
  • mvme68k - FTP のみ
  • alpha - FTP のみ
  • vax
  • hppa - FTP のみ * 3.3 から新登場 *

ブート可能というのは、OpenBSD が CD から直接起動できるということです。 CD セットは、いくつかのハードウェアプラットホーム上でブートできます。 OpenBSD の CD 入手については、この FAQ の 3 章を 御参照ください。

過去の OpenBSD リリースでは、以下のプラットホーム用のポートもありました。

  • amiga - 3.2 リリース後に削除
  • sun3 - 2.9 リリース後に削除
  • arc - 2.3 リリース後に削除
  • mvme88k
  • pmax

OpenBSD は、現状ではひとつ以上のプロセッサをサポートしていません。 より詳くは、FAQ 8, マルチプロセッサを参照してください。

1.3 - OpenBSD って本当にフリーなのでしょうか ?

OpenBSD はすべてフリーです。バイナリはフリーですし、ソースも フリーです。OpenBSD のすべての部分は、フリーの配布を認める、 まっとうなコピーライト条件を持っています。これは、個人的に であっても商業的な目的であっても、OpenBSD のほとんどの部分を 再利用できるということです。OpenBSD は、もともとの BSD ライセンスで含意されているもの以上の制限を一切加えていません。 もっと厳しいライセンス条件のもとで書かれたソフトは、 OpenBSD の正規ディストリビューションには含められません。 これは、OpenBSD のフリーな利用を守るためのものです。 たとえば、OpenBSD は個人利用でも、アカデミックな利用でも、 政府機関でも非営利組織でも、商業組織でも自由に使うことができます。

他の一般的なライセンスについてのより詳しいことは、以下を参照してください。 http://www.openbsd.org/policy.html

OpenBSD のメンテナたちは、プロジェクトをもっぱら自分の持ち出しで サポートしています。これはプロジェクトのためのプログラミングにかけた時間、 各種の移植をサポートするための機器、OpenBSD をあなたに届けるために使われた ネットワーク資源、そしてユーザからの質問に答えたり、バグ報告を調べたり するのに使う時間なんかも含んでいます。しかし、OpenBSD 開発者たち個人は お金持ちではありませんから、ほんの少しでも時間や機器やリソースで貢献して いただければ、大きな違いとなって表われてきます。

1.4 - なぜ私は OpenBSD を使わなければならないのでしょう ?

新しいユーザは、OpenBSD が他のフリーな UNIX 系 OS に比べて優れている のかどうか、頻繁に知りたがるものです。しかし、この質問は、おおむね 回答不能ですし、無数の (そして無益な) 宗教論争のネタになってきたものです。 いかなる状況においても、このような質問は OpenBSD のメーリングリストに 投げたりしないようにしてください。

以下に、なぜ私たちが OpenBSD を有用な OS だと思うのか、 いくつか理由をあげてみました。しかし、OpenBSD があなたに とって適切かどうかは、あなた自身が判断するしかありません。

  • OpenBSD はいろいろな ハードウェアプラットホーム上で動きます。
  • OpenBSD は、10 人のメンバによる 1.5 年にわたる徹底的な ソースコードのセキュリティ監査のおかげで、多くのセキュリティ専門家 によって、最も高いセキュリティ を誇る UNIX 系 OS と判断されています。
  • OpenBSD は、ソースコードの形で提供されている、 完全な UNIX 系 OS で、無料です。
  • OpenBSD には、最先端のセキュリティ技術が統合されていますので、 ファイアウォールやプライベート ネットワークサービスを分散環境で構築するのに適しています。
  • OpenBSD は、いろいろな分野での強力な継続的開発の成果の提供を受けて、 できたての技術を国際的なプログラマやエンドユーザのコミュニティとともに 活用する機会を提供しています。
  • OpenBSD は、ふつうの人々が製品の開発やテストに 参加する機会を提供しています。

1.5 - OpenBSD をサポートするにはどうすれば良いでしょう ?

私たちは、OpenBSD プロジェクトに貢献してきた人々や組織に大いなる 恩があります。彼らのお名前を 寄付のページに掲載しています。

OpenBSD は、ユーザコミュニティからの、常に何通りかのサポートを必要としています。 もし OpenBSD が使えると思ったなら、ぜひともプロジェクトに貢献してください。 もし以下の提案がどれもあなたにとって適切でないのでしたら、 メールで別の方法を提案していただければ幸いです。送り先は donations@openbsd.org です。

  • OpenBSD CD セットの購入。今の OpenBSD のフルリリースが入っていますし、多くのプラットホームでは起動 CD として使えます。 それに、OpenBSD を支える収入を生み出してくれますし、ディストリビューションを インターネット経由で配布するための、ネットワーク資源を節約してくれます。 この、廉価な 3 枚組の CD セットには、すべてのソースコードも含まれています。 そうそう、お友だちのためにも CD を買っておいてあげましょう !
  • お金の寄付。プロジェクトは、 機器やネットワーク接続、CD を発行するための費用など、いつも現金を 必要としているのです。CD のプレスは、OpenBSD 開発者にとって前払いの 持ち出し投資を必要とするもので、しかもそれが帰ってくる保証はありません。 寄付の方法についての詳しいことは、 donations@openbsd.org までメールをお送りください。 ほんの少しのお金でも、大きな差が出るものなのです。
  • 機器やパーツの寄付。プロジェクトは絶えず、 一般的なハードや、特定のハードを欲しいと思っています。IDE や SCSI ディスク、 各種 RAM はいつでも大歓迎しています。その他、各種コンピュータシステムや マザーボードの場合、いま何が必要とされているかについては、 まずお問い合わせください。発送のための手続き等については、 donations@openbsd.org まで御連絡ください。
  • 時間や技能で貢献。オペレーティングシステムを書くのが 大好きなプログラマはもちろん大歓迎ですが、 役にたつ方法はそれ以外にも何十とあります。 メーリングリストを読んで、 新しいユーザの質問に答えてあげてください。
  • FAQ のネタを送って、ドキュメンテーションの充実に貢献してください (送り先は faq@openbsd.org です)。 地域のユーザグループを組織して、友だちを OpenBSD の虜にしてしまいましょう。雇用者に、仕事で OpenBSD を使うよう 説得しましょう。学生なら、コンピュータ科学やエンジニアリングの講義で OpenBSD を使いませんか、と教授に話してみましょう。 また、OpenBSD プロジェクトを助けようとして、オペレーティングシステムの フレーム合戦に加わって時間を無駄にしたりしないことが、最も重要なことの ひとつであると意識することも、価値のあることです。 プロジェクトの新しいユーザ探しにも役に立ちませんし、 開発者同士の関係にも非常に有害な影響を与える場合さえあるのです。

1.6 - OpenBSD は誰がメンテナンスしているのでしょう ?

OpenBSD は、世界各国に分散した開発チーム によって維持されています。また、プロジェクトをとりまとめているのは、 カナダ在住の Theo de Raadt です。

1.7 - OpenBSD の次のリリースはいつでしょうか ?

OpenBSD チームでは、5 月 1 日と 11 月 1 日のリリースを目標に、新しい リリースを 6 ヶ月ごとに行っています。開発サイクルに関するより詳しい情報は、 here を参照してください。

1.8 - OpenBSD には何が含まれているのでしょう ?

OpenBSD は、以下のものを含む、多数のサードパーティソフトウェア製品とともに 出荷されています。
  • X Window 環境 XFree86 4.2.1。 i386 については、v3.3 の X サーバも、さらなるチップセットのサポートのために 含まれています。インストール用ファイルセット x*.tgz がインストールされています。
  • GNU C コンパイラ GCC 2.95。 OpenBSD チームは、スタック保護技術 Propolice をこれに追加し、デフォルトで有効化しています。そして、OpenBSD のユーザランドすべてと OpenBSD 上でコンパイルされたアプリケーションは、デフォルトでこれが使用されています。 インストール用ファイルセット comp33.tgz の一部としてインストールされます。
  • Perl 5.8.0 (+ パッチ) および OpenBSD チームによる改良。
  • Apache 1.3.27 web サーバ。 OpenBSD チームは、デフォルトでの chroot、特権分離、およびその他のセキュリティ関連の改良を 追加しています。また、mod_ssl 2.8 と DSO のサポートも組み込まれています。
  • OpenSSL 0.9.7beta3 (+ パッチ) および OpenBSD チームによる改良。
  • Groff 1.15 テキストプロセッサ。
  • Sendmail 8.12.9 メールサーバ。
  • BIND 9.2.2 DNS サーバ。 OpenBSD では、chroot による運用や他のセキュリティ関連の問題等に対する 多数の改良を実装しています。
  • テキストベース web ブラウザ Lynx 2.8.2rel.1。 また、OpenBSD チームによるパッチと HTTPS サポートが追加されています。
  • Sudo v1.6.7。これは、ユーザが 個々のコマンドを root として実行することを許可できるようにするためのものです。
  • Ncurses 5.2。
  • KAME IPv6。
  • KTH Kerberos 1.1.1。
  • Heimdal 0.4e (+ パッチ)。
  • OpenSSH 3.6。
上記のように、OpenBSD チームでは、(通常は) セキュリティやコードの品質の改良のため、 サードパーティの製品にしばしばパッチを当てることがあります。いくつかのケースでは、 ユーザにとって、操作性の違いを感ずることはありませんが、他の場合では、 操作性の違いが一部のユーザに対して影響を及ぼすことがあるかも知れません。 同じソフトウェアの異なるバージョンをを盲目的に追加する前に、これらの拡張の維持を 心がけてください。

もちろん、OpenBSD では、さらなるアプリケーションを packages や ports システムを使用して追加することができます。
