3.1 - Покупка комплекта дисков с OpenBSD

Покупка набора CD OpenBSD - лучший способ для начала. Для приобретения набора зайдите на страницу заказов OpenBSD.

Есть несколько причин для приобретения дисков с OpenBSD:

  • Продажи CD поддерживают разработку OpenBSD.
  • Разработка многоплатформенной ОС постоянно требует средств для приобретения оборудования.
  • Ваша поддержка в форме приобретения CD - это реальная помощь будущим разработкам.
  • CD содержат бинарные файлы (и исходники) для наиболее популярных поддерживаемых платформ.
  • CD являются загрузочными для нескольких платформ и могут использоваться для установки на машины без предустановленной ОС.
  • CD будут полезны для установки на голое железо, даже если вы решите поставить снимок разрабатываемой версии.
  • Установка с CD быстрее! Установка с CD экономит затраты на сетевой трафик.
  • В комплекте с OpenBSD CD всегда идут красивые наклейки. Без них ваша система неполноценна. Наклейки можно получить только вместе с компакт-дисками или при пожертвованиях оборудования.
  • На компакт-дисках OpenBSD также располагается набор полезных и популярных пакетов. CD позволяют получить полноценную среду для работы и разработки вообще без подключения к сети.

Если вы устанавливаете версию release OpenBSD, вам следует использовать официальный набор CD.

3.3 - Могу ли я скачать ISO-образы OpenBSD?

Для некоторых платформ, да!

Для большинства платформ, на которых можно загружаться с CD-ROM, есть ISO образы, доступные для загрузки, которые могут использоваться для создания установочных CD OpenBSD. Для этих платформ есть два типа ISO образов: "full install" (все файлы, необходимые для установки OpenBSD, называемые install52.iso) и для "boot only" (для установки по сети, называемые cd52.iso).

Заметим, эти ISO-образы не являются копиями официального набора компакт-дисков. Это образы только для одной платформы и они не включают в себя откомпилированных пакетов, наклеек и другой продукции, включенной в официальный набор OpenBSD.

Для начала, заметим, ISO образы - НЕ самый оптимальный способ для большинства. Это, конечно, быстро и просто - скачать загрузочный boot диск и выбрать только то, что необходимо. Однако для тех, кто не хочет или не умеет провести ряд простых операций и записать десяток файлов на CD-диск или локальный HTTP-/FTP-сервер, инсталяционные ISO образы доступны для скачивания.

OpenBSD не предоставляет для загрузки ISO-образы, используемые для изготовления официальных CD. Причина проста - мы хотим, чтобы пользователи приобретали официальные наборы CD и тем самым помогали разработке OpenBSD. Авторские права на образы официальных компакт-дисков OpenBSD принадлежат Тео де Раадту (Theo de Raadt). Тео не не разрешает распространение образов официальных CD OpenBSD. Для стимулирования приобретения наборов CD, в них также включаются бонусы (картинки, наклейки, и т.п.).

Обратите внимание, авторские права распространяются только на образы CD, сама OpenBSD свободна. Ничто не может помешать Вам самостоятельно скачать OpenBSD и создать собственные CD.

Для тех, кому необходим загрузочный CD для своей системы, bootdisk ISO images (именуемый cd52.iso) также доступен для ряда платформ и позволяет установить систему с HTTP/FTP. Эти ISO образы размером всего несколько мегабайт, и содержат только средства установки, без содержания самих комплектов установочных пакетов системы.

3.4 - Загрузка с HTTP, FTP, RSYNC или AFS

Существует множество зеркал по всему миру, предоставляющих FTP или HTTP доступ к релизам и снапшотам OpenBSD. Также доступна закачка по AFS и RSYNC. Вам всегда следует использовать ближайший к вам сервера. Перед началом закачки релиза или снимка вы можете определить ближайшее к вам зеркало и его работоспособность с помощью ping(8) и traceroute(8). Конечно, ваши компакт-диски с релизом OpenBSD всегда будут ближе, чем любое зеркало. Информация о зеркалах есть здесь:

3.5 - Проверка загрузки

В том же каталоге, в котором находятся установочные iso-образы, каждое зеркало также содержит файл с именем SHA256. Этот файл содержит так называемые контрольные суммы для различных установочных файлов. Вы можете проверить целостность установочного образа после загрузки следующей командой: 
  $ sha256 -c SHA256
Мы не занимаемся цифровой подписью своих релизов. Вывод этой команды просто поможет обнаружить случайное повреждение при копировании, а не специальное вмешательство. Если люди в черном захотят подсунуть вам свой образ, они смогут сделать это.

3.6 - Выбор железа

Достаточно важным является выбор оборудования, на котором вы собираетесь запускать OpenBSD, так как от этого зависит успех её использования.

Если вы собираетесь купить новый компьютер (не важно покупаете ли вы его по частям или уже собранным), вы скорее всего думаете о его качестве. Сегодня не так уж и просто быть уверенным в качестве покупаемого оборудования. Дешевые, ненадежные или просто плохо работающие вместе компоненты могут испортить работу OpenBSD или сделать её ненадежной. Лучшим советом в этом случае будет покупка железа от известных вам производителей, или просто тех марок, которые выбирают те, кому доверяете вы. Не стоит забывать, что высокая цена в одних случаях означает хорошее качество, а в других - лишь высокую цену.

Есть определенные критерии, с помощью которых вы можете выявить максимальную производительность вашей системы:

  • Дайте ПО выбрать железо: лучше изменить аппаратную часть системы, чем идти на компромисс с программами, которые вы "действительно хотите использовать".
  • Определите ваши узкие/слабые места: не стоить покупать последнюю модель процессора, если ваше приложение ограничено дисковым I/O. Не переплачивайте за быстрый диск, если ваша система ограничена скоростю сети. И наконец не стоит вообще покупать ничего нового, если у вас 128kbps DSL.
  • Пусть компоненты будут простыми: простое железо имеет, как правило, простые проблемы. Починка навороченных компонентов, которые не должны никогда сломаться, займет больше времени, когда они все же сломаются.
  • Используйте то железо, с которым вы знакомы, или изучайте новое, но прежде, чем собрать из него стабильную систему: Вне зависимости от технических характеристик того или иного компонента, глупо использовать что-то, с чем вы еще не знакомы и не знаете как это работает.
  • Используйте несколько дисков: вместо того чтобы покупать один большой диск, купите лучше несколько небольших дисков. Да, это может стоить больше, но распределение нагрузки между несколькими дисками уменьшает количество времени, необходимое для доступа к данным. Используя несколько дисков вы получаете более надежную систему, а также возможность более быстрого доступа к данным с RAID.
  • Разбивайте большие блоки данных: многие люди делают ошибку, проектировав свою систему таким образом, что в последствии хранится большой объем данных как один большой блок памяти. Это не самая лучшая идея. На много лучше разбить массив данных на управляемые блоки. У этого подхода много преимуществ. Во-первых, вы можете добавить память в будущем, если в этом будет необходимость, а пока обойтись небольшим её количеством. Во-вторых, в будущем цена на память будет ниже, а ее качество и скорость работы будет увеличена.
  • Избегайте дешевых сетевых карт: OpenBSD поддерживает множество дешевых сетевых карт. Их можно встретить в домашних системах, а также небольших офисах. Если же вам нужна стабильность и высокая производительность (пропускная способность), а также низкое воздейстие на сервер, вам лучше не экономить на сетевой карте. К сожалению, некоторые брендовые карты не намного лучше чем дешевых, а некоторые хорошие карты не имеют документации, на основании которой можно было бы написать драйвер. Карты с пропускной способностью в Gigabit часто работают лучше, чем адаптеры 10Mbps/100Mbps, даже когда используется на медленной скорости сети, благодаря превосходной буферизации.

3.7 - На сколько OpenBSD подходит в качестве первой системы для изучения?

Немотря на то, что OpenBSD будет работать на более старом и менее мощном компьютере чем почти любая другая современная ОС, её использование не принесет столько удовольствия на слишком старом железе. Приведенные ниже рекомендации являются ТОЛЬКО рекомендациями; OpenBSD будет очень хорошо работать и на более скромном оборудовании, чем указано здесь, все же это может разачаровать начинающего пользователя.
  • Платформа: Используйте ту платформу, с которой вы уже знакомы. Если вы решили изучить новую операционную систему, то не стоит одновременно браться за изучение новой платформы. Мы предполагаем, что вы скорее всего будете использовать i386, так как это наиболее знакомая платформа для большинства из нас.
  • Процессор: Pentium с частотой 100 МГц или выше. Да, OpenBSD будет работать и на 25MHz 80486, но вы же не хотите проделывать эксперименты с перегрузкой. Главным раздражителем, с которым вы столкнетесь на более медленном процессоре, будет время, которое потребуется для работы с SSH. Если вы хотите запустить X, вам, вероятнее всего, потребуется процессор с тактовой частотой не менее 200 МГц. X на самом деле будет неплохо работать и на более медленных машинах, но для его загрузки и запуска потребуется некоторое время.
  • 64M RAM или больше: Если вы хотите запустить X, 128M будет оптимальной отправной точкой.
  • Жесткий диск: жесткого диска объемом в 1 Gb будет вполне достаточно для развертывания таких систем, как брандмауэр, DNS сервер, или аналогичный. Если вы хотите пересобрать base-систему из исходников, вам, вероятней всего, потребуется около 4 Gb на диске; если вы хотите также пересобрать X, то 6 Gb или больше. Мы рекомендуем использовать IDE. Если у вас есть диск большего размера, то можете без проблем выделить для системы больше места - не должно случится ничего плохого, если вы выделите для системы, к примеру, 80 Gb, несмотря на то, что вам нужно всего 8.
  • Сетевая карта: Используйте PCI-карту. Если вы планируете использовать несколько сетевых карт, запишите где-нибудь их МАС-адреса перед установкой. Не стоит начинать с картами ISA, их настройка может вызовать некоторые трудности.
  • Мультизагрузка: Не стоит использовать мультизагрузку OpenBSD с другими ОС при первых опытов с установкой. Мультизагрузка является сложным процессом, который требует понимания. Это не то, с чем стоит эксперементировать во время вашей первой установки. Не забывайте, что вы можете случайно удалить всю информацию на диске, которая относится к другой ОС. Хорошей идеей будет установка OpenBSD на другой компьютер, или, по крайней мере, на отдельный диск.
  • Ноутбуки: OpenBSD работает хорошо на многих ноутбуках, однако встречаются модели ноутбуков, компоненты которых не поддерживаются в OpenBSD. Мы не советуем использовать ноутбук для знакомства с OpenBSD, хотя для опытных пользователей ноутбук может быть очень полезным инструментом.
  • Новое железо: самое последние и новые компоненты могут не поддерживаются в OpenBSD. Мы не советуем использовать самые последине новинки для ваших первых опытов с OpenBSD.
Очевидно, что чем мощнее железо, тем лучше. Помните, что некоторые популярные приложения используют столько ресурсов, сколько вы им предоставите.
